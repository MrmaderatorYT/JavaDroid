Структура програми Java

Последнее обновление: 23.05.2025

Основным строительным блоком программы на языке Java являются інструкції (statement). Каждая инструкция выполняет некоторое действие, например, вызовы методов, объявление переменных и присвоение им значений. После завершения инструкции в Java ставится точка с запятой (;). Данный знак указывает компилятору на конец инструкции. Например:

System.out.println("Hello METANIT.COM!");

Данная строка представляет вызов метода System.out.println, который выводит на консоль строку "Hello METANIT.COM!". В данном случае вызов метода является инструкцией и поэтому завершается точкой с запятой.

Крім окремих інструкцій, поширеною конструкцією є блок коду. Блок коду містить набір інструкцій, він полягає у фігурних дужках, а інструкції поміщаються між відкриває і закриває фігурними дужками:

{
	System.out.println("Hello!");
	System.out.println("Welcome to Java!");
}

У цьому блоці коду дві інструкції, які виводять на консоль певний рядок.

Виконання програми. Метод main

Java є об'єктно-орієнтованою мовою, тому всю програму можна представити як набір взаємодіючих між собою класів та об'єктів. У першому розділі під час створення першого додатка програма було визначено так:

public class Program{ 
     
    public static void main (String args[]){
         
		System.out.println("Hello METANIT.COM!");
    }
}

То есть основу нашей программы составляет класс Program. При определении класса вначале идет модификатор доступа public, который указывает, что данный класс будет доступен всем, то есть мы сможем его запустить из командной строки. Далее идет ключевое слово class, а затем название класса. После названия класса идет блок кода, в котором расположено содержимое класса.

Входной точкой в программу на языке Java является метод main, который определен в классе Program. Именно с него начинается выполнение программы. Он обязательно должен присутствовать в программе. При этом его заголовок может быть только таким:

public static void main (String args[])

При запуску програми віртуальна машина Java шукає в головному класі програми метод main з подібним заголовком і після його виявлення запускає його.

Вначале заголовка метода идет модификатор public, который указывает, что метод будет доступен извне. Слово static указывает, что метод main - статический, а слово void - что он не возвращает никакого значения. Далее в скобках у нас идут параметры метода - String args[] - это массив args, который хранит значения типа String, то есть строки. При запуске программы через этот массив мы можем передать в программу различные данные.

Після заголовка методу йде його блок, який містить набір інструкцій, що виконуються.

Скорочені файли коду та методи main c JDK 25

С выходом JDK 21 появилась экспериментальная функция, которая позволяет упростить программу на Java. В JDK 25 эта возможность перестает быть экспериментальной и стала полноценной для применения. Суть этой возможности: теперь можно писать код без объявления класса с сокращением определения кода main. Например, также определим файл Program.java, в котором будет следующий код:

void main() {
    System.out.println("Hello METANIT.COM!");
}

Як можна побачити, цей код значно коротший за попередній, що спрощує написання програм. Версії JDK 21, 22, 23 і 24 підтримували таке визначення методу main як експериментальний, а для компіляції програми треба було вказувати додаткові опції. Наприклад, для компіляції за допомогою JDK 22 застосовувалась команда:

javac --release 22 --enable-preview Program.java

Здесь 22 указывает на версию JDK, которую соответственно можно заменить на 21, 23, 24. И для запуска программы - также надо было указать дополнительную опцию --enable-preview:

java --enable-preview Program

Починаючи з версії JDK 25, жодних додаткових опцій ні при компіляції, ні при запуску програми вказувати не потрібно:

eugene@Eugene:/workspace/java$ javac Program.java
eugene@Eugene:/workspace/java$ java Program
Hello METANIT.COM!
eugene@Eugene:/workspace/java$

Також можна явно оформити метод main у клас:

class Program{
    void main() {
        System.out.println("Hello METANIT.COM");
    }
}

Коментарі

Код програми може містити коментарі. Коментарі дозволяють зрозуміти зміст програми, що роблять ті чи інші її частини. При компіляції коментарі ігноруються і не мають жодного впливу на роботу додатка та на його розмір.

В Java есть два типа комментариев: однострочный и многострочный. Однострочный комментарий размещается на одной строке после двойного слеша //. А многострочный комментарий заключается между символами /* Текст коментаря */. Он может размещаться на нескольких строках. Например:

/*
	многострочный комментарий
	Объявление нового класса,
	который содержит код программы
*/
public class Program{ 	// начало объявления класса Program
     
	// определение метода main
	public static void main (String args[]){ 	// объявление нового метода
         
		 System.out.println("Hello METANIT.COM!"); 	// вывод строки на консоль
    } 	// конец объявления нового метода
} // конец объявления класса Program
