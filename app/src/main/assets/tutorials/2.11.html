Структура програми Java

Останнє оновлення: 23.05.2025

Основным строительным блоком програми на мові Java є інструкції (statement). Каждая инструкция виконує некоторое действие, наприклад, вызовы методів, оголошення змінних и присвоение им значений. Після завершення инструкции в Java ставится крапка с комою (;). Данный знак вказує компилятору на конец инструкции. Наприклад:


System.out.println("Hello !");

Данная рядок представляє вызов методу System.out.println, який виводить на консоль рядок "Hello !". У даному випадку вызов методу є инструкцией и поцему завершуєся точкой с комою.

Крім окремих інструкцій, поширеною конструкцією є блок коду. Блок коду містить набір інструкцій, він полягає у фігурних дужках, а інструкції поміщаються між відкриває і закриває фігурними дужками:


{
	System.out.println("Hello!");
	System.out.println("Welcome to Java!");
}

У цьому блоці коду дві інструкції, які виводять на консоль певний рядок.

Виконання програми. Метод main

Java є об'єктно-орієнтованою мовою, тому всю програму можна представити як набір взаємодіючих між собою класів та об'єктів. У першому розділі під час створення першого додатка програма було визначено так:


public class Program{ 
     
    public static void main (String args[]){
         
		System.out.println("Hello !");
    }
}

То есть основу нашей програми составляет клас Program. При визначенийии класа вначале идет модификатор доступа public, який вказує, что данный клас буде доступен всем, то есть мы сможемо его запустить из командного рядка. Далїї идет ключевое слово class, а затем название класа. Після названия класа идет блок коду, в котором расположено вміст класа.

Входной точкой в програму на мові Java є метод main, який визначений в класе Program. Именно с него начинается виконання програми. Он обязательно повинен присутствовать в програме. При цем его заголовок может быть лише таким:

public static void main (String args[])

При запуску програми віртуальна машина Java шукає в головному класі програми метод main з підібним заголовком і після його виявлення запускує його.

Вначале заголовка методу идет модификатор public, який вказує, что метод буде доступен зовні. Слово static вказує, что метод main - статический, а слово void - что он не повертає ниякого значення. Далїї в скобках у нас идут параметри методу - String args[] - це масив args, який хранит значення типу String, то є рядки. При запуску програми через цей масив мы можемо передать в програму различные дані.

Після заголовка методу йде його блок, який містить набір інструкцій, що виконуються.

Скорочені файли коду та методи main c JDK 25

С выходом JDK 21 появилась экспериментальная функція, которая дозволяє упростить програму на Java. В JDK 25 эта возможнасть перестает быть экспериментальной и стала полноценной для применения. Суть цей возможнасти: теперь можна писать код без оголошення класа с сокращением визначенийия коду main. Наприклад, також визначимо файл Program.java, в котором буде следующий код:


void main() {
    System.out.println("Hello !");
}

Як можна побачити, цей код значно коротший за попередній, що спрощує написання програм. Версії JDK 21, 22, 23 і 24 підтримували таке визначення методу main як експериментальний, а для компіляції програми треба було вказувати додаткові опції. Наприклад, для компіляції за допомогою JDK 22 застосовувалась команда:

javac --release 22 --enable-preview Program.java

Здесь 22 вказує на версию JDK, яку соответственно можна заменить на 21, 23, 24. И для запуску програми - також треба буво указать дополнительную опцию --enable-preview:

java --enable-preview Program

Починаючи з версії JDK 25, жодних додаткових опцій ні при компіляції, ні при запуску програми вказувати не потрібно:


eugene@Eugene:/workspace/java$ javac Program.java
eugene@Eugene:/workspace/java$ java Program
Hello !
eugene@Eugene:/workspace/java$

Також можна явно оформити метод main у клас:


class Program{
    void main() {
        System.out.println("Hello ");
    }
}

Коментарі

Код програми може містити коментарі. Коментарі дозволяють зрозуміти зміст програми, що роблять ті чи інші її частини. При компіляції коментарі ігноруються і не мають жодного впливу на роботу додатка та на його розмір.

В Java есть два типу комментариев: однострочный и многострочный. Однострочный комментарий размещается на одной рядківе після двойного слеша //. А многострочный комментарий заключается між символами /* Текст коментаря */. Он может размещаться на нескольких рядокх. Наприклад:


/*
	многострочный комментарий
	Оголошення нового класа,
	який містить код програми
*/
public class Program{ 	// начало оголошення класа Program
     
	// визначенийие методу main
	public static void main (String args[]){ 	// оголошення нового методу
         
		 System.out.println("Hello !"); 	// вивід рядки на консоль
    } 	// конец оголошення нового методу
} // конец оголошення класа Program
