Структура програми Java

Останнє оновлення: 23.05.2025

Основным строительным блоком программы на мові Java є інструкції (statement). Каждая инструкция выполняет некоторое действие, наприклад, вызовы методів, оголошення переменных и присвоение им значений. После завершения инструкции в Java ставится крапка с комою (;). Данный знак указывает компилятору на конец инструкции. Например:


System.out.println("Hello !");

Данная строка представляє вызов метода System.out.println, який выводит на консоль строку "Hello !". В данном случае вызов метода є инструкцией и поцему завершуєся точкой с комою.

Крім окремих інструкцій, поширеною конструкцією є блок коду. Блок коду містить набір інструкцій, він полягає у фігурних дужках, а інструкції поміщаються між відкриває і закриває фігурними дужками:


{
	System.out.println("Hello!");
	System.out.println("Welcome to Java!");
}

У цьому блоці коду дві інструкції, які виводять на консоль певний рядок.

Виконання програми. Метод main

Java є об'єктно-орієнтованою мовою, тому всю програму можна представити як набір взаємодіючих між собою класів та об'єктів. У першому розділі під час створення першого додатка програма було визначено так:


public class Program{ 
     
    public static void main (String args[]){
         
		System.out.println("Hello !");
    }
}

То есть основу нашей программы составляет класс Program. При определении класса вначале идет модификатор доступа public, який указывает, что данный класс буде доступен всем, то есть мы сможемо его запустить из командного рядка. Далїї идет ключевое слово class, а затем название класса. После названия класса идет блок кода, в котором расположено содержимое класса.

Входной точкой в программу на мові Java є метод main, який определен в классе Program. Именно с него начинается выполнение программы. Он обязательно повинен присутствовать в программе. При цем его заголовок может быть лише таким:

public static void main (String args[])

При запуску програми віртуальна машина Java шукає в головному класі програми метод main з подібним заголовком і після його виявлення запускає його.

Вначале заголовка метода идет модификатор public, який указывает, что метод буде доступен зовні. Слово static указывает, что метод main - статический, а слово void - что он не повертає ниякого значения. Далїї в скобках у нас идут параметры метода - String args[] - це массив args, який хранит значения типу String, то есть строки. При запуске программы через цей массив мы можемо передать в программу различные дані.

Після заголовка методу йде його блок, який містить набір інструкцій, що виконуються.

Скорочені файли коду та методи main c JDK 25

С выходом JDK 21 появилась экспериментальная функция, которая дозволяє упростить программу на Java. В JDK 25 эта возможнасть перестает быть экспериментальной и стала полноценной для применения. Суть цей возможнасти: теперь можна писать код без оголошення класса с сокращением определения кода main. Наприклад, також определим файл Program.java, в котором буде следующий код:


void main() {
    System.out.println("Hello !");
}

Як можна побачити, цей код значно коротший за попередній, що спрощує написання програм. Версії JDK 21, 22, 23 і 24 підтримували таке визначення методу main як експериментальний, а для компіляції програми треба було вказувати додаткові опції. Наприклад, для компіляції за допомогою JDK 22 застосовувалась команда:

javac --release 22 --enable-preview Program.java

Здесь 22 указывает на версию JDK, яку соответственно можна заменить на 21, 23, 24. И для запуска программы - також треба буво указать дополнительную опцию --enable-preview:

java --enable-preview Program

Починаючи з версії JDK 25, жодних додаткових опцій ні при компіляції, ні при запуску програми вказувати не потрібно:


eugene@Eugene:/workspace/java$ javac Program.java
eugene@Eugene:/workspace/java$ java Program
Hello !
eugene@Eugene:/workspace/java$

Також можна явно оформити метод main у клас:


class Program{
    void main() {
        System.out.println("Hello ");
    }
}

Коментарі

Код програми може містити коментарі. Коментарі дозволяють зрозуміти зміст програми, що роблять ті чи інші її частини. При компіляції коментарі ігноруються і не мають жодного впливу на роботу додатка та на його розмір.

В Java есть два типу комментариев: однострочный и многострочный. Однострочный комментарий размещается на одной строке после двойного слеша //. А многострочный комментарий заключается между символами /* Текст коментаря */. Он может размещаться на нескольких строках. Например:


/*
	многострочный комментарий
	Оголошення нового класса,
	який містить код программы
*/
public class Program{ 	// начало оголошення класса Program
     
	// определение метода main
	public static void main (String args[]){ 	// оголошення нового метода
         
		 System.out.println("Hello !"); 	// вывод строки на консоль
    } 	// конец оголошення нового метода
} // конец оголошення класса Program
