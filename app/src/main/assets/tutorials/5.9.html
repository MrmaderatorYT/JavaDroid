Інтерфейси SortedMap и NavigableMap. Клас TreeMap

Останнє оновлення: 13.10.2025

Інтерфейс Map, який застосовується в мові Java для створенняя словарей, не предусматривает нияких возможнастей для сортировки. Но Java також надає ряд дополнительных інтерфейсов: SortedMap и NavigableMap

SortedMap

Інтерфейс SortedMap расширяет Map и дозволяє обеспечить полное упорядочение своих ключей. По умолчанию елементы упорядочиваются в соответствии с естественным порядком ключей (за зростанням) або за допомогою компаратора. Цет порядок, наприклад, отражается при переборе колекції.

SortedMap додає ряд методів:

  • K firstKey(): повертає ключ первого елемента колекції

  • K lastKey(): повертає ключ післяднего елемента колекції

  • SortedMap<K, V> headMap(K end): повертає об'єкт SortedMap, який містить всі елементи оригинального SortedMap вплоть до елемента с ключом end

  • SortedMap<K, V> tailMap(K start): повертає об'єкт SortedMap, який містить всі елементи оригинального SortedMap, начиная с елемента с ключом start

  • SortedMap<K, V> subMap(K start, K end): повертає об'єкт SortedMap, которые містить всі елементи оригинального SortedMap вплоть от елемента с ключом start до елемента с ключом end

NavigableMap

Інтерфейс NavigableMap расширяет інтерфейс SortedMap и забезпечує возможнасть получения елементов колекції относительно других елементов (условно говоря, навигацию). Его основные методи:

  • Map.Entry<K, V> ceilingEntry(K key): повертає елемент с наименьшим ключом k, який больше або равен ключу key (k >=key). Если такого ключа нет, то повертаєся null.

  • Map.Entry<K, V> floorEntry(K key): повертає елемент с наибольшим ключом k, який меньше або равен ключу key (k <=key). Если такого ключа нет, то повертаєся null.

  • Map.Entry<K, V> higherEntry(K key): повертає елемент с наименьшим ключом k, який больше ключа key (k >key). Если такого ключа нет, то повертаєся null.

  • Map.Entry<K, V> lowerEntry(K key): повертає елемент с наибольшим ключом k, який меньше ключа key (k <key). Если такого ключа нет, то повертаєся null.

  • Map.Entry<K, V> firstEntry(): повертає первый елемент колекції

  • Map.Entry<K, V> lastEntry(): повертає післядний елемент колекції

  • Map.Entry<K, V> pollFirstEntry(): повертає и одновременно видаляє первый елемент из колекції

  • Map.Entry<K, V> pollLastEntry(): повертає и одновременно видаляє післядний елемент из колекції

  • K ceilingKey(K key): повертає наименьший ключ k, який больше або равен ключу key (k >=key). Если такого ключа нет, то повертаєся null.

  • K floorKey(K key): повертає наибольший ключ k, який меньше або равен ключу key (k <=key). Если такого ключа нет, то повертаєся null.

  • K lowerKey(K key): повертає наибольший ключ k, який меньше ключа key (k <key). Если такого ключа нет, то повертаєся null.

  • K higherKey(K key): повертає наименьший ключ k, який больше ключа key (k >key). Если такого ключа нет, то повертаєся null.

  • NavigableSet<K> descendingKeySet(): повертає об'єкт NavigableSet, який містить все ключи колекції в обратном порядке

  • NavigableMap<K, V> descendingMap(): повертає колекцію NavigableMap, которое містить всі елементи в обратном порядке

  • NavigableSet<K> navigableKeySet(): повертає об'єкт NavigableSet, який містить все ключи колекції

  • NavigableMap<K, V> headMap(K upperBound, boolean incl): повертає колекцію NavigableMap, которое містить все елементы оригинальной колекції NavigableMap вплоть от елемента с ключом upperBound. Параметр incl при значении true вказує, что елемент с ключом upperBound також включается в выходной набор.

  • NavigableMap<K, V> tailMap(K lowerBound, boolean incl): повертає колекцію NavigableMap, которое містить все елементы оригинальной колекції NavigableMap, начиная с елемента с ключом lowerBound. Параметр incl при значении true вказує, что елемент с ключом lowerBound також включается в выходной набор.

  • NavigableMap<K, V> subMap(K lowerBound, boolean lowIncl, K upperBound, boolean highIncl): повертає колекцію NavigableMap, которое містить все елементы оригинальной колекції NavigableMap от елемента с ключом lowerBound до елемента с ключом upperBound. Параметри lowIncl и highIncl при значении true включают в выходной набор елементы с ключами lowerBound и upperBound соответственно.

TreeMap

Встроенной реализацией выше упомянутых інтерфейсов є клас TreeMap<K, V>. Он представляє колекцію в виде дерева, где всі елементи отсортированы. Цет клас найдеся от класа AbstractMap и реалізує інтерфейс NavigableMap, а следовательно, також и інтерфейсы SortedMap и Map. Тому для управления елементами в дереве можна використовувати методи інтерфейса Map. Но в отличие от колекції HashMap в TreeMap все об'єкты автоматически ортуються за зростанням их ключей.

Клас TreeMap имїїт следующие конструктори:

  • TreeMap(): створює пустое колекцію в виде дерева

  • TreeMap(Map<? extends K, ? extends V> map): створює дерево, в которое додає всі елементи из колекції map

  • TreeMap(SortedMap<K, ? extends V> smap): створює дерево, в которое додає всі елементи из колекції smap

  • TreeMap(Comparator<? super K> comparator): створює пустое дерево, где все додаємоые елементы впіслядствии будуть отсортированы компаратором.

Елементы дерева повинні реализовать інтерфейс Comparable() для сортировки, або необхідно предоставлять компаратор через післяднюю версию конструктора. При цем порядок, піддерживаемый в TreeMap, (независимо от того, предоставлен ли явный компаратор), повинен быть согласован с equals(), чтобы эта сортированная колекція правильно реализовала інтерфейс Map. Примеры створенняя дерева:


import java.util.Map;
import java.util.TreeMap;
 
public class Program{
      
    public static void main(String[] args) {
          
        // пустое дерево
        var treeMap = new TreeMap<String,String>();
        System.out.println(treeMap);     // {}

        // дерево на основе другого словника
        var dict = new TreeMap<String,String>(
                                Map.ofEntries( 
                                    Map.entry("+79876543210", "Tom"),
                                    Map.entry("+79666543210", "Bob"),
                                    Map.entry("+79176543210", "Sam"),
                                    Map.entry("+79276543210", "Alice")
                                ));
        System.out.println(dict);  // {blue=синий, green=зеленый, red=красный, yellow=желтый}
    }
}

Зверніть увагу на вивід второго дерева-словника - dict:

{blue=синий, green=зеленый, red=красный, yellow=желтый}

Вне зависимости в яком порядке об'єкты буви добавлены в дерево, они упорядочиваются за зростанням. Так, у даному випадку у нас змінна dict представляє условный словник, а TreeMap дозволяє упорядочить дані словника за зростанням.

Додавання елементов:


import java.util.Map;
import java.util.TreeMap;
 
public class Program{
      
    public static void main(String[] args) {
          
        // пустое дерево
        var dict = new TreeMap<String,String>();
 
        // додаємо в него елементы
        dict.put("red", "красный");
        dict.put("green", "зеленый");
        dict.put("blue", "синий");
        System.out.println(dict);   // {blue=синий, green=зеленый, red=красный}
 
        // додаємо другой словник с данными
        var map = Map.ofEntries( 
                Map.entry("yellow", "желтый"),
                Map.entry("black", "черный") 
        );
        dict.putAll(map);

        System.out.println(dict);   // {black=черный, blue=синий, green=зеленый, red=красный, yellow=желтый}
    }
}

Отримання елементов:


import java.util.Map;
import java.util.Set;
import java.util.Collection;
import java.util.TreeMap;
 
public class Program{
      
    public static void main(String[] args) {
          
        // порожній словник
        var dict = new TreeMap<String,String>(
            Map.ofEntries( 
                Map.entry("red", "красный"),
                Map.entry("green", "зеленый"),
                Map.entry("yellow", "желтый"),
                Map.entry("blue", "синий"),
                Map.entry("black", "черный") 
            )
        );

        // перебор елементов
       for(var word : dict.entrySet()){
         
           System.out.printf("%s: %s \n", word.getKey(), word.getValue());
       }

        // получим об'єкт по ключу "red"
        String red = dict.get("red");
        System.out.println("`red` переводится як `" + red + "`");  // `red` переводится як `красный`

        // получим весь набор ключей
        Set<String> keys = dict.keySet();
        System.out.println(keys);       // [black, blue, green, red, yellow]

        // отримати набор всех значений
        Collection<String> values = dict.values();
        System.out.println(values);     // [черный, синий, зеленый, красный, желтый]
            
        // отримуємо все об'єкты, которые стоят начиная с об'єкта с ключом "green"
        Map<String, String> afterGreen = dict.tailMap("green");
        System.out.println(afterGreen);     // {green=зеленый, red=красный, yellow=желтый}
            
        // отримуємо все об'єкты, которые стоят до об'єкта с ключом "green"
        Map<String, String> beforeGreen = dict.headMap("green");
        System.out.println(beforeGreen);    // {black=черный, blue=синий}
         
        // получим післядний елемент дерева
        Map.Entry<String, String> lastItem = dict.lastEntry();
        System.out.println(lastItem);   // yellow=желтый
    }
}

Зміна и видалення елементов:


import java.util.Map;
import java.util.TreeMap;
 
public class Program{
      
    public static void main(String[] args) {
          
        // порожній словник
        var dict = new TreeMap<String,String>(
            Map.ofEntries( 
                Map.entry("red", "красный"),
                Map.entry("green", "зеленый"),
                Map.entry("yellow", "желтый"),
                Map.entry("blue", "синий"),
                Map.entry("black", "черный") 
            )
        );

        // меняем по ключу "red" значення на "червонный"
        dict.replace("red", "червонный");
        System.out.println(dict);   // {black=черный, blue=синий, green=зеленый, red=червонный, yellow=желтый}
 
        // меняем по ключу "red" значення на "алый", лише якщо значення "красный"
        dict.replace("red", "алый", "красный");
 
        // видаляємо yellow
        dict.remove("yellow");
        System.out.println(dict);   // {black=черный, blue=синий, green=зеленый, red=червонный}

        // отримуємо с видаленням первый елемент
        Map.Entry<String, String> firstWord = dict.pollFirstEntry();
        System.out.println(firstWord);      // black=черный

        // отримуємо с видаленням післядний елемент
        Map.Entry<String, String> lastWord = dict.pollLastEntry();
        System.out.println(lastWord);       // red=червонный
 
        // видаляємо всі елементи
        dict.clear();
        System.out.println(dict);   // {}
    }
}
