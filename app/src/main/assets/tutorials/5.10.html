Ітератори. Iterator та Iterable

Останнє оновлення: 13.10.2025

Одним из ключевых інтерфейсов коллекций в мові Java є інтерфейс Iterator. Он дозволяє проходить по элементам коллекции.

Интерфейс Iterator имїїт следующїї определение:


public interface Iterator <E>{
    
   boolean hasNext();
   E next();
   default void remove();
   default void forEachRemaining(Consumer<? super E> action);
}

Реализация інтерфейса предполагает, что за допомогою вызова метода next() можна отримати следующий элемент. С помощью метода hasNext() можна узнать, есть ли следующий элемент, и не достигнут ли конец коллекции. И якщо элементы ще имеются, то hasNext() вернет значение true. Метод hasNext() йде вызывать перед методом next(), оскільки при достижении конца коллекции метод next() выбрасывает исключение NoSuchElementException. И метод remove() видаляє текущий элемент, який був получен последним вызовом next().

Наприклад, используем итератор для перебора коллекции ArrayList:


import java.util.Collection;
import java.util.ArrayList;
import java.util.Iterator;
 
public class Program{
      
    public static void main(String[] args) {
          
        Collection<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
         
        // получаем интератор коллекции people
        Iterator<String> iter = people.iterator();
        // используя итератор, перебираем коллекцию
        while(iter.hasNext()){
         
            System.out.println(iter.next());
        }
    }
}

Консольный вывод:


Tom
Bob
Sam

Конечно, для перебора коллекции гораздо проще використовувати стандартный цикл for each:


import java.util.Collection;
import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        Collection<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
         
        for(var p : people){
            System.out.println(p);
        }
    }
}

Однак "под капотом" компилятор преобразует цикл for-each в цикл с итератором.

Встроенные реализации итераторов

Интерфейс Iterator надає ограниченный функционал. Но Java имїїт встроенные типы, которые расширяют его функционал и призначенийы для перебора определенных типов коллекций. Наприклад, інтерфейс ListIterator використовується классами, которые реализуют інтерфейс List и представляють списки, то есть классами LinkedList, ArrayList и др.

Интерфейс ListIterator расширяет інтерфейс Iterator и определяет ряд дополнительных методів:

  • void add(E obj): вставляет об'єкт obj перед элементом, який повинен быть возвращен наступним вызовом next()

  • boolean hasNext(): повертає true, якщо в коллекции имїїтся следующий элемент, иначе повертає false

  • boolean hasPrevious(): повертає true, якщо в коллекции имїїтся предыдущий элемент, иначе повертає false

  • E next(): повертає текущий элемент и переходит к следующему, якщо такого нет, то генеруєся исключение NoSuchElementException

  • E previous(): повертає текущий элемент и переходит к предыдущему, якщо такого нет, то генеруєся исключение NoSuchElementException

  • int nextIndex(): повертає индекс следующего элемента. Если такого нет, то повертаєся размер списка

  • int previousIndex(): повертає индекс предыдущего элемента. Если такого нет, то повертаєся число -1

  • void remove(): видаляє текущий элемент из списка. Таким чином, цей метод повинен быть вызван после методів next() або previous(), иначе буде сгенерировано исключение IlligalStateException

  • void set(E obj): присваивает текущему элементу, выбранному вызовом методів next() або previous(), ссылку на об'єкт obj

Используем ListIterator:


import java.util.ListIterator;
import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam"); 
         
        ListIterator<String> listIter = people.listIterator();
         
        while(listIter.hasNext()){
         
            System.out.println(listIter.next());
        }
        // сейчас текущий элемент - Sam
        // изменим значение цього элемента
        listIter.set("Sammy");
        // пройдемся по элементам в обратном порядке
        while(listIter.hasPrevious()){
         
            System.out.println(listIter.previous());
        }  
    }
}

Консольный вывод программы:


Tom
Bob
Sam
Sammy
Bob
Tom

Интерфейс Iterable

Цикл for-each работает с любым об'єктом, який реализует інтерфейс Iterable и реализует его метод iterator():


public interface Iterable<E>
{
    Iterator<E> iterator();
    ....
}

В частности, цей метод повертає итератор - то есть об'єкт, реализующий інтерфейс Iterator. И за допомогою цього итератора буде осуществляться перебор об'єкта класса, которые реализует інтерфейс Iterable.

Интерфейс Collection, а через него и все остальные коллекции, як раз расширяет інтерфейс Iterable и найде метод Iterator<E> iterator().:


public interface Collection<E>
{
    Iterator<E> iterator();
    ................

Тому вы можете використовувати цикл "for each" с любой коллекцией в Java Collections Framework.

Кастомная реализация итераторов

Хотя итераторы и інтерфейс Iterable призначенийы прежде всего для работы с коллекциями, но фактически мы можемо их використовувати с любыми классами, рассматривая класс як набор некоторых об'єктов. Наприклад, реализуем інтерфейсы Iterable и Iterator для перебора об'єкта кастомного класса:


import java.util.Iterator;
 
public class Program{
      
    public static void main(String[] args) {
          
        Person tom = new Person(22, "Tom", 41);
        for(var prop : tom){
            System.out.println(prop);
        }
    }
}

class PersonIterator implements Iterator<String>{

    int currentProp = 0;
    Person person;

    PersonIterator(Person person){ this.person = person; }

    public boolean hasNext(){ return currentProp < 3; }

    public String next(){
        return switch(currentProp++){

            case 0 -> String.valueOf(person.getId());
            case 1 -> person.getName();
            case 2 -> String.valueOf(person.getAge());
            default -> null;
        };
    }
}

class Person implements Iterable{

    private int id;
    private String name;
    private int age;

    int getId() { return id;}
    String getName() { return name;}
    int getAge() { return age;}

    private PersonIterator iter = null;

    Person(int id, String name, int age){

        this.id = id;
        this.age = age;
        this.name = name;
    }

    public PersonIterator iterator(){
        if(iter == null){
            iter = new PersonIterator(this);
        }
        return iter;
    }
}

В данном случае демонстрируется, як сделать собственный класс итерируемым, то есть дать возможнасть перебирать его в цикле "for-each". И для начала определяем класс, об'єкт которого буде перебираться - класс Person, який описує человека:


class Person implements Iterable{
    // ... поля и конструктор ...

    public PersonIterator iterator(){
        if(iter == null){
            iter = new PersonIterator(this);
        }
        return iter;
    }
}

Основные моменты:

  • implements Iterable: реализуя цей інтерфейс, класс Person обязуется предоставить метод iterator(), який неявно вызывается циклом "for-each".

  • public PersonIterator iterator(): цей метод повинен вернуть об'єкт, який, в свою очередь, реализует інтерфейс Iterator. В данном случае он повертає экземпляр нашего кастомного итератора PersonIterator.

Класс PersonIterator містить всю логику перебора свойств об'єкта Person:


class PersonIterator implements Iterator<String>{
    // ... поля и конструктор ...

    public boolean hasNext(){ return currentProp < 3; }

    public String next(){
        return switch(currentProp++){
            // ...
        };
    }
}

Основные моменты:

  • implements Iterator<String>: применяем інтерфейс, який буде перебирать значения типу String

  • boolean hasNext(): цей метод вызывается перед каждой итерацией цикла "for-each". Он повинен вернуть true, якщо есть следующий элемент для перебора, и false, якщо элементы закончились. Здесь логика проста: поскольку у Person три свойства, метод повертає true, поки счетчик currentProp меньше 3.

  • String next(): цей метод повертає следующий элемент. Он вызывается, якщо hasNext() вернул true. С помощью оператора switch он последовательно повертає значения полей id, name и age в виде строки (String), увеличивая счетчик currentProp на единицу после каждого вызова.

Таким чином, код створює специальную конструкцию в виде PersonIterator о том, як пошагово перебрать свойства об'єкта Person, и связывает эту конструкцию с самим классом Person через інтерфейс Iterable

И далїї в методе main мы можемо создать об'єкт класса Person и перебрать его свойства:


public static void main(String[] args) {
      
    Person tom = new Person(22, "Tom", 41);
    for(var prop : tom){
        System.out.println(prop);
    }
}

Результат виконання програми:


22
Tom
41

Метод forEach

Вместо цикла "for each" для перебора коллекции можна вызвать методы Collection.forEach або Iterator.forEachRemaining с лямбда-выражением, которое использует элемент. Лямбда-выражение застосовується ко всем элементам коллекции або к оставшимся элементам, которые может обойти итератор:


coll.forEach(element -> действия с element);
iter.forEachRemaining(element -> действия с element);

Например:


import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam"); 
         
        // p представляє каждый об'єкт в people
        people.forEach(p -> System.out.println(p) );
    }
}
