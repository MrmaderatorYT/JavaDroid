Ітератори. Iterator та Iterable

Останнє оновлення: 13.10.2025

Одним из ключевых інтерфейсов колекцій в мові Java є інтерфейс Iterator. Он дозволяє проходить по елементам колекції.

Інтерфейс Iterator имїїт следующїї визначенийие:


public interface Iterator <E>{
    
   boolean hasNext();
   E next();
   default void remove();
   default void forEachRemaining(Consumer<? super E> action);
}

Реалізація інтерфейса предполагает, что за допомогою вызова методу next() можна отримати следующий елемент. За допомогою методу hasNext() можна узнать, чи є следующий елемент, и не достигнут ли конец колекції. И якщо елементы ще имеются, то hasNext() вернет значення true. Метод hasNext() йде вызывать перед методом next(), оскільки при достижении конца колекції метод next() выбрасывает виняток NoSuchElementException. И метод remove() видаляє текущий елемент, який був получен післядним вызовом next().

Наприклад, використовуємо ітератор для перебора колекції ArrayList:


import java.util.Collection;
import java.util.ArrayList;
import java.util.Iterator;
 
public class Program{
      
    public static void main(String[] args) {
          
        Collection<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
         
        // отримуємо интератор колекції people
        Iterator<String> iter = people.iterator();
        // використовуючи ітератор, перебираем колекцію
        while(iter.hasNext()){
         
            System.out.println(iter.next());
        }
    }
}

Консольный вивід:


Tom
Bob
Sam

Конечно, для перебора колекції гораздо проще використовувати стандартный цикл for each:


import java.util.Collection;
import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        Collection<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
         
        for(var p : people){
            System.out.println(p);
        }
    }
}

Однак "під капотом" компилятор преобразует цикл for-each в цикл с ітератором.

Встроенные реалізації ітераторов

Інтерфейс Iterator надає ограниченный функционал. Но Java имїїт встроенные типи, которые расширяют его функционал и призначенийы для перебора визначенийных типів колекцій. Наприклад, інтерфейс ListIterator використовується класами, которые реалізують інтерфейс List и представляютьь списки, то есть класами LinkedList, ArrayList и др.

Інтерфейс ListIterator расширяет інтерфейс Iterator и визначає ряд дополнительных методів:

  • void add(E obj): вставляет об'єкт obj перед елементом, який повинен быть возвращен наступним вызовом next()

  • boolean hasNext(): повертає true, якщо в колекції имїїтся следующий елемент, иначе повертає false

  • boolean hasPrevious(): повертає true, якщо в колекції имїїтся предыдущий елемент, иначе повертає false

  • E next(): повертає текущий елемент и переходит к следующему, якщо такого нет, то генеруєся виняток NoSuchElementException

  • E previous(): повертає текущий елемент и переходит к предыдущему, якщо такого нет, то генеруєся виняток NoSuchElementException

  • int nextIndex(): повертає индекс следующего елемента. Если такого нет, то повертаєся размер списку

  • int previousIndex(): повертає индекс предыдущего елемента. Если такого нет, то повертаєся число -1

  • void remove(): видаляє текущий елемент из списку. Таким чином, цей метод повинен быть вызван після методів next() або previous(), иначе буде сгенерировано виняток IlligalStateException

  • void set(E obj): присваивает текущему елементу, выбранному вызовом методів next() або previous(), ссылку на об'єкт obj

Використовуємо ListIterator:


import java.util.ListIterator;
import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam"); 
         
        ListIterator<String> listIter = people.listIterator();
         
        while(listIter.hasNext()){
         
            System.out.println(listIter.next());
        }
        // сейчас текущий елемент - Sam
        // изменим значення цього елемента
        listIter.set("Sammy");
        // пройдемся по елементам в обратном порядке
        while(listIter.hasPrevious()){
         
            System.out.println(listIter.previous());
        }  
    }
}

Консольный вивід програми:


Tom
Bob
Sam
Sammy
Bob
Tom

Інтерфейс Iterable

Цикл for-each работает с любым об'єктом, який реалізує інтерфейс Iterable и реалізує его метод iterator():


public interface Iterable<E>
{
    Iterator<E> iterator();
    ....
}

Зокрема, цей метод повертає ітератор - то есть об'єкт, реализующий інтерфейс Iterator. И за допомогою цього ітератора буде осуществляться перебор об'єкта класа, которые реалізує інтерфейс Iterable.

Інтерфейс Collection, а через него и все остальные колекції, як раз расширяет інтерфейс Iterable и найде метод Iterator<E> iterator().:


public interface Collection<E>
{
    Iterator<E> iterator();
    ................

Тому вы можете використовувати цикл "for each" с любой коллекцией в Java Collections Framework.

Кастомная реалізація ітераторов

Хотя ітераторы и інтерфейс Iterable призначенийы прежде всего для работы с колекціями, но фактически мы можемо их використовувати с любыми класами, рассматривая клас як набор некоторых об'єктов. Наприклад, реализуем інтерфейсы Iterable и Iterator для перебора об'єкта кастомного класа:


import java.util.Iterator;
 
public class Program{
      
    public static void main(String[] args) {
          
        Person tom = new Person(22, "Tom", 41);
        for(var prop : tom){
            System.out.println(prop);
        }
    }
}

class PersonIterator implements Iterator<String>{

    int currentProp = 0;
    Person person;

    PersonIterator(Person person){ this.person = person; }

    public boolean hasNext(){ return currentProp < 3; }

    public String next(){
        return switch(currentProp++){

            case 0 -> String.valueOf(person.getId());
            case 1 -> person.getName();
            case 2 -> String.valueOf(person.getAge());
            default -> null;
        };
    }
}

class Person implements Iterable{

    private int id;
    private String name;
    private int age;

    int getId() { return id;}
    String getName() { return name;}
    int getAge() { return age;}

    private PersonIterator iter = null;

    Person(int id, String name, int age){

        this.id = id;
        this.age = age;
        this.name = name;
    }

    public PersonIterator iterator(){
        if(iter == null){
            iter = new PersonIterator(this);
        }
        return iter;
    }
}

У даному випадку демонстрируется, як сделать собственный клас итерируемым, то есть дать возможнасть перебирать его в цикле "for-each". И для начала определяем клас, об'єкт которого буде перебираться - клас Person, який описує человека:


class Person implements Iterable{
    // ... поля и конструктор ...

    public PersonIterator iterator(){
        if(iter == null){
            iter = new PersonIterator(this);
        }
        return iter;
    }
}

Основные моменты:

  • implements Iterable: реализуя цей інтерфейс, клас Person обязуется предоставить метод iterator(), який неявно вызывается циклом "for-each".

  • public PersonIterator iterator(): цей метод повинен вернуть об'єкт, який, в свою очередь, реалізує інтерфейс Iterator. У даному випадку он повертає экземпляр нашего кастомного ітератора PersonIterator.

Клас PersonIterator містить всю логику перебора свойств об'єкта Person:


class PersonIterator implements Iterator<String>{
    // ... поля и конструктор ...

    public boolean hasNext(){ return currentProp < 3; }

    public String next(){
        return switch(currentProp++){
            // ...
        };
    }
}

Основные моменты:

  • implements Iterator<String>: застосовуємо інтерфейс, який буде перебирать значення типу String

  • boolean hasNext(): цей метод вызывается перед каждой итерацией цикла "for-each". Он повинен вернуть true, якщо есть следующий елемент для перебора, и false, якщо елементы закончились. Здесь логика проста: поскольку у Person три свойства, метод повертає true, поки счетчик currentProp меньше 3.

  • String next(): цей метод повертає следующий елемент. Он вызывается, якщо hasNext() вернул true. За допомогою оператора switch он післядовательно повертає значення полей id, name и age в виде рядки (String), увеличивая счетчик currentProp на единицу після каждого вызова.

Таким чином, код створює специальную конструкцию в виде PersonIterator о том, як пошагово перебрать свойства об'єкта Person, и связывает эту конструкцию с самим класом Person через інтерфейс Iterable

И далїї в методі main мы можемо створити об'єкт класа Person и перебрать его свойства:


public static void main(String[] args) {
      
    Person tom = new Person(22, "Tom", 41);
    for(var prop : tom){
        System.out.println(prop);
    }
}

Результат виконання програми:


22
Tom
41

Метод forEach

Вместо цикла "for each" для перебора колекції можна вызвать методи Collection.forEach або Iterator.forEachRemaining с лямбда-выражением, которое використовує елемент. Лямбда-выражение застосовується ко всем елементам колекції або к оставшимся елементам, которые может обойти ітератор:


coll.forEach(element -> действия с element);
iter.forEachRemaining(element -> действия с element);

Наприклад:


import java.util.ArrayList;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam"); 
         
        // p представляє каждый об'єкт в people
        people.forEach(p -> System.out.println(p) );
    }
}
