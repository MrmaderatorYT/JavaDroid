Java Native Interface (JNI)

Останнє оновлення: 21.12.2025

Мы можемо написать всю програму целиком на Java, учитывая большие возможнасти языка, велике кількість досьупных фреймворков и библиотек на цем языке. Но бывают ситуации, когда может потребоваться одновременно вместе с кодом на Java використовувати також код на другом языке, зокрема, на C/C++. Наприклад, якщо приложению требуется доступ к системным функціям або устройствам, недоступным через платформу Java, або якщо вже имїїтся значний обсяг протестованого и відлагодженого коду на другом языке, який треба інтегрувати в програму на мові Java. Тому розглянемо як мы можемо взаємодіяти в програме на Java с нативным кодом на C/C++

Начиная вже с версии Java 1.1 платформа Java имїїт специальный API для взаимодействия с нативным кодом C, який називається Java Native Interface (JNI). Начиная с версии Java 20 був добавлен новый, болїї простой и удобный API - Foreign Functions and Memory API (FFM API). Тем не менїї JNI по прежнему может використовуватися, особенно на старых версиях платформы. Тому вначале вкратце розглянемо, базовые моменты JNI

Общий процес связывания нативного методу с Java-програмой за допомогою JNI выглядит наступним чином:

  1. Оголошуємо нативний метод-обгортку для функції Си в Java-класе.

  2. За допомогою команди javac -h генерируем заголовочный файл с оголошенням методу на языке C.

  3. Визначаємо код для нативного методу на мові C.

  4. Компилируем код на Си в розділяемую бібліотеку.

  5. Загружаем эту бібліотеку в програме на Java

Рассмотрим по шагово весь процес.

Визначенийие обертки функції на Java

При использовании JNI для функції языка C/C++, яку мы хочемо використовувати, необхідно оголосити проксі-метод за допомогою ключевого слова native. Це ключевое слово повідомляє компилятору, что метод буде визначений зовні. Нативные методи не містять коду Java, и за заголовком методу відразу йде крапка с комою. Тому оголошення нативных методів выглядят аналогично оголошенням абстрактных методів.

Предположим, есть функція на языке C, яку необхідно використовувати в коді на мові Java. Для наглядности возьмем стандартную библиотечную функцію printf с одним аргументом и для її применения визначимо файл HelloNative.java со наступним кодом на мові Java:


class HelloNative {  
    public static native int printf(String str);
}

Здесь визначений клас HelloNative с одним нативным методів, ім'я которого відповідає названию функції printf. Нативные методи можуть быть як статическими, так и нестатическими. У даному випадку нативний метод також оголошений як статический.

Далїї нам треба визначити заголовочный файл на языке С. Для цього необхідно скомпилироть данный клас с флагом -h, указав каталог, в який повинні быть помщены заголовочные файли:

javac -h . HelloNative.java

В итоге буде скомпилирован файл HelloNative.h со наступним кодом:

/* DO NOT EDIT THIS FILE - it is machine generated */
#include <jni.h>
/* Header for class HelloNative */

#ifndef _Included_HelloNative
#define _Included_HelloNative
#ifdef __cplusplus
extern "C" {
#endif
/*
 * Class:     HelloNative
 * Method:    printf
 * Signature: (Ljava/lang/String;)I
 */
JNIEXPORT jint JNICALL Java_HelloNative_printf
  (JNIEnv *, jclass, jstring);

#ifdef __cplusplus
}
#endif
#endif

Данный файл містить оголошення функції Java_HelloNative_printf. Причем зверніть увагу, як називається функція: "Java_HelloNative_printf", а не просто "printf". В JNI имена функцій строго регламентированы: сначала а додаєся префикс "Java", затем додаєся пакет класа (якщо пакет вложенный, наприклад, "com..test", то в полном названии пакету крапка заменяется на прочерк - "com__test") и в конце додаєся название класа и название функції через підчеркивание. То есть отримуєся: Java_ + ИмяКласа_ + ИмяМетоду. Значит, в Java цей метод оголошений в класе HelloNative як native int printf(String s)

В принципе мы могли бы и вручную прописать підобный файл, но всегда есть вероятность ошибиться, поцему лучше використовувати автогенерацию.

Визначенийие коду на Си

Далїї визначимо ще один файл - HelloNative.c со наступним кодом на языке С:

#include "HelloNative.h"
#include <stdio.h>

JNIEXPORT jint JNICALL Java_HelloNative_printf (JNIEnv *env, jclass cl, jstring jstr) {

    const char *cstr = (*env)->GetStringUTFChars(env, jstr, NULL);
    int numchars = printf(cstr);
    (*env)->ReleaseStringUTFChars(env, jstr, cstr);
    return numchars;
}

Рассмотрим підробно, что происходит в каждой рядківе:

  1. #include "HelloNative.h

    Прежде всего підключаем наш автосгенерированный заголовочный файл, чтобы Java смогла визначити функцію при її вызове.

  2. #include <stdio.h>

    Підключение стандартного заголовочного файлу языка С, який містить прототипи функцій для работы с підсистемой ввода-вивіда

  3. JNIEXPORT jint JNICALL Java_HelloNative_printf

    JNIEXPORT и JNICALL - це специальные макросы, которые гарантируют, что функція буде видна Java-машине (JVM) и буде використовувати правильное соглашение о вызовах. Між макросами указан повертаємоый тип - jint, которому в мові Java відповідає int. И після макросов идет ім'я функції - Java_HelloNative_printf и її визначенийие.

  4. (JNIEnv *env, jclass cl, jstring jstr)

    Каждая нативная функція отримує параметр env, який необходим для доступа к API языка Си для нативных вызовов.

    Параметр cl - це дескриптор Java-класа (для нестатических методів цему параметру передаєся дескриптор об'єкта).

    Параметр jstr типу jstring - собственно то значення (рядок), которое буде передаваться при вызове данного методу в коді Java.

  5. Далїї идет собственно код методу. И начинается он с получения рядки (GetStringUTFChars)

    const char *cstr = (*env)->GetStringUTFChars(env, jstr, NULL);

    В Java рядки хранятся в формате Unicode, а функції C (як printf) работают с масивами char.

    Эта рядок конвертирует Java-рядок (jstring) в стандартную C-рядок (UTF-8).

  6. Вивід на экран

    int numchars = printf(cstr);

    Здесь вызывается стандартная функція языка C printf. Она виводить рядок в консоль. Змінна numchars сохраняет кількість напечатанных символів.

  7. Звільнення пам'яті (ReleaseStringUTFChars)

    (*env)->ReleaseStringUTFChars(env, jstr, cstr);

    Це критически важный шаг. Поскольку JNI выделил память під cstr, її необхідно освободить вручную, чтобы избежать утечек памяти.

    Виртуальная машина повинна знать, когда мы закончили використовувати рядок, чтобы она могла выполнить сборку мусора. Поскольку сборщик мусора работает в отдельном потіке и может прерывать виконання нативных методів, то необхідно вызвать функцію ReleaseStringUTFChars.

  8. Підзврат значення

    return numchars;

    Количество напечатанных символів повертаєся обратно в Java-код.

Компиляция бібліотеки на Си

Итак, у нас есть заголовочный файл и файл коду C, и теперь нам треба скомпилировать весь цей нативний код на языке C в динамически загружаемую бібліотеку. Конкретные підробности зависят от выбранного компилятора. У даному випадку посмотрим, як це сделать, використовуючи 2 распространенных компилятора.

Так, при использовании компилятора GCC на Linux команда компиляции буде выглядеть наступним чином:


gcc -fPIC -I $JAVA_HOME/include -I $JAVA_HOME/include/linux -shared -o libHelloNative.so HelloNative.c

Здесь варто учитывать, что змінна окрвжения $JAVA_HOME повинна быть визначенийа и указывать на папку установки java. Соответственно путь $JAVA_HOME/include буде указывать на папку с заголовочными файлуми, которые необходимы для компиляции бібліотеки на C, зокрема, файл jni.h, який підключался выше в одном из файлів. Тем не менїї эта змінна может быть и не установлена, в цем случае (якщо нет желания устанавливать змінну $JAVA_HOME) можна передать конретный путь к папке, где располагаются заголовочные файли. Наприклад, якщо java локально установлена в каталогу "/usr/java/jdk-25", то соответственно нам треба прописать


gcc -fPIC -I /usr/java/jdk-25/include -I /usr/java/jdk-25/include/linux -shared -o libHelloNative.so HelloNative.c

В любом случае після виконання цей команди в текущей папке повинен появиться файл бібліотеки libHelloNative.so (название самой бібліотеки - "HelloNative" - без префикса "lib" и расширения файлу)

Если текущая ОС - Windows и компилятор - VisualC++ от Microsoft, то там мы могли бы выполнить следующую команду


cl -I %JAVA_HOME%\include -I %JAVA_HOME%\include\win32 -LD HelloNative.c -FeHelloNative.dll

В цем случае в текущей папке появится файл "HelloNative.dll" ((название самой бібліотеки також "HelloNative").

Визначенийие коду Java

Для загрузки бібліотеки нам треба добавить в код програми вызов методу System.loadLibrary, в який передаєся название бібліотеки. Щоб гарантировать загрузку бібліотеки виртуальной машиной до первого использования класа, можна використовувати блок статической инициализации. Наприклад, визначимо в текущей папке следующий файл Program.java:


class Program {
  
    public static void main(String[] args) {
        
        HelloNative.printf("Hello \n");    
    }

    static {  
        System.loadLibrary("HelloNative");
    }
}

Таким чином, в папке проєкта у нас будуть следующие файли:

  • HelloNative.java: клас с нативным методом-оберткой для функції языка Си

  • HelloNative.class: скомпилированный клас

  • HelloNative.h: автосгенерированный заголовочный файл

  • HelloNative.c: реалізація функції из заголовочного файлу

  • libHelloNative.so/HelloNative.dll: скомпилированная бібліотека

  • Program.java: основной файл програми

Запуск програми

Скомпилируем програму на Java стандартной командой

javac Program.java

Для запуску програми прежде всего нам треба передать утилите java дополнительную опцию, которая як-бы разрешает використання нативного коду в програме на Java. Если модулі не використовуються, то програма запускуєся наступним чином:

java --enable-native-access=ALL-UNNAMED ...

В модульной же програме необхідно перечислить все модулі, для которых разрешен нативний доступ:

java --enable-native-access=module1, module2 ...

При запуску на Linux також необхідно добавить текущий каталог в путь к бібліотекам, чтобы java нашла нашу бібліотеку. Для цього можна установить змінну среды LD_LIBRARY_PATH:

export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH

Здесь за допомогою точки указываем текущую папку. Або можна використовувати системное свойство java.library.path:

java --enable-native-access=ALL-UNNAMED -Djava.library.path=. Program

Наприклад:


eugene@Eugene:/workspace/java/test$ java --enable-native-access=ALL-UNNAMED -Djava.library.path=. Program
Hello 
eugene@Eugene:/workspace/java/test$
