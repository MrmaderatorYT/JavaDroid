CompletableFuture та проміси. Обробка результату асинхронних завдань

Останнє оновлення: 29.10.2025

Об'єкт Future, який представляє асинхронную задачу, дозволяє отримати результат задачи за допомогою метода get(), який приведет к блокировки текущего потока до тех пор, поки результат асинхронной задачи не станет доступен. Класс CompletableFuture, який реализует інтерфейс Future, надає альтернативний механизм для получения результата. CompletableFuture использует функцию обратного вызова, которая вызвается с результатом асинхронной задачи, когда он станет доступен. Можна сказать, что CompletableFuture - це Future, який может быть явно завершен (с установкой его значения и статуса)

Крім того, CompletableFuture реализует інтерфейс CompletionStage, який представляє этап асинхронного вычисления и выполняет действие або вычисляет значение после завершения другого этапа. Можна сказать, что CompletableFuture дозволяє реализовать нечто похожїї на промисы (promise) в некоторых языках программирования (як, наприклад, в JavaScript)

Створення CompletableFuture

Для непосредственного создания об'єкта CompletableFuture можна використовувати конструктор без параметров:

CompletableFuture future = new CompletableFuture();

Класс CompletableFuture<T> є обобщенным и типизируется типом результата. Так, в коде выше предполагается, что результатом асинхронной задачи буде значение типу Integer.

После создания об'єкта за допомогою конструктора его можна настроить на выполнение якой-або задачи. Однак класс CompletableFuture також представляє ряд методів, которые дозволяють создать об'єкт данного класса с вже добавленной асинхронной задачей, яку йде выполнить. Наприклад, це статический метод CompletableFuture<U>.supplyAsync():


static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier)
static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier, Executor executor)

Цет метод имїїт две версии. Обе версии в качестве результата повертає новый об'єкт CompletableFuture. И обе цих версии в качестве первого параметра принимают об'єкт Supplier, який и представляє выполняемое действие. Данный інтерфейс є функциональным и имїїт один метод get(), який повертає некоторое значение


interface Supplier<T>{
    T get();
}

Результат метода get() затем передаєся в обратный вызов - асинхронную задачу, которая и обробляє данный результат и которая устанавливается за допомогою дополнительных методів.

Различие между цими обеими версиями соварто лишь в том, як именно виконується асинхронная задача. Під второй версии вторым параметром передаєся об'єкт Executor, який и выполняет задачу. Для первой же версии асинхронная задача виконується за допомогою потока из пула ForkJoinPool (ForkJoinPool представляє специализированный пул потоков, разработанный для параллельного программирования). То есть при выполнении задач вы можете положиться на ForkJoinPool, або для большего контроля предоставить свой об'єкт Executor, який выполнит задачу.

Рассмотрим на простейшем примере вычисления факториала:


int number = 5;  // исходное число для вычисления факториала

CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
 
    // якщо число меньше 1, генерируем исключение
    if(number < 1) throw new RuntimeException("Number must be greater than 0");

    int n = 1;
    int result = 1;
    while(n <= number) result *= n++; 
    return result;  // возвращаем результат задачи
});

В данном случае в метод CompletableFuture.supplyAsync() неявно передаєся об'єкт Supplier в виде лямбда-выражения. Це лямбда-выражение берет внешнюю переменную number и вычисляет її факториал и повертає результат - це и буде результат метода get(). Конечно, можна буво бы и явным чином определить об'єкт Supplier:


int number = 5;  // исходное число для вычисления факториала

Supplier<Integer> task = () -> {
 
    // якщо число меньше 1, генерируем исключение
    if(number < 1) throw new RuntimeException("Number must be greater than 0");

    int n = 1;
    int result = 1;
    while(n <= number) result *= n++;
    return result;  // возвращаем результат задачи
};
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(task);

Обратите внимание, что функция, которая представляє інтерфейс Supplier, не может генерировать проверяемое исключение (checked excetion). Тому в примере выше для генерации исключения при передаче некорректного числа використовується тип RuntimeException.

Добавление обратного вызова

После создания CompletableFuture и установки выполняемой задачи нам треба добавить обратный вызов, який буде вызываться при завершении CompletableFuture. Для цього в классе определен большой набор методів. Приведу лишь некоторые:

  • CompletableFuture<Void> thenAccept(Consumer<? super T> action)

    Применяет функцию к результату и повертає void.

  • <U> CompletableFuture<U> thenApply(Function<? super T, ? extends U> fn)

    Применяет функцию к результату.

  • CompletableFuture<Void> thenRun(Runnable action)

    Виконує Runnable с результатом типу void.

  • <U> CompletableFuture<U> thenCompose(Function<? super T, ? extends CompletionStage<U>> fn)

    Вызывает функцию для результата и выполняет возвращаемый об'єкт Future.

Наприклад, используем метод thenAccept(), який приймає об'єкт інтерфейса Consumer. Цет інтерфейс имїїт метод accept(), який приймає некоторое значение:

void accept(T t)

Таким чином, обратный вызов повинен представлять функцию, которая приймає значение произвольного типу, а в качестве возвращаемого типу испольует void

Рассмотрим на примере факториала:


import java.util.concurrent.*;
import java.util.function.Supplier;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        int number = 5;  // исходное число для вычисления факториала
       
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> task = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0");
            int n = 1;
            int result = 1;
            while(n <= number) {

                result *= n++;
            }
            return result;  // возвращаем результат задачи
        };
 
        CompletableFuture<Integer> future = CompletableFuture.supplyAsync(task);

        // передаем в thenAccept обратный вызов
        future.thenAccept(result -> System.out.printf("factorial of %d is %d\n", number, result));

        // future.thenAccept - не блокує основной поток 
        // и одновременно мы можемо выполнять в основном потоке деяку работу 
        System.out.println("Main thread works...");
 
        Thread.sleep(2000);

        System.out.println("Main thread finished...");
    }
}

Рассмотрим на передачу обратного вызова:

future.thenAccept(result -> System.out.printf("factorial of %d is %d\n", number, result));

Здесь в метод передаєся лямбда-выражение, в котором параметр result представляє результ асинхронной задачи - результат вычисления факториала. Тело лямбда-выражения же представляє вывод результата вычисления.

Если суммировать все по этапам, то программа буде выполняться наступним чином:

  • System.out.println("Main thread started...")

    Основной поток (main) запускаєся и выводит це сообщение.

  • CompletableFuture future = CompletableFuture.supplyAsync(task);

    Передає ранїї определенную задачу task (вычисление факториала) в общий пул потоков (по умолчанию ForkJoinPool.commonPool()). Эта задача начинает выполняться параллельно в другом потоке.

  • future.thenAcceptAsync(result -> System.out.printf("factorial of %d is %d\n", number, result))

    Регистрирует обратный вызов (колбэк). Цет колбэк не виконується немедленно. Он ждет, поки задача task не завершится и не возвратит неокторый результат.

  • System.out.println("Main thread works...")

    Основной поток не ждет завершения задачи task, а немедленно продолжает работу (в данном случае для имитации работы выводим сообщение)

  • Выполнение задачи

    В це час (main або друкує "Main thread works...", або вже спит) параллельный поток очень быстро вычисляет факториал 5 (це 120). Це занимает доли миллисекунды.

  • Выполнение колбэка

    Сразу после завершения задачи task запускаєся зарегистрированный за допомогою функції thenAccept обратный вызов (обычно в том же потоке из пула) и выводит: factorial of 5 is 120.

  • Thread.sleep(2000);

    Основной поток "спит" 2000 мс. К цему моменту колбэк почти наверняка вже отработал и вывел результат.

  • System.out.println("Main thread finished...")

    Через 2 секунды основной поток просыпается и выводит сообщение, после чего программа завершуєся.

    • Наиболїї вероятный порядок вывода буде таким:

    
Main thread started...
Main thread works...
factorial of 5 is 120
Main thread finished...

    Таким чином, можна обробляти результат, як лише он станет доступен, не блокируя основой поток.

    Но в данном случае варто відзначити, что, поскольку потоки, используемые CompletableFuture по умолчанию (из пула ForkJoinPool.commonPool()), є потоками-демонами (daemon threads) - по сути фоновыми потоками, то JVM (Виртуальная машина Java) не ждет, поки они завершатся. JVM работает до тех пор, поки жив хотя бы один поток, який не є "демоном" . В программе выше единственный поток, який не є демоном - це поток метода main. Тому якщо убрать Thread.sleep, то поток main очень быстро выполнит методы supplyAsync() и thenAccept(), надрукує "Main thread works..." и "Main thread finished..." и немедленно завершится.

    Как лише main завершуєся, JVM видит, что активных потоков-не-демонов не осталось, и принудительно завершує всю программу. И в цем случае асинхронная задача task и її колбэк в thenAccept() просто не успевают выполниться до завершения основного потока. Тому именно Thread.sleep(2000) в коде выше "удерживает" основной поток живым, давая асинхронной задаче достаточно времени, чтобы выполниться и напечатать результат.

    Вместо использования задержек мы могли бы явным чином определить об'єкт Executor, який буде выполнять задачи. Например:

    
import java.util.concurrent.*;
import java.util.function.Supplier;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        int number = 5;  // исходное число для вычисления факториала
       
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> task = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0");
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++; 

            // имитируем долгую работу
            try {Thread.sleep(2000);}
            catch(InterruptedException _){}

            return result;  // возвращаем результат задачи
        };

        // определяем об'єкт Executor, який буде выполнять задачу
        ExecutorService executor = Executors.newCachedThreadPool();
        CompletableFuture<Integer> future = CompletableFuture.supplyAsync(task, executor);
        future.thenAccept(result -> System.out.printf("factorial of %d is %d\n", number, result));

        // future.thenAccept - не блокує основной поток 
        // и одновременно мы можемо выполнять в основном потоке деяку работу 
        System.out.println("Main thread works...");
        System.out.println("Main thread finished...");
        // закрываем исполнителя
        executor.close();  
    }
}

    В принципе здесь выполняются те же самые действия, лише теперь я убрал задержку для основного потока и добавил аналогичную задержку в задачу по вычислению факториала, чтобы эта задача завершалась после всех действий в основном потоке метода main.

    И також добавлено створення об'єкта Executor, який передаєся в метод supplyAsync():

    
ExecutorService executor = Executors.newCachedThreadPool();
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(task, executor);

    В данном случае создаваемый об'єкт Executor буде використовувати один из платформенных потоков из пула. Причем далїї цей Executor буде выполнять як задачу по вычислению факториала, так и коллбек, який выводит результат на консоль.

    В итоге мы скорїї всего получим следующий консольный вывод:

    
Main thread started...
Main thread works...
Main thread finished...
factorial of 5 is 120

    Поскольку в асинхронной задаче действует 2-секундная задержка, то скорїї всего асинхронная задача факториала буде выполнена после вывода на консоль строки "Main thread finished..." в методе main.

    Створення цепочки обратных вызовов

    Аналогичным чином мы можемо застосовувати и другие методы CompletedFuture для добавления обратных вызовов для обработки результатов. Болїї того мы можемо обробляти результаты других обратных вызовов. Наприклад, возьмем метод thenApply():

    <U> CompletableFuture<U> thenApply(Function<? super T, ? extends R> fn)

    Параметром метода є об'єкт інтерфейса Function - по сути це некоторая функция, которая приймає об'єкт типу T и повертає об'єкт типу R, то есть преобразует результат типу T в об'єкт типу R. Або якщо быть точнїї функция обратного вызова повинна соответствовать методу apply() інтерфейса Function:

    R apply(T t)

    Допустим, после вычисления факториала нам треба увеличить его в 2 раза:

    
import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.Function;
import java.util.function.Consumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        int number = 5;  // исходное число для вычисления факториала
       
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> factorialTask = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0");
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++; 
            return result; 
        };
        // определяем задачу, которая увеличивает в два раза
        Function<Integer, Integer> doubleTask = result ->  result * 2;
        // определяем задачу, которая выводит на консоль конечный результат
        Consumer<Integer> printTask = result -> System.out.printf("Final result: %d\n", result);

        // определяем об'єкт Executor, який буде выполнять задачи
        ExecutorService executor = Executors.newCachedThreadPool();
        // определяем коллбеки
        CompletableFuture
                .supplyAsync(factorialTask, executor)
                .thenApply(doubleTask)
                .thenAccept(printTask);

        System.out.println("Main thread works...");
        System.out.println("Main thread finished...");
        
        executor.close();  // закрываем исполнителя
    }
}

    Ключевой момент программы - це, конечно, определение цепочки выполняемых задач:

    
CompletableFuture
    .supplyAsync(factorialTask, executor)
    .thenApply(doubleTask)
    .thenAccept(printTask);

    Рассмотрим поэтапно:

    1. supplyAsync(factorialTask, executor)

      Визначає асинхронную задачу, которая виконується - це задача factorialTask, которая вычисляет факториала. Эта задача представляє тип Supplier<Integer> и поцему повертає значение типу Integer

    2. thenApply(doubleTask)

      Применяет к результату предыдущей задачи (в нашем случае це вычисление факториала) функцию doubleTask. Эта задача представляє тип Function<Integer, Integer>. То есть функция отримує значение типу Integer (факториал числа - результат задачи factorialTask) и повертає також значение типу Integer - удвоенное значение факториала.

    3. thenAccept(printTask)

      Применяет к результату предыдущей задачи (в нашем случае це результат задачи doubleTask) функцию printTask, которая представляє тип Consumer<Integer>. Эта задача отримує значение типу Integer (результат задачи doubleTask) и ничего не повертає (фактически повертає void) и просто выводит результат на консоль.

    Консольный вывод программы:

    
Main thread started...
Main thread works...
Main thread finished...
Final result: 240
