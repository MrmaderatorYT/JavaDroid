Область дії та час життя змінних

Останнє оновлення: 16.09.2025

Область дії (scope) и життя (lifetime) змінних визначають, где к переменным можна отримати доступ и як долго они хранятся в памяти. Область дії змінної — це область коду, где змінна видима и может быть использована. Час життя змінної — це период, в течение которого она находится в памяти. Локальные змінні існують лише во час виконання блока або методу, в котором они оголошенийы, и уничтожаются після завершення блока або методу. Змінні экземпляра класа існують до тех пор, поки об'єкт, которому они принадлежат, находится в памяти. Статические змінні існують в течение всего времени существования програми. Рассмотрим эти моменты підробнїї.

Область дії

Область дії або scope — це область програми, в которой змінна доступна або видима.

Java змінні можуть мати різні області дії в залежності від місця їх оголошення:

Локальна сфера дії

Змінні, оголошенийные всередині блока коду (заключенные в фигурные скобки {}), имеют локальну сферу дії и доступны лише всередині цього блока. Они обычно використовуються для временного хранения и уничтожаются після выхода блока:


class Program{
    // метод образует локальную область видимости
    public static void main(String[] args) {  

        // Визначаємо локальную змінну
        int x = 5;
        // використовуємо змінну x
        System.out.println("x: " + x); 

        // блок образует другую локальную область видимости
        {
            // Ще одна локальная змінна
            int y = 10;
            System.out.println("y: " + y);
        }  // здесь область действия змінної y завершуєся

        // System.out.println("y: " + y);  // здесь використовувати змінну y нельзя

    }   // здесь область действия змінної x завершуєся

    // метод test образует новую локальную область видимости
    public static void test() {  
        // System.out.println("x: " + x); // здесь використовувати змінну x нельзя
    }
}

Тут змінна x визначена локальна всередині методу main. Відповідно область її дії (scope) обмежена цим методом, і, скажімо, в іншому методі - методі test вона недоступна.

Аналогічно, змінна y також є локальною, але оголошена всередині вкладеного блоку і її область дії обмежена цим блоком. Спроба доступу до y поза цим блоком призведе до помилки компіляції, оскільки змінна в цьому випадку перебуватиме поза своєю сферою дії.

Область дії класу та змінні примірники

Змінні, оголошенийные на рівні класу (вне любого методу), є глобальными переменными. Они доступны всем методум и конструкторам класа и ,таким чином, имеют глобальную область действия - область класа. Наприклад:


class Person{
    // Глобальая змінна (область действия класа)
    String name;
    // Конструктор
    public Person(String name) {
        this.name = name;
    }

    public void print() {
        System.out.println("Name: " + name);
    }
}

Тут на рівні класу Person визначено глобальну змінну назву. Відповідно ми можемоо звертатися до цієї змінної всередині будь-якого методу класу, будь-де класу.

Час життя

У Java час життя змінної визначається часом, протягом якого вона перебуває у пам'яті. Локальні змінні, оголошені всередині методу чи блоку, мають час життя, обмежений виконанням цього методу або блоку, і знищуються після його завершення.

Глобальні змінні можуть мати два типи часу життя:

  • Нестатичні глобальні змінні (інакше змінні екземпляра/об'єкта) мають час життя, що відповідає часу життя об'єкта, якому вони належать. Такі змінні створюються під час створення об'єкта і знищуються при збиранні сміття.

  • Статические змінні (оголошенийные на уровне класа с ключевым словом static) имеют час жизни, охватывающїї всю програму, поскольку они создаются при первой загрузке класа и уничтожаются при завершении програми.

Наприклад:


class Program{

    public static void main(String[] args) {  

        Person.printCount();    // Count: 0

        Person bob = new Person("Bob");

        {
            Person tom = new Person("Tom");
            tom.printName();    // Name: Tom
        } // здесь формально завершуєся час жизни об'єкта tom и его змінної name
        // чуть позже сборщик мусора удалит змінну tom и її змінну name

        bob.printName();        // Name: Bob

        Person.printCount();    // Count: 2
    } // здесь формально завершуєся час жизни об'єкта bob и его змінної name

}  // здесь завершуєся час жизни статической змінної count

class Person{
    
    String name;  // час жизни - час жизни об'єкта
    static int count = 0;  // час жизни - час жизни програми

    public Person(String name) {
        this.name = name;
        count++;        // для теста увеличиваем счетчик об'єктов
    }

    public void printName() {
        System.out.println("Name: " + name);
    }

    public static void printCount() {
        System.out.println("Count: " + count);
    }
}

Тут у класі Person визначено дві глобальні змінні: name і count. Змінна назва належить об'єкту, тому буде створюватися окремо для кожного об'єкта (при створенні цього об'єкта) і знищується при знищенні об'єкта. А змінна count є статичною, тому знищуватиметься при завершенні програми.
