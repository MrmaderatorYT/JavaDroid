Встановлення пути к класам

Останнє оновлення: 21.09.2025

При створенні програм на Java мы так або иначе работаем с класами. Какие-то якие-то созданы непосредственно нами, а якие-то класы созданы другими разработчиками, и мы хотми підключить эти класы себе в програму. Эти класы можуть быть доступны в виде файлів с расширением .class, або в виде архивов jar. И у даному випадку мы можемо столкнуться с проблемой установки путей к цим класам: куда нам треба поместить эти файли або як указать компилятору/среде виконання, что мы хочемо використовувати именно эти файли.

Многие распространенные инструменты разработки и сборки, як наприклад, среды разработки IDE, имеют свои механизмы управления путями к класам. Однак разработчик при необхідности может задать пути к класам вручную. Тому розглянемо, як мы можемо управлять путями к класам Java.

Расположение класов и формат пути

Підключаемые класы можуть распространяться в виде отдельных файлів с расширением .class або в качестве бібліотеки в виде jar-файлу.

Файли класов

Путь к файлум .class представляє путь к папке. Причем файли располагаются с учетом из пакетів. Наприклад, в качестве примера створимо в системе папку /home/user/classdir (на Linux) або c:\classdir (на Windows)

Пусть у нас есть следующий клас Person


package com.;

public class Person 
{
    String name; 

    public Person(String name){

        this.name = name;
    }

    public void print() {

        System.out.printf("Person %s\n", name);
    }
}

Цет код компилируется в файл Person.class. Поскольку цей клас принадлежит пакету "com.", соответственно полное название класа "com..Person". Тому цей клас помещается в підкаталог /home/user/classdir/com//Person (на Linux) або c:\classdir\com\\Person (на Windows). То есть при размщении класов мы учитываем пакети класа и проецируем эти пакети на структуру каталогов.

jar-файли

Путь к класам всередині .jar- файлу представляє путь к самому .jar- файлу. Наприклад, возьмем в качестве примера папки для jar-файлу папку /home/user/jardir (на Linux) або c:\jardir (на Windows).

Пусть в цей папке у нас располагается бібліотека mathlib.jar. В цем случае путь к її класам цей библиотеке буде путь:

  • /home/user/jardir/mathlib.jar (на Linux)

  • c:\jardir\mathlib.jar (на Windows)

Формат пути

Путь к класам — це совокупность всех расположений, где можуть находиться файли класов. То есть может быть место (допустим, папка classdir), где храняться по отдельности скомпилированные класы, наприклад:

// Linux
/home/user/classdir

// Windows
c:\classdir

И також отдельно по іншому пути можуть располагаться бібліотеки jar. Наприклад, у нас есть бібліотека "mathlib.jar" по следующему пути:

// Linux
/home/user/jardir/mathlib.jar

// Windows
c:\jardir\mathlib.jar

В цем случае общий путь к класам буде містятьь оба цих підпути. Формат пути к класам следующий:

// Linux
путь1:путь2:путь3:путьN

// Windows
путь1;путь2;путь3;путьN

Все підпути к класам перечисляются друг за другом и отделяются друг от друга розділителем. В Linux таким розділителем є двоеточие (":"), а в Windows - крапка с комою (";").

Кроме конкретных путей к класам мы можемо використовувати специальные относительные пути, наприклад, "." (путь к текущей папке), ".." (путь к родительскому каталогу) и т.д.

И якщо перейти непосредственно к нашим класам, то мы можемо визначити пути к класам наступним чином:


// Для Linux
/home/user/classdir:.:/home/user/jardir/mathlib.jar

// Windows
c:\classdir;.;c:\jardir\mathlib.jar

То есть у даному випадку для хранения класов мы будем використовувати три підпути:

  • Базовый каталог "/home/user/classdir" або "c:\classdir"

  • Текущий каталог (представляє символ крапка = ".")

  • Путь к jar - "/home/user/jardir/mathlib.jar" або "c:\jardir\mathlib.jar"

Варто відзначити, что у нас может быть декілька jar-файлів, которые мы хочемо використовувати. В цем случае при визначенийии пути можна використовувати символ підстановки, чтобы указать каталог для произвольного количества JAR-файлів, наприклад:

// Unix/Linux - символ * необхідно экранировать, чтобы предотвратить расширение командной оболочкой
/home/user/classdir:.:/home/user/jardir/'*'

// Windows
c:\classdir;.;c:\jardir\*

Теперь посмотрим, як мы можемо установить цей путь непосредственно при компиляции/виконання програми.

Змінна окрвжения CLASSPATH

Прежде всего пути к класам управляются через змінну окрвжения CLASSPATH, которая обычно додаєся в систему при установке JDK. Обычно в качестве путей к файлум класов застосовуються текущий и родительский каталог. Наприклад, выведем значення цей змінної на консоль:


eugene@Eugene:~$  echo $CLASSPATH
.:..
eugene@Eugene:~$

В моем случае вивід покизывает следующую рядок ".:..". У даному випадку первая крапка "." вказує на текущий каталог, а післядние две точки ".." на родительский каталог. Двоеточие між ними служит в качестве розділителя між путями. То есть у даному випадку змінна CLASSPATH містить два пути: текущий и родительский каталог.

Таким чином, мы можемо поместить файли в текущий або в родительский каталог, и компилятор/среда виконання их найдут.

Настройка змінної окрвжения CLASSPATH

Обычно не рекомендуется менять глобальные настройки. И тогда при работе мы можемо задать пути на час работы консоли. Встановлення змінної окрвжения CLASSPATH у даному випадку зависит от конкретной оболочки и системы:


// Для Linux (bash)
export CLASSPATH=/home/user/classdir:.:/home/user/jardir/mathlib.jar

// Windows
set CLASSPATH=c:\classdir;.;c:\jardir\mathlib.jar

Путь к класам сохраняется до завершення работы оболочки. Таким чином, мы сохраним начальное значення змінної CLASSPATH. С другой стороны, в следующий раз при работе с Java, нам потребуется снова устанавливать эту змінну.

Параметр -classpath

Самый простой способ установки пути к класам заключается в установке параметра -classpath (или -cp, або болїї современного варианта --class-path). При чем цей параметр действует як при компиляции с помощь. javac, так и при выполнении програми за допомогою утилиты java:


// Для Linux
javac -classpath /home/user/classdir:.:/home/user/jardir/mathlib.jar Program.java
java -classpath /home/user/classdir:.:/home/user/jardir/mathlib.jar Program

// Windows
javac -classpath c:\classdir;.;c:\jardir\mathlib.jar  Program.java
java -classpath c:\classdir;.;c:\jardir\mathlib.jar Program

Вся команда повинна быть набрана в одну рядок.

Поиск класов

Как происходит поиск файлів и класов?

Поиск компилятора

Если вы в коді програми використовуєе клас, не указывая его пакет, компилятору сначала необхідно найти пакет, содержащий данный клас. Для цього компилятор просматривает все директивы импорта як возможные источники расположения цього класа. Предположим, в коді використовується неякий клас com..Person, и компилятору треба найти путь при компиляции следующей програми:


import java.util.*;
import com..*;

class Program{

    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        tom.print();
    }
}

Данный исходный файл містить две директивы импорта:


import java.util.*;
import com..*;

В цем случае компилятор пытается найти класы java.lang.Person (поскольку пакет java.lang всегда импортируется по умолчанию), java.util.Person, com..Person и Person в текущем пакеті. Компилятор ищет каждый из цих класов во всех местах пути к класам, зокрема, пути, которые установлены за допомогою змінної CLASSPATH и параметра -classpath. При цем порядок директив импорта або поиска не имїїт значення, оскільки имена класов повинні быть уникальными. А якщо буде найдено болїї одного класа, возникнет ошибка компиляции.

Крім того, компилятор також проверяет исходные файли, чтобы визначити, є ли исходный файл болїї новым, чем файл класа. В цем случае исходный файл автоматически перекомпилируется.

Поиск виртуальной машины

Предположим, виртуальная машина ищет файл класа com..Person. Сначала она просматривает стандартные класы API Java. Там клас не найден, поцему она обращается к пути к класам. Допустим, путь к класам установлен на "/home/user/classdir:.:/home/user/jardir/mathlib.jar"

Використовуючи эти пути виртуальная машина ищет следующие файли:

  • /home/user/classdir/com//Person.class

  • com//Person.class в текущем каталогу

  • com//Person.class в архиве /home/user/jardir/mathlib.jar

Компилятор Javac всегда ищет файли в текущем каталогу, но модуль запуску виртуальной машины Java ищет файли в текущем каталогу лише в том случае, якщо каталог "." присутствует в пути к класам.
