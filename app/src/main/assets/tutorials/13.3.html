Установка пути к классам

Останнє оновлення: 21.09.2025

При создании программ на Java мы так або иначе работаем с классами. Какие-то якие-то созданы непосредственно нами, а якие-то классы созданы другими разработчиками, и мы хотми подключить эти классы себе в программу. Эти классы можуть быть доступны в виде файлов с расширением .class, або в виде архивов jar. И в данном случае мы можемо столкнуться с проблемой установки путей к цим классам: куда нам треба поместить эти файлы або як указать компилятору/среде виконання, что мы хочемо використовувати именно эти файлы.

Многие распространенные инструменты разработки и сборки, як наприклад, среды разработки IDE, имеют свои механизмы управления путями к классам. Однак разработчик при необхідности может задать пути к классам вручную. Тому розглянемо, як мы можемо управлять путями к классам Java.

Расположение классов и формат пути

Подключаемые классы можуть распространяться в виде отдельных файлов с расширением .class або в качестве библиотеки в виде jar-файла.

Файлы классов

Путь к файлам .class представляє путь к папке. Причем файлы располагаются с учетом из пакетів. Наприклад, в качестве примера создадим в системе папку /home/user/classdir (на Linux) або c:\classdir (на Windows)

Пусть у нас есть следующий класс Person


package com.;

public class Person 
{
    String name; 

    public Person(String name){

        this.name = name;
    }

    public void print() {

        System.out.printf("Person %s\n", name);
    }
}

Цет код компилируется в файл Person.class. Поскольку цей класс принадлежит пакету "com.", соответственно полное название класса "com..Person". Тому цей класс помещается в подкаталог /home/user/classdir/com//Person (на Linux) або c:\classdir\com\\Person (на Windows). То есть при размщении классов мы учитываем пакеты класса и проецируем эти пакеты на структуру каталогов.

jar-файлы

Путь к классам внутри .jar- файла представляє путь к самому .jar- файлу. Наприклад, возьмем в качестве примера папки для jar-файла папку /home/user/jardir (на Linux) або c:\jardir (на Windows).

Пусть в цей папке у нас располагается библиотека mathlib.jar. В цем случае путь к її классам цей библиотеке буде путь:

  • /home/user/jardir/mathlib.jar (на Linux)

  • c:\jardir\mathlib.jar (на Windows)

Формат пути

Путь к классам — це совокупность всех расположений, где можуть находиться файлы классов. То есть может быть место (допустим, папка classdir), где храняться по отдельности скомпилированные классы, например:

// Linux
/home/user/classdir

// Windows
c:\classdir

И також отдельно по іншому пути можуть располагаться библиотеки jar. Наприклад, у нас есть библиотека "mathlib.jar" по следующему пути:

// Linux
/home/user/jardir/mathlib.jar

// Windows
c:\jardir\mathlib.jar

В цем случае общий путь к классам буде містятьь оба цих подпути. Формат пути к классам следующий:

// Linux
путь1:путь2:путь3:путьN

// Windows
путь1;путь2;путь3;путьN

Все подпути к классам перечисляются друг за другом и отделяются друг от друга разделителем. В Linux таким разделителем є двоеточие (":"), а в Windows - крапка с комою (";").

Кроме конкретных путей к классам мы можемо використовувати специальные относительные пути, наприклад, "." (путь к текущей папке), ".." (путь к родительскому каталогу) и т.д.

И якщо перейти непосредственно к нашим классам, то мы можемо определить пути к классам наступним чином:


// Для Linux
/home/user/classdir:.:/home/user/jardir/mathlib.jar

// Windows
c:\classdir;.;c:\jardir\mathlib.jar

То есть в данном случае для хранения классов мы будем використовувати три подпути:

  • Базовый каталог "/home/user/classdir" або "c:\classdir"

  • Текущий каталог (представляє символ крапка = ".")

  • Путь к jar - "/home/user/jardir/mathlib.jar" або "c:\jardir\mathlib.jar"

Варто відзначити, что у нас может быть декілька jar-файлов, которые мы хочемо використовувати. В цем случае при определении пути можна використовувати символ подстановки, чтобы указать каталог для произвольного количества JAR-файлов, например:

// Unix/Linux - символ * необхідно экранировать, чтобы предотвратить расширение командной оболочкой
/home/user/classdir:.:/home/user/jardir/'*'

// Windows
c:\classdir;.;c:\jardir\*

Теперь посмотрим, як мы можемо установить цей путь непосредственно при компиляции/виконання програми.

Переменная окрвжения CLASSPATH

Прежде всего пути к классам управляются через переменную окрвжения CLASSPATH, которая обычно додаєся в систему при установке JDK. Обычно в качестве путей к файлам классов застосовуються текущий и родительский каталог. Наприклад, выведем значение цей змінної на консоль:


eugene@Eugene:~$  echo $CLASSPATH
.:..
eugene@Eugene:~$

В моем случае вывод покизывает следующую строку ".:..". В данном случае первая крапка "." указывает на текущий каталог, а последние две точки ".." на родительский каталог. Двоеточие между ними служит в качестве разделителя между путями. То есть в данном случае переменная CLASSPATH містить два пути: текущий и родительский каталог.

Таким чином, мы можемо поместить файлы в текущий або в родительский каталог, и компилятор/среда виконання их найдут.

Настройка змінної окрвжения CLASSPATH

Обычно не рекомендуется менять глобальные настройки. И тогда при работе мы можемо задать пути на час работы консоли. Установка змінної окрвжения CLASSPATH в данном случае зависит от конкретной оболочки и системы:


// Для Linux (bash)
export CLASSPATH=/home/user/classdir:.:/home/user/jardir/mathlib.jar

// Windows
set CLASSPATH=c:\classdir;.;c:\jardir\mathlib.jar

Путь к классам сохраняется до завершения работы оболочки. Таким чином, мы сохраним начальное значение змінної CLASSPATH. С другой стороны, в следующий раз при работе с Java, нам потребуется снова устанавливать эту переменную.

Параметр -classpath

Самый простой способ установки пути к классам заключается в установке параметра -classpath (или -cp, або болїї современного варианта --class-path). При чем цей параметр действует як при компиляции с помощь. javac, так и при выполнении программы за допомогою утилиты java:


// Для Linux
javac -classpath /home/user/classdir:.:/home/user/jardir/mathlib.jar Program.java
java -classpath /home/user/classdir:.:/home/user/jardir/mathlib.jar Program

// Windows
javac -classpath c:\classdir;.;c:\jardir\mathlib.jar  Program.java
java -classpath c:\classdir;.;c:\jardir\mathlib.jar Program

Вся команда повинна быть набрана в одну строку.

Поиск классов

Как происходит поиск файлов и классов?

Поиск компилятора

Если вы в коде программы используете класс, не указывая его пакет, компилятору сначала необхідно найти пакет, содержащий данный класс. Для цього компилятор просматривает все директивы импорта як возможные источники расположения цього класса. Предположим, в коде використовується неякий класс com..Person, и компилятору треба найти путь при компиляции следующей программы:


import java.util.*;
import com..*;

class Program{

    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        tom.print();
    }
}

Данный исходный файл містить две директивы импорта:


import java.util.*;
import com..*;

В цем случае компилятор пытается найти классы java.lang.Person (поскольку пакет java.lang всегда импортируется по умолчанию), java.util.Person, com..Person и Person в текущем пакеті. Компилятор ищет каждый из цих классов во всех местах пути к классам, в частности, пути, которые установлены за допомогою змінної CLASSPATH и параметра -classpath. При цем порядок директив импорта або поиска не имїїт значения, оскільки имена классов повинні быть уникальными. А якщо буде найдено болїї одного класса, возникнет ошибка компиляции.

Крім того, компилятор також проверяет исходные файлы, чтобы определить, є ли исходный файл болїї новым, чем файл класса. В цем случае исходный файл автоматически перекомпилируется.

Поиск виртуальной машины

Предположим, виртуальная машина ищет файл класса com..Person. Сначала она просматривает стандартные классы API Java. Там класс не найден, поцему она обращается к пути к классам. Допустим, путь к классам установлен на "/home/user/classdir:.:/home/user/jardir/mathlib.jar"

Используя эти пути виртуальная машина ищет следующие файлы:

  • /home/user/classdir/com//Person.class

  • com//Person.class в текущем каталоге

  • com//Person.class в архиве /home/user/jardir/mathlib.jar

Компилятор Javac всегда ищет файлы в текущем каталоге, но модуль запуска виртуальной машины Java ищет файлы в текущем каталоге лише в том случае, якщо каталог "." присутствует в пути к классам.
