Інтерфейси Comparable и Comparator. Сортування

Останнє оновлення: 01.10.2025

Сортування - одна из распротсраненных завдань при работе с данными. В Java в различных структурах данных есть встроенная піддержка сортировки. Однак для піддержки сортировки обычно требуется реалізація двух інтерфейсов: Comparable и Comparator

Інтерфейс Comparator

Многие встроенные колекції в Java имеют методи сортировки, которые обычно називаються sort(). Наприклад, в інтерфейсе List, який визначає базовый функционал для колекцій-списків, есть підобный метод:


default void sort(Comparator<? super E> c)

То есть, чтобы отсортировать список, нам треба в метод sort() передать компаратор - об'єкт типу Comparator, який типизирован типом елементов списку. Компаратор дозволяє сравнить два об'єкта.

Інтерфейс Comparator містить ряд методів, ключевым из которых є метод compare():

   
public interface Comparator<T> {

	int compare(T a, T b);
	// остальные методи
}

Метод compare приймає два сравниваемых об'єкта и повертає числовое значення в зависимости от результата сравнения:

  • Если об'єкт a предшествует об'єкту b (грубо говоря, a < b), то повертаєся отрицательное значення

  • Если об'єкт a идет після об'єкта b (a > b), то повертаєся положительное значення

  • Если об'єкты равны (a == b), то повертаєся ноль

Для применения інтерфейса необхідно створити клас компаратора, який реалізує цей інтерфейс. Наприклад, у нас есть следующий клас Person:


class Person{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }
}

Допустим, мы хочемо сравнивать два об'єкта Person, благодаря чему буде производиться сортування. Для цього визначимо следующий клас компаратора:


class PersonNameComparator implements Comparator<Person>{
 
    public int compare(Person a, Person b){
     
        return a.getName().compareTo(b.getName());
    }
}

Здесь в реалізації мы сравниваем по полю name. Для цього використовуємо метод compareTo(), який есть у рядків и який дозволяє сравнить две рядки. Це удобно, поскольку нам не треба дополнительно визначати логику сравнения символів рядків, а можна воспользоваться готовым функционалом. То есть, якщо a.name условно меньше (в лексекографическом поряке размещается ранїї), чем b.name, то метод compareTo() повертає отрицательное число. Если a.name условно больше, чем b.name, то повертаєся положительное число. Если рядки равны, то повертаєся 0.

Хотя в реальности мы могли бы визначити и свою логику, наприклад, сравнивать по длине імені:


class PersonNameComparator implements Comparator<Person>{
 
    public int compare(Person a, Person b){
     
        return a.getName().length()-p.getName().length();
    }
}

Посмотрим, як все це використовується:


import java.util.ArrayList; 
import java.util.Comparator; 

public class Program{

        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Sam", 28));

        people.sort(new PersonNameComparator());

        for(var p : people){
            p.print();
        }
    }
}

Здесь мы определяем список об'єктов Person и використовуємо компаратор PersonNameComparator для сортировки списку:

people.sort(new PersonComparator());

Благодаря цему при вивіде на консоль мы увидим список, отсортированный в зависимости от поля name:

Аналогично мы могли бы визначити компаратор для сравнения по полю age:


import java.util.ArrayList; 
import java.util.Comparator; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Sam", 28));

        people.sort(new PersonAgeComparator());

        for(var p : people){
            p.print();
        }
    }
}
class PersonAgeComparator implements Comparator<Person>{
  
    public int compare(Person a, Person b){

        return a.getAge() - b.getAge();
    }
}

class Person{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }
}

Здесь в качестве компаратора застосовується клас PersonAgeComparator, логика которого тоже не отличается сложностью: просто возващаем результат выражения a.getAge() - b.getAge(). Что фактически эквивалентно, якщо бы мы расписали це наступним чином:


class PersonAgeComparator implements Comparator<Person>{
  
    public int compare(Person a, Person b){
      
        if(a.getAge() > b.getAge())
            return 1;
        else if(a.getAge() < b.getAge())
            return -1;
        else
            return 0;
    }
}

Но операція вычитания a.getAge() - b.getAge() гораздо проще и производительнїї, чем условная конструкция.

В итоге теперь мы получим другой вивід - с сортировкой по полю age:


Person Name: Sam;  Age: 28;
Person Name: Tom;  Age: 41;
Person Name: Bob;  Age: 46;

Сочетание компараторов

Також мы можемо застосовувати відразу декілька компараторов по принципу приоритета. Наприклад, мы хочемо, чтобы сначала сортування шла по полю name, а якщо есть декілька об'єктов с одинаковым полем name, то застосовувати сортировку по полю age.

Інтерфейс компаратора визначає специальный метод thenComparing(), який дозволяє використовувати цепочки компараторов для сортировки набора::


import java.util.ArrayList; 
import java.util.Comparator; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 22));
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Bob", 37));
        people.add(new Person("Sam", 28));

        Comparator<Person> totalComparator = new PersonNameComparator().thenComparing(new PersonAgeComparator());
        people.sort(totalComparator);

        for(var p : people){
            p.print();
        }
    }
}

У даному випадку за допомогою вызовов

PersonNameComparator().thenComparing(new PersonAgeComparator());

створюємо цепочку компараторов в виде условно общего компаратора totalComparator, який затем використовується для сортировки. В итоге теперь мы получим следующий консольный вивід:


Person Name: Bob;  Age: 37;
Person Name: Bob;  Age: 46;
Person Name: Sam;  Age: 28;
Person Name: Tom;  Age: 22;
Person Name: Tom;  Age: 41;

Інтерфейс Comparable

Кроме інтерфейса Comparator для сравнения об'єктов застосовується другой інтерфейс - Comparable. Данный інтерфейс досить прост, определяя один метод compareTo():


public interface Comparable
{
   int compareTo(Object other);
}

Цет метод служит тем же целям, что и метод compare в інтерфейсе Comparator. Метод compareTo() приймає об'єкт, сравнивает его с текущим и повертає числовое значення в зависимости от результата сравнения:

  • Если текущий об'єкт меньше об'єкта other, то повертаєся отрицательное значення

  • Если текущий об'єкт больше об'єкта other, то повертаєся положительное значення

  • Если об'єкты равны, то повертаєся ноль

Некоторые встроенные класы в JDK вже по умолчанию реалізують цей інтерфейс. Так, выше в примере с компаратором PersonNameComparator для сравнения рядків мы использовали метод compareTo()

return a.getName().compareTo(b.getName());

То есть у даному випадку мы як раз и использовали реализацию інтерфейса Comparable в рядокх. И мы можемо використовувати її при сортировке об'єктов, наприклад, в том же списке:


import java.util.ArrayList; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");

        people.sort(null);

        for(var p : people){
            System.out.println(p);
        }

    }
}

Здесь мы определяем и сортуємо список рядків. Консольный вивід:


Bob
Sam
Tom

Почему здесь виконується сортування? Если в метод sort() для компаратора передаєся значення null, то для виконання сортировки всі елементи в цем списке повинні реализовывать інтерфейс Comparable. И эта реалізація и буде застосовуватися для сортировки. То есть в случае выше для сортировки рядків неявно застосовується реалізація інтерфейса Comparable.

И мы також можемо реализовать данный інтерфейс и в своих кастомных класах:


class Person implements Comparable<Person>{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }

    public int compareTo(Person p){
     
        return name.compareTo(p.getName());
    }
}

У даному випадку мы не повертаємо явным чином ниякое число, а опять же полагаемся на встроенный механизм сравнения, який есть у класа String.

Застосування в програме:


import java.util.ArrayList; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Sam", 28));

        people.sort(null); // сортування на основе реалізації Comparable

        for(var p : people){
            p.print();
        }

    }
}
