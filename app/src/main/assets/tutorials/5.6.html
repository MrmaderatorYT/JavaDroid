Інтерфейси Comparable и Comparator. Сортування

Останнє оновлення: 01.10.2025

Сортування - одна из распротсраненных задач при работе с данными. В Java в различных структурах данных есть встроенная поддержка сортировки. Однак для поддержки сортировки обычно требуется реализация двух інтерфейсов: Comparable и Comparator

Интерфейс Comparator

Многие встроенные коллекции в Java имеют методы сортировки, которые обычно називаються sort(). Наприклад, в інтерфейсе List, який определяет базовый функционал для коллекций-списков, есть подобный метод:


default void sort(Comparator<? super E> c)

То есть, чтобы отсортировать список, нам треба в метод sort() передать компаратор - об'єкт типу Comparator, який типизирован типом элементов списка. Компаратор дозволяє сравнить два об'єкта.

Интерфейс Comparator містить ряд методів, ключевым из которых є метод compare():

   
public interface Comparator<T> {

	int compare(T a, T b);
	// остальные методы
}

Метод compare приймає два сравниваемых об'єкта и повертає числовое значение в зависимости от результата сравнения:

  • Если об'єкт a предшествует об'єкту b (грубо говоря, a < b), то повертаєся отрицательное значение

  • Если об'єкт a идет после об'єкта b (a > b), то повертаєся положительное значение

  • Если об'єкты равны (a == b), то повертаєся ноль

Для применения інтерфейса необхідно создать класс компаратора, який реализует цей інтерфейс. Наприклад, у нас есть следующий класс Person:


class Person{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }
}

Допустим, мы хочемо сравнивать два об'єкта Person, благодаря чему буде производиться сортировка. Для цього определим следующий класс компаратора:


class PersonNameComparator implements Comparator<Person>{
 
    public int compare(Person a, Person b){
     
        return a.getName().compareTo(b.getName());
    }
}

Здесь в реализации мы сравниваем по полю name. Для цього используем метод compareTo(), який есть у строк и який дозволяє сравнить две строки. Це удобно, поскольку нам не треба дополнительно визначати логику сравнения символов строк, а можна воспользоваться готовым функционалом. То есть, якщо a.name условно меньше (в лексекографическом поряке размещается ранїї), чем b.name, то метод compareTo() повертає отрицательное число. Если a.name условно больше, чем b.name, то повертаєся положительное число. Если строки равны, то повертаєся 0.

Хотя в реальности мы могли бы определить и свою логику, наприклад, сравнивать по длине имени:


class PersonNameComparator implements Comparator<Person>{
 
    public int compare(Person a, Person b){

    }
}

import java.util.ArrayList;

        people.add(new Person("Sam", 28));

            p.print();
        }
    }
}

people.sort(new PersonComparator());

Аналогично мы могли бы определить компаратор для сравнения по полю age:


import java.util.ArrayList; 
import java.util.Comparator; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Sam", 28));

        people.sort(new PersonAgeComparator());

        for(var p : people){
            p.print();
        }
    }
}
class PersonAgeComparator implements Comparator<Person>{
  
    public int compare(Person a, Person b){

        return a.getAge() - b.getAge();
    }
}

class Person{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }
}

Здесь в качестве компаратора застосовується класс PersonAgeComparator, логика которого тоже не отличается сложностью: просто возващаем результат выражения a.getAge() - b.getAge(). Что фактически эквивалентно, якщо бы мы расписали це наступним чином:


class PersonAgeComparator implements Comparator<Person>{
  
    public int compare(Person a, Person b){
      
        if(a.getAge() > b.getAge())
            return 1;
        else if(a.getAge() < b.getAge())
            return -1;
        else
            return 0;
    }
}

Но операция вычитания a.getAge() - b.getAge() гораздо проще и производительнїї, чем условная конструкция.

В итоге теперь мы получим другой вывод - с сортировкой по полю age:


Person Name: Sam;  Age: 28;
Person Name: Tom;  Age: 41;
Person Name: Bob;  Age: 46;

Сочетание компараторов

Також мы можемо застосовувати відразу декілька компараторов по принципу приоритета. Наприклад, мы хочемо, чтобы сначала сортировка шла по полю name, а якщо есть декілька об'єктов с одинаковым полем name, то застосовувати сортировку по полю age.

Интерфейс компаратора определяет специальный метод thenComparing(), який дозволяє використовувати цепочки компараторов для сортировки набора::


import java.util.ArrayList; 
import java.util.Comparator; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 22));
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Bob", 37));
        people.add(new Person("Sam", 28));

        Comparator<Person> totalComparator = new PersonNameComparator().thenComparing(new PersonAgeComparator());
        people.sort(totalComparator);

        for(var p : people){
            p.print();
        }
    }
}

В данном случае за допомогою вызовов

PersonNameComparator().thenComparing(new PersonAgeComparator());

создаем цепочку компараторов в виде условно общего компаратора totalComparator, який затем використовується для сортировки. В итоге теперь мы получим следующий консольный вывод:


Person Name: Bob;  Age: 37;
Person Name: Bob;  Age: 46;
Person Name: Sam;  Age: 28;
Person Name: Tom;  Age: 22;
Person Name: Tom;  Age: 41;

Интерфейс Comparable

Кроме інтерфейса Comparator для сравнения об'єктов застосовується другой інтерфейс - Comparable. Данный інтерфейс довольно прост, определяя один метод compareTo():


public interface Comparable
{
   int compareTo(Object other);
}

Цет метод служит тем же целям, что и метод compare в інтерфейсе Comparator. Метод compareTo() приймає об'єкт, сравнивает его с текущим и повертає числовое значение в зависимости от результата сравнения:

  • Если текущий об'єкт меньше об'єкта other, то повертаєся отрицательное значение

  • Если текущий об'єкт больше об'єкта other, то повертаєся положительное значение

  • Если об'єкты равны, то повертаєся ноль

Некоторые встроенные классы в JDK вже по умолчанию реализуют цей інтерфейс. Так, выше в примере с компаратором PersonNameComparator для сравнения строк мы использовали метод compareTo()

return a.getName().compareTo(b.getName());

То есть в данном случае мы як раз и использовали реализацию інтерфейса Comparable в строках. И мы можемо використовувати її при сортировке об'єктов, наприклад, в том же списке:


import java.util.ArrayList; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");

        people.sort(null);

        for(var p : people){
            System.out.println(p);
        }

    }
}

Здесь мы определяем и сортируем список строк. Консольный вывод:


Bob
Sam
Tom

Почему здесь виконується сортировка? Если в метод sort() для компаратора передаєся значение null, то для виконання сортировки все элементы в цем списке повинні реализовывать інтерфейс Comparable. И эта реализация и буде застосовуватися для сортировки. То есть в случае выше для сортировки строк неявно застосовується реализация інтерфейса Comparable.

И мы також можемо реализовать данный інтерфейс и в своих кастомных классах:


class Person implements Comparable<Person>{

    private String name;
    private int age;

    String getName(){ return name; }
    int getAge(){ return age;}

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    void print(){
        System.out.printf("Person Name: %s;  Age: %d;\n", name, age);
    }

    public int compareTo(Person p){
     
        return name.compareTo(p.getName());
    }
}

В данном случае мы не возвращаем явным чином ниякое число, а опять же полагаемся на встроенный механизм сравнения, який есть у класса String.

Застосування в программе:


import java.util.ArrayList; 

public class Program{
      
    public static void main(String[] args) {
         
        var people = new ArrayList<Person>();
        people.add(new Person("Tom", 41));
        people.add(new Person("Bob", 46));
        people.add(new Person("Sam", 28));

        people.sort(null); // сортировка на основе реализации Comparable

        for(var p : people){
            p.print();
        }

    }
}
