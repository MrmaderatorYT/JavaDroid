Асинхронні завданьі FutureTask. Callable та Future

Останнє оновлення: 18.10.2025

Метод run() интрефейcа Runnable, який виконується потіком, не повертає ниякого результата и просто виконує некоторые действия. Однак иногда у потоці необхідно выполнять действия, которые производят неякий результат. И після завершення необхідно отримати из потіка цей результат и використовувати его якимось чином. Для цей завданьі язык Java надає інтерфейсы Callable и Future, которые визначенийы в пакеті java.util.concurrent.

Callable представляє некоторое действие, которое повертає неякий результат. Цет інтерфейс имїїт параметр типу и один метод - call(), який повертає значення цього типу:


public interface Callable<V>{

   V call() throws Exception;
}

То есть, грубо говоря, мы запускуємо у потоці об'єкт цього інтерфейса, а результат методу call() - це и буде то значення, которое мы получим из потіка.

Інтерфейс Future представляє результат асинхронной завданьі. Використовуючи об'єкт Future, можна отримати результат, наприклад, в результаті виконання методу call() інтерфейса Callable. Причем Future дозволяє отримати результат именно тогда, когда он буде готов.

Цет інтерфейс надає функционал для проверки завершення вычисления, ожидания его завершення и получения результата:

  • boolean cancel(boolean mayInterruptIfRunning)

    Пытается отменить виконання завданьі

  • default Throwable exceptionNow()

    Повертає виняток, сгенерированное завданьей, без ожидания

  • V get()

    При необхідности очікує завершення завданьі, а затем извлекает его результат

  • V get(long timeout, TimeUnit unit)

    При необхідности очікує завершення завданьі не болїї заданного времени, а затем извлекает її результат, якщо он доступен

  • boolean isCancelled()

    Повертає true, якщо завданьа бува отменена до її нормального завершення

  • boolean isDone()

    Повертає true, якщо завданьа завершена

  • default V resultNow()

    Повертає вычисленный результат без ожидания, неблокирующий вариант методу get()

  • default Future.State state()

    Повертає состояние завданьі. Представляє перечисление Future.State со наступними константами:

    • CANCELLED: завданьа отменена.

    • FAILED: завданьа завершилась неудачно, сгенерировав виняток.

    • RUNNING: завданьа виконується и ще не завершена.

    • SUCCESS: завданьа успешно завершена.

FutureTask

Один из способов запуску асинхронных завдань в виде Callable и Future представляє клас FutureTask. Цет клас представляє асинхронную завданьу, яку можна отменить. Цет клас надає базовую встроенную реализацию інтерфейса Future с методуми для запуску и отмены вычисления, запроса на завершення вычисления и получения результата вычисления.

Крім того, поскольку FutureTask реалізує Runnable, то об'єкты FutureTask можуть быть использованы для створенняя потіков.

Обычно FutureTask застосовується для обертки об'єкта Callable або Runnable для запуску завданьі, которая повинна быть выполнена в будущем. Для цього клас надає пару конструкторов:

  • FutureTask(Runnable runnable, V result)

    Створює FutureTask, який при запуску выполнит указанный Runnable и обеспечит возврат указанного результата методом get при успешном завершении.

  • FutureTask(Callable<V> callable)

    Створює FutureTask, який при запуску выполнит указанный Callable.

Функциональность класа FutureTask реалізуєся наступними методуми:

  • boolean cancel(boolean mayInterruptIfRunning)

    Пытается отменить виконання завданьі

  • Throwable exceptionNow()

    Повертає виняток, сгенерированное завданьей

  • V get()

    При необхідности очікує завершення завданьі и повертає результат завданьі

  • V get(long timeout, TimeUnit unit)

    При необхідности очікує завершення завданьі не болїї заданного времени, а затем извлекает его результат, якщо он доступен

  • boolean isCancelled()

    Повертає true, якщо завданьа бува отменена до її нормального завершення

  • boolean isDone()

    Повертає true, якщо завданьа завершена

  • V resultNow()

    Повертає результат завданьі без ожидания її завершення

  • void run()

    Встановлює використовуємоый об'єкт Future в качестве результата завданьі, якщо лише она не бува отменена

  • Future.State state()

    Повертає состояние завданьі

Ключевым є метод get(), який блокує виконання, якщо завданьа ще не завершена. И після завершення повертає її результат. Посмотрим на примере:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = 5;  // исходное число для вычисления факториала

        // определяем завданьу, которая вычисляет факториал
        Callable<Integer> task = () -> {

            int n = number;     // копируем число, чтобы не изменять его
            int result = 1;
            while(n > 0) result *= n--;
            return result;  // повертаємо результат завданьі
        };

        // определяем завданьу, которая буде выполнена в будущем
        var futureTask = new FutureTask<Integer>(task);
        // створюємо и запускуємо потік
        var t = new Thread(futureTask); // futureTask - це об'єкт Runnable
        t.start();

        // здесь можуть быть якие-або другие операції методу main
        // которые повинні выполняться одновременно с futureTask
        System.out.println("Main thread works...");

        try{
            // ожидаем виконання завданьі
            Integer factorial = futureTask.get();
            System.out.printf("factorial of %d is %d\n", number, factorial);
        }
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Консольный вивід програми:


Main thread started...
Main thread works...
factorial of 5 is 120
Main thread finished...

Разберем эту програму. Прежде всего определяем завданьу Callable, которая буде выполняться, в виде лямбда-выражения (мы можемо так сделать, потому что Callable имїїт лише один метод):


Callable<Integer> task = () -> {

    int n = number;     // копируем число, чтобы не изменять его
    int result = 1;
    while(n > 0) result *= n--;
    return result;  // повертаємо результат завданьі
};

Здесь простейший код виконання факториала для числа number, которое визначенийо во внешнем коді (лямбда-выражения можуть обращаться в нешним переменным). В конце лямбда-выражение повертає результат - значення змінної result.

Далїї определяем завданьу, которая буде выполнена в будущем - об'єкт FutureTask, передавая в его конструктор выше визначенийную завданьу Callable:


var futureTask = new FutureTask<Integer>(task);

Поскольку завданьа повертає значення типу int, преобразуемое в Integer, об'єкт FutureTask (як и Callable) типизируется типом Integer

Далїї для виконання завданьі створюємо и запускуємо потік:


var t = new Thread(futureTask); // futureTask - це об'єкт Runnable
t.start();

Поскольку об'єкт FutureTask реалізує інтерфейс Runnable, то мы можемо передать его в конструктор класа Thread.

Поки дочерний потік t виконує свою завданьу - вычисляет факториала, основной потік також может выполнять некоторые действия. У даному випадку для примера просто вивідим сообщение на консоль:


System.out.println("Main thread works...");

Затем для получения результата завданьі у FutureTask вызываем метод get()


try{
    // ожидаем виконання завданьі
    Integer factorial = futureTask.get();
    System.out.printf("factorial of %d is %d\n", number, factorial);
}
catch(Exception ex){ 
    System.out.println(ex.getMessage()); 
}

Цет метод генерує ряд винятків, поцему оборачиваем его вызов в конструкцию try..cath. В итоге за допомогою методу get() мы получим результат и выведем его на консоль.

Варто відзначити, что при вызове методу get()

Integer factorial = futureTask.get();

Главный потік - метод main приостанавливает виконання и ждет получения результата (або поки не произовдет ошибка)

Генерація та обробка помилки

Но выше визначенийный способ вычисления факториала имїїт большой недостаток - якщо переданное число буде менше 1, то завданьа возвратит число 1, что буде неправильно: для вычисления факториала число повинно быть больше 0. И мы можемо предупредить эту ситуацию, сгенерировав ошибку. И варто відзначити, что при метод call() інтерфейса Callable вже в своем визначенийие предусматривает возможную генерацию ошибки:

V call() throws Exception;

И також варто відзначити, что якщо FutureTask генерує виняток, то оно имїїт тип ExecutionException. Перехватывая винятки цього типу, мы можемо понять, что виняток произошло в методі call при выполнении FutureTask.

Наприклад, сгенерируем ошибку, якщо входное значення для факториала менше 1:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = -5;  // исходное число для вычисления факториала
        try{
            // определяем завданьу, которая вычисляет факториал
            Callable<Integer> task = () -> {

                // якщо число менше 1, генерируем виняток
                if(number < 1) throw new Exception("Number must be greater than 0");
                int n = number;     // копируем число, чтобы не изменять его
                int result = 1;
                while(n > 0) result *= n--;
                return result;  // повертаємо результат завданьі
            };

            var futureTask = new FutureTask<Integer>(task);
            var t = new Thread(futureTask);
            t.start();

            System.out.println("Main thread works...");
       
            // ожидаем виконання завданьі
            Integer factorial = futureTask.get();
            System.out.printf("factorial of %d is %d\n", number, factorial);
        }
        // перехватываем ошибки виконання FutureTask
        catch(ExecutionException ex){

            System.out.println("Factorial error: " + ex.getMessage());
        }
        // перехватываем все остальные ошибки
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Здесь перед вычислением факториала проверяем число и при необхідности генерируем ошибку:


if(number < 1) throw new Exception("Number must be greater than 0");

Для обработки ошибки виконання во FutureTask определяем специальный блок catch:


catch(ExecutionException ex){

    System.out.println("Factorial error: " + ex.getMessage());
}

И поскольку здесь для вычисления факториала передаєся заведомо некорректное число - -5, то мы столкнемся с ошибкой:


Main thread started...
Main thread works...
Factorial error: java.lang.Exception: Number must be greater than 0
Main thread finished...

Варто відзначити, что информацию о возникшем исключении мы можемо отримати и из об'єкта FutureTask через метод exceptionNow(). Наприклад:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = -5;  // исходное число для вычисления факториала

        FutureTask<Integer> futureTask = null;  // определяем завданьу

        try{
            ..............................................

            futureTask = new FutureTask<Integer>(task);

            ..............................................
        }
        // перехватываем ошибки виконання FutureTask
        catch(ExecutionException ex){

            var ex = futureTask.exceptionNow();  // отримуємо виняток
            System.out.println("Factorial error: " + ex.getMessage());
        }
        // перехватываем все остальные ошибки
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Це может быть полезно, когда у нас декілька об'єктов FutureTask, и каждый из которых может сгенерировать винятки, которые треба як-то разграничить.

Отмена завдань

Одним из достоинств інтерфейса Future є то, что підобную завданьу можна отменять. Для цього призначений метод cancel():


boolean cancel(boolean mayInterruptIfRunning)

Если завданьа вже запущена, то параметр mayInterruptIfRunning визначає, переривається ли потік завданьі. Підзвращаемое значення: false, якщо завданьу нельзя отменить (обычно потому, что она вже завершена), або true, якщо можна отменить. Но данный результат не обязательно вказує, отменена ли завданьа - для цей цели у Future використовується метод isCancelled().

Цет метод не действует, якщо завданьа вже завершена або отменена або не может быть отменена по якой-або другой причине. В противном случае, якщо эта завданьа не запустилась при вызове отмены, эта завданьа никогда не повинна запускуться.

Рассмотрим небольшой пример:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = 5;  // исходное число для вычисления факториала

        try{
            // определяем завданьу, которая вычисляет факториал
            Callable<Integer> task = () -> {

                if(number < 1) throw new Exception("Number must be greater than 0");
                int n = number;     // копируем число, чтобы не изменять его
                int result = 1;
                while(n > 0) {

                    result *= n--;
                    System.out.printf("[Factorial execution] result = %d   n = %d\n", result, n);
                    Thread.sleep(200);
                }
                return result;
            };

            var futureTask = new FutureTask<Integer>(task);
            // створюємо и запускуємо потік
            var t = new Thread(futureTask);
            t.start();

            System.out.println("Main thread works...");
            Thread.sleep(300); // имитируем работу, чтобы futureTask успела немного попрацювати

            // в конце отменяем завданьу
            futureTask.cancel(true);

            // проверяем отмену завданьі
            if(futureTask.isCancelled()) System.out.println("Factorial execution cancelled...");
        }
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Здесь для демонстрации в коді завданьі Callable делаем небольшую задержку в 200 миллисекунд:


while(n > 0) {

    result *= n--;
    System.out.printf("[Factorial execution] result = %d   n = %d\n", result, n);
    Thread.sleep(200);
}

В основном потіке main останавливаем завданьу, причем в прекращением работы потіка, и проверяем її статус остановки:


// отменяем завданьу
futureTask.cancel(true);

// проверяем отмену завданьі
if(futureTask.isCancelled()) System.out.println("Factorial execution cancelled...");

Консольный вивід програми:


Main thread started...
Main thread works...
[Factorial execution] result = 5   n = 4
[Factorial execution] result = 20   n = 3
Factorial execution cancelled...
Main thread finished...

Здесь мы видим, что після двух итераций цикла в завданье факториала завданьа бува отменена и прекратила свое виконання.
