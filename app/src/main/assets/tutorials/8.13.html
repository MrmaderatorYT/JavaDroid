Асинхронні задачі FutureTask. Callable та Future

Останнє оновлення: 18.10.2025

Метод run() интрефейcа Runnable, який виконується потоком, не повертає ниякого результата и просто выполняет некоторые действия. Однак иногда в потоке необхідно выполнять действия, которые производят неякий результат. И после завершения необхідно отримати из потока цей результат и використовувати его якимось чином. Для цей задачи язык Java надає інтерфейсы Callable и Future, которые определены в пакеті java.util.concurrent.

Callable представляє некоторое действие, которое повертає неякий результат. Цет інтерфейс имїїт параметр типу и один метод - call(), який повертає значение цього типу:


public interface Callable<V>{

   V call() throws Exception;
}

То есть, грубо говоря, мы запускаем в потоке об'єкт цього інтерфейса, а результат метода call() - це и буде то значение, которое мы получим из потока.

Интерфейс Future представляє результат асинхронной задачи. Используя об'єкт Future, можна отримати результат, наприклад, в результате виконання метода call() інтерфейса Callable. Причем Future дозволяє отримати результат именно тогда, когда он буде готов.

Цет інтерфейс надає функционал для проверки завершения вычисления, ожидания его завершения и получения результата:

  • boolean cancel(boolean mayInterruptIfRunning)

    Пытается отменить выполнение задачи

  • default Throwable exceptionNow()

    Повертає исключение, сгенерированное задачей, без ожидания

  • V get()

    При необхідности очікує завершения задачи, а затем извлекает его результат

  • V get(long timeout, TimeUnit unit)

    При необхідности очікує завершения задачи не болїї заданного времени, а затем извлекает її результат, якщо он доступен

  • boolean isCancelled()

    Повертає true, якщо задача бува отменена до її нормального завершения

  • boolean isDone()

    Повертає true, якщо задача завершена

  • default V resultNow()

    Повертає вычисленный результат без ожидания, неблокирующий вариант метода get()

  • default Future.State state()

    Повертає состояние задачи. Представляє перечисление Future.State со наступними константами:

    • CANCELLED: задача отменена.

    • FAILED: задача завершилась неудачно, сгенерировав исключение.

    • RUNNING: задача виконується и ще не завершена.

    • SUCCESS: задача успешно завершена.

FutureTask

Один из способов запуска асинхронных задач в виде Callable и Future представляє класс FutureTask. Цет класс представляє асинхронную задачу, яку можна отменить. Цет класс надає базовую встроенную реализацию інтерфейса Future с методами для запуска и отмены вычисления, запроса на завершение вычисления и получения результата вычисления.

Крім того, поскольку FutureTask реализует Runnable, то об'єкты FutureTask можуть быть использованы для создания потоков.

Обычно FutureTask застосовується для обертки об'єкта Callable або Runnable для запуска задачи, которая повинна быть выполнена в будущем. Для цього класс надає пару конструкторов:

  • FutureTask(Runnable runnable, V result)

    Створює FutureTask, який при запуске выполнит указанный Runnable и обеспечит возврат указанного результата методом get при успешном завершении.

  • FutureTask(Callable<V> callable)

    Створює FutureTask, який при запуске выполнит указанный Callable.

Функциональность класса FutureTask реализуется наступними методами:

  • boolean cancel(boolean mayInterruptIfRunning)

    Пытается отменить выполнение задачи

  • Throwable exceptionNow()

    Повертає исключение, сгенерированное задачей

  • V get()

    При необхідности очікує завершения задачи и повертає результат задачи

  • V get(long timeout, TimeUnit unit)

    При необхідности очікує завершения задачи не болїї заданного времени, а затем извлекает его результат, якщо он доступен

  • boolean isCancelled()

    Повертає true, якщо задача бува отменена до її нормального завершения

  • boolean isDone()

    Повертає true, якщо задача завершена

  • V resultNow()

    Повертає результат задачи без ожидания її завершения

  • void run()

    Устанавливает используемый об'єкт Future в качестве результата задачи, якщо лише она не бува отменена

  • Future.State state()

    Повертає состояние задачи

Ключевым є метод get(), який блокує выполнение, якщо задача ще не завершена. И после завершения повертає її результат. Посмотрим на примере:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = 5;  // исходное число для вычисления факториала

        // определяем задачу, которая вычисляет факториал
        Callable<Integer> task = () -> {

            int n = number;     // копируем число, чтобы не изменять его
            int result = 1;
            while(n > 0) result *= n--;
            return result;  // возвращаем результат задачи
        };

        // определяем задачу, которая буде выполнена в будущем
        var futureTask = new FutureTask<Integer>(task);
        // создаем и запускаем поток
        var t = new Thread(futureTask); // futureTask - це об'єкт Runnable
        t.start();

        // здесь можуть быть якие-або другие операции метода main
        // которые повинні выполняться одновременно с futureTask
        System.out.println("Main thread works...");

        try{
            // ожидаем виконання задачи
            Integer factorial = futureTask.get();
            System.out.printf("factorial of %d is %d\n", number, factorial);
        }
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Консольный вывод программы:


Main thread started...
Main thread works...
factorial of 5 is 120
Main thread finished...

Разберем эту программу. Прежде всего определяем задачу Callable, которая буде выполняться, в виде лямбда-выражения (мы можемо так сделать, потому что Callable имїїт лише один метод):


Callable<Integer> task = () -> {

    int n = number;     // копируем число, чтобы не изменять его
    int result = 1;
    while(n > 0) result *= n--;
    return result;  // возвращаем результат задачи
};

Здесь простейший код виконання факториала для числа number, которое определено во внешнем коде (лямбда-выражения можуть обращаться в нешним переменным). В конце лямбда-выражение повертає результат - значение змінної result.

Далїї определяем задачу, которая буде выполнена в будущем - об'єкт FutureTask, передавая в его конструктор выше определенную задачу Callable:


var futureTask = new FutureTask<Integer>(task);

Поскольку задача повертає значение типу int, преобразуемое в Integer, об'єкт FutureTask (як и Callable) типизируется типом Integer

Далїї для виконання задачи создаем и запускаем поток:


var t = new Thread(futureTask); // futureTask - це об'єкт Runnable
t.start();

Поскольку об'єкт FutureTask реализует інтерфейс Runnable, то мы можемо передать его в конструктор класса Thread.

Поки дочерний поток t выполняет свою задачу - вычисляет факториала, основной поток також может выполнять некоторые действия. В данном случае для примера просто выводим сообщение на консоль:


System.out.println("Main thread works...");

Затем для получения результата задачи у FutureTask вызываем метод get()


try{
    // ожидаем виконання задачи
    Integer factorial = futureTask.get();
    System.out.printf("factorial of %d is %d\n", number, factorial);
}
catch(Exception ex){ 
    System.out.println(ex.getMessage()); 
}

Цет метод генерує ряд исключений, поцему оборачиваем его вызов в конструкцию try..cath. В итоге за допомогою метода get() мы получим результат и выведем его на консоль.

Варто відзначити, что при вызове метода get()

Integer factorial = futureTask.get();

Главный поток - метод main приостанавливает выполнение и ждет получения результата (або поки не произовдет ошибка)

Генерация и обработка ошибки

Но выше определенный способ вычисления факториала имїїт большой недостаток - якщо переданное число буде меньше 1, то задача возвратит число 1, что буде неправильно: для вычисления факториала число повинно быть больше 0. И мы можемо предупредить эту ситуацию, сгенерировав ошибку. И варто відзначити, что при метод call() інтерфейса Callable вже в своем определение предусматривает возможную генерацию ошибки:

V call() throws Exception;

И також варто відзначити, что якщо FutureTask генерує исключение, то оно имїїт тип ExecutionException. Перехватывая исключения цього типу, мы можемо понять, что исключение произошло в методе call при выполнении FutureTask.

Наприклад, сгенерируем ошибку, якщо входное значение для факториала меньше 1:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = -5;  // исходное число для вычисления факториала
        try{
            // определяем задачу, которая вычисляет факториал
            Callable<Integer> task = () -> {

                // якщо число меньше 1, генерируем исключение
                if(number < 1) throw new Exception("Number must be greater than 0");
                int n = number;     // копируем число, чтобы не изменять его
                int result = 1;
                while(n > 0) result *= n--;
                return result;  // возвращаем результат задачи
            };

            var futureTask = new FutureTask<Integer>(task);
            var t = new Thread(futureTask);
            t.start();

            System.out.println("Main thread works...");
       
            // ожидаем виконання задачи
            Integer factorial = futureTask.get();
            System.out.printf("factorial of %d is %d\n", number, factorial);
        }
        // перехватываем ошибки виконання FutureTask
        catch(ExecutionException ex){

            System.out.println("Factorial error: " + ex.getMessage());
        }
        // перехватываем все остальные ошибки
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Здесь перед вычислением факториала проверяем число и при необхідности генерируем ошибку:


if(number < 1) throw new Exception("Number must be greater than 0");

Для обработки ошибки виконання во FutureTask определяем специальный блок catch:


catch(ExecutionException ex){

    System.out.println("Factorial error: " + ex.getMessage());
}

И поскольку здесь для вычисления факториала передаєся заведомо некорректное число - -5, то мы столкнемся с ошибкой:


Main thread started...
Main thread works...
Factorial error: java.lang.Exception: Number must be greater than 0
Main thread finished...

Варто відзначити, что информацию о возникшем исключении мы можемо отримати и из об'єкта FutureTask через метод exceptionNow(). Например:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = -5;  // исходное число для вычисления факториала

        FutureTask<Integer> futureTask = null;  // определяем задачу

        try{
            ..............................................

            futureTask = new FutureTask<Integer>(task);

            ..............................................
        }
        // перехватываем ошибки виконання FutureTask
        catch(ExecutionException ex){

            var ex = futureTask.exceptionNow();  // получаем исключение
            System.out.println("Factorial error: " + ex.getMessage());
        }
        // перехватываем все остальные ошибки
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Це может быть полезно, когда у нас декілька об'єктов FutureTask, и каждый из которых может сгенерировать исключения, которые треба як-то разграничить.

Отмена задач

Одним из достоинств інтерфейса Future є то, что подобную задачу можна отменять. Для цього призначений метод cancel():


boolean cancel(boolean mayInterruptIfRunning)

Если задача вже запущена, то параметр mayInterruptIfRunning определяет, переривається ли поток задачи. Підзвращаемое значение: false, якщо задачу нельзя отменить (обычно потому, что она вже завершена), або true, якщо можна отменить. Но данный результат не обязательно указывает, отменена ли задача - для цей цели у Future використовується метод isCancelled().

Цет метод не действует, якщо задача вже завершена або отменена або не может быть отменена по якой-або другой причине. В противном случае, якщо эта задача не запустилась при вызове отмены, эта задача никогда не повинна запускаться.

Рассмотрим небольшой пример:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        int number = 5;  // исходное число для вычисления факториала

        try{
            // определяем задачу, которая вычисляет факториал
            Callable<Integer> task = () -> {

                if(number < 1) throw new Exception("Number must be greater than 0");
                int n = number;     // копируем число, чтобы не изменять его
                int result = 1;
                while(n > 0) {

                    result *= n--;
                    System.out.printf("[Factorial execution] result = %d   n = %d\n", result, n);
                    Thread.sleep(200);
                }
                return result;
            };

            var futureTask = new FutureTask<Integer>(task);
            // создаем и запускаем поток
            var t = new Thread(futureTask);
            t.start();

            System.out.println("Main thread works...");
            Thread.sleep(300); // имитируем работу, чтобы futureTask успела немного поработать

            // в конце отменяем задачу
            futureTask.cancel(true);

            // проверяем отмену задачи
            if(futureTask.isCancelled()) System.out.println("Factorial execution cancelled...");
        }
        catch(Exception ex){ 
            System.out.println(ex.getMessage()); 
        }

        System.out.println("Main thread finished...");
    }
}

Здесь для демонстрации в коде задачи Callable делаем небольшую задержку в 200 миллисекунд:


while(n > 0) {

    result *= n--;
    System.out.printf("[Factorial execution] result = %d   n = %d\n", result, n);
    Thread.sleep(200);
}

В основном потоке main останавливаем задачу, причем в прекращением работы потока, и проверяем її статус остановки:


// отменяем задачу
futureTask.cancel(true);

// проверяем отмену задачи
if(futureTask.isCancelled()) System.out.println("Factorial execution cancelled...");

Консольный вывод программы:


Main thread started...
Main thread works...
[Factorial execution] result = 5   n = 4
[Factorial execution] result = 20   n = 3
Factorial execution cancelled...
Main thread finished...

Здесь мы видим, что после двух итераций цикла в задаче факториала задача бува отменена и прекратила свое выполнение.
