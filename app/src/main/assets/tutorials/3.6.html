Абстрактні класи

Останнє оновлення: 26.09.2025

Кроме обычных класов в Java есть абстрактні класи. Абстрактный клас похож на обычный клас. В абстрактном класе також можна визначити поля и методи, но в то же час нельзя створити об'єкт або экземпляр абстрактного класа. Абстрактні класи призваны предоставлять базовый функционал для класов-наследников. А производные класы вже реалізують цей функционал.

При визначенийии абстрактных класов використовується ключевое слово abstract:


abstract class ім'я_класа{

  // вміст класа
}

Зазвичай абстрактні класи створюються для таких сутностей, які репрезентують лише абстракцію і насправді не мають конкретного втілення. Хрестоматійним прикладом є система геометричних фігур. Насправді немає геометричної постаті як такої. Є коло, прямокутник, квадрат, але просто постаті немає. Однак і коло, і прямокутник мають щось спільне і є фігурами.

Також абстрактні класи можуть представляти і, на перший погляд, конкретні сутності, але об'єкти яких не створюватимуться у програмі та мають бути лише основою для створення класів-спадкоємців.

Наприклад, визначимо абстрактний клас геометричної фігури:


abstract class Shape{
    
    int x; // x-координата точки
    int y; // y-координата точки
 
    Shape(int x, int y){
        
        this.x = x;
        this.y = y;
    }

    void print(){ System.out.println("Геометрическая фигура"); }
}

Застосування абстрактних класів

Геометрические фигуры, як правило, имеют деяку точку, относительно которой располагается фигура. Наприклад, для прямоугольника це обычно координаты верхнего левого угла, для круга - це центр и т.д. И у даному випадку за допомогою полей x и y сохраняем эти координаты, которые передаються через конструктор. А для вивіда информации о фигуре призначений метод print()

Особливістю абстрактних класів і те, що ми можемоо використовувати їх конструктор безпосередньо до створення об'єкта класа:


class Program{
      
    public static void main(String[] args) {

        // так нельзя
        Shape someShape = new Shape(10, 15);  // ! Ошибка: Shape is abstract; cannot be instantiated
    }
}

Ми можемоо викликати конструктор абстрактного класу лише у похідних класах. Створимо пару похідних класів:


// абстрактный клас фигуры
abstract class Shape{
    
    int x; // x-координата точки
    int y; // y-координата точки
 
    Shape(int x, int y){
        
        this.x = x;
        this.y = y;
    }

    void print(){ System.out.println("Геометрическая фигура"); }
}
// производный клас прямоугольника
class Rectangle extends Shape
{
    int width;
    int height;
 
    // конструктор с обращением к конструктору класа Figure
    Rectangle(int x, int y, int width, int height){
        
        super(x,y);

        this.width = width;
        this.height = height;
    }
  
	  @Override
    void print(){ 
        System.out.printf("Пpямоугольник. Верхний левый угол: (%d, %d); Ширина: %d; Длина: %d\n", x, y, width, height);
    }
}
// производный клас круга
class Circle extends Shape
{
    int radius;
 
    // конструктор с обращением к конструктору класа Shape
    Circle(int x, int y, int radius){
        
        super(x,y);
        this.radius = radius;
    }
	
	  @Override
    void print(){ 
        System.out.printf("Круг. Центр: (%d, %d); Радиус: %d\n", x, y, radius);
    }
}

У даному випадку визначенийы клас прямоугольника и клас круга. Они наследуют функционал абстрактного класа, добавляя дополнительные поля типу ширины и длины або радиуса, а також переопределяя метод print(). Далїї мы можемо використовувати об'єкты цих класов як об'єкты типу Shape:


class Program{
      
    public static void main(String[] args) {

        Shape shape1 = new Rectangle(10, 15, 200, 150);
        Shape shape2 = new Circle(50, 60, 200);

        printShape(shape1);
        printShape(shape2);
    }

    static void printShape(Shape shape){
        shape.print();
    }
}

Консольний висновок:


Пpямоугольник. Верхний левый угол: (10, 15); Ширина: 200; Длина: 150
Круг. Центр: (50, 60); Радиус: 200

Абстрактні методи

Наряду с абстрактными класами в мові Java есть абстрактные методи. Такие методи також предваряются модификатором abstract, но при цем не имеют тела - не виконують нияких конкретных действий:


abstract тип_методу ім'я_методу(параметри_методу);

Похідний клас повинен перевизначити і реалізувати всі абстрактні методи, які є в базовому абстрактному класі. Також слід враховувати, що якщо клас має хоча б один абстрактний метод, то цей клас має бути визначений як Анотація.

Наприклад, в примере с фигурами выше метод print() в класе Shape не виконує якой-то полезной нагрузки. Производные класы его в любом случае перевизначають, вивідя в нем свою информацию. И мы можемо сделать его абстрактным:


abstract class Shape{
    
    int x;
    int y;
 
    Shape(int x, int y){
        
        this.x = x;
        this.y = y;
    }
    // абстрактный метод
    abstract void print();
}
// прямоугольник
class Rectangle extends Shape
{
    int width;
    int height;

    Rectangle(int x, int y, int width, int height){
        
        super(x,y);
        this.width = width;
        this.height = height;
    }
	  @Override
    void print(){ 
        System.out.printf("Пpямоугольник. Верхний левый угол: (%d, %d); Ширина: %d; Длина: %d\n", x, y, width, height);
    }
}
// круг
class Circle extends Shape
{
    int radius;
 
    Circle(int x, int y, int radius){
        
        super(x,y);
        this.radius = radius;
    }
	  @Override
    void print(){ 
        System.out.printf("Круг. Центр: (%d, %d); Радиус: %d\n", x, y, radius);
    }
}

Причем производные НЕабстрактные класы обязаны реализовать абстрактный метод.

Іншим прикладом абстрактних методів можуть бути такі дії, які знову ж таки не мають конкретного втілення, є абстракцією і залежать від конкретного класу. Наприклад, у геометричної фігури можна обчислити периметр та площу. Але для прямокутника ці покизники обчислюватимуться одним чином, для кола - іншим чином і т.д. Для абстрактної геометричної фігури ми не можемоо визначити універсальний спосіб обчислення периметра та площі, тому такі дії можна визначити у вигляді абстрактних методів. А конкретні класи за своїм реалізують їх:


abstract class Shape{
    
    int x;
    int y;
 
    Shape(int x, int y){
        
        this.x = x;
        this.y = y;
    }
    // абстрактный метод для вычисления периметра
    abstract double getPerimeter();
    // абстрактный метод для вычисления площади
    abstract double getArea();
}
// прямоугольник
class Rectangle extends Shape
{
    int width;
    int height;

    Rectangle(int x, int y, int width, int height){
        
        super(x,y);
        this.width = width;
        this.height = height;
    }
	@Override
    double getPerimeter(){ return this.width * 2 + this.height * 2; }

	@Override
    double getArea(){ return this.width * this.height; }
}
// круг
class Circle extends Shape
{
    int radius;
 
    Circle(int x, int y, int radius){
        
        super(x,y);
        this.radius = radius;
    }
    
	@Override
    double getPerimeter(){ return this.radius * 3.14 * 2; }

	@Override
    double getArea(){ return this.radius * 3.14; }
}

Застосування класів:


class Program{
      
    public static void main(String[] args) {

        Shape shape1 = new Rectangle(10, 15, 200, 150);
        System.out.println("Площадь прямоугольника: " + shape1.getArea());
        System.out.println("Периметер прямоугольника: " + shape1.getPerimeter());

        Shape shape2 = new Circle(50, 60, 100);
        System.out.println("Площадь круга: " + shape2.getArea());
        System.out.println("Периметер круга: " + shape2.getPerimeter());
    }
}

Площадь круга: 314.0
Периметер круга: 628.0
