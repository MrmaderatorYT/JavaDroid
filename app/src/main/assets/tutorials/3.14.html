Об'єкти як параметри методів

Останнє оновлення: 16.09.2025

У минулій темі було розглянуто передачу параметрів у метод. Об'єкти класів, як і дані примітивних типів, можуть передаватися в методи. Однак у цьому випадку є одна особливість - при передачі об'єктів як значення передається копія посилання область в пам'яті, де розташований об'єкт.

Сначала розглянемо болїї детально, что у нас происходит при передача параметрів примитивных типів - числовых типів як int, double и т.д., типу char и boolean:


class Program{

    public static void main(String[] args) {  

        int num = 22;
        Calcualtor calc = new Calcualtor();
        calc.twice(num);
        System.out.printf("num in main (after twice): %d\n", num);
    }
}
class Calcualtor{

    void twice(int n){
		
        n = n + n;
        System.out.printf("n in twice: %d\n", n);
	}
}

Здесь в методі main значення змінної num передаєся в метод calc.twice() параметру n:


calc.twice(num);

В методі calc.twice() значення параметра n (которому передаєся значення змінної num) увеличивается в два раза:


void twice(int n){
		
    n = n + n;
    System.out.printf("n in twice: %d\n", n);
}

И тут возникает главный вопрос: увеличится ли значення змінної num або нет після виконання методу calc.twice() и почему? Посмотрим на консольный вивід програми:


n in twice: 44
num in main (after twice): 22

По результату програми мы видим, что значення параметра n в методі twice увеличилось в два раза (як и ожидалось), а значення змінної num після виконання методу twice не изменилось. При вызове методу calc.twice() параметр n в цем методі отримує копію значення змінної num, то есть число 22:

Когда вызывается метод calc.twice(), в область памяти, которая призначенийа для параметра n, копируется значення змінної num - то есть число 22. Після цього параметр n и змінна num нияк не связаны - у них разные участки памяти, и зміна значениия одного из них нияк не сказывается на другом. Тому зміна параметра n нияк не влияет на змінну num.

Особливістю змінних інших типів є те, що вони зберігають посилання на область пам'яті, де розташований об'єкт. Розглянемо невеликий приклад:


class Program{

    public static void main(String[] args) {  

        Person bob = new Person();
        bob.age = 22;
        System.out.printf("bob.age in main (before twice): %d\n", bob.age);

        Calcualtor calc = new Calcualtor();
        calc.twice(bob);

        System.out.printf("bob.age in main (after twice): %d\n", bob.age);
    }
}
class Person{ 

    int age;
}

class Calcualtor{

    void twice(Person p){
		
        p.age = p.age + p.age;
        System.out.printf("p.age in twice: %d\n", p.age);
	}
}

Тут метод twice класу Calculator тепер приймає об'єкт типу Person, який має ціле поле age. І в методі twice аналогічно це поле збільшується вдвічі.


 void twice(Person p){
		
    p.age = p.age + p.age;
    System.out.printf("p.age in twice: %d\n", p.age);
}

В методі main определяем змінну bob типу Person и передаємо її значення в метод twice:


Person bob = new Person();
.......................
calc.twice(bob);

Погляньмо на консольний висновок програми:


bob.age in main (before twice): 22
p.age in twice: 44
bob.age in main (after twice): 44

Тут бачимо, що значення поля age у об'єкта, який вказує змінна bob, змінилася. Чому? Так як метод twice по суті передається копія посилання на область пам'яті, в якій знаходиться об'єкт Person, то змінна bob і параметр p методу twice будутьь вказувати на той самий об'єкт у пам'яті:

Тому після виконання методу об'єкт, адреса якого зберігає змінна bob і який передається в метод, буде змінено значення age c 22 на 44.

Від цього випадку слід відрізняти інший випадок:


class Program{

    public static void main(String[] args) {  

        Person bob = new Person();
        bob.age = 22;
        System.out.printf("bob.age in main (before twice): %d\n", bob.age);

        Calcualtor calc = new Calcualtor();
        calc.twice(bob);

        System.out.printf("bob.age in main (after twice): %d\n", bob.age);
    }
}
class Person{ 

    int age;
}

class Calcualtor{

    void twice(Person p){
		
        p = new Person();      // створюємо новый об'єкт
        p.age = 44;
        System.out.printf("p.age in twice: %d\n", p.age);
	}
}

У метод twice також передається копія посилання об'єкт Person. Однак у самому методі ми змінюємо окремі значення об'єкта, а перетворюємо об'єкт за допомогою конструктора та оператора new. В результаті в пам'яті буде виділено нове місце для нового об'єкта Person, і посилання на цей об'єкт буде надано параметру p:


void twice(Person p){
		
    p = new Person();      // створюємо новый об'єкт
    p.age = 44;
    System.out.printf("p.age in twice: %d\n", p.age);
}

Тобто після створення нового об'єкта Person параметр p та змінна bob у методі main зберігатимуть посилання на різні об'єкти. Змінна bob, яка передавалася в метод, продовжуватиме зберігати посилання на старий об'єкт у пам'яті. Тому її значення не змінюється, що ми побачимо з консольного висновку:


bob.age in main (before twice): 22
p.age in twice: 44
bob.age in main (after twice): 22
