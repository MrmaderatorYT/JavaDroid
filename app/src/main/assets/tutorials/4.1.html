Оператор throws и throw

Останнє оновлення: 06.10.2025

Генерация исключения и оператор throw

Обычно система сама генерує исключения при определенных ситуациях, наприклад, при делении числа на ноль. Но язык Java також дозволяє генерировать исключения вручную за допомогою оператора throw. Наприклад, в нашей программе пользователь повинен вводить имя, но имя имїїт свои ограничения. Наприклад, очевидно, что имя из одного символа - це некорректно имя. Підзможна, мы пред'ємо ще якие-то ограничения к имени. И в цем случае мы можемо при вводе некорректных данных генерировать исключение.

Общая форма оператора:


throw об'єкт_исключения;

То есть после оператора указывается об'єкт исключения. Как его создать? Прежде всего мы можемо використовувати встроенные классы исключений, в том числе базовые - Throwable и Exception. Так, один из конструкторов типу Throwable приймає сообщение об ошибке:


Throwable(String message)

По сути це то сообщение, которое повертаєся методом getMessage() и которое при возникновении ошибки отображается на консоли. Ну и соответственно все другие типы исключений також имеют подобный конструктор.

Рассмотрим генерацию исключения на примере:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        Scanner in = new Scanner(System.in);
        System.out.print("Введите имя: ");
        String name = in.nextLine(); // считываем введенную строку

        // якщо строка имїїт имїїт меньше 2-х символов, генерируем исключение
        if(name.length() < 2) throw new Throwable("Малобукоф");

        System.out.println("Привет, " + name);
    }
}

Здесь за допомогою класса Scanner и метода in.nextLine() считываем с консоли строку - условное имя пользователя. Затем, используя метол length() у строки смотрим на длину - якщо в имени меньше 2 символов, то генерируем исключение:

if(name.length() < 2) throw new Throwable("Малобукоф");

В конструктор класса Throwable передаем сообщение об ошибке.

Но якщо мы попробуем скомпилировать программу, нас постигнет неудача:


Program.java:12: error: unreported exception Throwable; must be caught or declared to be thrown
        if(name.length() < 2) throw new Throwable("Малобукоф");
                              ^
1 error

Компилятор нам говорит, что нам треба або обработать исключение, або указать, что метод main генерує данное исключение.

Проверяемые и непроверяемые исключения

КАк говорилось в прошлой статье, исключения в Java делятся условно на две группы: проверяемые (checked exceptions) и непроверяемые (unchecked exceptions). К непроверяемым исключениям относят классы Error и RuntimeException и унаследованные от них классы. А проверяемые исключения по сути все остальные. И в контексте работы с оператором throw подобное разделение имїїт большое значение. При генерации проверяемых исключений нам треба або указать, что метод генерує исключение, або обработать це исключение. Если же генеруються непроверяемые исключения, то ничего цього делать не треба.

В нашем примере выше генерируемое исключение типу Throwable рассматривается компилятором як проверяемое исключение (хотя классы Error и RuntimeException унаследованы от Throwable). Тому и при компиляции возникает ошибка.

Для решения проблемы изменим код и обработаем исключение за допомогою рассмотренной в прошлых статьях конструкции try..catch:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        Scanner in = new Scanner(System.in);
        System.out.print("Введите имя: ");
        String name = in.nextLine(); // считываем введенную строку

        try{

            // якщо строка имїїт имїїт меньше 2-х символов, генерируем исключение
            if(name.length() < 2) throw new Throwable("Малобукоф");
            System.out.println("Привет, " + name);
        }
        catch(Throwable ex){
            System.out.println("Підзникла ошибка: " + ex.getMessage());
        }
    }
}

Здесь та, часть кода, которая генерує исключения и которая зависит от проверки змінної name, помещается в блок try. Если буде сгенерировано исключение, то управление перейдет к блоку catch, где мы перехватим исключение и выведем сообщение об ошибке.

Пример работы программы:


Введите имя: e
Підзникла ошибка: Малобукоф

Обработка других типов исключений

Подобным чином мы можемо генерировать исключения в любом месте программы. Крім того, мы можемо використовувати и другие встроенные типы исключений. Наприклад, встроенный тип IllegalArgumentException призначений для генерации исключения при некорректном параметре метода. Используем данный тип:


class Program{
     
	public static void main(String[] args) {
         
        var tom = new Person("Tom", -20);
        tom.print();
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age){

        if(age < 1 || age > 110) throw new IllegalArgumentException("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Здесь в конструктор класса Person через параметр age передаєся возраст человека. Однак не все значения типу int можуть быть разумными значениями для возраста. Наприклад, возраст не может быть отрицательным. И в конструкторе класса Person проверяем значение параметра age на соответствие некоторому диапазону. Ив случае несоответствия, генерируем исключение типу IllegalArgumentException


if(age < 1 || age > 110) 
    throw new IllegalArgumentException("Некорректный возраст: " + age);

В методе main передаем в конструктор Person заведомо некорректное значение:

var tom = new Person("Tom", -20);

В итоге при запуске программы и создании цього об'єкта буде сгенерировано исключение


Exception in thread "main" java.lang.IllegalArgumentException: Некорректный возраст: -20
	at Person.<init>(Program.java:20)
	at Program.main(Program.java:7)

Обратите внимание, что при компиляции программы компилятор не выдал ошибку, як при компиляции предыдущего примера. Потому что в данном случае мы обрабатываем исключение типу IllegalArgumentException, которое є непроверяемым. Тому в принципе его можна не обробляти, якщо нас устраивает подобное завершение программы с ошибкой. Тем не менїї обработаем исключением:


class Program{
     
	public static void main(String[] args) {
         
        try{

            var tom = new Person("Tom", -20);
            tom.print();
        }
        catch(IllegalArgumentException ex){
            System.out.println(ex.getMessage());
        }
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age){

        if(age < 1 || age > 110) throw new IllegalArgumentException("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Оператор throws

Если метод генерує проверяемое исключение, то нам треба або обработать це исключение, або сообщить компилятору, что метод генерує данное исключение. Тому обработка исключения непосредственно в методе - не единственный способ работы с проверяемыми исключениями. Нередко метод сам не обробляє исключение, а надає возможнасть обработки внешнему коду, где цей метод вызывается. В цем случае в оголошенийии метода необхідно указать тип исключения, який генерує метод. Для цього використовується оператор throws в следующем виде:


возвращаемый_тип имя_метода(параметры) throws тип_исключения_1, тип_исключения_2,... тип_исключения_N  {

}

Оператор throws указывается после списка параметров метода, но до открывающей фигурной скобки. А после оператора идет перечисление типов генерируемых исключений. Например:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        try{
            printName();  // здесь генеруєся исключение
        }
        catch(Throwable ex){
            System.out.println(ex.getMessage());
        }
    }

    static void printName() throws Throwable {

        Scanner in = new Scanner(System.in);
        System.out.print("Введите имя: ");
        String name = in.nextLine(); // считываем введенную строку

        // якщо строка имїїт имїїт меньше 2-х символов, генерируем исключение
        if(name.length() < 2) throw new Throwable("Недостаточная длина имени: " + name.length());
        System.out.println("Привет, " + name);
    }
}

Здесь метод printName() считывает через консоль имя пользователя и за допомогою оператора throw генерує исключение типу Throwable, якщо длина введенного имени меньше 2 символов. При цем, поскольку цей метод сам не обробляє исключение, то в оголошенийии метода указываем, что он генерує исключение типу Throwable:

static void printName() throws Throwable {

А внешний метод - main(), в котором вызывается printName(), обробляє возможна исключение за допомогою конструкции try..catch.

ПРи цем йде учитывать, что якщо вызываемый метод (printName()) сам не обробляє проверяемое исключение, то внешний - вызывающий метод (main()) повинен або обработать исключение (як в примере выше), або також указать компилятору за допомогою оператора throws, что он генерує данный тип исключений.

Выше був пример, где внешний метод main() обрабаотывал исключение, возникшїї в printName(). Теперь же используем другую стратегию:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) throws Throwable {
         
        printName(); // здесь возникает исключение, но его никто не обробляє
    }

    static void printName() throws Throwable{

        Scanner in = new Scanner(System.in);
        System.out.print("Введите имя: ");
        String name = in.nextLine(); // считываем введенную строку

        // якщо строка имїїт имїїт меньше 2-х символов, генерируем исключение
        if(name.length() < 2) throw new Throwable("Недостаточная длина имени: " + name.length());
        System.out.println("Привет, " + name);
    }
}

Данная программа успешно скомпилируется, оскільки в заголовке метода main() мы указали выражение throws Throwable:


public static void main(String[] args) throws Throwable {

Однак при выполнении при некорректном вводе, поскольку исключение не обробляєся, программа упадет с ошибкой:


Введите имя: r
Exception in thread "main" java.lang.Throwable: Недостаточная длина имени: 1
	at Program.printName(Program.java:22)
	at Program.main(Program.java:7)

И в конце варто відзначити, что метод может генерировать декілька типов исключений. В цем случае они указываются после оператора throws через запятую:


public static void main(String[] args) throws FileNotFoundException, EOFException {

Однак якщо метод генерує непроверяемые исключения типу IllegalArgumentException, то нет смысла их указывать, оскільки компилятор их все равно не проверяет.
