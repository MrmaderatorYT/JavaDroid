Оператор throws и throw

Останнє оновлення: 06.10.2025

Генерація винятку та оператор throw

Обычно система сама генерує винятки при визначенийных ситуациях, наприклад, при делении числа на ноль. Но язык Java також дозволяє генерировать винятки вручную за допомогою оператора throw. Наприклад, в нашей програме пользователь повинен вводить ім'я, но ім'я имїїт свои ограничения. Наприклад, очевидно, что ім'я из одного символа - це некорректно ім'я. Підзможна, мы пред'ємо ще якие-то ограничения к імені. И в цем случае мы можемо при вводе некорректных данных генерировать виняток.

Общая форма оператора:


throw об'єкт_винятки;

То есть після оператора вказуєся об'єкт винятки. Как его створити? Прежде всего мы можемо використовувати встроенные класы винятків, в том числе базовые - Throwable и Exception. Так, один из конструкторов типу Throwable приймає сообщение об ошибке:


Throwable(String message)

По сути це то сообщение, которое повертаєся методом getMessage() и которое при возникновении ошибки отображается на консоли. Ну и соответственно все другие типи винятків також имеют підобный конструктор.

Рассмотрим генерацию винятки на примере:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        Scanner in = new Scanner(System.in);
        System.out.print("Введите ім'я: ");
        String name = in.nextLine(); // считываем введенную рядок

        // якщо рядок имїїт имїїт меньше 2-х символів, генерируем виняток
        if(name.length() < 2) throw new Throwable("Малобукоф");

        System.out.println("Привет, " + name);
    }
}

Здесь за допомогою класа Scanner и методу in.nextLine() считываем с консоли рядок - условное ім'я пользователя. Затем, використовуючи метол length() у рядки смотрим на длину - якщо в імені меньше 2 символів, то генерируем виняток:

if(name.length() < 2) throw new Throwable("Малобукоф");

В конструктор класа Throwable передаємо сообщение об ошибке.

Но якщо мы попробуем скомпилировать програму, нас постигнет неудача:


Program.java:12: error: unreported exception Throwable; must be caught or declared to be thrown
        if(name.length() < 2) throw new Throwable("Малобукоф");
                              ^
1 error

Компилятор нам говорит, что нам треба або обпрацювати виняток, або указать, что метод main генерує данное виняток.

Проверяемые и непроверяемые винятки

КАк говорилось в прошлой статье, винятки в Java делятся условно на две группы: проверяемые (checked exceptions) и непроверяемые (unchecked exceptions). К непроверяемым винятким относят класы Error и RuntimeException и унаследованные от них класы. А проверяемые винятки по сути все остальные. И в контексте работы с оператором throw підобное розділение имїїт велике значення. При генерации проверяемых винятків нам треба або указать, что метод генерує виняток, або обпрацювати це виняток. Если же генеруються непроверяемые винятки, то ничего цього делать не треба.

В нашем примере выше генерируемое виняток типу Throwable рассматривается компилятором як проверяемое виняток (хотя класы Error и RuntimeException унаследованы от Throwable). Тому и при компиляции возникает ошибка.

Для решения проблемы изменим код и обработаем виняток за допомогою рассмотренной в прошлых статьях конструкции try..catch:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        Scanner in = new Scanner(System.in);
        System.out.print("Введите ім'я: ");
        String name = in.nextLine(); // считываем введенную рядок

        try{

            // якщо рядок имїїт имїїт меньше 2-х символів, генерируем виняток
            if(name.length() < 2) throw new Throwable("Малобукоф");
            System.out.println("Привет, " + name);
        }
        catch(Throwable ex){
            System.out.println("Підзникла ошибка: " + ex.getMessage());
        }
    }
}

Здесь та, часть коду, которая генерує винятки и которая зависит от проверки змінної name, помещается в блок try. Если буде сгенерировано виняток, то управление перейдет к блоку catch, где мы перехватим виняток и выведем сообщение об ошибке.

Пример работы програми:


Введите ім'я: e
Підзникла ошибка: Малобукоф

Обробка других типів винятків

Підобным чином мы можемо генерировать винятки в любом месте програми. Крім того, мы можемо використовувати и другие встроенные типи винятків. Наприклад, встроенный тип IllegalArgumentException призначений для генерации винятки при некорректном параметрі методу. Використовуємо данный тип:


class Program{
     
	public static void main(String[] args) {
         
        var tom = new Person("Tom", -20);
        tom.print();
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age){

        if(age < 1 || age > 110) throw new IllegalArgumentException("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Здесь в конструктор класа Person через параметр age передаєся возраст человека. Однак не все значення типу int можуть быть разумными значеннями для возраста. Наприклад, возраст не может быть отрицательным. И в конструкторе класа Person проверяем значення параметра age на соответствие некоторому диапазону. Ив случае несоответствия, генерируем виняток типу IllegalArgumentException


if(age < 1 || age > 110) 
    throw new IllegalArgumentException("Некорректный возраст: " + age);

В методі main передаємо в конструктор Person заведомо некорректное значення:

var tom = new Person("Tom", -20);

В итоге при запуску програми и створенні цього об'єкта буде сгенерировано виняток


Exception in thread "main" java.lang.IllegalArgumentException: Некорректный возраст: -20
	at Person.<init>(Program.java:20)
	at Program.main(Program.java:7)

Зверніть увагу, что при компиляции програми компилятор не выдал ошибку, як при компиляции предыдущего примера. Потому что у даному випадку мы обробляємо виняток типу IllegalArgumentException, которое є непроверяемым. Тому в принципе его можна не обробляти, якщо нас устраивает підобное завершення програми с ошибкой. Тем не менїї обработаем винятокм:


class Program{
     
	public static void main(String[] args) {
         
        try{

            var tom = new Person("Tom", -20);
            tom.print();
        }
        catch(IllegalArgumentException ex){
            System.out.println(ex.getMessage());
        }
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age){

        if(age < 1 || age > 110) throw new IllegalArgumentException("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Оператор throws

Если метод генерує проверяемое виняток, то нам треба або обпрацювати це виняток, або сообщить компилятору, что метод генерує данное виняток. Тому обробка винятки непосредственно в методі - не единственный способ работы с проверяемыми виняткими. Нередко метод сам не обробляє виняток, а надає возможнасть обработки внешнему коду, где цей метод вызывается. В цем случае в оголошенийии методу необхідно указать тип винятки, який генерує метод. Для цього використовується оператор throws в следующем виде:


повертаємоый_тип ім'я_методу(параметри) throws тип_винятки_1, тип_винятки_2,... тип_винятки_N  {

}

Оператор throws вказуєся після списку параметрів методу, но до открывающей фигурной скобки. А після оператора идет перечисление типів генерируемых винятків. Наприклад:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) {
         
        try{
            printName();  // здесь генеруєся виняток
        }
        catch(Throwable ex){
            System.out.println(ex.getMessage());
        }
    }

    static void printName() throws Throwable {

        Scanner in = new Scanner(System.in);
        System.out.print("Введите ім'я: ");
        String name = in.nextLine(); // считываем введенную рядок

        // якщо рядок имїїт имїїт меньше 2-х символів, генерируем виняток
        if(name.length() < 2) throw new Throwable("Недостаточная длина імені: " + name.length());
        System.out.println("Привет, " + name);
    }
}

Здесь метод printName() считывает через консоль ім'я пользователя и за допомогою оператора throw генерує виняток типу Throwable, якщо длина введенного імені меньше 2 символів. При цем, поскольку цей метод сам не обробляє виняток, то в оголошенийии методу указываем, что он генерує виняток типу Throwable:

static void printName() throws Throwable {

А внешний метод - main(), в котором вызывается printName(), обробляє возможна виняток за допомогою конструкции try..catch.

ПРи цем йде учитывать, что якщо вызываемый метод (printName()) сам не обробляє проверяемое виняток, то внешний - вызывающий метод (main()) повинен або обпрацювати виняток (як в примере выше), або також указать компилятору за допомогою оператора throws, что он генерує данный тип винятків.

Выше був пример, где внешний метод main() обрабаотывал виняток, возникшїї в printName(). Теперь же використовуємо другую стратегию:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) throws Throwable {
         
        printName(); // здесь возникает виняток, но его никто не обробляє
    }

    static void printName() throws Throwable{

        Scanner in = new Scanner(System.in);
        System.out.print("Введите ім'я: ");
        String name = in.nextLine(); // считываем введенную рядок

        // якщо рядок имїїт имїїт меньше 2-х символів, генерируем виняток
        if(name.length() < 2) throw new Throwable("Недостаточная длина імені: " + name.length());
        System.out.println("Привет, " + name);
    }
}

Данная програма успешно скомпилируется, оскільки в заголовке методу main() мы указали выражение throws Throwable:


public static void main(String[] args) throws Throwable {

Однак при выполнении при некорректном вводе, поскольку виняток не обробляєся, програма упадет с ошибкой:


Введите ім'я: r
Exception in thread "main" java.lang.Throwable: Недостаточная длина імені: 1
	at Program.printName(Program.java:22)
	at Program.main(Program.java:7)

И в конце варто відзначити, что метод может генерировать декілька типів винятків. В цем случае они вказуються після оператора throws через запятую:


public static void main(String[] args) throws FileNotFoundException, EOFException {

Однак якщо метод генерує непроверяемые винятки типу IllegalArgumentException, то нет смысла их указывать, оскільки компилятор их все равно не проверяет.
