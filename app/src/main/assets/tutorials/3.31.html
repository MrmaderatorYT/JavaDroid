Type Erasure (Стирання типів)

Останнє оновлення: 10.10.2025

Хотя язык Java дозволяє создавать обобщенные типи и методи, которые використовують параметри типу, однак на уровне виртуальной машины Java в реальности не існує нияких обобщенных типів и методів — все об'єкты принадлежат обычным класам. При компиляции параметри типу видаляються и заменяются або типом Object для параметрів типу без ограничений, або их типуми-ограничениями. Данный процес називається type erasure (стирание типів).

Наприклад, візьмемо наступний узагальнений клас:


class Person<T>{

    private T id;
    private String name;

    T getId(){ return id; }
    String getName() { return name; }

    Person(T id, String name){

        this.id = id;
        this.name = name;
    }
}

Здесь поскольку для параметра T не установлено ограничений, то он просто заменяется на Object. То есть в итоге мы получим следующий клас:


class Person{

    private Object id;
    private String name;

    Object getId(){ return id; }
    String getName() { return name; }

    Person(Object id, String name){

        this.id = id;
        this.name = name;
    }
}

Причому байдвже, якими типуми типізується об'єкт Person у програмі:


Person tom = new Person<Integer>(456, "Tom");
Person bob = new Person<String>("qwert", "Bob");

В обоих цих случаях для среды виконання Java параметр T буде представлять тип Object.

Якщо результат методу чи полі класу представляє параметр типу і це результат методу чи полі присвоюються зовнішньої змінної, то середовище виконує необхідні перетворення. Наприклад, якщо ми маємо наступний код:


var tom = new Person<Integer>(456, "Tom");
int tomId = tom.getId();
System.out.println(tomId);

Те середовище виконання трактує цей код на кшталт наступного:


var tom = new Person(456, "Tom");
int tomId = (int) tom.getId();
System.out.println(tomId);

Параметри типу з обмеженнями

Тепер розглянемо інший приклад – застосування обмежень типу:


class Message{
 
    private String text; // текст сообщения
    String getText(){ return text; }
 
    Message(String text)
    {
        this.text = text;
    }
}

class Messenger<T extends Message> {
 
    private T message;
    T getMessage() { return this.message; }
 
    Messenger(T message) { this.message = message;  }
 
    void send() {
        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}

У даному випадку клас Messenger типизируется параметром T, для которого установлено ограничение в виде типу Message, поцему всередині класа Messenger тип T заменяется на Message:


class Messenger {
 
    private Message message;
    Message getMessage() { return this.message; }
 
    Messenger(Message message) { this.message = message;  }
 
    void send() {
        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}

Якщо для одного параметра типу встановлено кілька обмежень, то параметр тпіа замінюється на тип першого обмеження. Наприклад:


class Messenger<T extends Message & Printable> {
 
    void sendMessage(T message){
 
        message.print();
    }
}
interface Printable{
 
    void print();
}
 
class Message{
 
    private String text; // текст сообщения
    String getText(){ return text; }
 
    Message(String text){

        this.text = text;
    }
}

Здесь в класе Messenger параметр типу T повинен представлять одновременно клас Message и інтерфейс Printable (клас, який найдеся от Message и реалізує інтерфейс Printable). В итоге для виртуальной срежы Java цей клас буде выглядеть напідобие следующего:


class Messenger {
 
    void sendMessage(Message message){
 
        ((Printable)message).print();
    }
}

Тобто параметр типу T замінюється перше обмеження - тип Message, і якщо використовується функціонал з другого обмеження, то виконується перетворення.
