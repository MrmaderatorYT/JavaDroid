Робота з ZIP-архівами

Останнє оновлення: 26.04.2018

Кроме общего функционала для работы с файлуми Java надає функциональность для работы с таким видом файлів як zip-архивы. Для цього в пакеті java.util.zip визначенийы два класа - ZipInputStream и ZipOutputStream

ZipOutputStream. Запис архівів

Для створенняя архива використовується клас ZipOutputStream. Для створенняя об'єкта ZipOutputStream в его конструктор передаєся потік вивіда:


ZipOutputStream(OutputStream out)

Для записи файлів в архив для каждого файлу створюєся об'єкт ZipEntry, в конструктор которого передаєся ім'я архивируемого файлу. А чтобы добавить каждый об'єкт ZipEntry в архив, застосовується метод putNextEntry().

Створимо архив:


import java.io.*;
import java.util.zip.*;
 
public class Program {
 
    public static void main(String[] args) {
        
        String filename = "notes.txt";
        try(ZipOutputStream zout = new ZipOutputStream(new FileOutputStream("output.zip"));
                FileInputStream fis= new FileInputStream(filename);)
        {
            ZipEntry entry1=new ZipEntry("notes.txt");
            zout.putNextEntry(entry1);
            // считываем вміст файлу в масив byte
            byte[] buffer = new byte[fis.available()];
            fis.read(buffer);
            // додаємо вміст к архиву
            zout.write(buffer);
            // закрываем текущую запис для новой записи
            zout.closeEntry();
        }
        catch(Exception ex){
             
            System.out.println(ex.getMessage());
        } 
    } 
}

Після добавления об'єкта ZipEntry в потік нам також треба добавить в него и вміст файлу. Для цього використовується метод write, записывающий в потік масив байтов: zout.write(buffer);. В конце треба закрыть ZipEntry за допомогою методу closeEntry(). Після цього можна добавлять в архив новые файли - в цем случае все вышеописанные действия для каждого нового файлу повторяются.

Читання архівів. ZipInputStream

Для читання архивов застосовується клас ZipInputStream. В конструкторе он приймає потік, указывающий на zip-архив:

ZipInputStream(InputStream in)

Для считывания файлів из архива ZipInputStream використовує метод getNextEntry(), який повертає об'єкт ZipEntry. Об'єкт ZipEntry представляє отдельную запис в zip-архиве. Наприклад, считаємо якой-нибудь архив:


import java.io.*;
import java.util.zip.*;
 
public class Program {
 
    public static void main(String[] args) {
        
        try(ZipInputStream zin = new ZipInputStream(new FileInputStream("output.zip")))
        {
            ZipEntry entry;
            String name;
            while((entry=zin.getNextEntry())!=null){
                 
                name = entry.getName(); // получим название файлу
                System.out.printf("File name: %s \n", name);
				
				// распаковка
				FileOutputStream fout = new FileOutputStream("new" + name);
				for (int c = zin.read(); c != -1; c = zin.read()) {
					fout.write(c);
				}
				fout.flush();
				zin.closeEntry();
				fout.close();
            }
        }
        catch(Exception ex){
             
            System.out.println(ex.getMessage());
        } 
    } 
}

ZipInputStream в конструкторе отримує ссылку на потік ввода. И затем в цикле виводяться все файли и их размер в байтах, которые находятся в данном архиве.

Затем дані извлекаются из архива и сохраняются в новые файли, которые находятся в той же папке и которые начинаются с "new".
