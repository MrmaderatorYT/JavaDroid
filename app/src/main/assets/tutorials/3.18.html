Класи Records

Останнє оновлення: 22.09.2025

Начиная с версии Java 16 в язык бува добавлена новая функциональность - Records (на русском нередко называют "записями"). Records представляютьь класы, состояние которых незмінюємоо (immutable), а их поля доступны для внешнего коду. Крім того, records дозволяють упростить разработку, сократив обсяг коду. По сути record - це болїї удобная компактная форма визначенийия контейнера незмінюємоых данных.

Для визначенийия класов record застосовується ключевое слово record, після которого идет название и далїї в круглых скобках список полей record:

record название (поле1, поле2,...полеN){
	// тело record
}

Розглянемо наступний приклад


import java.util.Objects;

public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom", 36);
        System.out.println(tom.toString());
    }
}

class Person {
    private final String name;
    private final int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    String name() { return name; }
    int age() { return age; }

    public boolean equals(Object o) {
        if (!(o instanceof Person)) return false;
        Person other = (Person) o;
        return other.name == name && other.age == age;
    }

    public int hashCode() {
        return Objects.hash(name, age);
    }

    public String toString() {
        return String.format("Person[name=%s, age=%d]", name, age);
    }
}

Тут визначено клас Person, який визначає дві константи - name та age:


private final String name;
private final int age;

Їхні значення встановлюються в конструкторі. Більше їх встановити ми не можемоо. Таким чином, після створення об'єкта Person вони будутьь зберігати незмінні дані.

Для отримання значень name та age передбачені однойменні методи:


String name() { return name; }
int age() { return age; }

Крім того, здесь перевизначенийы унаследованные от класа Object методи equals(), hashCode() и toString().

В методі main створюємо один об'єкт класа Person и виводить на консоль его текстовое представление:


Person tom = new Person("Tom", 36);
 System.out.println(tom.toString());

У результаті консоль нам виведе:

Person[name=Tom, age=36]

Теперь посмотрим, что нам предлагают Records - визначимо record, которая буде полностью аналогична вышевизначенийному класу:


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom", 36);
        System.out.println(tom.toString());
    }
}
record Person(String name, int age) { }

Records визначаються за допомогою ключевого слова record, за которым йде название записи. Дальше идет список полей записи. То есть у даному випадку визначаєся два поля - name и age. Причем по умолчанию все они будуть приватными и иметь модификатор final.

Також буде создаваться конструктор с двумя параметрами name и age. А каждого поля автоматически буде создаваться одноименный общедоступный метод для получения значення це поля. Наприклад, для поля name створюєся метод name(), який повертає значення поля name.

И також автоматически будуть создаваться методи equals, hashCode и toString. Підобщем, данная record буде полностью аналогична вышевизначенийному класу, но при цем містить гораздо меньше коду.

За потреби ми можемоо викликати всі наявні методи:


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom",  36);
		
        System.out.println(tom.name());		// Tom
        System.out.println(tom.age());		// 36
        System.out.println(tom.hashCode());
		
		Person bob = new Person("Bob", 21);
		Person tomas = new Person("Tom", 36);
		System.out.println(tom.equals(bob));	// false
		System.out.println(tom.equals(tomas));	// true
    }
}
record Person(String name, int age){ }

Конструктор record

У прикладі вище застосовувалася форма record:


record Person(String name, int age) { }

яка фактично створювала конструктор:


Person(String name, int age) { 
	this.name = name;
	this.age = age;
}

Цет конструктор називається канонічним. Он приймає параметри, которые називаються також як и поля record, и передає полям значення соответствующих параметрів.

Проте за потреби ми можемоо змінити логіку конструктора. Так ми можемоо перевизначити логіку конструктора. Наприклад, якщо при створенні об'єкта буде передано невалідний вік? Передбачимо цю ситуацію, перевизначивши логіку конструктора:


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom", -116);
        System.out.println(tom.toString());
    }
}
record Person(String name, int age) { 

	Person{
		
		if(age<1 || age > 110){
			age = 18;
		}
	}
}

У разі якщо передано невалідне значення, то застосовуємо деяке значення за промовчанням (число 18). У результаті ми отримаємо конструктор з наступною дією:


Person(String name, int age) { 
	if(age<1 || age > 110){
		age = 18;
	}
	this.name = name;
	this.age = age;
}

У результаті програма виведе на консоль таке:

Person[name=Tom, age=18]

Ми можемоо повністю перевизначити канонічний конструктор:


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom",  36);
        System.out.println(tom.toString());
        System.out.println(tom.name());
    }
}
record Person(String name, int age) { 

	Person(String name, int age){
		
		if(age < 0 || age > 120) age = 18;
		
		this.name = name;
		this.age = age;
	}
}

Також ми можемоо визначати якісь інші конструктори, але вони повинні викликати канонічний конструктор:


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom", "Smith",  36);
        System.out.println(tom.toString());
    }
}
record Person(String name, int age) { 

	Person(String firstName, String lastName, int age){
		
		this(firstName + " " + lastName, age);
		
	}
}

Здесь визначений конструктор, який условно приймає ім'я, фамилию и возраст пользователя. Цет конструктор вызывает канонический конструктор, передая ему значення для полей name и age: this(firstName + " " + lastName, age).

Консольне виведення програми:

Person[name=Tom Smith, age=36]

Перевизначення методів

Також мы можемо перевизначити методи, которые имїїт record по умолчанию. А це методи equals(), hashCode() и toString() и методи, которые називаються також, як и поля записи, и которые повертають значення соответствующих полей.

Наприклад, перепвизначимо для записи Person методи toString() и name():


public class Program{ 
      
    public static void main (String args[]){
          
		Person tom = new Person("Tom", 36);
        System.out.println(tom.toString());
        System.out.println(tom.name());
    }
}
record Person(String name, int age) { 

	public String name() { return "Mister " + name; }
	
    public String toString() {
        return String.format("Person %s, Age: %d", name, age);
    }
}

Консольне виведення програм:


Person Tom, Age: 36
Mister Tom

Обмеження Records

Слід враховувати, що ми не можемоо успадковувати запис record від інших класів. Також не можна успадковувати класи від records. Однак класи record можуть реалізувати інтерфейси. Крім того, класи record не можуть бути абстрактними.

У record не можна явно визначати нестатичні поля та ініціалізатори. Але можна визначати статичні змінні та ініціалізатори, також як статичні та нестатичні методи:


record Person(String name, int age){ 

	static int minAge;
	static{
        minAge = 18;
        System.out.println("Static initializer");
    }
}
