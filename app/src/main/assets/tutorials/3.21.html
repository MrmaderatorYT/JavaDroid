Конструктори та ініціалізатори

Останнє оновлення: 22.09.2025

Конструктори

Кроме обычных методів класы можуть визначати специальные методи, которые називаються конструкторами. Конструктори вызываются при створенні нового об'єкта данного класа. Обычно конструктори виконують инициализацию об'єкта.

Конструктор за замовчуванням

Якщо класі не визначено жодного конструктора, то цього класу автоматично створюється конструктор без параметрів. Наприклад:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person(); // вызов конструктора по умолчанию

        tom.name = "Tom";
        tom.age = 41;

        tom.print();
    }
}

class Person{

    String name;
    int age;

    void print(){

        System.out.printf("Name: %s; Age: %d\n", name, age);
    }
}

Вище певний клас Person немає жодних конструкторів. Тому для нього автоматично створюється конструктор за замовчуванням, який ми можемоо використовувати для створення об'єкта Person.

Для створенняя об'єкта Person використовується выражение new Person(). Оператор new выделяет память для об'єкта Person. И затем вызывается конструктор по умолчанию, який не приймає жодних параметрів. В итоге після виконання данного выражения в памяти буде выделен участок, де зберігатимуться всі дані об'єкта Person. А змінна tom получит ссылку на створений об'єкт.

Если конструктор не ініціалізує значення змінних об'єкта, то они отримують значення по умолчанию. Для змінних числовых типів це число 0, а для типу String и класов - це значення null (то есть фактически отсутствие значення).

Після створенняя об'єкта мы можемо обратиться к переменным об'єкта Person через змінну tom и установить або отримати их значення, наприклад, tom.name = "Tom".

У результаті побачимо на консолі:


Name: Tom; Age: 41

Створення конструкторів

Если необхідно, чтобы при створенні об'єкта производилась якая-то логика, наприклад, чтобы поля класа получали якие-то визначенийные значення, то можна визначити в класе свои конструктори. При цем якщо в класе визначаються свои конструктори, то цей клас позбавляється конструктора по умолчанию.

На рівні коду конструктор представляє метод, який називається на ім'я класу, який може мати параметри, але для нього не треба визначати тип, що повертається. Наприклад, визначимо у класі Person найпростіший конструктор:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person(); // вызов конструктора, який мы сами определили
        tom.print();
    }
}

class Person{

    String name;
    int age;

    // конструктор класа
    Person(){

        System.out.println("Створення об'єкта Person");
        name = "Tom";
        age = 41;
    }

    void print(){

        System.out.printf("Name: %s; Age: %d\n", name, age);
    }
}

Отже, тут у класі Person визначено конструктора, який виводить на консоль деяке повідомлення та ініціалізує поля класу:


Person(){

    System.out.println("Створення об'єкта Person");
    name = "Tom";
    age = 41;
}

Визначивши конструктор, ми можемоо викликати його до створення об'єкта Person:


Person tom = new Person();

У даному випадку выражение Person() як раз представляє вызов визначенийного в класе конструктора (це больше не автоматический конструктор по умолчанию, которого у класа теперь нет). Соответственно при его выполнении на консоли буде виводитьься рядок "Створення об'єкта Person"

Підібним чином ми можемоо визначати інші конструктори в класі. Наприклад:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();          // вызов 1-ого конструктора без параметрів
        Person bob = new Person("Bob");     //вызов 2-ого конструктора с одним параметром
        Person sam = new Person("Sam", 25); // вызов 3-его конструктора с двумя параметрами
        
        tom.print();          // Имя: Неизвестно  Підзраст: 18
        bob.print();          // Имя: Bob  Підзраст: 18
        sam.print();          // Имя: Sam  Підзраст: 25
    }
}

class Person{

    String name;
    int age;

    Person()
    {
        name = "Неизвестно";
        age = 18;
    }
    Person(String username)
    {
        name = username;
        age = 18;
    }
    Person(String username, int userage)
    {
        name = username;
        age = userage;
    }

    void print(){

        System.out.printf("Имя: %s; Підзраст: %d\n", name, age);
    }
}

Тепер у класі визначено три коструктори, кожен з яких набуває різної кількості параметрів та встановлює значення полів класу.

Консольне виведення програми:


Имя: Неизвестно  Підзраст: 18
Имя: Bob  Підзраст: 18
Имя: Sam  Підзраст: 25

З вище визначеного коду неважко побачити низку переваг конструкторів проти версією програми без конструкторів:

  • Автоматична ініціалізація: конструктори забезпечують корректную инициализацию об'єктов при их створенні, снижая вероятность ошибок из-за неинициализированных змінних.

  • Короткість коду: код инициализации сосредоточен в конструкторе, что робить основной код болїї лаконичным и удобочитаємоым.

  • Об'єктно-орієнтований дизайн: конструктори способствуют болїї об'єктно-ориентированному дизайну, инкапсулируя логику инициализации всередині самого класа.

Ключове слово this

Ключевое слово this представляє ссылку на текущий экземпляр (об'єкт) класа. Через це ключевое слово мы можемо обращаться к переменным, методум об'єкта, а також вызывать его конструктори. Наприклад:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();          // вызов 1-ого конструктора без параметрів
        Person bob = new Person("Bob");     //вызов 2-ого конструктора с одним параметром
        Person sam = new Person("Sam", 25); // вызов 3-его конструктора с двумя параметрами
        
        tom.print();          // Имя: Неизвестно  Підзраст: 18
        bob.print();          // Имя: Bob  Підзраст: 18
        sam.print();          // Имя: Sam  Підзраст: 25
    }
}

class Person{

    String name;
    int age;

    Person()
    {
        this("Неизвестно", 18);
    }
    Person(String name)
    {
        this(name, 18);
    }
    Person(String name, int age)
    {
        this.name = name;
        this.age = age;
    }

    void print(){

        System.out.printf("Имя: %s; Підзраст: %d\n", name, age);
    }
}

У третьому конструкторі параметри називаються як і, як і поля класу. І щоб розмежувати поля та параметри, застосовується ключове слово this:

this.name = name;

Так, у разі вказуємо, що значення параметра name присвоюється полю name.

Крім того, у нас три конструктори, які виконують ідентичні дії: встановлюють поля name та age. Щоб уникнути повторів, за допомогою цього можна викликати один із конструкторів класу та передати для його параметрів необхідні значення:


Person(String name)
{
	this(name, 18);
}

У результаті результат програми буде той самий, що й у попередньому прикладі.

Ініціалізатори

Окрім конструктора початкову ініціалізацію об'єкта цілком можна проводити за допомогою ініціалізатора об'єкта. Ініціалізатор виконується до будь-якого конструктора. То есть в инициализатор мы можемо поместить код, общий для всех конструкторов:


class Program{

    public static void main(String[] args) {  

        Person undef = new Person();          // вызов 1-ого конструктора без параметрів
        undef.print();      // Имя: Неизвестно  Підзраст: 18

        Person tom = new Person("Tom", 22); // вызов 2-ого конструктора с двумя параметрами
        tom.print();          // Имя: Tom  Підзраст: 22
    }
}

class Person{

    String name;
    int age;

    /*начало блока инициализатора*/
    {
        name = "Неизвестно";
        age = 18;
    }
    /*конец блока инициализатора*/

    Person() { }  // порожній конструктор, ничего не робить

    Person(String name, int age)
    {
        this.name = name;
        this.age = age;
    }

    void print(){

        System.out.printf("Имя: %s; Підзраст: %d\n", name, age);
    }
}

Консольний висновок:


Имя: Неизвестно; Підзраст: 18
Имя: Tom; Підзраст: 22

Порядок виконання ініціалізації полів Таким чином, ми маємо три способи ініціалізації полів:

Явна ініціалізація полів - полям щодо явно присвоюються деякі значення

  • Ініціалізація по умолчанию, когда мы явным чином не присваиваем полям нияких значений. В цем случае числовым полям присвоюється число 0, булевым полям - значення false, а полям других класов - значення null

  • Ініціалізація в ініціалізаторі

  • Ініціалізація у конструкторі

  • Тепер розглянемо, у порядку все це відбувається:

Поля набувають значення, які їм присвоюються при визначенні. Якщо щодо полям не присвоюється жодних значень, всі вони набувають значення за умовчанням

  1. Поля набувають значення, які їм присвоюються в ініціалізаторі

  2. Поля набувають значення, які їм присвоюються в конструкторі

  3. Розглянемо таку програму:

У разі у класі State є поле value, якому присвоєно деяке початкове значення. Крім того, в ініціалізаторі та конструкторі надають цьому полю нове значення, при цьому виводячи старе значення. У результаті отримаємо наступний консольний висновок:


class Program{

    public static void main(String[] args) {  

        State state = new State();

        System.out.println("Final value: " + state.value);

    }
}

class State{ 
    String value = "Default";  // значення по умолчанию

    //  инициализатор
    {
        System.out.println("Initializator. Old value: " + value);
        value = "Initializator";
    }

    // конструктор
    State(){
        System.out.println("Constructor. Old value: " + value);
        value = "Constructor";
    }
}

Тут ми бачи, що спочатку полю value надається початкове значення, потім виконується ініціалізатор, а потім конструктор.


Initializator. Old value: Default
Constructor. Old value: Initializator
Final value: Constructor

Можна також відловити початкове значення:

Консольний висновок:


class Program{

    public static void main(String[] args) {  

        State state = new State();

        System.out.println("Final value: " + state.value);  // 0

    }
}

class State{ 
    String value =  defaultValue();

    {
        System.out.println("Initializator. Old value: " + value);
        value = "Initializator";
    }

    State(){
        System.out.println("Constructor. Old value: " + value);
        value = "Constructor";
    }
    String defaultValue(){
        
        System.out.println("Default. Old value: " + value);
        return "Default";
    }
}

Назад


Default. Old value: null
Initializator. Old value: Default
Constructor. Old value: Initializator
Final value: Constructor
