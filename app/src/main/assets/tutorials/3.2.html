Пакети

Останнє оновлення: 18.09.2025

Як правило, Java класи об'єднуються в пакети. Пакети дозволяють організувати класи логічно набори. Організація класів як пакетів дозволяє уникнути конфлікту імен між класами. Адже нерідкі ситуації, коли розробники називають свої класи однаквими іменами. Належність до пакету дозволяє гарантувати однозначність імен.

По умолчанию в мові Java вже имїїтся ряд встроенных пакетів, наприклад, java.lang, java.util, java.io и другие, которые мы можемо використовувати. Крім того, пакеты можуть иметь вложенные пакеты. В частности, для работы с консолью застосовується класс System, який расположен в пакеті java.lang.

Імпорт пакетів та класів

Если треба використовувати классы из других пакетів, то нам треба подключить эти пакеты и классы. Исключение составляют классы из пакета java.lang (наприклад, System), которые подключаются в программу автоматически.

Наприклад, для работы с датами призначений класс Date находится в пакеті java.util, поцему мы можемо к нему обратиться за допомогою выражения:


java.util.Date

Наприклад:


class Program {
  
    public static void main(String[] args) {
        
        java.util.Date curDate = new java.util.Date();
        System.out.println(curDate);
    }
}

То есть мы указываем полный путь к файлу в пакеті при создании его об'єкта. Однак такое нагромождение имен пакетів не всегда удобно, и в качестве альтернативы мы можемо импортировать пакеты и классы в проект за допомогою директивы import:


import пакет.класс;

Наприклад, импортируем класс Date и используем его для вывода на консоль текущих даты и времени:


import java.util.Date;

class Program {
  
    public static void main(String[] args) {
        
        Date curDate = new Date();
        System.out.println(curDate);
    }
}

Директива import указывается в самом начале кода, после чего идет имя подключаемого класса (в данном случае класса Date).

Повторюсь, что классы из пакета java.lang подключаются автоматически, поцему на не треба отдельно импортировать класс System для вывода на консоль.

В примере выше мы подключили лише один класс, однак пакет java.util містить ще множество классов. И чтобы не подключать по отдельности каждый класс, мы можемо відразу подключить весь пакет:


import java.util.*; // импорт всех классов из пакета java.util

Тепер ми можемоо використовувати будь-який клас із пакету java.util.

Підзможна ситуация, когда мы используем два класса с одним и тем же названием из двух разных пакетів, наприклад, класс Date имїїтся и в пакеті java.util, и в пакеті java.sql. И якщо нам треба одновременно використовувати два цих класса, то необхідно указывать полный путь к цим классам в пакеті:


java.util.Date utilDate = new java.util.Date();
java.sql.Date sqlDate = new java.sql.Date();

Статичний імпорт

В java есть також особая форма импорта - статический импорт. Для цього вместе с директивой import використовується модификатор static:


import static java.lang.System.*;
import static java.lang.Math.*;

class Program {

    public static void main(String[] args) {
        
        double result = sqrt(20);
        out.println(result);
    } 
}

Здесь происходит статический импорт классов System и Math. Эти классы имеют статические методы. Благодаря операции статического импорта мы можемо використовувати эти методы без названия класса. Наприклад, писать не Math.sqrt(20), а sqrt(20), оскільки функция sqrt(), которая повертає квадратный корень числа, є статической. (Позже мы розглянемо статические члены класса).

То же самое в отношении класса System: в нем определен статический об'єкт out, поцему мы можемо его використовувати без указания класса.

Безіменні пакети

Класи необов'язково визначати пакети. Якщо для класу пакет не визначено, то вважається, що цей клас знаходиться в стандартному пакеті, який не має імені. Наприклад, нехай вихідний файл має наступний код:


class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        tom.print();
    }
}

class Person 
{
    String name; 

    Person(String name){

        this.name = name;
    }

    void print() {

        System.out.printf("Person %s\n", name);
    }
}

В даному випадку у файлі визначено два класи - Program і Person, і обидва вони визначені у безіменному пакеті. Власне, всі класи, які визначалися в попередніх статтях, визначалися саме в безіменних пакетах.

Встановлення пакета

Щоб вказати, що клас належить до певного пакету, треба використовувати директиву package, після якої вказується ім'я пакета:

package название_пакета;

Але подивимося, як нам визначити пакети. Як правило, назви пакетів відповідають фізичній структурі проекту, тобто організації каталогів, де знаходяться файли з вихідним кодом. А шлях до файлів усередині проекту відповідає назві цих файлів. Наприклад, якщо класи належать пакету mypack, ці класи поміщаються у проекті в папку mypack. А якщо класи належать пакету "com.mypack", вони поміщаються в папку "com/mypack".

Наприклад, визначимо таку структуру проекту:

  • Файл Program.java

  • Папка types

    • Файл Person.java

Таким чином, в корневой папке у нас есть файл Program.java и папка types, в которой лежит файл Person.java

Пусть файл Person.java буде выглядеть наступним чином:


package types;

public class Person 
{
    String name; 

    public Person(String name){

        this.name = name;
    }

    public String getName() {

        return this.name;
    }
}

Первая строка (package types) указывает, что класс Person принадлежит пакету "types". Принадлежность класса пакету указывается в самом начале кода файла.

Другой важный момент - ключевое слово public. Если мы хочемо, чтобы якие-то компоненты класса - поля, методы, конструкторы буви доступны в других пакетах, то эти компоненты повинні предваряться ключевым словом public, як в данном случае конструктор класса и метод getName(). Болїї того, чтобы класс в целом був доступен в других пакетах, он також повинен быть определен за допомогою слова public

Далїї в файле Program.java используем класс Person из пакета types:


import types.Person;  // испортируем класс Person из пакета types

class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        System.out.println(tom.getName());   // Tom
    }
}

Вначале кода подключаем класс Person из пакета types (import types.Person) и затем мы можемо обращаться к функционалу класса, який определен с ключевым словом public

Створення вкладених пакетів

Пойдем дальше и добавим в папку types каталог test, а в него - новый файл PersonTest. То есть вся структура проекта буде выглядеть наступним чином:

  • Файл Program.java

  • Папка types

    • Файл Person.java

    • Папка test

      • Файл PersonTest.java

В файле PersonTest.java определим следующїї содержимое:


package types.test;  // класс PersonTest находится в пакеті types.test

import types.Person;  // импортируем класс Person из пакета types

public class PersonTest
{

    public static void personNameTest(){

        String name = "Bob";
        Person bob = new Person(name);
        if(name == bob.getName()){
            System.out.println("personNameTest passed!");
        }
        else{
            System.out.println("personNameTest failed...");
        }
    }
}

Першим рядком вказуємо, що клас PersonTest знаходиться у пакеті "types.test". Знову ж таки зверніть увагу на відповідність назви пакету структурі каталогів: клас PersonTest знаходиться в папці "types/test" та в пакеті "types.test".

Другим рядком імпортуємо клас Person із пакета types.

Далїї определяем класс с единственным статическим методом personNameTest(), який проверяет имя человека, используя функционал класса Person. Опять же, чтобы метод personNameTest() був доступен в других пакетах, он определен с модификатором public.

Далїї изменим файл Program.java:


import types.test.PersonTest;  // испортируем класс PersonTest из пакета types.test

class Program {
  
    public static void main(String[] args) {
        
        PersonTest.personNameTest();  // personNameTest passed!
    }
}

Первой строкой испортируем класс PersonTest из пакета types.test и в методе main вызываем метод PersonTest.personNameTest(), тем самым проверяя, что функционал класса Person работает як треба.

Головний файл в окремому пакеті

До сих пор и примерах выше главный файл программы с методом main располагался в безымянном пакеті по умолчанию. Но естественно он також можемо располагаться в яком-то именнованном пакеті. Наприклад, добавим в текущую папку проектов новую папку с названием , а в нїї поместим файл Program.java. Щоб в итоге у нас получилась следующая структура проектов:

  • Папка

    • Файл Program.java

  • Папка types

    • Файл Person.java

    • Папка test

      • Файл PersonTest.java

И в файле Program.java определим следующий код:


package ;

import types.test.PersonTest;  // испортируем класс PersonTest из пакета types.test

class Program {
  
    public static void main(String[] args) {
        
        PersonTest.personNameTest();  // personNameTest passed!
    }
}

Першим рядком визначаємо, що наш пакет знаходиться в пакеті "".

Тепер нам треба все це скомпілювати та запустити. У консолі перейдемо до кореневого каталогу, де розташовуються папки "" та "types". І для компіляції програми виконаємо команду:

javac /Program.java

Далі для запуску програми виконаємо команду

java .Program

Повний приклад роботи:


eugene@Eugene:/workspace/java$ javac /Program.java
eugene@Eugene:/workspace/java$ java .Program
personNameTest passed!
eugene@Eugene:/workspace/java$

Застереження

Під час роботи з пакетами слід враховувати, що компілятор не перевіряє структуру каталогів під час компіляції вихідних файлів. Наприклад, припустимо, що у вас є вихідний файл, який починається з директиви

package com.mycompany;

Компілятор javac може скомпілювати файл, навіть якщо він не знаходиться у підкаталозі "com/mycompany". Вихідний файл скомпілюється без помилок, якщо він не використовує інші невбудовані пакети. Однак скомпільована програма не запуститься, доки ви попередньо не перемістите всі файли класів у потрібне місце. Віртуальна машина не виявить класів, якщо пакети не відповідають каталогам.
