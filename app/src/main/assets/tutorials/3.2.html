Пакети

Останнє оновлення: 18.09.2025

Як правило, Java класи об'єднуються в пакети. Пакети дозволяють організувати класи логічно набори. Організація класів як пакетів дозволяє уникнути конфлікту імен між класами. Адже нерідкі ситуації, коли розробники називають свої класи однаквими іменами. Належність до пакету дозволяє гарантувати однозначність імен.

По умолчанию в мові Java вже имїїтся ряд встроенных пакетів, наприклад, java.lang, java.util, java.io и другие, которые мы можемо використовувати. Крім того, пакети можуть иметь вложенные пакети. Зокрема, для работы с консолью застосовується клас System, який расположен в пакеті java.lang.

Імпорт пакетів та класів

Если треба використовувати класы из других пакетів, то нам треба підключить эти пакети и класы. Виняток составляют класы из пакету java.lang (наприклад, System), которые підключаются в програму автоматически.

Наприклад, для работы с датами призначений клас Date находится в пакеті java.util, поцему мы можемо к нему обратиться за допомогою выражения:


java.util.Date

Наприклад:


class Program {
  
    public static void main(String[] args) {
        
        java.util.Date curDate = new java.util.Date();
        System.out.println(curDate);
    }
}

То есть мы указываем полный путь к файлу в пакеті при створенні его об'єкта. Однак такое нагромождение имен пакетів не всегда удобно, и в качестве альтернативы мы можемо импортировать пакети и класы в проєкт за допомогою директивы import:


import пакет.клас;

Наприклад, импортируем клас Date и використовуємо его для вивіда на консоль текущих даты и времени:


import java.util.Date;

class Program {
  
    public static void main(String[] args) {
        
        Date curDate = new Date();
        System.out.println(curDate);
    }
}

Директива import вказуєся в самом начале коду, після чего идет ім'я підключаемого класа (у даному випадку класа Date).

Повторюсь, что класы из пакету java.lang підключаются автоматически, поцему на не треба отдельно импортировать клас System для вивіда на консоль.

В примере выше мы підключили лише один клас, однак пакет java.util містить ще множина класов. И чтобы не підключать по отдельности каждый клас, мы можемо відразу підключить весь пакет:


import java.util.*; // импорт всех класов из пакету java.util

Тепер ми можемоо використовувати будь-який клас із пакету java.util.

Підзможна ситуация, когда мы використовуємо два класа с одним и тем же названием из двух разных пакетів, наприклад, клас Date имїїтся и в пакеті java.util, и в пакеті java.sql. И якщо нам треба одновременно використовувати два цих класа, то необхідно указывать полный путь к цим класам в пакеті:


java.util.Date utilDate = new java.util.Date();
java.sql.Date sqlDate = new java.sql.Date();

Статичний імпорт

В java есть також особая форма импорта - статический импорт. Для цього вместе с директивой import використовується модификатор static:


import static java.lang.System.*;
import static java.lang.Math.*;

class Program {

    public static void main(String[] args) {
        
        double result = sqrt(20);
        out.println(result);
    } 
}

Здесь происходит статический импорт класов System и Math. Эти класы имеют статические методи. Благодаря операції статического импорта мы можемо використовувати эти методи без названия класа. Наприклад, писать не Math.sqrt(20), а sqrt(20), оскільки функція sqrt(), которая повертає квадратный корень числа, є статической. (Позже мы розглянемо статические члены класа).

То же самое в отношении класа System: в нем визначений статический об'єкт out, поцему мы можемо его використовувати без указания класа.

Безіменні пакети

Класи необов'язково визначати пакети. Якщо для класу пакет не визначено, то вважається, що цей клас знаходиться в стандартному пакеті, який не має імені. Наприклад, нехай вихідний файл має наступний код:


class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        tom.print();
    }
}

class Person 
{
    String name; 

    Person(String name){

        this.name = name;
    }

    void print() {

        System.out.printf("Person %s\n", name);
    }
}

В даному випадку у файлі визначено два класи - Program і Person, і обидва вони визначені у безіменному пакеті. Власне, всі класи, які визначалися в попередніх статтях, визначалися саме в безіменних пакетух.

Встановлення пакету

Щоб вказати, що клас належить до певного пакету, треба використовувати директиву package, після якої вказується ім'я пакету:

package название_пакету;

Але підивимося, як нам визначити пакети. Як правило, назви пакетів відповідають фізичній структурі проєкту, тобто організації каталогів, де знаходяться файли з вихідним кодом. А шлях до файлів усередині проєкту відповідає назві цих файлів. Наприклад, якщо класи належать пакету mypack, ці класи поміщаються у проєкті в папку mypack. А якщо класи належать пакету "com.mypack", вони поміщаються в папку "com/mypack".

Наприклад, визначимо таку структуру проєкту:

  • Файл Program.java

  • Папка types

    • Файл Person.java

Таким чином, в корневой папке у нас есть файл Program.java и папка types, в которой лежит файл Person.java

Пусть файл Person.java буде выглядеть наступним чином:


package types;

public class Person 
{
    String name; 

    public Person(String name){

        this.name = name;
    }

    public String getName() {

        return this.name;
    }
}

Первая рядок (package types) вказує, что клас Person принадлежит пакету "types". Принадлежность класа пакету вказуєся в самом начале коду файлу.

Другой важный момент - ключевое слово public. Если мы хочемо, чтобы якие-то компоненты класа - поля, методи, конструктори буви доступны в других пакетух, то эти компоненты повинні предваряться ключевым словом public, як у даному випадку конструктор класа и метод getName(). Болїї того, чтобы клас в целом був доступен в других пакетух, он також повинен быть визначений за допомогою слова public

Далїї в файле Program.java використовуємо клас Person из пакету types:


import types.Person;  // испортируем клас Person из пакету types

class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        System.out.println(tom.getName());   // Tom
    }
}

Вначале коду підключаем клас Person из пакету types (import types.Person) и затем мы можемо обращаться к функционалу класа, який визначений с ключевым словом public

Створення вкладених пакетів

Пойдем дальше и додамо в папку types каталог test, а в него - новый файл PersonTest. То есть вся структура проєкта буде выглядеть наступним чином:

  • Файл Program.java

  • Папка types

    • Файл Person.java

    • Папка test

      • Файл PersonTest.java

В файле PersonTest.java визначимо следующїї вміст:


package types.test;  // клас PersonTest находится в пакеті types.test

import types.Person;  // импортируем клас Person из пакету types

public class PersonTest
{

    public static void personNameTest(){

        String name = "Bob";
        Person bob = new Person(name);
        if(name == bob.getName()){
            System.out.println("personNameTest passed!");
        }
        else{
            System.out.println("personNameTest failed...");
        }
    }
}

Першим рядком вказуємо, що клас PersonTest знаходиться у пакеті "types.test". Знову ж таки зверніть увагу на відповідність назви пакету структурі каталогів: клас PersonTest знаходиться в папці "types/test" та в пакеті "types.test".

Другим рядком імпортуємо клас Person із пакету types.

Далїї определяем клас с единственным статическим методом personNameTest(), який проверяет ім'я человека, використовуючи функционал класа Person. Опять же, чтобы метод personNameTest() був доступен в других пакетух, он визначений с модификатором public.

Далїї изменим файл Program.java:


import types.test.PersonTest;  // испортируем клас PersonTest из пакету types.test

class Program {
  
    public static void main(String[] args) {
        
        PersonTest.personNameTest();  // personNameTest passed!
    }
}

Первой рядківой испортируем клас PersonTest из пакету types.test и в методі main вызываем метод PersonTest.personNameTest(), тем самым проверяя, что функционал класа Person работает як треба.

Головний файл в окремому пакеті

До сих пор и примерах выше главный файл програми с методом main располагался в безымянном пакеті по умолчанию. Но естественно он також можемо располагаться в яком-то именнованном пакеті. Наприклад, додамо в текущую папку проєктов новую папку с названием , а в нїї поместим файл Program.java. Щоб в итоге у нас получилась следующая структура проєктов:

  • Папка

    • Файл Program.java

  • Папка types

    • Файл Person.java

    • Папка test

      • Файл PersonTest.java

И в файле Program.java визначимо следующий код:


package ;

import types.test.PersonTest;  // испортируем клас PersonTest из пакету types.test

class Program {
  
    public static void main(String[] args) {
        
        PersonTest.personNameTest();  // personNameTest passed!
    }
}

Першим рядком визначаємо, що наш пакет знаходиться в пакеті "".

Тепер нам треба все це скомпілювати та запустити. У консолі перейдемо до кореневого каталогу, де розташовуються папки "" та "types". І для компіляції програми виконаємо команду:

javac /Program.java

Далі для запуску програми виконаємо команду

java .Program

Повний приклад роботи:


eugene@Eugene:/workspace/java$ javac /Program.java
eugene@Eugene:/workspace/java$ java .Program
personNameTest passed!
eugene@Eugene:/workspace/java$

Застереження

Під час роботи з пакетуми слід враховувати, що компілятор не перевіряє структуру каталогів під час компіляції вихідних файлів. Наприклад, припустимо, що у вас є вихідний файл, який починається з директиви

package com.mycompany;

Компілятор javac може скомпілювати файл, навіть якщо він не знаходиться у підкаталозі "com/mycompany". Вихідний файл скомпілюється без помилок, якщо він не використовує інші невбудовані пакети. Однак скомпільована програма не запуститься, доки ви попередньо не перемістите всі файли класів у потрібне місце. Віртуальна машина не виявить класів, якщо пакети не відповідають каталогум.
