Функціональні інтерфейси та посилання на методи

Останнє оновлення: 02.10.2025

Функциональный інтерфейс

В мові Java есть особый тип інтерфейсов, який називається функциональным інтерфейсом. Функциональный інтерфейс - це інтерфейс, в котором визначений лише один абстрактный метод (метод без реалізації). Простейший пример:


interface Message{
    void print();
}

Данный інтерфейс Message можна назвать функциональным.

И як и в случае с любым інтерфейсом, мы можемо визначити змінну цього інтерфейса:


Message mes;

Ссылки на метод

Хотя функциональный інтерфейс, на первый взгляд, кажется обычным інтерфейсом, но у него есть особенность в том, что он может представлять ссылки на методи, то есть на некоторое действие.

Ссылка на метод (method reference) представляє синтаксическую конструкцию следующего вида:


ім'я_класа::ім'я_статического_методу     // якщо метод статический

об'єкт_класа::ім'я_методу       // якщо метод нестатический

Наприклад:


public class Program {

    public static void main(String[] args) {
        
        Message mes;
        mes = Program::hello;
    }

    static void hello(){
        System.out.println("Hello ");
    }
}
interface Message{
    void print();
}

Здесь змінної mes передаємо ссылку на метод hello, який визначений в класе Program як статический. Тому ссылка на метод выглядит наступним чином:

Program::hello

И тут також важливо, что ссылка на метод из ссылки (hello) повинен соответствовать единственному методу функционального інтерфейса по типуми параметрів и типу результат. Так, в нашем случае метод print в інтерфейсе Message имїїт тип void и не приймає нияких параметрів. Соответственно и статический метод hello, ссылка на який присвоюється змінної інтерфейса, також не приймає параметрів и имїїт тип void

Далїї мы можемо вызвать метод по ссылке:


public class Program {

    public static void main(String[] args) {
        
        Message mes;
        mes = Program::hello;

        mes.print(); // вызываем метод по ссылке
    }

    static void hello(){
        System.out.println("Hello ");
    }
}
interface Message{
    void print();
}

То есть при вызове

mes.print();

фактически вызывается метод по ссылке - метод hello()

Ссылка на метод предписывает компилятору створити экземпляр функционального інтерфейса, переопределяя единственный метод інтерфейса для его вызова. То есть у даному випадку мы отримуємо неякий об'єкт інтерфейса Message, где метод hello() выступает в качестве реалізації методу print() из інтерфейса.

Если нам треба вызвать нестатические методи, то в ссылке вместо імені класа застосовується ім'я об'єкта цього класа:


public class Program {

    public static void main(String[] args) {
        
        Program prog = new Program();
        Message mes = prog::hello;

        mes.print();    // Hello 
    }

    void hello(){
        System.out.println("Hello ");
    }
}
interface Message{
    void print();
}

У даному випадку метод hello() визначений як нестатический. Тому для формирования ссылки на него створюємо об'єкт класа Program и передаємо ссылку змінної mes:


Program prog = new Program();
Message mes = prog::hello;

Пример методу с параметрами

Рассмотрим другой пример, когда змінна ссылается на метод с параметрами и некоторым повертаємоым результатом:


public class Program {

    public static void main(String[] args) {
        
        Operation op = Operations::add;
        System.out.println(op.execute(5, 4));       // 9

        op = Operations::sub;
        System.out.println(op.execute(5, 4));       // 1

        op = Operations::mul;
        System.out.println(op.execute(5, 4));       // 20
    }
}
interface Operation{
    int execute(int x, int y);
}
class Operations{

    static int add(int a, int b){ return a + b; }
    static int sub(int a, int b){ return a - b; }
    static int mul(int a, int b){ return a * b; }
}

Здесь мы определяем функциональный інтерфейс Operation с методом execute()


interface Operation{
    int execute(int x, int y);
}

В програме в методі main() створюємо змінну op, которая представляє данный інтерфейс, и присваиваем ей ссылку на статический метод Operations.add:


Operation op = Operations::add;

Мы можемо так сделать, поскольку типи параметрів и повертаємоый тип методу Operations.add відповідають типум параметрів и результата абстрактного методу execute() из інтерфейса Operation.

Затем через змінну op вызываем метод по ссылке:


System.out.println(op.execute(5, 4));

Далїї присваиваем змінної ссылку на другой метод и повторно вызываем и соответственно отримуємо другой результат при тех же аргументах:


op = Operations::sub;
System.out.println(op.execute(5, 4));       // 1

Ссылки на метод як параметри методів

Підзможнасть визначати ссылки на методи открываем нам дополнительные возможнасти при написании програм. Так, мы можемо визначити параметр, який представляє некоторое действие, и передать ему ссылку на метод. Рассмотрим на примере:


public class Program {

    public static void main(String[] args) {
         
        doOperation(10, 4, Operations::add);  // 14
        doOperation(10, 4, Operations::sub);  // 6
        doOperation(10, 4, Operations::mul);  // 40
    } 

    static void doOperation(int a, int b, Operation op)
    {
        System.out.println(op.execute(a, b));
    }
}
interface Operation{
    int execute(int x, int y);
}
class Operations{

    static int add(int a, int b){ return a + b; }
    static int sub(int a, int b){ return a - b; }
    static int mul(int a, int b){ return a * b; }
}

Здесь возможные вычисления вынесены в отдельный клас Operations и представляютьь статические методи. Тому для передачи цих методів застосовуються выражения типу Operations::add

doOperation(10, 4, Operations::add);  // 14

Если нам треба вызвать нестатические методи, то в ссылке вместо імені класа застосовується ім'я об'єкта цього класа:


public class Program {

    public static void main(String[] args) {
         
        Operations ops = new Operations();
        doOperation(10, 4, ops::add);  // 14
        doOperation(10, 4, ops::sub);  // 6
        doOperation(10, 4, ops::mul);  // 40
    } 

    static void doOperation(int a, int b, Operation op)
    {
        System.out.println(op.execute(a, b));
    }
}
interface Operation{
    int execute(int x, int y);
}
class Operations{

    int add(int a, int b){ return a + b; }
    int sub(int a, int b){ return a - b; }
    int mul(int a, int b){ return a * b; }
}

Ссылки на метод як результати методів

Крім того, мы можемо возвращать ссылки на методи из других методів. Наприклад:


import java.util.function.BiFunction;

public class Program {

    public static void main(String[] args) {
        
        Operation action = select(1);  // отримуємо ссылку на функцію Sum
        System.out.println(action.execute(8, 5)); // 13

        action = select(2);  // отримуємо ссылку на функцію Subtract
        System.out.println(action.execute(8, 5));// 3

        action = select(3);  // отримуємо ссылку на функцію Multiply
        System.out.println(action.execute(8, 5)); // 40
    }

    static Operation select(int choice){

        // повертаємо нужную функцію в зависимости от choice
        switch (choice)
        {
            case 2:
                return Program::subtract;
            case 3:
                return Program::multiply;
            default:
                return Program::sum;
        }
    }
    static int sum(int a, int b) { return a + b; }
    static int subtract(int a, int b) { return a - b; }
    static int multiply(int a, int b) { return a * b; }
}
// Operation представляє метод, який приймає два числа int и повертає int
interface Operation {
    int execute(int a, int b);
}

Здесь метод select() в зависимости от значення параметра повертає ссылку на один из методів: sum, subtract або multiply.


static Operation select(int choice){

    switch (choice){

        case 2:
            return Program::subtract;
        case 3:
            return Program::multiply;
        default:
            return Program::sum;
    }
}

В методі main() отримуємо результат методу select() в змінну:

Operation action = select(1);

То есть у даному випадку змінна action фактически буде представлять ссылку Program::sum

Ссылки на конструктори

Підобным чином мы можемо використовувати конструктори: название_класа::new. Наприклад:


public class Program {

    public static void main(String[] args) {
        
        UserBuilder userBuilder = User::new;
        User user = userBuilder.create("Tom");
        System.out.println(user.getName());
    }
}
interface UserBuilder{
    User create(String name);
}

class User{
    
    private String name;
    String getName(){
        return name;
    }
    
    User(String n){
        this.name=n;
    }
}

При использовании конструкторов методи функциональных інтерфейсов повинні принимать тот же список параметрів, что и конструктори класа, и повинні возвращать об'єкт данного класа.

Аналогично выражение Person[]::new представляє створення масива об'єктов Person.
