Буферизовані потіки BufferedInputStream та BufferedOutputStream

Останнє оновлення: 26.04.2018

Для оптимизации операцій ввода-вивіда використовуються буферизуемые потіки. Эти потіки додають к стандартным специальный буфер в памяти, за допомогою которого повышается производительность при чтении и записи потіков.

Клас BufferedInputStream

Клас BufferedInputStream накапливает вводимые дані в специальном буфере без постоянного обращения к устройству ввода. Клас BufferedInputStream визначає два конструктора:


BufferedInputStream(InputStream inputStream)
BufferedInputStream(InputStream inputStream, int bufSize)

Первый параметр - це потік ввода, с которого дані будуть считываться в буфер. Второй параметр - размер буфера.

Наприклад, буферизируем считывание данных из потіка ByteArrayInputStream:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
         String text = "Hello world!";
         byte[] buffer = text.getBytes();
        ByteArrayInputStream in = new ByteArrayInputStream(buffer);
        
        try(BufferedInputStream bis = new BufferedInputStream(in)){
            
            int c;
            while((c=bis.read())!=-1){
        
                System.out.print((char)c);
            }
        }
        catch(Exception e){
            
            System.out.println(e.getMessage());
        }
        System.out.println();
    } 
}

Клас BufferedInputStream в конструкторе приймає об'єкт InputStream. У даному випадку таким об'єктом є экземпляр класа ByteArrayInputStream.

Как и все потіки ввода BufferedInputStream обладает методом read(), який считывает дані. И здесь мы считываем за допомогою методу read каждый байт из масива buffer.

Фактические все то же самое можна буво сделать и за допомогою одного ByteArrayInputStream, не прибегая к буферизированному потіку. Клас BufferedInputStream просто оптимизирует производительность при работе с потіком ByteArrayInputStream. Естественно вместо ByteArrayInputStream может використовуватися любой другой клас, який унаследован от InputStream.

Клас BufferedOutputStream

Клас BufferedOutputStream аналогично створює буфер для потіков вивіда. Цет буфер накапливает вивідимые байты без постоянного обращения к устройству. И когда буфер заполнен, производится запис данных.

BufferedOutputStream визначає два конструктора:


BufferedOutputStream(OutputStream outputStream)
BufferedOutputStream(OutputStream outputStream, int bufSize)

Первый параметр - це потік вивіда, який унаследован от OutputStream, а второй параметр - размер буфера.

Рассмотрим на примере записи в файл:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        String text = "Hello world!"; // рядок для записи
        try(FileOutputStream out=new FileOutputStream("notes.txt"); 
                BufferedOutputStream bos = new BufferedOutputStream(out))
        {
			// перевод рядки в байты
            byte[] buffer = text.getBytes();
            bos.write(buffer, 0, buffer.length);
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
    } 
}

Клас BufferedOutputStream в конструкторе приймає в качестве параметра об'єкт OutputStream - у даному випадку це файлівый потік вивіда FileOutputStream. И також производится запис в файл. Опять же BufferedOutputStream не додає много новой функциональности, он просто оптимизирует действие потіка вивіда.
