Введение в лямбда-выражения

Останнє оновлення: 02.10.2025

Лямбда-вираження в мові Java представляютьь набор инструкций, которые можна выделить в отдельную змінну и затем многократно вызвать в различных местах програми.

Визначенийие лямбда-выражения и функциональные інтерфейсы

Для створенняя лямбда-выражения застосовується лямбда-оператор (стрелка ->). Цет оператор розділяет лямбда-выражение на две части:

 (параметри) -> действия

левая часть (до стрелки) містить параметри через запятую, а правая часть (після стрелки) собственно представляє выполняемые действия лямбда-выражения.

Лямбда-выражение не виконується само по себе, а образует реализацию методу, визначенийного в функциональном інтерфейсе. Рассмотрим пример:


public class Program {

    public static void main(String[] args) {
        
        Operationable op;
        op = (x,y)->x+y;
        
        int result = op.execute(10, 20);
        System.out.println(result); //30
    }   
}
interface Operationable{
    int execute(int x, int y);
}

Здесь в роли функционального інтерфейса выступает інтерфейс Operationable, в котором визначений один метод без реалізації - метод execute. Данный метод приймає два параметра - целых числа, и повертає некоторое целое число.

Щоб оголосити и використовувати лямбда-выражение, основная програма разбивается на ряд этапов:

  1. Визначенийие ссылки на функциональный інтерфейс:

    Operationable op;

  2. Створення лямбда-выражения:

    op = (x,y)->x+y;

    Причем параметри лямбда-выражения відповідають параметрам единственного методу інтерфейса Operationable, а результат відповідає повертаємоому результату методу інтерфейса. При цем нам не треба використовувати ключевое слово return для возврата результата из лямбда-выражения.

    Так, в методі інтерфейса оба параметра представляютьь тип int, значит, в теле лямбда-выражения мы можемо пріменіть к ним сложение. Результат сложения також представляє тип int, об'єкт которого повертаєся методом інтерфейса.

  3. Використання лямбда-выражения в виде вызова методу інтерфейса:

    int result = operation.execute(10, 20);

    Так як в лямбда-выражении визначенийа операція сложения параметрів, результатом методу буде сумма чисел 10 и 20.

По факту лямбда-выражения є в некотором роде сокращенной формой внутренних анонимных класов, которые ранїї применялись в Java. Зокрема, предыдущий пример мы можемо переписать наступним чином:


public class Program {

    public static void main(String[] args) {
        
        Operationable op = new Operationable(){
        
            public int execute(int x, int y){
            
				return x + y;
            }
        };
         int z = op.execute(20, 10);
         System.out.println(z); // 30
    }   
}
interface Operationable{
    int execute(int x, int y);
}

При цем для одного функционального інтерфейса мы можемо визначити множина лямбда-выражений. Наприклад:


Operationable sum = (int x, int y)-> x + y;
Operationable sub = (int x, int y)-> x - y;
Operationable mul = (int x, int y)-> x * y;
        
System.out.println(sum.execute(20, 10)); //30
System.out.println(sub.execute(20, 10)); //10
System.out.println(mul.execute(20, 10)); //200

Если лямбда-выражение приймає один параметр, то скобки вокруг параметрів можна не використовувати:


public class Program {

    public static void main(String[] args) {
        
        Printable printer = message -> System.out.println(message);

        printer.print("Hello World");
        printer.print("Hello Work");
    }   
}
interface Printable{
    void print(String message);
}

Здесь лямбда-выражение printer представляє функциональный інтерфейс Printable, единственный метод которого - print приймає лише один параметр.

Другая ситуация - в методі функционального інтерфейса есть параметр, но в лямбда-выражении он не використовується. В цем случае вместо параметра можна указать прочерк:


public class Program {

    public static void main(String[] args) {
        
        Printable printer = _ -> System.out.println("Test");

        // при вызове параметру все равно треба передать значення
        // но оно не имїїт значення, оскільки не буде використовуватися
        printer.print(null);    // Test
        printer.print("brr");   // Test
    }   
}
interface Printable{
    void print(String message);
}

Если метод інтерфейса вообще не приймає нияких параметрів, то при визначенийии лямбда-выражения вказуються пустые скобки:


public class Program {

    public static void main(String[] args) {
        
        Hello speaker = () -> System.out.println("Привед");

        speaker.sayHello();
    }   
}
interface Hello{
    void sayHello();
}

Отложенное виконання

Одним из ключевых моментов в использовании лямбд є отложенное виконання (deferred execution). То есть мы определяем в одном месте програми лямбда-выражение и затем можемо его вызывать при необхідности невизначенийное кількість раз в различных частях програми. Отложенное виконання может потребоваться, к примеру, в следующих случаях:

  • Виконання коду отдельном потіке

  • Виконання одного и того же коду декілька раз

  • Виконання коду в результаті якого-то события

  • Виконання коду лише в том случае, когда он действительно необходим и якщо он необходим

Лямбды и локальные змінні

Лямбда-выражение может використовувати змінні, которые оголошенийы во вне в болїї общей области видимости - на уровне класа або методу, в котором лямбда-выражение визначенийо. Однак в зависимости от того, як и где визначенийы змінні, можуть различаться способы их использования в лямбдах. Рассмотрим первый пример - использования змінних уровня класа:


public class Program {

    static int x = 10;
    static int y = 20;
    public static void main(String[] args) {
        
        Operation op = ()->{ 
            
            x=30;
            return x+y;
        };
        System.out.println(op.execute()); // 50
        System.out.println(x); // 30 - значення x изменилось
    }
}
interface Operation{
    int execute();
}

Змінні x и y оголошенийы на уровне класа, и в лямбда-выражении мы их можемо отримати и даже змінити. Так, у даному випадку після виконання выражения змінюєся значення змінної x.

Теперь розглянемо другой пример - локальные змінні на уровне методу:


public static void main(String[] args) {
        
    int n = 70;
    int m = 30;
    Operation op = ()->{ 
            
        //n = 100; - так нельзя сделать
        return m+n;
    };
	// n=100;  - так теж не можна
    System.out.println(op.execute()); // 100
}

Локальные змінні уровня методу мы також можемо використовувати в лямбдах, но изменять их значення нельзя. Если мы попробуем це сделать, то среда разработки (Netbeans) может нам высветить ошибку и то, что такую змінну треба пометить за допомогою ключевого слова final, то есть сделать константой: final int n=70;. Однак це необязательно.

Болїї того, мы не сможемо змінити значення змінної, которая використовується в лямбда-выражении, вне цього выражения. То есть даже якщо такая змінна не оголошенийа як константа, по сути она є константой.

Блоки коду в лямбда-выражениях

Існують два типу лямбда-выражений: однострочное выражение и блок коду. Примеры однострочных выражений демонстрировались выше. Блочные выражения обрамляются фигурными скобками. В блочных лямбда-выражениях можна використовувати внутренние вложенные блоки, циклы, конструкции if, switch, создавать змінні и т.д. Если блочное лямбда-выражение повинно возвращать значення, то явным чином застосовується оператор return:


Operationable operation = (int x, int y)-> {
        
    if(y==0)
        return 0;
    return x/y;
};
        
System.out.println(operation.execute(20, 10)); //2
System.out.println(operation.execute(20, 0)); //0

Обобщенный функциональный інтерфейс

Функциональный інтерфейс может быть обобщенным, однак в лямбда-выражении використання обобщений не допускается. В цем случае нам треба типизировать об'єкт інтерфейса визначенийным типом, який потом буде застосовуватися в лямбда-выражении. Наприклад:


public class Program {

    public static void main(String[] args) {
        
        Operationable<Integer> operation1 = (x, y)-> x + y;
        Operationable<String> operation2 = (x, y) -> x + y;
        
        System.out.println(operation1.execute(20, 10)); //30
        System.out.println(operation2.execute("20", "10")); //2010
    } 
}
interface Operationable<T>{
    T execute(T x, T y);
}

Таким чином, при оголошенийии лямбда-выражения ему вже известно, якой тип параметри будуть представлять и якой тип они будуть возвращать.
