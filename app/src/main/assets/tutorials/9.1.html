Введение в лямбда-выражения

Останнє оновлення: 02.10.2025

Лямбда-вираження в мові Java представляють набор инструкций, которые можна выделить в отдельную переменную и затем многократно вызвать в различных местах программы.

Определение лямбда-выражения и функциональные інтерфейсы

Для создания лямбда-выражения застосовується лямбда-оператор (стрелка ->). Цет оператор разделяет лямбда-выражение на две части:

 (параметры) -> действия

левая часть (до стрелки) містить параметры через запятую, а правая часть (после стрелки) собственно представляє выполняемые действия лямбда-выражения.

Лямбда-выражение не виконується само по себе, а образует реализацию метода, определенного в функциональном інтерфейсе. Рассмотрим пример:


public class Program {

    public static void main(String[] args) {
        
        Operationable op;
        op = (x,y)->x+y;
        
        int result = op.execute(10, 20);
        System.out.println(result); //30
    }   
}
interface Operationable{
    int execute(int x, int y);
}

Здесь в роли функционального інтерфейса выступает інтерфейс Operationable, в котором определен один метод без реализации - метод execute. Данный метод приймає два параметра - целых числа, и повертає некоторое целое число.

Щоб оголосити и використовувати лямбда-выражение, основная программа разбивается на ряд этапов:

  1. Определение ссылки на функциональный інтерфейс:

    Operationable op;

  2. Створення лямбда-выражения:

    op = (x,y)->x+y;

    Причем параметры лямбда-выражения соответствуют параметрам единственного метода інтерфейса Operationable, а результат соответствует возвращаемому результату метода інтерфейса. При цем нам не треба використовувати ключевое слово return для возврата результата из лямбда-выражения.

    Так, в методе інтерфейса оба параметра представляють тип int, значит, в теле лямбда-выражения мы можемо применить к ним сложение. Результат сложения також представляє тип int, об'єкт которого повертаєся методом інтерфейса.

  3. Использование лямбда-выражения в виде вызова метода інтерфейса:

    int result = operation.execute(10, 20);

    Так як в лямбда-выражении определена операция сложения параметров, результатом метода буде сумма чисел 10 и 20.

По факту лямбда-выражения є в некотором роде сокращенной формой внутренних анонимных классов, которые ранїї применялись в Java. В частности, предыдущий пример мы можемо переписать наступним чином:


public class Program {

    public static void main(String[] args) {
        
        Operationable op = new Operationable(){
        
            public int execute(int x, int y){
            
				return x + y;
            }
        };
         int z = op.execute(20, 10);
         System.out.println(z); // 30
    }   
}
interface Operationable{
    int execute(int x, int y);
}

При цем для одного функционального інтерфейса мы можемо определить множество лямбда-выражений. Например:


Operationable sum = (int x, int y)-> x + y;
Operationable sub = (int x, int y)-> x - y;
Operationable mul = (int x, int y)-> x * y;
        
System.out.println(sum.execute(20, 10)); //30
System.out.println(sub.execute(20, 10)); //10
System.out.println(mul.execute(20, 10)); //200

Если лямбда-выражение приймає один параметр, то скобки вокруг параметров можна не використовувати:


public class Program {

    public static void main(String[] args) {
        
        Printable printer = message -> System.out.println(message);

        printer.print("Hello World");
        printer.print("Hello Work");
    }   
}
interface Printable{
    void print(String message);
}

Здесь лямбда-выражение printer представляє функциональный інтерфейс Printable, единственный метод которого - print приймає лише один параметр.

Другая ситуация - в методе функционального інтерфейса есть параметр, но в лямбда-выражении он не використовується. В цем случае вместо параметра можна указать прочерк:


public class Program {

    public static void main(String[] args) {
        
        Printable printer = _ -> System.out.println("Test");

        // при вызове параметру все равно треба передать значение
        // но оно не имїїт значения, оскільки не буде використовуватися
        printer.print(null);    // Test
        printer.print("brr");   // Test
    }   
}
interface Printable{
    void print(String message);
}

Если метод інтерфейса вообще не приймає нияких параметров, то при определении лямбда-выражения указываются пустые скобки:


public class Program {

    public static void main(String[] args) {
        
        Hello speaker = () -> System.out.println("Привед");

        speaker.sayHello();
    }   
}
interface Hello{
    void sayHello();
}

Отложенное выполнение

Одним из ключевых моментов в использовании лямбд є отложенное выполнение (deferred execution). То есть мы определяем в одном месте программы лямбда-выражение и затем можемо его вызывать при необхідности неопределенное количество раз в различных частях программы. Отложенное выполнение может потребоваться, к примеру, в следующих случаях:

  • Выполнение кода отдельном потоке

  • Выполнение одного и того же кода декілька раз

  • Выполнение кода в результате якого-то события

  • Выполнение кода лише в том случае, когда он действительно необходим и якщо он необходим

Лямбды и локальные переменные

Лямбда-выражение может використовувати переменные, которые оголошенийы во вне в болїї общей области видимости - на уровне класса або метода, в котором лямбда-выражение определено. Однак в зависимости от того, як и где определены переменные, можуть различаться способы их использования в лямбдах. Рассмотрим первый пример - использования переменных уровня класса:


public class Program {

    static int x = 10;
    static int y = 20;
    public static void main(String[] args) {
        
        Operation op = ()->{ 
            
            x=30;
            return x+y;
        };
        System.out.println(op.execute()); // 50
        System.out.println(x); // 30 - значение x изменилось
    }
}
interface Operation{
    int execute();
}

Переменные x и y оголошенийы на уровне класса, и в лямбда-выражении мы их можемо отримати и даже изменить. Так, в данном случае после виконання выражения змінюєся значение змінної x.

Теперь розглянемо другой пример - локальные переменные на уровне метода:


public static void main(String[] args) {
        
    int n = 70;
    int m = 30;
    Operation op = ()->{ 
            
        //n = 100; - так нельзя сделать
        return m+n;
    };
	// n=100;  - так тоже нельзя
    System.out.println(op.execute()); // 100
}

Локальные переменные уровня метода мы також можемо використовувати в лямбдах, но изменять их значение нельзя. Если мы попробуем це сделать, то среда разработки (Netbeans) может нам высветить ошибку и то, что такую переменную треба пометить за допомогою ключевого слова final, то есть сделать константой: final int n=70;. Однак це необязательно.

Болїї того, мы не сможемо изменить значение змінної, которая використовується в лямбда-выражении, вне цього выражения. То есть даже якщо такая переменная не оголошенийа як константа, по сути она є константой.

Блоки кода в лямбда-выражениях

Существуют два типу лямбда-выражений: однострочное выражение и блок кода. Примеры однострочных выражений демонстрировались выше. Блочные выражения обрамляются фигурными скобками. В блочных лямбда-выражениях можна використовувати внутренние вложенные блоки, циклы, конструкции if, switch, создавать переменные и т.д. Если блочное лямбда-выражение повинно возвращать значение, то явным чином застосовується оператор return:


Operationable operation = (int x, int y)-> {
        
    if(y==0)
        return 0;
    return x/y;
};
        
System.out.println(operation.execute(20, 10)); //2
System.out.println(operation.execute(20, 0)); //0

Обобщенный функциональный інтерфейс

Функциональный інтерфейс может быть обобщенным, однак в лямбда-выражении использование обобщений не допускается. В цем случае нам треба типизировать об'єкт інтерфейса определенным типом, який потом буде застосовуватися в лямбда-выражении. Например:


public class Program {

    public static void main(String[] args) {
        
        Operationable<Integer> operation1 = (x, y)-> x + y;
        Operationable<String> operation2 = (x, y) -> x + y;
        
        System.out.println(operation1.execute(20, 10)); //30
        System.out.println(operation2.execute("20", "10")); //2010
    } 
}
interface Operationable<T>{
    T execute(T x, T y);
}

Таким чином, при оголошенийии лямбда-выражения ему вже известно, якой тип параметры будуть представлять и якой тип они будуть возвращать.
