CompletableFuture, обробка помилок та завершення асинхронних завдань

Останнє оновлення: 29.10.2025

Обработка ошибок

В прошлой статье був рассмотрен такой пример для работы с асинхронностью в мові Java, як тип CompletableFuture, який дозволяє устанавливать коллбеки для обработки результатов асинхронной задачи. Однак что, якщо при вычислениях возникнет ошибка? Наприклад, у нас есть следующая задача по вычислению факториала, где, якщо передаваемое число меньше 1, то генеруєся ошибка:


Supplier<Integer> factorialTask = () -> {

    // при некорректных данных генерируем ошибку
    if(number < 1) throw new RuntimeException("Number must be greater than 0");
    int n = 1;
    int result = 1;
    while(n <= number)  result *= n++; 
    return result; 
};

Как мы можемо обработать эту ошибку? Для обработки ошибок в API класса CompletedFuture определен ряд методів. Рассмотрим ряд из них:


CompletableFuture<T> exceptionally(Function<Throwable, ? extends T> fn)
CompletableFuture<T> exceptionallyCompose(Function<Throwable, ? extends CompletionStage<T>> fn)

Эти методы дозволяють установить обратный вызов, який виконується при возникновении ошибки на предыдущем этапе:

  • exceptionally(): вычисляет результат на основе ошибки, якщо она возникла на предыдущем этапе. То есть фактически обратный вызов производит переход Throwable -> T

  • exceptionallyCompose(): вызывает функцию для исключения, якщо она возникла на предыдущем этапе, и выполняет возвращенный результат - об'єкт CompletableFuture. То есть фактически обратный вызов производит переход Throwable -> CompletableFuture<T>

Рассмотрим на примере вычисления факториала:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.Function;
import java.util.function.Consumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        int number = -5;  // исходное число для вычисления факториала
       
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> factorialTask = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0");
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++; 
            return result; 
        };
        // задача для вывода финального результата
        Consumer<Integer> printTask = result -> System.out.printf("Final result: %d\n", result);

        // задача для обработки ошибки
        Function<Throwable, Integer> printException = ex -> { 
            System.out.println(ex.getMessage());    
            throw new RuntimeException(ex.getMessage());
        };

        // определяем об'єкт Executor, який буде выполнять задачи
        ExecutorService executor = Executors.newCachedThreadPool();
        CompletableFuture
                .supplyAsync(factorialTask, executor)
                .exceptionally(printException)
                .thenAccept(printTask);

        System.out.println("Main thread works...");
        System.out.println("Main thread finished...");
        
        executor.close();  // закрываем исполнителя
    }
}

Рассмотрим ключевые моменты. Прежде всего основная задача, которая виконується с помощь CompletedFuture - це задача вычисления факториала factorialTask. Эта задача повертає целое число (значение Integer)

Для виконання задачи и вызова коллбеков, обрабатывающих її результат, определяем следующий код:


CompletableFuture
    .supplyAsync(factorialTask, executor)
    .exceptionally(printException)
    .thenAccept(printTask);

  • supplyAsync(factorialTask, executor)

    Устанавливаем выполняемую асинхронную задачу и об'єкт Executor, який выполняет эту задачу.

    Если передачаемое для вычисления факториала число некорректно, то генерируем исключение

    if(number < 1) throw new RuntimeException("Number must be greater than 0");

  • exceptionally(printException)

    Для обработки исключения определяем задачу printException, которая просто выводит информацию об исключении на консоль:

    
Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getMessage());    
    throw new RuntimeException(ex.getMessage());
};

    Здесь варто відзначити, что поскольку задача факториала повертає значение типу Integer, то и эта задача повинна возвращать значение типу Integer. Однак в данном случае мы ничего не возвращаем и просто повторно генерируем исключение (причем непроверяемое исключение). Повторная генерация исключения позволит избежать виконання следующего колбека, для которого требуется результат вычисления факториала.

  • thenAccept(printTask)

    Задача printTask приймає результат - результат вычисления факториала и просто выводит его на консоль

В программе входное число є отрицательным:

int number = -5;

Соответственно при выполнении мы столкнемся с ошибкой и получим консольный вывод наподобие следующего:


Main thread started...
java.lang.RuntimeException: Number must be greater than 0
Main thread works...
Main thread finished...

И тут треба обратить внимание, что последний коллбек - printTask не виконується, а выполнение основного потока не переривається, хотя мы повторно генерируем исключение при его обработке.

Если же входное число буво бы больше 0:

int number = 5;

то мы увидим результат работы коллбека из вызова thenAccept(printTask), а коллбек из вызова exceptionally(printException) не буде выполняться:


Main thread started...
Final result: 120
Main thread works...
Main thread finished...

Естественно це не единственно возможная схема обработки ошибок. Наприклад, в примере выше мы генерировали повторно исключение. Но мы могли бы и сгенерировать результат, якщо переданное значение генерировало ошибку:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.Function;
import java.util.function.Consumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
 
        int age = -5;
       
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> validateAge = () -> {

            if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");
            return age; 
        };
        // задача для вывода финального результата
        Consumer<Integer> printAge = result -> System.out.printf("Final age: %d\n", result);

        // задача для обработки ошибки
        Function<Throwable, Integer> printException = ex -> { 
            System.out.println(ex.getMessage());    
            return 18;  // возвращаем некоторое значение по умолчанию
        };

        // определяем об'єкт Executor, який буде выполнять задачи
        ExecutorService executor = Executors.newCachedThreadPool();
        CompletableFuture
                .supplyAsync(validateAge, executor)
                .exceptionally(printException)
                .thenAccept(printAge);

        // закрываем исполнителя
        executor.close();  
    }
}

Здесь мы сначала проверяем возраст - значение змінної age в задаче validateAge: якщо оно не входит в неготорый диапазон, генерируем исключение:


Supplier<Integer> validateAge = () -> {

    if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");
    return age; 
};

Если генеруєся исключение, то вызывается задача printException:


Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getMessage());    
    return 18;  // возвращаем некоторое значение по умолчанию
};

Здесь також выводим сообщение об ошибке, но при цем возвращаем некоторое значение по умолчанию.

Таким чином при определении следующей цепочки:


CompletableFuture
    .supplyAsync(validateAge, executor)
    .exceptionally(printException)
    .thenAccept(printAge);

Если в validateAge возникает исключение, оно передаєся в задачу printException, которая генерує конечный результат. И цей результа передаєся в задачу printAge. То есть в данном случае последний коллбек - printAge виконується в любом случае, даже якщо возникает ошибка. Консольный вывод в данном случае


java.lang.RuntimeException: Age is invalid
Final age: 18

CompletionException

Исключение, которое генеруєся в задачах CompletedFuture представляє тип CompletionException, которое найдеся от RuntimeException и фактически є оборткой над исключением. Таким чином, когда мы генерируем исключение


if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");

В реальности в задачу для обработки исключения из вызова exceptionally() передаєся не сгенерированное исключения RuntimeException, а обертка над ним в виде об'єкта CompletionException. Тому в сообщении об исключении мы видим изначальный тип исключения - "java.lang.RuntimeException: Age is invalid". Щоб обратиться к цему внутреннему исключению, мы можемо використовувати метод getCause(), який определен в інтерфейсе Throwble:


Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getCause().getMessage());    
    return 18;  // возвращаем некоторое значение по умолчанию
};

Соответственно теперь при выводе на консоль не буде указываться тип исключения:


Age is invalid
Final age: 18

Завершение CompletedFuture

Асинхронные задачи об'єкта CompletableFuture можуть завершиться двумя способами: с результатом або с неперехваченным исключением. Для обработки обоих случаев можна використовувати метод whenComplete():


CompletableFuture<T> whenComplete(BiConsumer<? super T, ? super Throwable> action)

В качестве параметра цей метод приймає действие, которое соответствует методу accept() функционального інтерфейса BiConsumer<T,U>:

void accept(T t, U u)

То есть действие ничего не повертає и приймає два параметра, причем второй параметр повинен представлять об'єкт Throwable. В случае с CompletableFuture первый параметр буде представлять результат асинхронной задачи, а второй параметр - исключение, якщо оно возвникло в процессе виконання асинхронной задачи.

Если результат був успешно вычислен, а исключения не возникло, то второй параметр равен null. Если же наоборот - при выполнении задачи возникло исключение, то первый параметр равен null

Общая схема работы коллбека, який передаєся в данный метод, выглядит наступним чином:


f.whenComplete((result, error) -> {

      if (error == null){  // якщо исключения нет
      
         обробляє результат result;
      }
      else {    // якщо возникло исключение

         обрабатываем ошибку error;
      }
});

Рассмотрим на примере вычисления факториала:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.BiConsumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        // определяем об'єкт Executor, який буде выполнять задачи
        ExecutorService executor = Executors.newCachedThreadPool();
        work(executor, 5);
        work(executor, -5);

        System.out.println("Main thread finished...");

        executor.close(); 
    }

    static void work(Executor executor, int number){
        // определяем задачу, которая вычисляет факториал
        Supplier<Integer> factorialTask = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0. Invalid number: " + number);
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++;
            return result; 
        };

        BiConsumer<Integer, Throwable> completedTask = (result, ex) -> {

            if (ex == null)  // якщо нет исключения
                System.out.printf("factorial of %d is %d\n", number, result);
            else     // якщо возникло исключение
                System.out.println(ex.getCause().getMessage());
        };

        CompletableFuture
            .supplyAsync(factorialTask, executor)
            .whenComplete(completedTask);
    }
}

Основные действия здесь выполняются в методе work(), який приймає число для вычисления факториала и об'єкт Executor для виконання задач.

Сначала определяем собственно задачу для вычисления факториала:


Supplier<Integer> factorialTask = () -> {

    if(number < 1) throw new RuntimeException("Number must be greater than 0. Invalid number: " + number);
    int n = 1;
    int result = 1;
    while(n <= number)  result *= n++;
    return result; 
};

Если переданное число здесь меньше 1, то генерируем исключение.

Для обработки завершения задачи (вне зависимости успешного або неудачного) определяем отдельную задачу типу BiConsumer:


BiConsumer<Integer, Throwable> completedTask = (result, ex) -> {

    if (ex == null)  // якщо нет исключения
        System.out.printf("factorial of %d is %d\n", number, result);
    else     // якщо возникло исключение
         System.out.println(ex.getCause().getMessage());
};

Поскольку результат вычисления факториала представляє тип Integer, то параметры задачи представляють типы Integer и Throwable. В целях демонстрации задача просто выводит результат або сообщение об ошибке на консоль.

В конце делаем цепочку вызовов, в которые передаем выше определенные коллбеки:


CompletableFuture
    .supplyAsync(factorialTask, executor)
    .whenComplete(completedTask);

В методе main() вызываем метод work(), передавая в него для тестирования корркетные и некорректные значения:


ExecutorService executor = Executors.newCachedThreadPool();

work(executor, 5);
work(executor, -5);

В итоге мы получим консольный вывод типу следующего (порядок вывода сообщение не детерминирован):


Main thread started...
factorial of 5 is 120
Main thread finished...
Number must be greater than 0. Invalid number: -5
