CompletableFuture, обробка помилок та завершення асинхронних завдань

Останнє оновлення: 29.10.2025

Обробка ошибок

В прошлой статье був рассмотрен такой пример для работы с асинхронностью в мові Java, як тип CompletableFuture, який дозволяє устанавливать коллбеки для обработки результатів асинхронной завданьі. Однак что, якщо при вычислениях возникнет ошибка? Наприклад, у нас есть следующая завданьа по вычислению факториала, где, якщо передаваемое число менше 1, то генеруєся ошибка:


Supplier<Integer> factorialTask = () -> {

    // при некорректных данных генерируем ошибку
    if(number < 1) throw new RuntimeException("Number must be greater than 0");
    int n = 1;
    int result = 1;
    while(n <= number)  result *= n++; 
    return result; 
};

Как мы можемо обпрацювати эту ошибку? Для обработки ошибок в API класа CompletedFuture визначений ряд методів. Рассмотрим ряд из них:


CompletableFuture<T> exceptionally(Function<Throwable, ? extends T> fn)
CompletableFuture<T> exceptionallyCompose(Function<Throwable, ? extends CompletionStage<T>> fn)

Эти методи дозволяють установить обратный вызов, який виконується при возникновении ошибки на предыдущем этапе:

  • exceptionally(): вычисляет результат на основе ошибки, якщо она возникла на предыдущем этапе. То есть фактически обратный вызов производит переход Throwable -> T

  • exceptionallyCompose(): вызывает функцію для винятки, якщо она возникла на предыдущем этапе, и виконує возвращенный результат - об'єкт CompletableFuture. То есть фактически обратный вызов производит переход Throwable -> CompletableFuture<T>

Рассмотрим на примере вычисления факториала:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.Function;
import java.util.function.Consumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        int number = -5;  // исходное число для вычисления факториала
       
        // определяем завданьу, которая вычисляет факториал
        Supplier<Integer> factorialTask = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0");
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++; 
            return result; 
        };
        // завданьа для вивіда финального результата
        Consumer<Integer> printTask = result -> System.out.printf("Final result: %d\n", result);

        // завданьа для обработки ошибки
        Function<Throwable, Integer> printException = ex -> { 
            System.out.println(ex.getMessage());    
            throw new RuntimeException(ex.getMessage());
        };

        // определяем об'єкт Executor, який буде выполнять завданьі
        ExecutorService executor = Executors.newCachedThreadPool();
        CompletableFuture
                .supplyAsync(factorialTask, executor)
                .exceptionally(printException)
                .thenAccept(printTask);

        System.out.println("Main thread works...");
        System.out.println("Main thread finished...");
        
        executor.close();  // закрываем исполнителя
    }
}

Рассмотрим ключевые моменты. Прежде всего основная завданьа, которая виконується с помощь CompletedFuture - це завданьа вычисления факториала factorialTask. Эта завданьа повертає целое число (значення Integer)

Для виконання завданьі и вызова коллбеков, обрабатывающих її результат, определяем следующий код:


CompletableFuture
    .supplyAsync(factorialTask, executor)
    .exceptionally(printException)
    .thenAccept(printTask);

  • supplyAsync(factorialTask, executor)

    Устанавливаем выполняемую асинхронную завданьу и об'єкт Executor, який виконує эту завданьу.

    Если передачаемое для вычисления факториала число некорректно, то генерируем виняток

    if(number < 1) throw new RuntimeException("Number must be greater than 0");

  • exceptionally(printException)

    Для обработки винятки определяем завданьу printException, которая просто виводить информацию об исключении на консоль:

    
Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getMessage());    
    throw new RuntimeException(ex.getMessage());
};

    Здесь варто відзначити, что поскольку завданьа факториала повертає значення типу Integer, то и эта завданьа повинна возвращать значення типу Integer. Однак у даному випадку мы ничего не повертаємо и просто повторно генерируем виняток (причем непроверяемое виняток). Повторная генерація винятки дозволить избежать виконання следующего колбека, для которого требуется результат вычисления факториала.

  • thenAccept(printTask)

    Задача printTask приймає результат - результат вычисления факториала и просто виводить его на консоль

В програме входное число є отрицательным:

int number = -5;

Соответственно при выполнении мы столкнемся с ошибкой и получим консольный вивід напідобие следующего:


Main thread started...
java.lang.RuntimeException: Number must be greater than 0
Main thread works...
Main thread finished...

И тут треба обратить внимание, что післядний коллбек - printTask не виконується, а виконання основного потіка не переривається, хотя мы повторно генерируем виняток при его обработке.

Если же входное число буво бы больше 0:

int number = 5;

то мы увидим результат работы коллбека из вызова thenAccept(printTask), а коллбек из вызова exceptionally(printException) не буде выполняться:


Main thread started...
Final result: 120
Main thread works...
Main thread finished...

Естественно це не единственно возможная схема обработки ошибок. Наприклад, в примере выше мы генерировали повторно виняток. Но мы могли бы и сгенерировать результат, якщо переданное значення генерировало ошибку:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.Function;
import java.util.function.Consumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
 
        int age = -5;
       
        // определяем завданьу, которая вычисляет факториал
        Supplier<Integer> validateAge = () -> {

            if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");
            return age; 
        };
        // завданьа для вивіда финального результата
        Consumer<Integer> printAge = result -> System.out.printf("Final age: %d\n", result);

        // завданьа для обработки ошибки
        Function<Throwable, Integer> printException = ex -> { 
            System.out.println(ex.getMessage());    
            return 18;  // повертаємо некоторое значення по умолчанию
        };

        // определяем об'єкт Executor, який буде выполнять завданьі
        ExecutorService executor = Executors.newCachedThreadPool();
        CompletableFuture
                .supplyAsync(validateAge, executor)
                .exceptionally(printException)
                .thenAccept(printAge);

        // закрываем исполнителя
        executor.close();  
    }
}

Здесь мы сначала проверяем возраст - значення змінної age в завданье validateAge: якщо оно не входит в неготорый диапазон, генерируем виняток:


Supplier<Integer> validateAge = () -> {

    if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");
    return age; 
};

Если генеруєся виняток, то вызывается завданьа printException:


Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getMessage());    
    return 18;  // повертаємо некоторое значення по умолчанию
};

Здесь також вивідим сообщение об ошибке, но при цем повертаємо некоторое значення по умолчанию.

Таким чином при визначенийии следующей цепочки:


CompletableFuture
    .supplyAsync(validateAge, executor)
    .exceptionally(printException)
    .thenAccept(printAge);

Если в validateAge возникает виняток, оно передаєся в завданьу printException, которая генерує конечный результат. И цей результа передаєся в завданьу printAge. То есть у даному випадку післядний коллбек - printAge виконується в любом случае, даже якщо возникает ошибка. Консольный вивід у даному випадку


java.lang.RuntimeException: Age is invalid
Final age: 18

CompletionException

Виняток, которое генеруєся в завданьах CompletedFuture представляє тип CompletionException, которое найдеся от RuntimeException и фактически є оборткой над винятокм. Таким чином, когда мы генерируем виняток


if(age > 110 || age < 1) throw new RuntimeException("Age is invalid");

В реальности в завданьу для обработки винятки из вызова exceptionally() передаєся не сгенерированное винятки RuntimeException, а обертка над ним в виде об'єкта CompletionException. Тому в сообщении об исключении мы видим изначальный тип винятки - "java.lang.RuntimeException: Age is invalid". Щоб обратиться к цему внутреннему исключению, мы можемо використовувати метод getCause(), який визначений в інтерфейсе Throwble:


Function<Throwable, Integer> printException = ex -> { 
    System.out.println(ex.getCause().getMessage());    
    return 18;  // повертаємо некоторое значення по умолчанию
};

Соответственно теперь при вивіде на консоль не буде указываться тип винятки:


Age is invalid
Final age: 18

Завершення CompletedFuture

Асинхронные завданьі об'єкта CompletableFuture можуть завершиться двумя способами: с результатом або с неперехваченным винятокм. Для обработки обоих случаев можна використовувати метод whenComplete():


CompletableFuture<T> whenComplete(BiConsumer<? super T, ? super Throwable> action)

В качестве параметра цей метод приймає действие, которое відповідає методу accept() функционального інтерфейса BiConsumer<T,U>:

void accept(T t, U u)

То есть действие ничего не повертає и приймає два параметра, причем второй параметр повинен представлять об'єкт Throwable. В случае с CompletableFuture первый параметр буде представлять результат асинхронной завданьі, а второй параметр - виняток, якщо оно возвникло в процесе виконання асинхронной завданьі.

Если результат був успешно вычислен, а винятки не возникло, то второй параметр равен null. Если же наоборот - при выполнении завданьі возникло виняток, то первый параметр равен null

Общая схема работы коллбека, який передаєся в данный метод, выглядит наступним чином:


f.whenComplete((result, error) -> {

      if (error == null){  // якщо винятки нет
      
         обробляє результат result;
      }
      else {    // якщо возникло виняток

         обробляємо ошибку error;
      }
});

Рассмотрим на примере вычисления факториала:


import java.util.concurrent.*;
import java.util.function.Supplier;
import java.util.function.BiConsumer;
 
class Program{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Main thread started...");
 
        // определяем об'єкт Executor, який буде выполнять завданьі
        ExecutorService executor = Executors.newCachedThreadPool();
        work(executor, 5);
        work(executor, -5);

        System.out.println("Main thread finished...");

        executor.close(); 
    }

    static void work(Executor executor, int number){
        // определяем завданьу, которая вычисляет факториал
        Supplier<Integer> factorialTask = () -> {

            if(number < 1) throw new RuntimeException("Number must be greater than 0. Invalid number: " + number);
            int n = 1;
            int result = 1;
            while(n <= number)  result *= n++;
            return result; 
        };

        BiConsumer<Integer, Throwable> completedTask = (result, ex) -> {

            if (ex == null)  // якщо нет винятки
                System.out.printf("factorial of %d is %d\n", number, result);
            else     // якщо возникло виняток
                System.out.println(ex.getCause().getMessage());
        };

        CompletableFuture
            .supplyAsync(factorialTask, executor)
            .whenComplete(completedTask);
    }
}

Основные действия здесь виконуються в методі work(), який приймає число для вычисления факториала и об'єкт Executor для виконання завдань.

Сначала определяем собственно завданьу для вычисления факториала:


Supplier<Integer> factorialTask = () -> {

    if(number < 1) throw new RuntimeException("Number must be greater than 0. Invalid number: " + number);
    int n = 1;
    int result = 1;
    while(n <= number)  result *= n++;
    return result; 
};

Если переданное число здесь менше 1, то генерируем виняток.

Для обработки завершення завданьі (вне зависимости успешного або неудачного) определяем отдельную завданьу типу BiConsumer:


BiConsumer<Integer, Throwable> completedTask = (result, ex) -> {

    if (ex == null)  // якщо нет винятки
        System.out.printf("factorial of %d is %d\n", number, result);
    else     // якщо возникло виняток
         System.out.println(ex.getCause().getMessage());
};

Поскольку результат вычисления факториала представляє тип Integer, то параметри завданьі представляютьь типи Integer и Throwable. В целях демонстрации завданьа просто виводить результат або сообщение об ошибке на консоль.

В конце делаем цепочку вызовов, в которые передаємо выше визначенийные коллбеки:


CompletableFuture
    .supplyAsync(factorialTask, executor)
    .whenComplete(completedTask);

В методі main() вызываем метод work(), передавая в него для тестирования корркетные и некорректные значення:


ExecutorService executor = Executors.newCachedThreadPool();

work(executor, 5);
work(executor, -5);

В итоге мы получим консольный вивід типу следующего (порядок вивіда сообщение не детерминирован):


Main thread started...
factorial of 5 is 120
Main thread finished...
Number must be greater than 0. Invalid number: -5
