Встроенные функциональные інтерфейсы

Останнє оновлення: 02.10.2025

Для работы с лямбда-выражениями нужны функциональные інтерфейсы. Однак вместо того, чтобы самим визначати свои інтерфейсы, мы можемо взять вже говотые, которые есть в JDK. ОНи покрывают наиболїї часто встречающиеся случае и можуть використовуватися в различных ситуациях. Рассмотрим основные из цих інтерфейсов:

  • Predicate<T>

  • Consumer<T>

  • Function<T,R>

  • BiFunction<T, U, R>

  • Supplier<T>

  • UnaryOperator<T>

  • BinaryOperator<T>

Predicate<T>

Функциональный інтерфейс Predicate<T> проверяет соблюдение некоторого условия. Если оно соблюдается, то повертаєся значення true. В качестве параметра лямбда-выражение приймає об'єкт типу T:


public interface Predicate<T> {
	boolean test(T t);
}

Наприклад:


import java.util.function.Predicate;

public class Program {

    public static void main(String[] args) {
        
        Predicate<Integer> isPositive = x -> x > 0;
        
        System.out.println(isPositive.test(5)); // true
        System.out.println(isPositive.test(-7)); // false
	}
}

BinaryOperator<T>

BinaryOperator<T> приймає в качестве параметра два об'єкта типу T, виконує над ними бинарную операцію и повертає її результат також в виде об'єкта типу T:


public interface BinaryOperator<T> {
	T apply(T t1, T t2);
}

Наприклад:


import java.util.function.BinaryOperator;

public class Program {

    public static void main(String[] args) {
        
        BinaryOperator<Integer> multiply = (x, y) -> x*y;
		
        System.out.println(multiply.apply(3, 5)); // 15
        System.out.println(multiply.apply(10, -2)); // -20
	}
}

UnaryOperator<T>

UnaryOperator<T> приймає в качестве параметра об'єкт типу T, виконує над ними операції и повертає результат операцій в виде об'єкта типу T:


public interface UnaryOperator<T> {
	T apply(T t);
}

Наприклад:


import java.util.function.UnaryOperator;

public class Program {

    public static void main(String[] args) {
        
        UnaryOperator<Integer> square = x -> x*x;
        System.out.println(square.apply(5)); // 25
	}
}

Function<T,R>

Функциональный інтерфейс Function<T,R> представляє функцію перехода от об'єкта типу T к об'єкту типу R:


public interface Function<T, R> {
	R apply(T t);
}

Наприклад:


import java.util.function.Function;

public class Program {

    public static void main(String[] args) {
        
        Function<Integer, String> convert = x-> String.valueOf(x) + " долларов";
        System.out.println(convert.apply(5)); // 5 долларов
	}
}

BiFunction<T, U, R>

Функциональный інтерфейс BiFunction<T, U, R> представляє функцію с двумя параметрами типу T и U и результатом типу R:


public interface Function<T, U, R> {
	R apply(T t, U u);
}

Наприклад:


import java.util.function.BiFunction;

public class Program {

    public static void main(String[] args) {
         
        BiFunction<Integer, Integer, Integer> sum = (x, y) -> x + y;
         
        System.out.println(sum.apply(3, 5)); // 8
        System.out.println(sum.apply(10, -2)); // 8
    } 
}

Consumer<T>

Consumer<T> виконує некоторое действие над об'єктом типу T, при цем ничего не возвращая:


public interface Consumer<T> {
	void accept(T t);
}

Наприклад:


import java.util.function.Consumer;

public class Program {

    public static void main(String[] args) {
        
        Consumer<Integer> printer = x-> System.out.printf("%d долларов \n", x);
        printer.accept(600); // 600 долларов
	}
}

Supplier<T>

Supplier<T> не приймає нияких аргументов, но повинен возвращать об'єкт типу T:


public interface Supplier<T> {
	T get();
}

Данный інтерфейс використовуються для так называемых ленивых вычислений, когда результат вычисления может потребоваться лишь спустя некоторое час. Наприклад:


import java.util.function.Supplier;

public class Program {

    public static void main(String[] args) {
         
        Supplier<Integer> randomValue = ()-> (int)(Math.random() * 10) + 1;
         
        System.out.println("Случайное значення");
        System.out.println(randomValue.get());  // реальные вычисления происходят тут
    } 
}

У даному випадку змінна randomValue представляє выражение по вычислению случайного значення, но реальное вычисление происходит, когда вызвается метод get() для получения цейого значення.
