Обробка анотацій під час виконання

Останнє оновлення: 01.11.2025

Анотація сама по себе в мові Java ничего робить. Щоб використовувати анотацію якимось чином, нам потрібно її обробляти. Під час виконання мы можемо отримати дані анотацій за допомогою рефлексии. Для цього в пакеті java.lang.reflect визначений інтерфейс AnnotatedElement. Для работы с анотаціями цей інтерфейс оголошує ряд методів:


T getAnnotation(Class<T>)
T getDeclaredAnnotation(Class<T>)
T[] getAnnotationsByType(Class<T>)
T[] getDeclaredAnnotationsByType(Class<T>)    
Annotation[] getAnnotations()
Annotation[] getDeclaredAnnotations()
boolean isAnnotationPresent(Class<? extends Annotation> annotationClass)

Рассматривая эти методи, варто відзначити о различиях в анотаціях:

  • анотація к елементу типу - це анотація оголошення (declaration annotation). Для из получения призначенийы методи getDeclaredAnnotation() / getDeclaredAnnotations() / getDeclaredAnnotationByType()

    • ,

  • анотація к типу (type annotation) - це анотація типу. Для их получения призначенийы методи с суффиксом ByType.

    Методи getAnnotations() и getDeclaredAnnotations() повертають вообще все застосовуємоые к типу або его елементу анотації в виде масива об'єктов Annotation. Цет тип представляє інтерфейс, из методів которого можна відзначити метод annotationType() - он повертає клас конкретной анотації:

    
Class<? extends Annotation> annotationType()

    Поскольку класы Class, Field, Parameter, Method, Constructor и Package реалізують цей інтерфейс, то, получив за допомогою рефлексии визначенийный компонент програми, мы можемо отримати его анотації.

    Рассмотрим небольшой пример:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Retention(RetentionPolicy.RUNTIME)     // Анотація доступна во час виконання
@Target(ElementType.TYPE)             // Анотація застосовується к типум
@interface MyAnnotation {

    String value();
}

// застосовуємо анотацію
@MyAnnotation(value="Thing")
class Thing{
    String name;
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var obj = new Thing();
        var cl = obj.getClass();
        // отримуємо анотацію MyAnnotation
        MyAnnotation annot = cl.getAnnotation(MyAnnotation.class);
        System.out.println(annot);  // @MyAnnotation("Thing")

        if (annot != null){

            System.out.println(annot.value());  // Thing
        }
    }
}

    Здесь сначала определяем змінну класа, к которому застосовується анотація:

    var obj = new Thing();

    Отримуємо клас змінної:

    var cl = obj.getClass();

    И далїї отримуємо нашу анотацію MyAnnotation:

    MyAnnotation annot = cl.getAnnotation(MyAnnotation.class);

    Зверніть увагу, что в метод передаєся об'єкт класа анотації (у даному випадку MyAnnotation.class). В качестве результата метод повертає об'єкт некоторого прокси-класа, який реалізує інтерфейс MyAnnotation.

    Затем можна вызвать методи інтерфейса. Однак может быть, что у класа отсутствует данная анотація, и в цем случае метод getAnnotation() повертає null. Соответственно перед вызовом методів анотації необхідно проверить її об'єкт на null:

    
if (annot != null){

    System.out.println(annot.value());  // Thing
}

    Метод getAnnotation() повертає один об'єкт анотації. Если же к типу або его елементу застосовується декілька анотацій, и все их треба отримати, то можна використовувати метод getAnnotations(), який повертає масив об'єктов Annotation:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.annotation.Annotation;

@Retention(RetentionPolicy.RUNTIME) 
@Target({ElementType.TYPE, ElementType.FIELD, ElementType.METHOD}) 
@interface Description {

    String data();
}

@Retention(RetentionPolicy.RUNTIME) 
@Target({ElementType.TYPE, ElementType.FIELD, ElementType.METHOD}) 
@interface Name {

    String value();
}

@Name(value="Thing")
@Description(data="Represents some object with a name")
class Thing{
    String name;
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var obj = new Thing();
        var cl = obj.getClass();

        // отримуємо все анотації
        Annotation[] annotations = cl.getAnnotations();
        for(var annot: annotations){

             System.out.println(annot);
            // отримуємо конкретный тип анотації
            System.out.println(annot.annotationType());

        }
    }
}

    У даному випадку к класу Thing застосовується две анотації, и мы получим следующий консольный вивід:

    @Name("Thing")
interface Name
@Description(data="Represents some object with a name")
interface Description

    Практический пример

    Рассмотрим болїї практический пример применения анотацій. Для цього возьмем следующую програму:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.reflect.Field;

@Retention(RetentionPolicy.RUNTIME) 
@Target(ElementType.FIELD) 
@interface Range {

    int min();
    int max();
}

class Person{

    String name;

    @Range(min=1, max=110)
    int age;

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    String getName(){ return name; }
    int getAge(){ return age; }
}

class RangeValidator{

    public static boolean validate(Object obj) throws Exception{
        // Отримуємо клас об'єкта 
        var cl = obj.getClass();

        // Проверяем анотації на уровне полей
        for (Field field : cl.getDeclaredFields()) {
            // якщо к полю застосовується анотація Range
            if (field.isAnnotationPresent(Range.class) && field.getType()==int.class) {
                // отримуємо анотацію
                Range range = field.getAnnotation(Range.class);
                // делаем поле доступным для получения его значення
                field.setAccessible(true);
                // отримуємо значення поля
                int age = (int)field.get(obj);
                // проверяем значення поля
                if(age > range.max() || age < range.min()) return false;
            }
        }

        return true;
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Person tom = new Person("Tom", 41);
        validatePerson(tom);                // Person Tom is valid

        Person bob = new Person("Bob", 141);
        validatePerson(bob);                // Person Bob is invalid!!!
    }
    static void validatePerson(Person p) throws Exception {

        if(RangeValidator.validate(p)) System.out.printf("Person %s is valid\n", p.getName());
        else System.out.printf("Person %s is invalid!!!\n", p.getName());
    }
}

    Данная програма створює механизм, який дозволяє декларативно задавать правила для полей, а затем динамически проверять, відповідають ли реальные об'єкты цим правилам. Рассмотрим пошагово, что здесь происходит:

    • Створення анотації @Range

      
@Retention(RetentionPolicy.RUNTIME) 
@Target(ElementType.FIELD) 
@interface Range {

    int min();
    int max();
}

      Анотація имїїт два параметра, min и max, которые будуть хранить допустимый диапазон значений.

    • Клас Person имїїт поле age, которое аннотировано созданной нами аннотацией:

      
class Person{

    String name;

    @Range(min=1, max=110)
    int age;
    ......................
}

      Це "правило" означает, что допустимый возраст для Person повинен быть в диапазоне от 1 до 110.

    • Клас RangeValidator отвечает за проверку произвольного об'єкта. Для цього в нем визначений метод validate(), який приймає любой об'єкт obj и повертає true (якщо об'єкт валиден) або false (якщо нет).

      
public static boolean validate(Object obj) throws Exception{
        // Отримуємо клас об'єкта 
        var cl = obj.getClass();

        // Проверяем анотації на уровне полей
        for (Field field : cl.getDeclaredFields()) {
            // якщо к полю застосовується анотація Range
            if (field.isAnnotationPresent(Range.class) && field.getType()==int.class) {
                // отримуємо анотацію
                Range range = field.getAnnotation(Range.class);
                // делаем поле доступным для получения его значення
                field.setAccessible(true);
                // отримуємо значення поля
                int age = (int)field.get(obj);
                // проверяем значення поля
                if(age > range.max() || age < range.min()) return false;
            }
        }

        return true;
    }

      За допомогою рефлексии метод отримує "описание" класа переданного об'єкта (наприклад, класа Person). Далїї перебираем все поля класа

      for (Field field : cl.getDeclaredFields())

      Затем проверяем, чи є у текущего поля анотація @Range

      if (field.isAnnotationPresent(Range.class) ...)

      Если анотація найдена, то отримуємо саму анотацію и її параметри ( min=1 и max=110).

      Range range = field.getAnnotation(Range.class);

      Разрешаем доступ к полю, даже якщо бы оно буво private:

      field.setAccessible(true);

      За допомогою рефлексии отримуємо реальное значення поля age из конкретного об'єкта

      int age = (int)field.get(obj);

      И в конце сравниваем значення поля с границами из анотації (1 и 110). Если значення выходит за рамки, метод немедленно повертає false (об'єкт невалиден).

      if(age > range.max() || age < range.min()) return false;

      В противном случае повертаєся true - об'єкт считаєся валидным.

    • В класе Program для упрощени проверки визначає вспомогательный метод validatePerson и вызываем его, передавая в него пару типівых об'єктов.

    Консольный вивід програм:

    
Person Tom is valid
Person Bob is invalid!!!
