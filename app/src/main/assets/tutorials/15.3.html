Обробка анотацій під час виконання

Останнє оновлення: 01.11.2025

Аннотация сама по себе в мові Java ничего робить. Щоб використовувати аннотацию якимось чином, нам потрібно її обробляти. Під час виконання мы можемо отримати дані аннотаций за допомогою рефлексии. Для цього в пакеті java.lang.reflect определен інтерфейс AnnotatedElement. Для работы с аннотациями цей інтерфейс оголошує ряд методів:


T getAnnotation(Class<T>)
T getDeclaredAnnotation(Class<T>)
T[] getAnnotationsByType(Class<T>)
T[] getDeclaredAnnotationsByType(Class<T>)    
Annotation[] getAnnotations()
Annotation[] getDeclaredAnnotations()
boolean isAnnotationPresent(Class<? extends Annotation> annotationClass)

Рассматривая эти методы, варто відзначити о различиях в аннотациях:

  • аннотация к элементу типу - це аннотация оголошення (declaration annotation). Для из получения призначенийы методы getDeclaredAnnotation() / getDeclaredAnnotations() / getDeclaredAnnotationByType()

    • ,

  • аннотация к типу (type annotation) - це аннотация типу. Для их получения призначенийы методы с суффиксом ByType.

    Методи getAnnotations() и getDeclaredAnnotations() повертають вообще все применяемые к типу або его элементу аннотации в виде массива об'єктов Annotation. Цет тип представляє інтерфейс, из методів которого можна відзначити метод annotationType() - он повертає класс конкретной аннотации:

    
Class<? extends Annotation> annotationType()

    Поскольку классы Class, Field, Parameter, Method, Constructor и Package реализуют цей інтерфейс, то, получив за допомогою рефлексии определенный компонент программы, мы можемо отримати его аннотации.

    Рассмотрим небольшой пример:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Retention(RetentionPolicy.RUNTIME)     // Аннотация доступна во час виконання
@Target(ElementType.TYPE)             // Аннотация застосовується к типум
@interface MyAnnotation {

    String value();
}

// применяем аннотацию
@MyAnnotation(value="Thing")
class Thing{
    String name;
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var obj = new Thing();
        var cl = obj.getClass();
        // получаем аннотацию MyAnnotation
        MyAnnotation annot = cl.getAnnotation(MyAnnotation.class);
        System.out.println(annot);  // @MyAnnotation("Thing")

        if (annot != null){

            System.out.println(annot.value());  // Thing
        }
    }
}

    Здесь сначала определяем переменную класса, к которому застосовується аннотация:

    var obj = new Thing();

    Получаем класс змінної:

    var cl = obj.getClass();

    И далїї получаем нашу аннотацию MyAnnotation:

    MyAnnotation annot = cl.getAnnotation(MyAnnotation.class);

    Обратите внимание, что в метод передаєся об'єкт класса аннотации (в данном случае MyAnnotation.class). В качестве результата метод повертає об'єкт некоторого прокси-класса, який реализует інтерфейс MyAnnotation.

    Затем можна вызвать методы інтерфейса. Однак может быть, что у класса отсутствует данная аннотация, и в цем случае метод getAnnotation() повертає null. Соответственно перед вызовом методів аннотации необхідно проверить її об'єкт на null:

    
if (annot != null){

    System.out.println(annot.value());  // Thing
}

    Метод getAnnotation() повертає один об'єкт аннотации. Если же к типу або его элементу застосовується декілька аннотаций, и все их треба отримати, то можна використовувати метод getAnnotations(), який повертає массив об'єктов Annotation:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.annotation.Annotation;

@Retention(RetentionPolicy.RUNTIME) 
@Target({ElementType.TYPE, ElementType.FIELD, ElementType.METHOD}) 
@interface Description {

    String data();
}

@Retention(RetentionPolicy.RUNTIME) 
@Target({ElementType.TYPE, ElementType.FIELD, ElementType.METHOD}) 
@interface Name {

    String value();
}

@Name(value="Thing")
@Description(data="Represents some object with a name")
class Thing{
    String name;
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var obj = new Thing();
        var cl = obj.getClass();

        // получаем все аннотации
        Annotation[] annotations = cl.getAnnotations();
        for(var annot: annotations){

             System.out.println(annot);
            // получаем конкретный тип аннотации
            System.out.println(annot.annotationType());

        }
    }
}

    В данном случае к классу Thing застосовується две аннотации, и мы получим следующий консольный вывод:

    @Name("Thing")
interface Name
@Description(data="Represents some object with a name")
interface Description

    Практический пример

    Рассмотрим болїї практический пример применения аннотаций. Для цього возьмем следующую программу:

    
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.reflect.Field;

@Retention(RetentionPolicy.RUNTIME) 
@Target(ElementType.FIELD) 
@interface Range {

    int min();
    int max();
}

class Person{

    String name;

    @Range(min=1, max=110)
    int age;

    Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    String getName(){ return name; }
    int getAge(){ return age; }
}

class RangeValidator{

    public static boolean validate(Object obj) throws Exception{
        // Получаем класс об'єкта 
        var cl = obj.getClass();

        // Проверяем аннотации на уровне полей
        for (Field field : cl.getDeclaredFields()) {
            // якщо к полю застосовується аннотация Range
            if (field.isAnnotationPresent(Range.class) && field.getType()==int.class) {
                // получаем аннотацию
                Range range = field.getAnnotation(Range.class);
                // делаем поле доступным для получения его значения
                field.setAccessible(true);
                // получаем значение поля
                int age = (int)field.get(obj);
                // проверяем значение поля
                if(age > range.max() || age < range.min()) return false;
            }
        }

        return true;
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Person tom = new Person("Tom", 41);
        validatePerson(tom);                // Person Tom is valid

        Person bob = new Person("Bob", 141);
        validatePerson(bob);                // Person Bob is invalid!!!
    }
    static void validatePerson(Person p) throws Exception {

        if(RangeValidator.validate(p)) System.out.printf("Person %s is valid\n", p.getName());
        else System.out.printf("Person %s is invalid!!!\n", p.getName());
    }
}

    Данная программа створює механизм, який дозволяє декларативно задавать правила для полей, а затем динамически проверять, соответствуют ли реальные об'єкты цим правилам. Рассмотрим пошагово, что здесь происходит:

    • Створення аннотации @Range

      
@Retention(RetentionPolicy.RUNTIME) 
@Target(ElementType.FIELD) 
@interface Range {

    int min();
    int max();
}

      Аннотация имїїт два параметра, min и max, которые будуть хранить допустимый диапазон значений.

    • Класс Person имїїт поле age, которое аннотировано созданной нами аннотацией:

      
class Person{

    String name;

    @Range(min=1, max=110)
    int age;
    ......................
}

      Це "правило" означает, что допустимый возраст для Person повинен быть в диапазоне от 1 до 110.

    • Класс RangeValidator отвечает за проверку произвольного об'єкта. Для цього в нем определен метод validate(), який приймає любой об'єкт obj и повертає true (якщо об'єкт валиден) або false (якщо нет).

      
public static boolean validate(Object obj) throws Exception{
        // Получаем класс об'єкта 
        var cl = obj.getClass();

        // Проверяем аннотации на уровне полей
        for (Field field : cl.getDeclaredFields()) {
            // якщо к полю застосовується аннотация Range
            if (field.isAnnotationPresent(Range.class) && field.getType()==int.class) {
                // получаем аннотацию
                Range range = field.getAnnotation(Range.class);
                // делаем поле доступным для получения его значения
                field.setAccessible(true);
                // получаем значение поля
                int age = (int)field.get(obj);
                // проверяем значение поля
                if(age > range.max() || age < range.min()) return false;
            }
        }

        return true;
    }

      С помощью рефлексии метод отримує "описание" класса переданного об'єкта (наприклад, класса Person). Далїї перебираем все поля класса

      for (Field field : cl.getDeclaredFields())

      Затем проверяем, есть ли у текущего поля аннотация @Range

      if (field.isAnnotationPresent(Range.class) ...)

      Если аннотация найдена, то получаем саму аннотацию и її параметры ( min=1 и max=110).

      Range range = field.getAnnotation(Range.class);

      Разрешаем доступ к полю, даже якщо бы оно буво private:

      field.setAccessible(true);

      С помощью рефлексии получаем реальное значение поля age из конкретного об'єкта

      int age = (int)field.get(obj);

      И в конце сравниваем значение поля с границами из аннотации (1 и 110). Если значение выходит за рамки, метод немедленно повертає false (об'єкт невалиден).

      if(age > range.max() || age < range.min()) return false;

      В противном случае повертаєся true - об'єкт считаєся валидным.

    • В классе Program для упрощени проверки определяет вспомогательный метод validatePerson и вызываем его, передавая в него пару типовых об'єктов.

    Консольный вывод программ:

    
Person Tom is valid
Person Bob is invalid!!!
