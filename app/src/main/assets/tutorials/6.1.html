Потоки введення-виведення. Работа с файлами

Потоки введення-виведення

Останнє оновлення: 25.04.2018

Отличительной чертой многих языков программирования є работа с файлами и потоками. В Java основной функционал работы с потоками сосредоточен в классах из пакета java.io.

Ключевым понятием здесь є понятие потока. Хотя понятие "поток" в программировании довольно перегрвжено и может обозначать множество различных концепций. В данном случае применительно к работе с файлами и вводом-выводом мы будем говорить о потоке (stream), як об абстракции, которая використовується для чтения або записи информации (файлов, сокетов, текста консоли и т.д.).

Поток связан с реальным физическим устройством за допомогою системы ввода-вывода Java. У нас может быть определен поток, який связан с файлом и через який мы можемо вести чтение або запись файла. Це також может быть поток, связанный с сетевым сокетом, за допомогою которого можна отримати або отправить дані в сети. Все эти задачи: чтение и запись различных файлов, обмен информацией по сети, ввод-ввывод в консоли мы будем решать в Java за допомогою потоков.

Об'єкт, из которого можна считать дані, називається потоком ввода, а об'єкт, в який можна записывать дані, - потоком вывода. Наприклад, якщо треба считать вміст файла, то застосовується поток ввода, а якщо треба записать в файл - то поток вывода.

В основе всех классов, управляющих потоками байтов, находятся два абстрактных класса: InputStream (представляющий потоки ввода) и OutputStream (представляющий потоки вывода)

Но поскольку работать с байтами не очень удобно, то для работы с потоками символов буви добавлены абстрактные классы Reader (для чтения потоков символов) и Writer (для записи потоков символов).

Все остальные классы, работающие с потоками, є наследниками цих абстрактных классов. Основные классы потоков:

Потоки байтов

Класс InputStream

Класс InputStream є базовым для всех классов, управляющих байтовыми потоками ввода. Рассмотрим его основные методы:

  • int available(): повертає количество байтов, доступных для чтения в потоке

  • void close(): закрывает поток

  • int read(): повертає целочисленное представление следующего байта в потоке. Когда в потоке не останется доступных для чтения байтов, данный метод возвратит число -1

  • int read(byte[] buffer): считывает байты из потока в массив buffer. После чтения повертає число считанных байтов. Если ни одного байта не буво считано, то повертаєся число -1

  • int read(byte[] buffer, int offset, int length): считывает некоторое количество байтов, равное length, из потока в массив buffer. При цем считанные байты помещаются в массиве, начиная со смщения offset, то есть с элемента buffer[offset]. Метод повертає число успешно прочитанных байтов.

  • long skip(long number): пропускает в потоке при чтении некоторое количество байт, которое равно number

Класс OutputStream

Класс OutputStream є базовым классом для всех классов, которые работают с бинарными потоками записи. Свою функциональность он реализует через следующие методы:

  • void close(): закрывает поток

  • void flush(): очищает буфер вывода, записывая все его содержимое

  • void write(int b): записує в выходной поток один байт, який представлен целочисленным параметром b

  • void write(byte[] buffer): записує в выходной поток массив байтов buffer.

  • void write(byte[] buffer, int offset, int length): записує в выходной поток некоторое число байтов, равное length, из массива buffer, начиная со смщения offset, то есть с элемента buffer[offset].

Абстрактні класи Reader и Writer

Абстрактный класс Reader надає функционал для чтения текстовой информации. Рассмотрим его основные методы:

  • absract void close(): закрывает поток ввода

  • int read(): повертає целочисленное представление следующего символа в потоке. Если таких символов нет, и достигнут конец файла, то повертаєся число -1

  • int read(char[] buffer): считывает в массив buffer из потока символы, количество которых равно длине массива buffer. Повертає количество успешно считанных символов. При достижении конца файла повертає -1

  • int read(CharBuffer buffer): считывает в об'єкт CharBuffer из потока символы. Повертає количество успешно считанных символов. При достижении конца файла повертає -1

  • absract int read(char[] buffer, int offset, int count): считывает в массив buffer, начиная со смщения offset, из потока символы, количество которых равно count

  • long skip(long count): пропускает количество символов, равное count. Повертає число успешно пропущенных символов

Класс Writer определяет функционал для всех символьных потоков вывода. Его основные методы:

  • Writer append(char c): додає в конец выходного потока символ c. Повертає об'єкт Writer

  • Writer append(CharSequence chars): додає в конец выходного потока набор символов chars. Повертає об'єкт Writer

  • abstract void close(): закрывает поток

  • abstract void flush(): очищает буферы потока

  • void write(int c): записує в поток один символ, який имїїт целочисленное представление

  • void write(char[] buffer): записує в поток массив символов

  • absract void write(char[] buffer, int off, int len) : записує в поток лише декілька символов из массива buffer. Причем количество символов равно len, а отбор символов из массива начинается с индекса off

  • void write(String str): записує в поток строку

  • void write(String str, int off, int len): записує в поток из строки некоторое количество символов, которое равно len, причем отбор символов из строки начинается с индекса off

Функционал, описанный классами Reader и Writer, найдеся непосредственно классами символьных потоков, в частности классами FileReader и FileWriter соответственно, призначенийными для работы с текстовыми файлами.
