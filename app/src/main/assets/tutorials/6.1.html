Потіки введення-виведення. Работа с файлуми

Потіки введення-виведення

Останнє оновлення: 25.04.2018

Отличительной чертой многих языков програмирования є работа с файлуми и потіками. В Java основной функционал работы с потіками сосредоточен в класах из пакету java.io.

Ключевым понятием здесь є понятие потіка. Хотя понятие "потік" в програмировании досить перегрвжено и может обозначать множина различных концепций. У даному випадку стосовно к работе с файлуми и вводом-вивідом мы будем говорить о потіке (stream), як об абстракции, которая використовується для читання або записи информации (файлів, сокетов, текста консоли и т.д.).

Потік связан с реальным физическим устройством за допомогою системы ввода-вивіда Java. У нас может быть визначений потік, який связан с файлом и через який мы можемо вести читання або запис файлу. Це також может быть потік, связанный с сетевым сокетом, за допомогою которого можна отримати або отправить дані в сети. Все эти завданьі: читання и запис различных файлів, обмен информацией по сети, ввод-ввивід в консоли мы будем решать в Java за допомогою потіков.

Об'єкт, из которого можна считать дані, називається потіком ввода, а об'єкт, в який можна записывать дані, - потіком вивіда. Наприклад, якщо треба считать вміст файлу, то застосовується потік ввода, а якщо треба записать в файл - то потік вивіда.

В основе всех класов, управляющих потіками байтов, находятся два абстрактных класа: InputStream (представляющий потіки ввода) и OutputStream (представляющий потіки вивіда)

Но поскольку працювати с байтами не дуже удобно, то для работы с потіками символів буви добавлены абстрактные класы Reader (для читання потіков символів) и Writer (для записи потіков символів).

Все остальные класы, работающие с потіками, є наследниками цих абстрактных класов. Основные класы потіков:

Потіки байтов

Клас InputStream

Клас InputStream є базовым для всех класов, управляющих байтовыми потіками ввода. Рассмотрим его основные методи:

  • int available(): повертає кількість байтов, доступных для читання у потоці

  • void close(): закрывает потік

  • int read(): повертає целочисленное представление следующего байта у потоці. Когда у потоці не останется доступных для читання байтов, данный метод возвратит число -1

  • int read(byte[] buffer): считывает байты из потіка в масив buffer. Після читання повертає число считанных байтов. Если ни одного байта не буво считано, то повертаєся число -1

  • int read(byte[] buffer, int offset, int length): считывает некоторое кількість байтов, равное length, из потіка в масив buffer. При цем считанные байты помещаются в масиве, начиная со смщения offset, то есть с елемента buffer[offset]. Метод повертає число успешно прочитанных байтов.

  • long skip(long number): пропускает у потоці при чтении некоторое кількість байт, которое равно number

Клас OutputStream

Клас OutputStream є базовым класом для всех класов, которые работают с бинарными потіками записи. Свою функциональность он реалізує через следующие методи:

  • void close(): закрывает потік

  • void flush(): очищает буфер вивіда, записывая все его вміст

  • void write(int b): записує в выходной потік один байт, який представлен целочисленным параметром b

  • void write(byte[] buffer): записує в выходной потік масив байтов buffer.

  • void write(byte[] buffer, int offset, int length): записує в выходной потік некоторое число байтов, равное length, из масива buffer, начиная со смщения offset, то есть с елемента buffer[offset].

Абстрактні класи Reader и Writer

Абстрактный клас Reader надає функционал для читання текстовой информации. Рассмотрим его основные методи:

  • absract void close(): закрывает потік ввода

  • int read(): повертає целочисленное представление следующего символа у потоці. Если таких символів нет, и достигнут конец файлу, то повертаєся число -1

  • int read(char[] buffer): считывает в масив buffer из потіка символи, кількість которых равно длине масива buffer. Повертає кількість успешно считанных символів. При достижении конца файлу повертає -1

  • int read(CharBuffer buffer): считывает в об'єкт CharBuffer из потіка символи. Повертає кількість успешно считанных символів. При достижении конца файлу повертає -1

  • absract int read(char[] buffer, int offset, int count): считывает в масив buffer, начиная со смщения offset, из потіка символи, кількість которых равно count

  • long skip(long count): пропускает кількість символів, равное count. Повертає число успешно пропущенных символів

Клас Writer визначає функционал для всех символьных потіков вивіда. Его основные методи:

  • Writer append(char c): додає в конец выходного потіка символ c. Повертає об'єкт Writer

  • Writer append(CharSequence chars): додає в конец выходного потіка набор символів chars. Повертає об'єкт Writer

  • abstract void close(): закрывает потік

  • abstract void flush(): очищает буферы потіка

  • void write(int c): записує в потік один символ, який имїїт целочисленное представление

  • void write(char[] buffer): записує в потік масив символів

  • absract void write(char[] buffer, int off, int len) : записує в потік лише декілька символів из масива buffer. Причем кількість символів равно len, а отбор символів из масива начинается с индекса off

  • void write(String str): записує в потік рядок

  • void write(String str, int off, int len): записує в потік из рядки некоторое кількість символів, которое равно len, причем отбор символів из рядки начинается с индекса off

Функционал, описанный класами Reader и Writer, найдеся непосредственно класами символьных потіков, зокрема класами FileReader и FileWriter соответственно, призначенийными для работы с текстовыми файлуми.
