Читання та запис файлів. FileInputStream та FileOutputStream

Останнє оновлення: 08.03.2023

Запис файлів и клас FileOutputStream

Клас FileOutputStream призначений для записи байтов в файл. Он є производным от класа OutputStream, поцему найде всю его функциональность.

Через конструктор класа FileOutputStream задаєся файл, в який производится запис. Клас піддерживает декілька конструкторов:


FileOutputStream(String filePath)
FileOutputStream(File fileObj)
FileOutputStream(String filePath, boolean append)
FileOutputStream(File fileObj, boolean append)

Файл задаєся або через рядківовый путь, або через об'єкт File. Второй параметр - append задає способ записи: eсли он равен true, то дані дозаписуються в конец файлу, а при false - файл полностью перезаписуєся

Наприклад, запишем в файл рядок:


import java.io.*;

public class Program {
 
    public static void main(String[] args) {
         
        String text = "Hello world!"; // рядок для записи
        try(FileOutputStream fos=new FileOutputStream("notes.txt"))
        {
            // перевод рядки в байты
            byte[] buffer = text.getBytes();
             
            fos.write(buffer, 0, buffer.length);
			System.out.println("The file has been written");
        }
        catch(IOException ex){
             
            System.out.println(ex.getMessage());
        }
    }
}

Для створенняя об'єкта FileOutputStream використовується конструктор, принимающий в качестве параметра путь к файлу для записи. Если такого файлу нет, то он автоматически створюєся при записи. Так як здесь записуємо рядок, то її треба сначала перевести в масив байтов. И за допомогою методу write рядок записуєся в файл.

Для автоматического закрытия файлу и освобождения ресурса об'єкт FileOutputStream створюєся за допомогою конструктции try...catch.

При цем необязательно записывать весь масив байтов. Використовуючи перегрузку методу write(), можна записать и одиночный байт:


fos.write(buffer[0]); // запис первого байта

Читання файлів и клас FileInputStream

Для считывания данных из файлу призначений клас FileInputStream, який є наследником класа InputStream и поцему реалізує все его методи.

Для створенняя об'єкта FileInputStream мы можемо використовувати ряд конструкторов. Наиболїї використовуємоая версия конструктора в качестве параметра приймає путь к считываемому файлу:


FileInputStream(String fileName) throws FileNotFoundException

Если файл не может быть открыт, наприклад, по указанному пути такого файлу не існує, то генеруєся виняток FileNotFoundException.

Считаємо дані из ранїї записанного файлу и выведем на консоль:


import java.io.*;

public class Program {
 
    public static void main(String[] args) {

		try(FileInputStream fin=new FileInputStream("notes.txt"))
        {    
            int i;
            while((i=fin.read())!=-1){
             
                System.out.print((char)i);
            }   
        }
        catch(IOException ex){
             
            System.out.println(ex.getMessage());
        } 
    }
}

У даному випадку мы считываем каждый отдельный байт в змінну i:

while((i=fin.read())!=-1){

Когда у потоці больше нет данных для читання, метод повертає число -1.

Затем каждый считанный байт конвертируется в об'єкт типу char и виводиться на консоль.

Підобным чином можна считать дані в масив байтов и затем производить с ним манипуляции:


import java.io.*;
 
public class Program {
  
    public static void main(String[] args) {
 
        try(FileInputStream fin=new FileInputStream("notes.txt"))
        {
            byte[] buffer = new byte[256];
            System.out.println("File data:");
            
            int count;
            while((count=fin.read(buffer))!=-1){
              
                for(int i=0; i<count;i++){
            
                    System.out.print((char)buffer[i]);
                }
            }   
        }
        catch(IOException ex){
              
            System.out.println(ex.getMessage());
        } 
    }
}

У даному випадку за допомогою методу read() считываем дані в масив buffer длиной 256 байтов. Метод повертає кількість считанных байтов.

Поскольк файл может быть больше 256 байтов, то считываем в цикле while до конца файлу. Когда больше не останется файлів для считывания, то метод возвратит -1.

while((count=fin.read(buffer))!=-1){

Поскольку кількість считанных байтов/размер файлу можуть быть меньше 256 байт, то реальное кількість считанных байт сохраняем в змінну count. Затем вивідим считанное кількість данных на консоль в цикле for.


for(int i=0; i<count;i++){
            
    System.out.print((char)buffer[i]);
}

Совместим оба класа и виконаємо читання из одного и запис в другой файл:


import java.io.*;
 
public class Program {
 
    public static void main(String[] args) {
         
        try(FileInputStream fin=new FileInputStream("notes.txt");
                FileOutputStream fos=new FileOutputStream("notes_new.txt"))
        {
            byte[] buffer = new byte[256];
            
            int count;
            // считываем буфер
            while((count=fin.read(buffer))!=-1){
              
                // записуємо из буфера в файл
                fos.write(buffer, 0, count);
            }
            System.out.println("File has been written");
        }
        catch(IOException ex){
             
            System.out.println(ex.getMessage());
        } 
    } 
}

Класы FileInputStream и FileOutputStream призначенийы прежде всего для записи двоичных файлів, то есть для записи и читання байтов. И хотя они також можуть використовуватися для работы с текстовыми файлуми, но все же для цей завданьі больше підходят другие класы.
