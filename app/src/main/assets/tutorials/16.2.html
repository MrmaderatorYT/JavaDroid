Foreign Functions та Memory API

Останнє оновлення: 21.12.2025

Хотя підход підключения функционала на Си в програму на Java за допомогою JNI продолжает працювати и в самых післядних версиях Java, тем не менїї підключение нативного коду за допомогою JNI досить трудоемко, требует написания дополнительного коду на Си, компиляцию соответствующей бібліотеки и т.д. Щоб упростить всю эту работу в післядних версиях Java був добавлен новый API для доступа к "внешним" функціям и памяти - Foreign Functions and Memory API (FFM), ранїї развивавшийся по именем Project Panama. Начиная с версии Java 14 цей API був экспериментальным, а в Java 22 данный API достиг состояния релиза и стал полноценной функциональностью платформы. Предполагается, что со временем он вытеснит JNI.

Для примера опять же використовуємо стандартную библиотечную функцію printf из стандартной бібліотеки языка Си. И для цього визначимо файл Program.java со наступним кодом:


import java.lang.invoke.*;
import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) throws Throwable {

        Linker linker = Linker.nativeLinker();
        MethodHandle printf = linker.downcallHandle(
            linker.defaultLookup().findOrThrow("printf"),  // отримуємо функцію printf
            FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS));

        try (Arena arena = Arena.ofConfined()) {
            MemorySegment str = arena.allocateFrom("Hello \n");
            int result = (int) printf.invoke(str);  // отримуємо результат из нативной функції
            System.out.printf("Напечатано символів %d\n", result);
        }
    }
}

Разберем, что происходит в коді. Прежде всего створюємо линкер (компоновщик) для работы с нативными функціями:


Linker linker = Linker.nativeLinker();

Линкер по сути представляє связующїї звено між Java и нативной архитектурой текущей операционной системы.

Далїї определяем дескриптор для вызываемыой функції в виде об'єкта MethodHandle


MethodHandle printf = linker.downcallHandle(...)

MethodHandle printf представляє своего рода указатель на нативную функцію, який Java может вызвать як обычный метод.

Для створенняя цього об'єкта застосовується метод linker.downcallHandle(), який приймає два параметра:

  • Дескриптор внешней функції. Щоб найти эту функцію, вызывается післядовательность методів:

    linker.defaultLookup().findOrThrow("printf")

    Здесь выражение linker.defaultLookup() виконує поиск символів (функцій) в стандартных бібліотеках C, которые загрвжены по умолчанию. А выражение findOrThrow("printf") вказує ім'я искомой функції (то есть в нашем случае printf)

  • Параметри линкера. Застосовуються для описания сигнатури функції

    
FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS)

    У даному випадку указываем, что printf() повертає int (ValueLayout.JAVA_INT) и приймає указатель - ValueLayout.ADDRESS (char*).

Следующий важный момент - выделение памяти для рядки, которая буде передаваться в вызываемую функцію в качестве параметра:


try (Arena arena = Arena.ofConfined()) {
       MemorySegment str = arena.allocateFrom("Hello \n");

Поскольку нативные функції не умеют працювати с об'єктами Java (такими як String), дані потрібно разместить в "неуправляемой" памяти (вне кучи Java), для чего застосовуються следующие действия:

  1. Arena.ofConfined(): створює область памяти (арену), которая автоматически звільняєся при закрытии блока try. Це предотвращает утечки памяти.

  2. arena.allocateFrom("..."): выделяет память для переданной рядки Java и копирует эту рядок в выделенную нативную память, добавляя в конец нулевой байт (\0), чтобы формат соответствовал стандарту C-рядків. Повертає об'єкт MemorySegment - по сути, безопасный указатель для работы с сегментом памяти.

И далїї собственно идет вызов гативной функції printf() и отримання результата:


int result = (int) printf.invoke(str);

В итоге мы имїїм следующий общий алгоритм работы програми:

  1. Поиск. Находит функцію printf в системных бібліотеках.

  2. Выделение памяти. Резервирует блок памяти вне Java для рядки.

  3. Виконання. Вызывает нативную функцію, передавая ей адрес цього блока.

  4. Звільнення. Після виконання блока try пам'ять, виділена під рядок, миттєво звільняєся.

Варто відзначити, что як и в случае с JNI, для запуску програми необхідно передать утилите java дополнительную опцию, которая разрешает використання нативного коду в програме на Java. Если модулі не використовуються, то програма запускуєся наступним чином:

java --enable-native-access=ALL-UNNAMED ...

В модульной же програме необхідно перечислить все модулі, для которых разрешен нативний доступ:

java --enable-native-access=module1, module2 ...

Протестируем програму, запустив її на виконання:


eugene@Eugene:/workspace/java/test$ javac Program.java
eugene@Eugene:/workspace/java/test$ java --enable-native-access=ALL-UNNAMED Program
Hello 
Напечатано символів 18
eugene@Eugene:/workspace/java/test$
