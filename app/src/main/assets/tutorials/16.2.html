Foreign Functions та Memory API

Останнє оновлення: 21.12.2025

Хотя подход подключения функционала на Си в программу на Java за допомогою JNI продолжает работать и в самых последних версиях Java, тем не менїї подключение нативного кода за допомогою JNI довольно трудоемко, требует написания дополнительного кода на Си, компиляцию соответствующей библиотеки и т.д. Щоб упростить всю эту работу в последних версиях Java був добавлен новый API для доступа к "внешним" функциям и памяти - Foreign Functions and Memory API (FFM), ранїї развивавшийся по именем Project Panama. Начиная с версии Java 14 цей API був экспериментальным, а в Java 22 данный API достиг состояния релиза и стал полноценной функциональностью платформы. Предполагается, что со временем он вытеснит JNI.

Для примера опять же используем стандартную библиотечную функцию printf из стандартной библиотеки языка Си. И для цього определим файл Program.java со наступним кодом:


import java.lang.invoke.*;
import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) throws Throwable {

        Linker linker = Linker.nativeLinker();
        MethodHandle printf = linker.downcallHandle(
            linker.defaultLookup().findOrThrow("printf"),  // получаем функцию printf
            FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS));

        try (Arena arena = Arena.ofConfined()) {
            MemorySegment str = arena.allocateFrom("Hello \n");
            int result = (int) printf.invoke(str);  // получаем результат из нативной функції
            System.out.printf("Напечатано символов %d\n", result);
        }
    }
}

Разберем, что происходит в коде. Прежде всего создаем линкер (компоновщик) для работы с нативными функциями:


Linker linker = Linker.nativeLinker();

Линкер по сути представляє связующїї звено между Java и нативной архитектурой текущей операционной системы.

Далїї определяем дескриптор для вызываемыой функції в виде об'єкта MethodHandle


MethodHandle printf = linker.downcallHandle(...)

MethodHandle printf представляє своего рода указатель на нативную функцию, який Java может вызвать як обычный метод.

Для создания цього об'єкта застосовується метод linker.downcallHandle(), який приймає два параметра:

  • Дескриптор внешней функції. Щоб найти эту функцию, вызывается последовательность методів:

    linker.defaultLookup().findOrThrow("printf")

    Здесь выражение linker.defaultLookup() выполняет поиск символов (функций) в стандартных библиотеках C, которые загрвжены по умолчанию. А выражение findOrThrow("printf") указывает имя искомой функції (то есть в нашем случае printf)

  • Параметры линкера. Застосовуються для описания сигнатуры функції

    
FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS)

    В данном случае указываем, что printf() повертає int (ValueLayout.JAVA_INT) и приймає указатель - ValueLayout.ADDRESS (char*).

Следующий важный момент - выделение памяти для строки, которая буде передаваться в вызываемую функцию в качестве параметра:


try (Arena arena = Arena.ofConfined()) {
       MemorySegment str = arena.allocateFrom("Hello \n");

Поскольку нативные функції не умеют работать с об'єктами Java (такими як String), дані потрібно разместить в "неуправляемой" памяти (вне кучи Java), для чего застосовуються следующие действия:

  1. Arena.ofConfined(): створює область памяти (арену), которая автоматически звільняєся при закрытии блока try. Це предотвращает утечки памяти.

  2. arena.allocateFrom("..."): выделяет память для переданной строки Java и копирует эту строку в выделенную нативную память, добавляя в конец нулевой байт (\0), чтобы формат соответствовал стандарту C-строк. Повертає об'єкт MemorySegment - по сути, безопасный указатель для работы с сегментом памяти.

И далїї собственно идет вызов гативной функції printf() и получение результата:


int result = (int) printf.invoke(str);

В итоге мы имїїм следующий общий алгоритм работы программы:

  1. Поиск. Находит функцию printf в системных библиотеках.

  2. Выделение памяти. Резервирует блок памяти вне Java для строки.

  3. Выполнение. Вызывает нативную функцию, передавая ей адрес цього блока.

  4. Освобождение. После виконання блока try память, выделенная под строку, мгновенно звільняєся.

Варто відзначити, что як и в случае с JNI, для запуска программы необхідно передать утилите java дополнительную опцию, которая разрешает использование нативного кода в программе на Java. Если модули не використовуються, то программа запускаєся наступним чином:

java --enable-native-access=ALL-UNNAMED ...

В модульной же программе необхідно перечислить все модули, для которых разрешен нативний доступ:

java --enable-native-access=module1, module2 ...

Протестируем программу, запустив її на выполнение:


eugene@Eugene:/workspace/java/test$ javac Program.java
eugene@Eugene:/workspace/java/test$ java --enable-native-access=ALL-UNNAMED Program
Hello 
Напечатано символов 18
eugene@Eugene:/workspace/java/test$
