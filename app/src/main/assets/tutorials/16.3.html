Арена та сегменти пам'яті MemorySegment

Останнє оновлення: 21.12.2025

Арена

Одним из ключевых компонентов при взаимодействии с нативными (внешними) функціями є арена (arena). При обращении к нативным (внешним) функціями код Java повинен працювати с памятью вне кучи (хипа), то есть с памятью, которая не бува выделена за допомогою оператора new. И арена як раз управляет жизненным циклом сегментов памяти вне кучи, позволяя легко выделять и автоматически освобождать використовуємоую память.

Створення и закрытие арены

На платформе Java арена представлена класом Arena. Для створенняя арены застосовується один из статических методів класа:

  • global(): повертає "глобальную" арену. Сегменты памяти, выделенные в рамках глобальной арены, доступны любому потіку.

  • ofAuto(): cоздает новую арену, которая автоматически управляется сборщиком мусора. Сегменты арены доступны любому потіку

  • ofConfined(): повертає новую "ограниченную" арену. Сегменты, выделенные для ограниченной арены, доступны потіку, який создал арену, то есть потіку-владельцу арены.

  • ofShared(): повертає новую общую арену. Сегменты, выделенные для общей арены, доступны любому потіку.

Для закрытия арены призначений метод close(). Но в зависимости от типу арены принцип закрытия может различаться:

  • "Ограниченная" арена (ofConfined()) повинна быть закрыта в том же потіке, в котором она бува створена. Иначе буде выброшено виняток.

  • "Глобальная" арена (global()) никогда не закрывается, и выделенные ею сегменты памяти никогда не звільняються.

  • "Автоматическую" арену (ofAuto()) закрыть нельзя. Сборщик мусора сам в конечном итоге освободит сегменты памяти, когда они, як и арена, перестанут використовуватися. Це удобно, но менїї предсказуемо.

  • "Общая" арена (ofShared()) работает до тех пор, поки не буде вызван метод close в любом потіке

Таким чином, явно вызывать метод close() необхідно лише для ограниченной и общей арен, а для глобальной и автоматической вызов методу close() сгенерує виняток.

Після успешного виконання методу close() доступ ко всем связанным с ареной сегментам памяти станет невозможным, а область памяти вне кучи, связанная с ареной, буде освобождена.:


Arena arena = Arena.ofConfined();   // створюємо арену
// неокторая работа с ареной
arena.close();  // закрываем арену

Но поскольку клас арены реалізує інтерфейс AutoCloseable (в виде методу close()), то арену удобно создавать и використовувати через конструкцию try-с ресурсами, которая автоматически звільняє память арены:


try (Arena arena = Arena.ofConfined()) {

    // работа с ареной

} // Арена закрыта

Сегменты памяти MemorySegment

Сегмент памяти представляє непрерывный блок памяти. На уровне коду Java сегмент памяти представлен інтерфейсом MemorySegment. Сегмент памяти может находиться в куче Java, управляемой сборщиком мусора, або быть выделен вне кучи Java. Сегмент памяти вне кучи имїїт фиксированный физический адрес. Сегмент кучи, як и любой об'єкт Java, имїїт ссылку, по которой его можна найти, но не имїїт фиксированного адреса, поскольку сборщик мусора мог его переместить.

Для использования с внешними/нативными функціями выделяются сегменты памяти вне кучи.

Для выделения памяти у класа Arena есть ряд методів (в основном реализаций інтерфейса java.lang.foreign.SegmentAllocator). Некоторые из них:

  • allocate(long byteSize, long byteAlignment)

    Цет метод приймає два параметра:

    • byteSize: размер выделяемой памяти в байтах

    • byteAlignment: выравнивание памяти в байтах

    Метод повертає сегмент памяти заданного размера (первый параметр), адрес которого выравнивается исходя из значення второго параметра.

  • allocateFrom(String str)

    Преобразует рядок Java в рядок C, завершающуюся нулевым символом, використовуючи кодировку UTF-8, и сохраняет результат в сегмент памяти.

  • allocateFrom(ValueLayout.ofType elementLayout, Type[] elements)

    Выделяет память для хранения елементов заданного примитивного типу Type (це может быть int, float и т.д.).

  • allocateFrom(ValueLayout.of[Type] elementLayout, Type element)

    Аналогичен предыдущему варианту, лише выделяет память для хранения одного елемента element заданного примитивного типу Type и сохраняет значення element в выделенной памяти

  • allocate(MemoryLayout layout)

    Выделяет память вне кучи с заданным макетом layout.

  • allocate(MemoryLayout layout, long count)

    Выделяет память вне кучи с заданным макетом layout с кількістьм елементов равным count

Все эти методи повертають об'єкт типу MemorySegment. Пример применения методу allocate():


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            // выделяем сегмент памяти - в 16 байт с выравниванием по 8 байтам
            MemorySegment segment1 = arena.allocate(16, 8);
            // можемо выделить ще один сегмент
            MemorySegment segment2 = arena.allocate(32, 4);

            // просто выведем информацию на экран
            System.out.println(segment1);   // MemorySegment{ kind: native, address: 0x772c881780e0, byteSize: 16 }
            System.out.println(segment2);   // MemorySegment{ kind: native, address: 0x772c881783d0, byteSize: 32 }
        } 
    }
}

Метод allocateFrom() є наиболїї простым способом для сохранения в сегменте памяти вне кучи якой-нибудь рядки:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            String str = "Hello ";

            MemorySegment str_seg = arena.allocateFrom(str);

            System.out.println(str_seg);   // MemorySegment{ kind: native, address: 0x7aa79c1802d0, byteSize: 18 }
        } 
    }
}

Здесь происходит кодирование рядки Java в UTF-8 и выделение сегмент памяти, який містить байты рядки, включая нулевой терминатор.

Осовобождение памяти

Для освобождения сегмента памяти в ограниченной або общей арене необхідно закрыть эту арену за допомогою методу close().

В автоматической арене для освобождения сегмента сегменту передаєся значення null, и затем в якой-то момент времени сборщик мусора освободит память:


MemorySegment segment = Arena.ofAuto().allocate(...);

...
segment = null; // Память в конечном итоге звільняєся

Наконец, сегменты памяти глобальной арены в процесе работы програми никогда не звільняються.

Варто також відзначити, что освободить отдельные сегменты памяти невозможна.
