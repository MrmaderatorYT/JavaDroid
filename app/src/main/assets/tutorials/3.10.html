Ієрархія успадкування та перетворення типів

Останнє оновлення: 24.09.2025

У минулому розділі йшлося про перетворення об'єктів найпростіших типів. Однак з об'єктами класів все відбувається трохи інакше. Припустимо, ми маємо таку ієрархію класів:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom");
        tom.print();        // Person Tom
        Person sam = new Employee("Sam", "Yandex");
        sam.print();        // Employee Sam works in Yandex
        Person bob = new Client("Bob", "SberBank");
        bob.print();    // Client Bob has account in SberBank
    }
}
// клас человека
class Person {
     
    private String name;
    String getName() {return name;}
    
    Person(String name){
     
        this.name=name;
    }
  
    void print(){
         
        System.out.printf("Person %s \n", name);
    }
}
// служащий некоторой компании 
class Employee extends Person{
 
    private String company;
    String getCompany(){return company;}
     
    Employee(String name, String company) {
     
        super(name);
        this.company = company;
    }
     
    void print(){
         
        System.out.printf("Employee %s works in %s \n", getName(), company);
    }
}
// клас клієнта банка
class Client extends Person{
       
    private String bank;
    String getBank(){return bank;}
       
    Client(String name, String bank) {
       
        super(name);
        this.bank=bank;
    }
       
    void print(){
           
        System.out.printf("Client %s has account in %s \n", getName(), bank);
    }
}

У цій ієрархії класів можна простежити наступний ланцюг успадкування: Object (всі класи неявно успадковуються від типу Object) -> Person -> Employee | Client.

Суперкласи зазвичай розміщуються вище підкласів, тому на вершині успадкування знаходиться клас Object, а в самому низу Employee та Client.

upcasting або висхідні перетворення

Об'єкт підкласу також представляє об'єкт суперкласу. Тож у програмі ми можемоо написати так:


public class Program{
      
    public static void main(String[] args) {
             
        Object tom = new Person("Tom");                     // преобразование от Person к Object
        Object sam = new Employee("Sam", "Yandex");         // от Employee к Object
        Object kate = new Client("Kate", "SberBank");       // от Client к Object
        Person bob = new Client("Bob", "VTB");              // от Client к Person
        Person alice = new Employee("Alice", "VK");         // от Employee к Person
    }
}

Це так называемое висхідне перетворення (от підкласа внизу к суперкласу вверху иерархии) або upcasting. Такое преобразование осуществляется автоматически.

downcasting або низхідне перетворення

Обратное не всегда верно. Наприклад, об'єкт Person не всегда є об'єктом Employee або Client. Тому низхідне перетворення або downcasting от суперкласа к підкласу автоматически не виконується. В цем случае нам треба використовувати операцію преобразования типів. Наприклад:


Object sam = new Employee("Sam", "Yandex");
sam.print(); // ! Ошибка

Здесь хотя змінна sam и хранит ссылку на об'єкт типу Employee, но є змінної типу Object. Тому при компиляции мы столкнемся с ошибкой - поскольку об'єкт типу Object не обязательно представляє тип Employee або Person, у него может и не быть методу print(). И в цем случае нам треба пріменіть низхідне перетворення от типу Object к типу Employee або Person:


Object sam = new Employee("Sam", "Yandex");

// нисходящїї преобразование от Object к типу Employee
Employee emp = (Employee)sam;
emp.print();
System.out.println(emp.getCompany());

У разі змінна sam приводиться до типу Employee. Потім через об'єкт emp ми можемоо звернутися до функціоналу об'єкта Employee.

Ми можемоо перетворити об'єкт Employee по всій прямій лінії наслідування від Object до Employee.

Приклади низхідних перетворень:


public class Program{
      
    public static void main(String[] args) {
             
        Object kate = new Client("Kate", "Sberbank");
        ((Person)kate).print();
                
        Object sam = new Employee("Sam", "Yandex");
        ((Employee)sam).print();
    }
}

Але розглянемо ще одну ситуацію:


public class Program{
      
    public static void main(String[] args) {
             
        Person kate = new Client("Kate", "Sberbank");
        Employee emp = (Employee) kate;   // ! Ошибка - java.lang.ClassCastException
        emp.print();

        // або так
        ((Employee)kate).print();
    }
}

У даному випадку змінна kate, которая имїїт тип Person, хранит ссылку на об'єкт Client. Мы можемо без ошибок привести цей об'єкт к типу Client. Но при попытке преобразования к типу Employee мы получим ошибку во час виконання. Так як kate НЕ представляє об'єкт типу Employee.

Оператор взаємодії

В примере выше мы явно видим, что змінна kate - це ссылка на об'єкт Client, а не Employee. Однак нередко дані приходят зовні, и мы можемо точно не знать, якой именно об'єкт эти дані представляютьь. Соответственно возникает большая вероятность столкнуться с ошибкой. И перед тем, як провести преобразование типів, мы можемо проверить, а можемо ли мы выполнить приведение за допомогою оператора instanceof:


public class Program{
      
    public static void main(String[] args) {
             
        Person kate = new Client("Kate", "Sberbank");
        printPerson(kate);  // Person is not an employee

        Person sam = new Employee("Sam", "Yandex");
        printPerson(sam);  // Employee works in Yandex
    }

    static void printPerson(Person person){
        if(person instanceof Employee){
     
            Employee empl = (Employee) person;
            System.out.println("Employee works in " + empl.getCompany());
        }
        else{
                            
            System.out.println("Person is not an employee");
        }
    }
}

В методі printPerson выражение person instanceof Employee проверяет, представляє ли параметр person об'єкт типу Employee. И якщо не є, то такая проверка вернет значення false, и преобразование не сработает, и виконується блок else:


else{
                            
    System.out.println("Person is not an employee");
}

Если же параметр person представляє тип Employee, то виконується блко if, где преобразуем в тип Employee и вивідим на консоль информацию об об'єкте: 


if(person instanceof Employee){
     
    Employee empl = (Employee) person;
    System.out.println("Employee works in " + empl.getCompany());
}
