Синхронізація потоків. Оператор synchronized

Останнє оновлення: 22.10.2025

Состояние гонки

При работе потоки нередко обращаются к якимось общим ресурсам, которые определены вне потока, наприклад, обращение к якой-то змінної або даже файлу. Если одновременно декілька потоков обратятся к общему ресурсу, то результаты виконання програми можуть быть неожиданными и даже непредсказуемыми. Наприклад, определим следующий код:


public class Program {

    public static void main(String[] args) {
        
        CommonResource commonResource= new CommonResource();

        for (int i = 1; i < 6; i++){
            
            new Thread(new CountThread(commonResource), "Thread "+ i).start();
        }
    }
}

class CommonResource{
    
    int x = 0;
}

class CountThread implements Runnable{

    CommonResource res;

    CountThread(CommonResource res){
        this.res = res;
    }

    public void run(){
		res.x = 1;

        for (int i = 1; i < 5; i++){

			System.out.printf("%s %d \n", Thread.currentThread().getName(), res.x);
			res.x++;

            try{
				Thread.sleep(100);
            }
            catch(InterruptedException _){}
        }
    }
}

Здесь определен класс CommonResource, який представляє общий ресурс и в котором определено одно целочисленное поле x.

Цет ресурс використовується классом потока CountThread. Цет класс просто увеличивает в цикле значение x на единицу. Причем при входе в поток значение x=1:

res.x=1;

То есть в итоге мы ожидаем, что после виконання цикла res.x буде равно 4.

В главном классе программы запускаєся пять потоков. То есть мы ожидаем, что каждый поток буде увеличивать значение res.x с 1 до 4 и так пять раз. Но якщо мы посмотрим на результат работы программы, то он буде иным:

То есть поки один поток не окончил работу с полем res.x, с ним начинает работать другой поток. Эта ситуация називається состоянием гонки (race condition), при котором два и болїї потоков борятся за управление одинм и тем же ресурсом. И якщо наше приложение выполняет якие-то критически важные операции, наприклад, це финансовое приложение, где декілька потоков одновременно обращаются к счету пользователя для изменения баланса средств, то подобное состояние гонки может существенно нарушить корректность данных.

Щоб избежать подобной ситуации, треба синхронизировать потоки. Одним из способов синхронизации є использование ключевого слова synchronized. Цет оператор предваряет блок кода або метод, який подлежит синхронизации.

synchronized-блоки

synchronized-блок або синхронизированный блок представляє блок кода, к которому одномоментно имїїт доступ лише один поток. Синхронизированный блок имїїт следующую форму:


synchronized (obj)
{
    // критическая секция
}

При создании синхронизированного блока кода после оператора synchronized идет об'єкт-заглушка: synchronized(obj). Причем в качестве об'єкта может використовуватися лише об'єкт якого-нибудь класса, но не примитивного типу. Затем идет блок кода, в яку помещается критическая секция - код, к которому в одним момент времени имїїт доступ лише один поток.

Каждый об'єкт в Java имїїт ассоциированный с ним монитор. Монитор представляє своего рода инструмент для управления доступа к об'єкту. Когда выполнение кода доходит до оператора synchronized, монитор об'єкта блокуєся, и на час его блокировки монопольный доступ к блоку кода имїїт лише один поток, який и произвел блокировку. После окончания работы блока кода, монитор об'єкта звільняєся и становится доступным для других потоков.

После освобождения монитора его захоплює другой поток, а все остальные потоки продолжают ожидать его освобождения.

Например изменим предудущий пример, применив синхронизированные блоки для синхронизации потоков:


public class Program {
 
    public static void main(String[] args) {
         
        CommonResource commonResource= new CommonResource();
        for (int i = 1; i < 6; i++){
             
            new Thread(new CountThread(commonResource), "Thread "+ i).start();
        }
    }
}
 
class CommonResource{
     
    int x=0;
}
 
class CountThread implements Runnable{
 
    CommonResource res;

    CountThread(CommonResource res){ this.res = res; }

    public void run(){

        synchronized(res){  // определяем синхронизированный блок

            // до конца блок идет критическая секция
            res.x = 1;

            for (int i = 1; i < 5; i++){
                System.out.printf("%s %d \n", Thread.currentThread().getName(), res.x);

                res.x++;

                try{
                    Thread.sleep(100);
                }
                catch(InterruptedException _){}
            }
        }
    }
}

И так, у нас есть неякий участок кода, к которому треба дать доступ одномоментно лише одному потоку. В примере выше це код, связанный с изменением общего ресурса - res. Тому эту часть кода оформляем в синхронизированный блок:


synchronized(res){  // определяем синхронизированный блок

    res.x = 1;

    for (int i = 1; i < 5; i++){
        System.out.printf("%s %d \n", Thread.currentThread().getName(), res.x);

        res.x++;

        try{
            Thread.sleep(100);
        }
        catch(InterruptedException _){}
        }
    }
}

Первый из потоков, який начнет выполнять цей код, захватит об'єкт res и начнет выполнять код synchronized-блока. Все остальные потоки в це час будуть ждать освобождения ресурса res. Когда первый поток завершит выполнение кода в блоке, ресурс res звільняєся, и его захоплює другой (произвольный) поток. И опять остальные потоки будуть ожидать его освобождения. И так далїї, поки все потоки не выполнят код synchronized-блока. В итоге консольный вывод изменится и станет упорядоченным:

Синхронизированные методы

При применении оператора synchronized к методу, як и в случае с блоком, монопольный доступ к методу (выполнять метод в один момент времени) может лише один поток - первый, який начал его выполнение. Для применения оператора synchronized к методу, изменим классы программы:


public class Program {
 
    public static void main(String[] args) {
         
        CommonResource commonResource= new CommonResource();
        for (int i = 1; i < 6; i++){
             
            new Thread(new CountThread(commonResource), "Thread "+ i).start();
        }
    }
}
 
class CommonResource{
     
    int x;
    synchronized void increment(){  // синхронизированный метод

        x = 1;
        for (int i = 1; i < 5; i++){
            System.out.printf("%s %d \n", Thread.currentThread().getName(), x);
            x++;
            try{
                Thread.sleep(100);
            }
            catch(InterruptedException _){}
        }
    }
}
 
class CountThread implements Runnable{
 
    CommonResource res;
    CountThread(CommonResource res){

        this.res = res;
    }
     
    public void run(){
        res.increment();
    }
}

Результат работы в данном случае буде аналогичен примеру выше с блоком synchronized. Здесь опять в дело вступает монитор об'єкта CommonResource - общего об'єкта для всех потоков. Тому синхронизированным оголошується не метод run() в классе CountThread, а метод increment класса CommonResource. Когда первый поток начинает выполнение метода increment, он захоплює монитор об'єкта CommonResource. А все потоки також продолжают ожидать его освобождения.
