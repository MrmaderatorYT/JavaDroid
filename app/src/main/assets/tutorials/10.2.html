Створення потіку даних

Останнє оновлення: 02.05.2018

Для створення потіку даних можна застосовувати різні методи. Як джерело потіку ми можемоо використовувати колекції. Зокрема, в JDK 8 в інтерфейс Collection, який реалізуєся всеми класами колекцій, буви добавлены два методу для работы с потіками:

  • default Stream<E> stream: повертається потік даних з колекції

  • default Stream<E> parallelStream: повертається паралельний потік даних з колекції

Так, розглянемо приклад з ArrayList:


import java.util.stream.Stream;
import java.util.*;
public class Program {

    public static void main(String[] args) {
		
		ArrayList<String> cities = new ArrayList<String>();
        Collections.addAll(cities, "Париж", "Лондон", "Мадрид");
		cities.stream() // отримуємо потік
			.filter(s->s.length()==6) // застосовуємо фільтрацію за довжиною рядка
			.forEach(s->System.out.println(s)); // виводимо відфільтровані рядки на консоль
	}
}

Здесь за допомогою вызова cities.stream() отримуємо потік, який використовує дані из списку cities. За допомогою каждой промежуточной операції, которая застосовується к потіку, мы також можемо отримати потік с учетом модификаций. Наприклад, мы можемо змінити предыдущий пример наступним чином:


ArrayList<String> cities = new ArrayList<String>();
Collections.addAll(cities, "Париж", "Лондон", "Мадрид");

Stream<String> citiesStream = cities.stream(); // отримуємо потік
citiesStream = citiesStream.filter(s->s.length()==6); // застосовуємо фільтрацію за довжиною рядка
citiesStream.forEach(s->System.out.println(s)); // виводимо відфільтровані рядки на консоль

Важливо, що після використання термінальних операцій інші термінальні або проміжні операції до цього ж потіку не можуть бути застосовані, потік вже використаний. Наприклад, у наступному випадку ми отримаємо помилку:


citiesStream.forEach(s->System.out.println(s)); // термінальна операція використовує потік
long number = citiesStream.count(); // тут помилка, оскільки потік вже використаний
System.out.println(number);
citiesStream = citiesStream.filter(s->s.length()>5); // теж не можна, оскільки потік вже використаний

Фактично життєвий цикл потіку проходить наступні три стадії:

  1. Створення потіку

  2. Застосування к потіку ряда промежуточных операцій

  3. Застосування к потіку терминальной операції и отримання результата

Окрім вищерозглянутих методів ми можемоо використовувати ще ряд способів для створення потіку даних. Один из таких способов представляє метод Arrays.stream(T[] array), який створює потік данных из масива:


Stream<String> citiesStream = Arrays.stream(new String[]{"Париж", "Лондон", "Мадрид"}) ;
citiesStream.forEach(s->System.out.println(s)); // виводимо всі елементи масиву

Для створення потіків IntStream, DoubleStream, LongStream можна використовувати відповідні перевантажені версії цього методу:


IntStream intStream = Arrays.stream(new int[]{1,2,4,5,7});
intStream.forEach(i->System.out.println(i));

LongStream longStream = Arrays.stream(new long[]{100,250,400,5843787,237});
longStream.forEach(l->System.out.println(l));

DoubleStream doubleStream = Arrays.stream(new double[] {3.4, 6.7, 9.5, 8.2345, 121});
doubleStream.forEach(d->System.out.println(d));

И ще один способ створенняя потіка представляє статический метод of(T..values) класа Stream:


Stream<String> citiesStream =Stream.of("Париж", "Лондон", "Мадрид");
citiesStream.forEach(s->System.out.println(s));

String[] cities = {"Париж", "Лондон", "Мадрид"};
