Параметри методів

Останнє оновлення: 16.09.2025

За допомогою параметрів ми можемоо передати до методів різні дані, які використовуватимуться для обчислень. Наприклад, визначимо таку програму:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();
        tom.say();  // Hello
        tom.say();  // Hello
    }
}

class Person{ 

    void say(){
        System.out.println("Hello");
    }
}

Здесь визначений клас Person, в котором есть метод say(). Цет метод имитирует говорение некоторой фразы. В методі main створюєся один об'єкт класа Person, и у него вызывается два раза метод say. И ожидаемо мы увидим на консоли два раза слово "Hello":


Hello
Hello

Але скільки б ми не викликали метод say, результат не зміниться. І щоб мати можливість налаштувати ззовні поведінку методу, застосовуються параметри. Параметри визначаються через зайому в дужках після назви методу як:


тип_методу ім'я_методу (тип_параметра1 параметр1, тип_параметра2 параметр2, ...)
{
    // действия методу
}

Визначення параметра складається з двох частин: спочатку йде тип параметра, а потім його ім'я.

Так, використовуємо параметри для цього змінимо програму так:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();
        tom.say("Hello work");  // Hello work
        tom.say("Hello gold");  // Hello gold
    }
}

class Person{ 

    void say(String message){
        System.out.println(message);
    }
}

Здесь метод say() теперь приймає один параметр, який називається message и имїїт тип String.

Щоб виконати метод, який має параметри, під час виклику після імені методу в дужках йому передаютьься значення його параметрів, наприклад:


tom.say("Hello work");

Здесь параметру message передаєся рядок "Hello work". Значення, которые передаються параметрам, ще називаються аргументами. То есть передавана рядок "Hello work" у даному випадку є аргументом.

Иногда можна встретить такие визначенийия як формальні параметри и фактичні параметри. Формальные параметри - це собственно параметри методу (у даному випадку message), а фактические параметри - значення, которые передаються формальным параметрам. То есть фактические параметри - це и есть аргументы методу.

Розглянемо інший приклад, де ми маємо метод, який становить два числа:


class Program{

    public static void main(String[] args) {  

        Calcualtor calc = new Calcualtor();
        calc.sum(1, 2);   // 3
        calc.sum(5, 3);   // 8
        calc.sum(5, 6);   // 11
    }
}
class Calcualtor{

    void sum(int a, int b){

        int result = a + b;
		System.out.println(result);
    }
}

Здесь визначений клас Calculator с одним методом sum, имїїт два параметра - два числа и виводить на консоль их сумму. Поскольку оба параметра имеют тип int, то на место параметрів треба передать два значення типу int. Це можуть быть и числовые литералы, и змінні типів данных, которые представляютьь тип int або можуть быть автоматически преобразованы в тип int.

Значення передаються параметрам по позиции, то есть первый аргумент первому параметру, второй аргумент - второму параметру и так далїї. Наприклад, при вызове calc.sum(5, 3) число 5 передаєся параметру a, а число 3 - параметру b.

Параметри змінної довжини

Метод може набувати параметрів змінної довжини одного типу. Наприклад, нам треба передати в метод набір чисел та обчислити їхню суму, але ми точно не знаємо, скільки саме чисел буде передано – 3, 4, 5 або більше. Параметри змінної довжини дозволяють вирішити це завдання:


class Program{

    public static void main(String[] args) {  

        Calcualtor calc = new Calcualtor();
        calc.sum(1, 2, 3);			// 6
		calc.sum(1, 2, 3, 4, 5);    // 15
		calc.sum();					// 0
    }
}
class Calcualtor{

    void sum(int ...nums){
		
		int result =0;
		for(int n: nums)
			result += n;
		System.out.println(result);
	}
}

Троеточие перед названием параметра int ...nums вказує на то, что он буде необязательным и буде представлять масив. Мы можемо передать в метод sum одно число, декілька чисел, а можемо вообще не передавать нияких параметрів. Причем, якщо мы хочемо передать декілька параметрів, то необязательный параметр повинен указываться в конце:


class Program{

    public static void main(String[] args) {  

        Calcualtor calc = new Calcualtor();
        calc.sum("Sum of {1, 2, 3}: ", 1, 2, 3);    // Sum of {1, 2, 3}: 6
		calc.sum("Zero sum: ");					    // Zero sum: 0
    }
}
class Calcualtor{

    void sum(String message, int ...nums){
		
        System.out.print(message); 
		int result =0;
		for(int n: nums)
			result += n;
		System.out.println(result);
	}
}
