Взаимодействие потіков. Методи wait и notify

Останнє оновлення: 22.10.2025

Иногда при взаимодействии потіков встает вопрос о извщении одних потіков о действиях других. Наприклад, действия одного потіка зависят от результата действий другого потіка, и треба як-то известить один потік, что второй потік произвел некую работу. И для підобных ситуаций у класа Object визначенийо ряд методів:

  • wait(): звільняє монитор и переводит вызывающий потік в состояние ожидания до тех пор, поки другой потік не вызовет метод notify()

  • notify(): продолжает работу потіка, у которого ранїї був вызван метод wait()

  • notifyAll(): возобновляет работу всех потіков, у которых ранїї був вызван метод wait()

Все эти методи вызываются лише из синхронизированного контекста - синхронизированного блока або методу.

Рассмотрим, як мы можемо використовувати эти методи. Підзьмем стандартную завданьу из прошлой темы - "Производитель-Потребитель" ("Producer-Consumer"): поки производитель не произвел продукт, потребитель не может его купить. Пусть производитель повинен произвести 5 товаров, соответственно потребитель повинен их все купить. Но при цем одновременно на складе может находиться не болїї 3 товаров. Для решения цей завданьі задействуем методи wait() и notify():


public class Program {
 
    public static void main(String[] args) {
         
        Store store=new Store();
        Producer producer = new Producer(store);
        Consumer consumer = new Consumer(store);
        new Thread(producer).start();
        new Thread(consumer).start();
    }
}
// Клас Магазин, хранящий произведенные товары
class Store{
   private int product=0;
   public synchronized void get() {
      while (product<1) {
         try {
            wait();
         }
         catch (InterruptedException e) {
         }
      }
      product--;
      System.out.println("Покупатель купил 1 товар");
      System.out.println("Товаров на складе: " + product);
      notify();
   }
   public synchronized void put() {
       while (product>=3) {
         try {
            wait();
         }
         catch (InterruptedException e) { 
         } 
      }
      product++;
      System.out.println("Производитель добавил 1 товар");
      System.out.println("Товаров на складе: " + product);
      notify();
   }
}
// клас Производитель
class Producer implements Runnable{
 
    Store store;
    Producer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.put();
        }
    }
}
// Клас Потребитель
class Consumer implements Runnable{
     
     Store store;
    Consumer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.get();
        }
    }
}

Итак, здесь визначений клас магазина, потребителя и покупателя. Производитель в методі run() додає в об'єкт Store за допомогою его методу put() 5 товаров. Потребитель в методі run() в цикле обращается к методу get об'єкта Store для получения цих товаров. Оба методу Store - put и get є синхронизированными.

Для отслеживания наличия товаров в класе Store проверяем значення змінної product. По умолчанию товара нет, поцему змінна равна 0. Метод get() - отримання товара повинен срабатывать лише при наличии хотя бы одного товара. Тому в методі get проверяем, отсутствует ли товар:

while (product<1)

Если товар отсутсвует, вызывается метод wait(). Цет метод звільняє монитор об'єкта Store и блокує виконання методу get, поки для цього же монитора не буде вызван метод notify().

Когда в методі put() додаєся товар и вызывается notify(), то метод get() отримує монитор и выходит из конструкции while (product<1), оскільки товар добавлен. Затем имитируется отримання покупателем товара. Для цього виводиться сообщение, и уменьшается значення product: product--. И в конце вызов методу notify() дает сигнал методу put() продолжить работу.

В методі put() работает похожая логика, лише теперь метод put() повинен срабатывать, якщо в магазине не болїї трех товаров. Тому в цикле проверяется наличие товара, и якщо товар вже есть, то освобождаем монитор за допомогою wait() и ждем вызова notify() в методі get().

Таким чином, за допомогою wait() в методі get() мы ожидаем, когда производитель добавит новый продукт. А після добавления вызываем notify(), як бы говоря, что на складе освободилось одно место, и можна ще добавлять.

А в методі put() за допомогою wait() мы ожидаем освобождения места на складе. Після того, як место освободится, додаємо товар и через notify() уведомляем покупателя о том, что он может забирать товар.
