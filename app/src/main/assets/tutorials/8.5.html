Взаимодействие потоков. Методи wait и notify

Останнє оновлення: 22.10.2025

Иногда при взаимодействии потоков встает вопрос о извщении одних потоков о действиях других. Наприклад, действия одного потока зависят от результата действий другого потока, и треба як-то известить один поток, что второй поток произвел некую работу. И для подобных ситуаций у класса Object определено ряд методів:

  • wait(): звільняє монитор и переводит вызывающий поток в состояние ожидания до тех пор, поки другой поток не вызовет метод notify()

  • notify(): продолжает работу потока, у которого ранїї був вызван метод wait()

  • notifyAll(): возобновляет работу всех потоков, у которых ранїї був вызван метод wait()

Все эти методы вызываются лише из синхронизированного контекста - синхронизированного блока або метода.

Рассмотрим, як мы можемо використовувати эти методы. Підзьмем стандартную задачу из прошлой темы - "Производитель-Потребитель" ("Producer-Consumer"): поки производитель не произвел продукт, потребитель не может его купить. Пусть производитель повинен произвести 5 товаров, соответственно потребитель повинен их все купить. Но при цем одновременно на складе может находиться не болїї 3 товаров. Для решения цей задачи задействуем методы wait() и notify():


public class Program {
 
    public static void main(String[] args) {
         
        Store store=new Store();
        Producer producer = new Producer(store);
        Consumer consumer = new Consumer(store);
        new Thread(producer).start();
        new Thread(consumer).start();
    }
}
// Класс Магазин, хранящий произведенные товары
class Store{
   private int product=0;
   public synchronized void get() {
      while (product<1) {
         try {
            wait();
         }
         catch (InterruptedException e) {
         }
      }
      product--;
      System.out.println("Покупатель купил 1 товар");
      System.out.println("Товаров на складе: " + product);
      notify();
   }
   public synchronized void put() {
       while (product>=3) {
         try {
            wait();
         }
         catch (InterruptedException e) { 
         } 
      }
      product++;
      System.out.println("Производитель добавил 1 товар");
      System.out.println("Товаров на складе: " + product);
      notify();
   }
}
// класс Производитель
class Producer implements Runnable{
 
    Store store;
    Producer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.put();
        }
    }
}
// Класс Потребитель
class Consumer implements Runnable{
     
     Store store;
    Consumer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.get();
        }
    }
}

Итак, здесь определен класс магазина, потребителя и покупателя. Производитель в методе run() додає в об'єкт Store за допомогою его метода put() 5 товаров. Потребитель в методе run() в цикле обращается к методу get об'єкта Store для получения цих товаров. Оба метода Store - put и get є синхронизированными.

Для отслеживания наличия товаров в классе Store проверяем значение змінної product. По умолчанию товара нет, поцему переменная равна 0. Метод get() - получение товара повинен срабатывать лише при наличии хотя бы одного товара. Тому в методе get проверяем, отсутствует ли товар:

while (product<1)

Если товар отсутсвует, вызывается метод wait(). Цет метод звільняє монитор об'єкта Store и блокує выполнение метода get, поки для цього же монитора не буде вызван метод notify().

Когда в методе put() додаєся товар и вызывается notify(), то метод get() отримує монитор и выходит из конструкции while (product<1), оскільки товар добавлен. Затем имитируется получение покупателем товара. Для цього виводиться сообщение, и уменьшается значение product: product--. И в конце вызов метода notify() дает сигнал методу put() продолжить работу.

В методе put() работает похожая логика, лише теперь метод put() повинен срабатывать, якщо в магазине не болїї трех товаров. Тому в цикле проверяется наличие товара, и якщо товар вже есть, то освобождаем монитор за допомогою wait() и ждем вызова notify() в методе get().

Таким чином, за допомогою wait() в методе get() мы ожидаем, когда производитель добавит новый продукт. А после добавления вызываем notify(), як бы говоря, что на складе освободилось одно место, и можна ще добавлять.

А в методе put() за допомогою wait() мы ожидаем освобождения места на складе. После того, як место освободится, добавляем товар и через notify() уведомляем покупателя о том, что он может забирать товар.
