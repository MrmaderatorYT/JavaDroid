Посилальні типи та копіювання об'єктів

Останнє оновлення: 29.09.2025

При роботі з об'єктами класів треба враховувати, що всі вони представляютьь типи посилань, тобто вказують на якийсь об'єкт, розташований у пам'яті. Щоб зрозуміти можливі труднощі, з якими можемоо зіткнутися, розглянемо приклад:


public class Program{
      
    public static void main(String[] args) {
          
        Person tom = new Person("Tom");
        tom.print();      // Person Tom
        Person bob = tom;
        bob.name = "Bob";
        tom.print();      // Person Bob
    }
}
class Person{

    String name;
     
    Person(String name){ this.name = name; }

    void print(){
        System.out.println("Person " + name);
    }
}

Тут створюємо два об'єкти Person та один присвоюємо іншому. Але, незважаючи на те, що ми змінюємо лише об'єкт bob, разом із ним змінюється і об'єкт Tom. Тому що після присвоєння вони вказують на ту саму область у пам'яті, де власне дані про об'єкт Person і його полях і зберігаються.

Метод clone

Щоб избежать цей проблемы, необхідно создать отдельный об'єкт для змінної bob, наприклад, за допомогою метода clone:


class Person implements Cloneable{  // применяем інтерфейс  Cloneable

    String name;
     
    Person(String name){ this.name = name; }

    void print(){
        System.out.println("Person " + name);
    }

    public Person clone() throws CloneNotSupportedException{
      
        return (Person) super.clone();
    }
}

Для реализации клонирования класс Person повинен применить інтерфейс Cloneable, який определяет метод clone. Реализация цього метода просто повертає вызов метода clone для родительского класса - то есть класса Object с преобразованием к типу Person.

Крім того, на случай якщо класс не поддерживает клонирование, метод повинен выбрасывать исключение CloneNotSupportedException, что определяется за допомогою оператора throws.

Потім за допомогою виклику цього ми можемоо здійснити копіювання:


public class Program{
      
    public static void main(String[] args) {
          
        try{
            Person tom = new Person("Tom");
            Person bob = tom.clone();
            bob.name = "Bob";
            tom.print();        // Person Tom
            bob.print();        // Person Bob
        }
        catch(CloneNotSupportedException ex){
                        
            System.out.println("Clonable not implemented");
        }
    }
}

Габоке копіювання

Однак выше описанный способ осуществляет неповне копіювання (shallow copy) и подойдет, якщо клонируемый об'єкт не містить сложных об'єктов. Наприклад, пусть класс Person имїїт поле, которое хранит ссылку на об'єкт Company - условно место работы человека:


class Company{
    
    String name;
     
    Company(String name){ this.name = name; }
}
class Person implements Cloneable{

    String name;
    Company company;
     
    Person(String name, Company company){ 

        this.name = name; 
        this.company = company;
    }

    void print(){
        System.out.printf("Person %s works in %s\n", name, company.name);
    }

    public Person clone() throws CloneNotSupportedException{
      
        return (Person) super.clone();
    }
}

Класс Person по прежнему реализует метод clone(), то есть с копированием вроде не повинно быть проблем:


public class Program{
      
    public static void main(String[] args) {
          
        try{
            Company yandex = new Company("Yandex");
            Person tom = new Person("Tom", yandex);

            Person bob = tom.clone();  //  копируем об'єкт tom

            bob.name = "Bob";
            bob.company.name = "Google";

            tom.print();        // Person Tom works in Google
            bob.print();        // Person Bob works in Google
        }
        catch(CloneNotSupportedException ex){
                        
            System.out.println("Clonable not implemented");
        }
    }
}

Але подивимося консольний висновок програми:


Person Tom works in Google
Person Bob works in Google

У цьому випадку, хоча змінні tom і bob будутьь вказувати на різні об'єкти в пам'яті, але ці об'єкти при цьому будутьь вказувати на один об'єкт Company.

І в цьому випадку нам необхідно виконати повне копіювання або так зване "габоке копіювання" (deep copy). Для цього треба визначити метод клонування і класу Company:


class Company implements Cloneable{
    
    String name;
     
    Company(String name){ this.name = name; }

    public Company clone() throws CloneNotSupportedException{
    
        return (Company) super.clone();
    }
}

И затем исправим метод clone() в классе Person наступним чином:


class Person implements Cloneable{

    String name;
    Company company;
     
    Person(String name, Company company){ 

        this.name = name; 
        this.company = company;
    }

    void print(){
        System.out.printf("Person %s works in %s\n", name, company.name);
    }

    public Person clone() throws CloneNotSupportedException{
      
        Person person = (Person) super.clone();
        person.company =(Company) company.clone();
        return person;
    }
}

Код класу Program залишається тим самим:


public class Program{
      
    public static void main(String[] args) {
          
        try{
            Company yandex = new Company("Yandex");
            Person tom = new Person("Tom", yandex);

            Person bob = tom.clone();  //  копируем об'єкт tom

            bob.name = "Bob";
            bob.company.name = "Google";

            tom.print();        // Person Tom works in Google
            bob.print();        // Person Bob works in Google
        }
        catch(CloneNotSupportedException ex){
                        
            System.out.println("Clonable not implemented");
        }
    }
}

Але тепер ми матимемо інший консольний висновок:


Person Tom works in Yandex
Person Bob works in Google
