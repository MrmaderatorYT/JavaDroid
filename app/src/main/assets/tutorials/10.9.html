Паралельні потоки

Останнє оновлення: 12.01.2022

Кроме последовательных потоков Stream API поддерживает параллельные потоки. Распараллеливание потоков дозволяє задействовать декілька ядер процессора (якщо целевая машина многоядерная) и тем самым может повысить производительность и ускорить вычисления. В то же час говорить, что применение параллельных потоков на многоядерных машинах однозначно повысит производительность - не совсем корректно. В каждом конкретном случае треба проверять и тестировать.

Щоб сделать обычный последовательный поток параллельным, треба вызвать у об'єкта Stream метод parallel. Крім того, можна також використовувати метод parallelStream() інтерфейса Collection для создания параллельного потока из коллекции.

В то же час якщо рабочая машина не є многоядерной, то поток буде выполняться як последовательный.

Застосування параллельных потоков во многих случаях буде аналогично. Например:


import java.util.Optional;
import java.util.stream.Stream;

public class Program {
 
    public static void main(String[] args) {
		
		Stream<Integer> numbersStream = Stream.of(1, 2, 3, 4, 5, 6);
		Optional<Integer> result = numbersStream.parallel().reduce((x,y)-> x*y);
		System.out.println(result.get()); // 720
	}
}

Ще один пример:


import java.util.Arrays;
import java.util.List;

public class Program {

    public static void main(String[] args) {

        List<String> people = Arrays.asList("Tom","Bob", "Sam", "Kate", "Tim");

        System.out.println("Последовательный поток");
        people.stream().filter(p->p.length()==3).forEach(System.out::println);

        System.out.println("\nПараллельный поток");
        people.parallelStream().filter(p->p.length()==3).forEach(System.out::println);
    }
}

В данном случае сначала для списка people создаем поток и выполняем над ним ряд операций в последовательном режиме. В частности, находим в списке строки, длина которых равна 3 и выводим их на консоль. В цем случае все операции с потоком будуть производиться над элементами списка в том порядке, в котоом элементы идут в списке.

Затем за допомогою метода people.parallelStream() для списка створюєся параллельный поток. Причем застосовуються те же операции, однак теперь порядок, в котором над элементами списка будуть производиться операции, не детерминирован.

Консольный вывод:

Последовательный поток
Tom
Bob
Sam
Tim

Параллельный поток
Sam
Tim
Bob
Tom

В случае с параллельным потоком вывод недетерминирован и может отличаться.

Однак не все функції можна без ущерба для точности вычисления перенести с последовательных потоков на параллельные. Прежде всего такие функції повинні быть без сохранения состояния и ассоциативными, то есть при выполнении слева направо давать тот же результат, что и при выполнении справа налево, як в случае с произведением чисел. Например:


Stream<Integer> numbersStream = Stream.of(1, 2, 3, 4, 5, 6);
Integer result = numbersStream.parallel().reduce(1, (x,y)->x * y);
System.out.println(result);

Фактически здесь происходит перемножение чисел. При цем нет разницы между 1 * 2 * 3 * 4 * (5 * 6) або 5 * 6 * 1 * (2 * 3) * 4. Мы можемо расставить скобки любым чином, разместить последовательность чисел в любом порядке, и все равно мы получим один и тот же результат. То есть данная операция є ассоциативной и поцему может быть распараллелена.

Підпросы производительности в параллельных операциях

Фактически применение параллельных потоков сводится к тому, что дані в потоке будуть разделены на части, каждая часть обробляєся на отдельном ядре процессора, и в конце эти части соединяются, и над ними выполняются финальные операции. Рассмотрим некоторые критерии, которые можуть повлиять на производительность в параллельных потоках:

  • Размер данных. Чем больше данных, тем сложнїї сначала разделять дані, а потом их соединять.

  • Количество ядер процессора. Теоретически, чем больше ядер в компьютере, тем быстрїї программа буде работать. Если на машине одно ядро, нет смысла застосовувати параллельные потоки.

  • Чем проще структура данных, с которой работает поток, тем быстрїї будуть происходить операции. Наприклад, дані из ArrayList легко використовувати, оскільки структура данной коллекции предполагает последовательность несвязанных данных. А вот коллекция типу LinkedList - не лучший вариант, оскільки в последовательном списке все элементы связаны с предыдущими/понаступними. И такие дані трудно распараллелить.

  • Над данными примитивных типов операции будуть производиться быстрїї, чем над об'єктами классов

Упорядоченность в параллельных потоках

Как правило, элементы передаются в поток в том же порядке, в котором они определены в источнике данных. При работе с параллельными потоками система сохраняет порядок следования элементов. Исключение составляет метод forEach(), який может выводить элементы в произвольном порядке. И чтобы сохранить порядок следования, необхідно застосовувати метод forEachOrdered:


phones.parallelStream()
    .sorted()
    .forEachOrdered(s->System.out.println(s));

Сохранение порядка в параллельных потоках увеличивает издержки при выполнении. Но якщо нам порядок не важен, то мы можемо отключить его сохранение и тем самым увеличить производительность, использовав метод unordered:


phones.parallelStream()
    .sorted()
	.unordered()
    .forEach(s->System.out.println(s));
