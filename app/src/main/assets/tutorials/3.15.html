Спадкування та узагальнення

Останнє оновлення: 10.10.2025

Узагальнені класи можуть брати участь у ієрархії спадкування: можуть успадковуватися з інших, або виконувати роль базових класів. Розглянемо різноманітні ситуації.

Базовий узагальнений клас

При наслідуванні від узагальненого класу клас-спадкоємець повинен передавати дані про тип у конструкції базового класу:


class Account<T>
{
    private T _id;
    T getId(){return _id;}
    Account(T id)
    {
        _id = id;
    }
}

class DepositAccount<T> extends Account<T>{

    DepositAccount(T id){
        super(id);
    }
}

У конструкторі DepositAccount йде звернення до конструктора базового класу, який передаються дані про тип.

Варіанти використання класів:


DepositAccount dAccount1 = new DepositAccount(20);
System.out.println(dAccount1.getId());
        
DepositAccount dAccount2 = new DepositAccount("12345");
System.out.println(dAccount2.getId());

При цьому клас-спадкоємець може додавати та використовувати якісь свої параметри типів:


class Account<T>
{
    private T _id;
    T getId(){return _id;}
    Account(T id)
    {
        _id = id;
    }
}

class DepositAccount<T, S> extends Account<T>{

    private S _name;
    S getName(){return _name;}
    DepositAccount(T id, S name){
        super(id);
        this._name=name;
    }
}

Варіанти використання:


DepositAccount<Integer, String> dAccount1 = new DepositAccount(20, "Tom");
System.out.println(dAccount1.getId() + " : " + dAccount1.getName());
        
DepositAccount<String, Integer> dAccount2 = new DepositAccount("12345", 23456);
System.out.println(dAccount2.getId() + " : " + dAccount2.getName());

І ще одна ситуація – клас-спадкоємець взагалі може не бути узагальненим:


class Account<T>
{
    private T _id;
    T getId(){return _id;}
    Account(T id)
    {
        _id = id;
    }
}

class DepositAccount extends Account<Integer>{

    DepositAccount(){
        super(5);
    }
}

Здесь при наследовании явным чином указывается тип, який буде використовуватися конструкциями базового класса, то есть тип Integer. Затем в конструктор базового класса передаєся значение именно цього типу - в данном случае число 5.

Варіант використання:


DepositAccount dAccount1 = new DepositAccount();
System.out.println(dAccount1.getId());

Узагальнений клас-спадкоємець

Також можливо ситуація, коли базовий клас є звичайним неузагальненим класом. Наприклад:


class Account
{
    private String _name;
    String getName(){return _name;}
    Account(String name)
    {
        _name=name;
    }
}

class DepositAccount<T> extends Account{

    private T _id;
    T getId(){return _id;}
    DepositAccount(String name, T id){
        super(name);
		_id = id;
    }
}

У цьому випадку використання конструкцій базового класу у спадкоємці відбувається як завжди.

Перетворення узагальнених типів

Об'єкт одного узагальненого типу можна привести до іншого типу, якщо вони використовують один і той самий тип. Розглянемо перетворення типів з прикладу наступних двох узагальнених класів:


class Account<T>
{
    private T _id;
    T getId(){return _id;}
    Account(T id)
    {
        _id = id;
    }
}

class DepositAccount<T> extends Account<T>{

    DepositAccount(T id){
        super(id);
    }
}

Ми можемоо привести об'єкт DepositAccount<Integer> до Account<Integer> або DepositAccount<String> до Account<String>:


DepositAccount<Integer> depAccount = new DepositAccount(10);
Account<Integer> account = (Account<Integer>)depAccount;
System.out.println(account.getId());

Але зробити те саме з різнотипними об'єктами ми не можемоо. Наприклад, наступний код не працюватиме:


DepositAccount<Integer> depAccount = new DepositAccount(10);
Account<String> account = (Account<String>)depAccount;
