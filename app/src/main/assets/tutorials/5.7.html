Очереди и стеки. Классы ArrayDeque и PriorityQueue

Останнє оновлення: 12.10.2025

Очереди представляють структуру данных, работающую по принципу FIFO (first in - first out). То есть чем раньше був добавлен элемент в коллекцию, тем раньше он из нїї видаляєся. Це стандартная модель однонаправленной очереди. Однак бывают и двунаправленные - то есть такие, в которых мы можемо добавить элемент не лише в начала, но и в конец. И соответственно удалить элемент не лише из конца, но и из начала.

Особенностью классов очередей є то, что они реализуют специальные інтерфейсы Queue або Deque.

Интерфейс Queue

Обобщенный інтерфейс Queue<E> расширяет базовый інтерфейс Collection и определяет поведение класса в качестве однонаправленной очереди. Свою функциональность он раскрывает через следующие методы:

  • E element(): повертає, но не видаляє, элемент из начала очереди. Если очередь пуста, генерує исключение NoSuchElementException

  • boolean offer(E obj): додає элемент obj в конец очереди. Если элемент удачно добавлен, повертає true, иначе - false

  • E peek(): повертає без удаления элемент из начала очереди. Если очередь пуста, повертає значение null

  • E poll(): повертає с удалением элемент из начала очереди. Если очередь пуста, повертає значение null

  • E remove(): повертає с удалением элемент из начала очереди. Если очередь пуста, генерує исключение NoSuchElementException

Таким чином, у всех классов, которые реализуют данный інтерфейс, буде метод offer для добавления в очередь, метод poll для извлечения элемента из головы очереди, и методы peek и element, позволяющие просто отримати элемент из головы очереди.

Таким чином, у Queue есть декілька методів, которые дозволяють добавлять, удалять и получать дані, но эти методы различаются в зависимости от того, генерують они исключение або повертають специальное значение - результат успешности операции. Краткая сводка по методам Queue

Queue и определяет поведение двунаправленной очереди, которая работает як обычная однонаправленная очередь, або як стек, действующий по принципу LIFO (последний вошел - первый вышел).

Интерфейс Deque определяет следующие методы:

  • void addFirst(E obj): додає элемент в начало очереди (унаследован от інтерфейса SequencedCollection<E>)

  • void addLast(E obj): додає элемент obj в конец очереди (унаследован от інтерфейса SequencedCollection<E>)

  • E getFirst(): повертає без удаления элемент из головы очереди. Если очередь пуста, генерує исключение NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • E getLast(): повертає без удаления последний элемент очереди. Если очередь пуста, генерує исключение NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • boolean offerFirst(E obj): додає элемент obj в самое начало очереди. Если элемент удачно добавлен, повертає true, иначе - false

  • boolean offerLast(E obj): додає элемент obj в конец очереди. Если элемент удачно добавлен, повертає true, иначе - false

  • E peekFirst(): повертає без удаления элемент из начала очереди. Если очередь пуста, повертає значение null

  • E peekLast(): повертає без удаления последний элемент очереди. Если очередь пуста, повертає значение null

  • E pollFirst(): повертає с удалением элемент из начала очереди. Если очередь пуста, повертає значение null

  • E pollLast(): повертає с удалением последний элемент очереди. Если очередь пуста, повертає значение null

  • E pop(): повертає с удалением элемент из начала очереди. Если очередь пуста, генерує исключение NoSuchElementException

  • void push(E element): додає элемент в самое начало очереди

  • E removeFirst(): повертає с удалением элемент из начала очереди. Если очередь пуста, генерує исключение NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • E removeLast(): повертає с удалением элемент из конца очереди. Если очередь пуста, генерує исключение NoSuchElementException< (унаследован от інтерфейса SequencedCollection<E>)/p>

  • boolean removeFirstOccurrence(Object obj): видаляє первый встреченный элемент obj из очереди. Если удаление произшло, то повертає true, иначе повертає false.

  • boolean removeLastOccurrence(Object obj): видаляє последний встреченный элемент obj из очереди. Если удаление произшло, то повертає true, иначе повертає false.

Таким чином, наличие методів pop и push дозволяє классам, реализующим цей элемент, действовать в качестве стека. В тоже час имеющийся функционал також дозволяє создавать двунаправленные очереди, что робить классы, применяющие данный інтерфейс, довольно гибкими.

Класс ArrayDeque

В Java очереди представлены рядом классов. Одни из низ - класс ArrayDeque<E>. Цет класс представляють обобщенную двунаправленную очередь, наследуя функционал от класса AbstractCollection и применяя інтерфейс Deque.

В классе ArrayDeque определены следующие конструкторы:

  • ArrayDeque(): створює пустую очередь

  • ArrayDeque(Collection<? extends E> col): створює очередь, наполненную элементами из коллекции col

  • ArrayDeque(int capacity): створює очередь с начальной емкостью capacity. Если мы явно не указываем начальную емкость, то емкость по умолчанию буде равна 16

Добавление в ArrayDeque:


import java.util.ArrayDeque;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        System.out.println(people);      // []
          
        // добавим ряд элементов
        people.add("Tom");              // [Tom]
        people.add("Bob");              // [Tom, Bob]
        // добавляем элемент в самое начало
        people.push("Sam");             // [Sam, Tom, Bob]
        // добавляем элемент в самое начало
        people.addFirst("Alice");       // [Alice, Sam, Tom, Bob]
        // добавляем элемент в конец коллекции
        people.addLast("Kate");         // [Alice, Sam, Tom, Bob, Kate]

        // добавляем элемент в самое начало
        people.offerFirst("Alex");       // [Alex, Alice, Sam, Tom, Bob]
        // добавляем элемент в конец коллекции
        people.offerLast("Bill");         // [Alex, Alice, Sam, Tom, Bob, Kate, Bill]
        // добавляем элемент в конец коллекции
        people.offer("Tim");            // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]

        // выводим все
        System.out.println(people);      // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]
    }
}

Видалення элементов из ArrayDeque:


import java.util.ArrayDeque;
import java.util.Collections;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        // добавляем ряд об'єктов
        Collections.addAll(people, new String[]{ "Alex", "Alice", "Sam", "Tom", "Bob", "Kate", "Bill", "Tim"});
        System.out.println(people);      // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]

        // удаляем элемент из начала
        var person = people.poll();
        System.out.println(person);      // Alex

        // удаляем элемент из начала в [Alice, Sam, Tom, Bob, Kate, Bill, Tim]
        person = people.poll();
        System.out.println(person);      // Alice

        // удаляем элемент из конца в [Sam, Tom, Bob, Kate, Bill, Tim]
        person = people.pollLast();
        System.out.println(person);      // Tim

        // удаляем элемент из начала  в [Sam, Tom, Bob, Kate, Bill]
        person = people.remove();
        System.out.println(person);      // Sam

        // удаляем элемент из начала в [Tom, Bob, Kate, Bill]
        person = people.removeFirst();
        System.out.println(person);      // Tom

        // удаляем элемент из конца в [Bob, Kate, Bill]
        person = people.removeLast();
        System.out.println(person);      // Bill

        // удаление произвольного элемента из [Bob, Kate]
        boolean removed = people.remove("Kate");
        System.out.println("Kate is removed: " + removed);  // Kate is removed: true
        // удаление произвольного элемента из [Bob]
        removed = people.remove("Charl");
        System.out.println("Charl is removed: " + removed);  // Charl is removed: false

        // выводим оставшиеся
        System.out.println(people);      // [Bob]

        people.clear();
        // удаляем оставшиеся
        System.out.println(people);      // [Bob]
    }
}

Получение элементов из очереди:


import java.util.ArrayDeque;
import java.util.Collections;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        // добавляем ряд об'єктов
        Collections.addAll(people, new String[]{"Tom", "Bob", "Sam"});

        // получаем элемент из начала
        var first = people.element();
        System.out.println(first);      // Tom

        first = people.peek();
        System.out.println(first);      // Tom

        first = people.peekFirst();
        System.out.println(first);      // Tom

        first = people.getFirst();
        System.out.println(first);      // Tom

        // получаем элемент из конца
        var last = people.getLast();
        System.out.println(last);      // Sam

        last = people.peekLast();
        System.out.println(last);      // Sam

        // проверка наличия
        boolean available = people.contains("Tom");
        System.out.println("Tom is avalable: " + available);      // true
        available = people.contains("Alex");
        System.out.println("Alex is avalable: " + available);      // false

        // перебор без извлечения
        for(var p : people){
          
            System.out.println(p);
        }

        // перебор коллекции с извлечением      
        while(people.peek() != null){
            // извлечение c начала
            System.out.println(people.pop());
        }
    }
}

Очереди приоритетов PriorityQueue

Очередь приоритетов, представленная в мові Java классом PriorityQueue, извлекает элементы в отсортированном порядке. То есть, при каждом получении або удалении элемента с начала очереди мы будем получать условно говоря "наименьший элемент". Однак очередь приоритетов не сортує все свои элементы. При итерации по элементам они не обязательно будуть отсортированы. Очередь приоритетов использует структуру данных под названием куча. Куча — це самоорганизующїїся двоичное дерево, в котором операции добавления и удаления перемещают наименьший элемент в корень, не тратя час на сортировку всех элементов.

Часто очередь приоритетов використовується для планирования заданий. Каждое задание имїїт приоритет. Задания добавляются в случайном порядке. Всякий раз, когда удается запустить новое задание, задание с наивысшим приоритетом видаляєся из очереди.

Поскольку при работе очереди приоритетов застосовується сортировка, то PriorityQueue может містятьь або элементы класса, який реализет інтерфейс Comparable, або через конструктор при создании очереди приоритетов треба передать об'єкт Comparator, який определяет логику сортировки. Конструкторы класса PriortyQueue:

  • PriorityQueue(): створює очередь PriorityQueue с начальной вместимостью по умолчанию (11), элементы которой упорядочены в соответствии с их естественным порядком

  • PriorityQueue(int initialCapacity): створює очередь PriorityQueue с указанной начальной вместимостью, элементы которой упорядочены в соответствии с их естественным порядком

  • PriorityQueue(int initialCapacity, Comparator<? super E> comparator): створює очередь PriorityQueue с указанной начальной вместимостью, элементы которой упорядочены в соответствии с указанным компаратором

  • PriorityQueue(Collection<? extends E> c): створює очередь PriorityQueue, которая містить элементы изуказанной коллекции

  • PriorityQueue(Comparator<? super E> comparator): створює очередь PriorityQueue с начальной вместимостью по умолчанию, элементы которой упорядочены в соответствии с указанным компаратором

  • PriorityQueue(PriorityQueue<? extends E> c): створює PriorityQueue, которая містить элементы из другой PriorityQueue

  • PriorityQueue(SortedSet<? extends E> c): створює PriorityQueue, которая містить элементы из SortedSet

PriorityQueue реализует інтерфейсы Collection<E> и Queue<E> и соответственно методы цих інтерфейсов. Застосування PriorityQueue:


import java.util.PriorityQueue;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new PriorityQueue<String>();
        // добавляем ряд об'єктов
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");

        System.out.println(people);      // [Bob, Tom, Sam]

        var numbers = new PriorityQueue<Integer>();
        // добавляем ряд об'єктов
        numbers.add(3);
        numbers.add(1);
        numbers.add(5);
        numbers.add(4);

        System.out.println(numbers);      // [1, 3, 5, 4]
    }
}

Здесь определены две очереди приоритетов - для строк и чисел. И в обоих случаях "наименьший" элемент располагается в самом начале. Хотя в целом коллекция не сортуєся.
