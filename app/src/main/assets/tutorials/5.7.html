Очереди и стеки. Класы ArrayDeque и PriorityQueue

Останнє оновлення: 12.10.2025

Очереди представляютьь структуру данных, работающую по принципу FIFO (first in - first out). То есть чем раньше був добавлен елемент в колекцію, тем раньше он из нїї видаляєся. Це стандартная модель однонаправленной очереди. Однак бывают и двунаправленные - то есть такие, в которых мы можемо добавить елемент не лише в начала, но и в конец. И соответственно удалить елемент не лише из конца, но и из начала.

Особенностью класов очередей є то, что они реалізують специальные інтерфейсы Queue або Deque.

Інтерфейс Queue

Обобщенный інтерфейс Queue<E> расширяет базовый інтерфейс Collection и визначає поведение класа в качестве однонаправленной очереди. Свою функциональность он раскрывает через следующие методи:

  • E element(): повертає, но не видаляє, елемент из начала очереди. Если очередь пуста, генерує виняток NoSuchElementException

  • boolean offer(E obj): додає елемент obj в конец очереди. Если елемент удачно добавлен, повертає true, иначе - false

  • E peek(): повертає без удаления елемент из начала очереди. Если очередь пуста, повертає значення null

  • E poll(): повертає с видаленням елемент из начала очереди. Если очередь пуста, повертає значення null

  • E remove(): повертає с видаленням елемент из начала очереди. Если очередь пуста, генерує виняток NoSuchElementException

Таким чином, у всех класов, которые реалізують данный інтерфейс, буде метод offer для добавления в очередь, метод poll для извлечения елемента из головы очереди, и методи peek и element, позволяющие просто отримати елемент из головы очереди.

Таким чином, у Queue есть декілька методів, которые дозволяють добавлять, удалять и получать дані, но эти методи различаются в зависимости от того, генерують они виняток або повертають специальное значення - результат успешности операції. Краткая сводка по методум Queue

Queue и визначає поведение двунаправленной очереди, которая работает як обычная однонаправленная очередь, або як стек, действующий по принципу LIFO (післядний вошел - первый вышел).

Інтерфейс Deque визначає следующие методи:

  • void addFirst(E obj): додає елемент в начало очереди (унаследован от інтерфейса SequencedCollection<E>)

  • void addLast(E obj): додає елемент obj в конец очереди (унаследован от інтерфейса SequencedCollection<E>)

  • E getFirst(): повертає без удаления елемент из головы очереди. Если очередь пуста, генерує виняток NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • E getLast(): повертає без удаления післядний елемент очереди. Если очередь пуста, генерує виняток NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • boolean offerFirst(E obj): додає елемент obj в самое начало очереди. Если елемент удачно добавлен, повертає true, иначе - false

  • boolean offerLast(E obj): додає елемент obj в конец очереди. Если елемент удачно добавлен, повертає true, иначе - false

  • E peekFirst(): повертає без удаления елемент из начала очереди. Если очередь пуста, повертає значення null

  • E peekLast(): повертає без удаления післядний елемент очереди. Если очередь пуста, повертає значення null

  • E pollFirst(): повертає с видаленням елемент из начала очереди. Если очередь пуста, повертає значення null

  • E pollLast(): повертає с видаленням післядний елемент очереди. Если очередь пуста, повертає значення null

  • E pop(): повертає с видаленням елемент из начала очереди. Если очередь пуста, генерує виняток NoSuchElementException

  • void push(E element): додає елемент в самое начало очереди

  • E removeFirst(): повертає с видаленням елемент из начала очереди. Если очередь пуста, генерує виняток NoSuchElementException (унаследован от інтерфейса SequencedCollection<E>)

  • E removeLast(): повертає с видаленням елемент из конца очереди. Если очередь пуста, генерує виняток NoSuchElementException< (унаследован от інтерфейса SequencedCollection<E>)/p>

  • boolean removeFirstOccurrence(Object obj): видаляє первый встреченный елемент obj из очереди. Если видалення произшло, то повертає true, иначе повертає false.

  • boolean removeLastOccurrence(Object obj): видаляє післядний встреченный елемент obj из очереди. Если видалення произшло, то повертає true, иначе повертає false.

Таким чином, наличие методів pop и push дозволяє класам, реализующим цей елемент, действовать в качестве стека. В тоже час имеющийся функционал також дозволяє создавать двунаправленные очереди, что робить класы, применяющие данный інтерфейс, досить гибкими.

Клас ArrayDeque

В Java очереди представлены рядом класов. Одни из низ - клас ArrayDeque<E>. Цет клас представляютьь обобщенную двунаправленную очередь, наследуя функционал от класа AbstractCollection и применяя інтерфейс Deque.

В класе ArrayDeque визначенийы следующие конструктори:

  • ArrayDeque(): створює пустую очередь

  • ArrayDeque(Collection<? extends E> col): створює очередь, наполненную елементами из колекції col

  • ArrayDeque(int capacity): створює очередь с начальной емкостью capacity. Если мы явно не указываем начальную емкость, то емкость по умолчанию буде равна 16

Додавання в ArrayDeque:


import java.util.ArrayDeque;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        System.out.println(people);      // []
          
        // додамо ряд елементов
        people.add("Tom");              // [Tom]
        people.add("Bob");              // [Tom, Bob]
        // додаємо елемент в самое начало
        people.push("Sam");             // [Sam, Tom, Bob]
        // додаємо елемент в самое начало
        people.addFirst("Alice");       // [Alice, Sam, Tom, Bob]
        // додаємо елемент в конец колекції
        people.addLast("Kate");         // [Alice, Sam, Tom, Bob, Kate]

        // додаємо елемент в самое начало
        people.offerFirst("Alex");       // [Alex, Alice, Sam, Tom, Bob]
        // додаємо елемент в конец колекції
        people.offerLast("Bill");         // [Alex, Alice, Sam, Tom, Bob, Kate, Bill]
        // додаємо елемент в конец колекції
        people.offer("Tim");            // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]

        // вивідим все
        System.out.println(people);      // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]
    }
}

Видалення елементов из ArrayDeque:


import java.util.ArrayDeque;
import java.util.Collections;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        // додаємо ряд об'єктов
        Collections.addAll(people, new String[]{ "Alex", "Alice", "Sam", "Tom", "Bob", "Kate", "Bill", "Tim"});
        System.out.println(people);      // [Alex, Alice, Sam, Tom, Bob, Kate, Bill, Tim]

        // видаляємо елемент из начала
        var person = people.poll();
        System.out.println(person);      // Alex

        // видаляємо елемент из начала в [Alice, Sam, Tom, Bob, Kate, Bill, Tim]
        person = people.poll();
        System.out.println(person);      // Alice

        // видаляємо елемент из конца в [Sam, Tom, Bob, Kate, Bill, Tim]
        person = people.pollLast();
        System.out.println(person);      // Tim

        // видаляємо елемент из начала  в [Sam, Tom, Bob, Kate, Bill]
        person = people.remove();
        System.out.println(person);      // Sam

        // видаляємо елемент из начала в [Tom, Bob, Kate, Bill]
        person = people.removeFirst();
        System.out.println(person);      // Tom

        // видаляємо елемент из конца в [Bob, Kate, Bill]
        person = people.removeLast();
        System.out.println(person);      // Bill

        // видалення произвольного елемента из [Bob, Kate]
        boolean removed = people.remove("Kate");
        System.out.println("Kate is removed: " + removed);  // Kate is removed: true
        // видалення произвольного елемента из [Bob]
        removed = people.remove("Charl");
        System.out.println("Charl is removed: " + removed);  // Charl is removed: false

        // вивідим оставшиеся
        System.out.println(people);      // [Bob]

        people.clear();
        // видаляємо оставшиеся
        System.out.println(people);      // [Bob]
    }
}

Отримання елементов из очереди:


import java.util.ArrayDeque;
import java.util.Collections;
 
public class Program{
      
    public static void main(String[] args) {
          
        ArrayDeque<String> people = new ArrayDeque<String>();
        // додаємо ряд об'єктов
        Collections.addAll(people, new String[]{"Tom", "Bob", "Sam"});

        // отримуємо елемент из начала
        var first = people.element();
        System.out.println(first);      // Tom

        first = people.peek();
        System.out.println(first);      // Tom

        first = people.peekFirst();
        System.out.println(first);      // Tom

        first = people.getFirst();
        System.out.println(first);      // Tom

        // отримуємо елемент из конца
        var last = people.getLast();
        System.out.println(last);      // Sam

        last = people.peekLast();
        System.out.println(last);      // Sam

        // проверка наличия
        boolean available = people.contains("Tom");
        System.out.println("Tom is avalable: " + available);      // true
        available = people.contains("Alex");
        System.out.println("Alex is avalable: " + available);      // false

        // перебор без извлечения
        for(var p : people){
          
            System.out.println(p);
        }

        // перебор колекції с извлечением      
        while(people.peek() != null){
            // извлечение c начала
            System.out.println(people.pop());
        }
    }
}

Очереди приоритетов PriorityQueue

Очередь приоритетов, представленная в мові Java класом PriorityQueue, извлекает елементы в отсортированном порядке. То есть, при каждом получении або удалении елемента с начала очереди мы будем получать условно говоря "наименьший елемент". Однак очередь приоритетов не сортує все свои елементы. При итерации по елементам они не обязательно будуть отсортированы. Очередь приоритетов використовує структуру данных під названием куча. Куча — це самоорганизующїїся двоичное дерево, в котором операції добавления и удаления перемещают наименьший елемент в корень, не тратя час на сортировку всех елементов.

Часто очередь приоритетов використовується для планирования заданий. Каждое задание имїїт приоритет. Задания додаються в случайном порядке. Всякий раз, когда удается запустить новое задание, задание с наивысшим приоритетом видаляєся из очереди.

Поскольку при работе очереди приоритетов застосовується сортування, то PriorityQueue может містятьь або елементы класа, який реализет інтерфейс Comparable, або через конструктор при створенні очереди приоритетов треба передать об'єкт Comparator, який визначає логику сортировки. Конструктори класа PriortyQueue:

  • PriorityQueue(): створює очередь PriorityQueue с начальной вместимостью по умолчанию (11), елементы которой упорядочены в соответствии с их естественным порядком

  • PriorityQueue(int initialCapacity): створює очередь PriorityQueue с указанной начальной вместимостью, елементы которой упорядочены в соответствии с их естественным порядком

  • PriorityQueue(int initialCapacity, Comparator<? super E> comparator): створює очередь PriorityQueue с указанной начальной вместимостью, елементы которой упорядочены в соответствии с указанным компаратором

  • PriorityQueue(Collection<? extends E> c): створює очередь PriorityQueue, которая містить елементы изуказанной колекції

  • PriorityQueue(Comparator<? super E> comparator): створює очередь PriorityQueue с начальной вместимостью по умолчанию, елементы которой упорядочены в соответствии с указанным компаратором

  • PriorityQueue(PriorityQueue<? extends E> c): створює PriorityQueue, которая містить елементы из другой PriorityQueue

  • PriorityQueue(SortedSet<? extends E> c): створює PriorityQueue, которая містить елементы из SortedSet

PriorityQueue реалізує інтерфейсы Collection<E> и Queue<E> и соответственно методи цих інтерфейсов. Застосування PriorityQueue:


import java.util.PriorityQueue;
 
public class Program{
      
    public static void main(String[] args) {
          
        var people = new PriorityQueue<String>();
        // додаємо ряд об'єктов
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");

        System.out.println(people);      // [Bob, Tom, Sam]

        var numbers = new PriorityQueue<Integer>();
        // додаємо ряд об'єктов
        numbers.add(3);
        numbers.add(1);
        numbers.add(5);
        numbers.add(4);

        System.out.println(numbers);      // [1, 3, 5, 4]
    }
}

Здесь визначенийы две очереди приоритетов - для рядків и чисел. И в обоих случаях "наименьший" елемент располагается в самом начале. Хотя в целом колекція не сортуєся.
