Закриття потіків та конструкція try-з-ресурсами

Останнє оновлення: 07.10.2025

При завершении работы с потіком его треба закрыть за допомогою методу close(), який визначений в інтерфейсе Closeable. Метод close имїїт следующїї визначенийие:


void close() throws IOException

Цет інтерфейс вже реалізуєся в класах InputStream и OutputStream, а через них и во всех класах потіков.

При закрытии потіка звільняються все выделенные для него ресурсы, наприклад, файл. В случае, якщо потік окажется не закрыт, может происходить утечка памяти.

Есть два способа закрытия файлу. Первый традиционный заключается в использовании блока try..catch..finally. Наприклад, считаємо дані из файлу:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        FileInputStream fin=null;
        try
        {
            fin = new FileInputStream("notes.txt");
            
            int i=-1;
            while((i=fin.read())!=-1){
            
                System.out.print((char)i);
            }
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
        finally{
            
            try{
            
                if(fin!=null)
                    fin.close();
            }
            catch(IOException ex){
            
                System.out.println(ex.getMessage());
            }
        }  
    } 
}

Поскольку при открытии або считывании файлу может произойти ошибка ввода-вивіда, то код считывания помещается в блок try. И чтобы быть уверенным, что потік в любом случае закроется, даже якщо при работе с ним возникнет ошибка, вызов методу close() помещается в блок finally. И, оскільки метод close() також в случае ошибки может генерировать виняток IOException, то его вызов також помещается во вложенный блок try..catch

try-с-ресурсами

Но язык Java дозволяє використовувати болїї удобную конструкцию, котораяя автоматически вызывает метод close() - конструкция try-with-resources (try-с-ресурсами). Данная конструкция работает с об'єктами, которые реалізують інтерфейс AutoCloseable. Так як все класы потіков реалізують інтерфейс Closeable, який в свою очередь найдеся от AutoCloseable, то их також можна використовувати в данной конструкции

Синтаксис конструкции следующий:


try(Resource res = ...){

    // действия с ресурсом res
}

Данная конструкция також не исключает использования блоков catch, которые також можуть размещаться після блока try:


try(Resource res = ...){

    // действия с ресурсом res
}
catch(виняток){
    // обробка винятки
}

Итак, перепишем предыдущий пример с використанням конструкции try-with-resources:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        try(FileInputStream fin = new FileInputStream("notes.txt"))
        {
            int i=-1;
            while((i = fin.read()) != -1){
            
                System.out.print((char)i);
            }   
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
    } 
}

Після окончания работы в блоке try у ресурса (у даному випадку у об'єкта FileInputStream) автоматически вызывается метод close().

Если нам треба використовувати декілька потіков, которые після виконання треба закрыть, то мы можемо указать об'єкты потіков через точку с комою:


try(FileInputStream fin = new FileInputStream("hello.txt"); 
        FileOutputStream fos = new FileOutputStream("hello2.txt"))
{
	//..................
}

Також зверніть увагу, что у даному випадку ресурс визначаєся непосредственно в заголовке конструкции try:

try(FileInputStream fin = new FileInputStream("notes.txt"))

Однак ресурс может быть создан и вне цей конструкции, но при передаче в заголовок конструкции try після її завершення цей ресурс все равно буде закрыт:


import java.io.*;

public class Program {

    public static void main(String[] args) throws FileNotFoundException {
        
        FileInputStream fin = new FileInputStream("notes.txt");
        try(fin)
        {
            int i=-1;
            while((i = fin.read()) != -1){
            
                System.out.print((char)i);
            }   
        }  // здесь вызывается fin.close()
    } 
}
