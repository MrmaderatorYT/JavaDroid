Закриття потоків та конструкція try-з-ресурсами

Останнє оновлення: 07.10.2025

При завершении работы с потоком его треба закрыть за допомогою метода close(), який определен в інтерфейсе Closeable. Метод close имїїт следующїї определение:


void close() throws IOException

Цет інтерфейс вже реализуется в классах InputStream и OutputStream, а через них и во всех классах потоков.

При закрытии потока звільняються все выделенные для него ресурсы, наприклад, файл. В случае, якщо поток окажется не закрыт, может происходить утечка памяти.

Есть два способа закрытия файла. Первый традиционный заключается в использовании блока try..catch..finally. Наприклад, считаем дані из файла:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        FileInputStream fin=null;
        try
        {
            fin = new FileInputStream("notes.txt");
            
            int i=-1;
            while((i=fin.read())!=-1){
            
                System.out.print((char)i);
            }
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
        finally{
            
            try{
            
                if(fin!=null)
                    fin.close();
            }
            catch(IOException ex){
            
                System.out.println(ex.getMessage());
            }
        }  
    } 
}

Поскольку при открытии або считывании файла может произойти ошибка ввода-вывода, то код считывания помещается в блок try. И чтобы быть уверенным, что поток в любом случае закроется, даже якщо при работе с ним возникнет ошибка, вызов метода close() помещается в блок finally. И, оскільки метод close() також в случае ошибки может генерировать исключение IOException, то его вызов також помещается во вложенный блок try..catch

try-с-ресурсами

Но язык Java дозволяє використовувати болїї удобную конструкцию, котораяя автоматически вызывает метод close() - конструкция try-with-resources (try-с-ресурсами). Данная конструкция работает с об'єктами, которые реализуют інтерфейс AutoCloseable. Так як все классы потоков реализуют інтерфейс Closeable, який в свою очередь найдеся от AutoCloseable, то их також можна використовувати в данной конструкции

Синтаксис конструкции следующий:


try(Resource res = ...){

    // действия с ресурсом res
}

Данная конструкция також не исключает использования блоков catch, которые також можуть размещаться после блока try:


try(Resource res = ...){

    // действия с ресурсом res
}
catch(исключение){
    // обработка исключения
}

Итак, перепишем предыдущий пример с использованием конструкции try-with-resources:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        try(FileInputStream fin = new FileInputStream("notes.txt"))
        {
            int i=-1;
            while((i = fin.read()) != -1){
            
                System.out.print((char)i);
            }   
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
    } 
}

После окончания работы в блоке try у ресурса (в данном случае у об'єкта FileInputStream) автоматически вызывается метод close().

Если нам треба використовувати декілька потоков, которые после виконання треба закрыть, то мы можемо указать об'єкты потоков через точку с комою:


try(FileInputStream fin = new FileInputStream("hello.txt"); 
        FileOutputStream fos = new FileOutputStream("hello2.txt"))
{
	//..................
}

Також обратите внимание, что в данном случае ресурс определяется непосредственно в заголовке конструкции try:

try(FileInputStream fin = new FileInputStream("notes.txt"))

Однак ресурс может быть создан и вне цей конструкции, но при передаче в заголовок конструкции try после її завершения цей ресурс все равно буде закрыт:


import java.io.*;

public class Program {

    public static void main(String[] args) throws FileNotFoundException {
        
        FileInputStream fin = new FileInputStream("notes.txt");
        try(fin)
        {
            int i=-1;
            while((i = fin.read()) != -1){
            
                System.out.print((char)i);
            }   
        }  // здесь вызывается fin.close()
    } 
}
