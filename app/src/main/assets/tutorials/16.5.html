Складні дані та MemoryLayout

Останнє оновлення: 21.12.2025

Ранїї мы рассмотрели, як работать с сегментами памяти, которые містять строки UTF-8 и массивы значений примитивного типу. Но возникает вопрос, як быть с болїї сложными по структуре данными? В языке C для описания таких данных застосовуються структуры. В Java для преобразования в нативний формат и обратно застосовуються кастомные структуры памяти, которые визначаються за допомогою метода MemoryLayout.structLayout():

static StructLayout structLayout(MemoryLayout... elements)

В цей метод передаєся набор значений типу MemoryLayout.

К примеру, якщо на Си у нас есть следующая структура Point, которая представляє точку:

typedef struct {
    int x;
    int y;
} Point;

То в Java цей структуре буде соответствовать следующая компоновка памяти:

StructLayout POINT = MemoryLayout.structLayout(
    ValueLayout.JAVA_INT.withName("x"),
    ValueLayout.JAVA_INT.withName("y")
);

Интерфейс MemoryLayout надає ряд специальных методів для управления выделенной памятью, в частности, метод withName(String name) повертає структуру памяти (об'єкт MemoryLayout) с теми же характеристиками, что и данная структура, но с заданным именем. Таким чином, за допомогою выражения

ValueLayout.JAVA_INT.withName("x")

определяем сегмент памяти размером в 4 байта (размера типу int в Java), який сопоставлен с именем "x".

Выделение сегмента памяти для данной структуры:


import java.lang.foreign.*;
import static java.lang.foreign.ValueLayout.*;

public class Program {

    public static void main(String[] args) {

        StructLayout POINT = MemoryLayout.structLayout(
            ValueLayout.JAVA_INT.withName("x"),
            ValueLayout.JAVA_INT.withName("y")
        );
        try (Arena arena = Arena.ofConfined()) {

            // выделяем память для 1 об'єкта POINT
            MemorySegment point_seg = arena.allocate(POINT);

            System.out.println(point_seg);  // MemorySegment{ kind: native, address: 0x72760c180400, byteSize: 8 }
        } 
    }
}

При сопоставлении типов C и Java длины байтов повинні совпадать. Но беззнаковые типы C сопоставляются с их эквивалентами в Java. При обработке цих значений в Java необхідно самостоятельно обробляти их як беззнаковые значения.

В примере выше мы не просто так установили для выделенных сегментов памяти имена ("x" и "y"). Используя методы сегментов памяти, мы можемо затем отримати або установить компоненты структуры:


import java.lang.foreign.*;
import static java.lang.foreign.ValueLayout.*;

public class Program {

    public static void main(String[] args) {

        // определяем структуру
        StructLayout POINT = MemoryLayout.structLayout(
            ValueLayout.JAVA_INT.withName("x"),
            ValueLayout.JAVA_INT.withName("y")
        );
        try (Arena arena = Arena.ofConfined()) {

            // выделяем память для 1 об'єкта POINT
            MemorySegment point_seg = arena.allocate(POINT);

            // устанавливаем значения x и y
            point_seg.set(ValueLayout.JAVA_INT, POINT.byteOffset(PathElement.groupElement("x")), 20);
            point_seg.set(ValueLayout.JAVA_INT, POINT.byteOffset(PathElement.groupElement("y")), 15);

            // получаем значения x и y
            int x = point_seg.get(ValueLayout.JAVA_INT, POINT.byteOffset(PathElement.groupElement("x")));
            int y = point_seg.get(ValueLayout.JAVA_INT, POINT.byteOffset(PathElement.groupElement("y")));

            System.out.println("x: " + x);  // 20
            System.out.println("y: " + y);  // 15
        } 
    }
}

Методи get/set обращаются к определенным участкам памяти, используя смщения. Для получения смщения застосовується метод byteOffset(), в який передаєся путь к элементу в памяти. Для получения пути використовується имя. Наприклад, выражение

POINT.byteOffset(PathElement.groupElement("x"))

дозволяє отримати смщение относительно начала структуры для сегмента, який сопоставляется с именем "x".

В принципе мы могли бы передать и числовое значение смщения, якщо оно нам известно:


// поле y смщено на 4 байта относительно начала структуры
point_seg.set(ValueLayout.JAVA_INT, 4, 17);

// получаем значения  y
int y = point_seg.get(ValueLayout.JAVA_INT, POINT.byteOffset(PathElement.groupElement("y")));

System.out.println("y: " + y);  // y: 17

Об'єднання

Подобно тому, як StructLayout описує расположение структур C, тип UnionLayout описує об'єднание (union) языка C. Работа с UnionLayout во многом аналогична работе с StructLayout.

Составная компоновка SequenceLayout

Говоря о компоновке памяти, йде відзначити інтерфейс SequenceLayout, який надає составную компоновку - однородное повторение заданной компоновки элементов або последовательность. Количество повторений називається количеством элементов в последовательности. Последовательность можна рассматривать як структурную компоновку, в которой количество повторений элементов равно количеству элементов в последовательности.

Для создания цього типу компоновки памяти застосовується метод MemoryLayout.sequenceLayout()


static SequenceLayout sequenceLayout(long elementCount, MemoryLayout elementLayout)

Метод приймає количество элементов и компоновку элемента в виде значения MemoryLayout

Наприклад, используем SequenceLayout для определения массива структур:


// массив структур
SequenceLayout points_layout = MemoryLayout.sequenceLayout(count, 
    MemoryLayout.structLayout( 
        JAVA_INT.withName("x"),
        JAVA_INT.withName("y")
    )
);

Для обращения к компоненту y структуры с некоторым индексом i, нам потрібно вычислить смщение в байтах:


long offset = points_layout.byteOffset( 
    PathElement.sequenceElement(i), 
    PathElement.groupElement("y")
);

Здесь метод byteOffset отримує последовательность об'єктов MemoryLayout.PathElement, которые дозволяють отримати смщение к потрібному компоненту.

Получив соответствующий сегмент памяти, мы можемо отримати або установить само значение компонента:


MemorySegment points = Arena.global().allocate(point_layout);
int value = 6;  // значение для y
points.set(JAVA_FLOAT, offset, value, y);

Пример программы полностью:


import java.lang.foreign.*;
import static java.lang.foreign.ValueLayout.*;

public class Program {

    public static void main(String[] args) {

        int count = 4;  // количество структур
        // массив структур
        SequenceLayout points_layout = MemoryLayout.sequenceLayout(count, 
            MemoryLayout.structLayout( 
                JAVA_INT.withName("x"),
                JAVA_INT.withName("y")
            )
        );
        try (Arena arena = Arena.ofConfined()) {

            // выделяем память для массива структур
            MemorySegment points = arena.allocate(points_layout);

            int index = 1;      // индекс потрібной структуры в массиве

            // получаем смщение поля y для структуры с индексом index
            long offset = points_layout.byteOffset( 
                PathElement.sequenceElement(index), 
                PathElement.groupElement("y")
            );
            // устанавливаем значение y
            points.set(ValueLayout.JAVA_INT, offset, 24);

            // получаем значение y для структуры с индексом index
            int y = points.get(ValueLayout.JAVA_INT, offset);

            System.out.println("y: " + y);  // 24
        } 
    }
}
