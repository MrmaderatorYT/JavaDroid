Порозрядні операції

Останнє оновлення: 16.09.2025

Порозрядні операції виконуються над окремими розрядами чи бітами чисел. У цих операціях як операнди можуть виступати тільки цілі числа.

Кожне число має певну двійкову виставу. Наприклад, число 4 у двійковій системі 100, а число 5 - 101 і так далі.

Наприклад, візьмемо такі перемінні:


byte b = 7;		// 0000 0111
short s = 7;	// 0000 0000 0000 0111

Тип byte занимает 1 байт або 8 бит, соответственно представлен 8 разрядами. Тому значение змінної b в двоичном коде буде равно 00000111. Тип short занимает в памяти 2 байта або 16 бит, поцему число данного типу буде представлено 16 разрядами. И в данном случае переменная s в двоичной системе буде иметь значение 0000 0000 0000 0111.

Для спрощення розуміння ми можемоо використати двійковий запис чисел. Наприклад:


class Program{

    public static void main(String[] args) {  

        byte b = 0b0000_0111;               // 7
        short s = 0b0000_0000_0000_0111;	// 7
        System.out.println("byte b = " + b);        // byte b = 7
        System.out.println("short s = " + s);       // short s = 7
    }
}

Для записи чисел со знаком в Java застосовується додатковий код (two's complement), при котором старший разряд є знаковым. Если його значення равно 0, то число положительное, и его двоичное представление не отличается от представления беззнакового числа. Наприклад, 0000 0001 в десятичной системе 1.

Якщо старший розряд дорівнює 1, ми маємо справу з негативним числом. Наприклад, 1111 1111 у десятковій системі становить -1. Відповідно, 1111 0011 становить -13.

Розглянемо такий приклад:


class Program{

    public static void main(String[] args) {  

        int num = 0b1111_1111_1111_1111_1111_1111_1111_1110;               // -2

        System.out.println("num = " + num);        // num = -2
    }
}

Здесь переменная num представляє значение 0b1111_1111_1111_1111_1111_1111_1111_1110 - все единицы, кроме последнего нуля (формальное первого нуля), что представляє -2. Но получим из него положительное число - 2. Сначала инвертируем число - все единицы сделаем нулями, а все нули - единицами:


// ~ - операция инверсии
~ 0b1111_1111_1111_1111_1111_1111_1111_1110
= 
  0b0000_0000_0000_0000_0000_0000_0000_0001

І далі додаємо 1:


0b0000_0000_0000_0000_0000_0000_0000_0001
+                                       
                                        1
=
0b0000_0000_0000_0000_0000_0000_0000_0010

Число 102 в десятичной системе равно 2.

Логічні операції

Логічні операції над числами представляютьь порозрядні операції. У даному випадку числа розглядаються в двійковому поданні, наприклад, 2 у двійковій системі дорівнює 10 і має два розряди, число 7 - 111 і має три розряди.

  • & (логическое умножение)

    Множення проводиться порозрядно, і якщо в обох операндів значення розрядів дорівнює 1, то операція повертає 1, інакше повертається число 0. Наприклад:

    
 int a1 = 2; //010
 int b1 = 5;//101
 System.out.println(a1&b1); // результат 0
            
int a2 = 4; //100
int b2 = 5; //101
System.out.println(a2 & b2); // результат 4

    У першому випадку ми два числа 2 і 5. 2 у двійковому вигляді представляє число 010, а 5 - 101. Порозрядне множення чисел (0*1, 1*0, 0*1) дає результат 000.

    У іншому випадку у нас замість двійки число 4, яке має у першому розряді 1, так само як і у числа 5, тому тут результатом операції (1*1, 0*0, 0 *1) = 100 буде число 4 у десятковому форматі.

  • | (логическое сложение)

    Ця операція також проводиться за двійковими розрядами, але тепер повертається одиниця, якщо хоча б одного числа в даному розряді є одиниця (операція "логічне АБО"). Наприклад:

    
int a1 = 2; //010
int b1 = 5;//101
System.out.println(a1|b1); // результат 7 - 111
int a2 = 4; //100
int b2 = 5;//101
System.out.println(a2 | b2); // результат 5 - 101

  • ^ (логическое исключающїї ИЛИ)

    Також цю операцію називають XOR, нерідко її застосовують простого шифрування:

    
int number = 45; // 1001 Значение, которое треба зашифровать - в двоичной форме 101101
int key = 102; //Ключ шифрования - в двоичной системе 1100110
int encrypt = number ^ key; //Результатом буде число 1001011 або 75
System.out.println("Зашифрованное число: " +encrypt);

int decrypt = encrypt ^ key; // Результатом буде исходное число 45
System.out.println("Расшифрованное число: " + decrypt);

    Тут також виконуються порозрядні операції. Якщо значення поточного розряду в обох чисел різні, то повертається 1, інакше повертається 0. Наприклад, результатом виразу 9^5 буде число 12. Щоб розшифрувати число, ми застосовуємо зворотну операцію до результату.

  • ~ (логическое отрицание або инверсия)

    Порозрядна операція, яка інвертує всі розряди числа: якщо значення розряду дорівнює 1, то воно стає рівним нулю, і навпаки.

    
byte a = 12;                 // 0000 1100     
System.out.println(~a);		//  1111 0011   або -13

Операции сдвига

Операции сдвига також производятся над разрядами чисел. Сдвиг может происходить вправо и влево.

  • a<<b - сдвигает число a влево на b разрядов. Наприклад, выражение 4<<1 сдвигает число 4 (которое в двоичном представлении 100) на один разряд влево, в результате отримуєся число 1000 або число 8 в десятичном представлении.

  • a>>b - смещает число a вправо на b разрядов. Наприклад, 16>>1 сдвигает число 16 (которое в двоичной системе 10000) на один разряд вправо, то есть в итоге отримуєся 1000 або число 8 в десятичном представлении.

  • a>>>b - в отличие от предыдущих типов сдвигов данная операция представляє беззнаковый сдвиг - сдвигает число a вправо на b разрядов. Наприклад, выражение -8>>>2 буде равно 1073741822.

Таким чином, якщо исходное число, которое треба сдвинуть в ту або другую сторону, делится на два, то фактически отримуєся умножение або деление на два. Тому подобную операцию можна використовувати вместо непосредственного умножения або деления на два, оскільки операция сдвига на аппаратном уровне менїї дорогостоящая операция в отличие от операции деления або умножения.

Таким чином, якщо исходное число, которое треба сдвинуть в ту або другую строну, делится на два, то фактически отримуєся умножение або деление на два. Тому подобную операцию можна використовувати вместо непосредственного умножения або деления на два. Например:


class Program{

    public static void main(String[] args) {  

        int a = 16; // в двоичной форме 10000
        int b = 2; // в двоичной форме 00010
        int c = a << b; // Сдвиг числа 10000 влево на 2 разряда, равно 1000000 або 64 в десятичной системе
        
        System.out.println("с: " + c);    // 64
        
        int d = a >> b; // Сдвиг числа 10000 вправо на 2 разряда, равно 100 або 4 в десятичной системе
        System.out.println("d: " + d);     // 4
    }
}

При цем числа, которые участвую в операциях, необязательно повинні быть кратны 2:


class Program{

    public static void main(String[] args) {  

        int a = 22; // в двоичной форме 10110
        int b = 2; // в двоичной форме
        int c = a << b; // Сдвиг числа 10110 влево на 2 разряда, равно 1011000 або 88 в десятичной системе

        System.out.println("с: " + c);    // 88

        int d = a >> b; // Сдвиг числа 10110 вправо на 2 разряда, равно 101 або 5 в десятичной системе
        System.out.println("d: " + d);     // 5
    }
}

Пример практического применения операций

Многие недооценивают поразрядные операции, не понимают, для чего они нужны. Тем не менїї они можуть помочь в решении ряда задач. Прежде всего они дозволяють нам манипулировать данными на уровне отдельных битов. Один из примеров. У нас есть три числа, которые находятся в диапазоне от 0 до 3:


int value1 = 3;  // 0b0000_0011
int value2 = 2;  // 0b0000_0010
int value3 = 1;  // 0b0000_0001

Мы знаем, что значения цих чисел не будуть больше 3, и нам потрібно эти дані максимально сжать. Мы можемо три числа сохранить в одно число. И в цем нам поможуть поразрядные операции.


class Program{

    public static void main(String[] args) {  

        int value1 = 3;  // 0b0000_0011
        int value2 = 2;  // 0b0000_0010
        int value3 = 1;  // 0b0000_0001
        int result = 0b0000_0000;
        // сохраняем в result значения из value1
        result = result | value1; // 0b0000_0011
        // сдвигаем разряды в result на 2 разряда влево
        result = result << 2;   // 0b0000_1100
        // сохраняем в result значения из value2
        result = result | value2;  // 0b0000_1110
        // сдвигаем разряды в result на 2 разряда влево
        result = result << 2;   // 0b0011_1000
        // сохраняем в result значения из value3
        result = result | value3;  // 0b0011_1001

        System.out.println(result);  // 57
    }
}

Разберем цей код. Сначала определяем все сохраняемые числа value1, value2, value3. Для хранения результата определена переменная result, которая по умолчанию равна 0. Для большей наглядности ей присвоено значение в бинарном формате:

int result = 0b0000_0000;

Сохраняем первое число в result:


result = result | value1; // 0b0000_0011

Здесь мы имїїм дело с логической операцией поразрядного сложения - якщо один из соответствующих разрядов равен 1, то результирующий разряд тоже буде равен 1. То есть фактически


0b0000_0000
+
0b0000_0011
=
0b0000_0011

Итак, первое число сохранили в result. Мы будем сохранять числа по порядку. То есть сначала в result буде идти первое число, затем второе и далїї третье. Тому сдвигаем число result на два разряда влево (наши числа занимают в памяти не болїї двух разрядов):


result = result << 2;   // 0b0000_1100

То есть фактически


0b0000_0011 << 2 =
0b0000_1100

Далїї повторяем логическую операцию сложения, сохраняем второе число:

result = result | value2;  // 0b0000_1110

что эквивалентно


0b0000_1100
+
0b0000_0010
=
0b0000_1110

Далїї повторяем сдвиг на два разряда влево и сохраняем третье число. В итоге мы получим в двоичном представлении число 0b0011_1001. В десятично системе це число равно 57. Но це не имїїт значения, потому что нам важны конкретные биты числа. Варто відзначити, что мы сохранили в одно число три числа, и в змінної result ще есть сводобное место. Причем в реальности не важно, сколько именно битов треба сохранить. В данном случае для примера сохраняем лишь два бита.

Для восстановления данных прибегнем к обратному порядку:


class Program{

    public static void main(String[] args) {  

        int result = 0b0011_1001;
        // обратное получение данных
        int newValue3 = result & 0b000_0011;
        // сдвигаем дані на 2 разряда вправо
        result = result >> 2;
        int newValue2 = result & 0b000_0011;
        // сдвигаем дані на 2 разряда вправо
        result = result >> 2;
        int newValue1 = result & 0b000_0011;
        System.out.println(newValue1);    // 3
        System.out.println(newValue2);   // 2
        System.out.println(newValue3);   // 1
    }
}

Получаем числа в порядке, обратном тому, в котором они буви сохранены. Поскольку мы знаем, что каждое сохраненное число занимает лишь два разряда, то по сути нам треба отримати лишь последние два бита. Для цього применяем битовую маску 0b000_0011 и операцию логического умножения, которая повертає 1, якщо каждый из двух соответствующих разрядов равен 1. То есть операция 

int newValue3 = result & 0b000_0011;

эквивалентна


0b0011_1001
*
0b0000_0011
=
0b0000_0001

Таким чином, последнїї число равно 0b0000_0001 або 1 в десятичной системе

Варто відзначити, что якщо мы точно знаем структуру данных, то мы легко можемо составить битовую маску, чтобы отримати потрібно число:


result = 0b0011_1001;
int recreatedValue1 = (result & 0b0011_0000) >> 4;
System.out.println(recreatedValue1);

Здесь получаем первое число, которое, як мы знаем, занимает в числе биты 4 и 5. Для цього применяем умножение на битовую маску 0b0011_0000. И затем сдвигаем число на 4 разряда вправо.


0b0011_1001
*
0b0011_0000
=
0b0011_0000
>> 4
=
0b0000_0011

Аналогично, якщо мы точно знаем структуру, по которой сохраняются дані, то мы могли бы сохранить дані відразу в потрібное место в числе result:


class Program{

    public static void main(String[] args) {  

        int value1 = 3;  // 0b0000_0011
        int value2 = 2;  // 0b0000_0010
        int value3 = 1;  // 0b0000_0001
        int result = 0b0000_0000;
        // сохраняем в result значения из value1
        result = result | (value1 << 4);
        // сохраняем в result значения из value2
        result = result | (value2 << 2);
        // сохраняем в result значения из value3
        result = result | value3;  // 0b0011_1001

        System.out.println(result);  // 57
    }
}
