Метод reduce

Останнє оновлення: 06.11.2020

Метод reduce виконує терминальные операції сведения, возвращая некоторое значення - результат операції. Он имїїт следующие формы:


Optional<T> reduce(BinaryOperator<T> accumulator)
T reduce(T identity, BinaryOperator<T> accumulator)
U reduce(U identity, BiFunction<U,? super T,U> accumulator, BinaryOperator<U> combiner)

Первая форма повертає результат в виде об'єкта Optional<T>. Наприклад, вычислим произведение набора чисел:


import java.util.stream.Stream;
import java.util.Optional;

public class Program {

    public static void main(String[] args) {

        Stream<Integer> numbersStream = Stream.of(1,2,3,4,5,6);
        Optional<Integer> result = numbersStream.reduce((x,y)->x*y);
        System.out.println(result.get()); // 720
    } 
}

Об'єкт BinaryOperator<T> представляє функцію, которая приймає два елемента и виконує над ними деяку операцію, возвращая результат. При цем метод reduce сохраняет результат и затем опять же застосовує к цему результату и следующему елементу в наборе бинарную операцію. Фактически у даному випадку мы получим результат, який буде равен: n1 op n2 op n3 op n4 op n5 op n6, где op - це операція (у даному випадку умножения), а n1, n2, ... - елементы из потіка.

Затем за допомогою методу get() мы можемо отримати собственно результат вычислений: result.get()

Или ще один пример - об'єднание слов в предложение:


Stream<String> wordsStream = Stream.of("мама", "мыла", "раму");
Optional<String> sentence = wordsStream.reduce((x,y)->x + " " + y);
System.out.println(sentence.get());

Вторая версия методу reduce() приймає два параметра: 


T reduce(T identity, BinaryOperator<T> accumulator)

Первый параметр - T identity - елемент, який надає начальное значення для функції из второго параметра, а також надає значення по умолчанию, якщо потік не имїїт елементов.

Второй параметр - BinaryOperator<T> accumulator, як и первая форма методу reduce, представляє ассоциативную функцію, которая запускуєся для каждого елемента у потоці и приймає два параметра. Первый параметр представляяет промежуточный результат функції, а второй параметр - следующий елемент у потоці. Фактически код цього методу буде равноценен следующей записи:


T result = identity;
for (T element : this stream)
	result = accumulator.apply(result, element)
return result;

То есть при первом вызове функція accumulator в качестве первого параметра приймає значення identity, а в качестве второго параметра - первый елемент потіка. При втором вызове первым параметром служит результат первого вызова функції accumulator, а вторым параметром - второй елемент у потоці и так далїї. Наприклад:


Stream<Integer> numberStream = Stream.of(-4, 3, -2, 1);
int identity = 1;
int result = numberStream.reduce(identity, (x,y)->x * y);
System.out.println(result);  // 24

Фактически здесь виконується следующая цепь операцій: identity op n1 op n2 op n3 op n4...

В предыдущих примерах тип повертаємоых об'єктов совпадал с типом елементов, которые входят в потік. Однак це не всегда удобно. Підзможна, мы захочемо возвратить результат, тип которого отличается от типу об'єктов потіка. Наприклад, пусть у нас есть следующий клас Phone, представляющий телефон:


class Phone{
    
    private String name;
    private int price;
    
    public Phone(String name, int price){
        this.name=name;
        this.price=price;
    }
    
    public String getName() {
        return name;
    }
    
    public int getPrice() {
        return price;
    }
}

И мы хочемо найти сумму цен тех телефонов, у которых цена меньше визначенийного значення. Для цього використовуємо третью версию методу reduce:


Stream<Phone> phoneStream = Stream.of(new Phone("iPhone 6 S", 54000), 
            new Phone("Lumia 950", 45000),
            new Phone("Samsung Galaxy S 6", 40000),
            new Phone("LG G 4", 32000));

int sum = phoneStream.reduce(0, 
            (x,y)-> {
                    if(y.getPrice()<50000)
                        return x + y.getPrice();
                    else 
                        return x + 0;
            }, 
            (x, y)->x+y);
        
System.out.println(sum); // 117000

Опять же здесь в качестве первого параметра идет значення по умолчанию - 0. Второй параметр производит бинарную операцію, которая отримує промежуточное значення - суммарную цену текущего и предыдущего телефонов. Третий параметр представляє бинарную операцію, которая суммирует все промежуточные вычисления.
