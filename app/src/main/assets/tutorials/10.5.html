Метод reduce

Останнє оновлення: 06.11.2020

Метод reduce выполняет терминальные операции сведения, возвращая некоторое значение - результат операции. Он имїїт следующие формы:


Optional<T> reduce(BinaryOperator<T> accumulator)
T reduce(T identity, BinaryOperator<T> accumulator)
U reduce(U identity, BiFunction<U,? super T,U> accumulator, BinaryOperator<U> combiner)

Первая форма повертає результат в виде об'єкта Optional<T>. Наприклад, вычислим произведение набора чисел:


import java.util.stream.Stream;
import java.util.Optional;

public class Program {

    public static void main(String[] args) {

        Stream<Integer> numbersStream = Stream.of(1,2,3,4,5,6);
        Optional<Integer> result = numbersStream.reduce((x,y)->x*y);
        System.out.println(result.get()); // 720
    } 
}

Об'єкт BinaryOperator<T> представляє функцию, которая приймає два элемента и выполняет над ними деяку операцию, возвращая результат. При цем метод reduce сохраняет результат и затем опять же применяет к цему результату и следующему элементу в наборе бинарную операцию. Фактически в данном случае мы получим результат, який буде равен: n1 op n2 op n3 op n4 op n5 op n6, где op - це операция (в данном случае умножения), а n1, n2, ... - элементы из потока.

Затем за допомогою метода get() мы можемо отримати собственно результат вычислений: result.get()

Или ще один пример - об'єднание слов в предложение:


Stream<String> wordsStream = Stream.of("мама", "мыла", "раму");
Optional<String> sentence = wordsStream.reduce((x,y)->x + " " + y);
System.out.println(sentence.get());

Вторая версия метода reduce() приймає два параметра: 


T reduce(T identity, BinaryOperator<T> accumulator)

Первый параметр - T identity - элемент, який надає начальное значение для функції из второго параметра, а також надає значение по умолчанию, якщо поток не имїїт элементов.

Второй параметр - BinaryOperator<T> accumulator, як и первая форма метода reduce, представляє ассоциативную функцию, которая запускаєся для каждого элемента в потоке и приймає два параметра. Первый параметр представляяет промежуточный результат функції, а второй параметр - следующий элемент в потоке. Фактически код цього метода буде равноценен следующей записи:


T result = identity;
for (T element : this stream)
	result = accumulator.apply(result, element)
return result;

То есть при первом вызове функция accumulator в качестве первого параметра приймає значение identity, а в качестве второго параметра - первый элемент потока. При втором вызове первым параметром служит результат первого вызова функції accumulator, а вторым параметром - второй элемент в потоке и так далїї. Например:


Stream<Integer> numberStream = Stream.of(-4, 3, -2, 1);
int identity = 1;
int result = numberStream.reduce(identity, (x,y)->x * y);
System.out.println(result);  // 24

Фактически здесь виконується следующая цепь операций: identity op n1 op n2 op n3 op n4...

В предыдущих примерах тип возвращаемых об'єктов совпадал с типом элементов, которые входят в поток. Однак це не всегда удобно. Підзможна, мы захочемо возвратить результат, тип которого отличается от типу об'єктов потока. Наприклад, пусть у нас есть следующий класс Phone, представляющий телефон:


class Phone{
    
    private String name;
    private int price;
    
    public Phone(String name, int price){
        this.name=name;
        this.price=price;
    }
    
    public String getName() {
        return name;
    }
    
    public int getPrice() {
        return price;
    }
}

И мы хочемо найти сумму цен тех телефонов, у которых цена меньше определенного значения. Для цього используем третью версию метода reduce:


Stream<Phone> phoneStream = Stream.of(new Phone("iPhone 6 S", 54000), 
            new Phone("Lumia 950", 45000),
            new Phone("Samsung Galaxy S 6", 40000),
            new Phone("LG G 4", 32000));

int sum = phoneStream.reduce(0, 
            (x,y)-> {
                    if(y.getPrice()<50000)
                        return x + y.getPrice();
                    else 
                        return x + 0;
            }, 
            (x, y)->x+y);
        
System.out.println(sum); // 117000

Опять же здесь в качестве первого параметра идет значение по умолчанию - 0. Второй параметр производит бинарную операцию, которая отримує промежуточное значение - суммарную цену текущего и предыдущего телефонов. Третий параметр представляє бинарную операцию, которая суммирует все промежуточные вычисления.
