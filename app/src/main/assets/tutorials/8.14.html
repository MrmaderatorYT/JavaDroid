Executor – виконавець завдань

Останнє оновлення: 20.10.2025

Executor або исполнитель представляє об'єкт, який выполняет полученные задачи. Исполнитель обычно використовується вместо явного создания потоков и дозволяє отделить отправку задачи от механизма її виконання.

Executor представляє функциональный інтерфейс с одним методом execute():


void execute(Runnable command)

Предполагается, что данный метод приймає задачу Runnable и выполняет її в неякий момент в будущем. Простейший пример:


import java.util.concurrent.*;

class TaskExecutor implements Executor{
    
    public void execute(Runnable task){
        task.run();
    }
}

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        var executor = new TaskExecutor();
        executor.execute(() -> {
                    
            System.out.println("Executing task in " + Thread.currentThread());
        });
        System.out.println("Main thread finished...");
    }
}

В данном случае определяем класс TaskExecutor, який реализует інтерфейс Executor и в котором метод execute() просто запускає переданную задачу на выполнение.

В методе main создаем об'єкт цього класса и вызываем его метод execute(), передавая в него простешую задачу Runnable. Таким чином виконується переданная задача.

В таком запуске задачи, конечно, нет большого смысла, оскільки мы просто могли бы запустить ту же задачу Runnable в том же методе main() без реализации інтерфейса Executor. Но тем не менїї інтерфейс Executor не требует строго асинхронного виконання и может выполняться синхронно, як в примере выше, где задача выполнялась непосредственно в потоке метода main()

Однак чаще всего задачи выполняются в потоке, отличном от потока вызывающего об'єкта. Наприклад, изменим код исполнителя, чтобы он создавал новый поток для каждой задачи:


class TaskExecutor implements Executor{
    
    public void execute(Runnable task){
        new Thread(task).start();
    }
}

При запуске программы мы увидим, что теперь задача виконується в отдельном потоке:


Main thread started...
Main thread finished...
Executing task in Thread[#34,Thread-0,5,main]

Нередко реализации Executor використовуються для координации виконання задач. И в большинстве случаев нам не потребуется создавать свои собственные реализации інтерфейса Executor, оскільки в Java есть встроенные реализации цього інтерфейса, наприклад, класса ThreadPoolExecutor, який для управления выполнением задач использует пул потоков, позволяя оптимизировать выполнение программы.

ExecutorService

Интерфейс ExecutorService расширяет інтерфейс Executor, предоставляя методы управления завершением работы, а також методы создания об'єкта Future для отслеживания хода виконання одной або нескольких асинхронных задач. Цет інтерфейс упрощает применение исполнителей Executor для управления асинхронными задачами. Выше упомянутый класс ThreadPoolExecutor як раз реализует данный інтерфейс ExecutorService (а через него и Executor). Рассмотрим, як використовувати ExecutorService.

Створення исполнителя

Для создания исполнителей застосовується один из методів класса Executors. Подобные методы в основном создают пулы потоков. Пул потоков представляє технику для оптимизации работы с платформенными потоками. Поскольку створення платформенных потоков довольно затратно и требует больших ресурсов, то вместо создания новых потоков болїї производительным решением є использование вже готовых потоков. И пул потоков представляє набор свободных потоков, готовых к выполнению задач. Так, якщо треба выполнить в потоке якую-нибудь задачу Runnable або Callable, то программа берет из пула один из потоков и использует его для виконання задачи. После виконання задачи поток не завершуєся, а повертаєся обратно в пул для обслуживания следующего запроса.

Некоторые из них:

  • Executors.newCachedThreadPool()

    Створює пул потоков, який створює новые потоки по мере необхідности, и повторно использует ранїї создані потоки, когда они становятся доступны. Бездействующие потоки сохраняются в течение 60 секунд

  • Executors.newFixedThreadPool()

    Створює пул из фиксированного количества потоков; бездействующие потоки сохраняются неограниченное час. Если число отправленных задач превышает число бездействующих потоков, необработанные задачи помещаются в очередь. Они запускаются после завершения других задач.

  • Executors.newSingleThreadExecutor()

    Створює об'єкт Executor (фактически пул из одного потока) с одним рабочим потоком, який последовательно выполняет отправленные задачи.

  • Executors.newVirtualThreadPerTaskExecutor()

    Створює об'єкт Executor, який запускає каждую задачу в новом виртуальном потоке.

Все эти методы повертають об'єкты классов, которые реализуют інтерфейс ExecutorService, який в свою очередь расширяет інтерфейс Executor.

Отправка задачи исполнителю

Для отправки задачи Runnable або Callable в ExecutorService застосовується один из следующих методів:


Future<T> submit(Callable<T> task)
Future<?> submit(Runnable task)
Future<T> submit(Runnable task, T result)

При вызове метода submit() исполнитель - об'єкт Executor планирует выполнение отправленной задачи.

В качестве результата методы повертають об'єкт Future, який можна використовувати для получения результата або отмены задачи.

Вторая версия метода submit повертає об'єкт Future<?>. Его метод get() блокує выполнение до завершения задачи, а затем повертає значение null.

Третья версия метода submit повертає об'єкт Future, у которого метод get() после завершения задачи повертає об'єкт, переданный через второй параметр.

Завершение исполнителя

После завершения работы исполнителя для освобождения ресурсов вызывается метод close():

void close()

Закрытый исполнитель не приймає новых задач. Метод close() блокує выполнение до завершения всех отправленных задач. (Если поток, вызывающий close, переривається, отправленные задачи отменяются.) Для болїї удобного неявного вызова метода close() можна використовувати конструкцию try-with-resources (try с ресурсами):


try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {

    ...............

    Future<V> f = submit(myCallable);

    .......................
} // здесь вызывается метод executor.close()

В итоге общий план применения об'єкта Executor для виконання задачи:

  1. С помощью одного из статических методів класса Executors (newCachedThreadPool, newFixedThreadPool и т.д) створюєся об'єкт ExectorService

  2. У об'єкта ExectorService для отправки задач Callable або Runnable вызывается метод submit()

  3. Метод submit() повертає об'єкт Future. Цет об'єкт можна сохранить возвращаемые, чтобы иметь возможнасть запрашивать состояния задач, получать результат задачи або отменять її.

  4. В конце, якщо больше не планируется отправлять задачи на выполнение, у об'єкта ExectorService вызывается метод close(), який завершує работу исполнителя.

Рассмотрим простейший пример:


import java.util.concurrent.*;

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {

            // запускаем 5 задач в виртуальных потоках
            for (int i = 0; i < 5; i++) {

                int taskNumber = i;

                executor.submit(() -> {
                    try{
                        Thread.sleep(1000);
                    }
                    catch(Exception ex) { 
                        System.out.println(ex.getMessage());
                    }
                    System.out.println("Executing task " + taskNumber + " in " + Thread.currentThread());
                });
            }
        } // executor.close() вызывается автоматически

        System.out.println("Main thread finished...");
    }
}

Здесь ExecutorService створює новый виртуальный поток для каждой задачи Runnable:

try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {

Через метод submit() передаем на выполнение задачу в виде лямбда-выражения:

executor.submit(() -> {
