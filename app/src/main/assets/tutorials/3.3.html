Модифікатори доступу

Останнє оновлення: 18.09.2025

Все компоненты класса в мові Java - поля, методы и конструкторы имеют модифікатори доступу. Модифікатори доступу управляют видимостью классов, методів и переменных, тем самым определяя, як к компонентам класса можна отримати доступ из других частей программы. Они управляют тем, откуда можна отримати доступ к конкретному классу, методу або змінної.

У Java використовуються такі модифікатори доступу:

  • public: публичный, общедоступный класс або компонент класса. Поля и методы, оголошенийные с модификатором public, видны другим классам из текущего пакета и из внешних пакетів, вообще в любом месте программы.

    Варто відзначити, что в исходном файле может быть лише один класс с модификатором public, но количество классов с другими модификаторами (или без них) может быть любым. И якщо в файле есть класс с модификатором public, то имя файла повинно совпадать с именем цього класса.

  • private: закрытый класс або компонент класса, противоположность модификатору public. Закрытый класс або компонент класса доступен лише из кода в том же классе.

  • protected: такой класс або компонент класса доступен из любого места в текущем классе або пакеті або в производных классах, даже якщо они находятся в других пакетах

  • Модифікатор за промовчанням. Отсутствие модификатора у поля або метода класса предполагает применение к нему модификатора по умолчанию. Такие поля або методы видны всем классам в текущем пакеті.

Подивимося з прикладу і створимо наступний клас State:

Так як клас State не має явного модифікатора, тому він буде доступний з будь-якого місця цього пакета, однак не буде доступний з інших пакетів.


class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакета
    
    private String privateVar = "private";  // доступно лише из текущего класса

    protected String protectedVar = "protected";  // доступно из текущего пакета и производных классов

    public String publicVar = "public";  // доступно в любом месте программы

    // доступен в любом месте текущего пакета
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класса
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакета и производных классов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте программы
    public void printPublic() { System.out.println(publicVar);} 
}

Клас State має чотири поля для кожного рівня доступу. А також визначено 4 методи з різними модифікаторами, які виводять значення відповідних змінних консоль. Оскільки всі модифікатори дозволяють використовувати компоненти класу всередині даного класу, то і всі змінні класи, у тому числі закриті, у нас доступні всім його методам, оскільки всі перебувають у контексті класу State.

Теперь посмотрим, як мы сможемо використовувати переменные класса State в другом классе, який расположен Таким чином, у класі Program ми змогли тільки звернутися до змінних defaultVar, protectedVar і publicVar, оскільки їх модифікатори дозволяють використовувати у цьому контексті.:


class Program {

    public static void main(String[] args) {
        
        State state = new State();
 
        // переменная defaultVar с модификатором по умолчанию доступна из любого места текущего пакета
        // поцему можна отримати або изменить її значення
        System.out.println(state.defaultVar); //Ошибка, отримати доступ нельзя
 
        // обратиться к змінної privateVar у нас не получится,
        // оскільки она имїїт модификатор private и класс Program її не видит
        System.out.println(state.privateVar); // Ошибка, отримати доступ нельзя
 
        // переменная protectedVar так же доступна из любого места текущего пакета 
        // (а також в производных классах других пакетів)
        System.out.println(state.protectedVar);
 
        // переменная publicVar общедоступна везде
        System.out.println(state.publicVar);
    } 
}

class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакета
    
    private String privateVar = "private";  // доступно лише из текущего класса

    protected String protectedVar = "protected";  // доступно из текущего пакета и производных классов

    public String publicVar = "public";  // доступно в любом месте программы

    // доступен в любом месте текущего пакета
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класса
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакета и производных классов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте программы
    public void printPublic() { System.out.println(publicVar);} 
}

Тут нам виявилися доступні лише три методи: printDefault, printProtected, printPublic, які мають відповідно модифікатор за замовчуванням та модифікатори protected та public.


class Program {

        state.printPublic();      // норм
    } 
}

Модифікатори пакетів

Тепер можна побачити, що буде, якщо класи State і Program розташовані в різних пакетах. Припустимо, ми маємо проект із наступною структурою:

  • Файл Program.java

  • Папка types

    • Файл State.java

Пусть файл State.java буде містятьь определение аналогичного класса State:


package types;

public class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакета
    
    private String privateVar = "private";  // доступно лише из текущего класса

    protected String protectedVar = "protected";  // доступно из текущего пакета и производных классов

    public String publicVar = "public";  // доступно в любом месте программы

    // доступен в любом месте текущего пакета
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класса
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакета и производных классов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте программы
    public void printPublic() { System.out.println(publicVar);} 
}

Тут лише дві відмінність проти попереднім визначенням класу State:

  • Клас State належить пакету "types" (тому і розташовується у відповідному каталозі)

  • Класс State определен с модификатором public, благодаря чему он виден из других пакетів

Застосуємо цей клас у класі Program:


import types.State;

class Program {

    public static void main(String[] args) {
        
        State state = new State();
 
        state.printDefault(); // Ошибка

        state.printPrivate(); // Ошибка

        state.printProtected(); // Ошибка

        state.printPublic();  // норм
    } 
}

И здесь мы видим, что мы можемо обратиться лише к тем полям класса, которые имеют модификатор public. То же самое касается методів, пример с которыми я не буду приводить, оскільки все аналогично.
