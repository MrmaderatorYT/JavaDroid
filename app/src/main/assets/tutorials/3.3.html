Модифікатори доступу

Останнє оновлення: 18.09.2025

Все компоненты класа в мові Java - поля, методи и конструктори имеют модифікатори доступу. Модифікатори доступу управляют видимостью класов, методів и змінних, тем самым определяя, як к компонентам класа можна отримати доступ из других частей програми. Они управляют тем, откуда можна отримати доступ к конкретному класу, методу або змінної.

У Java використовуються такі модифікатори доступу:

  • public: публичный, общедоступный клас або компонент класа. Поля и методи, оголошенийные с модификатором public, видны другим класам из текущего пакету и из внешних пакетів, вообще в любом месте програми.

    Варто відзначити, что в исходном файле может быть лише один клас с модификатором public, но кількість класов с другими модификаторами (или без них) может быть любым. И якщо в файле есть клас с модификатором public, то ім'я файлу повинно совпадать с именем цього класа.

  • private: закрытый клас або компонент класа, противоположность модификатору public. Закрытый клас або компонент класа доступен лише из коду в том же класе.

  • protected: такой клас або компонент класа доступен из любого места в текущем класе або пакеті або в производных класах, даже якщо они находятся в других пакетух

  • Модифікатор за промовчанням. Отсутствие модификатора у поля або методу класа предполагает застосування к нему модификатора по умолчанию. Такие поля або методи видны всем класам в текущем пакеті.

Підивимося з прикладу і створимо наступний клас State:

Так як клас State не має явного модифікатора, тому він буде доступний з будь-якого місця цього пакету, однак не буде доступний з інших пакетів.


class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакету
    
    private String privateVar = "private";  // доступно лише из текущего класа

    protected String protectedVar = "protected";  // доступно из текущего пакету и производных класов

    public String publicVar = "public";  // доступно в любом месте програми

    // доступен в любом месте текущего пакету
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класа
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакету и производных класов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте програми
    public void printPublic() { System.out.println(publicVar);} 
}

Клас State має чотири поля для кожного рівня доступу. А також визначено 4 методи з різними модифікаторами, які виводять значення відповідних змінних консоль. Оскільки всі модифікатори дозволяють використовувати компоненти класу всередині даного класу, то і всі змінні класи, у тому числі закриті, у нас доступні всім його методум, оскільки всі перебувають у контексті класу State.

Теперь посмотрим, як мы сможемо використовувати змінні класа State в другом класе, який расположен Таким чином, у класі Program ми змогли тільки звернутися до змінних defaultVar, protectedVar і publicVar, оскільки їх модифікатори дозволяють використовувати у цьому контексті.:


class Program {

    public static void main(String[] args) {
        
        State state = new State();
 
        // змінна defaultVar с модификатором по умолчанию доступна из любого места текущего пакету
        // поцему можна отримати або змінити її значення
        System.out.println(state.defaultVar); //Ошибка, отримати доступ нельзя
 
        // обратиться к змінної privateVar у нас не получится,
        // оскільки она имїїт модификатор private и клас Program її не видит
        System.out.println(state.privateVar); // Ошибка, отримати доступ нельзя
 
        // змінна protectedVar так же доступна из любого места текущего пакету 
        // (а також в производных класах других пакетів)
        System.out.println(state.protectedVar);
 
        // змінна publicVar общедоступна везде
        System.out.println(state.publicVar);
    } 
}

class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакету
    
    private String privateVar = "private";  // доступно лише из текущего класа

    protected String protectedVar = "protected";  // доступно из текущего пакету и производных класов

    public String publicVar = "public";  // доступно в любом месте програми

    // доступен в любом месте текущего пакету
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класа
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакету и производных класов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте програми
    public void printPublic() { System.out.println(publicVar);} 
}

Тут нам виявилися доступні лише три методи: printDefault, printProtected, printPublic, які мають відповідно модифікатор за замовчуванням та модифікатори protected та public.


class Program {

        state.printPublic();      // норм
    }

Модифікатори пакетів

Тепер можна побачити, що буде, якщо класи State і Program розташовані в різних пакетух. Припустимо, ми маємо проєкт із наступною структурою:

  • Файл Program.java

  • Папка types

    • Файл State.java

Пусть файл State.java буде містятьь визначенийие аналогичного класа State:


package types;

public class State{
    
    String defaultVar ="default";  // доступно в любом месте текущего пакету
    
    private String privateVar = "private";  // доступно лише из текущего класа

    protected String protectedVar = "protected";  // доступно из текущего пакету и производных класов

    public String publicVar = "public";  // доступно в любом месте програми

    // доступен в любом месте текущего пакету
    void printDefault() { System.out.println(defaultVar); }
 
    // доступен лише из текущего класа
    private void printPrivate() { System.out.println(privateVar); }
 
    // доступен из текущего пакету и производных класов
    protected void printProtected() { System.out.println(protectedVar); }
 
    // доступен в любом месте програми
    public void printPublic() { System.out.println(publicVar);} 
}

Тут лише дві відмінність проти попереднім визначенням класу State:

  • Клас State належить пакету "types" (тому і розташовується у відповідному каталозі)

  • Клас State визначений с модификатором public, благодаря чему он виден из других пакетів

Застосуємо цей клас у класі Program:


import types.State;

class Program {

    public static void main(String[] args) {
        
        State state = new State();
 
        state.printDefault(); // Ошибка

        state.printPrivate(); // Ошибка

        state.printProtected(); // Ошибка

        state.printPublic();  // норм
    } 
}

И здесь мы видим, что мы можемо обратиться лише к тем полям класа, которые имеют модификатор public. То же самое касается методів, пример с которыми я не буду приводить, оскільки все аналогично.
