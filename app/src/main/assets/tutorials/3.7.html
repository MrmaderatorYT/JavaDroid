Інтерфейси

Останнє оновлення: 30.09.2025

Механізм успадкування двже зручний, але має свої обмеження. Зокрема ми можемоо успадковувати лише від одного класу, на відміну, наприклад, від мови С++, де є множинне успадкування.

У мові Java подібну проблему частково дозволяють вирішити інтерфейси. Інтерфейси визначають деякий функціонал, який не має конкретної реалізації, який потім реалізують класи, які використовують ці інтерфейси. І один клас може застосувати багато інтерфейсів.

Но суть інтерфейса не лише и не слише в цем. Ключевая мысли інтерфейса соварто в том, чтобы определить некий набор требований, которым повинен соответствовать класс. Интерфейс определяет, що повинен делать класс, а класс вже определяет, як он це робить.

Визначення інтерфейсу

Щоб определить інтерфейс, використовується ключевое слово interface. Например:


interface Movable{ }

Цей інтерфейс називається Movable і далі представлятиме якийсь транспортний засіб, щось, що може перещатися. Відповідно до умов назви інтерфейсів в Java починаються з великої літери і зазвичай закінчуються на суфікс "-able", хоча в принципі це необов'язково.

Використання інтерфейсу

Как буво сказано выше, інтерфейс определяет набор требований к классу, а класс применяет эти требования. Щоб указать, что класс застосовує або реалізує інтерфейс, после названия класса указывается ключевое слово implements, после которого идет название інтерфейса:


interface Movable{ }
class Car implements Movable{ }

Тобто в даному випадку ми можемоо сказати, що Car застосовує або реалізує інтерфейс Movable. Грубо кажучи, ми говоримо, що машина (Car) - це транспортний засіб, щось, що функціонал для переміщення.

Змінні інтерфейси

Інтерфейс фактично представляє новий тип. І ми можемоо визначати змінні інтерфейси:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj;    // определяем інтерфейс
    }
}
// определение інтерфейса
interface Movable{ }

Однак все же інтерфейсы - це не классы. В частности, мы не можемо використовувати оператор new для создания экземпляра інтерфейса:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj = new Movable();        // ! Ошибка, так нельзя сделать
    }
}
interface Movable{ }

Але змінної інтерфейсу можна привласнити посилання на об'єкт класу, який застосовує цей інтерфейс. Наприклад:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj = new Car();
        System.out.println(obj);
    }
}

interface Movable{ }
class Car implements Movable{ }

Оскільки клас Car застосовує інтерфейс Movable, змінної цього інтерфейсу ми можемоо привласнити об'єкт Car.

Визначення компонентів інтерфейсу

Выше определенный інтерфейс Movable був пустой и ничего не определял. Но інтерфейс может визначати ряд компонентов:

  • Методи без реализации

  • Методи по умолчанию (с реализацией)

  • Приватные методы

  • Статические методы (як приватные, так и публичные)

  • Константы

Определение методів в інтерфейсе

Как говорилось выше, інтерфейс определяет ряд требований або контракт, которым повинен соответствовать класс. И эти требования визначаються в інтерфейсе в виде методів. Методи інтерфейса можуть иметь, а можуть и не иметь реализации. Если неякий класс применяет определенный інтерфейс, то цей класс повинен реализовать все методы применяемого інтерфейса (исключение касается методів, для которых в інтерфейсе определена реализация по умолчанию).

Наприклад, изменим інтерфейс Movable, добавив в него метод:


interface Movable{

    void move();
}

В данном случае інтерфейс Movable представляє функционал транспортного средства и определяет один метод без реализации - метод move(), який условно призначений для передвижения транспортного средства. Методи без реализации похожи на абстрактные методы абстрактных классов. Все методы інтерфейса не имеют модификаторов доступа, но фактически по умолчанию доступ public, оскільки цель інтерфейса - определение функционала для реализации его классом. Тому весь функционал повинен быть открыт для реализации.

Таким чином, у нас еть інтерфейс Movable, которое представляє непонятно якое транспортное средство, и есть функция move, которая призначенийа для перемщения транспортного средства, но як именно це перемщение осуществляется - неизвестно.

Однак класс, який буде застосовувати інтерфейс, повинен реализовать все его методы (которые не имеют реализации). Теперь реализуем інтерфейс Movable:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине
    }
}

interface Movable{

    void move();
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}

В данном случае класс Car применяет інтерфейс Movable и поцему повинен реализовать его метод move(). Если класс не реализует якие-то методы інтерфейса, то такой класс повинен быть определен як абстрактный, но его неабстрактные классы-наследники затем все равно повинні будуть реализовать эти методы.

Таким чином, інтерфейс Movable определяет требования - реализовать метод перемщения (метод move). Класс Car применяет эти требования, но сам решает, як конкретно реализовать метод перемщения.

Подобным чином мы можемо реализовать інтерфейс в серии классов. Наприклад, добавим класс, який буде представлять самолет:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине

        Aircraft boing = new Aircraft();
        boing.move();   // Летим на самолете
    }
}

interface Movable{

    void move();
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}
class Aircraft implements Movable{ 

    public void move(){

        System.out.println("Летим на самолете");
    }
}

Методи по умолчанию

Інтерфейси в мові Java можуть иметь методы по умолчанию (default methods) - методы с реализацией по умолчанию, которая використовується, якщо класс, реализующий данный інтерфейс, не реализует метод.

Зачем нужны методы по умолчанию? Допустим, ранїї був создан інтерфейс, який затем реализовало множество классов. Если мы захочемо внести изменения в цей інтерфейс в силу яких-то новых условий, то нам також придется изменять все те методы, которые его применяют. Когда речь идет о паре классов, которые определены в одном файле с інтерфейсов, проблема может покизаться надуманной. Но когда Но когда буво определено множество классов множеством разработчиков по всему миру в течение многих лет, як в случае с інтерфейсами стандартной библиотеки Java, проблем становится болїї актуальной и трудно решаемой. Поскольку изменение інтерфейса приведет к тому, что классы больше не будуть компилироваться, поскольку не реализуют новые методы.

При наличии же метода по умолчанию класс все равно буде загружаться, даже с отсутствующим методом. Программы по-прежнему можуть создавать экземпляры классов, и ничего страшного не произойдет.

Наприклад, ранїї использованный інтерфейс Movable и добавим в него новый метод, який буде методом по умолчанию:


interface Movable{

    void move();
    default void stop() { 
        System.out.println("Останавливаемся"); 
    }
}

В данном случае метод stop() определен як метод по умолчанию. Метод по умолчанию - це обычный метод без модификаторов (формально имїїт модификатор public), який помечается ключевым словом default. В данном случае метод stop выводит строку на консоль. В принципе можна буво вообще ничего не делать и оставить пустые фигурные скобки (default void stop() { })

Затем в классе нам необязательно цей метод реализовать:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине
        car.stop();     // Останавливаемся
    }
}

interface Movable{

    void move();
    default void stop() { 
        System.out.println("Останавливаемся"); 
    }
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}

При желании також можна переопределить цей метод:


public class Program{
      
    public static void main(String[] args) {
        
        Movable car = new Car();
        car.move();     // Едем на машине
        car.stop();     // Машина останавливается
    }
}

interface Movable{

    void move();
    default void stop() {  }
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
    // переопределяем метод по умолчанию
    public void stop(){

        System.out.println("Машина останавливается");
    }
}

Статические методы

В інтерфейсах також можна визначати статические методы - они аналогичны методам класса:


interface Printable {
    
    void print();
	
    static void read(){
        
        System.out.println("Read printable");
    }
}

Щоб обратиться к статическому методу інтерфейса також, як и в случае с классами, пишут название інтерфейса и метод:


public static void main(String[] args) {
        
    Printable.read();
}

Приватные методы

По умолчанию все методы в інтерфейсе фактически имеют модификатор public. Однак начиная с Java 9 мы також можемо визначати в інтерфейсе методы с модификатором private. Они можуть быть статическими и нестатическими, но они не можуть иметь реализации по умолчанию.

Подобные методы можуть використовуватися лише внутри самого інтерфейса, в котором они определены. То есть к примеру нам треба выполнять в інтерфейсе некоторые повторяющиеся действия, и в цем случае такие действия можна выделить в приватные методы:


public class Program{
     
	public static void main(String[] args) {
         
		Operation c = new Operation(); 
		System.out.println(c.sum(1, 2));        // 3
		System.out.println(c.sum(1, 2, 4));     // 7
	}
}
class Operation implements Sumable{}

interface Sumable{

	default int sum(int a, int b){
		return sumAll(a, b);
	}
	default int sum(int a, int b, int c){
		return sumAll(a, b, c);
	}
	
    private int sumAll(int... values){
         int result = 0;
		 for(int n : values){
			 result += n;
		 }
		 return result;
    }
}

Константы в інтерфейсах

Кроме методів в інтерфейсах можуть быть определены статические константы:


interface Stateable{

	int OPEN = 1;
	int CLOSED = 0;
	
	void printState(int n);
}

Хотя такие константы також не имеют модификаторов, но по умолчанию они имеют модификатор доступа public static final, и поцему их значение доступно из любого места программы.

Застосування констант:


public class Program{
     
	public static void main(String[] args) {
         
		WaterPipe pipe = new WaterPipe();
		pipe.printState(1);
	}
}
class WaterPipe implements Stateable{
	
	public void printState(int n){
		if(n == OPEN)
			System.out.println("Water is opened");
		else if(n == CLOSED)
			System.out.println("Water is closed");
		else
			System.out.println("State is invalid");
	}
}
interface Stateable{

	int OPEN = 1;
	int CLOSED = 0;
	
	void printState(int n);
}

Вложенные інтерфейсы

Как и классы, інтерфейсы можуть быть вложенными, то есть можуть быть определены в классах або других інтерфейсах. Например:


class Printer{
    interface Printable {
    
        void print();
    }
}

При применении такого інтерфейса нам треба указывать его полное имя вместе с именем класса:


class Book implements Printer.Printable {

    public void print() {
        System.out.println("Печатаем книгу");
    }  
}

Использование інтерфейса буде аналогично предыдущим случаям:


Printer.Printable printable = new Book();
printable.print();
