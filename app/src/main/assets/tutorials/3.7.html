Інтерфейси

Останнє оновлення: 30.09.2025

Механізм успадкування двже зручний, але має свої обмеження. Зокрема ми можемоо успадковувати лише від одного класу, на відміну, наприклад, від мови С++, де є множинне успадкування.

У мові Java підібну проблему частково дозволяють вирішити інтерфейси. Інтерфейси визначають деякий функціонал, який не має конкретної реалізації, який потім реалізують класи, які використовують ці інтерфейси. І один клас може застосувати багато інтерфейсів.

Но суть інтерфейса не лише и не слише в цем. Ключевая мысли інтерфейса соварто в том, чтобы визначити некий набор требований, которым повинен соответствовать клас. Інтерфейс визначає, що повинен делать клас, а клас вже визначає, як он це робить.

Визначення інтерфейсу

Щоб визначити інтерфейс, використовується ключевое слово interface. Наприклад:


interface Movable{ }

Цей інтерфейс називається Movable і далі представлятиме якийсь транспортний засіб, щось, що може перещатися. Відповідно до умов назви інтерфейсів в Java починаються з великої літери і зазвичай закінчуються на суфікс "-able", хоча в принципі це необов'язково.

Використання інтерфейсу

Как буво сказано выше, інтерфейс визначає набор требований к класу, а клас застосовує эти требования. Щоб указать, что клас застосовує або реалізує інтерфейс, після названия класа вказуєся ключевое слово implements, після которого идет название інтерфейса:


interface Movable{ }
class Car implements Movable{ }

Тобто в даному випадку ми можемоо сказати, що Car застосовує або реалізує інтерфейс Movable. Грубо кажучи, ми говоримо, що машина (Car) - це транспортний засіб, щось, що функціонал для переміщення.

Змінні інтерфейси

Інтерфейс фактично представляє новий тип. І ми можемоо визначати змінні інтерфейси:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj;    // определяем інтерфейс
    }
}
// визначенийие інтерфейса
interface Movable{ }

Однак все же інтерфейсы - це не класы. Зокрема, мы не можемо використовувати оператор new для створенняя экземпляра інтерфейса:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj = new Movable();        // ! Ошибка, так нельзя сделать
    }
}
interface Movable{ }

Але змінної інтерфейсу можна привласнити посилання на об'єкт класу, який застосовує цей інтерфейс. Наприклад:


public class Program{
      
    public static void main(String[] args) {
        
        Movable obj = new Car();
        System.out.println(obj);
    }
}

interface Movable{ }
class Car implements Movable{ }

Оскільки клас Car застосовує інтерфейс Movable, змінної цього інтерфейсу ми можемоо привласнити об'єкт Car.

Визначення компонентів інтерфейсу

Выше визначенийный інтерфейс Movable був порожній и ничего не определял. Но інтерфейс может визначати ряд компонентов:

  • Методи без реалізації

  • Методи по умолчанию (с реализацией)

  • Приватные методи

  • Статические методи (як приватные, так и публичные)

  • Константи

Визначенийие методів в інтерфейсе

Как говорилось выше, інтерфейс визначає ряд требований або контракт, которым повинен соответствовать клас. И эти требования визначаються в інтерфейсе в виде методів. Методи інтерфейса можуть иметь, а можуть и не иметь реалізації. Если неякий клас застосовує визначенийный інтерфейс, то цей клас повинен реализовать все методи застосовуємоого інтерфейса (виняток касается методів, для которых в інтерфейсе визначенийа реалізація по умолчанию).

Наприклад, изменим інтерфейс Movable, добавив в него метод:


interface Movable{

    void move();
}

У даному випадку інтерфейс Movable представляє функционал транспортного средства и визначає один метод без реалізації - метод move(), який условно призначений для передвижения транспортного средства. Методи без реалізації похожи на абстрактные методи абстрактных класов. Все методи інтерфейса не имеют модификаторов доступа, но фактически по умолчанию доступ public, оскільки цель інтерфейса - визначенийие функционала для реалізації его класом. Тому весь функционал повинен быть открыт для реалізації.

Таким чином, у нас еть інтерфейс Movable, которое представляє непонятно якое транспортное средство, и есть функція move, которая призначенийа для перемщения транспортного средства, но як именно це перемщение осуществляется - неизвестно.

Однак клас, який буде застосовувати інтерфейс, повинен реализовать все его методи (которые не имеют реалізації). Теперь реализуем інтерфейс Movable:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине
    }
}

interface Movable{

    void move();
}
// клас машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}

У даному випадку клас Car застосовує інтерфейс Movable и поцему повинен реализовать его метод move(). Если клас не реалізує якие-то методи інтерфейса, то такой клас повинен быть визначений як абстрактный, но его неабстрактные класы-наследники затем все равно повинні будуть реализовать эти методи.

Таким чином, інтерфейс Movable визначає требования - реализовать метод перемщения (метод move). Клас Car застосовує эти требования, но сам решает, як конкретно реализовать метод перемщения.

Підобным чином мы можемо реализовать інтерфейс в серии класов. Наприклад, додамо клас, який буде представлять самолет:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине

        Aircraft boing = new Aircraft();
        boing.move();   // Летим на самолете
    }
}

interface Movable{

    void move();
}
// клас машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}
class Aircraft implements Movable{ 

    public void move(){

        System.out.println("Летим на самолете");
    }
}

Методи по умолчанию

Інтерфейси в мові Java можуть иметь методи по умолчанию (default methods) - методи с реализацией по умолчанию, которая використовується, якщо клас, реализующий данный інтерфейс, не реалізує метод.

Зачем нужны методи по умолчанию? Допустим, ранїї був создан інтерфейс, який затем реализовало множина класов. Если мы захочемо внести изменения в цей інтерфейс в силу яких-то новых условий, то нам також придется изменять все те методи, которые его застосовують. Когда речь идет о паре класов, которые визначенийы в одном файле с інтерфейсов, проблема может покизаться надуманной. Но когда Но когда буво визначенийо множина класов множинам разработчиков по всему миру в течение многих лет, як в случае с інтерфейсами стандартной бібліотеки Java, проблем становится болїї актуальной и трудно решаемой. Поскольку зміна інтерфейса приведет к тому, что класы больше не будуть компилироваться, поскольку не реалізують новые методи.

При наличии же методу по умолчанию клас все равно буде загружаться, даже с отсутствующим методом. Програми по-прежнему можуть создавать экземпляры класов, и ничего страшного не произойдет.

Наприклад, ранїї использованный інтерфейс Movable и додамо в него новый метод, який буде методом по умолчанию:


interface Movable{

    void move();
    default void stop() { 
        System.out.println("Останавливаемся"); 
    }
}

У даному випадку метод stop() визначений як метод по умолчанию. Метод по умолчанию - це обычный метод без модификаторов (формально имїїт модификатор public), який помечается ключевым словом default. В данном случае метод stop виводить рядок на консоль. В принципе можна буво вообще ничего не делать и оставить пустые фигурные скобки (default void stop() { })

Затем в класе нам необязательно цей метод реализовать:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        car.move();     // Едем на машине
        car.stop();     // Останавливаемся
    }
}

interface Movable{

    void move();
    default void stop() { 
        System.out.println("Останавливаемся"); 
    }
}
// клас машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}

При желании також можна перевизначити цей метод:


public class Program{
      
    public static void main(String[] args) {
        
        Movable car = new Car();
        car.move();     // Едем на машине
        car.stop();     // Машина останавливается
    }
}

interface Movable{

    void move();
    default void stop() {  }
}
// клас машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
    // переопределяем метод по умолчанию
    public void stop(){

        System.out.println("Машина останавливается");
    }
}

Статические методи

В інтерфейсах також можна визначати статические методи - они аналогичны методум класа:


interface Printable {
    
    void print();
	
    static void read(){
        
        System.out.println("Read printable");
    }
}

Щоб обратиться к статическому методу інтерфейса також, як и в случае с класами, пишут название інтерфейса и метод:


public static void main(String[] args) {
        
    Printable.read();
}

Приватные методи

По умолчанию все методи в інтерфейсе фактически имеют модификатор public. Однак начиная с Java 9 мы також можемо визначати в інтерфейсе методи с модификатором private. Они можуть быть статическими и нестатическими, но они не можуть иметь реалізації по умолчанию.

Підобные методи можуть використовуватися лише всередині самого інтерфейса, в котором они визначенийы. То есть к примеру нам треба выполнять в інтерфейсе некоторые повторяющиеся действия, и в цем случае такие действия можна выделить в приватные методи:


public class Program{
     
	public static void main(String[] args) {
         
		Operation c = new Operation(); 
		System.out.println(c.sum(1, 2));        // 3
		System.out.println(c.sum(1, 2, 4));     // 7
	}
}
class Operation implements Sumable{}

interface Sumable{

	default int sum(int a, int b){
		return sumAll(a, b);
	}
	default int sum(int a, int b, int c){
		return sumAll(a, b, c);
	}
	
    private int sumAll(int... values){
         int result = 0;
		 for(int n : values){
			 result += n;
		 }
		 return result;
    }
}

Константи в інтерфейсах

Кроме методів в інтерфейсах можуть быть визначенийы статические константи:


interface Stateable{

	int OPEN = 1;
	int CLOSED = 0;
	
	void printState(int n);
}

Хотя такие константи також не имеют модификаторов, но по умолчанию они имеют модификатор доступа public static final, и поцему их значення доступно из любого места програми.

Застосування констант:


public class Program{
     
	public static void main(String[] args) {
         
		WaterPipe pipe = new WaterPipe();
		pipe.printState(1);
	}
}
class WaterPipe implements Stateable{
	
	public void printState(int n){
		if(n == OPEN)
			System.out.println("Water is opened");
		else if(n == CLOSED)
			System.out.println("Water is closed");
		else
			System.out.println("State is invalid");
	}
}
interface Stateable{

	int OPEN = 1;
	int CLOSED = 0;
	
	void printState(int n);
}

Вложенные інтерфейсы

Как и класы, інтерфейсы можуть быть вложенными, то есть можуть быть визначенийы в класах або других інтерфейсах. Наприклад:


class Printer{
    interface Printable {
    
        void print();
    }
}

При применении такого інтерфейса нам треба указывать его полное ім'я вместе с именем класа:


class Book implements Printer.Printable {

    public void print() {
        System.out.println("Друкуємо книгу");
    }  
}

Використання інтерфейса буде аналогично предыдущим случаям:


Printer.Printable printable = new Book();
printable.print();
