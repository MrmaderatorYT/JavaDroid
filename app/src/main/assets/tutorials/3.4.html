Статичні компоненти класу та модифікатор static

Останнє оновлення: 16.09.2025

Крім звичайних методів та полів клас може мати статичні поля, методи, константи та ініціалізатори. Крім звичайних полів, методів, властивостей класи та структури можуть мати статичні поля, методи, властивості. Статичні поля та методи до всього класу, і для звернення до підібних полів та методів необов'язково створювати об'єкт класу.

Наприклад, главный клас програми на Java повинен иметь метод main, який є статическим:


public static void main(String[] args) {
    
}

Для оголошення статических змінних, констант, методів и инициализаторов перед их оголошенням вказуєся ключевое слово static.

Статичні поля

Якщо звичайні поля класу зберігають стан об'єкта, статичні поля зберігають стан всього класу. Під час створення об'єктів класу кожному за об'єкта створюється своя копія нестатичних звичайних полів. А статичні поля є спільними для всього класу і тому створюються у єдиному вигляді.

Статичне поле визначається як і просте, лише перед типом поля вказується ключове слово static. Наприклад, розглянемо статичні поля у межах класу Person, який представляє людини, та його застосування:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person(41);
        tom.checkAge();     // Сколько лет осталось до пенсии: 24

        Person bob = new Person(68);
        bob.checkAge();     // Вже на пенсии

        // отримання статического поля
        System.out.println(Person.retirementAge); // 65
        // зміна статического поля
        Person.retirementAge = 67;
        System.out.println(Person.retirementAge); // 67
    }
}

class Person{

    int age;        // обычное поле
    static int retirementAge = 65;  // статическое поле

    Person(int age){
        this.age = age;
	}

    void checkAge()
    {
        if (age >= retirementAge)
            System.out.println("Вже на пенсии");
        else
            System.out.printf("Сколько лет осталось до пенсии: %d\n", retirementAge - age);
    }
}

У даному випадку клас Person имїїт два поля: age (хранит возраст человека) и retirementAge (хранит пенсионный возраст). Однак поле retirementAge є статическим. Оно относится не к конкретному человеку, а ко всем людям. (У даному випадку для упрощения пренебрежем тем фактом, что в зависимости от пола и профессии пенсионный возраст может отличаться.) Таким чином, поле retirementAge относится не к отдельную об'єкту и хранит значення НЕ отдельного об'єкта класа Person, а относится ко всему класу Person и хранит общїї значення для всего класа.

Причем в самом класе мы можемо використовувати це поле як и любые другие. Так, в методі checkAge(), який поверяет пенсионный статус человека, для проверки використовуємо це поле:

if (age >= retirementAge)

Але якщо ми хочемо звернутися до цього поля поза своїм класом, то ми можемоо звертатися до цього поля на ім'я класу:


System.out.println(Person.retirementAge);
Person.retirementAge = 67;

На рівні пам'яті для статичних полів створюватиметься ділянка в пам'яті, яка буде спільною для всіх об'єктів класу.

При цьому пам'ять для статичних змінних виділяється навіть у тому випадку, якщо не створено жодного об'єкта цього класу.

Статичні ініціалізатори

Статичні ініціалізатори призначені для ініціалізації статичних змінних, або для виконання таких дій, які виконуються при створенні першого об'єкта. Наприклад, визначимо статичний ініціалізатор:


public class Program{
     
	public static void main(String[] args) {
        
        Person tom = new Person(41);
        Person bob = new Person(68);

        tom.checkAge();     // Сколько лет осталось до пенсии: 24
        bob.checkAge();     // Вже на пенсии
	}
}

class Person{

    int age;        // обычное поле
    static int retirementAge;  // статическое поле

    static{
        retirementAge = 65;
        System.out.println("Static initializer");
    }

    Person(int age){
        this.age = age;
        System.out.println("Constructor");
	}

    void checkAge()
    {
        if (age >= retirementAge)
            System.out.println("Вже на пенсии");
        else
            System.out.printf("Сколько лет осталось до пенсии: %d\n", retirementAge - age);
    }
}

Статический инициализатор визначаєся як обычный, лише перед ним ставится ключевое слово static. У даному випадку в статическом инициализаторе мы устанавливаем начальное значення статического поля retirementAgeи вивідим на консоль сообщение.

У самій програмі створюються два об'єкти класу Person. Тому консольний висновок виглядатиме так:


Static initializer
Constructor
Constructor
Сколько лет осталось до пенсии: 24
Вже на пенсии

Варто враховувати, що виклик статичного ініціалізатора здійснюється після завантаження класу і фактично до створення першого об'єкта класу.

Статичні константи

Також статичними бувають константи, які є загальними для класу.


public class Program{
     
	public static void main(String[] args) {
        
        double radius = 60;
		System.out.printf("Radisu: %f \n", radius);				// 60
		System.out.printf("Area: %f \n", Math.PI * radius);		// 188,4
	}
}
class Math{
	public static final double PI = 3.14;
}

Варто зазначити, що протягом усіх попередніх тем активно використовувалися статичні константи. Зокрема, у виразі:


System.out.println("hello");

out як раз представляє статическую константу класа System. Тому обращение к ней идет без створенняя об'єкта класа System.

Статичні методи

Статические методи визначають общїї для всех об'єктов поведение, которое не зависит от конкретного об'єкта. В любой програме на Java мы вже сталкиваемся як минмум с одним статическим методом - це метод main, який автоматически запускуєся средой. И согласно правилам языка цей метод повинен быть статическим:


public class Program{
     
	// метод main повинен быть статическим 
	public static void main(String[] args) {
        
        // здесь действия програми
	}
}

Для звернення до статичними методуми також застосовується ім'я класу. Наприклад, визначимо таку програму:


public class Program{
     
	public static void main(String[] args) {
        
        Person tom = new Person(41);
        Person.checkStatus(tom); // Сколько лет осталось до пенсии: 24
	}
}

class Person{

    int age; 
    static int retirementAge = 65;

    Person(int age){
        this.age = age;
	}
    // статический метод
    static void checkStatus(Person person)
    {
        if (person.age >= retirementAge)
            System.out.println("Вже на пенсии");
        else
            System.out.printf("До пенсии осталось %d лет\n", retirementAge - person.age);
    }
}

У даному випадку в класе Person визначений статический метод checkStatus(), який в качестве параметра приймає об'єкт Person и проверяет его пенсионный статус.

Следует учитывать, что статические методи можуть обращаться лише к статическим членам класа. Обращаться к нестатическим методум и полям цього же класа всередині статического методу мы не можемо. Тому, наприклад, чтобы вызвать в методі main другие методи, которые визначенийы в одном класе с методом main, они також повинні иметь модификатор static:


public class Program{
     
	public static void main(String[] args) {
        
        sum(4, 5);   // метод статический, поцему мы можемо его вызывать в методі main
        // add(4, 5);   // метод НЕстатический, поцему мы НЕ можемо его вызвать в методі main
	}

    static void sum(int a, int b){
        System.out.printf("Сумма чисел %d и %d равна %d\n", a, b, a+b);
    }

    void add(int a, int b){
        System.out.printf("Сумма чисел %d и %d равна %d\n", a, b, a+b);
    }
}

Підкреслю, що в цьому випадку йдеться про звернення до методів та полів, які визначені в тому самому класі.

Взагалі методи визначаються як статичні, коли методи не торкаються стану об'єкта, тобто його нестатичні поля та константи, і для виклику методу немає сенсу створювати екземпляр класу. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		System.out.println(Operation.sum(45, 23));			// 68
		System.out.println(Operation.subtract(45, 23));		// 22
		System.out.println(Operation.multiply(4, 23));		// 92
	}
}
class Operation{
	
	static int sum(int x, int y){
		return x + y;
	}
	static int subtract(int x, int y){
		return x - y;
	}
	static int multiply(int x, int y){
		return x * y;
	}
}

У разі для методів sum, subtract, multiply немає значення, який саме екземпляр класу Operation використовується. Ці методи працюють тільки з параметрами, не торкаючись стану класу. Тому їх можна визначити як статичні.
