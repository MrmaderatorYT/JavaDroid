Навантаження методів

Останнє оновлення: 16.09.2025

В программе мы можемо використовувати методы с одним и тем же именем, но с разными типуми и/или количеством параметров. Такой механизм називається перевантаження методів (method overloading).

Наприклад:


class Program{

    public static void main(String[] args) {  

        Calcualtor calc = new Calcualtor();
        System.out.println(calc.sum(2, 3));          // 5
        System.out.println(calc.sum(4.5, 3.2));      // 7.7
        System.out.println(calc.sum(4, 3, 7));       // 14
    }
}

class Calcualtor{

    int sum(int x, int y){
             
        return x + y;
    }
    double sum(double x, double y){
             
        return x + y;
    }
    int sum(int x, int y, int z){
             
        return x + y + z;
    }
}

Здесь определен класс Calculator с тремя вариантами або тремя перегрузками метода sum(), но при его вызове в зависимости от типу и количества передаваемых параметров система выберет именно ту версию, которая наиболїї подходит.

На перевантаження методів впливають кількість і типи параметрів. Однак різниця в типі значення для перевантаження не мають ніякого значення. Наприклад, у наступному випадку методи sum розрізняються за типом значення, що повертається:


class Program{

    public static void main(String[] args) {  

        Calcualtor calc = new Calcualtor();
        System.out.println(calc.sum(2, 3));
    }
}

class Calcualtor{

    int sum(int x, int y){
             
        return x + y;
    }
    // Ошибка - метод sum вже определен
    double sum(int x, int y){
             
        return x + y;
    }
}

Однак навантаженням це не вважатиметься. Більше того така програма некоректна і просто не скомілюється, тому що метод з одним і тим самим кількістю і типом параметрів визначено кілька разів.
