Перша програма у NetBeans

Останнє оновлення: 28.09.2020

Ще одной популярной средой программирования под Java є Netbeans. Проект NetBeans во многом поддерживается и спонсируется компанией Oracle, при цем его разработка ведется независимым сообществом разработчиков (Apache NetBeans Community). NetBeans є кроссплатформенным программным обеспечением, поцему есть версии среды для различных платформ и ОС. Болїї подробную информацию можна найти на официальном сайте даной среды https://netbeans.apache.org/.

На данный момент последней версией є 12-я версия. Итак, установим последнюю версию NetBeans, загрузив со страницы https://netbeans.apache.org/download/nb121/nb121.html.

На странице загрузки мы можемо найти два варианта: скомпилированные бинарные файлы в виде архива (пункт Binaries) и установщики для каждой конкретной операционной системы (пункт Installers). Загрузим среду в скомпилированном, выбрав пункт Binaries.

Після вибору цієї опції нас переадресує на сторінку із прямим посиланням на дзеркало для завантаження:

После загрузки распакуем архив в папку, где мы хочемо разместить среду Netbeans. Варто відзначити, что после распаковки архива мы можемо вже відразу запустить среду, не треба ничего устанавливать. В архиве вже готовые для использования файлы. Для запуска в распакованном архиве в папке bin мы можемо найти собственно запускаемые файлы. В случае с ОС Windows це netbeans.exe и netbeans64.exe

Запустимо netbeans.exe чи netbeans64.exe:

NetBeans довольно интеллектуально понятна, и с ней очень легко работать. Создадим новый проект. Для цього выберем в меню пункт File->New Project.... После цього перед нами откроется окно создания нового проекта:

В окне создания нового проекта выберем в левой колонке первый пункт - Java with Ant, а в правой в качестве типу проекта - Java Application. И нажмем кнопку Next >

При першому створенні проекту Java нам буде запропоновано активувати відповідну функцію:

И на данном этапе нажмем на кноку Activate.

Потім з'явиться вікно налаштувань проекту:

В поле Project Name дадим проекту якое-нибудь название (в моем случае це HelloApp). Для всех остальных полей можна оставить значения по умолчанию. Последнїї поле "Create Main Class" указывает, что автоматически в проекте буде создан класс программы HelloApp, який буде находиться в одноименном пакеті helloapp. И в завершении создания проекта нажмем на кнопку "Finish". И перед нами откроется новый проект в Netbeans:

Вузьке вікно зліва відображає всі відкриті проекти та їхню структуру. У даному випадку у нас відкритий поки що лише один проект. А велике вікно справа представляє редактор коду з деякими додатковими функціями, як підцвітання коду, інтелектуальна підказка і т.д. Редактор коду вже має певний зміст за замовчуванням:


/*
 * To change this license header, choose License Headers in Project Properties.
 * To change this template file, choose Tools | Templates
 * and open the template in the editor.
 */
package helloapp;

/**
 *
 * @author Eugene
 */
public class HelloApp {

    /**
     * @param args the command line arguments
     */
    public static void main(String[] args) {
        // TODO code application logic here
    }
    
}

Здесь, во-первых, много комментариев, которые нам не нужны и которые мы можемо удалить. Під-вторых, в начале файла указывается, что класс буде находиться в пакеті helloapp за допомогою директивы package: package helloapp;. Затем, як вже рассматривалось в прошлой теме, у нас есть класс программы - класс HelloApp, який имїїт метод main.

Змінимо код за замовчуванням на наступний, щоб програма виводила найпростіший рядок на консоль:


package helloapp;

public class HelloApp {

    public static void main(String[] args) {
        
        System.out.println("Hello Java 15 from Netbeans!");
    }
    
}

Тепер запустимо проект виконання. Для цього на панелі інструментів NetBeans натиснемо на зелену стрілку (або можна натиснути на клавішу F6, або вибрати меню пункт Run -> Run Project). І внизу NetBeans відкриється вікно виведення, в якому ми зможемоо спостерігати результати нашої програми:

Фактично це вікно виведення еквівалентної консолі за деякими винятками.
