Класи та об'єкти

Останнє оновлення: 22.09.2025

Java є об'єктно-орієнтованою мовою, тому такі поняття як "клас" та "об'єкт" відіграють у ньому ключову роль. Будь-яку програму на Java можна як набір взаємодіючих між собою об'єктів.

Шаблоном або описом об'єкта є клас, а об'єкт представляє экземпляр цього класа. Можна ще провести наступну аналогію. У нас у всех есть некоторое представление о человеке - наявність двох рук, двох ніг, голови, тулуба і т.д. Есть неякий шаблон - цей шаблон можна назвати класом. Реально ж існуюча людина (фактично екземпляр даного класу) є об'єктом цього класа.

У принципі, раніше вже використовувалися класи. Наприклад, у попередніх темах ми визначали клас Program, у якому визначався решта коду програми. Наприклад:


class Program{

    public static void main(String[] args) {  

        System.out.println("Hello ");
    }
}

Тут у нас клас Program, у якому визначений метод main, і цей метод виводить на консоль деяке повідомлення. Для виведення на консоль ми використовуємо ще один клас - System. Теперь же посмотрим, як мы можемо визначати и використовувати другие класы.

Визначення класів

Клас визначається за допомогою ключового слова сlass, після которого идет название класа:


class ім'я_класа{

}

Назва класу підпорядковується тим самим правилам, як і назви змінних. Але загалом Java має умовність, що зазвичай назви класів починаються з великої літери.

Після назви класу йдуть фігурні дужки, між якими міститься тіло класу - тобто його компоненти (поля та методи).

Наприклад, визначимо клас, який назвемо Person:


// клас Program
class Program{

    public static void main(String[] args) {  

    }
}

// клас Person
class Person{ }

Після імені класу Person йдуть порожні фігурні дужки, тобто клас Person не має жодних компонентів. Як правило, класи визначаються в різних файлух, але в даному випадку для простоти ми визначаємо два класи в одному файлі.

Об'єкти класів

Класи в Java представляютььь окремі типи даних. І коли ми визначаємо новий клас, який навіть нічого не робить і не має жодного вмісту, фактично ми визначаємо новий тип даних. І як у випадку зі стандартними і наявними в Java типуми даних ми можемоо визначити змінні цього типу. Наприклад:


class Program{

    public static void main(String[] args) {  

        Person tom;  // определяем змінну класа Person
    }
}

// клас Person
class Person{ }

У даному випадку в методі main класу Program визначена змінна типу Person, которая називається "tom". Поки така змінна не має жодного значення, то есть она не ініціалізована. По великому рахунку ми її поки не можемоо використовувати, поцему вначале необхідно створити об'єкт класа Person.

Для створенняя об'єкта застосовуються конструктори. По сути конструктори представляютьь специальные методи, які називаються так само як і клас, и которые вызываются при створенні нового об'єкта класа і виконують ініціалізацію об'єкта. Общий синтаксис вызова конструктора:

new конструктор_класа(параметри_конструктора);

Спочатку йде оператор new, який виділяє пам'ять для об'єкта, а після него идет вызов конструктора.

Конструктор за замовчуванням

Якщо в класі не визначено жодного конструктора (як в случае с нашим пустым класом Person), то для цього класа автоматически створюєся порожній конструктор по умолчанию, який не приймає нияких параметрів. Тому застосуємо конструктор по умолчанию для створенняя об'єкта класа Person:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();  // створення об'єкта класа Person
    }
}

// клас Person
class Person{ }

Для створенняя об'єкта Person використовується выражение new Person(). У результаті після виконання даного виразу в пам'яті буде виділено ділянку, де зберігатимуться всі дані об'єкта Person. А змінна tom получит ссылку на створений об'єкт, і через цю змінну ми можемоо використовувати даний об'єкт и обращаться к его функциональности.

Але поки в класі Person ми не визначили ніякої функціональності, Java дозволяє нам вивести об'єкт на консоль:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();  // створення об'єкта класа Person
        System.out.println(tom);  // вивідим об'єкт tom на консоль
    }
}

// клас Person
class Person{ }

У разі Java виводить на консоль деяке текстове уявлення об'єкта типу "Person@1dbd16a6".

Поля класу

Будь-який об'єкт може мати дві основні характеристики: стан - деякі дані, які зберігає об'єкт, і поведінка - дії, які може здійснювати об'єкт.

Для зберігання стану об'єкта у класі застосовуються поля чи змінні класу.

Наприклад, ми визначили порожній клас Person. Додамо до нього поля. Що за поля ми додаватимемо в нього? Клас Person у нас представлятиме деяку людину. У людини ми можемоо виділити такі ознаки як ім'я та вік (за бажання можна виділити і більше ознак). Тому додамо цих ознак до класу поля:


class Person{
	
	String name;		// ім'я
	int age;			// возраст
}

Поле name представляє ім'я людини і представляє рядок, а age - його вік і становить число.

Звернення до функціональності класу

Для звернення до функціональності класу - полів, методів (і навіть іншим елементам класу) застосовується точкова нотація точки - після об'єкта класу ставиться крапка, та був елемент класу::


об'єкт.поле_класа
об'єкт.метод_класа(параметри_методу)

Наприклад, звернемося до полів об'єкта Person::


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();  // створення об'єкта класа Person

        // отримуємо значення полей
        System.out.println(tom.name);  // null
        System.out.println(tom.age);  // 0

        // устанавливаем новые значення полей
        tom.name = "Tom";
        tom.age = 41;

        // повторно отримуємо значення полей
        System.out.println(tom.name);  // Tom
        System.out.println(tom.age);  // 41
    }
}

// клас Person
class Person{ 
    String name;
    int age;
}

Для звернення до поля name об'єкта tom застосовується вираз tom.name, а для звернення до поля age - вираз tom.age. У програмі ми спочатку отримуємо значення полів і виводимо їх на консоль:


System.out.println(tom.name);
System.out.println(tom.age);

А потім встановлюємо їх значення. Так, полю name передаємо рядок, оскільки це поле представляє тип String, а полю age передаємо число, оскільки воно є цілим типом int. У результаті отримаємо наступний консольний висновок програми:


null
0
Tom
41

Ініціалізація полів

Якщо ми підивимося на вивід попередньої програми, то побачимо, що до присвоєння значення полям вони приймають деякі значення за замовчуванням. Якщо полю класу явним чином не присвоюється значення, то йому автоматично буде присвоєно значення за замовчуванням: числам — 0, логічним значенням — false, а ссылкам на об'єкты (наприклад, рядокм) — null. У цьому полягає важлива різниця між полями та локальними змінними: локальні змінні в методі необхідно явно ініціалізувати, тоді як у класі, якщо ви не ініціалізуєте поле, воно автоматично ініціалізується значенням за замовчуванням (0, false або null).

Однак поля можна явно ініціалізувати деякими значеннями. Наприклад:


class Program{

    public static void main(String[] args) {  

        Person tom = new Person();

        System.out.println(tom.name);  // Undefined
        System.out.println(tom.age);  // 1
    }
}

// клас Person
class Person{ 
    String name = "Undefined";
    int age = 1;
}

Створення кількох об'єктів

Підібним чином ми можемоо створювати більшу кількість об'єктів. Наприклад:



class Program{

    public static void main(String[] args) {  

        Person tom = new Person();
        Person bob = new Person(); 

        // устанавливаем новые значення полей об'єкта tom
        tom.name = "Tom";
        tom.age = 41;

        // устанавливаем новые значення полей об'єкта bob
        bob.name = "Bob";
        bob.age = 46;

        // отримуємо значення полей об'єкта tom
        System.out.println(tom.name);  // Tom
        System.out.println(tom.age);  // 41

        // отримуємо значення полей об'єкта bob
        System.out.println(bob.name);  // Bob
        System.out.println(bob.age);  // 46
    }
}

// клас Person
class Person{ 
    String name;
    int age;
}

Тут створюється два об'єкти класу Person – tom і bob. Для кожного з цих об'єктів виділятиметься своя область у пам'яті, і кожен з цих об'єктів матиме свій набір полів name та age.
