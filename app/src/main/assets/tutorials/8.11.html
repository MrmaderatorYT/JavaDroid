Управління потіками

Останнє оновлення: 16.10.2025

Клас Thread надає ряд методів для управления потіками, которые мы можемо використовувати в програмах. Рассмотрим ряд цих методів.

Метод sleep

Статический метод Thread.sleep() приостанавливает текущий потік (переводит в спящий режим) на визначенийный период времени. Цет метод имїїт ряд версий:


static void sleep(long millis)
static void sleep(long millis, int nanos)
static void sleep(Duration duration)

Все три версии в качестве параметра приймають час приостановки потіка. В первой версии - це кількість миллисекунд, во второй версии дополнительно передаєся кількість наносекунд, которые додаються к милллисекундам. В третьей версии час передаєся через об'єкт Duration

Варто учитывать, что цей метод может сгенерировать виняток типу InterruptedException. Соответственно нам его треба обпрацювати або передать обработку внешнему коду.

Застосуємо первую версию:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // отримуємо ім'я текущего потіка

        System.out.println(name + " started...");

        System.out.println(name + " works"); 
        
        try{
            Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var task = new MyTask();
        new Thread(task, "MyTask").start();

        System.out.println("Main thread finished...");
    }
}

У даному випадку в класе MyTask в методі run(), який визначає действия потіка, в конструкции try..catch осуществляем задержку на 1000 миллисекунд (1 минуту)


try{
    Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

Таким чином, когда виконання дойдет до вызова Thread.sleep(1000) потік приостановит виконання и возобновит его лише через 1 секунду. В итоге мы получим следующий консольный вивід:


Main thread started...
Main thread finished...
MyTask started...
MyTask works
MyTask finished...

Очікування завершення потіка и метод join

При запуску потіков в примерах ранїї основной потік (Main thread) завершался до дочернего потіка. Как правило, болїї распространенной ситуацией є случай, когда основной потік завершуєся самым післядним - після всех своих дочерних потіков. Для цього треба пріменіть метод join():


void join()
void join(long millis)
void join(long millis, int nanos)
boolean join(Duration duration)

Цет метод також имїїт декілька версий. Но вне зависимости от версии текущий потік буде ожидать завершення потіка, для которого вызван метод join(). За допомогою дополнительных параметрів в метод можна передать час, в течение которго текущий потік буде ожидать буде ожидать завершення потіка.

Наприклад, изменим предыдущий пример - пусть главный потік теперь очікує завершення дочернего потіка:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // отримуємо ім'я текущего потіка

        System.out.println(name + " started...");

        System.out.println(name + " works");
        
        try{
            Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var task = new MyTask();
        var t = new Thread(task, "MyTask");
        t.start();

        try{
            t.join();  // текущий потік Main thread очікує завершення потіка t
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println("Main thread finished...");
    }
}

Варто учитывать, что метод join() також генерує виняток, поцему его вызов також помещаем в конструкцию try..catch с обработкой винятки InterruptedException.


var task = new MyTask();
var t = new Thread(task, "MyTask");
t.start();

try{
    t.join();  // текущий потік Main thread очікує завершення потіка t
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

В итоге теперь главный потік завершится після потіка MyTask:


Main thread started...
MyTask started...
MyTask works
MyTask finished...
Main thread finished...

Однак при ожидании потіка мы можемо столкнуться со слишком долгим або даже неограниченным очікуванням завершення потіка, особенно якщо в его коді існує якая-то логическая ошибка. В цем случае мы можемо ограничить очікування визначенийным временем. Наприклад, ограничим очікування 500 миллисекундами:


var task = new MyTask();
var t = new Thread(task, "MyTask");
t.start();

try{
    t.join(500);  // ждем завершення потіка t не болїї 500 миллисекунд
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

В примере выше потік MyTask виконується болїї 1 секунды, тогда як его очікування длится не болїї 500 миллисекунд. Тому главный потік завершит свою работу, не дожидаясь завершення MyTaskЖ


Main thread started...
MyTask started...
MyTask works
Main thread finished...
MyTask finished...

Ождание группы потіков

Если в програме використовується декілька дочерних потіков, и треба, чтобы главный потік (Main thread) завершался після дочерних, то для каждого дочернего потіка треба вызвать метод join(). Наприклад:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // отримуємо ім'я текущего потіка

        System.out.println(name + " started...");

        System.out.println(name + " works");
        
        try{
            Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        // определяем и запускуємо 4 потіка
        Thread[] tasks = new Thread[4];
        for(int i = 0; i < tasks.length; i++) {

            tasks[i] = new Thread(new MyTask(), "MyTask_" + i);
            tasks[i].start();
        }   

        //  ждем завершення 4 потіков
        try{
            
            for(int i = 0; i < tasks.length; i++)  tasks[i].join();
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println("Main thread finished...");
    }
}

Здесь запускуємо 4 потіка и затем ожидаем их завершення. В итоге главный потік буде завершаться лише після завершення всех четырех дочерних потіков:


Main thread started...
MyTask_2 started...
MyTask_1 started...
MyTask_2 works
MyTask_1 works
MyTask_3 started...
MyTask_3 works
MyTask_0 started...
MyTask_0 works
MyTask_1 finished...
MyTask_2 finished...
MyTask_3 finished...
MyTask_0 finished...
Main thread finished...
