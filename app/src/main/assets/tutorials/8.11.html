Управління потоками

Останнє оновлення: 16.10.2025

Класс Thread надає ряд методів для управления потоками, которые мы можемо використовувати в программах. Рассмотрим ряд цих методів.

Метод sleep

Статический метод Thread.sleep() приостанавливает текущий поток (переводит в спящий режим) на определенный период времени. Цет метод имїїт ряд версий:


static void sleep(long millis)
static void sleep(long millis, int nanos)
static void sleep(Duration duration)

Все три версии в качестве параметра принимают час приостановки потока. В первой версии - це количество миллисекунд, во второй версии дополнительно передаєся количество наносекунд, которые добавляются к милллисекундам. В третьей версии час передаєся через об'єкт Duration

Варто учитывать, что цей метод может сгенерировать исключение типу InterruptedException. Соответственно нам его треба обработать або передать обработку внешнему коду.

Применим первую версию:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // получаем имя текущего потока

        System.out.println(name + " started...");

        System.out.println(name + " works"); 
        
        try{
            Thread.sleep(1000); // приостанавливаем поток на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var task = new MyTask();
        new Thread(task, "MyTask").start();

        System.out.println("Main thread finished...");
    }
}

В данном случае в классе MyTask в методе run(), який определяет действия потока, в конструкции try..catch осуществляем задержку на 1000 миллисекунд (1 минуту)


try{
    Thread.sleep(1000); // приостанавливаем поток на 1000 миллисекунд
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

Таким чином, когда выполнение дойдет до вызова Thread.sleep(1000) поток приостановит выполнение и возобновит его лише через 1 секунду. В итоге мы получим следующий консольный вывод:


Main thread started...
Main thread finished...
MyTask started...
MyTask works
MyTask finished...

Ожидание завершения потока и метод join

При запуске потоков в примерах ранїї основной поток (Main thread) завершался до дочернего потока. Как правило, болїї распространенной ситуацией є случай, когда основной поток завершуєся самым последним - после всех своих дочерних потоков. Для цього треба применить метод join():


void join()
void join(long millis)
void join(long millis, int nanos)
boolean join(Duration duration)

Цет метод також имїїт декілька версий. Но вне зависимости от версии текущий поток буде ожидать завершения потока, для которого вызван метод join(). С помощью дополнительных параметров в метод можна передать час, в течение которго текущий поток буде ожидать буде ожидать завершения потока.

Наприклад, изменим предыдущий пример - пусть главный поток теперь очікує завершения дочернего потока:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // получаем имя текущего потока

        System.out.println(name + " started...");

        System.out.println(name + " works");
        
        try{
            Thread.sleep(1000); // приостанавливаем поток на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var task = new MyTask();
        var t = new Thread(task, "MyTask");
        t.start();

        try{
            t.join();  // текущий поток Main thread очікує завершения потока t
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println("Main thread finished...");
    }
}

Варто учитывать, что метод join() також генерує исключение, поцему его вызов також помещаем в конструкцию try..catch с обработкой исключения InterruptedException.


var task = new MyTask();
var t = new Thread(task, "MyTask");
t.start();

try{
    t.join();  // текущий поток Main thread очікує завершения потока t
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

В итоге теперь главный поток завершится после потока MyTask:


Main thread started...
MyTask started...
MyTask works
MyTask finished...
Main thread finished...

Однак при ожидании потока мы можемо столкнуться со слишком долгим або даже неограниченным ожиданием завершения потока, особенно якщо в его коде существует якая-то логическая ошибка. В цем случае мы можемо ограничить ожидание определенным временем. Наприклад, ограничим ожидание 500 миллисекундами:


var task = new MyTask();
var t = new Thread(task, "MyTask");
t.start();

try{
    t.join(500);  // ждем завершения потока t не болїї 500 миллисекунд
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

В примере выше поток MyTask виконується болїї 1 секунды, тогда як его ожидание длится не болїї 500 миллисекунд. Тому главный поток завершит свою работу, не дожидаясь завершения MyTaskЖ


Main thread started...
MyTask started...
MyTask works
Main thread finished...
MyTask finished...

Ождание группы потоков

Если в программе використовується декілька дочерних потоков, и треба, чтобы главный поток (Main thread) завершался после дочерних, то для каждого дочернего потока треба вызвать метод join(). Например:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // получаем имя текущего потока

        System.out.println(name + " started...");

        System.out.println(name + " works");
        
        try{
            Thread.sleep(1000); // приостанавливаем поток на 1000 миллисекунд
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        // определяем и запускаем 4 потока
        Thread[] tasks = new Thread[4];
        for(int i = 0; i < tasks.length; i++) {

            tasks[i] = new Thread(new MyTask(), "MyTask_" + i);
            tasks[i].start();
        }   

        //  ждем завершения 4 потоков
        try{
            
            for(int i = 0; i < tasks.length; i++)  tasks[i].join();
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println("Main thread finished...");
    }
}

Здесь запускаем 4 потока и затем ожидаем их завершения. В итоге главный поток буде завершаться лише после завершения всех четырех дочерних потоков:


Main thread started...
MyTask_2 started...
MyTask_1 started...
MyTask_2 works
MyTask_1 works
MyTask_3 started...
MyTask_3 works
MyTask_0 started...
MyTask_0 works
MyTask_1 finished...
MyTask_2 finished...
MyTask_3 finished...
MyTask_0 finished...
Main thread finished...
