Поліморфізм та динамічна диспетчеризація методів

Останнє оновлення: 24.09.2025

Полиморфизм є одной из ключевых концепций об'єктно-ориентированного програмирования и підразумевает, что сущность может принимать различные формы. В мові Java змінні є полиморфными. Це значит, что змінна базового класа может ссылаться на об'єкт производного класа. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		tom.print();

        System.out.println();

		Person sam = new Employee("Sam", "Oracle");
		sam.print();
	}
}
class Person {
    
    private String name;
   
    Person(String name){ this.name = name; }
 
    void print(){
		
		System.out.println("Name: " + name);
	}
}

class Employee extends Person{

    private String company;
    
    Employee(String name, String company) {
    
        super(name);
        this.company = company;
    }
    @Override
    void print(){
        
        super.print();
        System.out.println("Company: " + company);
    }
}

Оскільки Employee успадковується від Person, об'єкт Employee є водночас і об'єктом Person. Грубо кажучи, будь-який працівник підприємства одночасно є людиною. Тому змінної типу Person ми можемоо надати посилання на об'єкт типу Employee.

Person sam = new Employee("Sam", "Oracle");

Однак, незважаючи на те, що змінна представляє об'єкт Person, віртуальна машина бачить, що насправді вона вказує на об'єкт Employee. Тому при викликі методів цього об'єкта буде викликатися та версія методу, яка визначена в класі Employee, а не в Person. Наприклад:


Person sam = new Employee("Sam", "Oracle");
sam.print();  // вызов реалізації методу print из класа Employee

При вызове перевизначенийного методу виртуальная машина динамически находит и вызывает именно ту версию методу, которая визначенийа в підкласе. Данный процес ще називається dynamic method lookup або динамический поиск методу або динамическая диспетчеризация методів. А встановлення точной версии методу називається динамічним зв'язуванням (dynamic binding)

Для упрощения поиска виртуальная машина заранїї вычисляет таблицю методів для каждого класа (method table). В таблице методів перечисляются все сигнатури методів (названия методів и типи их параметрів) и сами вызываемые методи. Виртуальная машина может построить таблицю методів після загрузки класа, об'єднав методи, найденные в файле класа, с таблицей методів базового класа. При фактическом вызове методу виртуальная машина просто виконує поиск в таблице.

Если же метод визначений с модификаторами private, static, final або є конструктором, то компилятор точно знает, якой метод вызывать. Це називається статичним зв'язуванням (static binding).

Консольний висновок цієї програми:


Name: Tom

Name: Sam
Company: Oracle

Однак зворотне не вірно: ми не можемоо присвоїти змінній похідного класу посилання на об'єкт базового класу, наприклад, так:


Employee sam = new Person("Sam");

І це логічно: не всі люди працюють і є працівниками певного підприємства.

Поліморфізм у параметрах і типух, що повертаються

Те, що будь-який об'єкт похідного класу також розглядається як об'єкт базового класу, дозволяє параметру базового типу передавати об'єкти похідних класів. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		printPerson(tom);

        System.out.println();

		Employee sam = new Employee("Sam", "Oracle");
		printPerson(sam);
	}

    static void printPerson(Person person){

        person.print();
    }
}
class Person {
    
    private String name;
   
    Person(String name){ this.name = name; }
 
    void print(){
		
		System.out.println("Name: " + name);
	}
}

class Employee extends Person{

    private String company;
    
    Employee(String name, String company) {
    
        super(name);
        this.company = company;
    }
    @Override
    void print(){
        
        super.print();
        System.out.println("Company: " + company);
    }
}

У даному випадку в класе Program метод printPerson приймає об'єкт типу Person:


static void printPerson(Person person){

    person.print();
}

Але оскільки об'єкти похідного типу Employee також є об'єктами типу Person, ми можемоо передати ці об'єкти у цей метод:


Employee sam = new Employee("Sam", "Oracle");
printPerson(sam); // параметру типу Person передаємо об'єкт Employee

Те саме стосується повернення об'єкту з методу: якщо тип, що повертається, представляє базовий клас, то ми можемоо повернути з методу об'єкт похідного класу. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		Person person1 = defaultPerson(0);
		person1.print();

        System.out.println();

		Person person2 = defaultPerson(1);
		person2.print();
	}

    static Person defaultPerson(int type){

        if(type == 0){
            return new Person("Tom");
        }
        return new Employee("Tom", "OOO Undefined");
    }
}
class Person {
    
    private String name;
   
    Person(String name){ this.name = name; }
 
    void print(){
		
		System.out.println("Name: " + name);
	}
}

class Employee extends Person{

    private String company;
    
    Employee(String name, String company) {
    
        super(name);
        this.company = company;
    }
    @Override
    void print(){
        
        super.print();
        System.out.println("Company: " + company);
    }
}

У даному випадку в класе Program статический метод defaultPerson повертає значення типу Person:


static Person defaultPerson(int type){

    if(type == 0){
        return new Person("Tom");
    }
    return new Employee("Tom", "OOO Undefined");
}

Але в залежності від переданого параметра це може бути як об'єкт типу Person, так і об'єкт типу Employee (який також є об'єктом типу Person)
