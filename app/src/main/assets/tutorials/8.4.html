Завершення та переривання потіку

Останнє оновлення: 16.10.2025

Примеры потіков в прошлых статьях представляли потік як післядовательный набор операцій. То есть потік по сути представлял післядовательность инструкций, которые післядовательно выполнялись одна за другой. А після виконання післядней операції завершался и потік. Однак нередко имїїт место и другая организация потіка в виде бесконечного цикла. Наприклад, потік сервера в бесконечном цикле прослушивает визначенийный порт на предмет получения данных. И в цем случае мы також можемо предусмотреть механизм завершення потіка.

Завершення потіка

Распространенный способ завершення потіка представляє опрос логической змінної. И якщо она равна, наприклад, false, то потік завершує бесконечный цикл и заканчивает свое виконання.

Визначимо следующий клас потіка:


class MyTask implements Runnable {
      
    private boolean isActive;
      
    void disable(){ isActive=false; }
      
    MyTask(){ isActive = true; }
      
    public void run(){
          
        String taskName = Thread.currentThread().getName();

        System.out.printf("%s started... \n", taskName);

        int counter = 1; // счетчик циклов

        while(isActive){

            System.out.println("Loop " + counter++);

            try{
                Thread.sleep(400);
            }
            catch(InterruptedException _){
                
                System.out.printf("%s interrupted... \n", taskName);
            }
        }
        System.out.printf("%s finished... \n", taskName);
    }
}

Змінна isActive вказує на активность потіка: поки эта змінна равна true, потік буде продолжать працювати. Но за допомогою методу disable() мы можемо сбросить состояние цей змінної в false и таким чином завершить потік.

Теперь використовуємо цей клас:


class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        // створюємо и запускуємо потік
        var myTask = new MyTask();
        new Thread(myTask,"MyTask").start();
            
        try{
            Thread.sleep(1100);
                
            myTask.disable();  // завершаем потік
            
            Thread.sleep(1000);
        }
        catch(InterruptedException e){
            System.out.println("Main thread interrupted...");
        }
        System.out.println("Main thread finished...");
    }
}
class MyTask implements Runnable {
      
    private boolean isActive;
      
    void disable(){ isActive=false; }
      
    MyTask(){ isActive = true; }
      
    public void run(){
          
        String taskName = Thread.currentThread().getName();

        System.out.printf("%s started... \n", taskName);

        int counter=1; // счетчик циклов
        while(isActive){
            System.out.println("Loop " + counter++);
            try{
                Thread.sleep(400);
            }
            catch(InterruptedException _){
                
                System.out.printf("%s interrupted... \n", taskName);
            }
        }
        System.out.printf("%s finished... \n", taskName);
    }
}

Итак, вначале запускуєся дочерний потік:


var myTask = new MyTask();
new Thread(myTask,"MyTask").start();

Затем на 1100 миллисекунд останавливаем главный потік "Main thread" и потом вызываем метод disable(), який переключает у потоці флаг isActive, и дочерний потік завершуєся.

myTask.disable();  // завершаем потік

Консольный вивід програми:


Main thread started...
MyTask started... 
Loop 1
Loop 2
Loop 3
MyTask finished... 
Main thread finished...

Прерывание потіка и метод interrupt

В мові Java нельзя принудительно завершить потік. Однак за допомогою методу interrupt() можна направить запроса на прерывание потіка. При вызове методу interrupt() для потіка встановлюєся статус INTERRUPTED.

При выполнении работы за допомогою методу isInterrupted() потік может периодически проверять, був ли он прерван, и, якщо прерван, завершать выполняемую работу:


// поки потік не прерван
while (!Thread.currentThread().isInterrupted()){

    // делаем деяку работу
}

Однак, якщо потік заблокирован, он не может проверить статус за допомогою вызова isInterrupted(). Такая ситуация, наприклад, возникает, когда метод interrupt() вызывается для потіка, який блокуєся вызовом sleep() (а також ряд других, наприклад, методом wait()). В цем случае блокирующий вызов (наприклад, sleep()) завершуєся винятокм InterruptedException. Існують блокирующие вызовы ввода-вивіда, которые не можуть быть прерваны. Тогда при завершении операції ввода-вивіда також генеруєся виняток InterruptedException. И в цем случае мы можемо в коді потіка обробляти виняток:


Runnable r = () -> {   // действия потіка
    try  {

        // якщо действия в цикле, в качестве условия проверяем isInterrupted()
        while (!Thread.currentThread().isInterrupted()) {
            
            // выполняемые действия
        }
    }
    catch (InterruptedException ex){

        // обробляємо виняток прерывания потіка
    }

    // якие-то завершающие действия при необхідности
};

Хотя, якщо потік прерван, он не обязан завершаться - он может продолжать працювати. Но гораздо чаще потік просто интерпретирует прерывание як запрос на завершення. Наприклад, проэмулируем данную ситуацию:


class MyTask implements Runnable {

    public void run(){
         
        String name = Thread.currentThread().getName();  // отримуємо ім'я текущего потіка

        try{
            
            System.out.println(name + " started...");

            while(!Thread.currentThread().isInterrupted()){

                System.out.println(name + " works");

                Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
            }
        }
        catch(InterruptedException ex){

            System.out.println(ex.getMessage());
        }
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        // определяем потік
        var t = new Thread(new MyTask(), "MyTask");
        // запускуємо потік
        t.start();
        
        try{
            // делаем небольшую задержку, чтобы дочерний потік MyTask успел немного отпрацювати
            Thread.sleep(2000);
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }

        // прерываем потік
        t.interrupt();
        
        System.out.println("Main thread finished...");
    }
}

В методі run() класа MyTask в цикле while проверяем результат методу isInterupted():


try{
            
    .........................

    while(!Thread.currentThread().isInterrupted()){

        System.out.println(name + " works");
        Thread.sleep(1000); // приостанавливаем потік на 1000 миллисекунд
    }
}
catch(InterruptedException ex){

    System.out.println(ex.getMessage());
}

В цикле делаем задержку на 1 секунду за допомогою Thread.sleep(). Однак якщо во час цей задержки во вне буде вызван метод interrupt() для цього потіка, то мы столкнемся с винятокм InterruptedException. И за допомогою конструкции try..catch перехватываем и обробляємо це виняток.

В методі main запускуємо потік MyTask, приостанавливаем главный потік, чтобы MyTask успел немного отпрацювати, и далїї вызыаем у MyTask метод interrupt():


var t = new Thread(new MyTask(), "MyTask");
t.start();
        
try{
    Thread.sleep(2000);
}
catch(InterruptedException ex){
    System.out.println(ex.getMessage());
}

// прерываем потік
t.interrupt();

В итоге при вызове методу interrupt() во час приостановки MyTask обработает виняток InterruptedException и завершит работу:

MyTask started...
MyTask works
MyTask works
Main thread finished...
sleep interrupted
MyTask finished...
