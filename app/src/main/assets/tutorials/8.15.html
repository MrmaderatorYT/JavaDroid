Координація виконання асинхронних завдань

Останнє оновлення: 20.10.2025

Нередко требуется запустить декілька завдань одновременно и отримати результат первой успешно выполненной завданьі або об'єднати результати всех выполненных завдань. Для цього клас ExecutorService надає два методу: inVokeAll() и invokeAny()

invokeAll. Очікування завершення всей группы завдань

Метод invokeAll() надсилає на виконання все завданьі из колекції об'єктов Callable, блокує их виконання до завершення всех завдань и повертає список об'єктов Future, которые представляютьь результати всех завдань, в том же порядке, в котором эти завданьі буви отправлены. Соответственно в соответствии с порядком можна совместить завданьі и их результати.

Недостатком данного методу в некоторых ситуациях может быть то, что он ждет завершення всех завдань, даже якщо якие-то из них вже завершились с ошибкой.

Цет метод також имїїт две версии:


<T> List<Future<T>> invokeAll(Collection<? extends Callable<T>> tasks)
<T> List<Future<T>> invokeAll(Collection<? extends Callable<T>> tasks, long timeout, TimeUnit unit)

Первая версия приймає колекцію завдань, которые треба выполнить, и повертає список Future-об'єктов со статусом завдань и их результатами.

Вторая версия також дополнительно приймає параметри тайм-аута, до истечения которого повинні быть выполнены все завданьі.

Рассмотрим на примере вычисления факториалов в следующей програме:


import java.util.concurrent.*;
import java.util.List;
import java.util.ArrayList;

record FactorialResult(int number, int factorial){}

class FactorialTask implements Callable<FactorialResult>{

    private int number;
    FactorialTask(int number) {  this.number = number; }

    public FactorialResult call() throws Exception{

        if(number < 1) throw new Exception("Incorrect number: " + number + ". Number must be greater than 0");

        int result = 1;
        int n = number;
        while(n > 0)  result *= n--; 

        return new FactorialResult(number, result);  // повертаємо результат завданьі
    };
}

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {

            List<Callable<FactorialResult>> tasks = new ArrayList<Callable<FactorialResult>>();
            // додаємо декілька завдань
            for(int i = -1; i < 4; i++) tasks.add(new FactorialTask(i));

            // вызываем и ждем завершення всех завдань
            List<Future<FactorialResult>> results = executor.invokeAll(tasks);

            // отримуємо результати
            for (Future<FactorialResult> result : results) {
                if (result.state() == Future.State.SUCCESS){
                    FactorialResult factorialResult =  result.resultNow();
                    System.out.printf("Factorial of %d:  %d\n", 
                            factorialResult.number(), factorialResult.factorial());
                }
                else{
                    var ex = result.exceptionNow();
                    System.out.println("Factorial exception: " + ex.getMessage());
                }
            }

        } // здесь вызывается метод executor.close() 
        catch(Exception ex){
            System.out.println(ex.getMessage());
        }
        System.out.println("Main thread finished...");
    }
}

Здесь для представления результата завданьі визначений клас FactorialResult:

record FactorialResult(int number, int factorial){}

Зднсь поле number представляє число, для которого вычисляется факториал, а поле factorial - сам вычисленный факториал числа.

Для представления завданьі визначений клас FactorialTask, який реалізує інтерфейс Callable - в методі call вычисляет факториал числа, послученного через конструктор, и повертає результа - об'єкт FactorialResult.


public FactorialResult call() throws Exception{

    if(number < 1) throw new Exception("Incorrect number: " + number + ". Number must be greater than 0");

    int result = 1;
    int n = number;
    while(n > 0)  result *= n--; 

    return new FactorialResult(number, result);  // повертаємо результат завданьі
};

Допустим, нам треба вычислить факториалы от -1 до 4 (для теста выбираем ряд чисел заведо некорректных). Для цього створюємо об'єкт ExecutorService, який буде выполнять завданьі на виртуальных потіках, и собственно список завдань Callable:


try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {

    List<Callable<FactorialResult>> tasks = new ArrayList<Callable<FactorialResult>>();
    // додаємо декілька завдань
    for(int i = -1; i < 4; i++) tasks.add(new FactorialTask(i));

Затем вызываем все завданьі за допомогою методу executor.invokeAll() и ждем их завершення:

List<Future<FactorialResult>> results = executor.invokeAll(tasks);

На цей рядківе метод main приостанавливает виконання и ждет завершення всех завдань. И когда от них станут доступны результати, проходим в цикле по все результатам и обробляємо их:


for (Future<FactorialResult> result : results) {
    if (result.state() == Future.State.SUCCESS){
        FactorialResult factorialResult =  result.resultNow();
        System.out.printf("Factorial of %d:  %d\n", 
        factorialResult.number(), factorialResult.factorial());
    }
    else{
        var ex = result.exceptionNow();
        System.out.println("Factorial exception: " + ex.getMessage());
    }
}

Каждый результат представляє об'єкт Future, у которого за допомогою методу state() можна проверить состояние виконання: якщо завданьа выполнена успешно, то використовуємо метод resultNow() для получения непосредственно самого результата в виде об'єкта FactorialResult. Если виконання завершилось неудачно, то за допомогою методу exceptionNow() отримуємо возникшїї виняток и вивідим его сообщение на консоль.

Консольный вивід програми:


Main thread started...
Factorial exception: Incorrect number: -1. Number must be greater than 0
Factorial exception: Incorrect number: 0. Number must be greater than 0
Factorial of 1:  1
Factorial of 2:  2
Factorial of 3:  6
Main thread finished...

invokeAny. Очікування завершення одной завданьі их группы

Метод invokeAny() надсилає на виконання все завданьі из колекції об'єктов Callable и блокує виконання до тех пор, поки одна из них не завершится успешно, вернув свой результат. Цет метод имїїт две версии:

invokeAny. Очікування завершення одной завданьі их группы

Метод invokeAny() надсилає на виконання все завданьі из колекції об'єктов Callable и блокує виконання до тех пор, поки одна из них не завершится успешно, вернув свой результат. Цет метод имїїт две версии:


<T> T invokeAny(Collection<? extends Callable<T>> tasks)
<T> T invokeAny(Collection<? extends Callable<T>> tasks, long timeout, TimeUnit unit)

Первая версия методу приймає колекцію завдань, из которых треба завершить одну. Результатом методу є результат первой завершенной завданьі.

Вторая версия дополнительно приймає параметри тайм-аута - времени, до истечения которого повинна быть выполнена одна из завдань.

При цем все незавершенные завданьі отменяются. Если ни одна завданьа не бува выполнена успешно, генеруєся виняток ExecutionException.

Изменим предыдущий код - пусть теперь хотя бы одна из завдань по вычислению факториала буде выполнена:


import java.util.concurrent.*;
import java.util.List;
import java.util.ArrayList;

record FactorialResult(int number, int factorial){}
class FactorialTask implements Callable<FactorialResult>{

    private int number;
    FactorialTask(int number) {  this.number = number; }

    public FactorialResult call() throws Exception{

        if(number < 1) throw new Exception("Incorrect number: " + number + ". Number must be greater than 0");

        int result = 1;
        int n = number;
        while(n > 0)  result *= n--; 

        return new FactorialResult(number, result);  // повертаємо результат завданьі
    };
}

class Program{

    public static void main(String[] args) {
         
        System.out.println("Main thread started...");

        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            List<Callable<FactorialResult>> tasks = new ArrayList<Callable<FactorialResult>>();
            // додаємо декілька завдань
            for(int i = -1; i < 4; i++) tasks.add(new FactorialTask(i));

            // вызываем завданьі и ждем, поки хотя бы одна из них не завершенится
            FactorialResult factorialResult = executor.invokeAny(tasks);
            System.out.printf("Factorial of %d:  %d\n", 
                    factorialResult.number(), factorialResult.factorial());

        }
        catch(Exception ex){
            System.out.println(ex.getMessage());
        }
        System.out.println("Main thread finished...");
    }
}

Большая часть логики здесь та же, что и в предыдущем примере. Отличие лише в том что ждем виконання одной завданьі за допомогою invokeAny():


FactorialResult factorialResult = executor.invokeAny(tasks);
System.out.printf("Factorial of %d:  %d\n",  factorialResult.number(), factorialResult.factorial());

И у даному випадку вивід может быть наступним (консольный вивід не детерминирован):


Main thread started...
Factorial of 3:  6
Main thread finished...

Если при выполнении завданьі возникнет ошибка, то буде сгенерировано виняток, которое буде обработано в блоке catch:


Main thread started...
java.lang.Exception: Incorrect number: -1. Number must be greater than 0
Main thread finished...
