Створення та застосування анотацій

Останнє оновлення: 01.11.2025

Створення собственных анотацій

Хотя в мові Java, во фреймворках на цем языке есть различные анотації для разных ситуаций, но Java також дозволяє создавать свои собственные анотації. Каждая анотація визначаєся за допомогою ​​інтерфейса анотації - за допомогою ключевого слова @interface, після которого вказуєся ім'я анотації. @interface створює інтерфейс Java:


// оголошення анотації під названием MyAnnotation
@interface MyAnnotation {

}

Після імені анотації (як и в случае с інтерфейсами) идут фигурные скобки, всередині которых размещаются елементы анотації. У даному випадку анотація називається "MyAnnotation" и не містить нияких елементов.

Все інтерфейсы анотацій неявно расширяют інтерфейс java.lang.annotation.Annotation, який є обычным інтерфейсом. Однак расширять інтерфейсы анотацій або реализовать их в своих класах нельзя. Вместо цього инструменты обработки исходного коду и виртуальная машина при необхідности сами автоматически генерують необходимые прокси-класы и об'єкты.

При створенні анотації часто використовуються так называемые мета-анотації (анотації для анотацій):

  • @Retention: визначає политику хранения.

  • @Target: вказує, к яким елементам коду можна застосовувати анотацію (клас, метод, поле, параметр и т.д.).

  • @Documented: вказує, что анотація повинна быть включена в генерируемую Javadoc-документацию.

  • @Inherited: вказує, что анотація, примененная к класу, буде наследоваться его підкласами.

  • @Repeatable: дозволяє застосовувати одну и ту же анотацію к одному елементу декілька раз.

Далїї мы розглянемо некоторые из цих мета-анотацій. А поки посмотрим, як мы можемо пріменіть нашу анотацію.


@interface MyAnnotation{}

@MyAnnotation
class Thing{
    @MyAnnotation String name;
}

Здесь анотація MyAnnotation застосовується к оголошенням - к класу Thing и его полю name. Причем анотація сама по себе и її застосування ничего не делают, мы можемо використовувати цей клас як обычный клас:


// оголошуємо анотацію
@interface MyAnnotation{}

// застосовуємо анотацію к класу Thing
@MyAnnotation
class Thing{ }

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Thing thing = new Thing();
        System.out.println(thing);
    }
}

Варто відзначити, что по умолчанию анотація может використовуватися с любыми оголошеннями, но не с параметрами типу (в обобщениях) и використанням типів. Для болїї детальной настройки применения анотації застосовується мета-анотація @Target.

@Target

Мета-анотація @Target визначає, к яким елементам коду можна застосовувати анотацію. Ее значення представляє масив об'єктов ElementType, которые и визначають елементы для применения анотації:

  • ElementType.ANNOTATION_TYPE: оголошення типів анотацій

  • ElementType.MODULE: модулі

  • ElementType.PACKAGE: пакети

  • ElementType.TYPE: класы (включая перечисления) и інтерфейсы (включая типи анотацій)

  • ElementType.METHOD: методи

  • ElementType.CONSTRUCTOR: конструктори

  • ElementType.FIELD: поля (включая константи перечисления)

  • ElementType.PARAMETER: параметри методу або конструктора

  • ElementType.RECORD_COMPONENT: компоненты record-класов

  • ElementType.LOCAL_VARIABLE: локальные змінні

  • ElementType.TYPE_PARAMETER: параметри типи

  • ElementType.TYPE_USE: використання типу

Наприклад:


@Target({ElementType.FIELD, ElementType.RECORD_COMPONENT, ElementType.TYPE})
@interface MyAnnotation

Здесь анотація MyAnnotation может применться к полям, типум (класам и інтерфейсам) и компонентам класов record. Наприклад:


@MyAnnotation
class Thing{
    @MyAnnotation String name;
}

Но, наприклад, к методум и другим компонентам такая анотація застосовуватися не может.

Анотація без ограничения @Target может використовуватися с любыми оголошеннями, но не с параметрами типу и використанням типів.

Елементы анотацій

Анотація может містятьь елементы, которые фактически є методуми. Причем эти методи не можуть иметь параметрів, не можуть быть обобщенными, а також не можуть в оголошенийии містятьь выражения throws. Наприклад:


@interface MyAnnotation {

    // елементы анотації
    String value();
    int count();
}

У даному випадку анотація MyAnnotation имїїт два елемента: value (представляє рядок) и count (представляє число).

При применении анотації нам треба передать цим елементам некоторые значення:


@interface MyAnnotation {

    // елементы анотації
    String value();
    int count();
}

@MyAnnotation(value="Thing", count=1)
class Thing{ }

Значення для елементов анотацій передаються після названия анотації в скобках через запятую в виде название_елемента = значення_елемента. В якой-то степени це похоже на вызов конструктора класа.

Значення по умолчанию для елементов анотацій

Елементы анотацій можуть иметь значення по умолчанию. Для его установки після методу и перед значенням по умолчанию вказуєся ключевое слово default. Наприклад:


@interface MyAnnotation {

    // елементы анотації со значеннями по умолчанию
    String value() default "default value";
    int count() default 0;
}

Если елементы анотації имеют значення по умолчанию, то при применении анотації для цих елементов можна не передавать значення:


@interface MyAnnotation {

    // елементы анотації со значеннями по умолчанию
    String value() default "DefaultValue";
    int count() default 0;
}

@MyAnnotation(value="Type A", count=2)
class TypeA{ }

@MyAnnotation(value="Type B")  // count = 0
class TypeB{ }

@MyAnnotation(count=1)      // value = DefaultValue
class TypeC{ }

@MyAnnotation               // value = DefaultValue, count = 0
class TypeD{ }

Если елемент анотації представляє масив, то в качестве значення по умолчанию вказуються всередині фигурных скобок:


@interface MyAnnotation {
    String[] addresses() default {"address1@humail.com", "address2@humail.com"};
    String[] posts() default {};    // порожній масив
}

Значення по умолчанию не сохраняются вместе с аннотацией, а вычисляются динамически.

@Retention

Мета-анотація @Retention вказує, где можна отримати доступ к анотації, то есть по сути то, як долго анотація "живет". Эта анотація может принимать одно из трех значений перечисления RetentionPolicy:

  • RetentionPolicy.SOURCE: анотація існує лише в исходном коді и отбрасывается компилятором. Використовується, наприклад, для инструментов анализа коду або кодогенерации

  • RetentionPolicy.CLASS: анотація сохраняется в файле .class (в байт-коді), но недоступна во час виконання. Це значення по умолчанию.

  • RetentionPolicy.RUNTIME: анотація сохраняется в файле .class, но в отличие от предыдущего варианта анотація доступна во час виконання через рефлексию (reflection). (Це самый распространенный тип для фреймворков, наприклад, Spring або JUnit, которые анализируют класы во час работы програми)

Пример применения мета-анотації:


@Retention(RetentionPolicy.RUNTIME)     // Анотація доступна во час виконання
@Target(ElementType.TYPE)             // Анотація застосовується к типум
@interface MyAnnotation {

    String value() default "DefaultValue";
}
