Створення та застосування анотацій

Останнє оновлення: 01.11.2025

Створення собственных аннотаций

Хотя в мові Java, во фреймворках на цем языке есть различные аннотации для разных ситуаций, но Java також дозволяє создавать свои собственные аннотации. Каждая аннотация определяется за допомогою ​​інтерфейса аннотации - за допомогою ключевого слова @interface, после которого указывается имя аннотации. @interface створює інтерфейс Java:


// оголошення аннотации под названием MyAnnotation
@interface MyAnnotation {

}

После имени аннотации (як и в случае с інтерфейсами) идут фигурные скобки, внутри которых размещаются элементы аннотации. В данном случае аннотация називається "MyAnnotation" и не містить нияких элементов.

Все інтерфейсы аннотаций неявно расширяют інтерфейс java.lang.annotation.Annotation, який є обычным інтерфейсом. Однак расширять інтерфейсы аннотаций або реализовать их в своих классах нельзя. Вместо цього инструменты обработки исходного кода и виртуальная машина при необхідности сами автоматически генерують необходимые прокси-классы и об'єкты.

При создании аннотации часто використовуються так называемые мета-аннотации (аннотации для аннотаций):

  • @Retention: определяет политику хранения.

  • @Target: указывает, к яким элементам кода можна застосовувати аннотацию (класс, метод, поле, параметр и т.д.).

  • @Documented: указывает, что аннотация повинна быть включена в генерируемую Javadoc-документацию.

  • @Inherited: указывает, что аннотация, примененная к классу, буде наследоваться его подклассами.

  • @Repeatable: дозволяє застосовувати одну и ту же аннотацию к одному элементу декілька раз.

Далїї мы розглянемо некоторые из цих мета-аннотаций. А поки посмотрим, як мы можемо применить нашу аннотацию.


@interface MyAnnotation{}

@MyAnnotation
class Thing{
    @MyAnnotation String name;
}

Здесь аннотация MyAnnotation застосовується к оголошенням - к классу Thing и его полю name. Причем аннотация сама по себе и її применение ничего не делают, мы можемо використовувати цей класс як обычный класс:


// оголошуємо аннотацию
@interface MyAnnotation{}

// применяем аннотацию к классу Thing
@MyAnnotation
class Thing{ }

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Thing thing = new Thing();
        System.out.println(thing);
    }
}

Варто відзначити, что по умолчанию аннотация может використовуватися с любыми оголошеннями, но не с параметрами типу (в обобщениях) и использованием типов. Для болїї детальной настройки применения аннотации застосовується мета-аннотация @Target.

@Target

Мета-аннотация @Target определяет, к яким элементам кода можна застосовувати аннотацию. Ее значение представляє массив об'єктов ElementType, которые и определяют элементы для применения аннотации:

  • ElementType.ANNOTATION_TYPE: оголошення типов аннотаций

  • ElementType.MODULE: модули

  • ElementType.PACKAGE: пакеты

  • ElementType.TYPE: классы (включая перечисления) и інтерфейсы (включая типы аннотаций)

  • ElementType.METHOD: методы

  • ElementType.CONSTRUCTOR: конструкторы

  • ElementType.FIELD: поля (включая константы перечисления)

  • ElementType.PARAMETER: параметры метода або конструктора

  • ElementType.RECORD_COMPONENT: компоненты record-классов

  • ElementType.LOCAL_VARIABLE: локальные переменные

  • ElementType.TYPE_PARAMETER: параметры типы

  • ElementType.TYPE_USE: использование типу

Например:


@Target({ElementType.FIELD, ElementType.RECORD_COMPONENT, ElementType.TYPE})
@interface MyAnnotation

Здесь аннотация MyAnnotation может применться к полям, типум (классам и інтерфейсам) и компонентам классов record. Например:


@MyAnnotation
class Thing{
    @MyAnnotation String name;
}

Но, наприклад, к методам и другим компонентам такая аннотация застосовуватися не может.

Аннотация без ограничения @Target может використовуватися с любыми оголошеннями, но не с параметрами типу и использованием типов.

Элементы аннотаций

Аннотация может містятьь элементы, которые фактически є методами. Причем эти методы не можуть иметь параметров, не можуть быть обобщенными, а також не можуть в оголошенийии містятьь выражения throws. Например:


@interface MyAnnotation {

    // элементы аннотации
    String value();
    int count();
}

В данном случае аннотация MyAnnotation имїїт два элемента: value (представляє строку) и count (представляє число).

При применении аннотации нам треба передать цим элементам некоторые значения:


@interface MyAnnotation {

    // элементы аннотации
    String value();
    int count();
}

@MyAnnotation(value="Thing", count=1)
class Thing{ }

Значения для элементов аннотаций передаются после названия аннотации в скобках через запятую в виде название_элемента = значение_элемента. В якой-то степени це похоже на вызов конструктора класса.

Значения по умолчанию для элементов аннотаций

Элементы аннотаций можуть иметь значение по умолчанию. Для его установки после метода и перед значением по умолчанию указывается ключевое слово default. Например:


@interface MyAnnotation {

    // элементы аннотации со значениями по умолчанию
    String value() default "default value";
    int count() default 0;
}

Если элементы аннотации имеют значения по умолчанию, то при применении аннотации для цих элементов можна не передавать значения:


@interface MyAnnotation {

    // элементы аннотации со значениями по умолчанию
    String value() default "DefaultValue";
    int count() default 0;
}

@MyAnnotation(value="Type A", count=2)
class TypeA{ }

@MyAnnotation(value="Type B")  // count = 0
class TypeB{ }

@MyAnnotation(count=1)      // value = DefaultValue
class TypeC{ }

@MyAnnotation               // value = DefaultValue, count = 0
class TypeD{ }

Если элемент аннотации представляє массив, то в качестве значения по умолчанию указываются внутри фигурных скобок:


@interface MyAnnotation {
    String[] addresses() default {"address1@humail.com", "address2@humail.com"};
    String[] posts() default {};    // пустой массив
}

Значения по умолчанию не сохраняются вместе с аннотацией, а вычисляются динамически.

@Retention

Мета-аннотация @Retention указывает, где можна отримати доступ к аннотации, то есть по сути то, як долго аннотация "живет". Эта аннотация может принимать одно из трех значений перечисления RetentionPolicy:

  • RetentionPolicy.SOURCE: аннотация существует лише в исходном коде и отбрасывается компилятором. Використовується, наприклад, для инструментов анализа кода або кодогенерации

  • RetentionPolicy.CLASS: аннотация сохраняется в файле .class (в байт-коде), но недоступна во час виконання. Це значение по умолчанию.

  • RetentionPolicy.RUNTIME: аннотация сохраняется в файле .class, но в отличие от предыдущего варианта аннотация доступна во час виконання через рефлексию (reflection). (Це самый распространенный тип для фреймворков, наприклад, Spring або JUnit, которые анализируют классы во час работы программы)

Пример применения мета-аннотации:


@Retention(RetentionPolicy.RUNTIME)     // Аннотация доступна во час виконання
@Target(ElementType.TYPE)             // Аннотация застосовується к типум
@interface MyAnnotation {

    String value() default "DefaultValue";
}
