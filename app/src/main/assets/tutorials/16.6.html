Пошук та виклик зовнішніх функцій

Останнє оновлення: 21.12.2025

Для поиска библиотечных функций языка Си и их подготовке для вызова в API FFM застосовується класс java.lang.foreign.Linker. Для создания линкера, связанного с текущей нативной платформой ОС застосовується статический метод Linker.nativeLinker():

Linker linker = Linker.nativeLinker();

Фактически, часть "поиска" делегируется экземпляру SymbolLookup, який определяется за допомогою метода defaultLookup() линкера:


Linker linker = Linker.nativeLinker();
SymbolLookup stdlib = linker.defaultLookup();

Библиотеки, о которых знает поиск по умолчанию, зависят от платформы. В Linux це libc и libm. Цет поиск достаточен для таких известных функций, як printf и malloc.

Для использования со специализированной библиотекой (или своей) в метод SymbolLookup.libraryLookup() передаєся путь к файлу библиотеки (в виде строки або об'єкта Path) и арена:


SymbolLookup somelib = SymbolLookup.libraryLookup("/lib/x86_64-linux-gnu/libsxxx.so", arena);

Метод libraryLookup загружает библиотеку. А когда арена закрывается, библиотека выгружается.

Поиск функций

Получив об'єкт SymbolLookup, мы можемо перейти к поиску функций. Для цього SymbolLookup надає два метода:

  • Optional<MemorySegment> find(String name): повертає адрес символа с заданным именем

  • MemorySegment findOrThrow(String name): повертає адрес символа с заданным именем або генерує исключение, якщо функция не найдена

В обоих случаях указатель на полученную функцию повертаєся через об'єкт MemorySegment, работа с которым бува рассмотрена в предыдущеихз статьях. Наприклад, поиск функції printf:


Linker linker = Linker.nativeLinker();
SymbolLookup stdlib = linker.defaultLookup();
MemorySegment printf_addr = stdlib.findOrThrow("printf");

Дескриптор функції/метода

Получив адрес вызываемой функції, можн создать дескриптор метода для її вызова. Дескриптор метода - це об'єкт, через який можна передавать аргументы, вызывать метод и получать возвращаемое значение.

Дескриптор метода представляє тип MethodHandle. Для его создания линкер - об'єкт Linker надає метода downcallHandle()

  • downcallHandle(FunctionDescriptor function, Linker.Option... options)

    Створює дескриптор метода, используемый для вызова внешней функції с заданной сигнатурой

  • downcallHandle(MemorySegment address, FunctionDescriptor function, Linker.Option... options)

    Створює дескриптор метода, используемый для вызова внешней функції с заданной сигнатурой и адресом

Цет метод приймає до трех параметров:

  • address: сегмент памяти, базовый адрес которого совпадает с адресом вызываемой внешней функції

  • function: дескриптор вызываемой внешней функції

  • options: параметры линкера, связанные с цим запросом на компоновку

Ключевой здесь параметр - це десткриптор функції, представленный інтерфейсом FunctionDescriptor. Для определения цього об'єкта обычно застосовується статический метод FunctionDescriptor.of

FunctionDescriptor.of(MemoryLayout resLayout, MemoryLayout... argLayouts)

Его первый параметр представляє компоновку памяти типу результата функції, а второй параметр - компоновки памяти для параметров функції.

Наприклад, получим десткриптор метода для функції printf:


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) {

        // получаем линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // получаем стандартную библиотеку
        SymbolLookup stdlib = linker.defaultLookup();
        // получаем адрес функції printf
        MemorySegment printf_addr = stdlib.findOrThrow("printf");
        // дескриптор функції printf
        FunctionDescriptor printf_desc = FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS);
        // дескриптор метода для функції printf
        MethodHandle printf = linker.downcallHandle(printf_addr, printf_desc);
        System.out.println(printf);     // MethodHandle(MemorySegment)int
    }
}

При передаче указателя в функцию C або получении результата указателя використовується ValueLayout.ADDRESS. Тип блока памяти не имїїт значения. Крім того, необхідно точно указать тип возвращаемого значения функції и типы параметров. Если предоставить неверную информацию, виртуальная машина может завершиться с ошибкой.

Так, функция printf в C повертає целое число int, а в качестве параметра приймає строку (по сути указатель на символы), поцему її дескриптор имїїт форму

FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS)

Вызов функції

Получив дескриптор метода, наконец мы можемо вызвать функцию. Для цього у типу MethodHandle застосовується метод invoke()

public final Object invoke(Object... args) throws Throwable

В качестве аргументов в вызов метода передаются значения для параметров внешней функції. ПРи цем параметры повинні соответствовать типум языка Си, а не Java. Наприклад, мы не можемо написать так:

printf.invoke("Hello");

Хотя, казалось бы функция printf() приймає строку. Однак це повинна быть строка языка Си. Соответственно нам треба сконвертировать строку Java в строку Си (UTF-8). Пример такого преобразования и вызова функції printf:


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // получаем линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // получаем стандартную библиотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // получаем адрес функції printf
        MemorySegment printf_addr = stdlib.findOrThrow("printf");
        // дескриптор функції printf
        FunctionDescriptor printf_desc = FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS);
        // дескриптор метода для функції printf
        MethodHandle printf = linker.downcallHandle(printf_addr, printf_desc);

        // создаем арену
        try (Arena arena = Arena.ofConfined()) {
            
            // строка для вывода
            String str = "Hello \n";
            // преобразуем строку Java в строку UTF-8
            MemorySegment str_seg = arena.allocateFrom(str);

            // вызываем функцию printf и получаем результат
            int char_count = (int) printf.invoke(str_seg);  // Hello 
            
            System.out.println("Длина строки: " + char_count + " символов");  // Длина строки: 18 символов
        }
    }
}

Підзвращаемый тип void

Отдельно варто сказать про функції, який повертають void. В цем случае для определения дескриптора функції застосовується метод FunctionDescriptor.ofVoid


FunctionDescriptor ofVoid(MemoryLayout... argLayouts)

В качестве параметров методу також передаются значения, которые передаются вызываемой внешней функції. Указатель на void (void*), як и все указатели, представляють тип ValueLayout.ADDRESS. Наприклад, получим дескрипторы методів для функций malloc (выделение памяти) и free (освобождение памяти)


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // получаем линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // получаем стандартную библиотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // получаем адрес функції malloc
        MemorySegment mallocAddr = stdlib.findOrThrow("malloc");
        // void* malloc(size_t)
        MethodHandle mallocHandle = linker.downcallHandle(mallocAddr, FunctionDescriptor.of(ValueLayout.ADDRESS, ValueLayout.JAVA_LONG));
        System.out.println(mallocHandle);   // MethodHandle(long)MemorySegment

        // получаем адрес функції free
        MemorySegment freeAddr = stdlib.findOrThrow("free");
        // void free(void *ptr)
        MethodHandle freeHandle = linker.downcallHandle(freeAddr, FunctionDescriptor.ofVoid(ValueLayout.ADDRESS));
        System.out.println(freeHandle);     // MethodHandle(MemorySegment)void

    }
}
