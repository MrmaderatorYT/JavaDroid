Пошук та виклик зовнішніх функцій

Останнє оновлення: 21.12.2025

Для поиска библиотечных функцій языка Си и их підготовке для вызова в API FFM застосовується клас java.lang.foreign.Linker. Для створенняя линкера, связанного с текущей нативной платформой ОС застосовується статический метод Linker.nativeLinker():

Linker linker = Linker.nativeLinker();

Фактически, часть "поиска" делегируется экземпляру SymbolLookup, який визначаєся за допомогою методу defaultLookup() линкера:


Linker linker = Linker.nativeLinker();
SymbolLookup stdlib = linker.defaultLookup();

Бібліотеки, о которых знает поиск по умолчанию, зависят от платформы. В Linux це libc и libm. Цет поиск достаточен для таких известных функцій, як printf и malloc.

Для использования со специализированной библиотекой (или своей) в метод SymbolLookup.libraryLookup() передаєся путь к файлу бібліотеки (в виде рядки або об'єкта Path) и арена:


SymbolLookup somelib = SymbolLookup.libraryLookup("/lib/x86_64-linux-gnu/libsxxx.so", arena);

Метод libraryLookup загружает бібліотеку. А когда арена закрывается, бібліотека выгружается.

Поиск функцій

Получив об'єкт SymbolLookup, мы можемо перейти к поиску функцій. Для цього SymbolLookup надає два методу:

  • Optional<MemorySegment> find(String name): повертає адрес символа с заданным именем

  • MemorySegment findOrThrow(String name): повертає адрес символа с заданным именем або генерує виняток, якщо функція не найдена

В обоих случаях указатель на полученную функцію повертаєся через об'єкт MemorySegment, работа с которым бува рассмотрена в предыдущеихз статьях. Наприклад, поиск функції printf:


Linker linker = Linker.nativeLinker();
SymbolLookup stdlib = linker.defaultLookup();
MemorySegment printf_addr = stdlib.findOrThrow("printf");

Дескриптор функції/методу

Получив адрес вызываемой функції, можн створити дескриптор методу для її вызова. Дескриптор методу - це об'єкт, через який можна передавать аргументы, вызывать метод и получать повертаємоое значення.

Дескриптор методу представляє тип MethodHandle. Для его створенняя линкер - об'єкт Linker надає методу downcallHandle()

  • downcallHandle(FunctionDescriptor function, Linker.Option... options)

    Створює дескриптор методу, використовуємоый для вызова внешней функції с заданной сигнатурой

  • downcallHandle(MemorySegment address, FunctionDescriptor function, Linker.Option... options)

    Створює дескриптор методу, використовуємоый для вызова внешней функції с заданной сигнатурой и адресом

Цет метод приймає до трех параметрів:

  • address: сегмент памяти, базовый адрес которого збігається с адресом вызываемой внешней функції

  • function: дескриптор вызываемой внешней функції

  • options: параметри линкера, связанные с цим запросом на компоновку

Ключевой здесь параметр - це десткриптор функції, представленный інтерфейсом FunctionDescriptor. Для визначенийия цього об'єкта обычно застосовується статический метод FunctionDescriptor.of

FunctionDescriptor.of(MemoryLayout resLayout, MemoryLayout... argLayouts)

Его первый параметр представляє компоновку памяти типу результата функції, а второй параметр - компоновки памяти для параметрів функції.

Наприклад, получим десткриптор методу для функції printf:


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) {

        // отримуємо линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // отримуємо стандартную бібліотеку
        SymbolLookup stdlib = linker.defaultLookup();
        // отримуємо адрес функції printf
        MemorySegment printf_addr = stdlib.findOrThrow("printf");
        // дескриптор функції printf
        FunctionDescriptor printf_desc = FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS);
        // дескриптор методу для функції printf
        MethodHandle printf = linker.downcallHandle(printf_addr, printf_desc);
        System.out.println(printf);     // MethodHandle(MemorySegment)int
    }
}

При передаче указателя в функцію C або получении результата указателя використовується ValueLayout.ADDRESS. Тип блока памяти не имїїт значення. Крім того, необхідно точно указать тип повертаємоого значення функції и типи параметрів. Если предоставить неверную информацию, виртуальная машина может завершиться с ошибкой.

Так, функція printf в C повертає целое число int, а в качестве параметра приймає рядок (по сути указатель на символи), поцему її дескриптор имїїт форму

FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS)

Вызов функції

Получив дескриптор методу, наконец мы можемо вызвать функцію. Для цього у типу MethodHandle застосовується метод invoke()

public final Object invoke(Object... args) throws Throwable

В качестве аргументов в вызов методу передаються значення для параметрів внешней функції. ПРи цем параметри повинні соответствовать типум языка Си, а не Java. Наприклад, мы не можемо написать так:

printf.invoke("Hello");

Хотя, казалось бы функція printf() приймає рядок. Однак це повинна быть рядок языка Си. Соответственно нам треба сконвертировать рядок Java в рядок Си (UTF-8). Пример такого преобразования и вызова функції printf:


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // отримуємо линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // отримуємо стандартную бібліотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // отримуємо адрес функції printf
        MemorySegment printf_addr = stdlib.findOrThrow("printf");
        // дескриптор функції printf
        FunctionDescriptor printf_desc = FunctionDescriptor.of(ValueLayout.JAVA_INT, ValueLayout.ADDRESS);
        // дескриптор методу для функції printf
        MethodHandle printf = linker.downcallHandle(printf_addr, printf_desc);

        // створюємо арену
        try (Arena arena = Arena.ofConfined()) {
            
            // рядок для вивіда
            String str = "Hello \n";
            // преобразуем рядок Java в рядок UTF-8
            MemorySegment str_seg = arena.allocateFrom(str);

            // вызываем функцію printf и отримуємо результат
            int char_count = (int) printf.invoke(str_seg);  // Hello 
            
            System.out.println("Длина рядки: " + char_count + " символів");  // Длина рядки: 18 символів
        }
    }
}

Підзвращаемый тип void

Отдельно варто сказать про функції, який повертають void. В цем случае для визначенийия дескриптора функції застосовується метод FunctionDescriptor.ofVoid


FunctionDescriptor ofVoid(MemoryLayout... argLayouts)

В качестве параметрів методу також передаються значення, которые передаються вызываемой внешней функції. Указатель на void (void*), як и все указатели, представляютьь тип ValueLayout.ADDRESS. Наприклад, получим дескрипторы методів для функцій malloc (выделение памяти) и free (освобождение памяти)


import java.lang.foreign.*;
import java.lang.invoke.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // отримуємо линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // отримуємо стандартную бібліотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // отримуємо адрес функції malloc
        MemorySegment mallocAddr = stdlib.findOrThrow("malloc");
        // void* malloc(size_t)
        MethodHandle mallocHandle = linker.downcallHandle(mallocAddr, FunctionDescriptor.of(ValueLayout.ADDRESS, ValueLayout.JAVA_LONG));
        System.out.println(mallocHandle);   // MethodHandle(long)MemorySegment

        // отримуємо адрес функції free
        MemorySegment freeAddr = stdlib.findOrThrow("free");
        // void free(void *ptr)
        MethodHandle freeHandle = linker.downcallHandle(freeAddr, FunctionDescriptor.ofVoid(ValueLayout.ADDRESS));
        System.out.println(freeHandle);     // MethodHandle(MemorySegment)void

    }
}
