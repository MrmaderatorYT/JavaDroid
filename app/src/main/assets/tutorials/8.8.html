Клас Phaser

Останнє оновлення: 28.04.2018

Клас Phaser дозволяє синхронизировать потіки, представляющие отдельную фазу або стадию виконання общего действия. Phaser визначає об'єкт синхронизации, який ждет, поки не завершится визначенийная фаза. Затем Phaser переходит к следующей стадии або фазе и снова очікує її завершення.

Для створенняя об'єкта Phaser використовується один из конструкторов:


Phaser()
Phaser(int parties)
Phaser(Phaser parent)
Phaser(Phaser parent, int parties)

Параметр parties вказує на кількість участников (грубо говоря, потіков), которые повинні выполнять все фазы действия. Первый конструктор створює об'єкт Phaser без яких-або участников. Второй конструктор регистрирует передаваемое в конструктор кількість участников. Третий и четвертый конструктори також встановлюють родительский об'єкт Phaser.

Основные методи класа Phaser:

  • int register(): регистрирует участника, який виконує фазы, и повертає номер текущей фазы - обычно фаза 0

  • int arrive(): повідомляє, что участник завершил фазу, и повертає номер текущей фазы

  • int arriveAndAwaitAdvance(): аналогичен методу arrive, лише при цем заставляет phaser ожидать завершення фазы всеми остальными участниками

  • int arriveAndDeregister(): повідомляє о завершении всех фаз участником и снимает его с регистрации. Повертає номер текущей фазы або отрицательное число, якщо синхронизатор Phaser завершил свою работу

  • int getPhase(): повертає номер текущей фазы

При работае с класом Phaser обычно сначала створюєся его об'єкт. Далїї нам треба зарегистрировать всех участников. Для регистрации для каждоого участника вызывается метод register(), або можна обойтись и без цього методу, передав потрібное кількість участников в конструктор Phaser.

Затем каждый участник виконує неякий набор действий, составляющих фазу. А синхронизатор Phaser ждет, поки все участники не завершат виконання фазы. Щоб сообщить синхронизатору, что фаза завершена, участник повинен вызвать метод arrive() або arriveAndAwaitAdvance(). Після цього синхронизатор переходит к следующей фазе.

Застосуємо Phaser в додатку:


import java.util.concurrent.Phaser;

public class Program {

    public static void main(String[] args) {
        
        Phaser phaser = new Phaser(1);
        new Thread(new PhaseThread(phaser, "PhaseThread 1")).start();
        new Thread(new PhaseThread(phaser, "PhaseThread 2")).start();
        
        // ждем завершення фазы 0
        int phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        // ждем завершення фазы 1
        phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        
        // ждем завершення фазы 2
        phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        
        phaser.arriveAndDeregister();
    }
}

class PhaseThread implements Runnable{
    
    Phaser phaser;
    String name;

    PhaseThread(Phaser p, String n){
        
        this.phaser=p;
        this.name=n;
        phaser.register();
    }
    public void run(){
        
        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что первая фаза достигнута
        
        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что вторая фаза достигнута

        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndDeregister(); // сообщаем о завершении фаз и видаляємо с регистрации об'єкты 
    }
}

Итак, здесь у нас фазы виконуються тремя участниками - главным потіком и двумя потіками PhaseThread. Тому при створенні об'єкта Phaser ему передаєся число 1 - главный потік, а в конструкторе PhaseThread вызывается метод register(). Мы в принципе могли бы не використовувати метод register, но тогда нам треба буво бы указать Phaser phaser = new Phaser(3), оскільки у нас три участника.

Фаза для каждого участника представляє минимальный примитивный набор действий: для потіков PhaseThread це вивід сообщения, а для главного потіка - підсчет текущей фазы за допомогою методу getPhase(). При цем отсчет фаз начинается с нуля. Каждый участник завершує виконання фазы вызовом методу phaser.arriveAndAwaitAdvance(). При вызове цього методу поки післядний участник не завершит виконання текущей фазы, все остальные участники блокуються.

Після завершення виконання післядней фазы происходит отмена регистрации всех участников за допомогою методу arriveAndDeregister().

В итоге работа програми даст следующий вивід:

PhaseThread 1 виконує фазу 0
PhaseThread 2 виконує фазу 0
PhaseThread 1 виконує фазу 1
PhaseThread 2 виконує фазу 1
Фаза 0 завершена
Фаза 1 завершена
PhaseThread 1 виконує фазу 2
PhaseThread 2 виконує фазу 2
Фаза 2 завершена

У даному випадку отримуєся немного путанный вивід. Так, сообщения о выполнении фазы 1 виводиться після сообщения об окончании фазы 0. Что связано с многопоточностью - фазы завершились, но в одном потіке ще не выведено сообщение о завершении, тогда як другие потіки вже начали виконання следующей фазы. В любом случае все це происходит вже після завершення фазы.

Но чтобы буво болїї наглядно, мы можемо використовувати sleep в потіках:


public void run(){
        
        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что первая фаза достигнута
        try{
            Thread.sleep(200);
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что вторая фаза достигнута
        try{
            Thread.sleep(200);
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " виконує фазу " + phaser.getPhase());
        phaser.arriveAndDeregister(); // сообщаем о завершении фаз и видаляємо с регистрации об'єкты 
    }

И в цем случае вивід буде болїї привычным, хотя на работу фаз це нияк не повлияет.

PhaseThread 1 виконує фазу 0
PhaseThread 2 виконує фазу 0
Фаза 0 завершена
PhaseThread 2 виконує фазу 1
PhaseThread 1 виконує фазу 1
Фаза 1 завершена
PhaseThread 2 виконує фазу 2
PhaseThread 1 виконує фазу 2
Фаза 2 завершена
