Клас Phaser

Останнє оновлення: 28.04.2018

Класс Phaser дозволяє синхронизировать потоки, представляющие отдельную фазу або стадию виконання общего действия. Phaser определяет об'єкт синхронизации, який ждет, поки не завершится определенная фаза. Затем Phaser переходит к следующей стадии або фазе и снова очікує її завершения.

Для создания об'єкта Phaser використовується один из конструкторов:


Phaser()
Phaser(int parties)
Phaser(Phaser parent)
Phaser(Phaser parent, int parties)

Параметр parties указывает на количество участников (грубо говоря, потоков), которые повинні выполнять все фазы действия. Первый конструктор створює об'єкт Phaser без яких-або участников. Второй конструктор регистрирует передаваемое в конструктор количество участников. Третий и четвертый конструкторы також устанавливают родительский об'єкт Phaser.

Основные методы класса Phaser:

  • int register(): регистрирует участника, який выполняет фазы, и повертає номер текущей фазы - обычно фаза 0

  • int arrive(): повідомляє, что участник завершил фазу, и повертає номер текущей фазы

  • int arriveAndAwaitAdvance(): аналогичен методу arrive, лише при цем заставляет phaser ожидать завершения фазы всеми остальными участниками

  • int arriveAndDeregister(): повідомляє о завершении всех фаз участником и снимает его с регистрации. Повертає номер текущей фазы або отрицательное число, якщо синхронизатор Phaser завершил свою работу

  • int getPhase(): повертає номер текущей фазы

При работае с классом Phaser обычно сначала створюєся его об'єкт. Далїї нам треба зарегистрировать всех участников. Для регистрации для каждоого участника вызывается метод register(), або можна обойтись и без цього метода, передав потрібное количество участников в конструктор Phaser.

Затем каждый участник выполняет неякий набор действий, составляющих фазу. А синхронизатор Phaser ждет, поки все участники не завершат выполнение фазы. Щоб сообщить синхронизатору, что фаза завершена, участник повинен вызвать метод arrive() або arriveAndAwaitAdvance(). После цього синхронизатор переходит к следующей фазе.

Применим Phaser в приложении:


import java.util.concurrent.Phaser;

public class Program {

    public static void main(String[] args) {
        
        Phaser phaser = new Phaser(1);
        new Thread(new PhaseThread(phaser, "PhaseThread 1")).start();
        new Thread(new PhaseThread(phaser, "PhaseThread 2")).start();
        
        // ждем завершения фазы 0
        int phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        // ждем завершения фазы 1
        phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        
        // ждем завершения фазы 2
        phase = phaser.getPhase();
        phaser.arriveAndAwaitAdvance();
        System.out.println("Фаза " + phase + " завершена");
        
        phaser.arriveAndDeregister();
    }
}

class PhaseThread implements Runnable{
    
    Phaser phaser;
    String name;

    PhaseThread(Phaser p, String n){
        
        this.phaser=p;
        this.name=n;
        phaser.register();
    }
    public void run(){
        
        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что первая фаза достигнута
        
        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что вторая фаза достигнута

        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndDeregister(); // сообщаем о завершении фаз и удаляем с регистрации об'єкты 
    }
}

Итак, здесь у нас фазы выполняются тремя участниками - главным потоком и двумя потоками PhaseThread. Тому при создании об'єкта Phaser ему передаєся число 1 - главный поток, а в конструкторе PhaseThread вызывается метод register(). Мы в принципе могли бы не використовувати метод register, но тогда нам треба буво бы указать Phaser phaser = new Phaser(3), оскільки у нас три участника.

Фаза для каждого участника представляє минимальный примитивный набор действий: для потоков PhaseThread це вывод сообщения, а для главного потока - подсчет текущей фазы за допомогою метода getPhase(). При цем отсчет фаз начинается с нуля. Каждый участник завершує выполнение фазы вызовом метода phaser.arriveAndAwaitAdvance(). При вызове цього метода поки последний участник не завершит выполнение текущей фазы, все остальные участники блокуються.

После завершения виконання последней фазы происходит отмена регистрации всех участников за допомогою метода arriveAndDeregister().

В итоге работа программы даст следующий вывод:

PhaseThread 1 выполняет фазу 0
PhaseThread 2 выполняет фазу 0
PhaseThread 1 выполняет фазу 1
PhaseThread 2 выполняет фазу 1
Фаза 0 завершена
Фаза 1 завершена
PhaseThread 1 выполняет фазу 2
PhaseThread 2 выполняет фазу 2
Фаза 2 завершена

В данном случае отримуєся немного путанный вывод. Так, сообщения о выполнении фазы 1 виводиться после сообщения об окончании фазы 0. Что связано с многопоточностью - фазы завершились, но в одном потоке ще не выведено сообщение о завершении, тогда як другие потоки вже начали выполнение следующей фазы. В любом случае все це происходит вже после завершения фазы.

Но чтобы буво болїї наглядно, мы можемо використовувати sleep в потоках:


public void run(){
        
        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что первая фаза достигнута
        try{
            Thread.sleep(200);
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        
        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndAwaitAdvance(); // сообщаем, что вторая фаза достигнута
        try{
            Thread.sleep(200);
        }
        catch(InterruptedException ex){
            System.out.println(ex.getMessage());
        }
        System.out.println(name + " выполняет фазу " + phaser.getPhase());
        phaser.arriveAndDeregister(); // сообщаем о завершении фаз и удаляем с регистрации об'єкты 
    }

И в цем случае вывод буде болїї привычным, хотя на работу фаз це нияк не повлияет.

PhaseThread 1 выполняет фазу 0
PhaseThread 2 выполняет фазу 0
Фаза 0 завершена
PhaseThread 2 выполняет фазу 1
PhaseThread 1 выполняет фазу 1
Фаза 1 завершена
PhaseThread 2 выполняет фазу 2
PhaseThread 1 выполняет фазу 2
Фаза 2 завершена
