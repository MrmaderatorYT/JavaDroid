StringBuffer и StringBuilder

Останнє оновлення: 16.09.2025

Об'єкты String є незмінюємоыми, поцему все операції, которые змінюють рядки, фактически приводят к створенняю новой рядки, что сказывается на производительности додатки. Для решения цей проблемы, чтобы работа со рядокми проходила с меньшими издержками в Java буви добавлены класы StringBuffer и StringBuilder. По сути они напоминает расширяемую рядок, яку можна изменять без ущерба для производительности.

Эти класы похожи, практически двойники, они имеют одинаковые конструктори, одни и те же методи, которые одинаково використовуються. Единственное их различие соварто в том, что клас StringBuffer синхронизированный и потікобезопасный. То есть клас StringBuffer удобнїї використовувати в многопоточных додатких, где об'єкт данного класа может меняться в различных потіках. Если же речь о многопоточных додатких не идет, то лучше використовувати клас StringBuilder, який не потікобезопасный, но при цем работает быстрїї, чем StringBuffer в однопоточных додатких.

StringBuffer визначає четыре конструктора:


StringBuffer()
StringBuffer(int capacity)
StringBuffer(String str)
StringBuffer(CharSequence chars)

Аналогичные конструктори визначає StringBuilder:


StringBuilder()
StringBuilder(int capacity)
StringBuilder(String str)
StringBuilder(CharSequence chars)

Рассмотрим работу цих класов на примере функциональности StringBuilder.

При всех операціях со рядокми StringBuffer / StringBuilder перераспределяет выделенную память. И чтобы избежать слишком частого перераспределения памяти, StringBuffer/StringBuilder заранїї резервирует деяку область памяти, которая может використовуватися. Конструктор без параметрів резервирует в памяти место для 16 символів. Если мы хочемо, чтобы кількість символів буво иным, то мы можемо пріменіть второй конструктор, який в качестве параметра приймає кількість символів.

Третий и четвертый конструктори обоих класов приймають рядок и набор символів, при цем резервируя память для дополнительных 16 символів.

За допомогою методу capacity() мы можемо отримати кількість символів, для которых зарезервирована память. А за допомогою методу ensureCapacity() змінити минимальную емкость буфера символів:


String str = "Java";
StringBuilder strBuilder = new StringBuilder(str);
System.out.println("Емкость: " + strBuilder.capacity()); // 20
strBuilder.ensureCapacity(32);
System.out.println("Емкость: " + strBuilder.capacity()); // 42
System.out.println("Длина: " + strBuilder.length()); // 4

Так як в самом начале StringBuilder ініціалізуєся рядківой "Java", то его емкость составляет 4 + 16 = 20 символів. Затем мы увеличиваем емкость буфера за допомогою вызова strBuilder.ensureCapacity(32) повышаем минимальную емкость буфера до 32 символів. Однак финальная емкость может отличаться в большую сторону. Так, у даному випадку я получаю емкость не 32 и не 32 + 4 = 36, а 42 символа. Дело в том, что в целях повышения эффективности Java может дополнительно выделять память.

Но в любом случае вне зависимости от емкости длина рядки, яку можна отримати за допомогою методу length(), в StringBuilder остается прежней - 4 символа (оскільки в "Java" 4 символа).

Щоб отримати рядок, которая хранится в StringBuilder, мы можемо використовувати стандартный метод toString():


String str = "Java";
StringBuilder strBuilder = new StringBuilder(str);
System.out.println(strBuilder.toString()); // Java

По всем своим операціям StringBuffer и StringBuilder напоминают клас String.

Отримання и встановлення символів

Метод charAt() отримує, а метод setCharAt() встановлює символ по визначенийному индексу:


StringBuilder strBuilder = new StringBuilder("Java");
char c = strBuilder.charAt(0); // J
System.out.println(c);
strBuilder.setCharAt(0, 'c');
System.out.println(strBuilder.toString()); // cava

Метод getChars() отримує набор символів між визначенийными индексами:


StringBuilder strBuilder = new StringBuilder("world");
int startIndex = 1;
int endIndex = 4;
char[] buffer = new char[endIndex-startIndex];
strBuilder.getChars(startIndex, endIndex, buffer, 0);
System.out.println(buffer); // orl

Додавання в рядок

Метод append() додає підрядок в конец StringBuilder:


StringBuilder strBuilder = new StringBuilder("hello");
strBuilder.append(" world");
System.out.println(strBuilder.toString()); // hello world

Метод insert() додає рядок або символ по визначенийному индексу в StringBuilder:


StringBuilder strBuilder = new StringBuilder("word");
        
strBuilder.insert(3, 'l');
System.out.println(strBuilder.toString()); //world

strBuilder.insert(0, "s");
System.out.println(strBuilder.toString()); //sworld

Видалення символів

Метод delete() видаляє все символи с визначенийного индекса о визначенийной позиции, а метод deleteCharAt() видаляє один символ по визначенийному индексу:


StringBuilder strBuilder = new StringBuilder("assembler");
strBuilder.delete(0,2);
System.out.println(strBuilder.toString()); //sembler
        
strBuilder.deleteCharAt(6);
System.out.println(strBuilder.toString()); //semble

Обрезка рядки

Метод substring() обрезает рядок с визначенийного индекса до конца, або до визначенийного индекса:


StringBuilder strBuilder = new StringBuilder("hello java!");
String str1 = strBuilder.substring(6); // обрезка рядки с 6 символа до конца
System.out.println(str1); //java!
        
String str2 = strBuilder.substring(3, 9); // обрезка рядки с 3 по 9 символ 
System.out.println(str2); //lo jav

Зміна длины

Для изменения длины StringBuilder (не емкости буфера символів) застосовується метод setLength(). Если StringBuilder увеличивается, то его рядок просто дополняется в конце пустыми символами, якщо уменьшается - то рядок по сути обрезается:


StringBuilder strBuilder = new StringBuilder("hello");
strBuilder.setLength(10);
System.out.println(strBuilder.toString()); //"hello     "
        
strBuilder.setLength(4);
System.out.println(strBuilder.toString()); //"hell"

Замена в рядківе

Для замены підрядки між визначенийными позициями в StringBuilder на другую підрядок застосовується метод replace():


StringBuilder strBuilder = new StringBuilder("hello world!");
strBuilder.replace(6,11,"java");
System.out.println(strBuilder.toString()); //hello java!

Первый параметр методу replace вказує, с якой позиции треба начать замену, второй параметр - до якой позиции, а третий параметр вказує на підрядок замены.

Обратный порядок в рядківе

Метод reverse() меняет порядок в StringBuilder на обратный:


StringBuilder strBuilder = new StringBuilder("assembler");
strBuilder.reverse();
System.out.println(strBuilder.toString()); //relbmessa
