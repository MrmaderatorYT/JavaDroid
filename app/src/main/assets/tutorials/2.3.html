Арифметичні операції

Останнє оновлення: 30.10.2018

Більшість операцій у Java аналогічні тим, які застосовуються в інших сі-подібних мовах. Є унарні операції (виконуються над одним операндом), бінарні – над двома операндами, а також тернарні – виконуються над трьома операндами. Операндом є змінна або значення (наприклад, число), що бере участь у операції. Розглянемо всі види операцій.

У арифметичних операціях беруть участь числа. У Java є бінарні арифметичні операції (виробляються над двома операндами) та унарні (виконуються над одним операндом). До бінарних операцій відносять наступні:

  • +

    операція додавання двох чисел:

    
int a = 10;
int b = 7;
int c = a + b;  // 17
int d = 4 + b;  // 11

  • -

    операція віднімання двох чисел:

    
int a = 10;
int b = 7;
int c = a - b;  // 3
int d = 4 - a;  // -6

  • *

    операція множення двох чисел

    
int a = 10;
int b = 7;
int c = a * b;  // 70
int d = b * 5;  // 35

  • /

    операція поділу двох чисел:

    
int a = 20;
int b = 5;
int c = a / b;  // 4
double d = 22.5 / 4.5;  // 5.0

    При розподілі варто враховувати, що якщо в операції беруть участь два цілих числа, то результат розподілу округлятиметься до цілого числа, навіть якщо результат присвоюється змінною float або double:

    
double k = 10 / 4;     // 2
System.out.println(k);

    Щоб результат представляв число з точкою, що плаває, один з операндів також повинен представляти число з плаваючою точкою:

    
double k = 10.0 / 4;     // 2.5
System.out.println(k);

  • %

    одержання залишку від поділу двох чисел:

    
int a = 33;
int b = 5;
int c = a % b;  // 3
int d = 22 % 4; // 2 (22 - 4*5 = 2)

Також є дві унарні арифметичні операції, які проводяться над одним числом: ++ (інкремент) та - (декремент). Кожна з операцій має два різновиди: префіксний та постфіксний:

  • ++ (Префіксний інкремент)

    Предполагает увеличение змінної на единицу, наприклад, z=++y (вначале значение змінної y увеличивается на 1, а затем її значення присвоюється змінної z)

    
int a = 8;
int b = ++a;
System.out.println(a);	// 9
System.out.println(b);	// 9

  • ++ (постфіксний інкремент)

    Також представляє увеличение змінної на единицу, наприклад, z=y++ (вначале значение змінної y присвоюється змінної z, а потом значение змінної y увеличивается на 1)

    
int a = 8;
int b = a++;
System.out.println(a);	// 9
System.out.println(b);	// 8

  • -- (префиксный декремент)

    уменьшение змінної на единицу, наприклад, z=--y (вначале значение змінної y уменьшается на 1, а потом її значення присвоюється змінної z)

    
int a = 8;
int b = --a;
System.out.println(a);	// 7
System.out.println(b);	// 7

  • -- (постфиксный декремент)

    z=y-- (сначала значение змінної y присвоюється змінної z, а затем значение змінної y уменьшается на 1)

    
int a = 8;
int b = a--;
System.out.println(a);	// 7
System.out.println(b);	// 8

Пріоритет арифметичних операцій

Одні операції мають більший пріоритет ніж інші, і тому виконуються спочатку. Операції щодо зменшення пріоритету:

++ (постфіксний інкремент), - (постфіксний декремент)

++ (префіксний інкремент), - (префіксний декремент)

* (множення), / (розподіл), % (залишок від розподілу)

+ (додавання), - (віднімання)

Пріоритет операцій слід враховувати і під час набору арифметичних выражений:


int a = 8;
int b = 7;
int c = a + 5 * ++b;      
System.out.println(c);	// 48

Вначале буде выполняться операция инкремента ++b, которая имїїт больший приоритет - она увеличит значение змінної b и возвратит его в качестве результата. Затем виконується умножение 5 * ++b, и лише в последнюю очередь виконується сложение a + 5 * ++b

Дужки дозволяють перевизначити порядок обчислень:


int a = 8;
int b = 7;
int c = (a + 5) * ++b;    
System.out.println(c);	// 104

Незважаючи на те, що операція додавання має менший пріоритет, але спочатку буде виконуватися саме додавання, а не множення, тому що операція додавання укладена в дужки.

Асоціативність операцій

Кроме приоритета операции отличаются таким понятием як асоціативність. Когда операции имеют один и тот же приоритет, порядок вычисления определяется ассоциативностью операторов. В зависимости от ассоциативности есть два типу операторов:

  • Лівоасоціативні оператори, що виконуються зліва направо

  • Правоасоціативні оператори, які виконуються праворуч наліво

Так, деякі операції, наприклад, операції множення та поділу, мають один і той самий пріоритет. Який тоді буде результат у виразі:

int x = 10 / 5 * 2;

Варто нам трактовать це выражение як (10 / 5) * 2 або як 10 / (5 * 2)? Ведь в зависимости от трактовки мы получим разные результаты.

Все арифметические операторы (кроме префиксного инкремента и декремента) є левоассоциативными, то есть выполняются слева направо. Тому выражение 10 / 5 * 2 необхідно трактовать як (10 / 5) * 2, то есть результатом буде 4.

Операції з числами з плаваючою точкою

Слід зазначити, що числа з точкою, що плаває, не підходять для фінансових та інших обчислень, де помилки при округленні можуть бути критичними. Наприклад:


double d = 2.0 - 1.1;
System.out.println(d);

В даному випадку змінна d дорівнюватиме не 0.9, як можна було б спочатку припустити, а 0.89999999999999999. Подібні помилки точності виникають через те, що на низькому рівні для представлення чисел із плаваючою точкою застосовується двійкова система, проте для числа 0.1 немає двійкового уявлення, як і інших дробових значень. Тому в таких випадках зазвичай застосовується клас BigDecimal, який дозволяє оминути подібні ситуації.
