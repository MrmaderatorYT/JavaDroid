Умови блокування

Останнє оновлення: 22.10.2025

Застосування условий в блокировках дозволяє добиться контроля над управлением доступом к потокам. Условие блокировки представлет собой об'єкт інтерфейса Condition из пакета java.util.concurrent.locks.

Застосування об'єктов Condition во многом аналогично использованию методів wait/notify/notifyAll класса Object, которые буви рассмотрены в одной из прошлых тем. В частности, мы можемо використовувати следующие методы інтерфейса Condition:

  • await: поток очікує, поки не буде выполнено некоторое условие и поки другой поток не вызовет методы signal/signalAll. Під многом аналогичен методу wait класса Object

  • signalAll: сигнализирует всем потокам, у которых ранїї був вызван метод await(), что они можуть продолжить работу. Аналогичен методу notifyAll() класса Object

  • signal: сигнализирует, что поток, у которого ранїї був вызван метод await(), может продолжить работу. Застосування аналогично использованию методу notify класса Object.

    Поток для разблокировки выбирается из множества ожидающих случайным чином. С одной стороны, це эффективнїї, чем разблокировать все потоки. Но якщо случайно выбранный поток обнаруживает, что он все ще не может продолжить выполнение, он снова блокуєся. Если ни один другой поток снова не вызовет сигнал, программа блокуєся.

Эти методы вызываются из блока кода, який попадает под действие блокировки ReentrantLock. Сначала, используя эту блокировку, нам треба отримати об'єкт Condition:


ReentrantLock locker = new ReentrantLock();
Condition condition = locker.newCondition();

Как правило, сначала проверяется условие доступа. Если соблюдается условие, то поток очікує, поки условие не изменится:


while (условие)
    condition.await();

После виконання всех действий другим потокам подается сигнал об изменении условия:

condition.signalAll();

Важно в конце вызвать метод signal/signalAll, чтобы избежать возможнасти взаимоблокировки потоков.

Для примера возьмем задачу производителей и потребителей: производитель повинен произвести 5 товаров, а покупатель повинен эти товары купить. Однак на складе можна одновременно разместить не болїї 3 товаров. В то же час покупатель не может купить товар, якщо на складе нет нияких товаров. То есть покупатель повинен ждать, поки производитель не произведет хотя бы один товар. А производитель повинен ждать, поки покупатель не купит хотя бы один товар, якщо на складе 3 товара. И для решения цей задачи используем блокировки:


import java.util.concurrent.locks.ReentrantLock;
import java.util.concurrent.locks.Condition;

public class Program {
 
    public static void main(String[] args) {
         
        Store store=new Store();
        Producer producer = new Producer(store);
        Consumer consumer = new Consumer(store);
        new Thread(producer).start();
        new Thread(consumer).start();
    }
}
// Класс Магазин, хранящий произведенные товары
class Store{
   private int product=0;
   ReentrantLock locker;
   Condition condition;
   
   Store(){
       locker = new ReentrantLock(); // создаем блокировку
       condition = locker.newCondition(); // получаем условие, связанное с блокировкой
   }
   
   public void get() {
       
      locker.lock();
      try{
          // поки нет доступных товаров на складе, ожидаем
          while (product<1)
              condition.await();
          
          product--;
          System.out.println("Покупатель купил 1 товар");
          System.out.println("Товаров на складе: " + product);
          
          // сигнализируем
          condition.signalAll();
      }
      catch (InterruptedException e){
          System.out.println(e.getMessage());
      }
      finally{
          locker.unlock();
      }
   }
   public void put() {
       
       locker.lock();
       try{
          // поки на складе 3 товара, ждем освобождения места
          while (product>=3)
              condition.await();
          
          product++;
          System.out.println("Производитель добавил 1 товар");
          System.out.println("Товаров на складе: " + product);
          // сигнализируем
          condition.signalAll();
      }
      catch (InterruptedException e){
          System.out.println(e.getMessage());
      }
      finally{
          locker.unlock();
      }
   }
}
// класс Производитель
class Producer implements Runnable{
 
    Store store;
    Producer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.put();
        }
    }
}
// Класс Потребитель
class Consumer implements Runnable{
     
     Store store;
    Consumer(Store store){
       this.store=store; 
    }
    public void run(){
        for (int i = 1; i < 6; i++) {
            store.get();
        }
    }
}

Разберем пошагово, что происходит. Итак, у нас есть следующие компоненты:

  • Store (Магазин): общий ресурс (буфер), к которому обращаются два потока. У него есть ограничение: он может хранить не болїї 3 товаров (product >= 3) и не менїї 0 (product < 1).

  • Producer (Производитель): поток (Runnable), який пытается 5 раз добавить товар в Store (вызывая метод put()).

  • Consumer (Потребитель): поток (Runnable), який пытается 5 раз забрать товар из Store (вызывая метод get()).

  • Program: uлавный класс створює Store, Producer и Consumer, а затем запускає Producer и Consumer в двух отдельных потоках.

Ключевая логика находится в классе Store, який управляет доступом к змінної product.

  • ReentrantLock (locker): бокировка. Прежде чем производитель або потребитель сможуть изменить количество товаров (product), они повинні "захватить" эту блокировку. Це гарантирует, что лише один поток (або Producer, або Consumer) может одновременно выполнять код внутри блоков try...finally в методах get() и put().

  • Condition (condition): условие, которое реализует "механизм ожидания", связанный с блокировкой. Оно дозволяє потоку, який вже захватил блокировку, временно "уснуть" (освободив блокировку), поки не выполнится якое-то условие.

Метод put() в классе Producer (Производитель) выполняет следующие действия:

  • locker.lock(): Поток производителя захоплює блокировку.

  • while (product >= 3): Он проверяет, полон ли склад.

  • condition.await(): Если склад полон (3 товара), производитель засыпает и звільняє блокировку. Он буде ждать, поки его не "разбудит" потребитель.

  • product++: Если склад не полон (меньше 3), производитель додає 1 товар.

  • condition.signalAll(): Производитель "будит" все ожидающие потоки (в данном случае, он будит потребителя, якщо тот уснул из-за пустого склада).

  • locker.unlock(): Производитель отпускает блокировку.

Метод get() класса Consumer (Потребитель) выполняет следующие действия:

  • locker.lock(): Поток потребителя захоплює блокировку.

  • while (product < 1): Он проверяет, пуст ли склад.

  • condition.await(): Если склад пуст (0 товаров), потребитель засыпает и звільняє блокировку. Он буде ждать, поки его не "разбудит" производитель.

  • product--: Если на складе есть товары (1 або больше), потребитель забирает 1 товар.

  • condition.signalAll(): Потребитель "будит" все ожидающие потоки (производителя, якщо тот уснул из-за полного склада).

  • locker.unlock(): Потребитель отпускает блокировку.

Выполнение программы:

  • Два потока (Producer и Consumer) запускаются одновременно.

  • Они начинают "бороться" за блокировку locker.

  • Producer (скорїї всего) захватит блокировку первым, добавит 3 товара (вызывая put() 3 раза) и на четвертой попытке уснет (т.к. product станет равен 3).

  • Consumer тем временем буде забирать товары. Как лише он заберет хотя бы один товар, он вызовет signalAll(), что разбудит потребителя - об'єкта Producer.

  • Producer снова попытается добавить товар.

  • Процесс буде повторяться. Потоки будуть поочередно производить и потреблять товары, при цем количество товаров на складе (product) никогда не буде больше 3.

  • В итоге Producer 5 раз добавит товар, а Consumer 5 раз его заберет, после чего оба потока завершат свою работу.
