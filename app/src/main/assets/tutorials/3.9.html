Клас Object та його методи

Останнє оновлення: 25.09.2025

Хотя мы можемо створити обычный клас, який не є наследником, но фактически все класы наследуются от класа Object. Все остальные класы, даже те, которые мы додаємо в свой проєкт, є неявно производными от класа Object.

Мы можемо використовувати змінну типу Object для ссылки на об'єкты любого типу:


public class Program{
      
    public static void main(String[] args) {
             
        Object obj = "Some Object";
        System.out.println(obj);    // Some Object

        obj = 41;
        System.out.println(obj);    // 41

        obj = new Person("Tom");
        System.out.println(obj);    // Person@1dbd16a6
    }
}
class Person{
    private String name;

    Person(String name) { this.name = name; }

    void print(){  System.out.println("Person " + name); }
}

Поскольку все класы неявно наследуются от Object, то все эти класы також наследуют методи класа Object. Рассмотрим эти методи.

toString

Метод toString служит для получения представления данного об'єкта в виде рядки. При попытке вывести рядківовое представления якого-нибудь об'єкта, як правило, буде виводитьься полное ім'я класа. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		System.out.println(tom.toString()); // Буде виводитьь что-то напідобие Person@7960847b
	}
}
class Person {
    
    private String name;
	
    public Person(String name){
    
        this.name=name;
    }
}

Полученное мной значення (у даному випадку Person@7960847b) вряд ли может служить хорошим рядківовым описанием об'єкта. Тому метод toString() нередко перевизначають. Наприклад:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		System.out.println(tom.toString()); // Person Tom
	}
}
class Person {
    
    private String name;
   
    public Person(String name){
    
        this.name=name;
    }
	
	@Override
	public String toString(){
         
        return "Person " + name;
    }
}

Метод hashCode

Метод hashCode дозволяє задать некоторое числовое значення, которое буде соответствовать данному об'єкту або его хэш-код. По данному числу, наприклад, можна сравнивать об'єкты. Так, якщо x и y — два разных об'єкта, то існує высокая вероятность того, что вызовы x.hashCode() и y.hashCode() дадут различные результати.

Наприклад, виведемо уявлення вищевизначеного об'єкта:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom");
        System.out.println(tom.hashCode()); // 498931366
    }
}
class Person{
    private String name;
    Person(String name) {  this.name = name; }
}

Але ми можемоо задати свій алгоритм визначення хеш-коду об'єкта. Існує багато готових алгоритмів хешування. Наприклад, клас String застосовує такий алгоритм:


int hash = 0;
for (int i = 0; i < length(); i++)
   hash = 31 * hash + charAt(i);

Ми можемоо визначити свій алгоритм, головне треба враховувати, що алгоритм бажано підібрати такий, щоб при обчисленні коду для різних об'єктів їх код також відрізнявся. Наприклад:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom");
        System.out.println(tom.hashCode()); // 842740

        Person tomas = new Person("Tom");
        System.out.println(tomas.hashCode()); // 842740

        Person tim = new Person("Tim");
        System.out.println(tim.hashCode()); // 840880
    }
}
class Person{
    private String name;

    Person(String name) {  this.name = name; }
    @Override
    public int hashCode(){
 
        return 10 * name.hashCode();
    }
}

В даному випадку алгоритм хешування взятий від балди просто для демонстрації цілей. При цьому він залежить від хеш-коду поля name. Проте з консольного висновку бачимо, що з однаквим полем name збігаються, і з різними різняться.

Об'єднання хешів

Зазвичай хеші обчислюються з урахуванням полів об'єкта, як, наприклад, у прикладі вище обчислення хешу застосовувалося поле name. Якщо клас містить кілька полів, то, можливо, ми захочемо використовувати всі ці поля для обчислення загального об'єкта. Наприклад:


class Person{
    private String name;
    private int age;

    Person(String name, int age) { 

        this.name = name;
        this.age = age;
    }
    @Override
    public int hashCode(){
 
        return 31* name.hashCode() + age;
    }
}

У даному випадку для вычисления хеша застосовуються поля name и age. Но об'єднание нескольких хэш-значений может быть утомительным. Під многих случаях можна просто вызвать Objects.hash, передав в него все необходимые поля. Цет метод на выходе об'єднає хэш-коды своих аргументов:


import java.util.Objects;  // для Objects.hash
..................................
class Person{
    private String name;
    private int age;

    Person(String name, int age) { 

        this.name = name;
        this.age = age;
    }
    @Override
    public int hashCode(){
 
        return Objects.hash(name, age);
    }
}

Отримання типу об'єкта та метод getClass

Метод getClass дозволяє отримати тип данного об'єкта:


Person tom = new Person("Tom");
System.out.println(tom.getClass()); // class Person

Метод equals

Метод equals сравнивает два об'єкта на равенство. В качестве параметра метод приймає об'єкт, с який треба сравнить текущий, и повертає true, якщо об'єкты равны, и false, якщо не равны:


boolean equals(Object otherObject)

Варто відзначити, что визначенийия equals и hashCode повинні быть совместимы: якщо x.equals(y) повертає true, то x.hashCode() повинен возвращать то же значення, что и y.hashCode().

За замовчуванням метод equals визначає ідентичність двох посилань на об'єкти:


public class Program{
      
    public static void main(String[] args) {
             
        Point a = new Point(10, 15);
        Point b = a;
        Point c = new Point(10, 15);

        boolean result = a.equals(b);
        System.out.println(result);    // true

        result = a.equals(c);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }
}

Здесь в програме змінні a и b хранять ссылку на один и тот же об'єкт Point, який представляє точку с двумя координатами:


Point a = new Point(10, 15);
Point b = a;

boolean result = a.equals(b);    // true

Тож вони рівні.

А вот змінна c хранит ссылку на другой об'єкт, поцему змінні a/b и c не равны.


Point a = new Point(10, 15);
Point c = new Point(10, 15);

boolean result = a.equals(b);
result = a.equals(c);   // false

Однак в реальности мы видим, что оба об'єкта (a т c), даже несмотря на то, что они не равны, они имеют одинаковые значення. То есть мы имїїм две точки с одинаковыми значеннями. И буво бы болїї разумно, якщо бы они буви равны в силу равности их координат. В цем случае мы можемо перевизначити метод equals:


public class Program{
      
    public static void main(String[] args) {
             
        Point point1 = new Point(10, 15);
        Point point2 = point1;

        boolean result = point1.equals(point2);
        System.out.println(result);    // true

        result = point1.equals(new Point(10, 15));
        System.out.println(result);    // true

        result = point1 .equals("hello");
        System.out.println(result);    // false

        result = point1 .equals(null);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }

    // переопределяем метод equals
    @Override
    public boolean equals(Object otherObject){
         
        // проверяем об'єкты на идентичноть - равенство их ссылок
        if (this == otherObject) return true;

        // проверяем, что второй об'єкт не равен null
        if (otherObject == null) return false;

        // якщо класы об'єктов не совпадают, то они не равны
        if (getClass() != otherObject.getClass()) return false;
 
        // иначе преобразуем об'єкт к типу Point
        Point p = (Point)otherObject;
        // и проверяем равенство их координат
        return this.x == p.x  && this.y == p.y;
    }
}

Метод equals приймає в качестве параметра об'єкт любого типу, соответственно це может быть абсолютно любой об'єкт або даже значення null. И в методі equals сначала проверяем, что текущий и второй экземпляр хранят ссылки на один и тот же об'єкт:

if (this == otherObject) return true;

Якщо дві змінних представляютььь зберігають посилання однією і той самий об'єкт, всі вони рівні.

Далїї проверяем, что второй об'єкт не равен null, иначе дальше сравнивать смысла нет, и об'єкты будуть вже не равны:


if (otherObject == null) return false;

Оскільки другий об'єкт може представляти насправді об'єкт будь-якого типу, то дивимося, чи представляютььь об'єкти той самий клас. Якщо НЕ репрезентують, то об'єкти не рівні:


if (getClass() != otherObject.getClass()) return false;

І якщо об'єкти представляютььь той самий клас, то перетворимо другий об'єкт до цільового типу Point і порівнюємо з координати.

Оператор instanceof дозволяє упростить код. Он дозволяє выяснить, є ли переданный в качестве параметра об'єкт об'єктом визначенийного класа, у даному випадку класа Person. Если об'єкты принадлежат к разным класам, або второй об'єкт представляє null, то их сравнение не имїїт смысла, и повертаєся значення false:


public class Program{
      
    public static void main(String[] args) {
             
        Point point1 = new Point(10, 15);
        Point point2 = point1;

        boolean result = point1.equals(point2);
        System.out.println(result);    // true

        result = point1.equals(new Point(10, 15));
        System.out.println(result);    // true

        result = point1 .equals("hello");
        System.out.println(result);    // false

        result = point1 .equals(null);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }

    // переопределяем метод equals
    @Override
    public boolean equals(Object otherObject){
         
        // проверяем об'єкты на идентичноть - равенство их ссылок
        if (this == otherObject) return true;

        // проверяем, что второй об'єкт - це об'єкт типу Point
        if (!(otherObject instanceof Point)) return false;
 
        // иначе преобразуем об'єкт к типу Point
        Point p = (Point)otherObject;
        // и проверяем равенство их координат
        return this.x == p.x  && this.y == p.y;
    }
}

Спадкування та метод equals

При визначенийии методу equals для підкласа имїїт смысл сначала вызовать методу equals для суперкласа. Если проверка не пройдена, об'єкты не можуть быть равны. Если поля суперкласа равны, можна сравнивать поля об'єкта підкласа:


class Person{
    private String name;

    Person(String name) {  this.name = name; }

    @Override
    public boolean equals(Object otherObject){
         
        if (this == otherObject) return true;
        if (otherObject == null) return false;
        if (getClass() != otherObject.getClass()) return false;
 
        Person p = (Person)otherObject;
        return this.name == p.name;
    }
}

class Employee extends Person{
    private String company;

    Employee(String name, String company) {  

        super(name);
        this.company = company; 
    }

    @Override
    public boolean equals(Object otherObject){
         
        if (!super.equals(otherObject)) return false;
        // super.equals вже проверяет, что this и otherObject представляютьь один и тот же клас
        Employee empl = (Employee)otherObject;
        // и проверяем равенство их координат
        return this.company.equals(empl.company);
    }
}

У даному випадку в класе Employee в реалізації методу equals() сначала вызываем его реализацию у базового класа Person через super.equals(otherObject). Если цей вызов повертає true, то це значит, что оба об'єкта як минимум представляютьь один и тот же клас и значення их полей name равны. И далїї мы можемо сравнивать вже те поля, которые визначенийы непосредственно в класе Employee. Застосування методу:


public class Program{
      
    public static void main(String[] args) {
             
        Employee empl1 = new Employee("Tom", "Yandex");
        Employee empl2 = empl1;

        boolean result = empl1.equals(empl2);
        System.out.println(result);    // true

        result = empl1 .equals(new Employee("Tom", "Yandex"));
        System.out.println(result);    // true

        result = empl1 .equals(new Employee("Bob", "Google"));
        System.out.println(result);    // false

        result = empl1.equals(new Person("Tom"));
        System.out.println(result);    // false

        result = empl1 .equals(null);
        System.out.println(result);    // false
    }
}
