Клас Object та його методи

Останнє оновлення: 25.09.2025

Хотя мы можемо создать обычный класс, який не є наследником, но фактически все классы наследуются от класса Object. Все остальные классы, даже те, которые мы добавляем в свой проект, є неявно производными от класса Object.

Мы можемо використовувати переменную типу Object для ссылки на об'єкты любого типу:


public class Program{
      
    public static void main(String[] args) {
             
        Object obj = "Some Object";
        System.out.println(obj);    // Some Object

        obj = 41;
        System.out.println(obj);    // 41

        obj = new Person("Tom");
        System.out.println(obj);    // Person@1dbd16a6
    }
}
class Person{
    private String name;

    Person(String name) { this.name = name; }

    void print(){  System.out.println("Person " + name); }
}

Поскольку все классы неявно наследуются от Object, то все эти классы також наследуют методы класса Object. Рассмотрим эти методы.

toString

Метод toString служит для получения представления данного об'єкта в виде строки. При попытке вывести строковое представления якого-нибудь об'єкта, як правило, буде выводиться полное имя класса. Например:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		System.out.println(tom.toString()); // Буде выводить что-то наподобие Person@7960847b
	}
}
class Person {
    
    private String name;
	
    public Person(String name){
    
        this.name=name;
    }
}

Полученное мной значение (в данном случае Person@7960847b) вряд ли может служить хорошим строковым описанием об'єкта. Тому метод toString() нередко переопределяют. Например:


public class Program{
     
	public static void main(String[] args) {
			
		Person tom = new Person("Tom");
		System.out.println(tom.toString()); // Person Tom
	}
}
class Person {
    
    private String name;
   
    public Person(String name){
    
        this.name=name;
    }
	
	@Override
	public String toString(){
         
        return "Person " + name;
    }
}

Метод hashCode

Метод hashCode дозволяє задать некоторое числовое значение, которое буде соответствовать данному об'єкту або его хэш-код. По данному числу, наприклад, можна сравнивать об'єкты. Так, якщо x и y — два разных об'єкта, то существует высокая вероятность того, что вызовы x.hashCode() и y.hashCode() дадут различные результаты.

Наприклад, виведемо уявлення вищевизначеного об'єкта:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom");
        System.out.println(tom.hashCode()); // 498931366
    }
}
class Person{
    private String name;
    Person(String name) {  this.name = name; }
}

Але ми можемоо задати свій алгоритм визначення хеш-коду об'єкта. Існує багато готових алгоритмів хешування. Наприклад, клас String застосовує такий алгоритм:


int hash = 0;
for (int i = 0; i < length(); i++)
   hash = 31 * hash + charAt(i);

Ми можемоо визначити свій алгоритм, головне треба враховувати, що алгоритм бажано підібрати такий, щоб при обчисленні коду для різних об'єктів їх код також відрізнявся. Наприклад:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom");
        System.out.println(tom.hashCode()); // 842740

        Person tomas = new Person("Tom");
        System.out.println(tomas.hashCode()); // 842740

        Person tim = new Person("Tim");
        System.out.println(tim.hashCode()); // 840880
    }
}
class Person{
    private String name;

    Person(String name) {  this.name = name; }
    @Override
    public int hashCode(){
 
        return 10 * name.hashCode();
    }
}

В даному випадку алгоритм хешування взятий від балди просто для демонстрації цілей. При цьому він залежить від хеш-коду поля name. Проте з консольного висновку бачимо, що з однаквим полем name збігаються, і з різними різняться.

Об'єднання хешів

Зазвичай хеші обчислюються з урахуванням полів об'єкта, як, наприклад, у прикладі вище обчислення хешу застосовувалося поле name. Якщо клас містить кілька полів, то, можливо, ми захочемо використовувати всі ці поля для обчислення загального об'єкта. Наприклад:


class Person{
    private String name;
    private int age;

    Person(String name, int age) { 

        this.name = name;
        this.age = age;
    }
    @Override
    public int hashCode(){
 
        return 31* name.hashCode() + age;
    }
}

В данном случае для вычисления хеша застосовуються поля name и age. Но об'єднание нескольких хэш-значений может быть утомительным. Під многих случаях можна просто вызвать Objects.hash, передав в него все необходимые поля. Цет метод на выходе об'єднає хэш-коды своих аргументов:


import java.util.Objects;  // для Objects.hash
..................................
class Person{
    private String name;
    private int age;

    Person(String name, int age) { 

        this.name = name;
        this.age = age;
    }
    @Override
    public int hashCode(){
 
        return Objects.hash(name, age);
    }
}

Отримання типу об'єкта та метод getClass

Метод getClass дозволяє отримати тип данного об'єкта:


Person tom = new Person("Tom");
System.out.println(tom.getClass()); // class Person

Метод equals

Метод equals сравнивает два об'єкта на равенство. В качестве параметра метод приймає об'єкт, с який треба сравнить текущий, и повертає true, якщо об'єкты равны, и false, якщо не равны:


boolean equals(Object otherObject)

Варто відзначити, что определения equals и hashCode повинні быть совместимы: якщо x.equals(y) повертає true, то x.hashCode() повинен возвращать то же значение, что и y.hashCode().

За замовчуванням метод equals визначає ідентичність двох посилань на об'єкти:


public class Program{
      
    public static void main(String[] args) {
             
        Point a = new Point(10, 15);
        Point b = a;
        Point c = new Point(10, 15);

        boolean result = a.equals(b);
        System.out.println(result);    // true

        result = a.equals(c);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }
}

Здесь в программе переменные a и b хранять ссылку на один и тот же об'єкт Point, який представляє точку с двумя координатами:


Point a = new Point(10, 15);
Point b = a;

boolean result = a.equals(b);    // true

Тож вони рівні.

А вот переменная c хранит ссылку на другой об'єкт, поцему переменные a/b и c не равны.


Point a = new Point(10, 15);
Point c = new Point(10, 15);

boolean result = a.equals(b);
result = a.equals(c);   // false

Однак в реальности мы видим, что оба об'єкта (a т c), даже несмотря на то, что они не равны, они имеют одинаковые значения. То есть мы имїїм две точки с одинаковыми значениями. И буво бы болїї разумно, якщо бы они буви равны в силу равности их координат. В цем случае мы можемо переопределить метод equals:


public class Program{
      
    public static void main(String[] args) {
             
        Point point1 = new Point(10, 15);
        Point point2 = point1;

        boolean result = point1.equals(point2);
        System.out.println(result);    // true

        result = point1.equals(new Point(10, 15));
        System.out.println(result);    // true

        result = point1 .equals("hello");
        System.out.println(result);    // false

        result = point1 .equals(null);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }

    // переопределяем метод equals
    @Override
    public boolean equals(Object otherObject){
         
        // проверяем об'єкты на идентичноть - равенство их ссылок
        if (this == otherObject) return true;

        // проверяем, что второй об'єкт не равен null
        if (otherObject == null) return false;

        // якщо классы об'єктов не совпадают, то они не равны
        if (getClass() != otherObject.getClass()) return false;
 
        // иначе преобразуем об'єкт к типу Point
        Point p = (Point)otherObject;
        // и проверяем равенство их координат
        return this.x == p.x  && this.y == p.y;
    }
}

Метод equals приймає в качестве параметра об'єкт любого типу, соответственно це может быть абсолютно любой об'єкт або даже значение null. И в методе equals сначала проверяем, что текущий и второй экземпляр хранят ссылки на один и тот же об'єкт:

if (this == otherObject) return true;

Якщо дві змінних представляютьь зберігають посилання однією і той самий об'єкт, всі вони рівні.

Далїї проверяем, что второй об'єкт не равен null, иначе дальше сравнивать смысла нет, и об'єкты будуть вже не равны:


if (otherObject == null) return false;

Оскільки другий об'єкт може представляти насправді об'єкт будь-якого типу, то дивимося, чи представляютьь об'єкти той самий клас. Якщо НЕ репрезентують, то об'єкти не рівні:


if (getClass() != otherObject.getClass()) return false;

І якщо об'єкти представляютьь той самий клас, то перетворимо другий об'єкт до цільового типу Point і порівнюємо з координати.

Оператор instanceof дозволяє упростить код. Он дозволяє выяснить, є ли переданный в качестве параметра об'єкт об'єктом определенного класса, в данном случае класса Person. Если об'єкты принадлежат к разным классам, або второй об'єкт представляє null, то их сравнение не имїїт смысла, и повертаєся значение false:


public class Program{
      
    public static void main(String[] args) {
             
        Point point1 = new Point(10, 15);
        Point point2 = point1;

        boolean result = point1.equals(point2);
        System.out.println(result);    // true

        result = point1.equals(new Point(10, 15));
        System.out.println(result);    // true

        result = point1 .equals("hello");
        System.out.println(result);    // false

        result = point1 .equals(null);
        System.out.println(result);    // false
    }
}
class Point{

    private int x;
    private int y;

    Point(int x, int y){ 

        this.x = x;
        this.y = y;
    }

    void print(){  System.out.printf("(%d, %d)", x, y); }

    // переопределяем метод equals
    @Override
    public boolean equals(Object otherObject){
         
        // проверяем об'єкты на идентичноть - равенство их ссылок
        if (this == otherObject) return true;

        // проверяем, что второй об'єкт - це об'єкт типу Point
        if (!(otherObject instanceof Point)) return false;
 
        // иначе преобразуем об'єкт к типу Point
        Point p = (Point)otherObject;
        // и проверяем равенство их координат
        return this.x == p.x  && this.y == p.y;
    }
}

Спадкування та метод equals

При определении метода equals для подкласса имїїт смысл сначала вызовать метода equals для суперкласса. Если проверка не пройдена, об'єкты не можуть быть равны. Если поля суперкласса равны, можна сравнивать поля об'єкта подкласса:


class Person{
    private String name;

    Person(String name) {  this.name = name; }

    @Override
    public boolean equals(Object otherObject){
         
        if (this == otherObject) return true;
        if (otherObject == null) return false;
        if (getClass() != otherObject.getClass()) return false;
 
        Person p = (Person)otherObject;
        return this.name == p.name;
    }
}

class Employee extends Person{
    private String company;

    Employee(String name, String company) {  

        super(name);
        this.company = company; 
    }

    @Override
    public boolean equals(Object otherObject){
         
        if (!super.equals(otherObject)) return false;
        // super.equals вже проверяет, что this и otherObject представляють один и тот же класс
        Employee empl = (Employee)otherObject;
        // и проверяем равенство их координат
        return this.company.equals(empl.company);
    }
}

В данном случае в классе Employee в реализации метода equals() сначала вызываем его реализацию у базового класса Person через super.equals(otherObject). Если цей вызов повертає true, то це значит, что оба об'єкта як минимум представляють один и тот же класс и значения их полей name равны. И далїї мы можемо сравнивать вже те поля, которые определены непосредственно в классе Employee. Застосування метода:


public class Program{
      
    public static void main(String[] args) {
             
        Employee empl1 = new Employee("Tom", "Yandex");
        Employee empl2 = empl1;

        boolean result = empl1.equals(empl2);
        System.out.println(result);    // true

        result = empl1 .equals(new Employee("Tom", "Yandex"));
        System.out.println(result);    // true

        result = empl1 .equals(new Employee("Bob", "Google"));
        System.out.println(result);    // false

        result = empl1.equals(new Person("Tom"));
        System.out.println(result);    // false

        result = empl1 .equals(null);
        System.out.println(result);    // false
    }
}
