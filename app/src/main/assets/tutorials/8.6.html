Семафори

Останнє оновлення: 28.04.2018

Семафори представляютьь ще одно средство синхронизации для доступа к ресурсу. В Java семафоры представлены класом Semaphore, який располагается в пакеті java.util.concurrent.

Для управления доступом к ресурсу семафор використовує счетчик, представляющий кількість разрешений. Если значення счетчика больше нуля, то потік отримує доступ к ресурсу, при цем счетчик уменьшается на единицу. Після окончания работы с ресурсом потік звільняє семафор, и счетчик увеличивается на единицу. Если же счетчик равен нулю, то потік блокуєся и ждет, поки не получит разрешение от семафора.

Установить кількість разрешений для доступа к ресурсу можна за допомогою конструкторов класа Semaphore:


Semaphore(int permits)
Semaphore(int permits, boolean fair)

Параметр permits вказує на кількість допустимых разрешений для доступа к ресурсу. Параметр fair во втором конструкторе дозволяє установить очередность получения доступа. Если он равен true, то разрешения будуть предоставляться ожидающим потікам в том порядке, в яком они запрашивали доступ. Если же он равен false, то разрешения будуть предоставляться в невизначенийном порядке.

Для получения разрешения у семафора треба вызвать метод acquire(), який имїїт две формы:


void acquire() throws InterruptedException
void acquire(int permits) throws InterruptedException

Для получения одного разрешения застосовується первый вариант, а для получения нескольких разрешений - второй вариант.

Після вызова цього методу поки потік не получит разрешение, он блокуєся.

Після окончания работы с ресурсом полученное ранїї разрешение треба освободить за допомогою методу release():


void release()
void release(int permits)

Первый вариант методу звільняє одно разрешение, а второй вариант - кількість разрешений, указанных в permits.

Використовуємо семафор в простом примере:


import java.util.concurrent.Semaphore;

public class Program {

    public static void main(String[] args) {
        
        Semaphore sem = new Semaphore(1); // 1 разрешение
        CommonResource res = new CommonResource();
        new Thread(new CountThread(res, sem, "CountThread 1")).start();
        new Thread(new CountThread(res, sem, "CountThread 2")).start();
        new Thread(new CountThread(res, sem, "CountThread 3")).start();
    }
}
class CommonResource{
    
    int x=0;  
}

class CountThread implements Runnable{

    CommonResource res;
    Semaphore sem;
    String name;
    CountThread(CommonResource res, Semaphore sem, String name){
        this.res=res;
        this.sem=sem;
        this.name=name;
    }
     
    public void run(){
        
        try{
            System.out.println(name + " очікує разрешение");
            sem.acquire();
            res.x=1;
            for (int i = 1; i < 5; i++){
                System.out.println(this.name + ": " + res.x);
                res.x++;
                Thread.sleep(100);
            }
        }
        catch(InterruptedException e){System.out.println(e.getMessage());}
        System.out.println(name + " звільняє разрешение");
        sem.release();
    }
}

Итак, здесь есть общий ресурс CommonResource с полем x, которое змінюєся каждым потіком. Потіки представлены класом CountThread, який отримує семафор и виконує некоторые действия над ресурсом. В основном класе програми эти потіки запускуються. В итоге мы получим следующий вивід:

Семафори отлично підходят для решения завдань, где треба ограничивать доступ. Наприклад, класическая завданьа про обедающих философов. Ее суть: есть декілька философов, допустим, пять, но одновременно за столом можуть сидеть не болїї двух. И треба, чтобы все философы пообедали, но при цем не возникло взаимоблокировки философами друг друга в борьбе за тарелку и вилку:


import java.util.concurrent.Semaphore;

public class Program {

    public static void main(String[] args) {
        
        Semaphore sem = new Semaphore(2);
		for(int i=1;i<6;i++)
			new Philosopher(sem,i).start();
    }
}
// клас философа
class Philosopher extends Thread 
{
    Semaphore sem; // семафор. ограничивающий число философов
    // кол-во приемов пищи
    int num = 0;
    // условный номер философа
    int id;
    // в качестве параметрів конструктора передаємо идентификатор философа и семафор
    Philosopher(Semaphore sem, int id)
    {
		this.sem=sem;
		this.id=id;
    }
    
    public void run()
    {
		try
		{
			while(num<3)// поки кількість приемов пищи не достигнет 3
			{
				//Запрашиваем у семафора разрешение на виконання
				sem.acquire(); 
				System.out.println ("Философ " + id+" садится за стол");
				// философ ест
				sleep(500);
				num++;
					
				System.out.println ("Философ " + id+" выходит из-за стола");
				sem.release();
			
				// философ гуляет
				sleep(500);
			}
		}
		catch(InterruptedException e)
		{
			System.out.println ("у философа " + id + " проблемы со здоровьем");
		}
    }
}
