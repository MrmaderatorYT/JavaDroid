Proxy (Проксі)

Останнє оновлення: 04.10.2025

Proxy (Проксі) в мові Java застосовуються для створенняя таких об'єктов, которые действуют як экземпляры інтерфейсов, но при цем допускают настройку вызова методів цих інтерфейсов. Прокси представляютьь клас java.lang.reflect.Proxy и не є часто використовуємоым инструментом и, як привило, застосовуються в системном програмирования, наприклад, при створенні плагинов. Зокрема, прокси можна використовувати, чтобы во час виконання програми створити новые класоы, которые реалізують визначенийый набор інтерфейсов. Прокси-класы можуть використовуватися для перенаправления вызовов методів на удаленные серверы, для связывания событий пользовательского інтерфейса с некоторыми действиями, для отслеживание вызовов методів для отладки

Прокси-клас, як и все другие, имїїт методи, унаследованные от базового класа Object (як toString(), equals() и тд.). Но крім того прокси-клас також имїїт методи, которые визначенийы в застосовуємоых інтерфейсах.

Обробник викликів (invocation handler)

Главное отличие по сравнению со стандартной реализацией інтерфейсов заключается в том, что во час виконання нельзя визначити новый код для методів реализуемых інтерфейсов. Вместо цього необхідно предоставить обробник викликів (invocation handler). Обробник викликів представляє об'єкт произвольного класа, який реалізує інтерфейс InvocationHandler. Цет інтерфейс имїїт единственный метод:


Object invoke(Object proxy, Method method, Object[] args) throws Throwable

Метод приймає следующие параметри:

  • proxy: об'єкт прокси-класа, на котором був вызван метод

  • method: экземпляр методу, який відповідає методу вызываемого інтерфейса

  • args: масив аргументов, передаваемых в вызываемый метод при вызове, або null, якщо метод інтерфейса не приймає аргументов. Аргументы примитивных типів обертываются в соответствующие класа-обертки, наприклад, int в java.lang.Integer або boolean в java.lang.Boolean.

Метод повертає значення, повертаємоое класом-прокси при вызове методу реализуемого інтерфейса.

В итоге, при каждом вызове методу прокси-об'єкта у обработчика вызовов вызывается метод invoke(), в який передаєся об'єкт Method и аргументы для методи.

Рассмотрим небольшой пример. Допустим, мы хочемо отслеживать вызовы методів. И для цього визначимо следующий клас:


import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;

class DebugHandler implements InvocationHandler{

   private Object target;

   DebugHandler(Object target){

      this.target = target;
   }

   public Object invoke(Object proxy, Method m, Object[] args) throws Throwable {

      // вивідим ім'я аргумента
      System.out.print(target);
      // вивідим ім'я методу
      System.out.print("." + m.getName() + "(");
      // вивідим аргументы
      if (args != null) {

         for (int i = 0; i < args.length; i++){

            if (i != args.length - 1) System.out.print(", ");
            System.out.print(args[i]);
         }
      }
      System.out.println(")");

      // вызываем метод інтерфейса
      return m.invoke(target, args);
   }
}

Здесь визначаєся клас-реалізація інтерфейса InvokeHandler - клас DebugHandler. Через конструктор он приймає об'єкт, на котором вызывается метод реализуемого інтерфейса. А его метод invoke() просто виводить ім'я и аргументы вызываемого методу, а затем вызывает цей метод, передавая в него об'єкт и аргументы.

Створення прокси-об'єкта

Для створенняя прокси-об'єкта застосовується статический метод Proxy.newProxyInstance():


static Object newProxyInstance (ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)

Цет метод приймає три параметра:

  • loader: загрузчик класов для визначенийия прокси-класа. В качестве загрузчика можна використовувати системный загрузчик, який можна отримати за допомогою методу ClassLoader.getSystemClassLoader()

  • interfaces: список інтерфейсов, реализуемых прокси-класом

  • h: обробник викликів для диспетчеризации вызовов методів

Рассмотрим, як створити и використовувати прокси-об'єкт:


import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

class Program{
     
	public static void main(String[] args) {
         
        Object start = new Point(10, 15);
        var handler = new DebugHandler(start);
        Movable proxy = (Movable) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(), new Class[] { Movable.class }, handler);

        proxy.move(25);
        proxy.move(15);
        proxy.move(10);
    }
}

class Point implements Movable{

    private int x;
    private int y;

    Point(int x, int y){

        this.x = x;
        this.y = y;
    }
    @Override
    public String toString(){ return "(" + x + ", " + y + ")";}

    public void move(int distance){

        System.out.printf("Перемещаем точку (%d, %d) на %d пикселей\n\n", x, y, distance);
        x += distance;
        y += distance;
    }
}

interface Movable{
    void move(int distance);
}

class DebugHandler implements InvocationHandler{

   private Object target;

   public DebugHandler(Object target){

      this.target = target;
   }

   public Object invoke(Object proxy, Method m, Object[] args) throws Throwable {

      // вивідим ім'я аргумента
      System.out.print(target);
      // вивідим ім'я методу
      System.out.print("." + m.getName() + "(");
      // вивідим аргументы
      if (args != null) {

         for (int i = 0; i < args.length; i++){

            if (i != args.length - 1) System.out.print(", ");
            System.out.print(args[i]);
         }
      }
      System.out.println(")");

      // вызываем метод інтерфейса
      return m.invoke(target, args);
   }
}

Итак, у нас есть неякий інтерфейс Movable с одним методом move():


interface Movable{
    void move(int distance);
}

Метод move() в качестве параметра приймає условное расстояние, на которое треба переместить об'єкт.

Есть клас Point, який представляє точку на плоскости и реалізує цей інтерфейс:


class Point implements Movable{

    private int x;
    private int y;

    Point(int x, int y){

        this.x = x;
        this.y = y;
    }
    @Override
    public String toString(){ return "(" + x + ", " + y + ")";}

    public void move(int distance){

        System.out.printf("Перемещаем точку (%d, %d) на %d пикселей\n\n", x, y, distance);
        x += distance;
        y += distance;
    }
}

В реалізації методу move() просто прибавляем расстояние distance к координатам x и y у точки и вивідим некоторое диагностическое сообщение.

Теперь посмотрим на застосування прокси-об'єкта в методі main. Но прежде всего створюємо об'єкт Point, для которого буде затем вызываться метод move()


Object start = new Point(10, 15);

Затем створюємо об'єкт нашего обработчика вызовов - класа DebugHandler, которому передаємо целевой об'єкт:


var handler = new DebugHandler(start);

И далїї створюємо сам прокси-об'єкт:


Movable proxy = (Movable) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(), new Class[] { Movable.class }, handler);

У даному випадку прокси-об'єкт буде вести себя як об'єкт інтерфейса Movable, поцему преобразуем об'єкт к цему інтерфейсу.

Для створенняя прокси в качестве загрузчика застосовуємо системный загрузчик, який отримуємо за допомогою вызова ClassLoader.getSystemClassLoader()

В качестве второго параметра передаємо масив отслеживаемых класов інтерфейсов. В нашем случае у нас один інтерфейс - Movable:

new Class[] { Movable.class }

И третий параметр - наш обробник викликів handler.

Итак, мы создали прокси-об'єкт, який ведет себя як об'єкт інтерфейса Movable. А обробник викликів может перехватить вызовы методів цього інтерфейса. Далїї для демонстрации декілька раз вызываем на прокси-об'єкте метод move():


proxy.move(25);
proxy.move(15);
proxy.move(10);

При каждом вызове proxy.move() обробник викликів через метод invoke() получит вызванный метод и его аргументы, выведет дані на консоль и затем вызовет цей метод на об'єкте Point за допомогою выражения


m.invoke(target, args)

Таким чином, выражение

proxy.move(25)

фактически превратится в 


start.move(25); // где start = new Point(10, 15)

В итоге мы получим следующий консольный вивід:


(10, 15).move(25)
Перемещаем точку (10, 15) на 25 пикселей

(35, 40).move(15)
Перемещаем точку (35, 40) на 15 пикселей

(50, 55).move(10)
Перемещаем точку (50, 55) на 10 пикселей
