Proxy (Проксі)

Останнє оновлення: 04.10.2025

Proxy (Проксі) в мові Java застосовуються для создания таких об'єктов, которые действуют як экземпляры інтерфейсов, но при цем допускают настройку вызова методів цих інтерфейсов. Прокси представляють класс java.lang.reflect.Proxy и не є часто используемым инструментом и, як привило, застосовуються в системном программирования, наприклад, при создании плагинов. В частности, прокси можна використовувати, чтобы во час виконання програми создать новые классоы, которые реализуют определеный набор інтерфейсов. Прокси-классы можуть використовуватися для перенаправления вызовов методів на удаленные серверы, для связывания событий пользовательского інтерфейса с некоторыми действиями, для отслеживание вызовов методів для отладки

Прокси-класс, як и все другие, имїїт методы, унаследованные от базового класса Object (як toString(), equals() и тд.). Но крім того прокси-класс також имїїт методы, которые определены в применяемых інтерфейсах.

Обработчик вызовов (invocation handler)

Главное отличие по сравнению со стандартной реализацией інтерфейсов заключается в том, что во час виконання нельзя определить новый код для методів реализуемых інтерфейсов. Вместо цього необхідно предоставить обработчик вызовов (invocation handler). Обработчик вызовов представляє об'єкт произвольного класса, який реализует інтерфейс InvocationHandler. Цет інтерфейс имїїт единственный метод:


Object invoke(Object proxy, Method method, Object[] args) throws Throwable

Метод приймає следующие параметры:

  • proxy: об'єкт прокси-класса, на котором був вызван метод

  • method: экземпляр метода, який соответствует методу вызываемого інтерфейса

  • args: массив аргументов, передаваемых в вызываемый метод при вызове, або null, якщо метод інтерфейса не приймає аргументов. Аргументы примитивных типов обертываются в соответствующие класса-обертки, наприклад, int в java.lang.Integer або boolean в java.lang.Boolean.

Метод повертає значение, возвращаемое классом-прокси при вызове метода реализуемого інтерфейса.

В итоге, при каждом вызове метода прокси-об'єкта у обработчика вызовов вызывается метод invoke(), в який передаєся об'єкт Method и аргументы для методы.

Рассмотрим небольшой пример. Допустим, мы хочемо отслеживать вызовы методів. И для цього определим следующий класс:


import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;

class DebugHandler implements InvocationHandler{

   private Object target;

   DebugHandler(Object target){

      this.target = target;
   }

   public Object invoke(Object proxy, Method m, Object[] args) throws Throwable {

      // выводим имя аргумента
      System.out.print(target);
      // выводим имя метода
      System.out.print("." + m.getName() + "(");
      // выводим аргументы
      if (args != null) {

         for (int i = 0; i < args.length; i++){

            if (i != args.length - 1) System.out.print(", ");
            System.out.print(args[i]);
         }
      }
      System.out.println(")");

      // вызываем метод інтерфейса
      return m.invoke(target, args);
   }
}

Здесь определяется класс-реализация інтерфейса InvokeHandler - класс DebugHandler. Через конструктор он приймає об'єкт, на котором вызывается метод реализуемого інтерфейса. А его метод invoke() просто выводит имя и аргументы вызываемого метода, а затем вызывает цей метод, передавая в него об'єкт и аргументы.

Створення прокси-об'єкта

Для создания прокси-об'єкта застосовується статический метод Proxy.newProxyInstance():


static Object newProxyInstance (ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)

Цет метод приймає три параметра:

  • loader: загрузчик классов для определения прокси-класса. В качестве загрузчика можна використовувати системный загрузчик, який можна отримати за допомогою метода ClassLoader.getSystemClassLoader()

  • interfaces: список інтерфейсов, реализуемых прокси-классом

  • h: обработчик вызовов для диспетчеризации вызовов методів

Рассмотрим, як создать и використовувати прокси-об'єкт:


import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

class Program{
     
	public static void main(String[] args) {
         
        Object start = new Point(10, 15);
        var handler = new DebugHandler(start);
        Movable proxy = (Movable) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(), new Class[] { Movable.class }, handler);

        proxy.move(25);
        proxy.move(15);
        proxy.move(10);
    }
}

class Point implements Movable{

    private int x;
    private int y;

    Point(int x, int y){

        this.x = x;
        this.y = y;
    }
    @Override
    public String toString(){ return "(" + x + ", " + y + ")";}

    public void move(int distance){

        System.out.printf("Перемещаем точку (%d, %d) на %d пикселей\n\n", x, y, distance);
        x += distance;
        y += distance;
    }
}

interface Movable{
    void move(int distance);
}

class DebugHandler implements InvocationHandler{

   private Object target;

   public DebugHandler(Object target){

      this.target = target;
   }

   public Object invoke(Object proxy, Method m, Object[] args) throws Throwable {

      // выводим имя аргумента
      System.out.print(target);
      // выводим имя метода
      System.out.print("." + m.getName() + "(");
      // выводим аргументы
      if (args != null) {

         for (int i = 0; i < args.length; i++){

            if (i != args.length - 1) System.out.print(", ");
            System.out.print(args[i]);
         }
      }
      System.out.println(")");

      // вызываем метод інтерфейса
      return m.invoke(target, args);
   }
}

Итак, у нас есть неякий інтерфейс Movable с одним методом move():


interface Movable{
    void move(int distance);
}

Метод move() в качестве параметра приймає условное расстояние, на которое треба переместить об'єкт.

Есть класс Point, який представляє точку на плоскости и реализует цей інтерфейс:


class Point implements Movable{

    private int x;
    private int y;

    Point(int x, int y){

        this.x = x;
        this.y = y;
    }
    @Override
    public String toString(){ return "(" + x + ", " + y + ")";}

    public void move(int distance){

        System.out.printf("Перемещаем точку (%d, %d) на %d пикселей\n\n", x, y, distance);
        x += distance;
        y += distance;
    }
}

В реализации метода move() просто прибавляем расстояние distance к координатам x и y у точки и выводим некоторое диагностическое сообщение.

Теперь посмотрим на применение прокси-об'єкта в методе main. Но прежде всего создаем об'єкт Point, для которого буде затем вызываться метод move()


Object start = new Point(10, 15);

Затем создаем об'єкт нашего обработчика вызовов - класса DebugHandler, которому передаем целевой об'єкт:


var handler = new DebugHandler(start);

И далїї создаем сам прокси-об'єкт:


Movable proxy = (Movable) Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(), new Class[] { Movable.class }, handler);

В данном случае прокси-об'єкт буде вести себя як об'єкт інтерфейса Movable, поцему преобразуем об'єкт к цему інтерфейсу.

Для создания прокси в качестве загрузчика применяем системный загрузчик, який получаем за допомогою вызова ClassLoader.getSystemClassLoader()

В качестве второго параметра передаем массив отслеживаемых классов інтерфейсов. В нашем случае у нас один інтерфейс - Movable:

new Class[] { Movable.class }

И третий параметр - наш обработчик вызовов handler.

Итак, мы создали прокси-об'єкт, який ведет себя як об'єкт інтерфейса Movable. А обработчик вызовов может перехватить вызовы методів цього інтерфейса. Далїї для демонстрации декілька раз вызываем на прокси-об'єкте метод move():


proxy.move(25);
proxy.move(15);
proxy.move(10);

При каждом вызове proxy.move() обработчик вызовов через метод invoke() получит вызванный метод и его аргументы, выведет дані на консоль и затем вызовет цей метод на об'єкте Point за допомогою выражения


m.invoke(target, args)

Таким чином, выражение

proxy.move(25)

фактически превратится в 


start.move(25); // где start = new Point(10, 15)

В итоге мы получим следующий консольный вывод:


(10, 15).move(25)
Перемещаем точку (10, 15) на 25 пикселей

(35, 40).move(15)
Перемещаем точку (35, 40) на 15 пикселей

(50, 55).move(10)
Перемещаем точку (50, 55) на 10 пикселей
