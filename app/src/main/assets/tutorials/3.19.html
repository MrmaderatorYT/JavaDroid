Компактні файли коду та метод main

Останнє оновлення: 23.05.2025

С выходом JDK 21 появилась экспериментальная возможнасть - компактные файлы кода и метод main, которая дозволяє упростить программу на Java. А в JDK 25 эта возможнасть перестает быть экспериментальной и стала полноценной для применения. Суть цей возможнасти заключается в упрощении файла программы и метода main. В частности, теперь можна писать код без оголошення класса с сокращением определения кода main.

Компактний метод main

Прежде всего мы можемо определить болїї компактный метод main. Наприклад, возьмем стандартную программу, которая размщена в файле Program.java


public class Program{ 
     
    public static void main (String args[]){
         
		    System.out.println("Hello !");
    }
}

Таким чином, раніше для побудови та запуску програми на Java потрібно:

  • Оголосити класс (public class Program).

  • Создать статический метод main с параметром-массивом строк (public static void main (String args[])).

Теперь можна определить метод main без public static и параметра String args[]:


class Program{
    void main() {
        System.out.println("Hello ");
    }
}

Версії JDK 21, 22, 23 і 24 підтримували таке визначення методу main як експериментальний, а для компіляції програми треба було вказати додаткові опції. Наприклад, для компіляції за допомогою JDK 22 застосовувалась команда:

javac --release 22 --enable-preview Program.java

Здесь 22 указывает на версию JDK, яку соответственно можна заменить на 21, 23, 24. И для запуска программы - також треба буво указать дополнительную опцию --enable-preview:

java --enable-preview Program

Починаючи з версії JDK 25, ніяких додаткових опцій ні при компіляції, ні при запуску прогарм вказувати не потрібно:


eugene@Eugene:/workspace/java$ javac Program.java
eugene@Eugene:/workspace/java$ java Program
Hello !
eugene@Eugene:/workspace/java$

Розглянемо загальну механіку виклику методу main. При запуску програми лаунчер Java (програма запуску програми) вибирає та викликає основний метод запущеного класу:

  • Если класс определяет або найде метод main с параметром String[], то лаунчер выбирает цей метод.

  • В противном случае, якщо класс определяет або найде метод main без параметров, то лаунчер выбирает цей метод.

  • В іншому випадку лаунчер повідомляє про помилку та завершує роботу.

После выбора метода дальнейшие действия зависят от того, выбран статический або не статический метод main:

  • Если выбранный метод - статичний, то лаунчер вызывает его.

  • Иначе класс повинен иметь не закрытый (без модификатора sealed) конструктор без параметров. Лаунчер вызывает цей конструктор, а затем у созданного об'єкта класса вызывает выбранный метод main.

    Якщо клас не має конструктора без параметрів, то лаунчер повідомляє про помилку і завершує роботу.

Наприклад візьмемо файл програми з наступним кодом:


public class Program{
    void main(String[] args) {  
        System.out.println("With args");  
    } 

    void main() {  
        System.out.println("Without args");  
    } 
}

Здесь два определения метода main, оба нестатические, но один из них приймає параметр String[], поцему именно цей метод и буде выбран для виконання програми.

Компактні файли

Якщо компілятор Java виявить вихідний файл із полями та методами, які не укладені в оголошення класу, він вважатиме, що вихідний файл неявно оголошує клас, членами якого є ці поля та методи. Такий вихідний файл називається компактним файлом.

Таким чином, раніше для побудови та запуску програми на Java потрібно:

  • Оголосити класс (public class Program).

  • Создать статический метод main с параметром-массивом строк (public static void main (String args[])).

Тепер у файлі програми можна писати код без оголошення класу:


void main() {
    System.out.println("Hello !");
}

Для цього файлу неявно визначається клас верхнього рівня в неіменованому пакеті. Тобто весь код файлу фактично представлятиме цей клас, який:

  • Розширює java.lang.Object і не реалізує жодних інтерфейсів

  • Має конструктор за замовчуванням без параметрів та інших конструкторів

  • Має як своїх членів поля та методи у компактному вихідному файлі

  • Повинен мати метод main, що запускається; інакше видається помилка часу компіляції

Поскольку поля и методы, оголошенийные в компактном исходном файле, интерпретируются як члены неявно оголошенийного класса, мы можемо рядом с методом main определеть и використовувати другие методы и поля, которые автоматически будуть являться методами и полями класса. Наприклад, определение и вызов метода:


String sayHello(){ 
    return "Hello";
}

void main(String[] args) {  

    System.out.println(sayHello()); 
}

Або визначення та використання полів:


String message = "Hello Work";

void main(String[] args) {  

    System.out.println(message); 
}

Компактный исходный файл оголошує класс неявно, поцему у класса нет имени, которое можна використовувати в исходном коде. Компилятор Java генерує имя класса при компиляции компактного исходного файла, но це имя зависит от реализации и не повинно використовуватися ни в яком исходном коде — даже в исходном коде в самом компактном исходном файле. Мы можемо ссылаться на текущий экземпляр класса через this, явно или, як выше, неявно, но мы не можемо создать экземпляр класса за допомогою оператора new.

Визначення інших типів файлу

У тому самому файлі ми можемоо визначати інші типи, наприклад, класи:


void main(String[] args) {  

    Person tom = new Person("Tom");
    System.out.println(tom.getName()); 
}

class Person{
     
    private String name;
 
    Person(String name){
        this.name = name;
    }
    public void setName(String name){
        this.name = name;
    }
    public String getName(){
     
        return this.name;
    }
}

В даному випадку це буде аналогічно тому, що клас Person був визначений усередині класу, який представляє поточний файл.

Если вложенность класса нас не устраивает, мы можемо явным чином определить для метода main конкретный класс, а рядом с ним остальные типы:


class Program{
    void main(String[] args) {  

        Person bob = new Person("Bob");
        System.out.println(bob.getName()); 
    }
}

class Person{
     
    private String name;
 
    Person(String name){
        this.name = name;
    }
    public void setName(String name){
        this.name = name;
    }
    public String getName(){
     
        return this.name;
    }
}
