Компактні файли коду та метод main

Останнє оновлення: 23.05.2025

С выходом JDK 21 появилась экспериментальная возможнасть - компактные файли коду и метод main, которая дозволяє упростить програму на Java. А в JDK 25 эта возможнасть перестает быть экспериментальной и стала полноценной для применения. Суть цей возможнасти заключается в упрощении файлу програми и методу main. Зокрема, теперь можна писать код без оголошення класа с сокращением визначенийия коду main.

Компактний метод main

Прежде всего мы можемо визначити болїї компактный метод main. Наприклад, возьмем стандартную програму, которая размщена в файле Program.java


public class Program{ 
     
    public static void main (String args[]){
         
		    System.out.println("Hello !");
    }
}

Таким чином, раніше для побудови та запуску програми на Java потрібно:

  • Оголосити клас (public class Program).

  • Створити статический метод main с параметром-масивом рядків (public static void main (String args[])).

Теперь можна визначити метод main без public static и параметра String args[]:


class Program{
    void main() {
        System.out.println("Hello ");
    }
}

Версії JDK 21, 22, 23 і 24 підтримували таке визначення методу main як експериментальний, а для компіляції програми треба було вказати додаткові опції. Наприклад, для компіляції за допомогою JDK 22 застосовувалась команда:

javac --release 22 --enable-preview Program.java

Здесь 22 вказує на версию JDK, яку соответственно можна заменить на 21, 23, 24. И для запуску програми - також треба буво указать дополнительную опцию --enable-preview:

java --enable-preview Program

Починаючи з версії JDK 25, ніяких додаткових опцій ні при компіляції, ні при запуску прогарм вказувати не потрібно:


eugene@Eugene:/workspace/java$ javac Program.java
eugene@Eugene:/workspace/java$ java Program
Hello !
eugene@Eugene:/workspace/java$

Розглянемо загальну механіку виклику методу main. При запуску програми лаунчер Java (програма запуску програми) вибирає та викликає основний метод запущеного класу:

  • Если клас визначає або найде метод main с параметром String[], то лаунчер выбирает цей метод.

  • В противном случае, якщо клас визначає або найде метод main без параметрів, то лаунчер выбирает цей метод.

  • В іншому випадку лаунчер повідомляє про помилку та завершує роботу.

Після выбора методу дальнейшие действия зависят от того, выбран статический або не статический метод main:

  • Если выбранный метод - статичний, то лаунчер вызывает его.

  • Иначе клас повинен иметь не закрытый (без модификатора sealed) конструктор без параметрів. Лаунчер вызывает цей конструктор, а затем у созданного об'єкта класа вызывает выбранный метод main.

    Якщо клас не має конструктора без параметрів, то лаунчер повідомляє про помилку і завершує роботу.

Наприклад візьмемо файл програми з наступним кодом:


public class Program{
    void main(String[] args) {  
        System.out.println("With args");  
    } 

    void main() {  
        System.out.println("Without args");  
    } 
}

Здесь два визначенийия методу main, оба нестатические, но один из них приймає параметр String[], поцему именно цей метод и буде выбран для виконання програми.

Компактні файли

Якщо компілятор Java виявить вихідний файл із полями та методуми, які не укладені в оголошення класу, він вважатиме, що вихідний файл неявно оголошує клас, членами якого є ці поля та методи. Такий вихідний файл називається компактним файлом.

Таким чином, раніше для побудови та запуску програми на Java потрібно:

  • Оголосити клас (public class Program).

  • Створити статический метод main с параметром-масивом рядків (public static void main (String args[])).

Тепер у файлі програми можна писати код без оголошення класу:


void main() {
    System.out.println("Hello !");
}

Для цього файлу неявно визначається клас верхнього рівня в неіменованому пакеті. Тобто весь код файлу фактично представлятиме цей клас, який:

  • Розширює java.lang.Object і не реалізує жодних інтерфейсів

  • Має конструктор за замовчуванням без параметрів та інших конструкторів

  • Має як своїх членів поля та методи у компактному вихідному файлі

  • Повинен мати метод main, що запускується; інакше видається помилка часу компіляції

Поскольку поля и методи, оголошенийные в компактном исходном файле, интерпретируются як члены неявно оголошенийного класа, мы можемо рядом с методом main определеть и використовувати другие методи и поля, которые автоматически будуть являться методуми и полями класа. Наприклад, визначенийие и вызов методу:


String sayHello(){ 
    return "Hello";
}

void main(String[] args) {  

    System.out.println(sayHello()); 
}

Або визначення та використання полів:


String message = "Hello Work";

void main(String[] args) {  

    System.out.println(message); 
}

Компактный исходный файл оголошує клас неявно, поцему у класа нет імені, которое можна використовувати в исходном коді. Компилятор Java генерує ім'я класа при компиляции компактного исходного файлу, но це ім'я зависит от реалізації и не повинно використовуватися ни в яком исходном коді — даже в исходном коді в самом компактном исходном файле. Мы можемо ссылаться на текущий экземпляр класа через this, явно или, як выше, неявно, но мы не можемо створити экземпляр класа за допомогою оператора new.

Визначення інших типів файлу

У тому самому файлі ми можемоо визначати інші типи, наприклад, класи:


void main(String[] args) {  

    Person tom = new Person("Tom");
    System.out.println(tom.getName()); 
}

class Person{
     
    private String name;
 
    Person(String name){
        this.name = name;
    }
    public void setName(String name){
        this.name = name;
    }
    public String getName(){
     
        return this.name;
    }
}

В даному випадку це буде аналогічно тому, що клас Person був визначений усередині класу, який представляє поточний файл.

Если вложенность класа нас не устраивает, мы можемо явным чином визначити для методу main конкретный клас, а рядом с ним остальные типи:


class Program{
    void main(String[] args) {  

        Person bob = new Person("Bob");
        System.out.println(bob.getName()); 
    }
}

class Person{
     
    private String name;
 
    Person(String name){
        this.name = name;
    }
    public void setName(String name){
        this.name = name;
    }
    public String getName(){
     
        return this.name;
    }
}
