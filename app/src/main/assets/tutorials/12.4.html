Процеси. Process та ProcessBuilder

Останнє оновлення: 29.10.2025

Иногда в рамках одной програми на мові Java требуется выполнить другую, произвольную програму. И стандартная бібліотека мови Java надає для цей цели клас Process. Клас Process виконує команду в отдельном процесе операционной системы и дозволяє взаємодіяти с его стандартными потіками ввода, вивіда и ошибок. Зокрема, цей клас забезпечує управление процесами и надає методи для передачи данных в процес и, наоборот, получения данных из процеса, а також ожидания завершення процеса, проверки статуса процеса и его завершення.

Створення процеса

Для створенняя и настройки об'єкта Process застосовується клас ProcessBuilder. Цет клас имїїт два похожих конструктора


ProcessBuilder(String... command)
ProcessBuilder(List<String> command)

Оба цих конструктора приймають путь к запускуємоой програме и набор її аргументов в виде рядків. Наприклад:

var builder = new ProcessBuilder("java", "--version");

Первый аргумент - "java" представляє ім'я додатки, которае буде запускуться в процесе. Второй параметр - "--version" - аргумент, який передаєся приложению. То есть у даному випадку будем запускуть команду "java --version", пытаясь отримати текущую версию Java.

Запуск процеса

Для запуску процеса в класе ProcessBuilder визначений метод start()

Process start()

Цет метод повертає об'єкт запущенного процеса. Наприклад:


class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process javac = builder.start();
    }
}

Однак після компиляции и запуску програми мы ничего не увидим. Для цього нам треба дополнительно выполнить деяку настройку.

Настройка створенняя процеса

За допомогою ряда методів ProcessBuilder можна настроить створення и виконання процеса. Все эти методи повертають об'єкт ProcessBuilder:

  • ProcessBuilder command(String... command) / command(List command): встановлює запускуємоую програму и аргументы для її запускув.

  • ProcessBuilder directory(File directory): встановлює рабочий каталог для построителя процесов.

  • ProcessBuilder inheritIO(): встановлює источник ввода и назначення вивіда підпроцеса такими же, як у текущего процеса Java.

  • ProcessBuilder redirectError(File file): встановлює файл для вивіда ошибок.

  • ProcessBuilder redirectError(ProcessBuilder.Redirect destination): встановлює стандартный вивід ошибок.

  • ProcessBuilder redirectInput(File file): встановлює файл в качестве стандартного источника ввода.

  • ProcessBuilder redirectInput(ProcessBuilder.Redirect source): встановлює источник стандартного ввода.

  • ProcessBuilder redirectOutput(File file): встановлює файл в качестве стандартного вивіда.

  • ProcessBuilder redirectOutput(ProcessBuilder.Redirect destination): встановлює место вивіда.

  • ProcessBuilder redirectErrorStream(boolean redirectErrorStream): встановлює свойство redirectErrorStream - якщо оно равно true, то стандартный вивід и вивід ошибок производятся в одно место.

    • Здесь треба відзначити следующие важные аспекты:

    • Вивід: куда буде йти вивід запускуємоого процеса. Задаєся методом redirectOutput()

    • Ввод: откуда буде йти ввод данных для запускуємоого процеса. Задаєся методом redirectInput()

    • Вивід ошибок: куда буде йти вивід ошибок процеса. Задаєся методом redirectError() и redirectErrorStream()

    Мы також можемо отримати устанавливаемые параметри за допомогою одноименных методів:

    • List<String>> command()

      Повертає запускуємоую програму и аргументы для її запуску.

    • File directory()

      Повертає рабочий каталог для построителя процесов.

    • boolean redirectErrorStream(): Вказує, об'єднує ли цей построитель процесов стандартный вивід ошибок и стандартный вивід.

    • ProcessBuilder.Redirect redirectError(): Повертає стандартный приемник ошибок цього построителя процесов.

    • ProcessBuilder.Redirect redirectInput(): Повертає стандартный источник ввода.

    • ProcessBuilder.Redirect redirectOutput(): Повертає стандартный вивід.

    Можете указать, что потіки ввода, вивіда и ошибок нового процеса повинні совпадать с потіками JVM. Если пользователь запускує JVM в консоли, любой пользовательский ввод перенаправляется в процес, а вивід процеса отображается в консоли. Для цього застосовується метод inheritIO(). Наприклад, наша основная програма запускуєся в консоли, и мы хочемо, чтобы запускуємоый процес також вивідил дані на консоль:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process javac = builder
                            .inheritIO()    // запускуємоый процес найде параметри ввода/вивіда текущего процеса
                            .start();       // запускуємо процес
    }
}

    В итоге при запуску мы увидим версию Java - по сути вивід запущенного підпроцеса:

    
eugene@Eugene:/workspace/java/hello$ javac Program.java && java Program
java 25 2025-09-16 LTS
Java(TM) SE Runtime Environment (build 25+37-LTS-3491)
Java HotSpot(TM) 64-Bit Server VM (build 25+37-LTS-3491, mixed mode, sharing)

    Також можна по отдельности направить потіки підпроцеса на текущие процесы JVM. Для цього використовується константа ProcessBuilder.Redirect.INHERIT

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        builder
            .redirectOutput(ProcessBuilder.Redirect.INHERIT)
            .start();
    }
}

    Ще один распространенный способ вивіда - вивід в файл. Для цього в метод redirectOutput() передаєся об'єкт File:

    
import java.nio.file.Path;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        builder
            .redirectOutput(Path.of("output.txt").toFile())
            .start();
    }
}

    Здесь для створенняя пути застосовується встроенный інтерфейс java.nio.file.Path. За допомогою статического методу Path.of() определяем путь к файлу. У даному випадку буде використовуватися файл "output.txt" из текущего каталогу (якщо такого файлу нет, то он створюєся). А метод .toFile() преобразует путь Path в об'єкт File. В итоге вивід команди "java --version" буде записан в файл "output.txt"

    Нередо потіки вивіда и ошибок пишут свои дані в один потік. Щоб об'єднати потіки вивіда и ошибок можна выполнить вызов

    builder.redirectErrorStream(true);

    Рабочий каталог

    У каждого процеса есть рабочий каталог (working directory), який використовується для вычисления относительных имен каталогов. По умолчанию рабочий каталог процеса збігається с рабочим каталогом виртуальной машины - обычно це каталог текущей програми на Java. За допомогою методу directory() можна отримати данный каталог в виде об'єкта File:

    
var builder = new ProcessBuilder("java", "--version");
System.out.println(builder.directory());  // null - текущий каталог

    Если метод повертає null, то в качестве рабочего каталогу установлен каталог текуй програми на Java.

    Но також можна змінити рабочий каталог за допомогою одноіменного методу directory(), який приймає об'єкт File.

    Наприклад, створимо в текущей папке програми новый каталог с именем "test". А в цем каталогу визначимо файл с именем "Main.java" и наступним содержимым:

    
class Main{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Hello ");
    }
}

    То есть файл Main.java из папки "test" містить простейшую програму на Java. И допустим, мы хочемо, чтобы наша програма запускула на виконання цей файл. Для цього определеим следующую програму:

    
import java.nio.file.Path;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "Main.java");
        builder
            .directory(Path.of("test").toFile())                // устанавливаем рабочий каталог на підкаталог "test"
            .redirectOutput(ProcessBuilder.Redirect.INHERIT)    // устанавливаем вивід на консоль
            .redirectErrorStream(true)                          // вивід ошибок також идет на консоль
            .start();
    }
}

    То есть фактически наша програму буде запускуть команду "java Main.java". И поскольку мы установили в качестве рабочего каталогу підпадку "test", то процес сможет найти файл "Main.java". Запустимо данную програму, и в отдельном підпроцесе буде выполняться команда java Main.java:

    
eugene@Eugene:/workspace/java$ java Program.java
Hello 
eugene@Eugene:/workspace/java$

    Управление процесами

    Клас Process надає ряд методів для управления процесом:

    • abstract InputStream getErrorStream()

      Повертає входной потік, підключенный к потіку ошибок процеса.

    • abstract InputStream getInputStream()

      Повертає потік ввода, підключенный к обычному вивіду процеса.

    • abstract OutputStream getOutputStream()

      Повертає потік вивіда, підключенный к обычному вводу процеса.

    • final BufferedReader errorReader()

      Повертає об'єкт BufferedReader, підключенный к стандартному потіку ошибок процеса.

    • final BufferedReader inputReader()

      Повертає об'єкт BufferedReader, підключенный к стандартному вивіду процеса.

    • final BufferedWriter outputWriter()

      Повертає об'єкт BufferedWriter, підключенный к обычному входу процеса, використовуючи собственную кодировку.

    • Stream<ProcessHandle> children()

      Повертає потік прямых дочерних процесов процеса.

    • Stream<ProcessHandle> descendants()

      Повертає потік потомков процеса.

    • abstract void destroy()

      Завершує процес.

    • Process destroyForcibly()

      Принудительно завершує процес.

    • boolean isAlive()

      Проверяет, активен ли запущенный процес.

    • long pid()

      Повертає идентификатор запущенного процеса.

    • ProcessHandle toHandle()

      Повертає об'єкт ProcessHandle для процеса.

    • CompletableFuture onExit()

      Повертає об'єкт CompletableFuture для завершення процеса.

    • boolean supportsNormalTermination()

      Повертає true, якщо метод destroy() завершує процес нормально. Повертає false, якщо метод destroy завершує процес принудительно и немедленно.

    • abstract int waitFor()

      При необхідности заставляет текущий потік ожидать завершення запущенного процеса. Модификации методу дозволяють передать период ожидания до завершення процеса

      boolean waitFor(long timeout, TimeUnit unit)

      boolean waitFor(Duration duration)

    Отримання потіков ввода-вивіда процеса

    Ряд методів дозволяють отримати потіки ввода-вивіда процеса:

    • abstract InputStream getErrorStream()

      Повертає входной потік, підключенный к потіку ошибок процеса.

    • abstract InputStream getInputStream()

      Повертає потік ввода, підключенный к обычному вивіду процеса.

    • abstract OutputStream getOutputStream()

      Повертає потік вивіда, підключенный к обычному вводу процеса.

    • final BufferedReader errorReader()

      Повертає об'єкт BufferedReader, підключенный к стандартному потіку ошибок процеса.

    • final BufferedReader inputReader()

      Повертає об'єкт BufferedReader, підключенный к стандартному вивіду процеса.

    • final BufferedWriter outputWriter()

      Повертає об'єкт BufferedWriter, підключенный к обычному входу процеса, використовуючи собственную кодировку.

    Конечно, мы можемо в целом перенаправить потік вивіда або ошибок в заданное место (наприклад, в файл або на консоль). Однак потіки ввода-вивіда дозволяють вручную настроить взаимодействие с потіком. Наприклад:

    
import java.util.Scanner;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.start();
        try (var in = new Scanner(process.getInputStream()))
        {
            int i = 0;
            // проходим по каждой рядківе из потіка
            while (in.hasNextLine()){
                // при вивіде на консоль додаємо перед рядківой її номер
                System.out.println(++i + ". " + in.nextLine());
            }
        }
    }
}

    У даному випадку запускуємо команду "java --version", которая виводить версию Java. Но цей вивід может включать в себя декілька рядків. И у даному випадку мы вручную обробляємо кажду рядок из входного потіка, добавляя к рядківе вначале її номер. В итоге мы получим консольный вивід на підобие следующего:

    
1. java 25 2025-09-16 LTS
2. Java(TM) SE Runtime Environment (build 25+37-LTS-3491)
3. Java HotSpot(TM) 64-Bit Server VM (build 25+37-LTS-3491, mixed mode, sharing)

    Аналогично можна буво бы отримати потік ввода за допомогою другого методу:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.start();
        try (var in = process.inputReader();)
        {
            int i = 0;
            String s;
            // считываем построчно
            while ((s = in.readLine())!=null){
                System.out.println(++i + ". " + s);
            }
        }
    }
}

    Варто відзначити, что буферное пространство для потіков процесов ограничено. Тому не йде перегружать входные дані, а выходные йде считывать быстро.

    Завершення процеса

    destroy и destroyForcibly

    Для завершення процеса застосовуються методи destroy() або destroyForcibly().

    
process.destroy();
// или
process.destroyForcibly();

    Разница між ними зависит от платформы. В UNIX-підобных системах (наприклад, в Linux) метод destroy() завершує процес за допомогою сигнала SIGTERM, а destroyForcibly() - за допомогою сигнала SIGKILL. (Метод supportsNormalTermination() повертає true, якщо метод destroy() может завершить процес нормально.)

    Уничтожение процеса підает операционной системе сигнал о необхідности его завершення. Операционная система повинна очистить и освободить ресурсы цього процеса. Обычно файлівые дескрипторы и дескрипторы закрываются. При их закрытии все з'єднання с другими процесами разрываются.

    Очікування завершення процеса и waitFor

    Щоб дождаться завершення процеса, можна вызвать метод waitFor():

    int result = process.waitFor();

    Цет метод повертає числовой код завершення процеса (по угоді, 0 в случае успешного завершення або ненулевой код ошибки).

    Однак цей метод неограничен по времени. Щоб ограничить час ожидания, можна пріменіть другую версию методу, которая приймає временной интервал:

    
long delay = 10; // период ожидания
if (process.waitFor(delay, TimeUnit.SECONDS)){

    int result = process.exitValue();
}
else{

    process.destroyForcibly();
}

    Здесь вызов process.waitFor() повертає true, якщо час ожидания процеса не истекло. В цем случае за допомогою методу exitValue() отримуємо статусный код завершення.

    Асинхронное уведомление о завершении процеса и onExit()

    За допомогою методу onExit() можна отримати асинхронное уведомление о завершении процеса. Цет метод повертає об'єкт CompletableFuture<Process>, який можна використовувати для планирования любого действия по завершении процеса:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.redirectOutput(ProcessBuilder.Redirect.INHERIT).start();
        process.onExit().thenAccept(p -> System.out.println("Процес завершился с кодом: " + p.exitValue()));
        Thread.sleep(500); // задержка, что успел отпрацювати
    }
}
