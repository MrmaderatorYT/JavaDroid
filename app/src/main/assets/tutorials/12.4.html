Процеси. Process та ProcessBuilder

Останнє оновлення: 29.10.2025

Иногда в рамках одной программы на мові Java требуется выполнить другую, произвольную программу. И стандартная библиотека мови Java надає для цей цели класс Process. Класс Process выполняет команду в отдельном процессе операционной системы и дозволяє взаємодіяти с его стандартными потоками ввода, вывода и ошибок. В частности, цей класс забезпечує управление процессами и надає методы для передачи данных в процесс и, наоборот, получения данных из процесса, а також ожидания завершения процесса, проверки статуса процесса и его завершения.

Створення процесса

Для создания и настройки об'єкта Process застосовується класс ProcessBuilder. Цет класс имїїт два похожих конструктора


ProcessBuilder(String... command)
ProcessBuilder(List<String> command)

Оба цих конструктора принимают путь к запускаемой программе и набор її аргументов в виде строк. Например:

var builder = new ProcessBuilder("java", "--version");

Первый аргумент - "java" представляє имя приложения, которае буде запускаться в процессе. Второй параметр - "--version" - аргумент, який передаєся приложению. То есть в данном случае будем запускать команду "java --version", пытаясь отримати текущую версию Java.

Запуск процесса

Для запуска процесса в классе ProcessBuilder определен метод start()

Process start()

Цет метод повертає об'єкт запущенного процесса. Например:


class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process javac = builder.start();
    }
}

Однак после компиляции и запуска программы мы ничего не увидим. Для цього нам треба дополнительно выполнить деяку настройку.

Настройка создания процесса

С помощью ряда методів ProcessBuilder можна настроить створення и выполнение процесса. Все эти методы повертають об'єкт ProcessBuilder:

  • ProcessBuilder command(String... command) / command(List command): устанавливает запускаемую программу и аргументы для її запускав.

  • ProcessBuilder directory(File directory): устанавливает рабочий каталог для построителя процессов.

  • ProcessBuilder inheritIO(): устанавливает источник ввода и назначение вывода подпроцесса такими же, як у текущего процесса Java.

  • ProcessBuilder redirectError(File file): устанавливает файл для вывода ошибок.

  • ProcessBuilder redirectError(ProcessBuilder.Redirect destination): устанавливает стандартный вывод ошибок.

  • ProcessBuilder redirectInput(File file): устанавливает файл в качестве стандартного источника ввода.

  • ProcessBuilder redirectInput(ProcessBuilder.Redirect source): устанавливает источник стандартного ввода.

  • ProcessBuilder redirectOutput(File file): устанавливает файл в качестве стандартного вывода.

  • ProcessBuilder redirectOutput(ProcessBuilder.Redirect destination): устанавливает место вывода.

  • ProcessBuilder redirectErrorStream(boolean redirectErrorStream): устанавливает свойство redirectErrorStream - якщо оно равно true, то стандартный вывод и вывод ошибок производятся в одно место.

    • Здесь треба відзначити следующие важные аспекты:

    • Вывод: куда буде идти вывод запускаемого процесса. Задается методом redirectOutput()

    • Ввод: откуда буде идти ввод данных для запускаемого процесса. Задается методом redirectInput()

    • Вывод ошибок: куда буде идти вывод ошибок процесса. Задается методом redirectError() и redirectErrorStream()

    Мы також можемо отримати устанавливаемые параметры за допомогою одноименных методів:

    • List<String>> command()

      Повертає запускаемую программу и аргументы для її запуска.

    • File directory()

      Повертає рабочий каталог для построителя процессов.

    • boolean redirectErrorStream(): Указывает, об'єднує ли цей построитель процессов стандартный вывод ошибок и стандартный вывод.

    • ProcessBuilder.Redirect redirectError(): Повертає стандартный приемник ошибок цього построителя процессов.

    • ProcessBuilder.Redirect redirectInput(): Повертає стандартный источник ввода.

    • ProcessBuilder.Redirect redirectOutput(): Повертає стандартный вывод.

    Можете указать, что потоки ввода, вывода и ошибок нового процесса повинні совпадать с потоками JVM. Если пользователь запускає JVM в консоли, любой пользовательский ввод перенаправляется в процесс, а вывод процесса отображается в консоли. Для цього застосовується метод inheritIO(). Наприклад, наша основная программа запускаєся в консоли, и мы хочемо, чтобы запускаемый процесс також выводил дані на консоль:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process javac = builder
                            .inheritIO()    // запускаемый процесс найде параметры ввода/вывода текущего процесса
                            .start();       // запускаем процесс
    }
}

    В итоге при запуске мы увидим версию Java - по сути вывод запущенного подпроцесса:

    
eugene@Eugene:/workspace/java/hello$ javac Program.java && java Program
java 25 2025-09-16 LTS
Java(TM) SE Runtime Environment (build 25+37-LTS-3491)
Java HotSpot(TM) 64-Bit Server VM (build 25+37-LTS-3491, mixed mode, sharing)

    Також можна по отдельности направить потоки подпроцесса на текущие процессы JVM. Для цього використовується константа ProcessBuilder.Redirect.INHERIT

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        builder
            .redirectOutput(ProcessBuilder.Redirect.INHERIT)
            .start();
    }
}

    Ще один распространенный способ вывода - вывод в файл. Для цього в метод redirectOutput() передаєся об'єкт File:

    
import java.nio.file.Path;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        builder
            .redirectOutput(Path.of("output.txt").toFile())
            .start();
    }
}

    Здесь для создания пути застосовується встроенный інтерфейс java.nio.file.Path. С помощью статического метода Path.of() определяем путь к файлу. В данном случае буде використовуватися файл "output.txt" из текущего каталога (якщо такого файла нет, то он створюєся). А метод .toFile() преобразует путь Path в об'єкт File. В итоге вывод команды "java --version" буде записан в файл "output.txt"

    Нередо потоки вывода и ошибок пишут свои дані в один поток. Щоб об'єднати потоки вывода и ошибок можна выполнить вызов

    builder.redirectErrorStream(true);

    Рабочий каталог

    У каждого процесса есть рабочий каталог (working directory), який використовується для вычисления относительных имен каталогов. По умолчанию рабочий каталог процесса совпадает с рабочим каталогом виртуальной машины - обычно це каталог текущей программы на Java. С помощью метода directory() можна отримати данный каталог в виде об'єкта File:

    
var builder = new ProcessBuilder("java", "--version");
System.out.println(builder.directory());  // null - текущий каталог

    Если метод повертає null, то в качестве рабочего каталога установлен каталог текуй программы на Java.

    Но також можна изменить рабочий каталог за допомогою одноіменного метода directory(), який приймає об'єкт File.

    Наприклад, создадим в текущей папке программы новый каталог с именем "test". А в цем каталоге определим файл с именем "Main.java" и наступним содержимым:

    
class Main{
 
    public static void main(String[] args) throws Exception{
          
        System.out.println("Hello ");
    }
}

    То есть файл Main.java из папки "test" містить простейшую программу на Java. И допустим, мы хочемо, чтобы наша программа запускала на выполнение цей файл. Для цього определеим следующую программу:

    
import java.nio.file.Path;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "Main.java");
        builder
            .directory(Path.of("test").toFile())                // устанавливаем рабочий каталог на подкаталог "test"
            .redirectOutput(ProcessBuilder.Redirect.INHERIT)    // устанавливаем вывод на консоль
            .redirectErrorStream(true)                          // вывод ошибок також идет на консоль
            .start();
    }
}

    То есть фактически наша программу буде запускать команду "java Main.java". И поскольку мы установили в качестве рабочего каталога подпадку "test", то процесс сможет найти файл "Main.java". Запустим данную программу, и в отдельном подпроцессе буде выполняться команда java Main.java:

    
eugene@Eugene:/workspace/java$ java Program.java
Hello 
eugene@Eugene:/workspace/java$

    Управление процессами

    Класс Process надає ряд методів для управления процессом:

    • abstract InputStream getErrorStream()

      Повертає входной поток, подключенный к потоку ошибок процесса.

    • abstract InputStream getInputStream()

      Повертає поток ввода, подключенный к обычному выводу процесса.

    • abstract OutputStream getOutputStream()

      Повертає поток вывода, подключенный к обычному вводу процесса.

    • final BufferedReader errorReader()

      Повертає об'єкт BufferedReader, подключенный к стандартному потоку ошибок процесса.

    • final BufferedReader inputReader()

      Повертає об'єкт BufferedReader, подключенный к стандартному выводу процесса.

    • final BufferedWriter outputWriter()

      Повертає об'єкт BufferedWriter, подключенный к обычному входу процесса, используя собственную кодировку.

    • Stream<ProcessHandle> children()

      Повертає поток прямых дочерних процессов процесса.

    • Stream<ProcessHandle> descendants()

      Повертає поток потомков процесса.

    • abstract void destroy()

      Завершує процесс.

    • Process destroyForcibly()

      Принудительно завершує процесс.

    • boolean isAlive()

      Проверяет, активен ли запущенный процесс.

    • long pid()

      Повертає идентификатор запущенного процесса.

    • ProcessHandle toHandle()

      Повертає об'єкт ProcessHandle для процесса.

    • CompletableFuture onExit()

      Повертає об'єкт CompletableFuture для завершения процесса.

    • boolean supportsNormalTermination()

      Повертає true, якщо метод destroy() завершує процесс нормально. Повертає false, якщо метод destroy завершує процесс принудительно и немедленно.

    • abstract int waitFor()

      При необхідности заставляет текущий поток ожидать завершения запущенного процесса. Модификации метода дозволяють передать период ожидания до завершения процесса

      boolean waitFor(long timeout, TimeUnit unit)

      boolean waitFor(Duration duration)

    Получение потоков ввода-вывода процесса

    Ряд методів дозволяють отримати потоки ввода-вывода процесса:

    • abstract InputStream getErrorStream()

      Повертає входной поток, подключенный к потоку ошибок процесса.

    • abstract InputStream getInputStream()

      Повертає поток ввода, подключенный к обычному выводу процесса.

    • abstract OutputStream getOutputStream()

      Повертає поток вывода, подключенный к обычному вводу процесса.

    • final BufferedReader errorReader()

      Повертає об'єкт BufferedReader, подключенный к стандартному потоку ошибок процесса.

    • final BufferedReader inputReader()

      Повертає об'єкт BufferedReader, подключенный к стандартному выводу процесса.

    • final BufferedWriter outputWriter()

      Повертає об'єкт BufferedWriter, подключенный к обычному входу процесса, используя собственную кодировку.

    Конечно, мы можемо в целом перенаправить поток вывода або ошибок в заданное место (наприклад, в файл або на консоль). Однак потоки ввода-вывода дозволяють вручную настроить взаимодействие с потоком. Например:

    
import java.util.Scanner;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.start();
        try (var in = new Scanner(process.getInputStream()))
        {
            int i = 0;
            // проходим по каждой строке из потока
            while (in.hasNextLine()){
                // при выводе на консоль добавляем перед строкой її номер
                System.out.println(++i + ". " + in.nextLine());
            }
        }
    }
}

    В данном случае запускаем команду "java --version", которая выводит версию Java. Но цей вывод может включать в себя декілька строк. И в данном случае мы вручную обрабатываем кажду строку из входного потока, добавляя к строке вначале її номер. В итоге мы получим консольный вывод на подобие следующего:

    
1. java 25 2025-09-16 LTS
2. Java(TM) SE Runtime Environment (build 25+37-LTS-3491)
3. Java HotSpot(TM) 64-Bit Server VM (build 25+37-LTS-3491, mixed mode, sharing)

    Аналогично можна буво бы отримати поток ввода за допомогою другого метода:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.start();
        try (var in = process.inputReader();)
        {
            int i = 0;
            String s;
            // считываем построчно
            while ((s = in.readLine())!=null){
                System.out.println(++i + ". " + s);
            }
        }
    }
}

    Варто відзначити, что буферное пространство для потоков процессов ограничено. Тому не йде перегружать входные дані, а выходные йде считывать быстро.

    Завершение процесса

    destroy и destroyForcibly

    Для завершения процесса застосовуються методы destroy() або destroyForcibly().

    
process.destroy();
// или
process.destroyForcibly();

    Разница между ними зависит от платформы. В UNIX-подобных системах (наприклад, в Linux) метод destroy() завершує процесс за допомогою сигнала SIGTERM, а destroyForcibly() - за допомогою сигнала SIGKILL. (Метод supportsNormalTermination() повертає true, якщо метод destroy() может завершить процесс нормально.)

    Уничтожение процесса подает операционной системе сигнал о необхідности его завершения. Операционная система повинна очистить и освободить ресурсы цього процесса. Обычно файловые дескрипторы и дескрипторы закрываются. При их закрытии все соединения с другими процессами разрываются.

    Ожидание завершения процесса и waitFor

    Щоб дождаться завершения процесса, можна вызвать метод waitFor():

    int result = process.waitFor();

    Цет метод повертає числовой код завершения процесса (по соглашению, 0 в случае успешного завершения або ненулевой код ошибки).

    Однак цей метод неограничен по времени. Щоб ограничить час ожидания, можна применить другую версию метода, которая приймає временной интервал:

    
long delay = 10; // период ожидания
if (process.waitFor(delay, TimeUnit.SECONDS)){

    int result = process.exitValue();
}
else{

    process.destroyForcibly();
}

    Здесь вызов process.waitFor() повертає true, якщо час ожидания процесса не истекло. В цем случае за допомогою метода exitValue() получаем статусный код завершения.

    Асинхронное уведомление о завершении процесса и onExit()

    С помощью метода onExit() можна отримати асинхронное уведомление о завершении процесса. Цет метод повертає об'єкт CompletableFuture<Process>, який можна використовувати для планирования любого действия по завершении процесса:

    
class Program{
 
    public static void main(String[] args) throws Exception{
          
        var builder = new ProcessBuilder("java", "--version");
        Process process = builder.redirectOutput(ProcessBuilder.Redirect.INHERIT).start();
        process.onExit().thenAccept(p -> System.out.println("Процесс завершился с кодом: " + p.exitValue()));
        Thread.sleep(500); // задержка, что успел отработать
    }
}
