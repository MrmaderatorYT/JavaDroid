Клас Constructor та створення об'єктів

Останнє оновлення: 29.09.2025

Клас Constructor из пакету java.lang.reflect представляє конструктор класа. Он надає функционал для получения информации о конструкторе, а також возможнасти для створенняя новых об'єктов. Отмечу основные его возможнасти:

  • String getName(): повертає ім'я конструктора

  • int getModifiers(): повертає целое число с различными включенными и выключенными битами, которые описують використовуємоые модификаторы.

  • Object newInstance(Object... initargs): створює новый экземпляр класа, передавая конструктору аргументы через параметр initargs.

  • Class[] getParameterTypes(): повертає масив об'єктов Class, которые представляютьь типи параметрів.

  • Class[] getExceptionTypes(): повертає масив об'єктов Class, представляющих типи винятків, генерируемых методом.

Отримання конструктора

Для получения конструктора класа в виде об'єкта Constructor застосовується ряд методів об'єкта Class:

  • Constructor[] getConstructors(): повертає масив об'єктов Constructor для открытых конструкторов (с модификатором public) данного класа

  • Constructor[] getDeclaredConstructors(): повертає масив об'єктов Constructor для всех конструкторов данного класа. Метод повертають масив нулевой длины, якщо таких конструкторов нет або якщо об'єкт представляє собой примитивный тип або масив.

  • Constructor getConstructor(Class... parameterTypes): повертає об'єкт Constructor для публичного конструктора. Параметр parameterTypes — це масив об'єктов Class, которые идентифицируют формальные типи параметрів конструктора в порядке их оголошення. Благодаря цим визначенийиям параметрів Java может выбрать нужную версию, якщо визначенийо декілька конструкторов. Если підобный публичный конструктор отсутствует, то метод генерує виняток NoSuchConstructorException

  • Constructor getDeclaredConstructor(String name): повертає об'єкт Constructor для любого конструктора класа. Параметр parameterTypes — це масив об'єктов Class, которые идентифицируют формальные типи параметрів конструктора в порядке их оголошення. Благодаря цим визначенийиям параметрів Java может выбрать нужную версию, якщо визначенийо декілька конструкторов. Если підобный публичный конструктор отсутствует, то метод генерує виняток NoSuchConstructorException

Наприклад, возьмем следующий клас:


class Person {
     
    private String name;
    private int age;

    Person(){
        this("Undefined", 1);
    }

    Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
    
    void print(){
        System.out.printf("Person. Name: %s;  Age: %d\n", name, age);
    }
}

Получим конструктори цього класа:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Person.class;
        Constructor[] ctors = cl.getDeclaredConstructors();
        for(Constructor ctor : ctors){

            System.out.println(ctor);
        }
    }
}

Консольный вивід:


Person()
Person(java.lang.String,int)

Теперь получим один конструктор, исходя из его параметрів. Параметы передаються в виде набора об'єктов Class, який представляютьь типи соответствующих параметрів. Наприклад, в случае с класом Person второй конструктор приймає два параметра с типуми String и int. Получим цей конструктор:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Person.class;
        try{
            Constructor ctor = cl.getDeclaredConstructor(String.class, int.class);
            
            System.out.println(ctor);   // Person(java.lang.String,int)
        }
        catch(Exception ex){
            System.out.println(ex);
        }
    }
}

Отримання параметрів

За допомогою методу getParameterTypes() можна отримати масив параметрів, где каждый параметр представляє тип Class:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Person.class;
        Constructor[] ctors = cl.getDeclaredConstructors();
        for(Constructor ctor : ctors){

            // отримуємо ім'я конструктора
            String name = ctor.getName();
            System.out.print(name + "(");

            // отримуємо типи параметрів
            Class[] params = ctor.getParameterTypes();
            for(Class p : params){  
                System.out.printf("%s, ", p.getName());
            }

            System.out.println(")");
        }
    }
}

Консольный вивід:


Person()
Person(java.lang.String, int, )

Створення об'єкта

За допомогою методу Object newInstance(Object... initargs) можна створити новый об'єкт класа:


public T newInstance(Object... initargs)
              throws InstantiationException,
                    IllegalAccessException,
                    IllegalArgumentException,
                    InvocationTargetException

Параметри методу представляютьь набор об'єктов, которые предоставляют значення параметрам конструктора. Наприклад, створимо об'єкт Person:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Person.class;
        try{
            Constructor ctor = cl.getDeclaredConstructor(String.class, int.class);
            Person person = (Person) ctor.newInstance("Sam", 23);   
            person.print();     // Person. Name: Sam;  Age: 23
        }
        catch(Exception ex){
            System.out.println(ex);
        }
    }
}

Здесь сначала отримуємо конструктор с двумя параметрами:

Constructor ctor = cl.getDeclaredConstructor(String.class, int.class);

А затем використовуємо полученный конструктор, передавая в метод newInstance() параметри для него

Person person = (Person) ctor.newInstance("Sam", 23);
