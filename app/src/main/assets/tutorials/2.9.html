Консольне введення/виведення в Java

Останнє оновлення: 16.09.2025

Наиболїї простой способ взаимодействия с пользователем представляє консоль: мы можемо виводитьь на консоль деяку информацию или, наоборот, считывать с консоли некоторые дані. Для взаимодействия с консолью в Java застосовується клас System, а его функциональность собственно забезпечує консольный ввод и вивід.

Виведення на консоль

Для створенняя потіка вивіда в клас System визначений об'єкт out. В цем об'єкте визначений метод println, який дозволяє вывести на консоль некоторое значення с понаступним переводом курсора консоли на следующую рядок. Наприклад:


class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Hello world!");
		System.out.println("Bye world...");
    }
}

В метод println передаєся любое значення, як правило, рядок, которое треба вывести на консоль. И у даному випадку мы получим следующий вивід:


Hello world!
Bye world...

При необхідности можна и не переводить курсор на следующую рядок. В цем случае можна використовувати метод System.out.print(), який аналогичен println за тем винятокм, что не осуществляет перевода на следующую рядок.


class Program {
  
    public static void main(String[] args) {
          
        System.out.print("Hello world!");
		System.out.print("Bye world...");
    }
}

Консольний висновок цієї програми:


Hello world!Bye world...

Но за допомогою методу System.out.print також можна осуществить перевод каретки на следующую рядок. Для цього треба використовувати escape-післядовательность \n:


System.out.print("Hello world \n");

Нерідко необхідно підставляти у рядок якісь дані. Наприклад, у нас є два числа, і ми хочемо вивести їх значення на екрані. І тут ми можемоо, наприклад, написати так:


class Program {
  
    public static void main(String[] args) {
          
        int x=5;
		int y=6;
		System.out.println("x=" + x + "; y=" + y);
    }
}

Консольне виведення програми:


x=5; y=6

Но в Java есть також функція для форматированного вивіда, унаследованная от языка С: System.out.printf(). С її помощью мы можемо переписать предыдущий пример наступним чином:


int x=5;
int y=6;
System.out.printf("x=%d; y=%d \n", x, y);

У даному випадку символи %d обозначают спецификатор, вместо которого підставляет один из аргументов. Спецификаторов и соответствующих им аргументов может быть множина. У даному випадку у нас лише два аргумента, поцему вместо первого %d підставляет значення змінної x, а вместо второго - значення змінної y. Сама буква d означает, что данный спецификатор буде використовуватися для вивіда целочисленных значений.

Крім специфікатора %d, ми можемоо використовувати ще ряд специфікаторів для інших типів даних:

  • %x: для вивіда шестнадцатеричных чисел

  • %f: для вивіда чисел с плавающей точкой

  • %e: для вивіда чисел в экспоненциальной форме, наприклад, 1.3e+01

  • %c: для вивіда одиночного символа

  • %s: для вивіда рядківовых значений

  • %b: для вивіда значений типу boolean

Наприклад:


class Program {
  
    public static void main(String[] args) {
          
        String name = "Tom";
		int age = 30;
		float height = 1.7f;
         
		System.out.printf("Name: %s  Age: %d  Height: %.2f \n", name, age, height);
    }
}

При вивіде чисел с плавающей точкой мы можемо указать кількість знаков після комою, для цього використовуємо спецификатор на %.2f, где .2 вказує, что після комою буде два знака. В итоге мы получим следующий вивід:


Name: Tom  Age: 30  Height: 1,70

Введення із консолі

Для получения ввода с консоли в класе System визначений об'єкт in. Однак непосредственно через об'єкт System.in не дуже удобно працювати, поцему, як правило, використовують клас Scanner, який, в свою очередь використовує System.in. Наприклад, напишем маленькую програму, которая осуществляет ввод чисел:


import java.util.Scanner;

class Program {
  
    public static void main(String[] args) {
          
        Scanner in = new Scanner(System.in);
		System.out.print("Input a number: ");
        int num = in.nextInt();
         
		System.out.printf("Your number: %d \n", num);
		in.close();
    }
}

Так як клас Scanner находится в пакеті java.util, то мы вначале его импортируем за допомогою инструкции import java.util.Scanner.

Для створенняя самого об'єкта Scanner в его конструктор передаєся об'єкт System.in. Після цього мы можемо получать вводимые значення. Наприклад, у даному випадку вначале вивідим приглашение к вводу и затем отримуємо вводимое число в змінну num.

Щоб отримати введенное число, використовується метод in.nextInt();, який повертає введенное с клавиатуры целочисленное значення.

Приклад роботи програми:


Input a number: 5
Your number: 5

Клас Scanner має ще низку методів, які дозволяють отримати введені користувачем значення:

  • next(): считывает введенную рядок до первого пробела

  • nextLine(): считывает всю введенную рядок

  • nextInt(): считывает введенное число int

  • nextDouble(): считывает введенное число double

  • nextBoolean(): считывает значення boolean

  • nextByte(): считывает введенное число byte

  • nextFloat(): считывает введенное число float

  • nextShort(): считывает введенное число short

Тобто для введення значень кожного примітивного типу клас Scanner визначено свій метод.

Наприклад, створимо програму для введення інформації про людину:


import java.util.Scanner;

class Program {
  
    public static void main(String[] args) {
          
        Scanner in = new Scanner(System.in);
        System.out.print("Input name: ");
        String name = in.nextLine();
        System.out.print("Input age: ");
        int age = in.nextInt();
		System.out.print("Input height: ");
        float height = in.nextFloat();
        System.out.printf("Name: %s  Age: %d  Height: %.2f \n", name, age, height);
		in.close();
    }
}

Тут послідовно вводяться дані типів String, int, float і всі введені дані разом виводяться на консоль. Приклад роботи програми:


Input name: Tom
Input age: 34
Input height: 1,7
Name: Tom  Age: 34  Height: 1,70

Зверніть увагу, що для введення значення типу float (те саме відноситься до типу double) застосовується число "1,7", де роздільником є кома, а чи не "1.7", де роздільником є крапка. У разі все залежить від поточної мовної локалізації системи. У моєму випадку російськомовна локалізація, відповідно вводити необхідно числа, де роздільником є кома. Те саме стосується багатьох інших локалізацій, наприклад, німецької, французької і т.д., де застосовується кома.
