Консольне введення/виведення в Java

Останнє оновлення: 16.09.2025

Наиболїї простой способ взаимодействия с пользователем представляє консоль: мы можемо выводить на консоль деяку информацию или, наоборот, считывать с консоли некоторые дані. Для взаимодействия с консолью в Java застосовується класс System, а его функциональность собственно забезпечує консольный ввод и вывод.

Виведення на консоль

Для создания потока вывода в класс System определен об'єкт out. В цем об'єкте определен метод println, який дозволяє вывести на консоль некоторое значение с понаступним переводом курсора консоли на следующую строку. Например:


class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Hello world!");
		System.out.println("Bye world...");
    }
}

В метод println передаєся любое значение, як правило, строка, которое треба вывести на консоль. И в данном случае мы получим следующий вывод:


Hello world!
Bye world...

При необхідности можна и не переводить курсор на следующую строку. В цем случае можна використовувати метод System.out.print(), який аналогичен println за тем исключением, что не осуществляет перевода на следующую строку.


class Program {
  
    public static void main(String[] args) {
          
        System.out.print("Hello world!");
		System.out.print("Bye world...");
    }
}

Консольний висновок цієї програми:


Hello world!Bye world...

Но за допомогою метода System.out.print також можна осуществить перевод каретки на следующую строку. Для цього треба використовувати escape-последовательность \n:


System.out.print("Hello world \n");

Нерідко необхідно підставляти у рядок якісь дані. Наприклад, у нас є два числа, і ми хочемо вивести їх значення на екрані. І тут ми можемоо, наприклад, написати так:


class Program {
  
    public static void main(String[] args) {
          
        int x=5;
		int y=6;
		System.out.println("x=" + x + "; y=" + y);
    }
}

Консольне виведення програми:


x=5; y=6

Но в Java есть також функция для форматированного вывода, унаследованная от языка С: System.out.printf(). С її помощью мы можемо переписать предыдущий пример наступним чином:


int x=5;
int y=6;
System.out.printf("x=%d; y=%d \n", x, y);

В данном случае символы %d обозначают спецификатор, вместо которого подставляет один из аргументов. Спецификаторов и соответствующих им аргументов может быть множество. В данном случае у нас лише два аргумента, поцему вместо первого %d подставляет значение змінної x, а вместо второго - значение змінної y. Сама буква d означает, что данный спецификатор буде використовуватися для вывода целочисленных значений.

Крім специфікатора %d, ми можемоо використовувати ще ряд специфікаторів для інших типів даних:

  • %x: для вывода шестнадцатеричных чисел

  • %f: для вывода чисел с плавающей точкой

  • %e: для вывода чисел в экспоненциальной форме, наприклад, 1.3e+01

  • %c: для вывода одиночного символа

  • %s: для вывода строковых значений

  • %b: для вывода значений типу boolean

Наприклад:


class Program {
  
    public static void main(String[] args) {
          
        String name = "Tom";
		int age = 30;
		float height = 1.7f;
         
		System.out.printf("Name: %s  Age: %d  Height: %.2f \n", name, age, height);
    }
}

При выводе чисел с плавающей точкой мы можемо указать количество знаков после комою, для цього используем спецификатор на %.2f, где .2 указывает, что после комою буде два знака. В итоге мы получим следующий вывод:


Name: Tom  Age: 30  Height: 1,70

Введення із консолі

Для получения ввода с консоли в классе System определен об'єкт in. Однак непосредственно через об'єкт System.in не очень удобно работать, поцему, як правило, используют класс Scanner, який, в свою очередь использует System.in. Наприклад, напишем маленькую программу, которая осуществляет ввод чисел:


import java.util.Scanner;

class Program {
  
    public static void main(String[] args) {
          
        Scanner in = new Scanner(System.in);
		System.out.print("Input a number: ");
        int num = in.nextInt();
         
		System.out.printf("Your number: %d \n", num);
		in.close();
    }
}

Так як класс Scanner находится в пакеті java.util, то мы вначале его импортируем за допомогою инструкции import java.util.Scanner.

Для создания самого об'єкта Scanner в его конструктор передаєся об'єкт System.in. После цього мы можемо получать вводимые значения. Наприклад, в данном случае вначале выводим приглашение к вводу и затем получаем вводимое число в переменную num.

Щоб отримати введенное число, використовується метод in.nextInt();, який повертає введенное с клавиатуры целочисленное значение.

Приклад роботи програми:


Input a number: 5
Your number: 5

Клас Scanner має ще низку методів, які дозволяють отримати введені користувачем значення:

  • next(): считывает введенную строку до первого пробела

  • nextLine(): считывает всю введенную строку

  • nextInt(): считывает введенное число int

  • nextDouble(): считывает введенное число double

  • nextBoolean(): считывает значение boolean

  • nextByte(): считывает введенное число byte

  • nextFloat(): считывает введенное число float

  • nextShort(): считывает введенное число short

Тобто для введення значень кожного примітивного типу клас Scanner визначено свій метод.

Наприклад, створимо програму для введення інформації про людину:


import java.util.Scanner;

class Program {
  
    public static void main(String[] args) {
          
        Scanner in = new Scanner(System.in);
        System.out.print("Input name: ");
        String name = in.nextLine();
        System.out.print("Input age: ");
        int age = in.nextInt();
		System.out.print("Input height: ");
        float height = in.nextFloat();
        System.out.printf("Name: %s  Age: %d  Height: %.2f \n", name, age, height);
		in.close();
    }
}

Тут послідовно вводяться дані типів String, int, float і всі введені дані разом виводяться на консоль. Приклад роботи програми:


Input name: Tom
Input age: 34
Input height: 1,7
Name: Tom  Age: 34  Height: 1,70

Зверніть увагу, що для введення значення типу float (те саме відноситься до типу double) застосовується число "1,7", де роздільником є кома, а чи не "1.7", де роздільником є крапка. У разі все залежить від поточної мовної локалізації системи. У моєму випадку російськомовна локалізація, відповідно вводити необхідно числа, де роздільником є кома. Те саме стосується багатьох інших локалізацій, наприклад, німецької, французької і т.д., де застосовується кома.
