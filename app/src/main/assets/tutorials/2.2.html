Перетворення базових типів даних

Останнє оновлення: 16.09.2025

Кожен базовий тип даних займає певну кількість байт пам'яті. Це накладає обмеження на операції, які залучені різні типи даних. Розглянемо такий приклад:


int a = 4;
byte b = a;	// ! Ошибка

У цьому коді ми зіткнемося з помилкою. Хоча і тип byte, і тип int репрезентують цілі числа. Більше того, значення змінної a, яке надається змінної типу byte, цілком укладається в діапазон значень для типу byte (від -128 до 127). Тим не менш, ми стикаємося з помилкою на етапі компіляції. Оскільки в даному випадку ми намагаємося привласнити деякі дані, які займають 4 байти, змінної, яка займає лише один байт.

Тем не менїї в програме может потребоваться, чтобы підобное преобразование буво выполнено. В цем случае необхідно використовувати операцію преобразования типів (операція ()):


int a = 4;
byte b = (byte)a;  // преобразование типів: от типу int к типу byte
System.out.println(b); // 4

Операція преобразования типів предполагает указание в скобках того типу, к которому треба преобразовать значення. Наприклад, в случае операції (byte)a, идет преобразование данных типу int в тип byte. В итоге мы получим значення типу byte.

Явні та неявні перетворення

Коли в одній операції залучені дані різних типів, не завжди потрібно використовувати операцію перетворення типів. Деякі види перетворень виконуються неявно автоматично.

Автоматичні перетворення

Стрілки на малюнку покизують, які перетворення типів можуть виконуватися автоматично. Пунктирними стрілками покизані автоматичні перетворення із втратою точності.

Автоматически без яких-або проблем производятся розширювальні перетворення (widening) - они расширяют представление об'єкта в памяти. Наприклад:


byte b = 7;
int d = b;	// преобразование от byte к int

В даному випадку значення типу byte, яке займає в пам'яті 1 байт, розширюється до типу int, який займає 4 байти.

Розширювальні автоматичні перетворення представлені такими ланцюжками:

byte -> short -> int -> long

int -> double

short -> float -> double

char -> int

Автоматичні перетворення із втратою точності

Некоторые преобразования можуть производиться автоматически між типуми данных одинаковой разрядности або даже от типу данных с большей разрядностью к типу с меньшей разрядностью. Це следующие цепочки преобразований: int -> float, long -> float и long -> double. Они производятся без ошибок, но при преобразовании мы можемо столкнуться с потерей информации.

Наприклад:


int a = 2147483647;
float b = a;			// от типу int к типу float
System.out.println(b);	// 2.14748365E9

Явні перетворення

В інших перетвореннях примітивних типів явним чином застосовується операція перетворення типів. Зазвичай це зввження перетворення (narrowing) від типу із більшою розрядністю до типу із меншою розрядністю:


long a = 4;
int b = (int) a;

Втрата даних під час перетворення

При застосуванні явних змін ми можемоо зіткнутися з втратою даних. Наприклад, у наступному коді ми не матимемо жодних проблем:


int a = 5;
byte b = (byte) a;
System.out.println(b);		// 5

Число 5 цілком укладається в діапазон значень типу byte, тому після перетворення змінна b дорівнюватиме 5. Але що буде в наступному випадку:


int a = 258;
byte b = (byte) a;
System.out.println(b);		// 2

Результатом буде число 2. У цьому випадку число 258 поза діапазоном типу byte (від -128 до 127), тому відбудеься усічення значення. Чому результатом буде саме число 2?

Число a, которое равно 258, в двоичном системе буде равно 00000000 00000000 00000001 00000010. Значення типу byte занимают в памяти лише 8 бит. Тому двоичное представление числа int усекается до 8 правых разрядов, то есть 00000010, что в десятичной системе дает число 2.

Усічення раціональних чисел до цілих

При перетворенні значень з плаваючою точкою до цілих значень, відбувається усічення дробової частини:


double a = 56.9898;
int b = (int)a;

Тут значення числа b дорівнюватиме 56, незважаючи на те, що число 57 було б ближче до 56.9898. Щоб уникнути підібних казусів, треба застосовувати функцію округлення, яка є у математичній бібліотеці Java:


import java.lang.Math;  // для Math.round

public class Program {
  
    public static void main(String[] args) {
          
        double a = 56.9898;
        int b = (int)Math.round(a);
        System.out.println(b);  // 57
    }
}

Перетворення під час операцій

Непоодинокі ситуації, коли доводиться застосовувати різні операції, наприклад, додавання і твір, над значеннями різних типів. Тут також діють деякі правила:

  • якщо один из операндов операції относится к типу double, то и второй операнд преобразуется к типу double

  • якщо предыдущїї условие не соблюдено, а один из операндов операції относится к типу float, то и второй операнд преобразуется к типу float

  • якщо предыдущие условия не соблюдены, один из операндов операції относится к типу long, то и второй операнд преобразуется к типу long

  • иначе все операнды операції преобразуются к типу int

Приклади перетворень:


int a = 3;
double b = 4.6;
double c = a+b;

Так як в операції участвует значення типу double, то и другое значення приводится к типу double и сумма двух значений a+b буде представлять тип double.

Інший приклад:


byte a = 3;
short b = 4;
byte c = (byte)(a+b);

Две змінних типу byte и short (не double, float або long), поцему при сложении они преобразуются к типу int, и их сумма a+b представляє значення типу int. Тому якщо затем мы присваиваем эту сумму змінної типу byte, то нам опять треба сделать преобразование типів к byte.

Якщо операціях беруть участь дані типу char, всі вони перетворюються на int:


int d = 'a' + 5;
System.out.println(d);	// 102
