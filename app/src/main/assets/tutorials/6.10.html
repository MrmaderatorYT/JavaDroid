Сериализация

Останнє оновлення: 26.04.2018

Сериализация представляє процес записи состояния об'єкта в потік, соответственно процес извлечения або восстановления состояния об'єкта из потіка називається десериализацией. Сериализация дуже удобна, когда идет работа со сложными об'єктами.

Інтерфейс Serializable

Сразу треба сказать, что сериализовать можна лише те об'єкты, которые реалізують інтерфейс Serializable. Цет інтерфейс не визначає нияких методів, просто он служит указателем системе, что об'єкт, реализующий его, может быть сериализован.

Сериализация. Клас ObjectOutputStream

Для сериализации об'єктов в потік використовується клас ObjectOutputStream. Он записує дані в потік.

Для створенняя об'єкта ObjectOutputStream в конструктор передаєся потік, в який производится запис:


ObjectOutputStream(OutputStream out)

Для записи данных ObjectOutputStream використовує ряд методів, среди которых можна выделить следующие:

  • void close(): закрывает потік

  • void flush(): очищает буфер и сбрасывает его вміст в выходной потік

  • void write(byte[] buf): записує в потік масив байтов

  • void write(int val): записує в потік один младший байт из val

  • void writeBoolean(boolean val): записує в потік значення boolean

  • void writeByte(int val): записує в потік один младший байт из val

  • void writeChar(int val): записує в потік значення типу char, представленное целочисленным значенням

  • void writeDouble(double val): записує в потік значення типу double

  • void writeFloat(float val): записує в потік значення типу float

  • void writeInt(int val): записує целочисленное значення int

  • void writeLong(long val): записує значення типу long

  • void writeShort(int val): записує значення типу short

  • void writeUTF(String str): записує в потік рядок в кодировке UTF-8

  • void writeObject(Object obj): записує в потік отдельный об'єкт

Эти методи охватывают весь спектр данных, которые можна сериализовать.

Наприклад, сохраним в файл один об'єкт класа Person:


import java.io.*;

public class Program {

    public static void main(String[] args) {
        
        try(ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("person.dat")))
        {
			Person p = new Person("Sam", 33, 178, true);
            oos.writeObject(p);
        }
        catch(Exception ex){
            
            System.out.println(ex.getMessage());
        } 
    } 
}
class Person implements Serializable{
     
    private String name;
    private int age;
    private double height;
    private boolean married;
     
    Person(String n, int a, double h, boolean m){
         
        name=n;
        age=a;
        height=h;
        married=m;
    }
	String getName() {return name;}
	int getAge(){ return age;}
	double getHeight(){return height;}
	boolean getMarried(){return married;}
}

Десериализация. Клас ObjectInputStream

Клас ObjectInputStream отвечает за обратный процес - читання ранїї сериализованных данных из потіка. В конструкторе он приймає ссылку на потік ввода:


ObjectInputStream(InputStream in)

Функционал ObjectInputStream сосредоточен в методух, призначенийных для читання различных типів данных. Рассмотрим основные методи цього класа:

  • void close(): закрывает потік

  • int skipBytes(int len): пропускает при чтении декілька байт, кількість которых равно len

  • int available(): повертає кількість байт, доступных для читання

  • int read(): считывает из потіка один байт и повертає его целочисленное представление

  • boolean readBoolean(): считывает из потіка одно значення boolean

  • byte readByte(): считывает из потіка один байт

  • char readChar(): считывает из потіка один символ char

  • double readDouble(): считывает значення типу double

  • float readFloat(): считывает из потіка значення типу float

  • int readInt(): считывает целочисленное значення int

  • long readLong(): считывает значення типу long

  • short readShort(): считывает значення типу short

  • String readUTF(): считывает рядок в кодировке UTF-8

  • Object readObject(): считывает из потіка об'єкт

Наприклад, извлечем выше сохраненный об'єкт Person из файлу:


import java.io.*;

public class Program {

    public static void main(String[] args) {
       
        try(ObjectInputStream ois = new ObjectInputStream(new FileInputStream("person.dat")))
        {
            Person p=(Person)ois.readObject();
			System.out.printf("Name: %s \t Age: %d \n", p.getName(), p.getAge());
        }
        catch(Exception ex){
            
            System.out.println(ex.getMessage());
        }
    } 
}

Теперь совместим сохранение и восстановление из файлу на примере списку об'єктов:


import java.io.*;
import java.util.ArrayList;
 
public class Program {
	
	//@SuppressWarnings("unchecked")
    public static void main(String[] args) {
        
        String filename = "people.dat";
        // створимо список об'єктов, которые будем записывать
        ArrayList<Person> people = new ArrayList<Person>();
        people.add(new Person("Tom", 30, 175, false));
        people.add(new Person("Sam", 33, 178, true));
         
        try(ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(filename)))
        {
            oos.writeObject(people);
            System.out.println("File has been written");
        }
        catch(Exception ex){
             
            System.out.println(ex.getMessage());
        } 
         
        // десериализация в новый список
        ArrayList<Person> newPeople= new ArrayList<Person>();
        try(ObjectInputStream ois = new ObjectInputStream(new FileInputStream(filename)))
        {
			
			newPeople=((ArrayList<Person>)ois.readObject());
        }
        catch(Exception ex){
             
            System.out.println(ex.getMessage());
        } 
         
        for(Person p : newPeople)
            System.out.printf("Name: %s \t Age: %d \n", p.getName(), p.getAge());
    } 
}
class Person implements Serializable{
     
    private String name;
    private int age;
    private double height;
    private boolean married;
     
    Person(String n, int a, double h, boolean m){
         
        name=n;
        age=a;
        height=h;
        married=m;
    }
	String getName() {return name;}
	int getAge(){ return age;}
	double getHeight(){return height;}
	boolean getMarried(){return married;}
}

Виняток данных из сериализации

По умолчанию сериализуются все змінні об'єкта. Однак, возможна, мы хочемо, чтобы некоторые поля буви исключены из сериализации. Для цього они повинні быть оголошенийы с модификатором transient. Наприклад, исключим из сериализации об'єкта Person змінні height и married:


class Person implements Serializable{
     
    private String name;
    private int age;
    private transient double height;
    private transient boolean married;
     
    Person(String n, int a, double h, boolean m){
         
        name=n;
        age=a;
        height=h;
        married=m;
    }
	String getName() {return name;}
	int getAge(){ return age;}
	double getHeight(){return height;}
	boolean getMarried(){return married;}
}
