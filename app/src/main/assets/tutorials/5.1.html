Типи колекцій. Інтерфейс Collection

Останнє оновлення: 11.10.2025

Для хранения наборов данных в Java призначенийы массивы. Однак их не всегда удобно використовувати, прежде всего потому, что они имеют фиксированную длину. Эту проблему в Java решают функционал коллекций в виде Java Collections Framework - подсистема по работе с коллекциями. Однак суть не лише в гибких по размеру наборах об'єктов, но в и том, что классы коллекций реализуют различные алгоритмы и структуры данных, наприклад, такие як стек, очередь, дерево и ряд других.

Типы коллекций располагаются в пакеті java.util, поцему перед применением коллекций йде подключить данный пакет.

Хотя в Java существует множество коллекций, но все они образуют стройную и логичную систему. Під-первых, Java Collections Framework отделяет інтерфейс/определение от реализации. В основе всех коллекций лежит тот або иной інтерфейс, який определяет базовый функционал. Среди цих інтерфейсов можна выделить следующие:

  • Collection: базовый інтерфейс для всех коллекций и других інтерфейсов коллекций

  • Queue: найде інтерфейс Collection и представляє функционал для структур данных в виде очереди

  • Deque: найде інтерфейс Queue и представляє функционал для двунаправленных очередей

  • List: найде інтерфейс Collection и представляє функциональность простых списков

  • Set: також расширяет інтерфейс Collection и використовується для хранения множеств уникальных об'єктов

  • SortedSet: расширяет інтерфейс Set для создания сортированных коллекций

  • NavigableSet: расширяет інтерфейс SortedSet для создания коллекций, в которых можна осуществлять поиск по соответствию

  • Map: призначений для созданий структур данных в виде словаря, где каждый элемент имїїт определенный ключ и значение. В отличие от других інтерфейсов коллекций не найдеся от інтерфейса Collection

Эти інтерфейсы частично реализуются абстрактными классами:

  • AbstractCollection: базовый абстрактный класс для других коллекций, який применяет інтерфейс Collection

  • AbstractList: расширяет класс AbstractCollection і застосовує інтерфейс List, призначений для создания коллекций в виде списков

  • AbstractSet: расширяет класс AbstractCollection и применяет інтерфейс Set для создания коллекций в виде множеств

  • AbstractQueue: расширяет класс AbstractCollection и применяет інтерфейс Queue, призначений для создания коллекций в виде очередей и стеков

  • AbstractSequentialList: також расширяет класс AbstractList и реализует інтерфейс List. Використовується для создания связанных списков

  • AbstractMap: применяет інтерфейс Map, призначений для создания наборов по типу словаря с об'єктами в виде пары "ключ-значение"

С помощью применения вышеописанных інтерфейсов и абстрактных классов в Java реализуется широкая палитра классов коллекций - списки, множества, очереди, отображения и другие, среди которых можна выделить следующие:

  • ArrayList: простой список об'єктов

  • LinkedList: представляє связанный список

  • ArrayDeque: класс двунаправленной очереди, в которой мы можемо произвести вставку и удаление як в начале коллекции, так и в її конце

  • HashSet: набор об'єктов або хеш-множество, где каждый элемент имїїт ключ - уникальный хеш-код

  • TreeSet: набор отсортированных об'єктов в виде дерева

  • LinkedHashSet: связанное хеш-множество

  • PriorityQueue: очередь приоритетов

  • HashMap: структура данных в виде словаря, в котором каждый об'єкт имїїт уникальный ключ и некоторое значение

  • TreeMap: структура данных в виде дерева, где каждый элемент имїїт уникальный ключ и некоторое значение

Схематично всю систему коллекций вкратце можна представить наступним чином:

Интерфейс Collection

Интерфейс Collection є базовым для всех коллекций, определяя основной функционал:


public interface Collection<E> extends Iterable<E>{

	// определения методів
}

Интерфейс Collection є обобщенным и расширяет інтерфейс Iterable, поцему все об'єкты коллекций можна перебирать в цикле по типу for-each.

Среди методів інтерфейса Collection можна выделить следующие:

  • boolean add (E item): додає в коллекцию об'єкт item. При удачном добавлении повертає true, при неудачном - false

  • boolean addAll (Collection<? extends E> col): додає в коллекцию все элементы из коллекции col. При удачном добавлении повертає true, при неудачном - false

  • void clear (): видаляє все элементы из коллекции

  • boolean contains (Object item): повертає true, якщо об'єкт item міститься в коллекции, иначе повертає false

  • boolean containsAll (Collection<?> col): повертає true, якщо коллекция містить все элементы из коллекции col, иначе повертає false

  • boolean isEmpty (): повертає true, якщо коллекция пуста, иначе повертає false

  • Iterator<E> iterator (): повертає об'єкт Iterator для обхода элементов коллекции

  • boolean remove (Object item): повертає true, якщо об'єкт item удачно удален из коллекции, інакше повертається false

  • boolean removeAll (Collection<?> col): видаляє все об'єкты коллекции col из текущей коллекции. Если текущая коллекция изменилась, повертає true, інакше повертається false

  • boolean removeIf (Predicate<? super E> filter): видаляє все элементы коллекции, якщо они соответствуют предикату filter.

  • boolean retainAll (Collection<?> col): видаляє все об'єкты из текущей коллекции, кроме тех, которые містяться в коллекции col. Если текущая коллекция после удаления изменилась, повертає true, інакше повертається false

  • int size (): повертає число элементов в коллекции

  • Object[] toArray (): повертає массив, содержащий все элементы коллекции

Все эти и остальные методы, которые имеются в інтерфейсе Collection, реализуются всеми коллекциями, поцему в целом общие принципы работы с коллекциями будуть одни и те же. Единообразный інтерфейс упрощает понимание и работу с различными типуми коллекций. Так, добавление элемента буде производиться за допомогою метода add, який приймає добавляемый элемент в качестве параметра. Для удаления вызывается метод remove(). Метод clear буде очищать коллекцию, а метод size возвращать количество элементов в коллекции.
