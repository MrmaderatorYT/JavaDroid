Типи колекцій. Інтерфейс Collection

Останнє оновлення: 11.10.2025

Для хранения наборов данных в Java призначенийы масивы. Однак их не всегда удобно використовувати, прежде всего потому, что они имеют фиксированную длину. Эту проблему в Java решают функционал колекцій в виде Java Collections Framework - підсистема по работе с колекціями. Однак суть не лише в гибких по размеру наборах об'єктов, но в и том, что класы колекцій реалізують различные алгоритмы и структуры данных, наприклад, такие як стек, очередь, дерево и ряд других.

Типи колекцій располагаются в пакеті java.util, поцему перед застосуванням колекцій йде підключить данный пакет.

Хотя в Java існує множина колекцій, но все они образуют стройную и логичную систему. Під-первых, Java Collections Framework отделяет інтерфейс/визначенийие от реалізації. В основе всех колекцій лежит тот або иной інтерфейс, який визначає базовый функционал. Среди цих інтерфейсов можна выделить следующие:

  • Collection: базовый інтерфейс для всех колекцій и других інтерфейсов колекцій

  • Queue: найде інтерфейс Collection и представляє функционал для структур данных в виде очереди

  • Deque: найде інтерфейс Queue и представляє функционал для двунаправленных очередей

  • List: найде інтерфейс Collection и представляє функциональность простых списків

  • Set: також расширяет інтерфейс Collection и використовується для хранения множеств уникальных об'єктов

  • SortedSet: расширяет інтерфейс Set для створенняя сортированных колекцій

  • NavigableSet: расширяет інтерфейс SortedSet для створенняя колекцій, в которых можна осуществлять поиск по соответствию

  • Map: призначений для створенняй структур данных в виде словника, где каждый елемент имїїт визначенийный ключ и значення. В отличие от других інтерфейсов колекцій не найдеся от інтерфейса Collection

Эти інтерфейсы частично реалізуються абстрактными класами:

  • AbstractCollection: базовый абстрактный клас для других колекцій, який застосовує інтерфейс Collection

  • AbstractList: расширяет клас AbstractCollection і застосовує інтерфейс List, призначений для створенняя колекцій в виде списків

  • AbstractSet: расширяет клас AbstractCollection и застосовує інтерфейс Set для створенняя колекцій в виде множеств

  • AbstractQueue: расширяет клас AbstractCollection и застосовує інтерфейс Queue, призначений для створенняя колекцій в виде очередей и стеков

  • AbstractSequentialList: також расширяет клас AbstractList и реалізує інтерфейс List. Використовується для створенняя связанных списків

  • AbstractMap: застосовує інтерфейс Map, призначений для створенняя наборов по типу словника с об'єктами в виде пары "ключ-значення"

За допомогою применения вышеописанных інтерфейсов и абстрактных класов в Java реалізуєся широкая палитра класов колекцій - списки, множини, очереди, отображения и другие, среди которых можна выделить следующие:

  • ArrayList: простой список об'єктов

  • LinkedList: представляє связанный список

  • ArrayDeque: клас двунаправленной очереди, в которой мы можемо произвести вставку и видалення як в начале колекції, так и в її конце

  • HashSet: набор об'єктов або хеш-множина, где каждый елемент имїїт ключ - уникальный хеш-код

  • TreeSet: набор отсортированных об'єктов в виде дерева

  • LinkedHashSet: связанное хеш-множина

  • PriorityQueue: очередь приоритетов

  • HashMap: структура данных в виде словника, в котором каждый об'єкт имїїт уникальный ключ и некоторое значення

  • TreeMap: структура данных в виде дерева, где каждый елемент имїїт уникальный ключ и некоторое значення

Схематично всю систему колекцій вкратце можна представить наступним чином:

Інтерфейс Collection

Інтерфейс Collection є базовым для всех колекцій, определяя основной функционал:


public interface Collection<E> extends Iterable<E>{

	// визначенийия методів
}

Інтерфейс Collection є обобщенным и расширяет інтерфейс Iterable, поцему все об'єкты колекцій можна перебирать в цикле по типу for-each.

Среди методів інтерфейса Collection можна выделить следующие:

  • boolean add (E item): додає в колекцію об'єкт item. При удачном добавлении повертає true, при неудачном - false

  • boolean addAll (Collection<? extends E> col): додає в колекцію всі елементи из колекції col. При удачном добавлении повертає true, при неудачном - false

  • void clear (): видаляє всі елементи из колекції

  • boolean contains (Object item): повертає true, якщо об'єкт item міститься в колекції, иначе повертає false

  • boolean containsAll (Collection<?> col): повертає true, якщо колекція містить всі елементи из колекції col, иначе повертає false

  • boolean isEmpty (): повертає true, якщо колекція пуста, иначе повертає false

  • Iterator<E> iterator (): повертає об'єкт Iterator для обхода елементов колекції

  • boolean remove (Object item): повертає true, якщо об'єкт item удачно удален из колекції, інакше повертається false

  • boolean removeAll (Collection<?> col): видаляє все об'єкты колекції col из текущей колекції. Если текущая колекція изменилась, повертає true, інакше повертається false

  • boolean removeIf (Predicate<? super E> filter): видаляє всі елементи колекції, якщо они відповідають предикату filter.

  • boolean retainAll (Collection<?> col): видаляє все об'єкты из текущей колекції, кроме тех, которые містяться в колекції col. Если текущая колекція після удаления изменилась, повертає true, інакше повертається false

  • int size (): повертає число елементов в колекції

  • Object[] toArray (): повертає масив, содержащий всі елементи колекції

Все эти и остальные методи, которые имеются в інтерфейсе Collection, реалізуються всеми колекціями, поцему в целом общие принципы работы с колекціями будуть одни и те же. Единообразный інтерфейс упрощает понимание и работу с различными типуми колекцій. Так, додавання елемента буде производиться за допомогою методу add, який приймає додаємоый елемент в качестве параметра. Для удаления вызывается метод remove(). Метод clear буде очищать колекцію, а метод size возвращать кількість елементів в колекції.
