Клас File. Робота з файлуми та каталогуми

Останнє оновлення: 26.04.2018

Клас File, визначенийный в пакеті java.io, не работает напрямую с потіками. Его завданьей є управление информацией о файлух и каталогух. Хотя на уровне операционной системы файли и каталоги отличаются, но в Java они описуються одним класом File.

В зависимости от того, что повинен представлять об'єкт File - файл або каталог, мы можемо використовувати один из конструкторов для створенняя об'єкта:


File(String путь_к_каталогу)
File(String путь_к_каталогу, String ім'я_файлу)
File(File каталог, String ім'я_файлу)

Наприклад:


// створюємо об'єкт File для каталогу
File dir1 = new File("C://SomeDir");
// створюємо об'єкты для файлів, которые находятся в каталогу
File file1 = new File("C://SomeDir", "Hello.txt");
File file2 = new File(dir1, "Hello2.txt");

Клас File имїїт ряд методів, которые дозволяють управлять файлуми и каталогуми. Рассмотрим некоторые из них:

  • boolean createNewFile(): створює новый файл по пути, який передан в конструктор. В случае удачного створенняя повертає true, иначе false

  • boolean delete(): видаляє каталог або файл по пути, який передан в конструктор. При удачном удалении повертає true.

  • boolean exists(): проверяет, існує ли по указанному в конструкторе пути файл або каталог. И якщо файл або каталог існує, то повертає true, иначе повертає false

  • String getAbsolutePath(): повертає абсолютный путь для пути, переданного в конструктор об'єкта

  • String getName(): повертає краткое ім'я файлу або каталогу

  • String getParent(): повертає ім'я родительского каталогу

  • boolean isDirectory(): повертає значення true, якщо по указанному пути располагается каталог

  • boolean isFile(): повертає значення true, якщо по указанному пути находится файл

  • boolean isHidden(): повертає значення true, якщо каталог або файл є скрытыми

  • long length(): повертає размер файлу в байтах

  • long lastModified(): повертає час післяднего изменения файлу або каталогу. Значення представляє кількість миллисекунд, прошедших с начала эпохи Unix

  • String[] list(): повертає масив файлів и підкаталогов, которые находятся в визначенийном каталогу

  • File[] listFiles(): повертає масив файлів и підкаталогов, которые находятся в визначенийном каталогу

  • boolean mkdir(): створює новый каталог и при удачном створенні повертає значення true

  • boolean renameTo(File dest): переименовывает файл або каталог

Работа с каталогуми

Если об'єкт File представляє каталог, то его метод isDirectory() повертає true. И поцему мы можемо отримати его вміст - вложенные підкаталоги и файли за допомогою методів list() и listFiles(). Получим все підкаталоги и файли в визначенийном каталогу:


import java.io.File;

public class Program {

    public static void main(String[] args) {
        
        // определяем об'єкт для каталогу
        File dir = new File("C://SomeDir");
        // якщо об'єкт представляє каталог
        if(dir.isDirectory())
        {
            // отримуємо все вложенные об'єкты в каталогу
            for(File item : dir.listFiles()){
             
                 if(item.isDirectory()){
                     
                     System.out.println(item.getName() + "  \t folder");
                 }
                 else{
                     
                     System.out.println(item.getName() + "\t file");
                 }
             }
        }
    }
}

Теперь виконаємо ще ряд операцій с каталогуми, як видалення, переименование и створення:


import java.io.File;

public class Program {

    public static void main(String[] args) {
        
        // определяем об'єкт для каталогу
        File dir = new File("C://SomeDir//NewDir");
        boolean created = dir.mkdir();
        if(created)
            System.out.println("Folder has been created");
        // переименуем каталог
        File newDir = new File("C://SomeDir//NewDirRenamed");
        dir.renameTo(newDir);
        // удалим каталог
        boolean deleted = newDir.delete();
        if(deleted)
            System.out.println("Folder has been deleted");
    } 
}

Работа с файлуми

Работа с файлуми аналогична работе с каталогу. Наприклад, получим дані по одному из файлів и створимо ще один файл:


import java.io.File;
import java.io.IOException;

public class Program {

    public static void main(String[] args) {
        
        // определяем об'єкт для каталогу
        File myFile = new File("C://SomeDir//notes.txt");
        System.out.println("File name: " + myFile.getName());
        System.out.println("Parent folder: " + myFile.getParent());
        if(myFile.exists())
            System.out.println("File exists");
        else
            System.out.println("File not found");
        
        System.out.println("File size: " + myFile.length());
        if(myFile.canRead())
            System.out.println("File can be read");
        else
            System.out.println("File can not be read");
        
        if(myFile.canWrite())
            System.out.println("File can be written");
        else
            System.out.println("File can not be written");
        
        // створимо новый файл
        File newFile = new File("C://SomeDir//MyFile");
        try
        {
            boolean created = newFile.createNewFile();
            if(created)
                System.out.println("File has been created");
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        }  
    } 
}

При створенні нового файлу метод createNewFile() в случае неудачи выбрасывает виняток IOException, поцему нам треба его отлавливать, наприклад, в блоке try...catch, як робиться в примере выше.
