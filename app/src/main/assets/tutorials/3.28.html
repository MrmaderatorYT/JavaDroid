Інтерфейси у перетвореннях типів

Все сказане щодо перетворення типів притаманно і інтерфейсів. Наприклад, оскільки клас реалізує деякий інтерфейс, то об'єкт цього класу є об'єктом реалізованого інтерфейсу. Наприклад, розглянемо таку програму:


public class Program{
     
	public static void main(String[] args) {
         
		  Movable tesla = new Car("Tesla");
      tesla.move();
	}
}
interface Movable{

    void move();
}

// класс машины
class Car implements Movable{ 

    private String model;

    Car(String model){ this.model = model; }

    public void move(){

        System.out.println("Едем на машине");
    }

    public void print(){

        System.out.println("Модель: " + model);
    }
}

Тут клас Сar реалізує інтерфейс Movable, відповідно змінна типу Movable може зберігати посилання об'єкт типу Car.

І якщо ми хочемо звернутися до методів класу Car, які визначені не в інтерфейсі Movable, а в самому класі Car, нам треба явно виконати перетворення типів:


public class Program{
     
	public static void main(String[] args) {
         
		  Movable tesla = new Car("Tesla");
      tesla.move();
      ((Car) tesla).print();      // Модель: Tesla

      // або через вспомогательную переменную
      Car someCar = (Car) tesla;
      someCar.print();        // Модель: Tesla
	}
}

Інтерфейси як параметри та результати методів

І також як і у випадку з класами, інтерфейси можуть використовуватися як тип параметрів методу або як тип, що повертається. Як параметри методів і повертаються результатів, які представляютьь інтерфейс, можна використовувати об'єкти класів, які застосовують цей інтерфейс:


public class Program{
      
    public static void main(String[] args) {
        
        Car car = new Car();
        travel(car);

        Movable boing = new Aircraft();
        travel(boing);  // Летим на самолете
    }

    static void travel(Movable obj)  {

        System.out.println("Отправляемся в путешествие");
        obj.move();
    }
}

interface Movable{

    void move();
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}
class Aircraft implements Movable{ 

    public void move(){

        System.out.println("Летим на самолете");
    }
}

В даному випадку метод travel (умовна функція подорожі на транспорті) як параметр отримує об'єкт Movable. Це може бути машина і літак, і будь-який інший об'єкт, клас якого реалізує інтерфейс Movable. Який саме це буде об'єкт, ми не знаємо. Ми лише знаємо, що цей об'єкт має реалізувати інтерфейс Movable, отже, мати метод move.

Консольний висновок:


Отправляемся в путешествие
Едем на машине
Отправляемся в путешествие
Летим на самолете

ПРімер з поверненням об'єкта інтерфейсу:


public class Program{
      
    public static void main(String[] args) {
        
        Movable car = createMovable(true);
        car.move();

        Movable aircraft = createMovable(false);
        aircraft.move();
    }

    static Movable createMovable(boolean isCar)  {

        if(isCar) return new Car();
        return new Aircraft();
    }
}

interface Movable{

    void move();
}
// класс машины
class Car implements Movable{ 

    public void move(){

        System.out.println("Едем на машине");
    }
}
class Aircraft implements Movable{ 

    public void move(){

        System.out.println("Летим на самолете");
    }
}

Метод createMovable() в качестве результата повертає об'єкт інтерфейса Movable. В качестве параметра метод приймає булевое значение, которое указывает, об'єкт якого именно класса буде создаваться: якщо true, то створюєся об'єкт Car, иначе створюєся об'єкт Aircraft..
