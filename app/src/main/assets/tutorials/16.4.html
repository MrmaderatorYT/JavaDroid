Компонування пам'яті MemoryLayout та робота з MemorySegment

Останнє оновлення: 21.12.2025

Ряд методів создания сегментов памяти в классе Arena принимают в качестве параметра об'єкт типу MemoryLayout. Цет інтерфейс описує компоновку памяти - размер сегмента памяти, выравнивание, порядок байтов. Интерфейс ValueLayout (який найдеся от MemoryLayout) надає ряд констант, которые представляють встроенную компоновку памяти для определенного типу данных:

  • ADDRESS: константа компоновки адреса, размер которой совпадает с размером машинного адреса (size_t в языке С), выравнивание по байтам установлено равным sizeof(size_t)

  • ADDRESS_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером машинного адреса (size_t)

  • JAVA_BOOLEAN: константа компоновки значения, размер которой совпадает с размером типу boolean в Java, выравнивание по байтам установлено равным 1

  • JAVA_BYTE: константа компоновки значения, размер которой совпадает с размером типу byte в Java, выравнивание по байтам установлено на 1

  • JAVA_CHAR: константа компоновки значения, размер которой совпадает с размером типу char в Java, выравнивание по байтам установлено на 2

  • JAVA_CHAR_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу char в Java

  • JAVA_DOUBLE: константа компоновки значений, размер которой совпадает с размером типу double в Java, выравнивание по байтам установлено на 8

  • JAVA_DOUBLE_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу double в Java

  • JAVA_FLOAT: константа компоновки значений, размер которой совпадает с размером типу float в Java, выравнивание по байтам установлено на 4

  • JAVA_FLOAT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу float в Java

  • JAVA_INT: константа расположения значений, размер которой совпадает с размером типу int в Java, выравнивание байтов установлено на 4

  • JAVA_INT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу int в Java

  • JAVA_LONG: константа компоновки значений, размер которой совпадает с размером типу long в Java, выравнивание по байтам установлено на 8

  • JAVA_LONG_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу long в Java

  • JAVA_SHORT: константа компоновки значений, размер которой совпадает с размером типу short в Java, выравнивание по байтам установлено на 2

  • JAVA_SHORT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой совпадает с размером типу short в Java

Для выделения массива необхідно указать структуру элементов и количество элементов. Для цього можна використовувати константы компоновки памяти, определенные в классе ValueLayout. Наприклад, выделим массив значений типу double:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            // выделяем память для 16 значений double
            MemorySegment double_seg = arena.allocate(ValueLayout.JAVA_DOUBLE, 16);

            System.out.println(double_seg);   // MemorySegment{ kind: native, address: 0x7477a8184cb0, byteSize: 128 }
        } 
    }
}

Если у нас вже есть значения в массиве, можна використовувати метод allocateFrom():


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для массива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            System.out.println(double_seg);   // MemorySegment{ kind: native, address: 0x75f5cc1851a0, byteSize: 32 }
        } 
    }
}

Работа с сегментами памяти

Интерфейс MemorySegment надає множество методів для различных операций с данными в памяти. Рассмотрим некоторые из них:

  • String getString(long offset)

    преобразует строку UTF-8 в строку Java.

  • type getAtIndex(ValueLayout.oftype layout, long index)

    считывает значение примитивного типу по заданному индексу массива.

  • void setAtIndex(ValueLayout.oftype layout, long index, type value)

    записує значение примитивного типу по заданному индексу массива.

  • type get(ValueLayout.oftype layout, long offset)

    считывает значение примитивного типу по заданному смщению.

  • void set(ValueLayout.oftype layout, long offset, type value)

    записує значение примитивного типу по заданному смщению.

  • type[] toArray(ValueLayout.oftype elementLayout)

    копирует содержимое сегмента памяти в массив примитивного типу type.

Наприклад, возьмем метод getString(). Предположим, что функция C заполнила сегмент памяти данными. Щоб преобразовать строку UTF-8 в строку Java, в метод getString() передаєся смщение 0:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            String str = "Hello ";

            // преобразуем строку Java в строку UTF-8
            MemorySegment str_seg = arena.allocateFrom(str);

            System.out.println(str_seg);   // MemorySegment{ kind: native, address: 0x7aa79c1802d0, byteSize: 18 }

            // обратное преобразование в строку Java
            String java_str = str_seg.getString(0);

            System.out.println(java_str);   // Hello 
        } 
    }
}

Для чтения або записи по индексу массива указывается структура памяти и индекс:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для массива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            // устанавливаем для элемента с индексом 2 значение 5.0
            int index = 2;
            double_seg.setAtIndex(ValueLayout.JAVA_DOUBLE, index, 5.0);
            // получаем один элемент double по индексу 2
            double second_double = double_seg.getAtIndex(ValueLayout.JAVA_DOUBLE, index);

            System.out.println(second_double);  // 5.0
        } 
    }
}

Обратите внимание, что сегмент не сохраняет ниякой информации о структуре данных внутри. Вам необхідно указывать структуру данных (в данном случае ValueLayout.JAVA_DOUBLE) при каждом доступе к элементам або при их обновлении.

Також можна отримати массив Java со всеми значениями:


double[] double_nums = double_seg.toArray(ValueLayout.JAVA_DOUBLE);

Методи get()/set() аналогичны методам getAtIndex/setAtIndex, но принимают смщение в байтах:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для массива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            // устанавливаем значение 11 по смщению 8 байт
            double_seg.set(ValueLayout.JAVA_DOUBLE, 8, 11);
            // получаем один элемент double по смщению 16 байт
            double second_double = double_seg.get(ValueLayout.JAVA_DOUBLE, 16);

            System.out.println(second_double);  // 3.0
        } 
    }
}
