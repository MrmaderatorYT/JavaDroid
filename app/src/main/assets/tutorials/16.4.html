Компонування пам'яті MemoryLayout та робота з MemorySegment

Останнє оновлення: 21.12.2025

Ряд методів створенняя сегментов памяти в класе Arena приймають в качестве параметра об'єкт типу MemoryLayout. Цет інтерфейс описує компоновку памяти - размер сегмента памяти, выравнивание, порядок байтов. Інтерфейс ValueLayout (який найдеся от MemoryLayout) надає ряд констант, которые представляютьь встроенную компоновку памяти для визначенийного типу данных:

  • ADDRESS: константа компоновки адреса, размер которой збігається с размером машинного адреса (size_t в языке С), выравнивание по байтам установлено равным sizeof(size_t)

  • ADDRESS_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером машинного адреса (size_t)

  • JAVA_BOOLEAN: константа компоновки значення, размер которой збігається с размером типу boolean в Java, выравнивание по байтам установлено равным 1

  • JAVA_BYTE: константа компоновки значення, размер которой збігається с размером типу byte в Java, выравнивание по байтам установлено на 1

  • JAVA_CHAR: константа компоновки значення, размер которой збігається с размером типу char в Java, выравнивание по байтам установлено на 2

  • JAVA_CHAR_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу char в Java

  • JAVA_DOUBLE: константа компоновки значений, размер которой збігається с размером типу double в Java, выравнивание по байтам установлено на 8

  • JAVA_DOUBLE_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу double в Java

  • JAVA_FLOAT: константа компоновки значений, размер которой збігається с размером типу float в Java, выравнивание по байтам установлено на 4

  • JAVA_FLOAT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу float в Java

  • JAVA_INT: константа расположения значений, размер которой збігається с размером типу int в Java, выравнивание байтов установлено на 4

  • JAVA_INT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу int в Java

  • JAVA_LONG: константа компоновки значений, размер которой збігається с размером типу long в Java, выравнивание по байтам установлено на 8

  • JAVA_LONG_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу long в Java

  • JAVA_SHORT: константа компоновки значений, размер которой збігається с размером типу short в Java, выравнивание по байтам установлено на 2

  • JAVA_SHORT_UNALIGNED: константа компоновки адреса без выравнивания, размер которой збігається с размером типу short в Java

Для выделения масива необхідно указать структуру елементов и кількість елементів. Для цього можна використовувати константи компоновки памяти, визначенийные в класе ValueLayout. Наприклад, выделим масив значений типу double:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            // выделяем память для 16 значений double
            MemorySegment double_seg = arena.allocate(ValueLayout.JAVA_DOUBLE, 16);

            System.out.println(double_seg);   // MemorySegment{ kind: native, address: 0x7477a8184cb0, byteSize: 128 }
        } 
    }
}

Если у нас вже есть значення в масиве, можна використовувати метод allocateFrom():


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для масива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            System.out.println(double_seg);   // MemorySegment{ kind: native, address: 0x75f5cc1851a0, byteSize: 32 }
        } 
    }
}

Работа с сегментами памяти

Інтерфейс MemorySegment надає множина методів для различных операцій с данными в памяти. Рассмотрим некоторые из них:

  • String getString(long offset)

    преобразует рядок UTF-8 в рядок Java.

  • type getAtIndex(ValueLayout.oftype layout, long index)

    считывает значення примитивного типу по заданному индексу масива.

  • void setAtIndex(ValueLayout.oftype layout, long index, type value)

    записує значення примитивного типу по заданному индексу масива.

  • type get(ValueLayout.oftype layout, long offset)

    считывает значення примитивного типу по заданному смщению.

  • void set(ValueLayout.oftype layout, long offset, type value)

    записує значення примитивного типу по заданному смщению.

  • type[] toArray(ValueLayout.oftype elementLayout)

    копирует вміст сегмента памяти в масив примитивного типу type.

Наприклад, возьмем метод getString(). Предположим, что функція C заполнила сегмент памяти данными. Щоб преобразовать рядок UTF-8 в рядок Java, в метод getString() передаєся смщение 0:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            String str = "Hello ";

            // преобразуем рядок Java в рядок UTF-8
            MemorySegment str_seg = arena.allocateFrom(str);

            System.out.println(str_seg);   // MemorySegment{ kind: native, address: 0x7aa79c1802d0, byteSize: 18 }

            // обратное преобразование в рядок Java
            String java_str = str_seg.getString(0);

            System.out.println(java_str);   // Hello 
        } 
    }
}

Для читання або записи по индексу масива вказуєся структура памяти и индекс:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для масива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            // устанавливаем для елемента с индексом 2 значення 5.0
            int index = 2;
            double_seg.setAtIndex(ValueLayout.JAVA_DOUBLE, index, 5.0);
            // отримуємо один елемент double по индексу 2
            double second_double = double_seg.getAtIndex(ValueLayout.JAVA_DOUBLE, index);

            System.out.println(second_double);  // 5.0
        } 
    }
}

Зверніть увагу, что сегмент не сохраняет ниякой информации о структуре данных всередині. Вам необхідно указывать структуру данных (у даному випадку ValueLayout.JAVA_DOUBLE) при каждом доступе к елементам або при их обновлении.

Також можна отримати масив Java со всеми значеннями:


double[] double_nums = double_seg.toArray(ValueLayout.JAVA_DOUBLE);

Методи get()/set() аналогичны методум getAtIndex/setAtIndex, но приймають смщение в байтах:


import java.lang.foreign.*;

public class Program {

    public static void main(String[] args) {

        try (Arena arena = Arena.ofConfined()) {

            double[] nums = {1.0, 2.0, 3.0, 4.0};
            // выделяем память для масива nums
            MemorySegment double_seg = arena.allocateFrom(ValueLayout.JAVA_DOUBLE, nums);

            // устанавливаем значення 11 по смщению 8 байт
            double_seg.set(ValueLayout.JAVA_DOUBLE, 8, 11);
            // отримуємо один елемент double по смщению 16 байт
            double second_double = double_seg.get(ValueLayout.JAVA_DOUBLE, 16);

            System.out.println(second_double);  // 3.0
        } 
    }
}
