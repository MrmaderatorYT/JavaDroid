Багатопоточне програмування та клас Thread

Останнє оновлення: 14.10.2025

Большинство языков программирования поддерживают такую важную функциональность як многопоточность, и Java в цем плане не исключение. При помощи многопоточности мы можемо выделить в приложении декілька потоков, которые будуть выполнять различные задачи одновременно. Если у нас, допустим, графическое приложение, которое посылает запрос к якому-нибудь серверу або считывает и обробляє огромный файл, то без многопоточности у нас бы блокировался графический інтерфейс на час виконання задачи. А благодаря потокам мы можемо выделить отправку запроса або любую другую задачу, которая может долго оброблятися, в отдельный поток. Тому большинство реальных приложений, которые многим из нас приходится використовувати, практически не мыслимы без многопоточности.

Варто відзначити, что начиная с версии 21 в мові Java есть два типу потоков: потоки платформы (platform thread) и виртуальные потоки (virtual thread). Поток платформы представляє поток, предоставляемый операционной системой, которая планирует его выполнение. Віртуальні потоки выполняются в потоках платформы и планируются средой виконання Java.

Каждый процессор может одновременно выполнять один поток платформы. На машине с несколькими процессорами можна запускать декілька потоков параллельно. Если количество потоков платформы больше, чем количество доступных процессоров, то каждому запущенному потоку выделяется определенный промежуток времени для виконання своей задачи. Когда цей промежуток времени исчерпывается, операционная система вытесняет поток и надає возможнасть іншому потоку выполнить его.

Класс Thread

В мові Java за работу с потоками отвечает класс Thread. Фактически цей класс и представляє функциональность потока. И чтобы создать новый поток, нам треба создать об'єкт цього класса.

Но все потоки необхідно создавать вручую. Когда запускаєся программа, начинает работать главный поток цей программы, в котором и виконується метод main(). От цього главного потока порождаются все остальные дочерние потоки.

Получение текущего потока

С помощью статического метода Thread.currentThread() мы можемо отримати текущий поток виконання в виде об'єкта Thread:


public class Program {
  
    public static void main(String[] args) {
          
        // получаем текущий поток
        Thread t = Thread.currentThread(); 
        System.out.println(t); // Thread[#3,main,5,main]
    }
}

В данном случае мы получаем текущий поток, в котором запущен метод main, в переменную t и выводим его на консоль. В итоге мы получим консольный вывод следующего вида:

Thread[#3,main,5,main]

Что значит цей вывод? Цет вывод сформирован за допомогою метода toString() и означает следующїї:

идентификатор_потока, имя_потока, приоритет, группа потока

То есть мы получаем:

  • идентификатор потока: 3

  • имя потока: main

  • приоритет потока: 5

  • группа потока: main

Варто відзначити, что за допомогою статического метода Thread.getAllStackTraces() мы можемо отримати трассировку стека для всех активных потоков в программе в виде словаря Map<Thread, StackTraceElement[]>. Наприклад, выведем на консоль все активные потоки:


public class Program {
  
    public static void main(String[] args) {
          
        // получаем лише потоки без трасировки стека
        var threads = Thread.getAllStackTraces().keySet(); 

        for(var thread : threads){
            System.out.println(thread); 
        }
    }
}

В цем случае мы получим консольный вывод что-то вроде следующего:


Thread[#3,main,5,main]
Thread[#15,Reference Handler,10,system]
Thread[#16,Finalizer,8,system]
Thread[#32,Notification Thread,9,system]
Thread[#17,Signal Dispatcher,9,system]
Thread[#33,Common-Cleaner,8,InnocuousThreadGroup]

Получение информации о потоках

Класс Thread надає ряд методів для управления и получения состояния потоков. Отмечу некоторые из цих методів:

  • String getName(): повертає имя потока

  • void setName(String name): устанавливает имя потока

  • int getPriority(): повертає приоритет потока

  • void setPriority(int proirity): устанавливает приоритет потока. Приоритет є одним из ключевых факторов для выбора системой потока из кучи потоков для виконання. В цей метод в качестве параметра передаєся числовое значение приоритета - от 1 до 10. По умолчанию главному потоку выставляется средний приоритет - 5.

  • StackTraceElement[] getStackTrace(): повертає трассировку стека текущего потока в виде массива об'єктов StackTraceElement

  • void setDaemon(boolean on): устанавливает, є ли поток фоновым

  • boolean isAlive(): повертає true, якщо поток активен

  • boolean isInterrupted(): повертає true, якщо поток був прерван

  • boolean isDaemon(): повертає true, якщо поток є фоновым (потоком-демоном)

  • boolean isVirtual(): повертає true, якщо поток є виртуальным

  • long threadId(): повертає идентификатор текущего потока

И також можна выделить ряд методів по управлению потоками:

  • void join(): очікує завершение потока

  • void run(): определяет действия, которые собственно и выполняет поток

  • static void sleep(long millis): приостанавливает поток на заданное количество миллисекунд

  • start(): запускає (или ставит в очередь на запуск) поток, вызывая его метод run()

  • static Thread startVirtualThread(Runnable task): запускає або ставит в очередь на запуск виртуальный поток

Наприклад, получим информацию о текущем потоке (главном потоке программы):


public class Program {
  
    public static void main(String[] args) {
          
        // получаем текущий поток
        Thread t = Thread.currentThread(); 
        // получаем его свойства

        System.out.println("Id: " + t.threadId()); 
        System.out.println("Name: " + t.getName()); 
        System.out.println("Priority: " + t.getPriority()); 
        System.out.println("State: " + t.getState()); 
        System.out.println("isAlive: " + t.isAlive()); 
        System.out.println("isDaemon: " + t.isDaemon()); 
        System.out.println("isVirtual: " + t.isVirtual()); 
    }
}

В данном случае мы получим следующий консольный вывод:


Id: 3
Name: main
Priority: 5
State: RUNNABLE
isAlive: true
isDaemon: false
isVirtual: false

Рассмотрим некоторые свойства потоков подробнїї.

Состояние потоков

Потоки можуть находиться в следующих 6 состояниях:

  • NEW: поток создан, но ще не запущен

  • RUNNABLE: поток виконується

  • BLOCKED: поток заблокирован в ожидании блокировки монитора

  • WAITING: поток неопределенно долго очікує, поки другой поток выполнит определенное действие

  • TIMED_WAITING: поток, ожидающий виконання действия другим потоком в течение заданного времени

  • TERMINATED: поток завершил выполнение

В любой момент времени поток может находиться лише в одном состоянии. Эти состояния представляють собой состояния виртуальной машины, которые не отражают состояния потоков операционной системы.

Приоритеты потоков

Каждый поток имїїт приоритет. Приоритеты представляють числа от 1 до 10. Для их описания Java представляє три константы:

  • MIN_PRIORITY: минимальный приоритет, соответствует числу 1

  • MAX_PRIORITY: максимальный приоритет, соответствует числу 10

  • NORM_PRIORITY: стандартный приоритет, соответствует числу 5

По умолчанию поток найде приоритет потока, який его создал. Но теоретически можна увеличить або уменьшить приоритет любого потока платформы за допомогою метода setPriority(). В частности, можна установить приоритет на любое значение между MIN_PRIORITY (1) и MAX_PRIORITY (10).

Всякий раз, когда у планировщика потоков платформы есть возможнасть выбрать новый поток, он отдает предпочтение потокам с болїї высоким приоритетом. Однак приоритеты потоков сильно зависят от системы. Поскольку виртуальная машина Java опирается на реализацию потоков текущей платформы, приоритеты потоков Java сопоставляются с уровнями приоритета текущей платформы, которая может иметь больше або меньше уровней приоритета потоков.

Наприклад, в Windows имїїтся семь уровней приоритета. Некоторые приоритеты Java будуть соответствовать одному и тому же уровню операционной системы. В виртуальной машине OpenJDK для Linux приоритеты потоков вообще игнорируются — все потоки имеют одинаковый приоритет. Приоритеты потоков могли быть полезны в ранних версиях Java, которые не использовали потоки операционных систем. Вам не йде використовувати их в настоящїї час.

Для обратной совместимости все виртуальные потоки имеют приоритет NORM_PRIORITY. Так, у главного потока программы по умолчанию приоритет - 5. Попытка изменить приоритет не имїїт ниякого эффекта.
