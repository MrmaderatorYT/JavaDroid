Assert

Останнє оновлення: 07.10.2025

assertion (утвержение) представляє распространенный паттерн защитного механизма (guard), який дозволяє во час тестирования проверить соблюдение некоторых условий. Для піддержки цього механизма в мові Java есть ключевое слово assert.

Оператор assert використовується для проверки таких ошибок, при которых нет смысла продолжать програму. Но обычно цей оператор застосовується лише во час разработки и тестирования, наприклад, для поиска внутренних ошибок програми во час тестирования.

Оператор assert

Оператор assert имїїт следующие формы:


assert условие;
assert условие : выражение;

В обоих формах після оператора assert идет некоторое условие. И среда проверяет истинность цього условия. И якщо условие не соблюдается, то система генерує ошибку типу AssertionError

Під второй форме після условия через двоеточие вказуєся выражение, которое передаєся в конструктор об'єкта AssertionError и застосовується для створенняя сообщения об ошибке.

Приведу абстрактные примеры. Первая форма:


assert n > 0;

Здесь оператор assert проверяет, что некоторая змінна n больше 0. Если змінна равна 0 або меньше, что система генерує ошибку AssertionError

Вторая форма:


assert n > 0 : n;

Здесь фактически все то же самое, лише теперь значення змінної n передаєся в конструктор класа AssertionError и додаєся в сообщение об ошибке.

Включение піддержки assert

По умолчанию оператор assert не оказывает ниякого действия на работу програми. Ошибка типу AssertionError генериурется, якщо лише при выполнении програми явным чином включена піддержка assert. Для цього при запуску програми утилите java передаєся опция -enableassertions або -ea. Наприклад:

java -enableassertions Program

Також Java дозволяє підключать піддержку assert лише для визначенийных класов и пакетів:

java -ea:SomeClass -ea:com.mycompany.somelib... Program

Підвыражение -ea:SomeClass включает assert для класа SomeClass, а -ea:com.mycompany.somelib... - для пакету "com.mycompany.somelib".

Если пакет не имїїт імені (пакет по умолчанию), то для підключения assert можна використовувати следующїї выражение:

-ea:...

При необхідности також можна отключить assert в визначенийных класах и пакетух за допомогою опции -disablessertions або -da:

java -ea:... -da: SomeClass Program

Пример использования assert

Рассмотрим небольшой пример использования оператора assert:


class Program {

    public static void main(String[] args) {
        
        int f1 = factorial(4);
        System.out.println("f1: " + f1);

        int f2 = factorial(-4);
        System.out.println("f2: " + f2);
    } 

    static int factorial(int n){

        assert n > 0;

        int result = 1;
        while(n > 1) result *= n--;
        return result;
    }
}

Здесь визначений метод factorial, який вычисляет факториал числа. Само число передаєся через параметр. Но по условию факториала число повинно быть больше 0. И в самом методі мы проверяем це условие за допомогою assert:


static int factorial(int n){

    assert n > 0;
    .................

Для демонстрации в методі main() мы два раза вызываем данный метод, передавая в него корректное и некорректное значення:


int f1 = factorial(4);

int f2 = factorial(-4);

Скомпилируем програму и виконаємо її с флагом -enableassertions:


eugene@Eugene:/workspace/java$ javac Program.java && java -enableassertions Program
f1: 24
Exception in thread "main" java.lang.AssertionError
	at Program.factorial(Program.java:14)
	at Program.main(Program.java:8)

eugene@Eugene:/workspace/java$

И мы видим, что первый вызов методу с корректным значенням отработал нормально, а при втором вызове возникла ошибка AssertionError на 14-й рядківе коду. Поймав підобную ошибку при тестировании або разработке, мы увидим, что дані, которые в передаються в метод factorial(), не проверяются, и, возможна, при вызове методу додамо код проверки или якой-нибудь другой механизм, який дозволить избежать падения програми.

И дополнительно розглянемо, что буде при использовании другой формы оператора assert. Для цього изменим немного метод factorial()


static int factorial(int n){

    assert n > 0 : n;

    int result = 1;
    while(n > 1) result *= n--;
    return result;
}

У даному випадку выражение assert n > 0 : n вказує, что значення параметра n буде передаваться в сообщение об ошибке. Так, при выполнении програми мы получим немного другое сообщение:


eugene@Eugene:/workspace/java$ javac Program.java && java -enableassertions Program
f1: 24
Exception in thread "main" java.lang.AssertionError: -4
	at Program.factorial(Program.java:14)
	at Program.main(Program.java:8)

eugene@Eugene:/workspace/java$

Теперь из сообщения об ошибке мы видим, что стало причиной ошибки. Ну и естественно мы можемо визначати болїї підробные сообщения. Наприклад:

assert n > 0 : "Параметр менше 1";
