Assert

Останнє оновлення: 07.10.2025

assertion (утвержение) представляє распространенный паттерн защитного механизма (guard), який дозволяє во час тестирования проверить соблюдение некоторых условий. Для поддержки цього механизма в мові Java есть ключевое слово assert.

Оператор assert використовується для проверки таких ошибок, при которых нет смысла продолжать программу. Но обычно цей оператор застосовується лише во час разработки и тестирования, наприклад, для поиска внутренних ошибок программы во час тестирования.

Оператор assert

Оператор assert имїїт следующие формы:


assert условие;
assert условие : выражение;

В обоих формах после оператора assert идет некоторое условие. И среда проверяет истинность цього условия. И якщо условие не соблюдается, то система генерує ошибку типу AssertionError

Під второй форме после условия через двоеточие указывается выражение, которое передаєся в конструктор об'єкта AssertionError и застосовується для создания сообщения об ошибке.

Приведу абстрактные примеры. Первая форма:


assert n > 0;

Здесь оператор assert проверяет, что некоторая переменная n больше 0. Если переменная равна 0 або меньше, что система генерує ошибку AssertionError

Вторая форма:


assert n > 0 : n;

Здесь фактически все то же самое, лише теперь значение змінної n передаєся в конструктор класса AssertionError и додаєся в сообщение об ошибке.

Включение поддержки assert

По умолчанию оператор assert не оказывает ниякого действия на работу программы. Ошибка типу AssertionError генериурется, якщо лише при выполнении программы явным чином включена поддержка assert. Для цього при запуске программы утилите java передаєся опция -enableassertions або -ea. Например:

java -enableassertions Program

Також Java дозволяє подключать поддержку assert лише для определенных классов и пакетів:

java -ea:SomeClass -ea:com.mycompany.somelib... Program

Подвыражение -ea:SomeClass включает assert для класса SomeClass, а -ea:com.mycompany.somelib... - для пакета "com.mycompany.somelib".

Если пакет не имїїт имени (пакет по умолчанию), то для подключения assert можна використовувати следующїї выражение:

-ea:...

При необхідности також можна отключить assert в определенных классах и пакетах за допомогою опции -disablessertions або -da:

java -ea:... -da: SomeClass Program

Пример использования assert

Рассмотрим небольшой пример использования оператора assert:


class Program {

    public static void main(String[] args) {
        
        int f1 = factorial(4);
        System.out.println("f1: " + f1);

        int f2 = factorial(-4);
        System.out.println("f2: " + f2);
    } 

    static int factorial(int n){

        assert n > 0;

        int result = 1;
        while(n > 1) result *= n--;
        return result;
    }
}

Здесь определен метод factorial, який вычисляет факториал числа. Само число передаєся через параметр. Но по условию факториала число повинно быть больше 0. И в самом методе мы проверяем це условие за допомогою assert:


static int factorial(int n){

    assert n > 0;
    .................

Для демонстрации в методе main() мы два раза вызываем данный метод, передавая в него корректное и некорректное значения:


int f1 = factorial(4);

int f2 = factorial(-4);

Скомпилируем программу и выполним її с флагом -enableassertions:


eugene@Eugene:/workspace/java$ javac Program.java && java -enableassertions Program
f1: 24
Exception in thread "main" java.lang.AssertionError
	at Program.factorial(Program.java:14)
	at Program.main(Program.java:8)

eugene@Eugene:/workspace/java$

И мы видим, что первый вызов метода с корректным значением отработал нормально, а при втором вызове возникла ошибка AssertionError на 14-й строке кода. Поймав подобную ошибку при тестировании або разработке, мы увидим, что дані, которые в передаются в метод factorial(), не проверяются, и, возможна, при вызове метода добавим код проверки или якой-нибудь другой механизм, який позволит избежать падения программы.

И дополнительно розглянемо, что буде при использовании другой формы оператора assert. Для цього изменим немного метод factorial()


static int factorial(int n){

    assert n > 0 : n;

    int result = 1;
    while(n > 1) result *= n--;
    return result;
}

В данном случае выражение assert n > 0 : n указывает, что значение параметра n буде передаваться в сообщение об ошибке. Так, при выполнении программы мы получим немного другое сообщение:


eugene@Eugene:/workspace/java$ javac Program.java && java -enableassertions Program
f1: 24
Exception in thread "main" java.lang.AssertionError: -4
	at Program.factorial(Program.java:14)
	at Program.main(Program.java:8)

eugene@Eugene:/workspace/java$

Теперь из сообщения об ошибке мы видим, что стало причиной ошибки. Ну и естественно мы можемо визначати болїї подробные сообщения. Например:

assert n > 0 : "Параметр меньше 1";
