Лямбди як параметри та результати методів

Останнє оновлення: 02.10.2025

Лямбды як параметры методів

Одним из преимуществ лямбд в мові Java є то, что их можна передавать в качестве параметров в методы. Рассмотрим пример:


public class Program {

    public static void main(String[] args) {
         
        doOperation(10, 4, (a, b) -> a + b);  // 14
        doOperation(10, 4, (a, b) -> a - b);  // 6
        doOperation(10, 4, (a, b) -> a * b);  // 40
    } 

    static void doOperation(int a, int b, Operation op)
    {
        System.out.println(op.execute(a, b));
    }
}
interface Operation{
    int execute(int a, int b);
}

Здесь для представления лямбда-выражений застосовується функциональный інтерфейс Operation с методом execute(), який приймає два числа, повинен выполнять над ними некоторые вычисления и повертає число - результат вычислений:


interface Operation{
    int execute(int a, int b);
}

В программе определен метод doOperation, який приймає два числа и об'єкт інтерфейса Operation - то есть деяку операцию над двумя числами:


static void doOperation(int a, int b, Operation op)
{
    System.out.println(op.execute(a, b));
}

То есть на момент определения метода мы точно не знаем, что це буде за операция. Мы лише знаем, что она приймає два числа и повертає число.

В методе main вызываем декілька раз метод doOperation, передавая в него через лямбда-выражения разные операции. При цем можна не визначати переменную інтерфейса, а відразу передать в метод лямбда-выражение:


doOperation(10, 4, (a, b) -> a + b);  // 14
doOperation(10, 4, (a, b) -> a - b);  // 6
doOperation(10, 4, (a, b) -> a * b);  // 40

Це особенно удобно тем, что нам не треба дополнительно создавать переменную для хранения лямбда-выражения, которое, возможна, использовалось бы один раз. Хотя при необхідности також можна буво бы создать переменную інтерфейса Operation и передать її в метод.

Другой пример:


public class Program {

    public static void main(String[] args) {
        
        Expression func = (n)-> n%2==0;
        int[] nums = { 1, 2, 3, 4, 5, 6, 7, 8, 9 };
        System.out.println(sum(nums, func)); // 20
    } 
    static int sum (int[] numbers, Expression func)
    {
        int result = 0;
        for(int i : numbers)
        {
            if (func.isEqual(i))
                result += i;
        }
        return result;
    }
}

interface Expression{
    boolean isEqual(int n);
}

Функциональный інтерфейс Expression определяет метод isEqual(), який повертає true, якщо в отношении числа n действует якое-нибудь равенство.

В основном классе программы определяется метод sum(), який вычисляет сумму всех элементов массива, соответствующих некоторому условию. А само условие передаєся через параметр Expression func. Причем на момент написания метода sum мы можемо абсолютно не знать, якое именно условие буде використовуватися. Само же условие определяется в виде лямбда-выражения:

Expression func = (n)-> n%2==0;

То есть в данном случае все числа повинні быть четными або остаток от их деления на 2 повинен быть равен 0. Затем це лямбда-выражение передаєся в вызов метода sum.

В методе main применяем метод sum, передавая в него лямбда-выражение:


int[] nums = { 1, 2, 3, 4, 5, 6, 7, 8, 9 };
int x = sum(nums, (n)-> n > 5); // сумма чисел, которые больше 5
System.out.println(x);	// 30

Лямбды як результат методів

Також метод в Java может возвращать лямбда-выражение. Рассмотрим следующий пример:


interface Operation{
    int execute(int a, int b);
}

public class Program {
 
    public static void main(String[] args) {
         
        Operation func = action(1);
		int a = func.execute(6, 5);
		System.out.println(a);			// 11
		
		int b = action(2).execute(8, 2);
        System.out.println(b); 			// 6
    } 
	
	private static Operation action(int number){
		switch(number){
			case 1: return (x, y) -> x + y; 
			case 2: return (x, y) -> x - y; 
			case 3: return (x, y) -> x * y; 
			default: return (x,y) -> 0;
		}
	}
}

В данном случае определен функциональный інтерфейс Operation, в котором метод execute приймає два значения типу int и повертає значение типу int.

Метод action приймає в качестве параметра число и в зависимости от его значения повертає то або иное лямбда-выражение. Оно может представлять або сложение, або вычитание, або умножение, або просто повертає 0. Варто учитывать, что формально возвращаемым типом метода action є інтерфейс Operation, а возвращаемое лямбда-выражение повинно соответствовать цему інтерфейсу.

В методе main мы можемо вызвать цей метод action. Наприклад, сначала отримати его результат - лямбда-выражение, которое присвоюється змінної Operation. А затем через метод execute выполнить це лямбда-выражение:


Operation func = action(1);
int a = func.execute(6, 5);
System.out.println(a);			// 11

Або можна відразу отримати и тут же выполнить лямбда-выражение:


int b = action(2).execute(8, 2);
System.out.println(b); 			// 6
