Модульний jar-файл

Останнє оновлення: 06.11.2025

Упаковка функционала пакетів в единый jar-файл також может використовуватися для распространения модулей Java. JAR-файл, який по сути представляє скомпилированный модуль, називається модульным JAR-файлом.

Для создания модульного JAR-файла аналогичным чином застосовується утилита jar. Если модуль містить декілька пакетів, то також лучше всего его компилировать с опцией -d, которая помещает файлы классов в отдельный каталог (якщо каталог ще не существует, то он створюєся). А при упаковке файлов в jar-файл для перехода в цей каталог застосовується опция -C команды jar. Рассмотрим на примере.

Определение модуля

Для начала где-нибудь на жестком диске (в моем случае це папка worlspace/java) создадим каталог, який назовем demo. Цет каталог буде представлять одноименный модуль. И в цем каталоге demo определим новый файл module-info.java - файл дескриптора модуля со наступним кодом:


module demo {
}

Далїї в каталоге demo создадим папку com. В папке com создадим папку , а в папке com/ - папку hello.

В папке com//hello определим новый файл Hello.java:


package com..hello;
 
public class Hello{
     
    public static void main(String[] args){
        System.out.println("Hello !");
    }
}

Название пакета файла - com..hello отражает структуру папок, в которых расположен файл. Сам файл определяет класс Hello, який в методе main выводит на консоль строку.

В итоге у нас получится следующая стуктура проекта:

Теперь скомпилируем все це и упакуем в файл jar.

Створення jar

Для создания jar перейдем в командной строке/терминале к папке, где находится папка demo с нашим модулем (в моем случае це папка worlspace/java). Затем для компиляции модуля выполним следующую команду:


javac --module-source-path . -d mods demo/module-info.java demo/com//hello/Hello.java

После компиляции модуля в текущей папке появится папка "mods", в яку будуть помещаться все скомпилированные классы модуля demo и которая буде иметь следующую структуру:

mods
|_demo
    |_module-info.class
    |_com
        |_
            |_hello
                |_Hello.class

Далїї для упаковки в jar выполним следующую команду:


jar -c -v -f demo.jar -C mods/demo .

Команда приймає следующие параметры

  • -c: указывает на створення нового архива.

  • -v: надає подробный вывод виконання команды

  • -f demo.jar: задает имя создаваемого JAR-файла. То есть создаваемый файл буде называться "demo.jar"

  • -C mods/demo .: переходит в папку с скомпилированными классами (mods/имя__модуля) и додає все її содержимое (.) в JAR. В данном случае це папка "mods" (название произвольное) и її подкаталог "demo", название которого повинно соответствовать имени модуля, указананного в module-info.java (якщо указано). В примере выше модуль називається "demo", соответственно все компилируемые файлы помещаются в папку "mods/demo"

eugene@Eugene:/workspace/java$ javac --module-source-path . -d mods demo/module-info.java demo/com//hello/Hello.java

Створення исполняемого jar

Но в примере выше класс Hello є исполняемым, то есть он имїїт метод main() и может быть запущен на выполнение. И як и в обычных JAR-файлах, в модульном JAR-файле можна указать основной класс, який буде выполняться при запуске. Для цього призначенийа опция -e (полное название опции - --main-class), которой передаєся название основного класса с учетом его пакетів:


jar -c -v -f demo.jar -e com..hello.Hello -C mods/demo .

В примере выше класс Hello определен в пакеті com..hello, поцему указываем в качестве полного имени файла "com..hello.Hello".

После цього можна запустить створений модульный JAR-файл за допомогою команды java:

java --module-path demo.jar --module demo

Компиляция взаимодействующих модулей

Рассмотрим другую, болїї сложную ситуацию, где у нас есть декілька модулей, которые взаимодействуют друг с другом. Допустим, у нас есть следующая структура проектов:

Здесь определено два модуля: demo и operations. Каждый располагается в своей одноименной папке.

В папке operations/com//factorial определен следующий файл Factorial.java:


package com..factorial;

public class Factorial{
	
	public static int calculate(int n){
		if (n < 1) return -1;
        int result = 1;
        while (n > 1) result *= n--;
        return result;
	}
}

Класс Factorial определяет функцию вычисления факториала и принадлежит пакету com..factorial.

Щоб сделать цей класс и в целом его пакет доступным для использования в других модулях в папке operations определен следующий файл module-info.java:


module operations {
    exports com..factorial;
}

В папке demo определен следующий файл module-info.java:


module demo {
    requires operations;
}

С помощью оператора requires< указываем, что модуль demo буде використовувати модуль operations.

А в папке demo/com//hello/ определим следующий файл Hello.java:


package com..hello;

import com..factorial.Factorial;

public class Hello{
	
	public static void main(String[] args){
		
		int a = 5;
		int b = Factorial.calculate(a);
		System.out.printf("Factorial of %d is equal to %d \n", a, b);
	}
}

Здесь мы подключаем класс Factorial из пакета "com./factorial" и вычисляем факториал числа.

Теперь нам треба оба модуля скомпилировать в jar-файлы. Стандартный подход предполагает, что оба модуля компилируются по отдельности каждый в свой jar-файл. В консоли перейдем к корневой папке, где размещаются оба модуля (в моем случае це папка "/workspace/java") и скомпилируем их. Для цього выполним следующую команду:

javac --module-source-path . -d mods demo/module-info.java demo/com//hello/Hello.java

Хотя здесь указан лише один модуль, зависимый модуль також буде скомпилирован. Оба модуля в данном случае будуть помещаться в папку "mods", которая в итоге буде иметь следующую структуру:

mods
|_demo
|   |_module-info.class
|   |_com
|       |_
|           |_hello
|                |_Hello.class
|_factorial
    |_module-info.class
    |_com
        |_
            |_factorial
                |_Factorial.class

Далїї скомпилируем модуль operations:

jar -c -v -f libs/operations.jar -C mods/operations .

Компилируем из модуля файл "operations.jar" буде помещаться в папку "libs". Далїї скомпилируем модуль demo:

jar -c -v -f libs/demo.jar -e com..hello.Hello -C mods/demo .

В итоге в папке "libs" будуть располагаться оба скомпилированных модуля. Теперь запустим главный модуль, передав путь к папке с модульными jar:

java --module-path libs --module demo/com..hello.Hello
