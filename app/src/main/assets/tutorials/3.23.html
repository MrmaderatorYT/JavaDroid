Заборона наслідування та перевизначення методів

Останнє оновлення: 24.09.2025

Хотя наследование дуже интересный и эффективный механизм, но в некоторых ситуациях его застосування может быть нежелательным. И в цем случае можна запретить наследование за допомогою ключевого слова final. Наприклад:


final class Person { }

Якби клас Person був би визначений таким чином, то наступний код був би помилковим і не спрацював, тому що ми тим самим заборонили успадкування:


class Employee extends Person{ }  // ! Ошибка - унаследовать от класа Person нельзя

Кроме запрета наследования можна також запретить перевизначенийие отдельных методів. Наприклад, в примере выше перевизначений метод print(), запретим его перевизначенийие:


class Person {
     
    private String name;

    Person(String name){ this.name = name;  }

    // запрет перевизначенийия
    final void print(){
         
        System.out.println("Name: " + name);
    }
}

І тут клас-спадкоємець зможе перевизначити метод print:


class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }
    // ! Ошибка - перевизначити метод нельзя
    void print(){
        super.print();
        System.out.println("Company: " + company);
    }
}

Запрет наследования класа або перевизначенийия методу за допомогою final может быть полезным, якщо мы хочемо гарантировать, что семантика класа або методу не буде изменена в производном класе, то есть клас або метод буде действовать ровно так, як он був изначально визначений. Поскольку якщо всередині базового класа вызываются его методи, то перевизначенийие цих методів в производном класе может нарушить стандартную и ожидаемую работу цього класа. Рассмотрим небольшой пример:


public class Program{ 
       
    public static void main (String args[]){

        Employee bob = new Employee("Bob", "Google");
    }
}
class Person {
     
    private String name;

    Person(String name){ 

        this.name = name;  
        print();
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){

        super(name); 
        this.company = company;
    }
    @Override
    void print(){
        super.print();
        System.out.println("Company: " + company);
    }
}

В конструкторе класа Person вызывается метод print() для вивіда информации о созданном об'єкте. Но в класе Employee переопределяем цей метод, чтобы також вывести значення поля company.

Оскільки у конструкторі Employee ми вже викликаємо конструктор базового класу Person, нам не потрібно вдруге викликати метод print:


Employee(String name, String company){

    super(name);   // здесь вызывается метод print
    this.company = company;
}

Однак підивимося на консольне виведення програми:


Name: Bob
Company: null

Конструктор базового класа Person, где вызывается метод print, отработал ДО установки поля company. В итоге при вивіде на консоль виводиться некорректная информация.

Вызов методу базового класа в конструкторе - це одна из распространенных причин запрета перевизначенийия методу. И в цем случае метод print в класе Person мы можемо визначити як final:


public class Program{ 
       
    public static void main (String args[]){

        Employee bob = new Employee("Bob", "Google");
    }
}
class Person {
     
    private String name;

    Person(String name){ 
        this.name = name;  
        printPerson();
    }
    // запрет перевизначенийия
    final void printPerson(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){

        super(name);  // в цей точке в базовом класе вызывается метод printPerson 

        this.company = company;
        printEmployee();
    }

    void printEmployee(){
        System.out.println("Company: " + company);
    }
}

І тут отримаємо коректний висновок:


Name: Bob
Company: Google

Варто відзначити, що проблема була пов'язана насамперед з тим, що виклик конструктора базового класу виконувався до решти коду (установки поля company). І в до версії Java 25 ми могли викликати конструктор базового класу тільки на початку конструктоора похідного класу до решти коду:


Employee(String name, String company){

    super(name); 
    this.company = company;
}

Однак, починаючи з Java 25, можна викликати конструктор базового класу в будь-якому місці конструктора похідного класу:


Employee(String name, String company){

    this.company = company;
    super(name); 
}

Конкретно эта деталь у даному випадку бы решила проблему, тем не менїї в болїї сложных програмах нам треба учитывать все возможные зависимости, которые можуть повлять на створення об'єкта базового класа. Тому в конструкторе лучше вызывать лише методи с модификаторами final або private.
