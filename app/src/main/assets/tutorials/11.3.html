Взаємодія між модулями. Експорт та імпорт

Останнє оновлення: 02.10.2025

Нередко разработчики визначають не один, а декілька модулів, которые використовуються в програме. И при взаимодействии модулів есть некоторые особенности, которые йде учитывать. Рассмотрим, як модулі можуть взаємодіяти друг с другом.

Экспорт

Для виконання своих завдань одному модулю Java можуть потребоваться пакети из другого модуля. Однак якщо в модуля Java визначенийы некоторые пакети, они не є автоматически доступными для других модулів. Модуль вказує, якие из его пакетів доступны, за допомогою ключевого слова exports. Наприклад, в оголошенийии модуля java.base, який надає базовую функциональность Java и який мы можемо посмотреть в репозитории OpenJDK на github, мы можемо увидеть такие рядки:


module java.base {

    exports java.io;
    exports java.lang;
    exports java.lang.annotation;
    exports java.lang.classfile;
...................................
    exports java.util;
.........................

Таким чином, модуль "java.base" робить доступными такие пакети як java.io, java.util и java.lang. При экспорте пакету его открытые и защищенные класы и інтерфейсы (с модификаторами доступа public и protected), а також их открытые и защищенные компоненты доступны вне модуля. Другие же пакети, которые явным чином не экспортируются за допомогою оператора exports, скрыты от доступа из вне и недоступны вне своего модуля.

Квалифицированный экспорт

Кроме стандартных выражений экспорта за допомогою оператора exports в исходном коді конфигурации модуля java.base можна увидеть такие рядки як


exports jdk.internal.javac to
    java.compiler,
    jdk.compiler;

Выражения exports ... to представляютьь квалифицированный экспорт (qualified export). Он вказує, что лише перечисленные модулі (java.compiler и jdk.compiler) можуть отримати доступ к экспортированному пакету, но другие модулі не можуть.

Импорт модуля

Для импорта функциональности модуля, як и пакетів и отдельных класов, застосовується оператор import

import module java.base;

Це эквивалентно импорту всех экспортируемых пакетів модуля java.base. Наприклад:


import module java.base;

class Program {
    
    static void main(String[] args) {

        List<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
        System.out.println(people);     // [Tom, Bob, Sam]
    }
}

У даному випадку мы явно не импортируем по отдельности типи List и ArrayList, ни их пакет. Однак импорт всего модуля приведет к тому, что эти типи також станут доступны для использования, оскільки располагаются в пакеті java.util из данного модуля.

Варто відзначити, что до Java 25 оболчка JShell импортировала специальный набор пакетів, а теперь импортирует модуль java.base, что и дозволяє використовувати в оболочки функциональность данного модуля без якого-або дополнительного импорта. Крім того, в компактных програмах модуль java.base також импортируется автоматически.

Вто же час при импорте модулів (особенно когда импортируется декілька модулів) возрастает вероятность конфликтов имен, якщо ряд модулів або даже пакетів одного модуля имеют одноименные типи. Тому, возможна, в ряде ситуаций традиционный импорт пакетів або отдельных типів буде болїї лучшим решением.

Пример взаимодействия модулів

Визначимо проєкт со следующей структурой:

Здесь визначенийо два модуля: demo и operations. Каждый располагается в своей одноименной папке.

В папке operations/com//factorial визначений следующий файл Factorial.java:


package com..factorial;

public class Factorial{
	
	public static int calculate(int n){
		if (n < 1) return -1;
        int result = 1;
        while (n > 1) result *= n--;
        return result;
	}
}

Клас Factorial визначає функцію вычисления факториала и принадлежит пакету com..factorial.

Допустим, мы хочемо, чтобы данный клас мог быть использован в других модулях. Но по умолчанию, все пакети модуля невидимы для модулів зовні. Щоб сделать пакет, який визначений в модуле, видимым для других модулів, цей пакет треба экспортировать при визначенийии модуля за допомогою оператора exports.

Для цього в папке operations визначений следующий файл module-info.java:


module operations {
    exports com..factorial;
}

Файл вказує, что данный модуль буде называться operations. За допомогою оператора exports экспортируется пакет com..factorial. Таким чином, другие модулі сможуть використовувати клас Factorial.

Теперь перейдем к модулю demo. В папке demo визначений следующий файл module-info.java:


module demo {
    requires operations;
}

За допомогою оператора requires< указываем, что модуль demo буде використовувати модуль operations.

А в папке demo/com//hello/ визначимо следующий файл Hello.java:


package com..hello;

import com..factorial.Factorial;

public class Hello{
	
	public static void main(String[] args){
		
		int a = 5;
		int b = Factorial.calculate(a);
		System.out.printf("Factorial of %d is equal to %d \n", a, b);
	}
}

Здесь мы підключаем клас Factorial из пакету "com./factorial" и вычисляем факториал числа.

В консоли перейдем к корневой папке, где размещаются оба модуля, и скомпилируем первый модуль - operations:


javac -d appmodules/operations operations/module-info.java operations/com//factorial/Factorial.java

За допомогою параметра -d здесь указываем, что скомпилированный модуль буде помещаться в каталог appmodules/operations (якщо по умолчанию такого каталогу нет, то он буде автоматически буде создан).

Скомпилируем второй модуль - demo, який використовує первый модуль:


javac -p appmodules -d appmodules/demo demo/module-info.java demo/com//hello/Hello.java

Параметр -p представляє сокращение от --module-path и вказує, где искать використовуємоые модулі. То есть у даному випадку це папка appmodules, куда ранїї був помщен скомпилированный модуль operations.

Параметр -d, як и в случае выше, вказує, куда буде помещаться скомпилированный модуль - то есть в папку appmodules/demo.

Після того, як все скомпилировано, запустимо на виконання модуль demo:


java -p appmodules -m demo/com..hello.Hello

При компиляции за допомогою параметра -p вказуєся папка, где находятся оба скомпилированных модуля. В итоге на консоль буде выведен факториал числа:


eugene@Eugene:/java$ javac -d appmodules/operations operations/module-info.java operations/com//factorial/Factorial.java
eugene@Eugene:/java$ javac -p appmodules -d appmodules/demo demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java -p appmodules -m demo/com..hello.Hello
Factorial of 5 is equal to 120 
eugene@Eugene:/java$
