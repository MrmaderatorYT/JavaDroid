Взаємодія між модулями. Експорт та імпорт

Останнє оновлення: 02.10.2025

Нередко разработчики определяют не один, а декілька модулей, которые використовуються в программе. И при взаимодействии модулей есть некоторые особенности, которые йде учитывать. Рассмотрим, як модули можуть взаємодіяти друг с другом.

Экспорт

Для виконання своих задач одному модулю Java можуть потребоваться пакеты из другого модуля. Однак якщо в модуля Java определены некоторые пакеты, они не є автоматически доступными для других модулей. Модуль указывает, якие из его пакетів доступны, за допомогою ключевого слова exports. Наприклад, в оголошенийии модуля java.base, який надає базовую функциональность Java и який мы можемо посмотреть в репозитории OpenJDK на github, мы можемо увидеть такие строки:


module java.base {

    exports java.io;
    exports java.lang;
    exports java.lang.annotation;
    exports java.lang.classfile;
...................................
    exports java.util;
.........................

Таким чином, модуль "java.base" робить доступными такие пакеты як java.io, java.util и java.lang. При экспорте пакета его открытые и защищенные классы и інтерфейсы (с модификаторами доступа public и protected), а також их открытые и защищенные компоненты доступны вне модуля. Другие же пакеты, которые явным чином не экспортируются за допомогою оператора exports, скрыты от доступа из вне и недоступны вне своего модуля.

Квалифицированный экспорт

Кроме стандартных выражений экспорта за допомогою оператора exports в исходном коде конфигурации модуля java.base можна увидеть такие строки як


exports jdk.internal.javac to
    java.compiler,
    jdk.compiler;

Выражения exports ... to представляють квалифицированный экспорт (qualified export). Он указывает, что лише перечисленные модули (java.compiler и jdk.compiler) можуть отримати доступ к экспортированному пакету, но другие модули не можуть.

Импорт модуля

Для импорта функциональности модуля, як и пакетів и отдельных классов, застосовується оператор import

import module java.base;

Це эквивалентно импорту всех экспортируемых пакетів модуля java.base. Например:


import module java.base;

class Program {
    
    static void main(String[] args) {

        List<String> people = new ArrayList<String>();
        people.add("Tom");
        people.add("Bob");
        people.add("Sam");
        System.out.println(people);     // [Tom, Bob, Sam]
    }
}

В данном случае мы явно не импортируем по отдельности типы List и ArrayList, ни их пакет. Однак импорт всего модуля приведет к тому, что эти типы також станут доступны для использования, оскільки располагаются в пакеті java.util из данного модуля.

Варто відзначити, что до Java 25 оболчка JShell импортировала специальный набор пакетів, а теперь импортирует модуль java.base, что и дозволяє використовувати в оболочки функциональность данного модуля без якого-або дополнительного импорта. Крім того, в компактных программах модуль java.base також импортируется автоматически.

Вто же час при импорте модулей (особенно когда импортируется декілька модулей) возрастает вероятность конфликтов имен, якщо ряд модулей або даже пакетів одного модуля имеют одноименные типы. Тому, возможна, в ряде ситуаций традиционный импорт пакетів або отдельных типов буде болїї лучшим решением.

Пример взаимодействия модулей

Определим проект со следующей структурой:

Здесь определено два модуля: demo и operations. Каждый располагается в своей одноименной папке.

В папке operations/com//factorial определен следующий файл Factorial.java:


package com..factorial;

public class Factorial{
	
	public static int calculate(int n){
		if (n < 1) return -1;
        int result = 1;
        while (n > 1) result *= n--;
        return result;
	}
}

Класс Factorial определяет функцию вычисления факториала и принадлежит пакету com..factorial.

Допустим, мы хочемо, чтобы данный класс мог быть использован в других модулях. Но по умолчанию, все пакеты модуля невидимы для модулей зовні. Щоб сделать пакет, який определен в модуле, видимым для других модулей, цей пакет треба экспортировать при определении модуля за допомогою оператора exports.

Для цього в папке operations определен следующий файл module-info.java:


module operations {
    exports com..factorial;
}

Файл указывает, что данный модуль буде называться operations. С помощью оператора exports экспортируется пакет com..factorial. Таким чином, другие модули сможуть використовувати класс Factorial.

Теперь перейдем к модулю demo. В папке demo определен следующий файл module-info.java:


module demo {
    requires operations;
}

С помощью оператора requires< указываем, что модуль demo буде використовувати модуль operations.

А в папке demo/com//hello/ определим следующий файл Hello.java:


package com..hello;

import com..factorial.Factorial;

public class Hello{
	
	public static void main(String[] args){
		
		int a = 5;
		int b = Factorial.calculate(a);
		System.out.printf("Factorial of %d is equal to %d \n", a, b);
	}
}

Здесь мы подключаем класс Factorial из пакета "com./factorial" и вычисляем факториал числа.

В консоли перейдем к корневой папке, где размещаются оба модуля, и скомпилируем первый модуль - operations:


javac -d appmodules/operations operations/module-info.java operations/com//factorial/Factorial.java

С помощью параметра -d здесь указываем, что скомпилированный модуль буде помещаться в каталог appmodules/operations (якщо по умолчанию такого каталога нет, то он буде автоматически буде создан).

Скомпилируем второй модуль - demo, який использует первый модуль:


javac -p appmodules -d appmodules/demo demo/module-info.java demo/com//hello/Hello.java

Параметр -p представляє сокращение от --module-path и указывает, где искать используемые модули. То есть в данном случае це папка appmodules, куда ранїї був помщен скомпилированный модуль operations.

Параметр -d, як и в случае выше, указывает, куда буде помещаться скомпилированный модуль - то есть в папку appmodules/demo.

После того, як все скомпилировано, запустим на выполнение модуль demo:


java -p appmodules -m demo/com..hello.Hello

При компиляции за допомогою параметра -p указывается папка, где находятся оба скомпилированных модуля. В итоге на консоль буде выведен факториал числа:


eugene@Eugene:/java$ javac -d appmodules/operations operations/module-info.java operations/com//factorial/Factorial.java
eugene@Eugene:/java$ javac -p appmodules -d appmodules/demo demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java -p appmodules -m demo/com..hello.Hello
Factorial of 5 is equal to 120 
eugene@Eugene:/java$
