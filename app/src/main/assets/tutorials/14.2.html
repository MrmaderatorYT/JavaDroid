Дослідження типів

Останнє оновлення: 29.09.2025

Для исследования типів в програме кроме класа Class язык Java надає ще ряд класов, которые визначенийы главным чином в пакеті java.lang.reflect:

  • Field: описує поле класа

  • Method: описує метод типу

  • Constructor: описує конструктор класа

Для получения цих компонентов класа застосовуються соответственно следующие методи класа Class:

  • Field[] getFields() / getDeclaredFields(): повертає поля класа

  • Method[] getMethods() / getDeclaredMethods(): повертає методи класа

  • Constructor[] getConstructors() / getDeclaredConstructors(): повертає конструктори класа

Методи getFields(), getMethods() и getConstructors() повертають масивы общедоступных полей, методів и конструкторов (визначенийных с модификатором public), піддерживаемых класом, в том числе и поля и методи, унаследованные от суперкласов. Методи getDeclaredFields(), getDeclaredMethods() и getDeclaredConstructors() повертають масивы из всех полей, методів и конструкторов, оголошенийных непосредственно в класе, в том числе и закрытые, приватные компоненты. Но эти масивы НЕ включают компоненты суперкласов.

Таким чином, в зависимости от уровня доступа компонентов а також от того, хочемо мы отримати також компоненты суперкласов або хочемо ограничиться компонентами, которые визначенийы непосредственно в класе, зависит выбор конкретного методу для получения компонентов.

Допустим, у нас есть следующая иерархия класов:



class Person {
     
    private String name;
    private int age;

    public String getName() {return name;}
    public int getAge() {return age;}

    public Person(){
        this.name = "Undefined";
        this.age = 18;
    }
    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
 
}

Исследуем тип Person:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // отримуємо клас для класа Person
        Class personClass = Person.class;

        Field[] fields = personClass.getFields();
        Method[] methods = personClass.getMethods();
        Constructor[] ctors = personClass.getConstructors();

        System.out.println("Public Fields");
        for(Field field : fields){
            System.out.println(field);
        }
        System.out.println("\nPublic Ctors");
        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }

        System.out.println("\nPublic Methods");
        for(Method method : methods){
            System.out.println(method);
        }
    }
}

Преимуществом класов Field/Method/Constructor є то, что в Java вже по умолчанию реалізований полноценный вивід компонента, что дуже удобно при вивіде на консоль. И у даному випадку мы видим, что методи вывели лише публичные компоненты, в том числе и унаследованные от класа Object.

Теперь застосуємо другой набор методів для получения информации о класе - getDeclaredFields()/getDeclaredMethods()/getDeclaredConstructors():


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // отримуємо клас для класа Person
        Class personClass = Person.class;

        Field[] fields = personClass.getDeclaredFields();
        Method[] methods = personClass.getDeclaredMethods();
        Constructor[] ctors = personClass.getDeclaredConstructors();

        System.out.println("Fields");
        for(Field field : fields){
            System.out.println(field);
        }
        System.out.println("\nCtors");
        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }

        System.out.println("\nMethods");
        for(Method method : methods){
            System.out.println(method);
        }
    }
}

Підобным чином мы можемо исследовать и встроенные типи данных. Наприклад, исследуем тип String (ограничимся конструкторами):


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class strClass = String.class;

        Constructor[] ctors = strClass.getConstructors();

        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }
    }
}

Проверка типу

Ряд методів класа Class дозволяють проверить исследуемый тип:

  • boolean isAnonymousClass(): повертає true, якщо тип є анонимным класом

  • boolean isArray(): повертає true, якщо тип представляє масив

  • boolean isPrimitive(): повертає true, якщо тип є примитивным типом

  • boolean isInterface(): повертає true, якщо об'єкт представляє інтерфейс

  • boolean isEnum(): повертає true, якщо об'єкт представляє перечисление

  • boolean isRecord(): повертає true, якщо об'єкт представляє клас record

  • boolean isInstance(Object obj): повертає true, якщо об'єкт-параметр є экземпляром цього типу

  • boolean isSealed(): повертає true, якщо клас визначений с модификатором sealed

  • Class getSuperclass(): повертає суперклас для текущего типу

Пример применения некоторых из цих методів:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // отримуємо клас для типу Person
        Class cl = Person.class;

        // отримуємо ім'я класа
        System.out.println("Name: " + cl.getName());
        // отримуємо суперклас
        System.out.println("superclass: " + cl.getSuperclass());
        // проверяем, є ли тип примитивным типом
        System.out.println("primitive: " + cl.isPrimitive());
        // проверяем, є ли тип масивом
        System.out.println("array: " + cl.isArray());
        // проверяем, є ли тип класом record
        System.out.println("record: " + cl.isRecord());
        // проверяем, є ли тип інтерфейсом
        System.out.println("interface: " + cl.isInterface());

        // проверям, что tom - об'єкт класа
        Person tom = new Person("Tom", 41);
        System.out.println("tom is a Person:: " + cl.isInstance(tom));

        // проверям, что message - об'єкт класа
        String message = "hello";
        System.out.println("message is a Person:: " + cl.isInstance(message));
    }
}
