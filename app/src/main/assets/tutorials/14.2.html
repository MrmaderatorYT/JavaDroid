Дослідження типів

Останнє оновлення: 29.09.2025

Для исследования типов в программе кроме класса Class язык Java надає ще ряд классов, которые определены главным чином в пакеті java.lang.reflect:

  • Field: описує поле класса

  • Method: описує метод типу

  • Constructor: описує конструктор класса

Для получения цих компонентов класса застосовуються соответственно следующие методы класса Class:

  • Field[] getFields() / getDeclaredFields(): повертає поля класса

  • Method[] getMethods() / getDeclaredMethods(): повертає методы класса

  • Constructor[] getConstructors() / getDeclaredConstructors(): повертає конструкторы класса

Методи getFields(), getMethods() и getConstructors() повертають массивы общедоступных полей, методів и конструкторов (определенных с модификатором public), поддерживаемых классом, в том числе и поля и методы, унаследованные от суперклассов. Методи getDeclaredFields(), getDeclaredMethods() и getDeclaredConstructors() повертають массивы из всех полей, методів и конструкторов, оголошенийных непосредственно в классе, в том числе и закрытые, приватные компоненты. Но эти массивы НЕ включают компоненты суперклассов.

Таким чином, в зависимости от уровня доступа компонентов а також от того, хочемо мы отримати також компоненты суперклассов або хочемо ограничиться компонентами, которые определены непосредственно в классе, зависит выбор конкретного метода для получения компонентов.

Допустим, у нас есть следующая иерархия классов:



class Person {
     
    private String name;
    private int age;

    public String getName() {return name;}
    public int getAge() {return age;}

    public Person(){
        this.name = "Undefined";
        this.age = 18;
    }
    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
 
}

Исследуем тип Person:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // получаем класс для класса Person
        Class personClass = Person.class;

        Field[] fields = personClass.getFields();
        Method[] methods = personClass.getMethods();
        Constructor[] ctors = personClass.getConstructors();

        System.out.println("Public Fields");
        for(Field field : fields){
            System.out.println(field);
        }
        System.out.println("\nPublic Ctors");
        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }

        System.out.println("\nPublic Methods");
        for(Method method : methods){
            System.out.println(method);
        }
    }
}

В данном случае используем методы getFields(), getMethods() и getConstructors() для получения всех публичных компонентов класса, в том числе унаследованных от класса Person. Консольный вывод программы:

Преимуществом классов Field/Method/Constructor є то, что в Java вже по умолчанию реализован полноценный вывод компонента, что очень удобно при выводе на консоль. И в данном случае мы видим, что методы вывели лише публичные компоненты, в том числе и унаследованные от класса Object.

Теперь применим другой набор методів для получения информации о классе - getDeclaredFields()/getDeclaredMethods()/getDeclaredConstructors():


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // получаем класс для класса Person
        Class personClass = Person.class;

        Field[] fields = personClass.getDeclaredFields();
        Method[] methods = personClass.getDeclaredMethods();
        Constructor[] ctors = personClass.getDeclaredConstructors();

        System.out.println("Fields");
        for(Field field : fields){
            System.out.println(field);
        }
        System.out.println("\nCtors");
        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }

        System.out.println("\nMethods");
        for(Method method : methods){
            System.out.println(method);
        }
    }
}

Подобным чином мы можемо исследовать и встроенные типы данных. Наприклад, исследуем тип String (ограничимся конструкторами):


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class strClass = String.class;

        Constructor[] ctors = strClass.getConstructors();

        for(Constructor ctor : ctors){
            System.out.println(ctor);
        }
    }
}

Проверка типу

Ряд методів класса Class дозволяють проверить исследуемый тип:

  • boolean isAnonymousClass(): повертає true, якщо тип є анонимным классом

  • boolean isArray(): повертає true, якщо тип представляє массив

  • boolean isPrimitive(): повертає true, якщо тип є примитивным типом

  • boolean isInterface(): повертає true, якщо об'єкт представляє інтерфейс

  • boolean isEnum(): повертає true, якщо об'єкт представляє перечисление

  • boolean isRecord(): повертає true, якщо об'єкт представляє класс record

  • boolean isInstance(Object obj): повертає true, якщо об'єкт-параметр є экземпляром цього типу

  • boolean isSealed(): повертає true, якщо класс определен с модификатором sealed

  • Class getSuperclass(): повертає суперкласс для текущего типу

Пример применения некоторых из цих методів:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // получаем класс для типу Person
        Class cl = Person.class;

        // получаем имя класса
        System.out.println("Name: " + cl.getName());
        // получаем суперкласс
        System.out.println("superclass: " + cl.getSuperclass());
        // проверяем, є ли тип примитивным типом
        System.out.println("primitive: " + cl.isPrimitive());
        // проверяем, є ли тип массивом
        System.out.println("array: " + cl.isArray());
        // проверяем, є ли тип классом record
        System.out.println("record: " + cl.isRecord());
        // проверяем, є ли тип інтерфейсом
        System.out.println("interface: " + cl.isInterface());

        // проверям, что tom - об'єкт класса
        Person tom = new Person("Tom", 41);
        System.out.println("tom is a Person:: " + cl.isInstance(tom));

        // проверям, что message - об'єкт класса
        String message = "hello";
        System.out.println("message is a Person:: " + cl.isInstance(message));
    }
}
