Успадкування

Останнє оновлення: 24.09.2025

Одним із ключових аспектів об'єктно-орієнтованого програмування є успадкування. За допомогою успадкування можна розширити функціонал вже наявних класів з допомогою додавання нового функціоналу чи зміни старого. Наприклад, є наступний клас Person, описує окрему людину:


class Person {
     
    String name;
	
    void print(){
         
        System.out.println("Name: " + name);
    }
}

І, можливо, згодом ми захочемо додати ще один клас, який описує співробітника підприємства – клас Employee:


class Employee {
     
    String name;
	
    void print(){
         
        System.out.println("Name: " + name);
    }
}

Однак оскільки цей класс реализует тот же функционал, что и класс Person, поскольку сотрудник - це також и человек, то буво бы целесообразнїї не визначати повторно один и тот же функционал, а як-то передать цей функционал из класса Person в класс Employee. И для цього можна успадкувати класс Employee от класса Person:


class Employee extends Person { }

Щоб унаследовать один класс от другого, треба використовувати после имени класса-наследника ключевое слово extends, после которого идет имя базового класса. Для класса Employee базовым є Person, и поцему класс Employee найде все те же поля и методы, которые есть в классе Person.

Таким чином, наследование реализует отношение is-a (є). То есть об'єкт класса Employee також є об'єктом класса Person. В цем отношении класс Employee ще называют похідним класом (а також дочірнім класом, класом-спадкоємцем, підкласом) а класс Person - базовим класом (а також батьківським класом або суперкласом).

Потім ми можемоо використовувати функціонал базового класу (Person), звертаючись до об'єкта похідного класу (Employee):


public class Program{ 
       
    public static void main (String args[]){
           
        Person tom = new Person();
        tom.name = "Tom";
        tom.print();

        Employee bob = new Employee();
        bob.name = "Bob";
        bob.print();
    }
}

class Person {
     
    String name;

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{}

Тобто в даному випадку завдяки успадкування у об'єкта класу Employee ми можемоо звернутися і до поля name, і методу print, хоча вони визначені в класі Person.

Спадкування конструкторів

Если в базовом классе определены конструкторы, то в конструкторе производного классы необхідно вызвать один из конструкторов базового класса с помощью ключевого слова super, в який передаются значения для параметров конструкторв:


super(аргументы);

Наприклад, визначимо до класу Person конструктор:


class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    // треба вызвать конструктор базового класса
    Employee(String name){
        super(name);  // вызываем конструктор базового класса
    }
}

Здесь конструктор класса Person приймає один параметр. Тому в классе Employee в конструкторе потрібно вызвать конструктор класса Person. То есть вызов super(name) буде представлять вызов конструктора класса Person.

При вызове конструктора после слова super в скобках идет перечисление передаваемых аргументов. Таким чином, установка имени сотрудника делегируется конструктору базового класса.

Причому навіть якщо похідний клас ніякої іншої роботи не робить у конструкторі, як у прикладі вище, все одно необхідно викликати конструктор базового класу.

Використання класів:


public class Program{ 
       
    public static void main (String args[]){
           
        Person tom = new Person("Tom");
        tom.print();

        Employee bob = new Employee("Bob");
        bob.print();
    }
}
class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    Employee(String name){
        super(name);
    }
}

Варто зазначити, що при визначенні конструктора у похідному класі до версії Java 25 виклик конструктора суперкласу повинен був виконуватись до всіх інших інструкцій. Наприклад:


Employee(String name){
    super(name);   // До Java 25 вызов конструктора мог быть лише в самом начале

    // все остальные инструкции
    System.out.println("Employee created");
}

Починаючи з версії Java 25, виклик конструктора суперкласу може розташовуватися в будь-якому місці конструктора похідного класу. Наприклад:


Employee(String name){

    System.out.println("Employee creation started");

    super(name);   // Начиная с Java 25 можна поместить вызов конструктора в любое место
}

Розширення базового класу

Похідний клас не тільки успадковує функціонал від базового класу, але також може розширювати його - додавати свої поля і методи. Наприклад, клас Employee представляє працівника деякої компанії, та відповідно нам хотілося б зберігати в ньому дані про цю компанію:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();    // Name: Bob
        bob.work();     // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // выводим дані о компании
    void work(){
        System.out.println("Company: " + company);
    }
}  

class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

У разі клас Employee додає поле company, яке зберігає місце роботи співробітника, і навіть метод work для виведення назви компанії.

Доступ до функціональності базового класу

Производный класс имїїт доступ ко всем методам и полям базового класса (даже якщо базовый класс находится в другом пакеті) кроме тех, которые определены с модификатором private. Так в примере выше мы сделали поле name в классе Person приватным (определенным с модификатором private). Тому класс Employee не имїїт к нему доступа:


class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // выводим дані о компании
    void work(){
        System.out.println("Name: " + name);  // ! Ошибка: так нельзя - переменная name - приватная
        System.out.println("Company: " + company);
    }
}

При попытке скомпилировать цей класс мы столкнемся с ошибкой, оскільки в методе work идет обращение к змінної name базового класса Person, которая є приватной.

Определение полей приватными представляє стандартную практику лучше инкапсулировать и защитить дані от некорретного вмешательства зовні. Однак бывают случаи, когда требуется открыть доступ к компоненту (полю або методу) базового класса лише для производных классов. В цем случае компонент класса оголошується за допомогою модификатора protected. Наприклад, якщо суперкласс Person оголошує поле name с модификатором protected, а не приватным, то методы класса Employee можуть обращаться к нему напрямую:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.work();     // Name: Bob
                        // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // выводим дані о компании
    void work(){
        System.out.println("Name: " + name);  // так можна - переменная name - protected
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    protected String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

В Java поле с модификатором protected доступно любому классу в том же пакеті. Тому для надлежащей інкапсуляції нам треба поместить класс Person в отдельный пакет. В цем случае поля и методы с модификатором protected будуть доступны лише классу Employee и другим унаследованным классам.

Перевизначення методів

Похідний клас може визначати свої методи, а може перевизначати методи, успадковані від базового класу. При перевизначенні методу похідний клас визначає метод із тим самим ім'ям, як і базовий клас. Наприклад:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }
    @Override
    void print(){
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    private String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

Здесь производный класс Employee переопределяет метод print из базового класса. Перед переопределяемым методом указывается аннотация @Override. Данная аннотация в принципе необязательна.

При перевизначенні методу він повинен мати рівень доступу не менше, ніж рівень доступу до базового класу. Наприклад, якщо у базовому класі метод має модифікатор public, те й у похідному класі метод повинен мати модифікатор public.

В классе Employee реализация метода print выводит значение поля company на консоль. Но что делать, якщо мы хочемо, чтобы переопределенный метод print також выводил и значение поля name из базового класса? Напрямую в классе Employee мы не можемо обращаться к полю name, потому что оно приватное. Однак мы можемо вызвать в производном классе реализацию метода базового класса за допомогою слова super:


super.метод(аргументы)

Наприклад:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();        // Name: Bob
                            // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }
    @Override
    void print(){
        super.print();  // вызов реализации метожжа print из класса Person
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    private String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

Здесь в методе print в классе Employee помощью ключевого слова super мы обращаемся к реализации метода print из базового класса, которая выводит имя:

super.print();
