Успадкування

Останнє оновлення: 24.09.2025

Одним із ключових аспектів об'єктно-орієнтованого програмування є успадкування. За допомогою успадкування можна розширити функціонал вже наявних класів з допомогою додавання нового функціоналу чи зміни старого. Наприклад, є наступний клас Person, описує окрему людину:


class Person {
     
    String name;
	
    void print(){
         
        System.out.println("Name: " + name);
    }
}

І, можливо, згодом ми захочемо додати ще один клас, який описує співробітника підприємства – клас Employee:


class Employee {
     
    String name;
	
    void print(){
         
        System.out.println("Name: " + name);
    }
}

Однак оскільки цей клас реалізує тот же функционал, что и клас Person, поскольку сотрудник - це також и человек, то буво бы целесообразнїї не визначати повторно один и тот же функционал, а як-то передать цей функционал из класа Person в клас Employee. И для цього можна успадкувати клас Employee от класа Person:


class Employee extends Person { }

Щоб унаследовать один клас от другого, треба використовувати після імені класа-наследника ключевое слово extends, після которого идет ім'я базового класа. Для класа Employee базовым є Person, и поцему клас Employee найде все те же поля и методи, которые есть в класе Person.

Таким чином, наследование реалізує отношение is-a (є). То есть об'єкт класа Employee також є об'єктом класа Person. В цем отношении клас Employee ще называют похідним класом (а також дочірнім класом, класом-спадкоємцем, підкласом) а клас Person - базовим класом (а також батьківським класом або суперкласом).

Потім ми можемоо використовувати функціонал базового класу (Person), звертаючись до об'єкта похідного класу (Employee):


public class Program{ 
       
    public static void main (String args[]){
           
        Person tom = new Person();
        tom.name = "Tom";
        tom.print();

        Employee bob = new Employee();
        bob.name = "Bob";
        bob.print();
    }
}

class Person {
     
    String name;

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{}

Тобто в даному випадку завдяки успадкування у об'єкта класу Employee ми можемоо звернутися і до поля name, і методу print, хоча вони визначені в класі Person.

Спадкування конструкторів

Если в базовом класе визначенийы конструктори, то в конструкторе производного класы необхідно вызвать один из конструкторов базового класа с помощью ключевого слова super, в який передаються значення для параметрів конструкторв:


super(аргументы);

Наприклад, визначимо до класу Person конструктор:


class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    // треба вызвать конструктор базового класа
    Employee(String name){
        super(name);  // вызываем конструктор базового класа
    }
}

Здесь конструктор класа Person приймає один параметр. Тому в класе Employee в конструкторе потрібно вызвать конструктор класа Person. То есть вызов super(name) буде представлять вызов конструктора класа Person.

При вызове конструктора після слова super в скобках идет перечисление передаваемых аргументов. Таким чином, встановлення імені сотрудника делегируется конструктору базового класа.

Причому навіть якщо похідний клас ніякої іншої роботи не робить у конструкторі, як у прикладі вище, все одно необхідно викликати конструктор базового класу.

Використання класів:


public class Program{ 
       
    public static void main (String args[]){
           
        Person tom = new Person("Tom");
        tom.print();

        Employee bob = new Employee("Bob");
        bob.print();
    }
}
class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

class Employee extends Person{

    Employee(String name){
        super(name);
    }
}

Варто зазначити, що при визначенні конструктора у похідному класі до версії Java 25 виклик конструктора суперкласу повинен був виконуватись до всіх інших інструкцій. Наприклад:


Employee(String name){
    super(name);   // До Java 25 вызов конструктора мог быть лише в самом начале

    // все остальные инструкции
    System.out.println("Employee created");
}

Починаючи з версії Java 25, виклик конструктора суперкласу може розташовуватися в будь-якому місці конструктора похідного класу. Наприклад:


Employee(String name){

    System.out.println("Employee creation started");

    super(name);   // Начиная с Java 25 можна поместить вызов конструктора в любое место
}

Розширення базового класу

Похідний клас не тільки успадковує функціонал від базового класу, але також може розширювати його - додавати свої поля і методи. Наприклад, клас Employee представляє працівника деякої компанії, та відповідно нам хотілося б зберігати в ньому дані про цю компанію:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();    // Name: Bob
        bob.work();     // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // вивідим дані о компании
    void work(){
        System.out.println("Company: " + company);
    }
}  

class Person {
     
    private String name;

    Person(String name){

        this.name = name;
    }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

У разі клас Employee додає поле company, яке зберігає місце роботи співробітника, і навіть метод work для виведення назви компанії.

Доступ до функціональності базового класу

Производный клас имїїт доступ ко всем методум и полям базового класа (даже якщо базовый клас находится в другом пакеті) кроме тех, которые визначенийы с модификатором private. Так в примере выше мы сделали поле name в класе Person приватным (визначенийным с модификатором private). Тому клас Employee не имїїт к нему доступа:


class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // вивідим дані о компании
    void work(){
        System.out.println("Name: " + name);  // ! Ошибка: так нельзя - змінна name - приватная
        System.out.println("Company: " + company);
    }
}

При попытке скомпилировать цей клас мы столкнемся с ошибкой, оскільки в методі work идет обращение к змінної name базового класа Person, которая є приватной.

Визначенийие полей приватными представляє стандартную практику лучше инкапсулировать и защитить дані от некорретного вмешательства зовні. Однак бывают случаи, когда требуется открыть доступ к компоненту (полю або методу) базового класа лише для производных класов. В цем случае компонент класа оголошується за допомогою модификатора protected. Наприклад, якщо суперклас Person оголошує поле name с модификатором protected, а не приватным, то методи класа Employee можуть обращаться к нему напрямую:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.work();     // Name: Bob
                        // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }

    // вивідим дані о компании
    void work(){
        System.out.println("Name: " + name);  // так можна - змінна name - protected
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    protected String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

В Java поле с модификатором protected доступно любому класу в том же пакеті. Тому для надлежащей інкапсуляції нам треба поместить клас Person в отдельный пакет. В цем случае поля и методи с модификатором protected будуть доступны лише класу Employee и другим унаследованным класам.

Перевизначення методів

Похідний клас може визначати свої методи, а може перевизначати методи, успадковані від базового класу. При перевизначенні методу похідний клас визначає метод із тим самим ім'ям, як і базовий клас. Наприклад:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }
    @Override
    void print(){
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    private String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

Здесь производный клас Employee перевизначає метод print из базового класа. Перед переопределяемым методом вказуєся анотація @Override. Данная анотація в принципе необязательна.

При перевизначенні методу він повинен мати рівень доступу не менше, ніж рівень доступу до базового класу. Наприклад, якщо у базовому класі метод має модифікатор public, те й у похідному класі метод повинен мати модифікатор public.

В класе Employee реалізація методу print виводить значення поля company на консоль. Но что делать, якщо мы хочемо, чтобы перевизначенийный метод print також вивідил и значення поля name из базового класа? Напрямую в класе Employee мы не можемо обращаться к полю name, потому что оно приватное. Однак мы можемо вызвать в производном класе реализацию методу базового класа за допомогою слова super:


super.метод(аргументы)

Наприклад:


public class Program{ 
       
    public static void main (String args[]){
           
        Employee bob = new Employee("Bob", "Google");
        bob.print();        // Name: Bob
                            // Company: Google
    }
}

class Employee extends Person{

    private String company;

    Employee(String name, String company){
        super(name); 
        this.company = company;
    }
    @Override
    void print(){
        super.print();  // вызов реалізації метожжа print из класа Person
        System.out.println("Company: " + company);
    }
}  
class Person {
     
    private String name;

    Person(String name){ this.name = name;  }

    void print(){
         
        System.out.println("Name: " + name);
    }
}

Здесь в методі print в класе Employee помощью ключевого слова super мы обращаемся к реалізації методу print из базового класа, которая виводить ім'я:

super.print();
