Поля класу та клас Field

Останнє оновлення: 29.09.2025

За работу с полями класов в пакеті java.lang.reflect отвечает клас Field. Для получения информации о полях и управления ими цей клас определеяет ряд методів, из которых отмечу основные:

  • String getName(): повертає ім'я поля

  • Class getType(): повертає тип поля в виде об'єкта Class

  • int getModifiers(): повертає целое число с различными включенными и выключенными битами, которые описують використовуємоые модификаторы.

  • Set<AccessFlag> accessFlags(): повертає набор флагов доступа для цього поля

  • Object get(Object obj): повертає ім'я поля

  • void set(Object obj, Object value): повертає ім'я поля

Отримання полей

Для манипулирования полями нам прежде всего треба отримати их. Для получения полей типу застосовується ряд методів класа Class:

  • Field[] getFields(): повертає масив об'єктов Field для открытых полей (с модификатором public) данного класа або его суперкласов

  • Field[] getDeclaredFields(): повертає масив об'єктов Field для всех полей данного класа. Метод повертають масив нулевой длины, якщо таких полей нет або якщо об'єкт представляє собой примитивный тип або масив.

  • Field getField(String name): повертає об'єкт Field для публичного поля, ім'я которого передаєся в качестве параметра. Поиск також идет среди полей суперкласов. Если підобное публичное поле отсутствует, то метод генерує виняток NoSuchFieldException

  • Field getDeclaredField(String name): повертає об'єкт Field для поля, ім'я которого передаєся в качестве параметра. Если підобное поле отсутствует, то метод генерує виняток NoSuchFieldException

Наприклад, получим поля класа:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // отримуємо клас для типу Person
        Class cl = Person.class;

        // отримуємо поля 
        Field[] fields = cl.getDeclaredFields();
        for(Field f : fields){

            // отримуємо тип и ім'я поля
            System.out.printf("%s %s;\n", f.getType().getName(), f.getName());
        }
    }
}

class Person {
     
    private String name;
    private int age;

    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
    
    void print(){
        System.out.printf("Person. Name: %s;  Age: %d\n", name, age);
    }
}

Консольный вивід:


java.lang.String name;
int age;

Теперь получим по отдельности поля, використовуючи их ім'я:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // отримуємо клас для типу Person
        Class cl = Person.class;

        try{
            // отримуємо поля name и age
            Field nameField = cl.getDeclaredField("name");
            Field ageField = cl.getDeclaredField("age");

            System.out.println(nameField);
            System.out.println(ageField);
        }
        catch(NoSuchFieldException ex){
            System.out.println(ex);
        }
    }
}

class Person {
     
    private String name;
    private int age;

    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
    
    void print(){
        System.out.printf("Person. Name: %s;  Age: %d\n", name, age);
    }
}

У даному випадку за допомогою методу getDeclaredField() отримуємо поля name и age класа Person. Зверніть увагу, что при передаче імені поля учитывается регистр імені. В итоге на консоль буде выведена информация о полученных полях:


private java.lang.String Person.name
private int Person.age

Манипуляции с полями

Манипуляции обычно включают отримання значення поля або его установку. для цього клас Field надає методи get()/set().

Обращение к приватным полям

По умолчанию механизм рефлексии учитывает контроль доступа Java. Так, при обращении к закрытым полям методи get() и set() по умолчанию вызываюют виняток IllegalAccessException - виняток недопустимого доступа. И по умолчанию методи get и set можна використовувати лише с доступными полями. Механизм безопасности Java дозволяє узнать, якие поля есть у об'єкта, но не дозволяє читать и записывать значення цих полей без соответствующего разрешения.

Но мы можемо перевизначити контроль доступа, вызвав метод setAccessible() для об'єкта Field и передав в метод true:


field.setAccessible(true);

Вызов setAccessible вызывает виняток, якщо доступ не предоставлен. Доступ может быть запрщен модульной системой.

Отримання значення поля

Для получения значення поля застосовується метод get(), який имїїт следующїї визначенийие:


public Object get(Object obj) throws IllegalArgumentException, IllegalAccessException

В качестве параметра в метод передаєся об'єкт, значення поля которого треба отримати. Наприклад, получим значення полей об'єкта Person:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // Исследуемый об'єкт
        Person tom = new Person("Tom", 41);
        // отримуємо клас для типу Person
        Class cl = tom.getClass();

        try{
            // отримуємо поля name и age
            Field nameField = cl.getDeclaredField("name");
            Field ageField = cl.getDeclaredField("age");

            // делаем поля доступными для доступа
            nameField.setAccessible(true);
            ageField.setAccessible(true);

            // отримуємо значення полей
            Object name = nameField.get(tom);  // Tom
            Object age = ageField.get(tom);     // 41

            System.out.println("Name: " + name);
            System.out.println("Age: " + age);  
        }
        catch(Exception ex){
            System.out.println(ex);
        }
    }
}

class Person {
     
    private String name;
    private int age;

    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
    
    void print(){
        System.out.printf("Person. Name: %s;  Age: %d\n", name, age);
    }
}

Зміна поля

Для изменения значення поля застосовується метод set():


public void set(Object obj, Object value) throws IllegalArgumentException, IllegalAccessException

ПЕрвый параметр (obj) представляє об'єкт, поле которого треба змінити, а второй параметр (value) - присваемое полю новое значення. Посмотрим на примере:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // Исследуемый об'єкт
        Person tom = new Person("Tom", 41);
        // отримуємо клас для типу Person
        Class cl = tom.getClass();

        try{
            // отримуємо поля name и age
            Field nameField = cl.getDeclaredField("name");
            Field ageField = cl.getDeclaredField("age");

            // делаем поля доступными для доступа
            nameField.setAccessible(true);
            ageField.setAccessible(true);

            // змінюємо значення полей
            nameField.set(tom, "Bob");
            ageField.set(tom, 22);

            tom.print();    // Person. Name: Bob;  Age: 22
        }
        catch(Exception ex){
            System.out.println(ex);
        }
    }
}

class Person {
     
    private String name;
    private int age;

    public Person(String name, int age){ 

        this.name = name;
        this.age = age;
    }
    
    void print(){
        System.out.printf("Person. Name: %s;  Age: %d\n", name, age);
    }
}

Таким чином, благодаря рефлексии приватность полей є преградой для их изменения.
