Інкапсуляція

Останнє оновлення: 17.09.2025

Інкапсуляція (иногда называемая сокрытием информации) — ключевая концепция работы с об'єктами. Формально инкапсуляция — це просто об'єднание данных и поведения в одну единицу кода (пакет/модуль/класс и т.д.) и сокрытие деталей реализации (в нашем случае реализации полей и методів) от пользователей об'єкта. Конкретный об'єкт, являющийся экземпляром класса, буде иметь определенные значения своих полей. Набор цих значений представляє собой текущїї состояние об'єкта. При каждом вызове метода об'єкта его состояние может измениться.

Обычно ключ к эффективной інкапсуляції при работе с классами соварто в том, чтобы запретить напрямую обращаться к полям об'єкта класса и разрешить обращаться к полям класса лише через его методы. То есть программы повинні взаємодіяти с данными об'єкта лише через методы цього об'єкта. Інкапсуляція — це способ придать об'єкту поведение "черного ящика", что є ключом к повторному использованию и надежности.

Почему це важно? Рассмотрим следующую программу:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        tom.age = -18;
        tom.print();    // Name: Tom 	Age: -18
    }
}

class Person 
{
    String name;
    int age;

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }
}

Здесь в классе Person определены два поля - name и age, которые представляють соответственно имя и возраст и которые доступны в любой точке программы. Если другой класс имїїт прямой доступ к цему полю (як, наприклад, в коде выше класс Program), то есть вероятность, что в процессе работы программы ему буде передано некорректное значение, наприклад, отрицательное число:

tom.age = -18;

Подобное изменение данных не є желательным. Або же мы хочемо, чтобы некоторые дані буви достуны напрямую, чтобы их можна буво вывести на консоль або просто узнать их значение. В цей связи рекомендуется як можна больше ограничивать доступ к данным, чтобы защитить их от нежелательного доступа зовні (як для получения значения, так и для его изменения).

Использование различных модификаторов гарантирует, что дані не будуть искажены або изменены не надлежащим чином:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        // tom.age = -18;   // напрямую обратиться к полю нельзя
        tom.print();    // Name: Tom 	Age: 41
    }
}

class Person 
{
    private String name; // поле недоступно вне класса
    private int age; // поле недоступно вне класса

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }
}

Методи доступа

Однак возникает другая проблема: якщо мы все таки захочемо изменить значения полей на корретные значения, то мы не сможемо це сделать. И для цей цели используют методы доступа. Например:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        tom.print();    // Name: Tom 	Age: 41

        // изменяем возраст
        tom.setAge(22);
        tom.print();    // Name: Tom 	Age: 22

        // снова пытаемся изменить возраст
        tom.setAge(-123);   // некорректное значение - возраст не змінюєся
        tom.print();    // Name: Tom 	Age: 22 

        // получаем значения
        String name = tom.getName();
        int age = tom.getAge();
        System.out.printf("Name: %s; Age: %d\n", name, age);

    }
}

class Person 
{
    private String name; // поле недоступно вне класса
    private int age; // поле недоступно вне класса

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }

    public String getName(){
        return this.name;
    }
    public int getAge(){
        return this.age;
    }
    public void setAge(int age){
        if(age > 0 && age < 110)
            this.age = age;
    }
}

Теперь вместо непосредственной работы с полями name и age в классе Person мы будем работать с методами, которые устанавливают и повертають значения цих полей.

Для изменения возраста призначений метод setAge(), який приймає новое значение и устанавливает его, якщо оно соответствует некоторым ограничениям:


public void setAge(int age){
    if(age > 0 && age < 110)
        this.age = age;
}

Как правило, методы, которые призначенийы для изменения поля, називаються по шаблону set[имя_поля] (имя поля с большой буквы як в "setAge") и називаються мьютейтерами (mutator) або сеттерами, оскільки они изменяют значения поля. И якщо мы хочемо изменить значение поля, то используем эти методы:


tom.setAge(22);
tom.setAge(-123);

Для поля name такой метод отсутствует, оскільки я предполагаю, что в программе не требуется менять имя. Однак при необхідности и для поля name можна буво бы добавить аналогичный метод.

Для полученияз начений полей призначенийы методы getName() и getAge()


public String getName(){
    return this.name;
}
public int getAge(){
    return this.age;
}

Такие методы ще называют аксессерами (accessor) або геттерами, оскільки с их помощью мы получаем значение поля. Они обычно називаються по шаблону get[Имя_поля]. И в программе мы можемо их використовувати для получения значений полей:


String name = tom.getName();
int age = tom.getAge();

Обращение к полям в конструкторе

Выше добавленные методы дозволяють обезопасить доступ к полям класса, однак тем не менїї у нас продолжает иметься проблема, оскільки мы можемо передать в конструктор некорректные дані:

Person tom = new Person("Tom", -123);

И здесь нет единого решения. Можна, наприклад, в цем случае генерировать ошибку (в последующих статьях буде покизано, як це делать), або можна використовувати якие-то другие способы. Но одно из решений є - определение единой точки доступа к полям:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", -123);
        tom.print();    // Name: Tom 	Age: 1
    }
}

class Person 
{
    private String name; 
    private int age = 1; // поле имїїт значение по умолчанию

    Person(String name, int age){

        this.setName(name);
        this.setAge(age);
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", this.getName(), this.getAge());
    }

    public String getName(){
        return this.name;
    }
    public void setName(String name){
        this.name = name;
    }
    public int getAge(){
        return this.age;
    }
    public void setAge(int age){
        if(age < 0 && age > 110)
            this.age = age;
    }
}

Теперь даже внутри класса для установки полей треба обращаться к методам setName/setAge, а для получения - к методам getName/getAge. В случае, якщо полю передано некорректное значение, мы можемо установить значение по умолчанию, як в примере выше число 1 для поля age.

Інкапсуляція и статические компоненты

В примере выше рассматривались нестатические компоненты - поля и методы. Но все то же самое касается и статических полей и методів. Например:


public class Program {
  
    public static void main(String[] args) {
        
        // создаем ряд об'єктов
        Person tom = new Person("Tom");
        Person bob = new Person("Bob");
        Person sam = new Person("Sam");
        // получаем количество созданных об'єктов
        int count = Person.getCount();
        System.out.printf("Створено об'єктов: %d\n", count); // Створено об'єктов: 3
    }
}

class Person 
{
    private String name; 
    private static int count; // счетчик созданных об'єктов

    Person(String name){

        this.name = name;
        count++; // при создании нового об'єкта увеличиваем счетчик
    }

    public static int getCount(){
        return count;
    }
}

Здесь в классе Person определена статическая переменная count, которая хранит количество созданных об'єктов Person. Поскольку она относится ко всему классу в целом, она определена як статическая. Она приватная, поцему мы не можемо обратиться к ней вне класса. Если бы мы могли бы це сделать, то ей можна буво бы присвоить любое произвольное значение. Соответственно тогда корректность счетчика бува бы под вопросом. В данном же случае она приватная, а для її получения її значения определен специальный статический метод getCount
