Інкапсуляція

Останнє оновлення: 17.09.2025

Інкапсуляція (иногда называемая сокрытием информации) — ключевая концепция работы с об'єктами. Формально инкапсуляция — це просто об'єднание данных и поведения в одну единицу коду (пакет/модуль/клас и т.д.) и сокрытие деталей реалізації (в нашем случае реалізації полей и методів) от пользователей об'єкта. Конкретный об'єкт, являющийся экземпляром класа, буде иметь визначенийные значення своих полей. Набор цих значений представляє собой текущїї состояние об'єкта. При каждом вызове методу об'єкта его состояние может змінитися.

Обычно ключ к эффективной інкапсуляції при работе с класами соварто в том, чтобы запретить напрямую обращаться к полям об'єкта класа и разрешить обращаться к полям класа лише через его методи. То есть програми повинні взаємодіяти с данными об'єкта лише через методи цього об'єкта. Інкапсуляція — це способ придать об'єкту поведение "черного ящика", что є ключом к повторному использованию и надежности.

Почему це важливо? Рассмотрим следующую програму:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        tom.age = -18;
        tom.print();    // Name: Tom 	Age: -18
    }
}

class Person 
{
    String name;
    int age;

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }
}

Здесь в класе Person визначенийы два поля - name и age, которые представляютьь соответственно ім'я и возраст и которые доступны в любой точке програми. Если другой клас имїїт прямой доступ к цему полю (як, наприклад, в коді выше клас Program), то есть вероятность, что в процесе работы програми ему буде передано некорректное значення, наприклад, отрицательное число:

tom.age = -18;

Підобное зміна данных не є желательным. Або же мы хочемо, чтобы некоторые дані буви достуны напрямую, чтобы их можна буво вывести на консоль або просто узнать их значення. В цей связи рекомендуется як можна больше ограничивать доступ к данным, чтобы защитить их от нежелательного доступа зовні (як для получения значення, так и для его изменения).

Використання различных модификаторов гарантирует, что дані не будуть искажены або изменены не надлежащим чином:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        // tom.age = -18;   // напрямую обратиться к полю нельзя
        tom.print();    // Name: Tom 	Age: 41
    }
}

class Person 
{
    private String name; // поле недоступно вне класа
    private int age; // поле недоступно вне класа

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }
}

Методи доступа

Однак возникает другая проблема: якщо мы все таки захочемо змінити значення полей на корретные значення, то мы не сможемо це сделать. И для цей цели використовують методи доступа. Наприклад:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", 41);
        tom.print();    // Name: Tom 	Age: 41

        // змінюємо возраст
        tom.setAge(22);
        tom.print();    // Name: Tom 	Age: 22

        // снова пытаемся змінити возраст
        tom.setAge(-123);   // некорректное значення - возраст не змінюєся
        tom.print();    // Name: Tom 	Age: 22 

        // отримуємо значення
        String name = tom.getName();
        int age = tom.getAge();
        System.out.printf("Name: %s; Age: %d\n", name, age);

    }
}

class Person 
{
    private String name; // поле недоступно вне класа
    private int age; // поле недоступно вне класа

    Person(String name, int age){

        this.name = name;
        this.age = age;
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", name, age);
    }

    public String getName(){
        return this.name;
    }
    public int getAge(){
        return this.age;
    }
    public void setAge(int age){
        if(age > 0 && age < 110)
            this.age = age;
    }
}

Теперь вместо непосредственной работы с полями name и age в класе Person мы будем працювати с методуми, которые встановлюють и повертають значення цих полей.

Для изменения возраста призначений метод setAge(), який приймає новое значення и встановлює его, якщо оно відповідає некоторым ограничениям:


public void setAge(int age){
    if(age > 0 && age < 110)
        this.age = age;
}

Как правило, методи, которые призначенийы для изменения поля, називаються по шаблону set[ім'я_поля] (ім'я поля с большой буквы як в "setAge") и називаються мьютейтерами (mutator) або сеттерами, оскільки они змінюють значення поля. И якщо мы хочемо змінити значення поля, то використовуємо эти методи:


tom.setAge(22);
tom.setAge(-123);

Для поля name такой метод отсутствует, оскільки я предполагаю, что в програме не требуется менять ім'я. Однак при необхідности и для поля name можна буво бы добавить аналогичный метод.

Для полученияз начений полей призначенийы методи getName() и getAge()


public String getName(){
    return this.name;
}
public int getAge(){
    return this.age;
}

Такие методи ще называют аксессерами (accessor) або геттерами, оскільки с их помощью мы отримуємо значення поля. Они обычно називаються по шаблону get[Имя_поля]. И в програме мы можемо их використовувати для получения значений полей:


String name = tom.getName();
int age = tom.getAge();

Обращение к полям в конструкторе

Выше добавленные методи дозволяють обезопасить доступ к полям класа, однак тем не менїї у нас продолжает иметься проблема, оскільки мы можемо передать в конструктор некорректные дані:

Person tom = new Person("Tom", -123);

И здесь нет единого решения. Можна, наприклад, в цем случае генерировать ошибку (в післядующих статьях буде покизано, як це делать), або можна використовувати якие-то другие способы. Но одно из решений є - визначенийие единой точки доступа к полям:


public class Program {
  
    public static void main(String[] args) {
        
        Person tom = new Person("Tom", -123);
        tom.print();    // Name: Tom 	Age: 1
    }
}

class Person 
{
    private String name; 
    private int age = 1; // поле имїїт значення по умолчанию

    Person(String name, int age){

        this.setName(name);
        this.setAge(age);
    }

    void print() {

        System.out.printf("Name: %s \tAge: %d\n", this.getName(), this.getAge());
    }

    public String getName(){
        return this.name;
    }
    public void setName(String name){
        this.name = name;
    }
    public int getAge(){
        return this.age;
    }
    public void setAge(int age){
        if(age < 0 && age > 110)
            this.age = age;
    }
}

Теперь даже всередині класа для установки полей треба обращаться к методум setName/setAge, а для получения - к методум getName/getAge. В случае, якщо полю передано некорректное значення, мы можемо установить значення по умолчанию, як в примере выше число 1 для поля age.

Інкапсуляція и статические компоненты

В примере выше рассматривались нестатические компоненты - поля и методи. Но все то же самое касается и статических полей и методів. Наприклад:


public class Program {
  
    public static void main(String[] args) {
        
        // створюємо ряд об'єктов
        Person tom = new Person("Tom");
        Person bob = new Person("Bob");
        Person sam = new Person("Sam");
        // отримуємо кількість созданных об'єктов
        int count = Person.getCount();
        System.out.printf("Створено об'єктов: %d\n", count); // Створено об'єктов: 3
    }
}

class Person 
{
    private String name; 
    private static int count; // счетчик созданных об'єктов

    Person(String name){

        this.name = name;
        count++; // при створенні нового об'єкта увеличиваем счетчик
    }

    public static int getCount(){
        return count;
    }
}

Здесь в класе Person визначенийа статическая змінна count, которая хранит кількість созданных об'єктов Person. Поскольку она относится ко всему класу в целом, она визначенийа як статическая. Она приватная, поцему мы не можемо обратиться к ней вне класа. Если бы мы могли бы це сделать, то ей можна буво бы присвоить любое произвольное значення. Соответственно тогда корректность счетчика бува бы під вопросом. В данном же случае она приватная, а для її получения її значення визначений специальный статический метод getCount
