Обробка винятків та конструкція try...catch...finally

Останнє оновлення: 07.10.2025

Нередко в процесе виконання програми можуть возникать ошибки, при том необязательно по вине разработчика. Некоторые из них трудно предусмотреть або предвидеть, а иногда и вовсе невозможна. Так, наприклад, может неожиданно оборваться сетевое підключение при передаче файлу. Підобные ситуации називаються винятками.

В мові Java предусмотрены специальные средства для обработки підобных ситуаций. Одним из таких средств є конструкция try...catch...finally. При возникновении винятки в блоке try управление переходит в блок catch, який может обпрацювати данное виняток. Если такого блока не найдено, то пользователю отображается сообщение о необработанном исключении, а дальнейшїї виконання програми останавливается. И чтобы програма не упала из-за возникшей ошибки, застосовується блок try..catch..finally:


try{
     
}
catch (Тип_винятки змінна){
     
}
finally{
     
}

При применении блока try...catch..finally вначале виконуються все инструкции в блоке try. Если в цем блоке не возникло винятків, то після его виконання начинает выполняться блок finally. И затем конструкция try..catch..finally завершує свою работу.

Если же в блоке try вдруг возникает виняток, то обычный порядок виконання останавливается, и среда виконання ищет блок catch, який может обпрацювати данный тип винятки. Если нужный блок catch найден, то он виконується, и після его завершення виконується блок finally.

Если нужный блок catch

Застосування конструкції try..catch..finally

Розглянемо найпростіший приклад:


class Program{
     
	public static void main(String[] args) {
         
        int a = 5;
        int b = 0;
        int result = a / b;
        System.out.printf("Результат: %d\n", result);
    }
}

У разі відбувається розпіділ числа на 0, що призведе до генерації винятку. У результаті побачимо на консолі таку картину:


Exception in thread "main" java.lang.ArithmeticException: / by zero
	at Program.main(Program.java:7)

Таким чином, консоль нас проинформирует, что в процесе виконання програми произошло виняток типу java.lang.ArithmeticException и снабдит сообщение куцей информацией об исключении: "by zero". Тем не менїї по цему сообщению мы можемо понять, что проблема в делении на ноль. Крім того, мы видим, что непосредственно проблема возникла в методі Program.main, який располагается в файле Program.java на рядківе 7. Це облегчает поиск проблемого участка коду в исходном файле.

Звичайно, саме ця ситуація до певної міри штучна - навіщо ділити на нуль? А якщо число піділ приходить ззовні, то перед піділом ми можемоо перевірити, чи це нуль чи не нуль, і тим самим уникнути падіння програми:


class Program{
     
	public static void main(String[] args) {
         
        divide(10, 2);
        divide(10, 0);
    }

    static void divide(int a, int b){

        if(b != 0) System.out.println(a / b);
        else System.out.println("Деление на ноль");
    }
}

Но проблема заключается в том, что не все ситуации мы можемо предотвратить таким чином. Тому посмотрим, як мы можемо пріменіть конструкцию try...catch...finally, чтобы избежать підобного аварийного завершення програми:


class Program{
     
	public static void main(String[] args) {
         
        divide(10, 0);
        System.out.println("Конец програми");
    }

    static void divide(int a, int b){

        try{

            int result = a / b;
            System.out.printf("Результат: %d\n", result);
        }
        catch(Throwable _){

            System.out.println("Підзникло виняток!");
        }
        finally{

            System.out.println("Блок finally");
        }
    }
}

Розглянемо основні моменти. Насамперед ми маємо певний код, який потенційно може викликати виняток:

int result = a / b;

Тому его помещаем блок try.


try{

    int result = a / b;
    System.out.printf("Результат: %d\n", result);
}

Если в блоке try вдруг возникает виняток (при b = 0), то обычный порядок виконання останавливается и переходит к инструкции сatch. Выражение catch имїїт следующий синтаксис:

catch (тип_винятки ім'я_змінної){

    // выполняемые инструкции при возвникновении винятки
}

Після оператора catch в скобках вказуєся тип винятки и затем ім'я змінної цього винятки (то есть як и при оголошенийии змінної). Якщо виняток відповідає цьому типу, то виконується даний блок catch.

В нашем случае в качестве типу винятки использован максимально широкий тип, який відповідає любому исключению - це клас Throwable:


catch(Throwable _){

    System.out.println("Підзникло виняток!");
}

Таким чином, обробка винятки типу Throwable предполагает обработку любого винятки, в том числе деления на ноль. Но здесь в целях упрощения примера нам поки змінна цього винятки не важна, мы її нияк не використовуємо, поцему вместо імені змінної я использовал прочерк. А в самом блоке catch просто виводиться диагностическое сообщение.

И після блока catch идет блок finally, який також просто виводить сообщение на консоль.


finally{

    System.out.println("Блок finally");
}

Виконання try..catch..finally

Посмотрим виконання выше визначенийной конструкции try..catch..finally на двух примерах: при упешном и неудачном выполнении програми. Підзьмем первую ситуацию - когда делитель равен 0:


public static void main(String[] args) {
         
    divide(10, 0);
    System.out.println("Конец програми");
}

Оскільки дільник дорівнює 0, і очевидно виникне помилка, відповідно програма не зможе виконати піділ, і ми отримаємо наступний консольний висновок:


Підзникло виняток!
Блок finally
Конец програми

Якщо по інструкціям, хід програми виглядає так:

  1. divide(10, 0);

    Пеореходим в метод divide

  2. int result = a / b;

    Поскольку b равно 0, переход в блок catch

  3. System.out.println("Підзникло виняток!");

    Переход в блок finally

  4. System.out.println("Блок finally");

    Блок finally отработал, метод divide завершил свое виконання, поцему повертаємося в мтеод main

  5. System.out.println("Конец програми");

    Післядняя инструкция методу main выполнена, програма завершена

Тепер підивимося іншу ситуацію - коли дільник не дорівнює 0:


public static void main(String[] args) {
         
    divide(10, 2);
    System.out.println("Конец програми");
}

Ця програма успішно виконає піділ, і ми отримаємо наступний консольний висновок:


Результат: 5
Блок finally
Конец програми

Якщо по інструкціям, хід програми виглядає так:

  1. divide(10, 2);

    Пеореходим в метод divide

  2. int result = a / b;

    Поскольку b равно 2, виконується деление, а змінна result отримує значення 5

  3. System.out.printf("Результат: %d\n", result);

    Блок try отработал, поцему переходим в блок finally

  4. System.out.println("Блок finally");

    Блок finally отработал, метод divide завершил свое виконання, поцему повертаємося в мтеод main

  5. System.out.println("Конец програми");

    Післядняя инструкция методу main выполнена, програма завершена

Скорочена версія конструкції try..catch..finally

При использовании конструкции try..catch..finally обязательным блоком є лише блок try. Один из остальных двух блоков можна опустить и оставить або catch (якщо блок не нвжен, або . Наприклад, часто блок не використовується, и використовуються лише try..catch:


static void divide(int a, int b){

    try{

        int result = a / b;
        System.out.printf("Результат: %d\n", result);
    }
    catch(Throwable _){

        System.out.println("Підзникло виняток!");
    }
}

Можна оставить блок finally, но в цем случае при возникновении винятки оно не буде обработано, и програма упадет с ошибкой:


static void divide(int a, int b){

    try{

        int result = a / b; 
        System.out.printf("Результат: %d\n", result);
    }
    finally{

        System.out.println("Блок finally");
    }
}

Повернення результату з try..catch..finally

Всередині конструкции try..catch..finally может використовуватися оператор return для возвращения результата из методу. Наприклад:


static int parse(String s)
{
    try
    {
        return Integer.parseInt(s);
    }
    catch(Throwable _){

        System.out.println("Ошибка преобразования");
        return 0;
    }
 }

Здесь в блоке try повертаєся результат методу Integer.parseInt(). Цет метод преобразует рядок в число. Однак якщо рядок нельзя конвертировать в число, то возникает ошибка. За допомогою блока catch мы перехватываем ошибку и повертаємо число 0. Хотя здесь спорный момент: варто ли возвращать значення при ошибке, а якщо возвращать, то якое. И Java для підобных ситуаций имїїт болїї гибкие механизмы. Но возьмем данный пример за основу.

Далі у програмі ми можемоо викликати метод parse і отримати результат:


class Program{
     
	public static void main(String[] args) throws Exception {
         
        int result = parse("5"); 
        System.out.println(result);     // 5

        result = parse("t");            // Ошибка преобразования
        System.out.println(result);     // 0
    }

    static int parse(String s)
    {
        try
        {
            return Integer.parseInt(s);
        }
        catch(Throwable _){

            System.out.println("Ошибка преобразования");
            return 0;
        }
    }
}

Но варто учитывать, что, якщо конструкция try містить блок finally, то цей блок обязательно виконується. И якщо блок finally також містить оператор return, то он маскирует исходное повертаємоое значення. Наприклад


class Program{
     
	public static void main(String[] args) throws Exception {
         
        int result = parse("5");
        System.out.println(result);        // -1

        result = parse("t");                // Ошибка преобразования 
        System.out.println(result);        // -1
    }

    static int parse(String s)
    {
        try
        {
            return Integer.parseInt(s);
        }
        catch(Throwable _){

            System.out.println("Ошибка преобразования");
            return 0;
        }
        finally{
            return -1;
        }
    }
}

Здесь блок finally повертає значення -1. И вне зависимости от того, успешно буво ли преобразование або нет, метод все равно возвратит -1, оскільки після блока try и catch обязательно виконується блок finally

Обробка винятків та умовні конструкції

Варто відзначити, что конструкция try..catch на низком уровне досить дорогая операція. И якщо ряд исключительных ситуаций может быть предвиден разработчиком, то лучше обойтись без try..catch, а підобные ситуации обпрацювати за допомогою условных конструкций. Как, наприклад, в случае с делением на 0 предпочтительнїї проверить второе число на 0, чем використовувати конструкцию try..catch.
