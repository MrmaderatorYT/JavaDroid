Обробка винятків та конструкція try...catch...finally

Останнє оновлення: 07.10.2025

Нередко в процессе виконання програми можуть возникать ошибки, при том необязательно по вине разработчика. Некоторые из них трудно предусмотреть або предвидеть, а иногда и вовсе невозможна. Так, наприклад, может неожиданно оборваться сетевое подключение при передаче файла. Подобные ситуации називаються винятками.

В мові Java предусмотрены специальные средства для обработки подобных ситуаций. Одним из таких средств є конструкция try...catch...finally. При возникновении исключения в блоке try управление переходит в блок catch, який может обработать данное исключение. Если такого блока не найдено, то пользователю отображается сообщение о необработанном исключении, а дальнейшїї выполнение программы останавливается. И чтобы программа не упала из-за возникшей ошибки, застосовується блок try..catch..finally:


try{
     
}
catch (Тип_исключения переменная){
     
}
finally{
     
}

При применении блока try...catch..finally вначале выполняются все инструкции в блоке try. Если в цем блоке не возникло исключений, то после его виконання начинает выполняться блок finally. И затем конструкция try..catch..finally завершує свою работу.

Если же в блоке try вдруг возникает исключение, то обычный порядок виконання останавливается, и среда виконання ищет блок catch, який может обработать данный тип исключения. Если нужный блок catch найден, то он виконується, и после его завершения виконується блок finally.

Если нужный блок catch

Застосування конструкції try..catch..finally

Розглянемо найпростіший приклад:


class Program{
     
	public static void main(String[] args) {
         
        int a = 5;
        int b = 0;
        int result = a / b;
        System.out.printf("Результат: %d\n", result);
    }
}

У разі відбувається розподіл числа на 0, що призведе до генерації винятку. У результаті побачимо на консолі таку картину:


Exception in thread "main" java.lang.ArithmeticException: / by zero
	at Program.main(Program.java:7)

Таким чином, консоль нас проинформирует, что в процессе виконання програми произошло исключение типу java.lang.ArithmeticException и снабдит сообщение куцей информацией об исключении: "by zero". Тем не менїї по цему сообщению мы можемо понять, что проблема в делении на ноль. Крім того, мы видим, что непосредственно проблема возникла в методе Program.main, який располагается в файле Program.java на строке 7. Це облегчает поиск проблемого участка кода в исходном файле.

Звичайно, саме ця ситуація до певної міри штучна - навіщо ділити на нуль? А якщо число поділ приходить ззовні, то перед поділом ми можемоо перевірити, чи це нуль чи не нуль, і тим самим уникнути падіння програми:


class Program{
     
	public static void main(String[] args) {
         
        divide(10, 2);
        divide(10, 0);
    }

    static void divide(int a, int b){

        if(b != 0) System.out.println(a / b);
        else System.out.println("Деление на ноль");
    }
}

Но проблема заключается в том, что не все ситуации мы можемо предотвратить таким чином. Тому посмотрим, як мы можемо применить конструкцию try...catch...finally, чтобы избежать подобного аварийного завершения программы:


class Program{
     
	public static void main(String[] args) {
         
        divide(10, 0);
        System.out.println("Конец программы");
    }

    static void divide(int a, int b){

        try{

            int result = a / b;
            System.out.printf("Результат: %d\n", result);
        }
        catch(Throwable _){

            System.out.println("Підзникло исключение!");
        }
        finally{

            System.out.println("Блок finally");
        }
    }
}

Розглянемо основні моменти. Насамперед ми маємо певний код, який потенційно може викликати виняток:

int result = a / b;

Тому его помещаем блок try.


try{

    int result = a / b;
    System.out.printf("Результат: %d\n", result);
}

Если в блоке try вдруг возникает исключение (при b = 0), то обычный порядок виконання останавливается и переходит к инструкции сatch. Выражение catch имїїт следующий синтаксис:

catch (тип_исключения имя_змінної){

    // выполняемые инструкции при возвникновении исключения
}

После оператора catch в скобках указывается тип исключения и затем имя змінної цього исключения (то есть як и при оголошенийии змінної). Якщо виняток відповідає цьому типу, то виконується даний блок catch.

В нашем случае в качестве типу исключения использован максимально широкий тип, який соответствует любому исключению - це класс Throwable:


catch(Throwable _){

    System.out.println("Підзникло исключение!");
}

Таким чином, обработка исключения типу Throwable предполагает обработку любого исключения, в том числе деления на ноль. Но здесь в целях упрощения примера нам поки переменная цього исключения не важна, мы її нияк не используем, поцему вместо имени змінної я использовал прочерк. А в самом блоке catch просто виводиться диагностическое сообщение.

И после блока catch идет блок finally, який також просто выводит сообщение на консоль.


finally{

    System.out.println("Блок finally");
}

Виконання try..catch..finally

Посмотрим выполнение выше определенной конструкции try..catch..finally на двух примерах: при упешном и неудачном выполнении программы. Підзьмем первую ситуацию - когда делитель равен 0:


public static void main(String[] args) {
         
    divide(10, 0);
    System.out.println("Конец программы");
}

Оскільки дільник дорівнює 0, і очевидно виникне помилка, відповідно програма не зможе виконати поділ, і ми отримаємо наступний консольний висновок:


Підзникло исключение!
Блок finally
Конец программы

Якщо по інструкціям, хід програми виглядає так:

  1. divide(10, 0);

    Пеореходим в метод divide

  2. int result = a / b;

    Поскольку b равно 0, переход в блок catch

  3. System.out.println("Підзникло исключение!");

    Переход в блок finally

  4. System.out.println("Блок finally");

    Блок finally отработал, метод divide завершил свое выполнение, поцему возвращаемся в мтеод main

  5. System.out.println("Конец программы");

    Последняя инструкция метода main выполнена, программа завершена

Тепер подивимося іншу ситуацію - коли дільник не дорівнює 0:


public static void main(String[] args) {
         
    divide(10, 2);
    System.out.println("Конец программы");
}

Ця програма успішно виконає поділ, і ми отримаємо наступний консольний висновок:


Результат: 5
Блок finally
Конец программы

Якщо по інструкціям, хід програми виглядає так:

  1. divide(10, 2);

    Пеореходим в метод divide

  2. int result = a / b;

    Поскольку b равно 2, виконується деление, а переменная result отримує значение 5

  3. System.out.printf("Результат: %d\n", result);

    Блок try отработал, поцему переходим в блок finally

  4. System.out.println("Блок finally");

    Блок finally отработал, метод divide завершил свое выполнение, поцему возвращаемся в мтеод main

  5. System.out.println("Конец программы");

    Последняя инструкция метода main выполнена, программа завершена

Скорочена версія конструкції try..catch..finally

При использовании конструкции try..catch..finally обязательным блоком є лише блок try. Один из остальных двух блоков можна опустить и оставить або catch (якщо блок не нвжен, або . Наприклад, часто блок не використовується, и використовуються лише try..catch:


static void divide(int a, int b){

    try{

        int result = a / b;
        System.out.printf("Результат: %d\n", result);
    }
    catch(Throwable _){

        System.out.println("Підзникло исключение!");
    }
}

Можна оставить блок finally, но в цем случае при возникновении исключения оно не буде обработано, и программа упадет с ошибкой:


static void divide(int a, int b){

    try{

        int result = a / b; 
        System.out.printf("Результат: %d\n", result);
    }
    finally{

        System.out.println("Блок finally");
    }
}

Повернення результату з try..catch..finally

Внутри конструкции try..catch..finally может використовуватися оператор return для возвращения результата из метода. Например:


static int parse(String s)
{
    try
    {
        return Integer.parseInt(s);
    }
    catch(Throwable _){

        System.out.println("Ошибка преобразования");
        return 0;
    }
 }

Здесь в блоке try повертаєся результат метода Integer.parseInt(). Цет метод преобразует строку в число. Однак якщо строку нельзя конвертировать в число, то возникает ошибка. С помощью блока catch мы перехватываем ошибку и возвращаем число 0. Хотя здесь спорный момент: варто ли возвращать значение при ошибке, а якщо возвращать, то якое. И Java для подобных ситуаций имїїт болїї гибкие механизмы. Но возьмем данный пример за основу.

Далі у програмі ми можемоо викликати метод parse і отримати результат:


class Program{
     
	public static void main(String[] args) throws Exception {
         
        int result = parse("5"); 
        System.out.println(result);     // 5

        result = parse("t");            // Ошибка преобразования
        System.out.println(result);     // 0
    }

    static int parse(String s)
    {
        try
        {
            return Integer.parseInt(s);
        }
        catch(Throwable _){

            System.out.println("Ошибка преобразования");
            return 0;
        }
    }
}

Но варто учитывать, что, якщо конструкция try містить блок finally, то цей блок обязательно виконується. И якщо блок finally також містить оператор return, то он маскирует исходное возвращаемое значение. Например


class Program{
     
	public static void main(String[] args) throws Exception {
         
        int result = parse("5");
        System.out.println(result);        // -1

        result = parse("t");                // Ошибка преобразования 
        System.out.println(result);        // -1
    }

    static int parse(String s)
    {
        try
        {
            return Integer.parseInt(s);
        }
        catch(Throwable _){

            System.out.println("Ошибка преобразования");
            return 0;
        }
        finally{
            return -1;
        }
    }
}

Здесь блок finally повертає значение -1. И вне зависимости от того, успешно буво ли преобразование або нет, метод все равно возвратит -1, оскільки после блока try и catch обязательно виконується блок finally

Обробка винятків та умовні конструкції

Варто відзначити, что конструкция try..catch на низком уровне довольно дорогая операция. И якщо ряд исключительных ситуаций может быть предвиден разработчиком, то лучше обойтись без try..catch, а подобные ситуации обработать за допомогою условных конструкций. Как, наприклад, в случае с делением на 0 предпочтительнїї проверить второе число на 0, чем використовувати конструкцию try..catch.
