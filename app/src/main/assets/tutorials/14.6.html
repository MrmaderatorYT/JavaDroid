Методи и клас Method

Останнє оновлення: 29.09.2025

Клас java.lang.reflect.Method надає информацию об одном методі класа або інтерфейса и доступ к нему. Для получения информации о методух и управления ими клас Method определеяет ряд методів. Отмечу основные из них:

  • String getName(): повертає ім'я методу

  • Class[] getExceptionTypes(): повертає генерируемые методом типи винятків в виде масива об'єктов Class

  • int getModifiers(): повертає целое число с различными включенными и выключенными битами, которые описують використовуємоые модификаторы.

  • int getParameterCount(): повертає кількість параметрів

  • Class[] getParameterTypes(): повертає типи параметрів методу

  • Class getReturnType(): повертає тип повертаємоого значення методу

  • Object invoke(Object obj, Object... args): вызывает метод, представленный цим об'єктом методу, для указанного об'єкта с указанными параметрами.

Отримання методів

Для получения методів типу застосовується ряд методів класа Class:

  • Method[] getMethods(): повертає масив об'єктов Method для открытых методів (с модификатором public) данного класа або его суперкласов

  • Method[] getDeclaredКлас java.lang.reflect.Methods(): повертає масив об'єктов Method для всех методів данного класа. Метод повертають масив нулевой длины, якщо таких методів нет або якщо об'єкт представляє собой примитивный тип або масив.

  • Method getMethod(String name, Class... parameterTypes): повертає об'єкт Method для публичного методу, ім'я которого передаєся в качестве параметра. Второй параметр вказує на типи параметрів методу. Поиск також идет среди методів суперкласов. Если підобное публичный метод отсутствует, то метод генерує виняток NoSuchMethodException

  • Method getDeclaredMethod(String name, Class... parameterTypes): повертає об'єкт Method для методу, ім'я которого передаєся в качестве параметра. Второй параметр вказує на типи параметрів методу. Если підобный метод отсутствует, то метод генерує виняток NoSuchMethodException

Наприклад, получим все методи некоторого класа:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Operation.class;
        Method[] methods = cl.getDeclaredMethods();
        for(Method m : methods){
            System.out.println(m);
        }
    }
}

class Operation{

    static int sum(int val1, int val2){
        return val1 + val2;
    }

    static double sum(double val1, double val2){
        return val1 + val2;
    }

    static int subtract(int val1, int val2){
        return val1 - val2;
    }
}

У даному випадку отримуємо методи класа Operation. И консоль повинна нам вывести следующие рядки:


static int Operation.sum(int,int)
static double Operation.sum(double, double)
static int Operation.subtract(int,int)

Исследование методу

Исследуем метод sum из выше визначенийного класа Operation:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Operation.class;
        try{
            // отримуємо метод sum с двумя параметрами типу int
            Method m = cl.getDeclaredMethod("sum", int.class, int.class);
            // отримуємо ім'я методу
            String methodName = m.getName();
            // отримуємо модификаторы
            String modifiers = Modifier.toString(m.getModifiers());

            // отримуємо кількість параметрів
            int paramsCount = m.getParameterCount();
            // отримуємо повертаємоый тип
            Class retType = m.getReturnType();
            // отримуємо типи параметрів
            Class[] paramTypes = m.getParameterTypes();

            // вивідим информацию о методі
            System.out.print(modifiers + " " + retType.getName() + " " + methodName + "(");
            for (int j = 0; j < paramsCount; j++)
            {
                if (j > 0) System.out.print(", ");
                System.out.print(paramTypes[j].getName());
            }
            System.out.println(");");
        }
        catch(Exception ex){

            System.out.println(ex);
        }
    }
}

class Operation{

    static int sum(int val1, int val2){
        return val1 + val2;
    }

    static double sum(double val1, double val2){
        return val1 + val2;
    }

    static int subtract(int val1, int val2){
        return val1 - val2;
    }
}

У даному випадку отримуємо метод sum, який приймає два параметра типу int:

Method m = cl.getDeclaredMethod("sum", int.class, int.class);

Далїї післядовательно отримуємо ім'я, повертаємоый тип, модификаторы и типи параметрів методу и вивідим их на консоль. В итоге консоль повинна нам вывести следующую рядок:

static int sum(int, int);

Вызов методу

Для вызова методу в класе Method визначений метод invoke():


public Object invoke(Object obj, Object... args)
              throws IllegalAccessException, InvocationTargetException

Первый параметр (obj) представляє об'єкт, для которого буде вызываться метод. Если вызываемый метод статический, то параметр obj игнорируется.

Второй параметр (args) представляє аргументы, передаваемые вызываемому методу. Если вызываемый метод не приймає нияких параметрів, то параметр args опускается.

Підзвращаемое значення - це результат вызываемого методу.

Вызов статического методу

Вызовем статический метод sum класа Operation:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Class cl = Operation.class;
        try{
            // отримуємо метод sum с двумя параметрами типу int
            Method m = cl.getDeclaredMethod("sum", int.class, int.class);
            int val1 = 23;
            int val2 = 4;
            // вызываем метод и отримуємо результат
            int result = (int) m.invoke(null, val1, val2);

            System.out.printf("sum(%d, %d) = %d\n", val1, val2, result);        // sum(23, 4) = 27
        }       
        catch(Exception ex){

            System.out.println(ex);
        }
    }
}

class Operation{

    static int sum(int val1, int val2){
        return val1 + val2;
    }

    static int subtract(int val1, int val2){
        return val1 - val2;
    }
}

Поскольку вызываемый метод sum - статический, то в качестве первого параметра передаєся null:

int result = (int) m.invoke(null, val1, val2);

Результаи преобразуется к типу int, оскільки мы значем, что метод повинен возвратить значення цього типу. Консольный вивід програми:

sum(23, 4) = 27

Вызов нестатического методу

Рассмотрим вызов нестатического методу:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        // об'єкт, для которого буде вызываться метод
        Person tom = new Person("Tom");
        Class cl = tom.getClass();
        try{
            // отримуємо метод setName с параметром типу String
            Method m = cl.getDeclaredMethod("setName", String.class);
            // новое ім'я
            String name = "Eugene";
            // вызываем метод
            m.invoke(tom, name);

            tom.print();        // Person Eugene
        }       
        catch(Exception ex){

            System.out.println(ex);
        }
    }
}
class Person {
     
    private String name;

    void setName(String name){ this.name = name; }

    Person(String name){  this.name = name; }
    
    void print(){
        System.out.println("Person " + name);
    }
}

У даному випадку для об'єкта Person вызываем метод setName(), який змінює значення поля name.

Приватные методи

Как и в случае с полями и конструкторами, механизм рефлексии Java запрещает обращение к приватным методум (визначенийным с модификатором private). В цем случае для доступа к методу необхідно сделать его доступным за допомогою методу setAccessible(), передав в него true:


import java.lang.reflect.*;

public class Program{
      
    public static void main(String[] args) {
        
        Person tom = new Person("Tom");
        Class cl = tom.getClass();

        try{
            Method m = cl.getDeclaredMethod("setName", String.class);
            m.setAccessible(true);      // открываем доступ к методу

            String name = "Eugene";

            m.invoke(tom, name);

            tom.print();        // Person Eugene
        }       
        catch(Exception ex){

            System.out.println(ex);
        }
    }
}
class Person {
     
    private String name;
    // приватный метод
    private void setName(String name){ this.name = name; }

    Person(String name){  this.name = name; }
    
    void print(){
        System.out.println("Person " + name);
    }
}
