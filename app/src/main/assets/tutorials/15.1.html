Введение в анотації

Останнє оновлення: 01.11.2025

Анотації в языке програмирования Java представляютьь специальную форму метаданных (данных о данных), яку можна добавлять в исходный код. Они предоставляют дополнительную информацию о програме, которая может використовуватися компилятором, инструментами сборки, фреймворками або во час виконання. Анотації дозволяє отделить конфигурацию и метадані от основной бизнес-логики. Они делают код болїї чистым, декларативным и легким для піддержки, перенося большую часть "магии" настройки на фреймворки и инструменты. Можна сказать, что анотації є основой многих современных Java-технологий, зокрема, веб-фреймворков, фреймворков для тестирования, валидации и т.д.

Сами по себе анотації не виконують нияких действий. Они лишь "маркируют" класы, методи, поля и другие елементы коду, а вже другой код (обработчик анотацій) ищет эти маркеры и виконує на их основе деяку визначенийную логику.

Синтаксически анотація начинается с символа @, за которым йде її ім'я, наприклад, @Override.

В мові Java есть ряд встроенных анотацій, которые використовуються для стандартных завдань. Рассмотрим некоторые наиболїї використовуємоые:

  • @Override: вказує компилятору, что текущий метод перевизначає метод из суперкласа.

  • @Deprecated: эта анотація помечает елемент (клас, метод, поле) як устаревший

  • @SuppressWarnings: отключает визначенийные предупреждения компилятора для аннотированного елемента.

  • @FunctionalInterface: вказує, что інтерфейс призначений для использования в качестве функционального інтерфейса (т.е. он повинен иметь ровно один абстрактный метод).

Застосування анотацій

Анотації оголошенийий можуть застосовуватися ко многим компонентам. И в зависимости от места применения анотації делятся на 2 группы:

  • анотації оголошенийий (declaration annotation)

  • анотації использования типів (type uses annotation)

Анотації оголошенийий

Анотації оголошенийий містять деяку информацию об оголошуємоом елементі и можуть застосовуватися к наступним оголошенням:

  • К класам (включая перечисления) и інтерфейсам (в том числе к інтерфейсам анотацій)

  • К методум

  • К конструкторам

  • К полям (в том числе к константам перечислений и полям класов record).

  • К локальным переменным (в том числе к оголошенийным в конструкциях for и try-with-resources).

  • К параметрам методів

  • К параметрам типів

  • К пакетум и модулям

Для класов, інтерфейсов и модулів анотації размещаются перед ключевым словом class, interface або module и любым модификаторам (при его наличии):

@MyAnnotation public class Person { }

Для змінних, параметрів, а також поле класов-record анотації размещаются перед типом:


@MyAnnotation String name = "Tom";
public User getUser(@MyAnnotation int userId)
record Person(@MyAnnotation String name) {}

Також анотація помещается перед параметром типу в обобщенном класе або методі:


class Person<@MyAnnotation T> { }

Анотації использования типів

Анотації использования типів можуть появляться в следующих местах:

  • С аргументами обобщенного типу:

    Person<@MyAnnotation String>

  • Перед скобками масива:

    
String[] @MyAnnotation [] data // Застосовується к String[]
String @MyAnnotation [][] data // Застосовується к String[][]

    От эих ситуаций треба отличать следующую:

    @MyAnnotation String[][] data

    Здесь анотація застосовується к String

  • С суперкласами и реалізованийными інтерфейсами:

    class Employee extends @MyAnnotation Person.

  • С вызовами конструктора

     new @MyAnnotation Person();

  • С вложенными типуми

     Map.@MyAnnotation Entry

  • С приведениями типів и проверками instanceof:

    
(@MyAnnotation String) text
if (text instanceof @MyAnnotation String)

    Анотації призначенийы лише для использования внешними инструментами. Они не влияют на поведение приведения типів або проверки instanceof.

  • С указанием генерируемых винятків

    void print() throws @MyAnnotation Exception.

  • С підстановочными типуми и ограничениями типів

    
List<@MyAnnotation ? extends Person>
List<? extends @MyAnnotation Person>

  • С ссылками на методи и конструктор:

    @MyAnnotation Person::getName

Анотації можна помещать до або після других модификаторов, таких як static и public. Обычно анотації использования типу размещаются після других модификаторов, а анотації оголошенийий - перед другими модификаторами. Наприклад:


private @MyAnnotation String name;   // Анотація использования типу
@MyAnnotation private int age;      // Анотація змінної / поля класа

Встроенные (стандартные) анотації Java

@Override

Анотація @Override вказує компилятору, что текущий метод перевизначає метод из суперкласа. Если метод с такой сигнатурой в суперкласе не найден, компилятор выдаст ошибку. Це помогает избежать опечаток.

Конечно, при перевизначенийии методу суперкласа в производном класе нам необязательно використовувати данную анотацію. Но розглянемо следующую програму:


class Animal {
    void makeSound() {
        System.out.println("Некий звук");
    }
}
class Cat extends Animal {
    
    void makeSounds() {
        System.out.println("Мяу");
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Cat barsik = new Cat();
        barsik.makeSound(); // Некий звук
    }
}

Клас Animal представляє животное и визначає метод makeSound() для имитации звука животного. Клас Cat перевизначає цей метод. А в методі main() вызываем цей метод. Но посмотрим на консольный вивід програми:

Некий звук

Результат, очевидно, не тот, який ожидался. А почему? А потому что я допустил опечатку: вместо "makeSound" в класе Cat определил метод "makeSounds". В итоге я по сути определил новый метод, який ничего не перевизначає. Но програма бува успешно скомпилирована и успешно выполнила все свои действия! И в том числе для предупреждения підобных ситуаций призначенийа анотація @Override. Застосуємо її:


class Animal {
    void makeSound() {
        System.out.println("Некий звук");
    }
}
class Cat extends Animal {
    
    @Override
    void makeSounds() {
        System.out.println("Мяу");
    }
}

В производном класе Cat попрежнему метод с опечаткой: "makeSounds" вместо "makeSound". Однак? пріменів к методу анотацію @Override, мы указываем, что в суперкласе повинен быть метод с підобным именем, и мы его хочемо перевизначити. Но поскольку в суперкласе нет методу "makeSounds", то при компиляции теперь мы получим ошибку:


Program.java:8: error: method does not override or implement a method from a supertype
    @Override
    ^
1 error
error: compilation failed

Таким чином, мы увидим ошибку и сможемо своевременно її исправить, недопустив некорректного виконання програми:


class Cat extends Animal {
    
    @Override
    void makeSound() {
        System.out.println("Мяу");
    }
}

@Deprecated

Анотація @Deprecated помечает елемент (клас, метод, поле) як устаревший, что может быть полезно, якщо після обновления API класа або пакету, мы хочемо, что якие-то компоненты больше не использовались. Компилятор буде выдавать предупреждение при каждой попытке використовувати такой компонент. Наприклад:


class Cat{
    
    @Deprecated
    void makeSound() {
        System.out.println("Мяу");
    }

    void say() {
        System.out.println("Мяу");
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Cat barsik = new Cat();
        barsik.makeSound();
    }
}

Здесь в класе Cat визначений метод makeSound(), який за допомогою анотації @Deprecated оголошений як устаревший (допустим, мы хочемо, чтобы в будущем разработчики использовали метод say(), який дублирует функциональность методу makeSound()). Данная програму успешно скомпилируется и буде выполняться, но при компиляции компилятор выдаст предупреждение:


eugene@Eugene:/workspace/java$ java Program.java
Program.java:18: warning: [deprecation] makeSound() in Cat has been deprecated
        barsik.makeSound();
              ^
1 warning
Мяу

@SuppressWarnings

Анотація @SuppressWarnings отключает визначенийные предупреждения компилятора для аннотированного елемента. Наприклад:


import java.util.ArrayList;
import java.util.List;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        methodWithRawType();
    }

    public static void methodWithRawType() {
        List names = new ArrayList();
        names.add("Tom");
        System.out.println(names);
    }
}

Эта програма успешно скомпилируется и виконується, однак при компиляции мы получим предупреждение:


eugene@Eugene:/workspace/java$ java Program.java
Program.java:13: warning: [unchecked] unchecked call to add(E) as a member of the raw type List
        names.add("Tom");
                 ^
  where E is a type-variable:
    E extends Object declared in interface List
1 warning
[Tom]

Проблема в том, что нам треба указать тип хранимых данных для ArrayList, напідобие следующего:

List<String> names = new ArrayList<String>();

Однак мы також можемо пріменіть анотацію @SuppressWarnings для отключения підобного (и других) предупреждений. Для цього після названия анотації в скобках треба передать рядок с типом предупреждения. В примере выше мы тип предупреждения - "unchecked", соответственно нам треба написать:


@SuppressWarnings("unchecked")
public static void methodWithRawType() {

    List names = new ArrayList();
    names.add("Tom");
    System.out.println(names);
}

@FunctionalInterface

Анотація @FunctionalInterface вказує, что інтерфейс призначений для использования в качестве функционального інтерфейса (т.е. он повинен иметь ровно один абстрактный метод). Компилятор проверяет це условие.


@FunctionalInterface
interface MyComparator {
    int compare(String a, String b);
}
