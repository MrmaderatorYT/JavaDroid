Введение в аннотации

Останнє оновлення: 01.11.2025

Аннотации в языке программирования Java представляють специальную форму метаданных (данных о данных), яку можна добавлять в исходный код. Они предоставляют дополнительную информацию о программе, которая может використовуватися компилятором, инструментами сборки, фреймворками або во час виконання. Аннотации дозволяє отделить конфигурацию и метадані от основной бизнес-логики. Они делают код болїї чистым, декларативным и легким для поддержки, перенося большую часть "магии" настройки на фреймворки и инструменты. Можна сказать, что аннотации є основой многих современных Java-технологий, в частности, веб-фреймворков, фреймворков для тестирования, валидации и т.д.

Сами по себе аннотации не выполняют нияких действий. Они лишь "маркируют" классы, методы, поля и другие элементы кода, а вже другой код (обработчик аннотаций) ищет эти маркеры и выполняет на их основе деяку определенную логику.

Синтаксически аннотация начинается с символа @, за которым йде її имя, наприклад, @Override.

В мові Java есть ряд встроенных аннотаций, которые використовуються для стандартных задач. Рассмотрим некоторые наиболїї используемые:

  • @Override: указывает компилятору, что текущий метод переопределяет метод из суперкласса.

  • @Deprecated: эта аннотация помечает элемент (класс, метод, поле) як устаревший

  • @SuppressWarnings: отключает определенные предупреждения компилятора для аннотированного элемента.

  • @FunctionalInterface: указывает, что інтерфейс призначений для использования в качестве функционального інтерфейса (т.е. он повинен иметь ровно один абстрактный метод).

Застосування аннотаций

Аннотации оголошенийий можуть застосовуватися ко многим компонентам. И в зависимости от места применения аннотации делятся на 2 группы:

  • аннотации оголошенийий (declaration annotation)

  • аннотации использования типов (type uses annotation)

Аннотации оголошенийий

Аннотации оголошенийий містять деяку информацию об оголошуємоом елементі и можуть застосовуватися к наступним оголошенням:

  • К классам (включая перечисления) и інтерфейсам (в том числе к інтерфейсам аннотаций)

  • К методам

  • К конструкторам

  • К полям (в том числе к константам перечислений и полям классов record).

  • К локальным переменным (в том числе к оголошенийным в конструкциях for и try-with-resources).

  • К параметрам методів

  • К параметрам типов

  • К пакетам и модулям

Для классов, інтерфейсов и модулей аннотации размещаются перед ключевым словом class, interface або module и любым модификаторам (при его наличии):

@MyAnnotation public class Person { }

Для переменных, параметров, а також поле классов-record аннотации размещаются перед типом:


@MyAnnotation String name = "Tom";
public User getUser(@MyAnnotation int userId)
record Person(@MyAnnotation String name) {}

Також аннотация помещается перед параметром типу в обобщенном классе або методе:


class Person<@MyAnnotation T> { }

Аннотации использования типов

Аннотации использования типов можуть появляться в следующих местах:

  • С аргументами обобщенного типу:

    Person<@MyAnnotation String>

  • Перед скобками массива:

    
String[] @MyAnnotation [] data // Застосовується к String[]
String @MyAnnotation [][] data // Застосовується к String[][]

    От эих ситуаций треба отличать следующую:

    @MyAnnotation String[][] data

    Здесь аннотация застосовується к String

  • С суперклассами и реализованными інтерфейсами:

    class Employee extends @MyAnnotation Person.

  • С вызовами конструктора

     new @MyAnnotation Person();

  • С вложенными типуми

     Map.@MyAnnotation Entry

  • С приведениями типов и проверками instanceof:

    
(@MyAnnotation String) text
if (text instanceof @MyAnnotation String)

    Аннотации призначенийы лише для использования внешними инструментами. Они не влияют на поведение приведения типов або проверки instanceof.

  • С указанием генерируемых исключений

    void print() throws @MyAnnotation Exception.

  • С подстановочными типуми и ограничениями типов

    
List<@MyAnnotation ? extends Person>
List<? extends @MyAnnotation Person>

  • С ссылками на методы и конструктор:

    @MyAnnotation Person::getName

Аннотации можна помещать до або после других модификаторов, таких як static и public. Обычно аннотации использования типу размещаются после других модификаторов, а аннотации оголошенийий - перед другими модификаторами. Например:


private @MyAnnotation String name;   // Аннотация использования типу
@MyAnnotation private int age;      // Аннотация змінної / поля класса

Встроенные (стандартные) аннотации Java

@Override

Аннотация @Override указывает компилятору, что текущий метод переопределяет метод из суперкласса. Если метод с такой сигнатурой в суперклассе не найден, компилятор выдаст ошибку. Це помогает избежать опечаток.

Конечно, при переопределении метода суперкласса в производном классе нам необязательно використовувати данную аннотацию. Но розглянемо следующую программу:


class Animal {
    void makeSound() {
        System.out.println("Некий звук");
    }
}
class Cat extends Animal {
    
    void makeSounds() {
        System.out.println("Мяу");
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Cat barsik = new Cat();
        barsik.makeSound(); // Некий звук
    }
}

Класс Animal представляє животное и определяет метод makeSound() для имитации звука животного. Класс Cat переопределяет цей метод. А в методе main() вызываем цей метод. Но посмотрим на консольный вывод программы:

Некий звук

Результат, очевидно, не тот, який ожидался. А почему? А потому что я допустил опечатку: вместо "makeSound" в классе Cat определил метод "makeSounds". В итоге я по сути определил новый метод, який ничего не переопределяет. Но программа бува успешно скомпилирована и успешно выполнила все свои действия! И в том числе для предупреждения подобных ситуаций призначенийа аннотация @Override. Применим її:


class Animal {
    void makeSound() {
        System.out.println("Некий звук");
    }
}
class Cat extends Animal {
    
    @Override
    void makeSounds() {
        System.out.println("Мяу");
    }
}

В производном классе Cat попрежнему метод с опечаткой: "makeSounds" вместо "makeSound". Однак? применив к методу аннотацию @Override, мы указываем, что в суперклассе повинен быть метод с подобным именем, и мы его хочемо переопределить. Но поскольку в суперклассе нет метода "makeSounds", то при компиляции теперь мы получим ошибку:


Program.java:8: error: method does not override or implement a method from a supertype
    @Override
    ^
1 error
error: compilation failed

Таким чином, мы увидим ошибку и сможемо своевременно її исправить, недопустив некорректного виконання програми:


class Cat extends Animal {
    
    @Override
    void makeSound() {
        System.out.println("Мяу");
    }
}

@Deprecated

Аннотация @Deprecated помечает элемент (класс, метод, поле) як устаревший, что может быть полезно, якщо после обновления API класса або пакета, мы хочемо, что якие-то компоненты больше не использовались. Компилятор буде выдавать предупреждение при каждой попытке використовувати такой компонент. Например:


class Cat{
    
    @Deprecated
    void makeSound() {
        System.out.println("Мяу");
    }

    void say() {
        System.out.println("Мяу");
    }
}

class Program{
 
    public static void main(String[] args) throws Exception{
          
        Cat barsik = new Cat();
        barsik.makeSound();
    }
}

Здесь в классе Cat определен метод makeSound(), який за допомогою аннотации @Deprecated оголошений як устаревший (допустим, мы хочемо, чтобы в будущем разработчики использовали метод say(), який дублирует функциональность метода makeSound()). Данная программу успешно скомпилируется и буде выполняться, но при компиляции компилятор выдаст предупреждение:


eugene@Eugene:/workspace/java$ java Program.java
Program.java:18: warning: [deprecation] makeSound() in Cat has been deprecated
        barsik.makeSound();
              ^
1 warning
Мяу

@SuppressWarnings

Аннотация @SuppressWarnings отключает определенные предупреждения компилятора для аннотированного элемента. Например:


import java.util.ArrayList;
import java.util.List;

class Program{
 
    public static void main(String[] args) throws Exception{
          
        methodWithRawType();
    }

    public static void methodWithRawType() {
        List names = new ArrayList();
        names.add("Tom");
        System.out.println(names);
    }
}

Эта программа успешно скомпилируется и виконується, однак при компиляции мы получим предупреждение:


eugene@Eugene:/workspace/java$ java Program.java
Program.java:13: warning: [unchecked] unchecked call to add(E) as a member of the raw type List
        names.add("Tom");
                 ^
  where E is a type-variable:
    E extends Object declared in interface List
1 warning
[Tom]

Проблема в том, что нам треба указать тип хранимых данных для ArrayList, наподобие следующего:

List<String> names = new ArrayList<String>();

Однак мы також можемо применить аннотацию @SuppressWarnings для отключения подобного (и других) предупреждений. Для цього после названия аннотации в скобках треба передать строку с типом предупреждения. В примере выше мы тип предупреждения - "unchecked", соответственно нам треба написать:


@SuppressWarnings("unchecked")
public static void methodWithRawType() {

    List names = new ArrayList();
    names.add("Tom");
    System.out.println(names);
}

@FunctionalInterface

Аннотация @FunctionalInterface указывает, что інтерфейс призначений для использования в качестве функционального інтерфейса (т.е. он повинен иметь ровно один абстрактный метод). Компилятор проверяет це условие.


@FunctionalInterface
interface MyComparator {
    int compare(String a, String b);
}
