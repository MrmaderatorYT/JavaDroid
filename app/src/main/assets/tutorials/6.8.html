Читання та запис текстових файлів

Останнє оновлення: 26.04.2018

Хотя за допомогою ранїї рассмотренных класов можна записывать текст в файли, однак они призначенийы прежде всего для работы с бинарными потіками данных, и их возможнастей для полноценной работы с текстовыми файлуми недостатньо. И для цей цели служат совсем другие класы, которые є наследниками абстрактных класов Reader и Writer.

Запис файлів. Клас FileWriter

Клас FileWriter є производным от класа Writer. Он використовується для записи текстовых файлів.

Щоб створити об'єкт FileWriter, можна використовувати один из следующих конструкторов:


FileWriter(File file)
FileWriter(File file, boolean append)
FileWriter(FileDescriptor fd)
FileWriter(String fileName)
FileWriter(String fileName, boolean append)

Так, в конструктор передаєся або путь к файлу в виде рядки, або об'єкт File, який ссылается на конкретный текстовый файл. Параметр append вказує, повинні ли дані дозаписываться в конец файлу (якщо параметр равен true), або файл повинен перезаписываться.

Запишем в файл якой-нибудь текст:


import java.io.*;

public class Program {

    public static void main(String[] args) {
       
        try(FileWriter writer = new FileWriter("notes3.txt", false))
        {
           // запис всей рядки
            String text = "Hello Gold!";
            writer.write(text);
            // запис по символам
            writer.append('\n');
            writer.append('E');
			
			writer.flush();
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        } 
    } 
}

В конструкторе использовался параметр append со значенням false - то есть файл буде перезаписываться. Затем за допомогою методів, визначенийных в базовом класе Writer производится запис данных.

Читання файлів. Клас FileReader

Клас FileReader найдеся от абстрактного класа Reader и надає функциональность для читання текстовых файлів.

Для створенняя об'єкта FileReader мы можемо використовувати один из его конструкторов:


FileReader(String fileName) 
FileReader(File file)
FileReader(FileDescriptor fd)

А використовуючи методи, визначенийные в базом класе Reader, произвести читання файлу:


import java.io.*;

public class Program {

    public static void main(String[] args) {
       
        try(FileReader reader = new FileReader("notes3.txt"))
        {
           // читаємо посимвольно
            int c;
            while((c=reader.read())!=-1){
                
                System.out.print((char)c);
            } 
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        }   
    } 
}

Також мы можемо считывать в промежуточный буфер из масива символів:


import java.io.*;
import java.util.Arrays;

public class Program {
 
    public static void main(String[] args) {
         
        try(FileReader reader = new FileReader("notes3.txt"))
        {
			char[] buf = new char[256];
            int c;
            while((c = reader.read(buf))>0){
                
				if(c < 256){
					buf = Arrays.copyOf(buf, c);
				}
                System.out.print(buf);
            } 
        }
        catch(IOException ex){
            
            System.out.println(ex.getMessage());
        }		
    }	  
}

У даному випадку считываем післядовательно символи из файлу в масив из 256 символів, поки не дойдем до конца файлу в цем случае метод read возвратит число -1.

Поскольку считанная порция файлу может быть меньше 256 символів (наприклад, в файле всего 73 символа), и якщо кількість считанных данных меньше размера буфера (256), то выполняем копирование масива за допомогою методу Arrays.copy. То есть фактически обрезаем масив buf, оставляя в нем лише те символи, которые считаны из файлу.
