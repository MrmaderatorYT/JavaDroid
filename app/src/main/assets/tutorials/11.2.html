Залежні модулі

Останнє оновлення: 02.11.2025

Одни модулі можуть зависить от других, наприклад, як-то використовувати их. Наприклад, возьмем проєкт из прошлой темы с простейшим модулем:

У даному випадку модуль назвается "demo" и располагается в одноименной папке. В цей же папке лежит файл визначенийия модуля - module-info.java. А весь код програми располагается в папке "com//hello" в файле "Hello.java".

В файле Hello.java визначимо вивід сообщения не на консоль, а через графическое окно, которое надаєся инфраструктурой Swing:


package com..hello;

import javax.swing.JOptionPane;

public class Hello{
	
	public static void main(String[] args){
		//System.out.println("Hello !");
		JOptionPane.showMessageDialog(null, "Hello Demo Module!");
	}
}

Для вивіда сообщения використовується статический метод showMessageDialog() класа JOptionPane, який расположен в пакеті javax.swing.JOptionPane. Щоб використовувати данный клас, он импортируется в начале файлу. Однак поскольку мы використовуємо данный клас в отдельном модуле, то просто импортировать клас або целый пакет недостатньо. Надо указать нашему модулю, чтобы он использовал модуль, где визначений данный клас.

Как узнать, в яком модуле расположен тот або иной клас? Если речь идет о встроенных класах, то для цього треба смотреть документацию, причем відразу по Java (JDK) 9+. В начале описания класа, як правило, вказуєся, к якому модулю он принадлежит.

Итак, документация нам повідомляє, что клас JOptionPane расположен в модуле java.desktop. Теперь укажем нашему модулю, что нам треба використовувати модуль java.desktop. Для цього перейдем к визначенийию модуля в файле module-info.java в корневой папке модуля и изменим его наступним чином:


module demo {
    requires java.desktop;
}

Після оператора requires вказуєся название модуля, от которого зависит наш модуль. Соответственно якщо наш модуль використовує декілька других модулів, то також за допомогою оператора require можна указать все використовуємоые модулі.

Далїї скомпилируем модуль и запустимо его на виконання.


eugene@Eugene:/java$ javac demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java  --module-path demo --module demo/com..hello.Hello
