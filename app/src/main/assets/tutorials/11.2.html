Залежні модулі

Останнє оновлення: 02.11.2025

Одни модули можуть зависить от других, наприклад, як-то використовувати их. Наприклад, возьмем проект из прошлой темы с простейшим модулем:

В данном случае модуль назвается "demo" и располагается в одноименной папке. В цей же папке лежит файл определения модуля - module-info.java. А весь код программы располагается в папке "com//hello" в файле "Hello.java".

В файле Hello.java определим вывод сообщения не на консоль, а через графическое окно, которое надаєся инфраструктурой Swing:


package com..hello;

import javax.swing.JOptionPane;

public class Hello{
	
	public static void main(String[] args){
		//System.out.println("Hello !");
		JOptionPane.showMessageDialog(null, "Hello Demo Module!");
	}
}

Для вывода сообщения використовується статический метод showMessageDialog() класса JOptionPane, який расположен в пакеті javax.swing.JOptionPane. Щоб використовувати данный класс, он импортируется в начале файла. Однак поскольку мы используем данный класс в отдельном модуле, то просто импортировать класс або целый пакет недостатньо. Надо указать нашему модулю, чтобы он использовал модуль, где определен данный класс.

Как узнать, в яком модуле расположен тот або иной класс? Если речь идет о встроенных классах, то для цього треба смотреть документацию, причем відразу по Java (JDK) 9+. В начале описания класса, як правило, указывается, к якому модулю он принадлежит.

Итак, документация нам повідомляє, что класс JOptionPane расположен в модуле java.desktop. Теперь укажем нашему модулю, что нам треба використовувати модуль java.desktop. Для цього перейдем к определению модуля в файле module-info.java в корневой папке модуля и изменим его наступним чином:


module demo {
    requires java.desktop;
}

После оператора requires указывается название модуля, от которого зависит наш модуль. Соответственно якщо наш модуль использует декілька других модулей, то також за допомогою оператора require можна указать все используемые модули.

Далїї скомпилируем модуль и запустим его на выполнение.


eugene@Eugene:/java$ javac demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java  --module-path demo --module demo/com..hello.Hello
