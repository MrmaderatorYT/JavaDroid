Класс Throwable

Базовым классом для всех исключений в мові Java є класс Throwable. Тому, якщо мы хочемо обработать в блоке catch все возможные исключения, то мы можемо використовувати цей тип.

Данный класс определяет ряд базовых методів, отмечу основные из них:

  • String getMessage(): повертає строку подробного сообщения об исключении

  • StackTraceElement[] getStackTrace(): повертає информацию о трассировке стека, выведенной функцией printStackTrace()

  • void printStackTrace(): выводит трассировку стека в стандартный поток ошибок

Используем эти методы для получения информации об исключении при его обработке:


class Program{
     
	public static void main(String[] args) {
         
        try{
            // для эмуляции ошибки делим число на 0
            System.out.println(5 / 0);
        }
        catch(Throwable ex){

            // выводим сообщение об ошибке
            System.out.println("Error: " + ex.getMessage());
            // выводим трасировку стека
            ex.printStackTrace();
        }
    }
}

Здесь для генерации исключения в блоке try делим число на 0. А в блоке catch оголошуємо переменную ex, которая представляє тип Throwable и выводим на консоль по отдельности сообщение об ошибке и трассировку стека (которая також включает сообщение об ошибке). В итоге мы получим следующий консольный вывод:


Error: / by zero
java.lang.ArithmeticException: / by zero
	at Program.main(Program.java:7)

Трассировка стека дозволяє болїї детально узнать, якой реально тип ошибки, где произвошла ошибка. Так, из выведенной информации мы видим, что в реальности исключение представляє тип java.lang.ArithmeticException (то есть тип исключений для арифметических операций). А сама ошибка возникла на 7 строке файла Program.java в методе Program.main

Классификация ошибок

От класса Throwable наследуются два других класса: Error и Exception. Все остальные классы є производными от цих двух классов.

Класс Error описує внутренние ошибки в исполняющей среде Java. Программист имїїт очень ограниченные возможнасти для обработки подобных ошибок. При возникновении такой ошибки мало что можна сделать, кроме як уведомить пользователя и попытаться корректно завершить программу. Такие ситуации встречаются довольно редко.

Собственно исключения наследуются от класса Exception. Среди цих исключений йде выделить класс RuntimeException. Все классы исключений, которые унаследованы от классов Error и RuntimeException ще називаються непроверяемыми исключениями (unchecked exceptions) - компилятор не проверяет факт обработки таких исключений. Такие исключения нередко є следствием ошибок разработчика, наприклад, неверное преобразование типов або выход за пределы массива.

Некоторые из классов непроверяемых исключений:

  • ArithmeticException: исключение, возникающїї при делении на ноль

  • IndexOutOfBoundException: индекс вне границ массива

  • IllegalArgumentException: использование неверного аргумента при вызове метода

  • NullPointerException: использование пустой ссылки

  • NumberFormatException: ошибка преобразования строки в число

Все остальные классы, образованные от класса Exception, називаються проверяемыми исключениями (checked exceptions).

Некоторые из классов проверяемых исключений:

  • CloneNotSupportedException: класс, для об'єкта которого вызывается клонирование, не реализует інтерфейс Cloneable

  • InterruptedException: поток прерван другим потоком

  • ClassNotFoundException: невозможна найти класс

Обработка нескольких исключений

В одной и той же программе на одном и том же участке кода может возникать декілька разных типов исключений. Обрабатывая исключения типу Throwable або Exception, мы можемо покрыть все эти типы исключений. Но Java також дозволяє разграничить обработку различных видов исключений, добавив дополнительные блоки catch для каждого обрабатываемого типу исключения:


class Program{
     
	public static void main(String[] args) {
         
        printNumber(0);     // По индексу 0 число 1
        printNumber(10);    // Выход за пределы массива
        printNumber(3);     // Ошибка преобразования из строки в число
    }

    static void printNumber(int index){

        String[] data = {"1", "a", "2", "b", "3", "c"};

        try{
            // пытаемся преобразовать значение массива по индексу в число
            int result =Integer.parseInt(data[index]);
            System.out.printf("По индексу %d число %d\n", index, result);
        }
        catch(ArrayIndexOutOfBoundsException _){
                    
            System.out.println("Выход за пределы массива");
        }
        catch(NumberFormatException _){
                    
            System.out.println("Ошибка преобразования из строки в число");
        }
    }
}

В данном случае основные действия разворачиваются в методе printNumber(). Цет метод отримує зовні индекс и пытается преобразовать значение из массива по цему индексу в число. И в цем случае у нас может быть декілька вариантов развития ситуации:

  1. По переданному индексу в массиве располагается число (лише в виде строки), тогда преобразование завершуєся успешно

  2. Переданный индекс находится вне допустимого диапазона, тогда мы столкнемся с исключением типу ArrayIndexOutOfBoundsException

  3. Переданный индекс действителен, но значение по нему невозможна преобразовать в число. Тогда мы столкнемся с исключением типу NumberFormatException

Для отслеживания последних двух ситуаций в конструкции try..catch определено два блока catch:


catch(ArrayIndexOutOfBoundsException _){
                    
    System.out.println("Выход за пределы массива");
}
catch(NumberFormatException _){
                    
    System.out.println("Ошибка преобразования из строки в число");
}

Если у нас возникает исключение определенного типу, то оно переходит к соответствующему блоку catch.

Подобная стратегия дозволяє разграничить обработку различных типов исключений. Однак, что, якщо у нас в коде может возникнуть большїї количество исключений - не создавать же на каждый тип исключений из стандартной библиотеки Java свой блок catch! В цем случае мы можемо определить ряд блоков catch для тех типов, для которых мы хочемо определить специальную обработку. А для всех остальных типов исключений определить блок catch с обработкой болїї общего типу як Throwable або Exception. Например:


class Program{
     
	public static void main(String[] args) {
         
        printNumber(0);     // По индексу 0 число 1
        printNumber(10);    // Выход за пределы массива
        printNumber(3);     // For input string: "b"
    }

    static void printNumber(int index){

        String[] data = {"1", "a", "2", "b", "3", "c"};

        try{
            // пытаемся преобразовать значение массива по индексу в число
            int result =Integer.parseInt(data[index]);
            System.out.printf("По индексу %d число %d\n", index, result);
        }
        catch(ArrayIndexOutOfBoundsException _){  // обрабатываем выход за границы массива
                    
            System.out.println("Выход за пределы массива");
        }
        catch(Exception ex){        // обрабатываем все остальные типы исключений
                    
            System.out.println(ex.getMessage());    // выводим сообщение об исключении
            ex.printStackTrace();
        }
    }
}

В данном случае отдельно обрабатываем исключение типу ArrayIndexOutOfBoundsException, то есть выход за границы массива. А для других типов обрабатываем исключения типу Exception

При цем варто учитывать, что блоки catch для обработки болїї конкретных (производных) типов (як ArrayIndexOutOfBoundsException) повинні располагаться сначала, а блоки catch для обработки болїї общих (базовых) типов (як Exception або Throwable) - в конце, як в примере выше.
