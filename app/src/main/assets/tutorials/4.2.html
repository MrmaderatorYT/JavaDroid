Класи винятків

Останнє оновлення: 05.10.2025

Клас Throwable

Базовым класом для всех винятків в мові Java є клас Throwable. Тому, якщо мы хочемо обпрацювати в блоке catch все возможные винятки, то мы можемо використовувати цей тип.

Данный клас визначає ряд базовых методів, отмечу основные из них:

  • String getMessage(): повертає рядок підробного сообщения об исключении

  • StackTraceElement[] getStackTrace(): повертає информацию о трассировке стека, выведенной функцією printStackTrace()

  • void printStackTrace(): виводить трассировку стека в стандартный потік ошибок

Використовуємо эти методи для получения информации об исключении при его обработке:


class Program{
     
	public static void main(String[] args) {
         
        try{
            // для эмуляции ошибки делим число на 0
            System.out.println(5 / 0);
        }
        catch(Throwable ex){

            // вивідим сообщение об ошибке
            System.out.println("Error: " + ex.getMessage());
            // вивідим трасировку стека
            ex.printStackTrace();
        }
    }
}

Здесь для генерации винятки в блоке try делим число на 0. А в блоке catch оголошуємо змінну ex, которая представляє тип Throwable и вивідим на консоль по отдельности сообщение об ошибке и трассировку стека (которая також включает сообщение об ошибке). В итоге мы получим следующий консольный вивід:


Error: / by zero
java.lang.ArithmeticException: / by zero
	at Program.main(Program.java:7)

Трассировка стека дозволяє болїї детально узнать, якой реально тип ошибки, где произвошла ошибка. Так, из выведенной информации мы видим, что в реальности виняток представляє тип java.lang.ArithmeticException (то есть тип винятків для арифметических операцій). А сама ошибка возникла на 7 рядківе файлу Program.java в методі Program.main

Класификация ошибок

От класа Throwable наследуются два других класа: Error и Exception. Все остальные класы є производными от цих двух класов.

Клас Error описує внутренние ошибки в исполняющей среде Java. Програмист имїїт дуже ограниченные возможнасти для обработки підобных ошибок. При возникновении такой ошибки мало что можна сделать, кроме як уведомить пользователя и попытаться корректно завершить програму. Такие ситуации встречаются досить редко.

Собственно винятки наследуются от класа Exception. Среди цих винятків йде выделить клас RuntimeException. Все класы винятків, которые унаследованы от класов Error и RuntimeException ще називаються непроверяемыми виняткими (unchecked exceptions) - компилятор не проверяет факт обработки таких винятків. Такие винятки нередко є следствием ошибок разработчика, наприклад, неверное преобразование типів або выход за пределы масива.

Некоторые из класов непроверяемых винятків:

  • ArithmeticException: виняток, возникающїї при делении на ноль

  • IndexOutOfBoundException: индекс вне границ масива

  • IllegalArgumentException: використання неверного аргумента при вызове методу

  • NullPointerException: використання порожній ссылки

  • NumberFormatException: ошибка преобразования рядки в число

Все остальные класы, образованные от класа Exception, називаються проверяемыми виняткими (checked exceptions).

Некоторые из класов проверяемых винятків:

  • CloneNotSupportedException: клас, для об'єкта которого вызывается клонирование, не реалізує інтерфейс Cloneable

  • InterruptedException: потік прерван другим потіком

  • ClassNotFoundException: невозможна найти клас

Обробка нескольких винятків

В одной и той же програме на одном и том же участке коду может возникать декілька разных типів винятків. Обрабатывая винятки типу Throwable або Exception, мы можемо покрыть все эти типи винятків. Но Java також дозволяє разграничить обработку различных видов винятків, добавив дополнительные блоки catch для каждого обробляємоого типу винятки:


class Program{
     
	public static void main(String[] args) {
         
        printNumber(0);     // По индексу 0 число 1
        printNumber(10);    // Выход за пределы масива
        printNumber(3);     // Ошибка преобразования из рядки в число
    }

    static void printNumber(int index){

        String[] data = {"1", "a", "2", "b", "3", "c"};

        try{
            // пытаемся преобразовать значення масива по индексу в число
            int result =Integer.parseInt(data[index]);
            System.out.printf("По индексу %d число %d\n", index, result);
        }
        catch(ArrayIndexOutOfBoundsException _){
                    
            System.out.println("Выход за пределы масива");
        }
        catch(NumberFormatException _){
                    
            System.out.println("Ошибка преобразования из рядки в число");
        }
    }
}

У даному випадку основные действия разворачиваются в методі printNumber(). Цет метод отримує зовні индекс и пытается преобразовать значення из масива по цему индексу в число. И в цем случае у нас может быть декілька вариантов развития ситуации:

  1. По переданному индексу в масиве располагается число (лише в виде рядки), тогда преобразование завершуєся успешно

  2. Переданий індекс знаходиться за межами допустимого діапазону, тоді ми зіткнемося з винятком типу ArrayIndexOutOfBoundsException

  3. Переданий індекс дійсний, але значення за ним неможливо перетворити на число. Тоді ми зіткнемося з винятком типу NumberFormatException

Для отслеживания післядних двух ситуаций в конструкции try..catch визначенийо два блока catch:


catch(ArrayIndexOutOfBoundsException _){
                    
    System.out.println("Выход за пределы масива");
}
catch(NumberFormatException _){
                    
    System.out.println("Ошибка преобразования из рядки в число");
}

Если у нас возникает виняток визначенийного типу, то оно переходит к соответствующему блоку catch.

Підобная стратегия дозволяє разграничить обработку различных типів винятків. Однак, что, якщо у нас в коді может возникнуть большїї кількість винятків - не создавать же на каждый тип винятків из стандартной бібліотеки Java свой блок catch! В цем случае мы можемо визначити ряд блоков catch для тех типів, для которых мы хочемо визначити специальную обработку. А для всех остальных типів винятків визначити блок catch с обработкой болїї общего типу як Throwable або Exception. Наприклад:


class Program{
     
	public static void main(String[] args) {
         
        printNumber(0);     // По индексу 0 число 1
        printNumber(10);    // Выход за пределы масива
        printNumber(3);     // For input string: "b"
    }

    static void printNumber(int index){

        String[] data = {"1", "a", "2", "b", "3", "c"};

        try{
            // пытаемся преобразовать значення масива по индексу в число
            int result =Integer.parseInt(data[index]);
            System.out.printf("По индексу %d число %d\n", index, result);
        }
        catch(ArrayIndexOutOfBoundsException _){  // обробляємо выход за границы масива
                    
            System.out.println("Выход за пределы масива");
        }
        catch(Exception ex){        // обробляємо все остальные типи винятків
                    
            System.out.println(ex.getMessage());    // вивідим сообщение об исключении
            ex.printStackTrace();
        }
    }
}

У даному випадку отдельно обробляємо виняток типу ArrayIndexOutOfBoundsException, то есть выход за границы масива. А для других типів обробляємо винятки типу Exception

При цем варто учитывать, что блоки catch для обработки болїї конкретных (производных) типів (як ArrayIndexOutOfBoundsException) повинні располагаться сначала, а блоки catch для обработки болїї общих (базовых) типів (як Exception або Throwable) - в конце, як в примере выше.
