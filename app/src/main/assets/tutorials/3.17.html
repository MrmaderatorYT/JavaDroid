Обмеження узагальнень

Останнє оновлення: 09.10.2025

Когда мы указываем универсальный параметр у обобщений, то по умолчанию он может представлять любой тип. Однак иногда необхідно, чтобы параметр соответствовал лише некоторому ограниченному набору типів. В цем случае застосовуються ограничения обобщений (type bound) - они дозволяють указать базовый клас, которому повинен соответствовать параметр.

Наприклад, ми маємо наступний клас Message, який представляє деяке повідомлення, та його похідні класи, які представляютььь email-повідомлення та sms-повідомлення:


// Клас сообщения
class Message {
    private String text;

    String getText() { return text;  }

    Message(String text) { this.text = text; }
}
// клас сообщения по email
class EmailMessage extends Message {

    private String address;  // адрес электронной почты
    String getAddress(){ return address; }

    EmailMessage(String text, String address) {

        super(text);
        this.address = address;
    }
}
// клас сообщения по sms
class SmsMessage extends Message {

    private String number;  // номер телефона
    String getNumber(){ return number; }

    SmsMessage(String text, String number) {
        super(text);
        this.number = number;
    }
}

І, припустимо, ми хочемо визначити функціонал для надсилання повідомлень. І цього створимо наступний клас Messenger:


class Messenger {

    private Message message;
    Message getMessage() { return this.message; }

    Messenger(Message message) { this.message = message;  }

    void send() {
        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}

Клас Messenger через конструктор приймає сообщение и через метод send() условно надсилає. За допомогою методу getMessage() мы можемо отримати отправленное сообщение. И чтобы клас Messenger був предельно универсален, в качестве типу сообщений використовуємо базовый тип Message.

Використовуємо певні класи:


class Program {

    public static void main(String[] args) {
        //  Створюємо сообщения
        Message message1 = new EmailMessage("Привет, ты спишь?", "someaddress@hmail.com");
        Message message2 = new SmsMessage("Не, ты скажи, ты спишь?", "+71234567890");

        // Надсилаємо повідомлення
        Messenger simpleMessenger = new Messenger(message1);
        simpleMessenger.send();

        simpleMessenger = new Messenger(message2);
        simpleMessenger.send();
    }
}

B все ніби чудово працює, і жодних проблем немає, оскільки клас Messenger приймає об'єкт Message і відповідно також і об'єкти похідних класів. Але підивимося, з якими проблемами ми можемоо зіткнутися.

Перетворення типів

Однак при спробі отримати з об'єкта Messenger надіслане повідомлення ми можемоо зіткнутися з необхідністю перетворення типів:


class Program {

    public static void main(String[] args) {

        Message message = new EmailMessage("Привет, ты спишь?", "someaddress@hmail.com");

        Messenger simpleMessenger = new Messenger(message);
        simpleMessenger.send(); // Надсилаєся сообщение: Привет, ты спишь?

        // отримуємо отправленное сообщение
        EmailMessage email = (EmailMessage) simpleMessenger.getMessage();
        System.out.println("Address: " + email.getAddress());  // Address: someaddress@hmail.com
    }
}

Здесь мы извлекаем за допомогою вызова simpleMessenger.getMessage() отправленное сообщение и преобразуем в EmailMessage, чтобы иметь обращаться к методум, визначенийным непосредственно в класе EmailMessage.

Типобезпека

Друга проблема, яка тут можлива – це проблема типобезпеки, якщо ми точно не знаємо, який тип приховує об'єкт Message та захочемо перетворити цей об'єкт до несумісного типу:


class Program {

    public static void main(String[] args) {

        Message message = new EmailMessage("Привет, ты спишь?", "someaddress@hmail.com");
        Messenger simpleMessenger = new Messenger(message);

        // отримуємо сообщение
        SmsMessage sms = (SmsMessage) simpleMessenger.getMessage();  // Сюрпрайз - java.lang.ClassCastException!
        System.out.println("Phone number: " + sms.getNumber());
    }
}

Здесь при попытке преобразовать к типу SmsMessage мы столкнемся с ошибкой, поскольку преобразуемый об'єкт представляє тип EmailMessage. И проблема усугубляется тем, что эту ошибку мы можемо отримати лише в процесе виконання. А компиляция пройдет успешно.

Використання узагальнень

На перший погляд, щоб уникнути перетворень та проблем із типобезпекою, ми могли б застосувати узагальнення:


class Messenger<T> {

    private T message;
    T getMessage() { return this.message; }

    Messenger(T message) { this.message = message;  }

    void send() {
        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}

Теперь в клас Messenger мы можемо передать сообщения любого типу, и об'єкт Messenger буде строго використовувати цей тип. Преобразования не потребуются, а ошибки можна буде отловить на этапе компиляции. Но теперь мы сталкиваемся с другой проблемой - универсальный параметр T підразумевает любой тип. Но не любой тип имїїт метод getText(). Соответственно метод getText() для об'єкта типу T не визначений и мы не можемо цей метод використовувати. Болїї того для об'єкта T по умолчанию достуны лише методи типу Object.

Визначення обмежень узагальнень

Таким чином, возникает проблема: треба избежать преобразований типів и соответственно використовувати обобщения, а с другой стороны, необхідно обращаться всередині методу к функционалу класа Message. И ограничения обобщений дозволяють решить эту проблему.

Для установки ограничения після универсального параметра ставится слово extends, після которого вказуєся тип ограничения:

<T extends тип_ограничения>

Обмеження узагальнень у типух

Обмеження узагальнень можна застосовувати лише на рівні типів:

class ім'я_класа<T extends тип_ограничения>

Як приклад змінимо клас месенджера:


class Messenger<T extends Message> {

    private T message;
    T getMessage() { return this.message; }

    Messenger(T message) { this.message = message;  }

    void send() {
        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}

Выражение <T extends Message> в визначенийии класа говорит, что через универсальный параметр T будуть передаваться об'єкты класа Message и производных класов. Благодаря цему компилятор буде знать, что T буде иметь функционал класа Message, и соответственно мы сможемо без проблем обратиться к методум класа Message всередині класа Messenger.

Застосуємо клас для надсилання повідомлень:



class Program {

    public static void main(String[] args) {

        EmailMessage email = new EmailMessage("Привет, ты спишь?", "someaddress@hmail.com");
        Messenger<EmailMessage> mailClient = new Messenger<EmailMessage>(email);
        mailClient.send();  // Надсилаєся сообщение: Привет, ты спишь?

        SmsMessage sms = new SmsMessage("Hello World", "+71234567890");
        Messenger<SmsMessage> phone = new Messenger<SmsMessage>(sms);
        phone.send();   // Надсилаєся сообщение: Hello World
    }
}

Слід зазначити, що з визначенні змінної класу ми можемоо скоротити її визначення:

var mailClient = new Messenger<EmailMessage>(email);

Або так:

Messenger<EmailMessage> mailClient = new Messenger<>(email);

Якщо ми спробуємо отримати об'єкт несумісного типу:


EmailMessage email = new EmailMessage("Привет", "someaddress@hmail.com");
var mailClient = new Messenger<EmailMessage>(email);

SmsMessage sms = mailClient.getMessage();  // Ошибка компиляции

ми отримаємо помилку на етапі компіляції.

Використання кількох універсальних параметрів

Якщо клас використовує кілька універсальних параметрів, то послідовно можна встановити обмеження до кожного з них:


class Messenger<T extends Message, P extends Person> {

    void sendMessage(P sender, P receiver, T message)
    {
        System.out.println("Відправник: " + sender.getName());
        System.out.println("Отримувач: " + receiver.getName());
        System.out.println("Сообщение: " + message.getText());
    }
}

class Person{

    private String name;
    String getName(){ return name;}

    Person(String name) {this.name = name; }
}

class Message{

    private String text; // текст сообщения
    String getText(){ return text; }

    Message(String text)
    {
        this.text = text;
    }
}

У цьому випадку для параметра P будутьь передаватися об'єкти типу Person, а для параметра T – об'єкти Message.

Застосуємо класи:


public class Program{
      
    public static void main(String[] args) {
          
        Messenger<Message, Person> telegram = new Messenger<Message, Person>();
        Person tom = new Person("Tom");
        Person bob = new Person("Bob");
        Message hello = new Message("Hello, Bob!");
        telegram.sendMessage(tom, bob, hello);
    }
}

Консольний висновок:


Відправник: Tom
Отримувач: Bob
Сообщение: Hello, Bob!

Множинні обмеження

Також можна установить, чтобы для одного параметра действовало відразу декілька ограничений. В цем случае ограничения об'єднуються за допомогою оператора &:


public class Program{
      
    public static void main(String[] args) {
          
        var icq = new Messenger<PrintedMessage>();
        var hello = new PrintedMessage("Hello ");
        icq.sendMessage(hello);     // Текст сообщения: Hello 
    }
}
class Messenger<T extends Message & Printable> {

    void sendMessage(T message){

        message.print();
    }
}
interface Printable{

    void print();
}

class Message{

    private String text; // текст сообщения
    String getText(){ return text; }

    Message(String text)
    {
        this.text = text;
    }
}

class PrintedMessage 
        extends Message 
        implements Printable{

    PrintedMessage(String text) { super(text); }

    public void print(){
        System.out.println("Текст сообщения: " + getText());
    }
}

У даному випадку клас Messenger використовує параметр типу T, який оновременно повинен представлять кдасс Message и інтерфейс Printable:

class Messenger<T extends Message & Printable> {

Як тест визначаємо підібний клас - PrintedMessage і потім передаємо його об'єкт у метод sendMessage:


var icq = new Messenger<PrintedMessage>();
var hello = new PrintedMessage("Hello ");
icq.sendMessage(hello);     // Текст сообщения: Hello

Обмеження методів

Аналогічно класам обмеження типізованих параметрів можна застосовувати і методум. Наприклад:


public class Program{
      
    public static void main(String[] args) {
          
        sendMessage(new Message("Hello World"));
        sendMessage(new EmailMessage("Bye World"));
    }
    static <T extends Message> void sendMessage(T message){

        System.out.println("Надсилаєся сообщение: " + message.getText());
    }
}
class EmailMessage extends Message{

    EmailMessage(String text) {  super(text); }
}
class SmsMessage extends Message{

    SmsMessage(String text)  { super(text); }
}
class Message{

    private String text; // текст сообщения
    String getText(){ return text; }

    Message(String text){

        this.text = text;
    }
}
