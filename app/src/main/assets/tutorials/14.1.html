Введення у рефлексію. Клас Class

Останнє оновлення: 29.09.2025

Рефлексия представляє мощный механизм, який дозволяє динамически во час виконання анализировать возможнасти классов и исследовать их и их функционал. Рефлексия может быть полезна при создании различных плагинов, анализаторов и других типов приложений, когда динамически треба получать информацию о программе, її типух данных и при возможнасти даже взаємодіяти с ними, даже якщо эти типы данных определяли не вы, и их код вам не доступен.

Class

Ключевым классом для исследования типов є класс Class в мові Java. Рассмотрим его основные методы:

  • Field[] getFields(): повертає массив об'єктов Field для открытых полей данного класса або его суперклассов. Метод повертають массив нулевой длины, якщо таких полей нет або якщо об'єкт представляє собой примитивный тип або массив

  • Field[] getDeclaredFields(): повертає массив об'єктов Field для всех полей данного класса. Метод повертають массив нулевой длины, якщо таких полей нет або якщо об'єкт представляє собой примитивный тип або массив.

  • Method[] getMethods(): повертає открытые методы в виде об'єктов Method (в том числе и унаследованные методы)

  • Method[] getDeclaredMethods(): повертає все методы данного класса або інтерфейса, но не включает унаследованные методы

  • Constructor[] getConstructors(): повертає массив об'єктов Constructor, которые представляють все открытые конструкторы

  • Constructor[] getDeclaredConstructors(): повертає массив об'єктов Constructor, которые представляють все конструкторы, определенные непосредственно в цем классе

  • boolean isAnonymousClass(): повертає true, якщо тип є анонимным классом

  • boolean isArray(): повертає true, якщо тип представляє массив

  • boolean isPrimitive(): повертає true, якщо тип є примитивным типом

  • boolean isInterface(): повертає true, якщо об'єкт представляє інтерфейс

  • boolean isEnum(): повертає true, якщо об'єкт представляє перечисление

  • boolean isRecord(): повертає true, якщо об'єкт представляє класс record

  • boolean isInstance(Object obj): повертає true, якщо об'єкт-параметр є экземпляром цього типу

  • boolean isSealed(): повертає true, якщо класс определен с модификатором sealed

  • Class getSuperclass(): повертає суперкласс для текущего типу

  • RecordComponent[] getRecordComponents(): повертає массив об'єктов RecordComponent, которые описують поля об'єкта, або null, якщо цей об'єкт не представляє класс record.

  • String getPackageName(): отримує имя пакета, содержащего цей тип, або пакета типу элемента, якщо цей тип є массивом, або "java.lang", якщо цей тип є примитивным типом.

Получение класса

Щоб исследовать типы, нам прежде всего треба отримати соответствуюзий типу об'єкт Class. Для цього есть три способа:

  • Метод getClass() в классе Object (а, значит, и у всех остальных классов) повертає экземпляр типу Class

  • Статический метод Class.forName() приймає полное (квалифицированное) имя типу с учетом пакетів и повертає об'єкт Class, який представляє цей тип

  • Поле class в классе Object (а, значит, и у всех остальных классов) повертає экземпляр типу Class

Метод getClass

Самый простой способ отримати связанный с типом об'єкт Class - це метод getClass(), який есть у всех типов в мові Java:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person();
        Class personClass = tom.getClass();
        System.out.println(personClass);    // class Person
    }
}
class Person{ }

Здесь получаем информацию о классе Person, а консоль выведет нам строку "class Person", что вообщем-то очевидно.

Если тип расположен в другом пакеті, то виводиться полное (квалифицированное) имя класса с учетом пакетів. Например:


public class Program{
      
    public static void main(String[] args) {

        String text = "";
        System.out.println(text.getClass());    // class java.lang.String
    }
}

Так, для строки (об'єкта String) полное имя класса буде "java.lang.String"

Метод forName

Cтатическоий метод Class.forName() приймає полное (квалифицированное) имя типу с учетом пакетів и повертає об'єкт Class, який представляє цей тип:


public class Program{
      
    public static void main(String[] args) {
        
        try{

            Class personClass = Class.forName("Person");
            System.out.println(personClass);    // class Person

            Class strClass = Class.forName("java.lang.String");
            System.out.println(strClass);    // class java.lang.String
        }
        catch(ClassNotFoundException ex){

            System.out.println("Class not found");
        }
    }
}
class Person{ }

При использовании цього метода варто учитывать, что якщо Java не найдет класс по переданному имени, то буде сгенерировано исключение ClassNotFoundException, поцему на уровне кода треба обробляти це исключение.

Поле class

Cтатическое поле class, которое есть у всех типов, повертає об'єкт Class, який представляє цей тип:


public class Program{
      
    public static void main(String[] args) {
        
        Class personClass = Person.class;
        System.out.println(personClass);    // class Person

        Class strClass = String.class;
        System.out.println(strClass);    // class java.lang.String
    }
}
class Person{ }
