Введення у рефлексію. Клас Class

Останнє оновлення: 29.09.2025

Рефлексия представляє мощный механизм, який дозволяє динамически во час виконання анализировать возможнасти класов и исследовать их и их функционал. Рефлексия может быть полезна при створенні различных плагинов, анализаторов и других типів приложений, когда динамически треба получать информацию о програме, її типух данных и при возможнасти даже взаємодіяти с ними, даже якщо эти типи данных определяли не вы, и их код вам не доступен.

Class

Ключевым класом для исследования типів є клас Class в мові Java. Рассмотрим его основные методи:

  • Field[] getFields(): повертає масив об'єктов Field для открытых полей данного класа або его суперкласов. Метод повертають масив нулевой длины, якщо таких полей нет або якщо об'єкт представляє собой примитивный тип або масив

  • Field[] getDeclaredFields(): повертає масив об'єктов Field для всех полей данного класа. Метод повертають масив нулевой длины, якщо таких полей нет або якщо об'єкт представляє собой примитивный тип або масив.

  • Method[] getMethods(): повертає открытые методи в виде об'єктов Method (в том числе и унаследованные методи)

  • Method[] getDeclaredMethods(): повертає все методи данного класа або інтерфейса, но не включает унаследованные методи

  • Constructor[] getConstructors(): повертає масив об'єктов Constructor, которые представляютьь все открытые конструктори

  • Constructor[] getDeclaredConstructors(): повертає масив об'єктов Constructor, которые представляютьь все конструктори, визначенийные непосредственно в цем класе

  • boolean isAnonymousClass(): повертає true, якщо тип є анонимным класом

  • boolean isArray(): повертає true, якщо тип представляє масив

  • boolean isPrimitive(): повертає true, якщо тип є примитивным типом

  • boolean isInterface(): повертає true, якщо об'єкт представляє інтерфейс

  • boolean isEnum(): повертає true, якщо об'єкт представляє перечисление

  • boolean isRecord(): повертає true, якщо об'єкт представляє клас record

  • boolean isInstance(Object obj): повертає true, якщо об'єкт-параметр є экземпляром цього типу

  • boolean isSealed(): повертає true, якщо клас визначений с модификатором sealed

  • Class getSuperclass(): повертає суперклас для текущего типу

  • RecordComponent[] getRecordComponents(): повертає масив об'єктов RecordComponent, которые описують поля об'єкта, або null, якщо цей об'єкт не представляє клас record.

  • String getPackageName(): отримує ім'я пакету, содержащего цей тип, або пакету типу елемента, якщо цей тип є масивом, або "java.lang", якщо цей тип є примитивным типом.

Отримання класа

Щоб исследовать типи, нам прежде всего треба отримати соответствуюзий типу об'єкт Class. Для цього есть три способа:

  • Метод getClass() в класе Object (а, значит, и у всех остальных класов) повертає экземпляр типу Class

  • Статический метод Class.forName() приймає полное (квалифицированное) ім'я типу с учетом пакетів и повертає об'єкт Class, який представляє цей тип

  • Поле class в класе Object (а, значит, и у всех остальных класов) повертає экземпляр типу Class

Метод getClass

Самый простой способ отримати связанный с типом об'єкт Class - це метод getClass(), який есть у всех типів в мові Java:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person();
        Class personClass = tom.getClass();
        System.out.println(personClass);    // class Person
    }
}
class Person{ }

Здесь отримуємо информацию о класе Person, а консоль выведет нам рядок "class Person", что вообщем-то очевидно.

Если тип расположен в другом пакеті, то виводиться полное (квалифицированное) ім'я класа с учетом пакетів. Наприклад:


public class Program{
      
    public static void main(String[] args) {

        String text = "";
        System.out.println(text.getClass());    // class java.lang.String
    }
}

Так, для рядки (об'єкта String) полное ім'я класа буде "java.lang.String"

Метод forName

Cтатическоий метод Class.forName() приймає полное (квалифицированное) ім'я типу с учетом пакетів и повертає об'єкт Class, який представляє цей тип:


public class Program{
      
    public static void main(String[] args) {
        
        try{

            Class personClass = Class.forName("Person");
            System.out.println(personClass);    // class Person

            Class strClass = Class.forName("java.lang.String");
            System.out.println(strClass);    // class java.lang.String
        }
        catch(ClassNotFoundException ex){

            System.out.println("Class not found");
        }
    }
}
class Person{ }

При использовании цього методу варто учитывать, что якщо Java не найдет клас по переданному імені, то буде сгенерировано виняток ClassNotFoundException, поцему на уровне коду треба обробляти це виняток.

Поле class

Cтатическое поле class, которое есть у всех типів, повертає об'єкт Class, який представляє цей тип:


public class Program{
      
    public static void main(String[] args) {
        
        Class personClass = Person.class;
        System.out.println(personClass);    // class Person

        Class strClass = String.class;
        System.out.println(strClass);    // class java.lang.String
    }
}
class Person{ }
