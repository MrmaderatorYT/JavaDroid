Внутрішні та вкладені класи

Останнє оновлення: 03.10.20125

У мові Java класи може бути вкладеними (nested), тобто можуть бути визначені всередині інших класів. Усі вкладені класи поділяються на два типи: внутрішні класи (вкладені нестатичні класи) та вкладені статичні класи.

Внутрішні класи

Внутрішні класи (inner class) є частным случаем вложенных классов. Использование внутренних классов прейде главным чином две цели:

  • Внутрішні класи можуть бути приховані з інших класів у тому пакеті.

  • Методам внутрішнього класу доступні дані, зокрема й приватні, із зовнішньої області видимості (із зовнішнього класу).

Наприклад, є клас Person, у якому визначено клас Account:


public class Program{
      
    public static void main(String[] args) {
             
        Person tom = new Person("Tom", "qwerty");
        tom.print();
        tom.account.print();
    }
}
class Person{
     
    private String name;
    Account account;
 
    Person(String name, String password){
        this.name = name;
        account = new Account(password);
    }
    void print(){
        System.out.printf("Person \t Name: %s \t Password: %s \n", name, account.password);
    }
 
    public class Account{
        private String password;
         
        Account(String pass){ this.password = pass; }

        void print(){
            System.out.printf("Account Login: %s \t Password: %s \n", Person.this.name, password);
        }
    }
}

Відзначу ключові моменти програми:

  • Внутрішній клас веде себе як звичайний клас за винятком, що його об'єкти можуть бути створені тільки всередині зовнішнього класу.

  • Внутренний класс имїїт доступ ко всем полям внешнего класса, в том числе закрытым за допомогою модификатора private. Аналогично внешний класс имїїт доступ ко всем компонентам внутреннего класса, в том числе к полям и методам с модификатором private.

  • Ссылку на об'єкт внешнего класса из внутреннего класса можна отримати за допомогою выражения Внешний_класс.this, наприклад, Person.this.

  • Об'єкти внутрішніх класів може бути створено лише у тому класі, у якому внутрішні класи випереджені. В інших зовнішніх класах об'єкти внутрішнього класу створити не можна.

Локальні внутрішні класи

Ще однією особливістю внутрішніх класів і те, що можна оголосити всередині будь-якого контексту, зокрема усередині методу і навіть у циклі. Мотивація для використання таких класів полягає в тому, що їх об'єкти можуть бути потрібні лише в якомусь певному контексті, наприклад, у якомусь певному методі, і більше ніде.

Локальні класи ніколи не оголошуються зі специфікатором доступу, а їх сфера дії завжди обмежена блоком, в якому вони оголошені. При цьому локальні класи повністю приховані від зовнішнього світу – до них можна звернутися лише з контексту, у якому вони визначені. Наприклад:


public class Program{
     
	public static void main(String[] args) {

        Person tom = new Person("Tom");
        tom.setTempAccount("qwerty");
	}
}

class Person{
	
	private String name;

	Person(String name){ this.name = name; }
	
	void setTempAccount (String password){
		
		class TempAccount{
			
			private void print(){
				System.out.printf("Account Login: %s \t Password: %s \n", name, password);
			}
		}
		TempAccount account = new TempAccount();
		account.print();
	}
}

В данном случае класс TempAccount и его об'єкты доступны лише в рамках метода setTempAccount. ПРи цем в данном методе мы можемо обращаться том числе и к приватным компонентам об'єкта: здесь я специально сделал метод print() приватным, но тем не менїї мы можемо его вызвать из метода setTempAccount.

Статичні вкладені класи

Окрім внутрішніх класів усередині інших класів також можна визначати статичні вкладені класи (nested static class). Статичний вкладений клас схожий на внутрішній клас, крім того, що об'єкт статичного класу немає посилання об'єкт зовнішнього класу. Статичні вкладені класи дозволяють приховати деяку комплексну інформацію всередині зовнішнього класу:


class Math{

    public static class Factorial{
    
        private int result;
        private int key;
        
        public Factorial(int number, int x){
            
            result=number;
            key = x;
        }
        
        public int getResult(){
            return result;
        }
        
        public int getKey(){
            return key;
        }
    }
    
    public static Factorial getFactorial(int x){
    
        int result=1;
        for (int i = 1; i <= x; i++){
		
            result *= i;
        }
        return new Factorial(result, x);
    }
}

Здесь определен вложенный класс для хранения данных о вычислении факториала. Основные действия выполняет метод getFactorial, який повертає об'єкт вложенного класса. И теперь используем классы в методе main:


public static void main(String[] args) {
        
    Math.Factorial fact = Math.getFactorial(6);
    System.out.printf("Факториал числа %d равен %d \n", fact.getKey(), fact.getResult());
}
