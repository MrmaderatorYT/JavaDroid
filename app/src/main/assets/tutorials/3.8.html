Перерахування enum

Останнє оновлення: 26.09.2025

Кроме отдельных примитивных типів данных и класов в Java есть такой тип як enum або перечисление. Перечисления представляютьь набор логически связанных констант. Оголошення перечисления происходит за допомогою оператора enum, після которого идет название перечисления:


enum ім'я_перечисления{

    константа_1,
    константа_2,
    ........................
    константа_N,
}

Після назви перерахування у фігурних дужках через кому йде список елементів перерахування. Наприклад:


enum DayTime{

    MORNING,
    AFTERNOON,
    EVENING,
    NIGHT
}

Тут визначено перелік DayTime, який має чотири значення: MORNING, AFTERNOON, EVENING, NIGHT. Назви констант довільні, але Java часто визначаються в повернем регістрі.

Перерахування фактично представляє новий тип, тому ми можемоо визначити змінну даного типу та використовувати її:


class Program{
      
    public static void main(String[] args) {

        DayTime current = DayTime.MORNING;
		System.out.println(current);	// MORNING
    }
}
enum DayTime{

    MORNING,
    AFTERNOON,
    EVENING,
    NIGHT
}

Зверніть увагу, що при використанні констант нам треба зазначати їхнє повне ім'я з урахуванням перерахування:

DayTime current = DayTime.MORNING;

При вивіде на консоль константа перечисления преобразуется в рядок, которая фактически представляє название цей константи, наприклад, для DayTime.MORNING це "MORNING".

Перерахування у конструкціях/виразах switch

Перечисления удобно використовувати в конструкциях/выражениях switch, которые дозволяють не указывать название перечисления при сопоставлении с константами:


class Program{
      
    public static void main(String[] args) {

        DayTime current = DayTime.MORNING;
        String message = getMessage(current);
		System.out.println(message);	// Доброе утро

        message = getMessage(DayTime.EVENING);
        System.out.println(message);	// Добрый вечер
    }

    static String getMessage(DayTime dt){

        return switch(dt){
            case MORNING -> "Доброе утро";
            case AFTERNOON -> "Добрый день";
            case EVENING -> "Добрый вечер";
            case NIGHT -> "Доброй ночи";
        };
    }
}
enum DayTime{

    MORNING,
    AFTERNOON,
    EVENING,
    NIGHT
}

Перерахування на зберігання стану

Перерахування зручно використовувати у класах для збереження стану:


class Program{
      
    public static void main(String[] args) {

        Operation sum = new Operation(OpType.ADD, 5, 4);
        System.out.println(sum.execute());	// 9

        Operation sub = new Operation(OpType.SUBTRACT, 5, 4);
        System.out.println(sub.execute());	// 1

        Operation mul = new Operation(OpType.MULTIPLY, 5, 4);
        System.out.println(mul.execute());	// 20
    }
}
enum OpType{

    ADD,
    SUBTRACT,
    MULTIPLY
}
class Operation{
    
    private OpType type;
    private int op1;
    private int op2;

    Operation(OpType type, int op1, int op2){
        this.type = type;
        this.op1 = op1;
        this.op2 = op2;
    }

    int execute(){
        return switch(type){

            case ADD -> op1 + op2;
            case SUBTRACT -> op1 - op2;
            case MULTIPLY -> op1 * op2;
        };
    }
}

Тут перелік OpType є типом операції. Клас Operation представляє арифметичну операцію і через конструктор встановлює її тип, і навіть два операнда. А в Метод execute залежно від типу операції виконує ті чи інші дії.

Методи перерахувань

Каждое перечисление имїїт статический метод values(). Он повертає масив всех констант перечисления:


class Program{
      
    public static void main(String[] args) {

        DayTime[] dts = DayTime.values();
        for(DayTime dt: dts){
            System.out.println(dt);    
        }
    }
}
enum DayTime{

    MORNING,
    AFTERNOON,
    EVENING,
    NIGHT
}

Консольний висновок:


MORNING
AFTERNOON
EVENING
NIGHT

Метод ordinal() повертає порядковый номер визначенийной константи (нумерация начинается с 0):


class Program{
      
    public static void main(String[] args) {

        var current = DayTime.MORNING;
        System.out.println(current.ordinal());  // 0

        System.out.println(DayTime.AFTERNOON.ordinal());  // 1
    }
}

Цей спосіб зручний створення карт зіставлення:


class Program{
      
    static String[] messages = {"Доброе утро", "Добрый день", "Добрый вечер", "Доброй ночи"};

    public static void main(String[] args) {

        var current = DayTime.EVENING;
        System.out.println(messages[current.ordinal()]);  // Добрый вечер
    }
}
enum DayTime{

    MORNING,
    AFTERNOON,
    EVENING,
    NIGHT
}

Конструктори, поля та методи перерахування

Перерахування, як і звичайні класи, можуть визначати конструктори, поля та методи. Наприклад:


class Program{

    public static void main(String[] args) {

        var current = DayTime.EVENING;
        System.out.println(current.getMessage());  // Добрый вечер
    }
}
enum DayTime{

    MORNING ("Доброе утро"), AFTERNOON("Добрый день"), EVENING("Добрый вечер"), NIGHT("Доброй ночи");

    private String message;
	DayTime(String message){
		this.message = message;
	}
	public String getMessage(){ return message;}
}

Перелік DayTome визначає приватне поле message для зберігання повідомлення, пов'язаного зі значенням перерахування, а за допомогою методу getMessage воно повертається. Через конструктор передається йому значення.

Следует відзначити, что конструктор по умолчанию приватный, то есть имїїт модификатор private. Любой другой модификатор буде считаться ошибкой. Тому створити константи перечисления за допомогою конструктора мы можемо лише всередині перечисления.

Визначення методів

Також можна визначати методи окремих констант:


class Program{
      
    public static void main(String[] args) {
          
        Operation op = Operation.SUM;
        System.out.println(op.action(10, 4));   // 14
        op = Operation.MULTIPLY;
        System.out.println(op.action(6, 4));    // 24
    }
}
enum Operation{
    SUM{
        int action(int x, int y){ return x + y;}
    },
    SUBTRACT{
        int action(int x, int y){ return x - y;}
    },
    MULTIPLY{
        int action(int x, int y){ return x * y;}
    };
    abstract int action(int x, int y);
}
