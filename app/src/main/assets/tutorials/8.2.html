Створення та виконання потоків

Останнє оновлення: 16.10.2025

Для создания нового потока (потока платформы) в Java мы можемо використовувати следующие варианты:

  • Реализуация інтерфейса Runnable

  • Створення класса, який найдеся от Thread

Рассмотрим оба варианта.

Реализация інтерфейса Runnable

Интерфейс Runnable представляє операцию, которая не повертає ниякого результата. Цет інтерфейс имїїт один метод run:


interface Runnable{
	
	void run();
}

В методе run() собственно определяется весь тот код, який виконується при запуске потока.

Общий процесс создания и запуска потока выглядит наступним чином:

  1. Определяем реализацию інтерфейса Runable. Например:

    
Runnable task = ()->{

    // здесь действия потока
};

    Поскольку Runnable представляє функциональный інтерфейс, який определяет один метод, то об'єкт цього інтерфейса мы можемо представить в виде лямбда-выражения.

  2. Создаем об'єкт Thread за допомогою Runable:

    
var t = new Thread(task);

    Класс Thread имїїт декілька конструкторов. В данном случае используем тот, який приймає об'єкт Runable

  3. Запускаем поток Thread за допомогою метода start():

    
t.start();

Посмотрим на примере:


public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        Runnable task = ()->{       // определяем действия потока
            System.out.println("Runnable started...");

            System.out.println("Runnable finished...");
        };

        var t = new Thread(task);  // определяем поток

        t.start();  // запускаем поток

        System.out.println("Main thread finished...");
    }
}

Здесь в создаваемом потоке просто выводим два сообщения на консоль. Консольный вывод в цем случае может быть наступним:


Main thread started...
Main thread finished...
Runnable started...
Runnable finished...

По консольному выводу видно, что, несмотря на то, что поток запускаєся до окончания метода main(), но его работа виконується после последней инструкции в методе main().

При создании потока - об'єкта Thread за допомогою оператора new и вызова конструктора:

var t = new Thread(task);

поток автоматически не запускаєся. Он лишь переходит в состояние NEW, а программа ще не начала выполнять его код.

После вызова метода start() поток переходит в состояние RUNNABLE - готовый к запуску. Однак в реальности такой поток ще не виконується: оне может быть запущен, а может и не запущен. Планировщик потоков решает, предоставить ли потоку час для виконання. После того, як поток запущен, он может периодически приостанавливаться, чтобы дать другим потокам возможнасть выполниться. Однак точные детали планирования потоков зависят от операционной системы и от природы потока.

Вместо лямбда-выражения мы можемо избрать болїї долгий путь - определить отдельный класс, який реализует інтерфейс Runable. Например:


class MyTask implements Runnable {
    
    public void run(){  // определяем действия потока
         
        System.out.println("Runnable started...");
        System.out.println("Runnable finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        Runnable task = new MyTask();

        var t = new Thread(task);  // определяем поток

        t.start();  // запускаем поток

        System.out.println("Main thread finished...");
    }
}

Успадкування от класса Thread

Другой способ создания потока представляє наследование от класса Thread. Например:


class MyThread extends Thread {
     
    MyThread(){
        super();  // вызываем конструктор базового класса
    }
     
    public void run(){
         
        System.out.println("MyThread started...");
        System.out.println("MyThread finished...");
    }
}

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var t = new MyThread();  // определяем поток

        t.start();  // запускаем поток

        System.out.println("Main thread finished...");
    }
}

Класс потока називається MyThread. В конструкторе класса вызываем один из конструкторов базового класса Thread. И також в MyThread переопределяется метод run(), код которого собственно и буде представлять весь тот код, який виконується в потоке.

В методе main для запуска потока JThread у него вызывается метод start(), после чего начинается выполнение того кода, який определен в методе run:


var t = new MyThread();  // определяем поток
t.start();  // запускаем поток

Консольный вывод:


Main thread started...
Main thread finished...
MyThread started... 
MyThread finished...

Запуск массива потоков

Аналогично созданию одного потока мы можемо запускать відразу декілька потоков:


class MyTask implements Runnable {
    
    private String name;

    MyTask(String name) { this.name = name;}

    public void run(){  // определяем действия потока
         
        System.out.println(name + " started...");
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        for(int i=1; i < 6; i++){

            var task = new MyTask("MyTask_" + i);
            new Thread(task).start();
        }

        System.out.println("Main thread finished...");
    }
}

В данном случае для разграничения потоков в класс MyTask через конструктор передаєся имя потока. А в цикле при запуске потока формируем имя на основе номера потока.

Консольный вывод в данном случае недетерминирован. Наприклад, он может быть наступним:


Main thread started...
Main thread finished...
MyTask_2 started...
MyTask_1 started...
MyTask_1 finished...
MyTask_4 started...
MyTask_3 started...
MyTask_3 finished...
MyTask_2 finished...
MyTask_4 finished...
MyTask_5 started...
MyTask_5 finished...

Но по данному выводу мы видимо, что потоки вне зависимости от запуска работают фактически одновременно, як бы параллельно

Конструкторы Thread

Для данного класса Thread определенно декілька конструкторов, которые можна використовувати. Но я отмечу один из них:


Thread(ThreadGroup group, Runnable task, String name)

Параметры конструктора:

  • group: группа потока. Если значение равно null, то в качестве группы устанавливается группа текущего потока.

  • task: об'єкт Runnable, який собственно и определяет выполняемые в потоке действия. Если значение равно null, то вызывается метод run() цього потока.

  • name: имя создаваемого потока

Я взял именно цей вариант конструктора, потому что большинство других неявно вызывают его. Наприклад, следующий вариант конструктора

Thread(Runnable task, String name)

Фактически имїїт тот же эффект, что и вызов

Thread (null, task, name)

А вариант конструктора

Thread(Runnable task)

Фактически аналогичен вызову

Thread (null, task, gname)

Здесь gname - автоматически генерируемое имя в виде "Thread-"+n, где n - целое число.

Наприклад, модифицтруем предыдущую программу:


class MyTask implements Runnable {

    public void run(){  // определяем действия потока
         
        // получаем имя текущего потока
        String name = Thread.currentThread().getName();
        System.out.println(name + " started...");
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        for(int i=1; i < 6; i++){

            var task = new MyTask();
            new Thread(task).start();
        }

        System.out.println("Main thread finished...");
    }
}

Здесь в классе MyTask в методе run() за допомогою вызова Thread.currentThread().getName() получаем имя потока, которое автоматически ему присвоюється. Далїї на консоли мы увидим автосгенерированные имена потоков:


Main thread started...
Main thread finished...
Thread-2 started...
Thread-0 started...
Thread-3 started...
Thread-4 started...
Thread-0 finished...
Thread-1 started...
Thread-2 finished...
Thread-3 finished...
Thread-4 finished...
Thread-1 finished...

Такие имена можуть быть неудобны, або мы захочемо подобрать болїї описательные имена для различения потоков. В цем случае мы можемо передавать имена через второй параметр конструктора Thread:


for(int i=1; i < 6; i++){

    var task = new MyTask();
    new Thread(task, "MyTask_" + i).start();
}

При консольном выводе мы увидим назначенные вручную потокам имена:


Main thread started...
Main thread finished...
MyTask_1 started...
MyTask_4 started...
MyTask_1 finished...
MyTask_3 started...
MyTask_4 finished...
MyTask_3 finished...
MyTask_5 started...
MyTask_5 finished...
MyTask_2 started...
MyTask_2 finished...
