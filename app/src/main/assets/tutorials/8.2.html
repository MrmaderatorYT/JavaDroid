Створення та виконання потіків

Останнє оновлення: 16.10.2025

Для створенняя нового потіка (потіка платформы) в Java мы можемо використовувати следующие варианты:

  • Реализуация інтерфейса Runnable

  • Створення класа, який найдеся от Thread

Рассмотрим оба варианта.

Реалізація інтерфейса Runnable

Інтерфейс Runnable представляє операцію, которая не повертає ниякого результата. Цет інтерфейс имїїт один метод run:


interface Runnable{
	
	void run();
}

В методі run() собственно визначаєся весь тот код, який виконується при запуску потіка.

Общий процес створенняя и запуску потіка выглядит наступним чином:

  1. Визначаємо реалізацію інтерфейсу Runable. Наприклад:

    
Runnable task = ()->{

    // здесь действия потіка
};

    Поскольку Runnable представляє функциональный інтерфейс, який визначає один метод, то об'єкт цього інтерфейса мы можемо представить в виде лямбда-выражения.

  2. Створюємо об'єкт Thread за допомогою Runable:

    
var t = new Thread(task);

    Клас Thread имїїт декілька конструкторов. У даному випадку використовуємо тот, який приймає об'єкт Runable

  3. Запускуємо потік Thread за допомогою методу start():

    
t.start();

Посмотрим на примере:


public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        Runnable task = ()->{       // определяем действия потіка
            System.out.println("Runnable started...");

            System.out.println("Runnable finished...");
        };

        var t = new Thread(task);  // определяем потік

        t.start();  // запускуємо потік

        System.out.println("Main thread finished...");
    }
}

Здесь в создаваемом потіке просто вивідим два сообщения на консоль. Консольный вивід в цем случае может быть наступним:


Main thread started...
Main thread finished...
Runnable started...
Runnable finished...

По консольному вивіду видно, что, несмотря на то, что потік запускуєся до окончания методу main(), но его работа виконується після післядней инструкции в методі main().

При створенні потіка - об'єкта Thread за допомогою оператора new и вызова конструктора:

var t = new Thread(task);

потік автоматически не запускуєся. Он лишь переходит в состояние NEW, а програма ще не начала выполнять его код.

Після вызова методу start() потік переходит в состояние RUNNABLE - готовый к запуску. Однак в реальности такой потік ще не виконується: оне может быть запущен, а может и не запущен. Планировщик потіков решает, предоставить ли потіку час для виконання. Після того, як потік запущен, он может периодически приостанавливаться, чтобы дать другим потікам возможнасть выполниться. Однак точные детали планирования потіков зависят от операционной системы и от природы потіка.

Вместо лямбда-выражения мы можемо избрать болїї долгий путь - визначити отдельный клас, який реалізує інтерфейс Runable. Наприклад:


class MyTask implements Runnable {
    
    public void run(){  // определяем действия потіка
         
        System.out.println("Runnable started...");
        System.out.println("Runnable finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        Runnable task = new MyTask();

        var t = new Thread(task);  // определяем потік

        t.start();  // запускуємо потік

        System.out.println("Main thread finished...");
    }
}

Успадкування от класа Thread

Другой способ створенняя потіка представляє наследование от класа Thread. Наприклад:


class MyThread extends Thread {
     
    MyThread(){
        super();  // вызываем конструктор базового класа
    }
     
    public void run(){
         
        System.out.println("MyThread started...");
        System.out.println("MyThread finished...");
    }
}

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        var t = new MyThread();  // определяем потік

        t.start();  // запускуємо потік

        System.out.println("Main thread finished...");
    }
}

Клас потіка називається MyThread. В конструкторе класа вызываем один из конструкторов базового класа Thread. И також в MyThread перевизначаєся метод run(), код которого собственно и буде представлять весь тот код, який виконується у потоці.

В методі main для запуску потіка JThread у него вызывается метод start(), після чего начинается виконання того коду, який визначений в методі run:


var t = new MyThread();  // определяем потік
t.start();  // запускуємо потік

Консольный вивід:


Main thread started...
Main thread finished...
MyThread started... 
MyThread finished...

Запуск масива потіков

Аналогично створенняю одного потіка мы можемо запускуть відразу декілька потіков:


class MyTask implements Runnable {
    
    private String name;

    MyTask(String name) { this.name = name;}

    public void run(){  // определяем действия потіка
         
        System.out.println(name + " started...");
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        for(int i=1; i < 6; i++){

            var task = new MyTask("MyTask_" + i);
            new Thread(task).start();
        }

        System.out.println("Main thread finished...");
    }
}

У даному випадку для разграничения потіков в клас MyTask через конструктор передаєся ім'я потіка. А в цикле при запуску потіка формируем ім'я на основе номера потіка.

Консольный вивід у даному випадку недетерминирован. Наприклад, он может быть наступним:


Main thread started...
Main thread finished...
MyTask_2 started...
MyTask_1 started...
MyTask_1 finished...
MyTask_4 started...
MyTask_3 started...
MyTask_3 finished...
MyTask_2 finished...
MyTask_4 finished...
MyTask_5 started...
MyTask_5 finished...

Но по данному вивіду мы видимо, что потіки вне зависимости от запуску работают фактически одновременно, як бы параллельно

Конструктори Thread

Для данного класа Thread визначенийно декілька конструкторов, которые можна використовувати. Но я отмечу один из них:


Thread(ThreadGroup group, Runnable task, String name)

Параметри конструктора:

  • group: группа потіка. Если значення равно null, то в качестве группы встановлюєся группа текущего потіка.

  • task: об'єкт Runnable, який собственно и визначає выполняемые у потоці действия. Если значення равно null, то вызывается метод run() цього потіка.

  • name: ім'я создаваемого потіка

Я взял именно цей вариант конструктора, потому что большинство других неявно вызывают его. Наприклад, следующий вариант конструктора

Thread(Runnable task, String name)

Фактически имїїт тот же эффект, что и вызов

Thread (null, task, name)

А вариант конструктора

Thread(Runnable task)

Фактически аналогичен вызову

Thread (null, task, gname)

Здесь gname - автоматически генерируемое ім'я в виде "Thread-"+n, где n - целое число.

Наприклад, модифицтруем предыдущую програму:


class MyTask implements Runnable {

    public void run(){  // определяем действия потіка
         
        // отримуємо ім'я текущего потіка
        String name = Thread.currentThread().getName();
        System.out.println(name + " started...");
        System.out.println(name + " finished...");
    }
} 

public class Program {
  
    public static void main(String[] args) {
          
        System.out.println("Main thread started...");

        for(int i=1; i < 6; i++){

            var task = new MyTask();
            new Thread(task).start();
        }

        System.out.println("Main thread finished...");
    }
}

Здесь в класе MyTask в методі run() за допомогою вызова Thread.currentThread().getName() отримуємо ім'я потіка, которое автоматически ему присвоюється. Далїї на консоли мы увидим автосгенерированные имена потіков:


Main thread started...
Main thread finished...
Thread-2 started...
Thread-0 started...
Thread-3 started...
Thread-4 started...
Thread-0 finished...
Thread-1 started...
Thread-2 finished...
Thread-3 finished...
Thread-4 finished...
Thread-1 finished...

Такие имена можуть быть неудобны, або мы захочемо підобрать болїї описательные имена для различения потіков. В цем случае мы можемо передавать имена через второй параметр конструктора Thread:


for(int i=1; i < 6; i++){

    var task = new MyTask();
    new Thread(task, "MyTask_" + i).start();
}

При консольном вивіде мы увидим назначенные вручную потікам имена:


Main thread started...
Main thread finished...
MyTask_1 started...
MyTask_4 started...
MyTask_1 finished...
MyTask_3 started...
MyTask_4 finished...
MyTask_3 finished...
MyTask_5 started...
MyTask_5 finished...
MyTask_2 started...
MyTask_2 finished...
