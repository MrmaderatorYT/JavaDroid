Анонімні класи

Останнє оновлення: 03.10.20125

Иногда бывает необхідно створити лише один об'єкт некоторого класа. Причем его ім'я даже не важливо. Такой клас можна визначити як анонімний. Часто анонимные класы создаются там, где требуется об'єкт некоторого інтерфейс, чтобы не визначати отдельный клас.

Загальний синтаксис визначення анонімного класу:


new SuperType(аргументы для конструктора суперкласа)
{
    // поля и методи анонимного класа
}

Тут SuperType може бути інтерфейсом, тоді анонімний клас продає цей інтерфейс. SuperType може бути класом, тоді анонімний клас розширює цей клас.

Анонімний внутрішній клас неспроможна мати конструкторів, оскільки ім'я конструктора має збігатися з ім'ям класу, а класу немає імені. Натомість аргументи конструювання передаютьься конструктору суперкласу (якщо у суперкласі визначено конструктор із параметрами).

Якщо SuperType представляє інтерфейс, то аргументи для конструктора природно не передаютьься (адже інтерфейси не мають конструктора), і просто вказуються аусті дужки:


new InterfaceType()
{
    // поля и методи анонимного класа
}

Анонімний клас замість інтерфейсу Поширеним варіантом використання анонімних класів є створення там, де потрібно об'єкт інтерфейсу. Наприклад:

Основні моменти:


public class Program{
     
	public static void main(String[] args) {

        var rectangle = new Movable(){

            public void move(int distance){
                System.out.printf("Перемещаем прямоугольник на %d пикселей\n", distance);
            }
        };
        moveObject(rectangle, 56);     // Перемещаем прямоугольник на 56 пикселей
	}
    // перемещаем об'єкт Movable на расстояние distance
    static void moveObject(Movable obj, int distance){
        obj.move(distance);
    }
}

interface Movable{
    void move(int distance);
}

Це схоже на звичайну реалізацію інтерфейсу, тільки в даному випадку клас не має імені - його замінює ім'я інтерфейсу і немає конструктора.

  1. У нас есть інтерфейс Movable, в котором визначений метод move()

    
interface Movable{
    void move(int distance);
}

  2. Визначаємо метод moveObject(), в який передаєся об'єкт інтерфейса Movable. І далі у цього об'єкта викликається метод move()

    
static void moveObject(Movable obj, int distance){
    obj.move(distance);
}

  3. Визначаємо змінну rectangle, которая по сути представляє об'єкт анонимного класа, який, в свою очередь, реалізує інтерфейс Movable:

    
var rectangle = new Movable(){

    public void move(int distance){
        System.out.printf("Перемещаем прямоугольник на %d пикселей\n", distance);
    }
};

    При цьому нам необов'язково створювати окремо змінну, яка містить посилання на об'єкт анонімного класу. Ми можемоо створити об'єкт анонімного класу безпосередньо за виклику методу:

  4. И в конце вызываем метод moveObject(), передавая в него об'єкт анонимного класа:

    moveObject(rectangle, 54);

Анонімний клас замість класу Менш поширений варіант, але можливий - це розширення анонімним класом звичайного класу. Наприклад:


public class Program{
     
	public static void main(String[] args) {

        // передаємо об'єкт анонимного класа
        moveObject(new Movable(){
            public void move(int distance){
                System.out.printf("Человек идет на расстояние %d метров\n", distance);
            }
        }, 
        100);
	}
    static void moveObject(Movable obj, int distance){
        obj.move(distance);
    }
}

interface Movable{
    void move(int distance);
}

Розглянемо основні моменти:

означає створення об'єкта анонімного класу, який розширює клас Person і передає його конструктор рядок " Tom " .


public class Program{
     
	public static void main(String[] args) {

        printPerson(
            new Person("Tom"){

                private String company = "Yandex";

                // реалізація методу print
                @Override
                void print(){
                    System.out.printf("Employee. Name: %s;  Company: %s\n", super.getName(), company);
                }
            }
        );
	}
    static void printPerson(Person person){
        person.print();
    }
}
abstract class Person{
    
    private String name;
    String getName(){ return name;}

    Person(String name){

        this.name = name;
    }

    abstract void print();
}

Потім у фігурних дужках визначаємо тіло анонімного класу:

  1. Здесь визначений абстрактный клас Person, який представляє некоторого человека. Через конструктор он приймає ім'я и визначає абстрактный метод ptint()

  2. Статический метод printPerson() приймає об'єкт Person и вызывает у него метод ptint()

    
static void printPerson(Person person){
    person.print();
}

  3. При вызове методу printPerson() передаємо об'єкт анонимного класа. Причем, як мы знаем, в Java мы не можемо використовувати вызов конструктора для створенняя об'єкта абстрактного класа. И у даному випадку вызов

    new Person("Tom")

    Ініціалізатори в анонімних класах Як говорилося вище, ми можемоо визначити конструктор в анонімних класах, проте блок ініціалізатора для початкової ініціалізації:

    
{
    private String company = "Yandex";
    @Override
    void print(){
        System.out.printf("Employee. Name: %s;  Company: %s\n", super.getName(), company);
    }
}

    В реалізації додаємо дополнительное поле (для хранения компании) и определяем реализацию для абстрактного методу print(). Причем в реалізації анонимного класа мы можемо, як и в производных класах, ссылаться на функционал базового класа через ключевое слово super


printPerson(
    new Person("Tom"){

        private String company;

        // блок инициализатора
        {
            company = "Sber";
        }
        // реалізація методу print
        @Override
        void print(){
            System.out.printf("Employee. Name: %s;  Company: %s\n", super.getName(), company);
        }
    }
);
