Нативні функції зворотного виклику

Останнє оновлення: 22.12.2025

Иногда одна функция в качестве параметра приймає другую функцию - функцию обратного вызова або callback и в процессе своей работы вызывает її. Наприклад, возьмем функцию быстрой сортировки qsort, которая есть в стандартной библиотеке языка C:


void qsort(void *base, size_t nmemb, size_t size, int (*compare)(const void *, const void *));

При вызове цей функції ей передаются:

  • base: массив сортируемых элементов

  • nmemb: длина массива

  • size: размер каждого элемента

  • compar: ​​адрес функції обратного вызова, которая собственно сравнивает два элемента

Последний параметр представляє указатель на функцию, которая вызывается при сравнении двух элементов. С точки зрения API FFM, як и любой указатель, указатель на функцию передаєся через значение ValueLayout.ADDRESS. Однак может возникнуть вопрос, як определить подобную функцию и як передать її в вызов. Рассмотрим на примере следующей программы:


import java.lang.foreign.*;
import java.lang.invoke.*;
import java.util.Arrays;
import static java.lang.foreign.ValueLayout.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // получаем линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // получаем стандартную библиотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // получаем адрес функції sqort
        MemorySegment qsortAddr = stdlib.findOrThrow("qsort");
        // определяем дескриптор функції
        FunctionDescriptor qsortDesc = FunctionDescriptor.ofVoid(ADDRESS, JAVA_LONG, JAVA_LONG, ADDRESS);
        // void qsort(void*, size_t, size_t, int (*)(const void *, const void *))
        MethodHandle qsort = linker.downcallHandle(qsortAddr, qsortDesc);

        // получаем тип метода
        MethodType compareType = MethodType.methodType(int.class, MemorySegment.class, MemorySegment.class);
        // получаем дескриптор статического метода
        MethodHandle compareHandle = MethodHandles.lookup().findStatic(Program.class, "compareInt", compareType);

        // определяем заглушку для функції
        MemorySegment compareStub = linker.upcallStub(
            compareHandle,
            FunctionDescriptor.of(JAVA_INT,
                ADDRESS.withTargetLayout(JAVA_INT),
                ADDRESS.withTargetLayout(JAVA_INT)),
            Arena.ofAuto());

        // дані для сортировки 
        int[] numbers = {3, 5, 1, 8, 0, 5, 2};

        try (Arena arena = Arena.ofConfined()) {

            // определяем для чисел numbers сегмент памяти
            MemorySegment numbers_seg = arena.allocateFrom(JAVA_INT, numbers);
            qsort.invoke(numbers_seg, numbers.length, JAVA_INT.byteSize(), compareStub);

            // преобразуем сегмент памяти в массив
            numbers = numbers_seg.toArray(JAVA_INT);
            System.out.println(Arrays.toString(numbers));   // [0, 1, 2, 3, 5, 5, 8]
        }
    }

    static int compareInt(MemorySegment a, MemorySegment b) {

        return Integer.compare(a.get(JAVA_INT, 0), b.get(JAVA_INT, 0));
    }
}

Рассмотрим поэтапно программный код:

  1. Допустим, мы хочемо, чтобы нативная функция qsort для сортировки использовала следующий метод Java:

    
static int compareInt(MemorySegment a, MemorySegment b) {

    return Integer.compare(a.get(JAVA_INT, 0), b.get(JAVA_INT, 0));
}

    Обратите внимание, что метод призначений для сортировки целочисленных значений типу int, однак параметры представляють значения MemorySegment

  2. Для начала нам нам, як и в общем случае, треба отримати дескриптор метода внешней функції:

    
// получаем линкер/компоновщик 
Linker linker = Linker.nativeLinker();
// получаем стандартную библиотеку
SymbolLookup stdlib = linker.defaultLookup();

// получаем адрес функції sqort
MemorySegment qsortAddr = stdlib.findOrThrow("qsort");
// определяем дескриптор функції
FunctionDescriptor qsortDesc = FunctionDescriptor.of(ADDRESS, JAVA_LONG, JAVA_LONG, ADDRESS);
// void qsort(void*, size_t, size_t, int (*)(const void *, const void *))
MethodHandle qsort = linker.downcallHandle(qsortAddr, qsortDesc);

  3. Щоб отримати адрес цього метода, сначала получаем об'єкт MemoryHandle для статического метода Java:

    
// получаем тип метода
MethodType compareType = MethodType.methodType(int.class, MemorySegment.class, MemorySegment.class);

// получаем дескриптор статического метода (Program - текущий класс, где располагается метод)
MethodHandle compareHandle = MethodHandles.lookup().findStatic(Program.class, "compareInt", compareType);

    Метод MethodType.methodType приймає тип возвращаемого значения и далїї типы параметров метода. В нашем случае метод сортировки compareInt повертає значение int и приймає два значения MemorySegment

  4. С помощью метода upcallStub() линкера сгенерируем заглушку для функції C, которая затем вызывает дескриптор метода обратного вызова:

    
MemorySegment compareStub = linker.upcallStub(
    compareHandle,
    FunctionDescriptor.of(JAVA_INT,
        ADDRESS.withTargetLayout(JAVA_INT),
        ADDRESS.withTargetLayout(JAVA_INT)),
    Arena.ofAuto());

    Метод upcallStub() створює заглушку вызова, яку можна передать другим внешним функциям в виде указателя на функцию. Вызов такого указателя на функцию из внешнего кода приведет к выполнению предоставленного дескриптора метода:

    
MemorySegment upcallStub(MethodHandle target,
 FunctionDescriptor function,
 Arena arena,
 Linker.Option... options)

    Метод приймає до 4 параметров:

    • target: дескриптор целевого метода

    • function: дескриптор функції-заглушки вызываемой функції

    • arena: арена, связанная с возвращаемым сегментом-заглушкой вызываемой функції

    • options: параметры линкера, связанные с цим запросом на компоновку

    Обратите внимание на вызовы withTargetLayout(). Они настраивают структуру памяти, чтобы линкер мог сгенерировать код для настройки экземпляров MemorySegment, которые отримує обратный вызов Java. Если цього не сделать, вызовы a.get(JAVA_INT, 0) и b.get(JAVA_INT, 0) в Java-коллбэке завершатся ошибкой, поскольку сегменты памяти a и b будуть иметь нулевой размер.

  5. И в конце можна вызвать внешнюю функцию с адресом коллбэка:

    
float[] samples = . . .;
MemorySegment csamples = arena.allocateFrom(JAVA_FLOAT, samples);

qsort.invoke(csamples, samples.length, JAVA_FLOAT.byteSize(), compStub);

При выполнении программы на вход функції qsort передаем массив {3, 5, 1, 8, 0, 5, 2} и после виконання мы увидим консольный вывод отсортированного массива:

[0, 1, 2, 3, 5, 5, 8]
