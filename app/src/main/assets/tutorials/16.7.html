Нативні функції зворотного виклику

Останнє оновлення: 22.12.2025

Иногда одна функція в качестве параметра приймає другую функцію - функцію обратного вызова або callback и в процесе своей работы вызывает її. Наприклад, возьмем функцію быстрой сортировки qsort, которая есть в стандартной библиотеке языка C:


void qsort(void *base, size_t nmemb, size_t size, int (*compare)(const void *, const void *));

При вызове цей функції ей передаються:

  • base: масив сортуємоых елементов

  • nmemb: длина масива

  • size: размер каждого елемента

  • compar: ​​адрес функції обратного вызова, которая собственно сравнивает два елемента

Післядний параметр представляє указатель на функцію, которая вызывается при сравнении двух елементов. С точки зрения API FFM, як и любой указатель, указатель на функцію передаєся через значення ValueLayout.ADDRESS. Однак может возникнуть вопрос, як визначити підобную функцію и як передать її в вызов. Рассмотрим на примере следующей програми:


import java.lang.foreign.*;
import java.lang.invoke.*;
import java.util.Arrays;
import static java.lang.foreign.ValueLayout.*;

public class Program {

    public static void main(String[] args) throws Throwable{

        // отримуємо линкер/компоновщик 
        Linker linker = Linker.nativeLinker();
        // отримуємо стандартную бібліотеку
        SymbolLookup stdlib = linker.defaultLookup();

        // отримуємо адрес функції sqort
        MemorySegment qsortAddr = stdlib.findOrThrow("qsort");
        // определяем дескриптор функції
        FunctionDescriptor qsortDesc = FunctionDescriptor.ofVoid(ADDRESS, JAVA_LONG, JAVA_LONG, ADDRESS);
        // void qsort(void*, size_t, size_t, int (*)(const void *, const void *))
        MethodHandle qsort = linker.downcallHandle(qsortAddr, qsortDesc);

        // отримуємо тип методу
        MethodType compareType = MethodType.methodType(int.class, MemorySegment.class, MemorySegment.class);
        // отримуємо дескриптор статического методу
        MethodHandle compareHandle = MethodHandles.lookup().findStatic(Program.class, "compareInt", compareType);

        // определяем заглушку для функції
        MemorySegment compareStub = linker.upcallStub(
            compareHandle,
            FunctionDescriptor.of(JAVA_INT,
                ADDRESS.withTargetLayout(JAVA_INT),
                ADDRESS.withTargetLayout(JAVA_INT)),
            Arena.ofAuto());

        // дані для сортировки 
        int[] numbers = {3, 5, 1, 8, 0, 5, 2};

        try (Arena arena = Arena.ofConfined()) {

            // определяем для чисел numbers сегмент памяти
            MemorySegment numbers_seg = arena.allocateFrom(JAVA_INT, numbers);
            qsort.invoke(numbers_seg, numbers.length, JAVA_INT.byteSize(), compareStub);

            // преобразуем сегмент памяти в масив
            numbers = numbers_seg.toArray(JAVA_INT);
            System.out.println(Arrays.toString(numbers));   // [0, 1, 2, 3, 5, 5, 8]
        }
    }

    static int compareInt(MemorySegment a, MemorySegment b) {

        return Integer.compare(a.get(JAVA_INT, 0), b.get(JAVA_INT, 0));
    }
}

Рассмотрим поэтапно програмный код:

  1. Допустим, мы хочемо, чтобы нативная функція qsort для сортировки использовала следующий метод Java:

    
static int compareInt(MemorySegment a, MemorySegment b) {

    return Integer.compare(a.get(JAVA_INT, 0), b.get(JAVA_INT, 0));
}

    Зверніть увагу, что метод призначений для сортировки целочисленных значений типу int, однак параметри представляютьь значення MemorySegment

  2. Для начала нам нам, як и в общем случае, треба отримати дескриптор методу внешней функції:

    
// отримуємо линкер/компоновщик 
Linker linker = Linker.nativeLinker();
// отримуємо стандартную бібліотеку
SymbolLookup stdlib = linker.defaultLookup();

// отримуємо адрес функції sqort
MemorySegment qsortAddr = stdlib.findOrThrow("qsort");
// определяем дескриптор функції
FunctionDescriptor qsortDesc = FunctionDescriptor.of(ADDRESS, JAVA_LONG, JAVA_LONG, ADDRESS);
// void qsort(void*, size_t, size_t, int (*)(const void *, const void *))
MethodHandle qsort = linker.downcallHandle(qsortAddr, qsortDesc);

  3. Щоб отримати адрес цього методу, сначала отримуємо об'єкт MemoryHandle для статического методу Java:

    
// отримуємо тип методу
MethodType compareType = MethodType.methodType(int.class, MemorySegment.class, MemorySegment.class);

// отримуємо дескриптор статического методу (Program - текущий клас, где располагается метод)
MethodHandle compareHandle = MethodHandles.lookup().findStatic(Program.class, "compareInt", compareType);

    Метод MethodType.methodType приймає тип повертаємоого значення и далїї типи параметрів методу. В нашем случае метод сортировки compareInt повертає значення int и приймає два значення MemorySegment

  4. За допомогою методу upcallStub() линкера сгенерируем заглушку для функції C, которая затем вызывает дескриптор методу обратного вызова:

    
MemorySegment compareStub = linker.upcallStub(
    compareHandle,
    FunctionDescriptor.of(JAVA_INT,
        ADDRESS.withTargetLayout(JAVA_INT),
        ADDRESS.withTargetLayout(JAVA_INT)),
    Arena.ofAuto());

    Метод upcallStub() створює заглушку вызова, яку можна передать другим внешним функціям в виде указателя на функцію. Вызов такого указателя на функцію из внешнего коду приведет к выполнению предоставленного дескриптора методу:

    
MemorySegment upcallStub(MethodHandle target,
 FunctionDescriptor function,
 Arena arena,
 Linker.Option... options)

    Метод приймає до 4 параметрів:

    • target: дескриптор целевого методу

    • function: дескриптор функції-заглушки вызываемой функції

    • arena: арена, связанная с повертаємоым сегментом-заглушкой вызываемой функції

    • options: параметри линкера, связанные с цим запросом на компоновку

    Зверніть увагу на вызовы withTargetLayout(). Они настраивают структуру памяти, чтобы линкер мог сгенерировать код для настройки экземпляров MemorySegment, которые отримує обратный вызов Java. Если цього не сделать, вызовы a.get(JAVA_INT, 0) и b.get(JAVA_INT, 0) в Java-коллбэке завершатся ошибкой, поскольку сегменты памяти a и b будуть иметь нулевой размер.

  5. И в конце можна вызвать внешнюю функцію с адресом коллбэка:

    
float[] samples = . . .;
MemorySegment csamples = arena.allocateFrom(JAVA_FLOAT, samples);

qsort.invoke(csamples, samples.length, JAVA_FLOAT.byteSize(), compStub);

При выполнении програми на вход функції qsort передаємо масив {3, 5, 1, 8, 0, 5, 2} и після виконання мы увидим консольный вивід отсортированного масива:

[0, 1, 2, 3, 5, 5, 8]
