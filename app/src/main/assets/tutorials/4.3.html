Створення своїх виняткових класів

Останнє оновлення: 06.10.2025

Хотя имеющиеся в стандартной библиотеке класов Java класы винятків описують большинство исключительных ситуаций, которые можуть возникнуть при выполнении програми, все-таки иногда требуется створити свои собственные класы винятків со своей логикой.

Щоб створити свой клас винятків, треба унаследовать его от класа Exception. Наприклад, у нас есть клас, вычисляющий факториал, и нам треба выбрасывать специальное виняток, якщо число, передаваемое в метод, менше 1:


import java.util.Scanner;

class Program{
     
	public static void main(String[] args) throws Exception {
         
        createPerson("Tom", -10);
    }

    static void createPerson(String name, int age){

        try{

            var person = new Person(name, age);
            person.print();
        }
        catch(Exception ex){

            System.out.println(ex.getMessage());
        }
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age) throws Exception{

        if(name.length() < 2) throw new Exception("Недостаточная длина імені: " + name.length());
        if(age < 1 || age > 110) throw new Exception("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

В класе Person в конструкторе отримуємо ім'я и возраст пользователя и проверяем их на соответствие некоторым условиям. Зокрема, якщо длина імені меньше 2 символів або возраст менше 1 або больше 110, то выбрасываются винятки. В качестве типу генерируемого винятки застосовується базовый клас Exception, в конструктор которого передаєся сообщение об ошибке:


Person(String name, int age) throws Exception{

    if(name.length() < 2) throw new Exception("Недостаточная длина імені: " + name.length());
    if(age < 1 || age > 110) throw new Exception("Некорректный возраст: " + age);
    this.name = name; 
    this.age = age;
}

И поскольку конструктор сам не обробляє эти винятки, то в его оголошенийии за допомогою оператора throws указываем компилятору, что конструктор генерує винятки типу Exception.

В методі createPerson() собственно обробляємо винятки при вызове конструктора и створенні об'єкта Person:


try{
    var person = new Person(name, age);
    person.print();
}
catch(Exception ex){

    System.out.println(ex.getMessage());
}

Підобный підход вполне можна використовувати. Но иногда удобнїї використовувати свои класы винятків. Наприклад, в якой-то ситуации мы хочемо обпрацювати визначенийным чином лише те винятки, которые относятся к класу Person. Или выделить исключительные случаи и ошибки для класа Person в отдельные класы. Для цих целей мы можемо сделать специальный клас PersonException:


class PersonException extends Exception{

    PersonException(String message){
        
        super(message);
    }
}

По сути клас кроме пустого конструктора ничего не имїїт, и то в конструкторе мы просто обращаемся к конструктору базового класа Exception, передавая в него рядок с сообщением об ошибке. Но теперь мы можемо змінити клас Person, чтобы он выбрасывал виняток именно цього типу и соответственно в основной програме обробляти це виняток:


class Program{
     
	public static void main(String[] args) throws Exception {
         
        createPerson("Tom", -10);
        createPerson("m", 20);
    }

    static void createPerson(String name, int age){

        try{

            var person = new Person(name, age);
            person.print();
        }
        catch(PersonException ex){

            System.out.println(ex.getMessage());
        }
    }
}

class PersonException extends Exception{

    PersonException(String message){
        
        super(message);
    }
}

class Person{
    
    private String name;
    private int age;

    Person(String name, int age) throws PersonException{

        if(name.length() < 2) throw new PersonException("Недостаточная длина імені: " + name.length());
        if(age > 1 || age > 110) throw new PersonException("Некорректный возраст: " + age);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Консольный вивід програми:


Некорректный возраст: -10
Недостаточная длина імені: 1

Но мы можемо пойти дальше и створити для отдельных ситуаций свои типи винятків. Наприклад:


class Program{
     
	public static void main(String[] args) throws Exception {
         
        createPerson("Tom", -10);
        createPerson("m", 20);
    }

    static void createPerson(String name, int age){

        try{

            var person = new Person(name, age);
            person.print();
        }
        catch(PersonNameException ex){

            System.out.println("Ошибка в імені:");
            System.out.println(ex.getMessage());
        }
        catch(PersonAgeException ex){

            System.out.println("Ошибка в возрасте:");
            System.out.println(ex.getMessage());
        }
    }
}

class PersonNameException extends Exception{

    PersonNameException(String name, int minLength){
        
        String message = "Недостаточная длина імені. Текущїї значення: " + name + ". Минимальная длина імені: " + minLength; 
        super(message);
    }
}

class PersonAgeException extends Exception{

    PersonAgeException(int age, int minAge, int maxAge){
        
        String message = "Некорректный возраст. Текущїї значення: " + age + 
                        ". Минимальный возраст: " + minAge + ". Максимальный возраст: " + maxAge; 
        super(message);
    }
}

class Person{
    
    private String name;
    private int age;
    private static int nameMinLength = 2;
    private static int minAge = 1;
    private static int maxAge = 110;

    Person(String name, int age) throws PersonNameException, PersonAgeException {

        if(name.length() < nameMinLength) throw new PersonNameException(name, nameMinLength);
        if(age < minAge || age > maxAge) throw new PersonAgeException(age, minAge, maxAge);
        this.name = name; 
        this.age = age;
    }

    void print(){ System.out.printf("Name: %s; Age: %d\n", name, age); }
}

Здесь для ошибок імені визначений тип PersonNameException, а для ошибок при указании возраста - тип PersonAgeException. Оба типу наследуются от Exception и через конструктор отримують дані об ошибке.

Конструктор класа Person генерує винятки цих типів:


Person(String name, int age) throws PersonNameException, PersonAgeException {

А в методі createPerson() при створенні об'єктов Person обробляємо по отдельности оба цих вида винятків:


catch(PersonNameException ex){

    System.out.println("Ошибка в імені:");
    System.out.println(ex.getMessage());
}
catch(PersonAgeException ex){

    System.out.println("Ошибка в возрасте:");
    System.out.println(ex.getMessage());
}

В итоге мы получим следующий консольный вивід:


Ошибка в возрасте:
Некорректный возраст. Текущїї значення: -10. Минимальный возраст: 1. Максимальный возраст: 110
Ошибка в імені:
Недостаточная длина імені. Текущїї значення: m. Минимальная длина імені: 2
