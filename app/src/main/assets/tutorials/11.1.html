Модульность

Створення модуля

Останнє оновлення: 02.11.2025

С выходом JDK 9 в мові Java появилась новая возможнасть - модульность. Модульность дозволяє разбить код на отдельные структурные единицы - модули. Фактически модуль представляє группу пакетів або ресурсов, об'єднаних в одно целое и к которым можна обращаться по имени модуля. Поддержка модулей реализована через подсистему Java Platform Module System (система модулей платформы Java), которая є частью мови Java и виртуальной машины.

До Java 9 буво декілька уровней інкапсуляції функционала. Первый уровень представлял класс, в котором мы могли определить переменные и методы с различным рівнем доступа. Следующий уровень представлял пакет, який, в свою очередь, представлял коллекцию классов. Однак со временем цих уровней виявилося недостатньо. И модуль стал наступним рівнем інкапсуляції, який об'єднував декілька пакетів.

Определение модуля

Модуль соварто из группы пакетів. Також модуль включает список все пакетів, которые входят в модуль, и список всех модулей, от которых зависит данный модуль. Дополнительно (но необязательно) он может включать файлы ресурсов або файлы нативных библиотек.

Перед створенням модуля прежде всего треба определиться с его именем. В качестве имени модуля в принципе может використовуватися произвольный идентификатор из алфавитно-цифровых символов и знаков подчеркивания. Обычно, як и в случае с пакетами, имена модулей следуют условности "reverse domain name" (обратного доменного имени). И в целом рекомендуется (хотя це и необязательно), чтобы название модуля соответствовало названию, c которого начинаются пакеты цього модуля. Це соглашение предотвращает конфликты имен пакетів в модулях. Любой пакет может быть размещtн лише в одном модуле. Если имена модулей уникальны и начинаются с имени модуля, то имена пакетів також будуть уникальными.

Определим и используем простейший модуль. Допустим, файлы с исходными кодами помещаются в папку "C:\java" або "Home/java" (або якую-нибудь другую папку на жестком диске). Создадим в цей папке каталог, який назовем demo. Цет каталог буде представлять модуль. По соглашению, имя базового каталога совпадает с именем модуля.

В каталоге demo определим новый файл module-info.java со наступним кодом:


module demo {
}

Цет файл представляє дескриптор модуля (module descriptor). Цет файл может містятьь лише определение модуля.

С помощью ключевого слова module определяется модуль, який називається "demo", то есть так же, як и каталог, в котором данный файл расположен. После имени модуля за допомогою фигурных скобок можна определить тело модуля, но в данном случае код модуля не містить нияких инструкций.

Далїї в каталоге demo создадим папку com. В папке com создадим папку , а в папке com/hello.

В папке com//hello определим новый файл Hello.java:


package com..hello;
 
public class Hello{
     
    public static void main(String[] args){
        System.out.println("Hello !");
    }
}

Название пакета файла - com..hello отражает структуру папок, в которых расположен файл. Сам файл определяет класс Hello, який в методе main выводит на консоль строку.

В итоге у нас получится следующая стуктура проекта:

Теперь скомпилируем все це. Для цього вначале перейдем в командной строке/терминале к папке, в которой находится модуль demo.

Затем для компиляции модуля выполним следующую команду:


javac demo/module-info.java demo/com//hello/Hello.java

После компиляции модуля demo выполним программу за допомогою следующей команды:


java  --module-path demo --module demo/com..hello.Hello

Параметр --module-path указывает на путь к модулю, а --module - на главный класс модуля.

При наборе команды вместо параметра --module-path можна указать его сокращение -p, а вместо параметра --module - сокращение -m.

И на консоли отобразится сообщение "Hello !"


eugene@Eugene:~$ cd java
eugene@Eugene:/java$ javac demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java  --module-path demo --module demo/com..hello.Hello
Hello !
eugene@Eugene:/java$
