Модульность

Створення модуля

Останнє оновлення: 02.11.2025

С выходом JDK 9 в мові Java появилась новая возможнасть - модульность. Модульность дозволяє разбить код на отдельные структурные единицы - модулі. Фактически модуль представляє группу пакетів або ресурсов, об'єднаних в одно целое и к которым можна обращаться по імені модуля. Піддержка модулів реалізованийа через підсистему Java Platform Module System (система модулів платформы Java), которая є частью мови Java и виртуальной машины.

До Java 9 буво декілька уровней інкапсуляції функционала. Первый уровень представлял клас, в котором мы могли визначити змінні и методи с различным рівнем доступа. Следующий уровень представлял пакет, який, в свою очередь, представлял колекцію класов. Однак со временем цих уровней виявилося недостатньо. И модуль стал наступним рівнем інкапсуляції, який об'єднував декілька пакетів.

Визначенийие модуля

Модуль соварто из группы пакетів. Також модуль включает список все пакетів, которые входят в модуль, и список всех модулів, от которых зависит данный модуль. Дополнительно (но необязательно) он может включать файли ресурсов або файли нативных библиотек.

Перед створенням модуля прежде всего треба визначитися с его именем. В качестве імені модуля в принципе может використовуватися произвольный идентификатор из алфавитно-цифровых символів и знаков підчеркивания. Обычно, як и в случае с пакетуми, имена модулів следуют условности "reverse domain name" (обратного доменного імені). И в целом рекомендуется (хотя це и необязательно), чтобы название модуля соответствовало названию, c которого начинаются пакети цього модуля. Це соглашение предотвращает конфликты имен пакетів в модулях. Любой пакет может быть размещtн лише в одном модуле. Если имена модулів уникальны и начинаются с імені модуля, то имена пакетів також будуть уникальными.

Визначимо и використовуємо простейший модуль. Допустим, файли с исходными кодуми помещаются в папку "C:\java" або "Home/java" (або якую-нибудь другую папку на жестком диске). Створимо в цей папке каталог, який назовем demo. Цет каталог буде представлять модуль. По угоді, ім'я базового каталогу збігається с именем модуля.

В каталогу demo визначимо новий файл module-info.java со наступним кодом:


module demo {
}

Цет файл представляє дескриптор модуля (module descriptor). Цет файл может містятьь лише визначенийие модуля.

За допомогою ключевого слова module визначаєся модуль, який називається "demo", то есть так же, як и каталог, в котором данный файл расположен. Після імені модуля за допомогою фигурных скобок можна визначити тело модуля, но у даному випадку код модуля не містить нияких инструкций.

Далїї в каталогу demo створимо папку com. В папке com створимо папку , а в папке com/hello.

В папке com//hello визначимо новий файл Hello.java:


package com..hello;
 
public class Hello{
     
    public static void main(String[] args){
        System.out.println("Hello !");
    }
}

Название пакету файлу - com..hello отражает структуру папок, в которых расположен файл. Сам файл визначає клас Hello, який в методі main виводить на консоль рядок.

В итоге у нас получится следующая стуктура проєкта:

Теперь скомпилируем все це. Для цього вначале перейдем в командной рядківе/терминале к папке, в которой находится модуль demo.

Затем для компиляции модуля виконаємо следующую команду:


javac demo/module-info.java demo/com//hello/Hello.java

Після компиляции модуля demo виконаємо програму за допомогою следующей команди:


java  --module-path demo --module demo/com..hello.Hello

Параметр --module-path вказує на путь к модулю, а --module - на главный клас модуля.

При наборе команди вместо параметра --module-path можна указать его сокращение -p, а вместо параметра --module - сокращение -m.

И на консоли отобразится сообщение "Hello !"


eugene@Eugene:~$ cd java
eugene@Eugene:/java$ javac demo/module-info.java demo/com//hello/Hello.java
eugene@Eugene:/java$ java  --module-path demo --module demo/com..hello.Hello
Hello !
eugene@Eugene:/java$
