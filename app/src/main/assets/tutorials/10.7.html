Групування

Останнє оновлення: 30.04.2018

Щоб сгруппировать дані по якому-нибудь признаку, нам треба використовувати в связке метод collect() об'єкта Stream и метод Collectors.groupingBy(). Допустим, у нас есть следующий клас:


class Phone{
    
    private String name;
    private String company;
    private int price;
    
    public Phone(String name, String comp, int price){
        this.name=name;
        this.company=comp;
        this.price = price;
    }
    
    public String getName() { return name; }
    public int getPrice() { return price; }
    public String getCompany() { return company; }
}

И, к примеру, у нас есть набор об'єктов Phone, которые мы хочемо сгруппировать по компании:


import java.util.List;
import java.util.Map;
import java.util.stream.Stream;
import java.util.stream.Collectors;

public class Program {
 
    public static void main(String[] args) {
         
        Stream<Phone> phoneStream = Stream.of(new Phone("iPhone X", "Apple", 600), 
            new Phone("Pixel 2", "Google", 500),
            new Phone("iPhone 8", "Apple",450),
            new Phone("Galaxy S9", "Samsung", 440),
            new Phone("Galaxy S8", "Samsung", 340));
         
        Map<String, List<Phone>> phonesByCompany = phoneStream.collect(
                Collectors.groupingBy(Phone::getCompany));
         
        for(Map.Entry<String, List<Phone>> item : phonesByCompany.entrySet()){
             
            System.out.println(item.getKey());
            for(Phone phone : item.getValue()){
                 
                System.out.println(phone.getName());
            }
            System.out.println();
        } 
    } 
}

Консольный вивід:

Google
Pixel 2

Apple
iPhone X
iPhone 8

Samsung
Galaxy S9
Galaxy S8

Итак, для створенняя групп в метод phoneStream.collect() передаєся вызов функції Collectors.groupingBy(), которая за допомогою выражения Phone::getCompany группирует об'єкты по компании. В итоге буде создан об'єкт Map, в котором ключами є названия компаний, а значеннями - список связанных с компаниями телефонов.

Метод Collectors.partitioningBy

Метод Collectors.partitioningBy() имїїт похожїї действие, лише он делит елементы на группы по принципу, відповідає ли елемент визначенийному условию. Наприклад:


Map<Boolean, List<Phone>> phonesByCompany = phoneStream.collect(
                Collectors.partitioningBy(p->p.getCompany()=="Apple"));
        
for(Map.Entry<Boolean, List<Phone>> item : phonesByCompany.entrySet()){
            
    System.out.println(item.getKey());
    for(Phone phone : item.getValue()){
                
        System.out.println(phone.getName());
    }
    System.out.println();
}

У даному випадку за допомогою условия p->p.getCompany()=="Apple" мы смотрим, принадлежит ли телефон компании Apple. Если телефон принадлежит цей компании, то он попадает в одну группу, якщо нет, то в другую.

Метод Coollectors.counting

Метод Collectors.counting застосовується в Collectors.groupingBy() для вычисления количества елементов в каждой группе:


Map<String, Long> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, Collectors.counting()));
        
for(Map.Entry<String, Long> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue());
}

Консольный вивід:

Google -1
Apple - 2
Samsung - 2

Метод Collectors.summing

Метод Collectors.summing застосовується для підсчета суммы. В зависимости от типу данных, к которым застосовується метод, он имїїт следующие формы: summingInt(), summingLong(), summingDouble(). Застосуємо цей метод для підсчета стоимости всех смартфонов по компаниям:


Map<String, Integer> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, Collectors.summingInt(Phone::getPrice)));
        
for(Map.Entry<String, Integer> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue());
}

За допомогою выражения Collectors.summingInt(Phone::getPrice)) мы указываем, что для каждой компании буде вычислять совокупная цена всех її смартфонов. И поскольку вычисляется результат - сумма для значений типу int, то в качестве типу повертаємоой колекції використовується тип Map<String, Integer>

Консольный вивід:

Google - 500
Apple - 1050
Samsung - 780

Методи maxBy и minBy

Методи maxBy и minBy застосовуються для підсчета минимального и максимального значення в каждой группе. В качестве параметра эти методи приймають функцію компаратора, которая нужна для сравнения значений. Наприклад, найдем для каждой компании телефон с минимальной ценой:


Map<String, Optional<Phone>> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, 
                Collectors.minBy(Comparator.comparing(Phone::getPrice))));
        
for(Map.Entry<String, Optional<Phone>> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue().get().getName());
}

Консольный вивід:

Google - Pixel 2
Apple - iPhone 8
Samsung - Galaxy S8

В качестве повертаємоого значення операції группировки використовується об'єкт Map<String, Optional<Phone>>. Опять же поскольку группируем по компаниям, то ключом буде выступать рядок, а значенням - об'єкт Optional<Phone>.

Метод summarizing

Методи summarizingInt() / summarizingLong() / summarizingDouble() дозволяють об'єднати в набор значення соответствующих типів:


import java.util.IntSummaryStatistics;
//....................................

 Map<String, IntSummaryStatistics> priceSummary = phoneStream.collect(
    Collectors.groupingBy(Phone::getCompany,
        Collectors.summarizingInt(Phone::getPrice)));
        
for(Map.Entry<String, IntSummaryStatistics> item : priceSummary.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue().getAverage());
}

Метод Collectors.summarizingInt(Phone::getPrice)) створює набор, в який помещаются цены для всех телефонов каждой из групп. Данный набор инкапсулируется в об'єкте IntSummaryStatistics. Соответственно якщо бы мы применяли методи summarizingLong() або summarizingDouble(), то соответственно бы получали об'єкты LongSummaryStatistics або DoubleSummaryStatistics.

У цих об'єктов есть ряд методів, який дозволяють выполнить различные атомарные операції над набором:

  • getAverage(): повертає среднїї значення

  • getCount(): повертає кількість елементів в наборе

  • getMax(): повертає максимальное значення

  • getMin(): повертає минимальное значення

  • getSum(): повертає сумму елементов

  • accept(): додає в набор новый елемент

У даному випадку мы отримуємо среднюю цену смартфонов для каждой группы.

Консольный вивід:

Google - 500.0
Apple - 525.0
Samsung - 390.0

Метод mapping

Метод mapping дозволяє дополнительно обпрацювати дані и задать функцію отображения об'єктов из потіка на якой-нибудь другой тип данных. Наприклад:


Map<String, List<String>> phonesByCompany = phoneStream.collect(
    Collectors.groupingBy(Phone::getCompany,
        Collectors.mapping(Phone::getName, Collectors.toList())));
        
for(Map.Entry<String, List<String>> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey());
    for(String name : item.getValue()){
        System.out.println(name);
    }
}

Выражение Collectors.mapping(Phone::getName, Collectors.toList()) вказує, что в группу будуть выделятся названия смартфонов, причем группа буде представлять об'єкт List.
