Групування

Останнє оновлення: 30.04.2018

Щоб сгруппировать дані по якому-нибудь признаку, нам треба використовувати в связке метод collect() об'єкта Stream и метод Collectors.groupingBy(). Допустим, у нас есть следующий класс:


class Phone{
    
    private String name;
    private String company;
    private int price;
    
    public Phone(String name, String comp, int price){
        this.name=name;
        this.company=comp;
        this.price = price;
    }
    
    public String getName() { return name; }
    public int getPrice() { return price; }
    public String getCompany() { return company; }
}

И, к примеру, у нас есть набор об'єктов Phone, которые мы хочемо сгруппировать по компании:


import java.util.List;
import java.util.Map;
import java.util.stream.Stream;
import java.util.stream.Collectors;

public class Program {
 
    public static void main(String[] args) {
         
        Stream<Phone> phoneStream = Stream.of(new Phone("iPhone X", "Apple", 600), 
            new Phone("Pixel 2", "Google", 500),
            new Phone("iPhone 8", "Apple",450),
            new Phone("Galaxy S9", "Samsung", 440),
            new Phone("Galaxy S8", "Samsung", 340));
         
        Map<String, List<Phone>> phonesByCompany = phoneStream.collect(
                Collectors.groupingBy(Phone::getCompany));
         
        for(Map.Entry<String, List<Phone>> item : phonesByCompany.entrySet()){
             
            System.out.println(item.getKey());
            for(Phone phone : item.getValue()){
                 
                System.out.println(phone.getName());
            }
            System.out.println();
        } 
    } 
}

Консольный вывод:

Google
Pixel 2

Apple
iPhone X
iPhone 8

Samsung
Galaxy S9
Galaxy S8

Итак, для создания групп в метод phoneStream.collect() передаєся вызов функції Collectors.groupingBy(), которая за допомогою выражения Phone::getCompany группирует об'єкты по компании. В итоге буде создан об'єкт Map, в котором ключами є названия компаний, а значениями - список связанных с компаниями телефонов.

Метод Collectors.partitioningBy

Метод Collectors.partitioningBy() имїїт похожїї действие, лише он делит элементы на группы по принципу, соответствует ли элемент определенному условию. Например:


Map<Boolean, List<Phone>> phonesByCompany = phoneStream.collect(
                Collectors.partitioningBy(p->p.getCompany()=="Apple"));
        
for(Map.Entry<Boolean, List<Phone>> item : phonesByCompany.entrySet()){
            
    System.out.println(item.getKey());
    for(Phone phone : item.getValue()){
                
        System.out.println(phone.getName());
    }
    System.out.println();
}

В данном случае за допомогою условия p->p.getCompany()=="Apple" мы смотрим, принадлежит ли телефон компании Apple. Если телефон принадлежит цей компании, то он попадает в одну группу, якщо нет, то в другую.

Метод Coollectors.counting

Метод Collectors.counting застосовується в Collectors.groupingBy() для вычисления количества элементов в каждой группе:


Map<String, Long> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, Collectors.counting()));
        
for(Map.Entry<String, Long> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue());
}

Консольный вывод:

Google -1
Apple - 2
Samsung - 2

Метод Collectors.summing

Метод Collectors.summing застосовується для подсчета суммы. В зависимости от типу данных, к которым застосовується метод, он имїїт следующие формы: summingInt(), summingLong(), summingDouble(). Применим цей метод для подсчета стоимости всех смартфонов по компаниям:


Map<String, Integer> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, Collectors.summingInt(Phone::getPrice)));
        
for(Map.Entry<String, Integer> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue());
}

С помощью выражения Collectors.summingInt(Phone::getPrice)) мы указываем, что для каждой компании буде вычислять совокупная цена всех її смартфонов. И поскольку вычисляется результат - сумма для значений типу int, то в качестве типу возвращаемой коллекции використовується тип Map<String, Integer>

Консольный вывод:

Google - 500
Apple - 1050
Samsung - 780

Методи maxBy и minBy

Методи maxBy и minBy застосовуються для подсчета минимального и максимального значения в каждой группе. В качестве параметра эти методы принимают функцию компаратора, которая нужна для сравнения значений. Наприклад, найдем для каждой компании телефон с минимальной ценой:


Map<String, Optional<Phone>> phonesByCompany = phoneStream.collect(
        Collectors.groupingBy(Phone::getCompany, 
                Collectors.minBy(Comparator.comparing(Phone::getPrice))));
        
for(Map.Entry<String, Optional<Phone>> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue().get().getName());
}

Консольный вывод:

Google - Pixel 2
Apple - iPhone 8
Samsung - Galaxy S8

В качестве возвращаемого значения операции группировки використовується об'єкт Map<String, Optional<Phone>>. Опять же поскольку группируем по компаниям, то ключом буде выступать строка, а значением - об'єкт Optional<Phone>.

Метод summarizing

Методи summarizingInt() / summarizingLong() / summarizingDouble() дозволяють об'єднати в набор значения соответствующих типов:


import java.util.IntSummaryStatistics;
//....................................

 Map<String, IntSummaryStatistics> priceSummary = phoneStream.collect(
    Collectors.groupingBy(Phone::getCompany,
        Collectors.summarizingInt(Phone::getPrice)));
        
for(Map.Entry<String, IntSummaryStatistics> item : priceSummary.entrySet()){

    System.out.println(item.getKey() + " - " + item.getValue().getAverage());
}

Метод Collectors.summarizingInt(Phone::getPrice)) створює набор, в який помещаются цены для всех телефонов каждой из групп. Данный набор инкапсулируется в об'єкте IntSummaryStatistics. Соответственно якщо бы мы применяли методы summarizingLong() або summarizingDouble(), то соответственно бы получали об'єкты LongSummaryStatistics або DoubleSummaryStatistics.

У цих об'єктов есть ряд методів, який дозволяють выполнить различные атомарные операции над набором:

  • getAverage(): повертає среднїї значення

  • getCount(): повертає количество элементов в наборе

  • getMax(): повертає максимальное значение

  • getMin(): повертає минимальное значение

  • getSum(): повертає сумму элементов

  • accept(): додає в набор новый элемент

В данном случае мы получаем среднюю цену смартфонов для каждой группы.

Консольный вывод:

Google - 500.0
Apple - 525.0
Samsung - 390.0

Метод mapping

Метод mapping дозволяє дополнительно обработать дані и задать функцию отображения об'єктов из потока на якой-нибудь другой тип данных. Например:


Map<String, List<String>> phonesByCompany = phoneStream.collect(
    Collectors.groupingBy(Phone::getCompany,
        Collectors.mapping(Phone::getName, Collectors.toList())));
        
for(Map.Entry<String, List<String>> item : phonesByCompany.entrySet()){

    System.out.println(item.getKey());
    for(String name : item.getValue()){
        System.out.println(name);
    }
}

Выражение Collectors.mapping(Phone::getName, Collectors.toList()) указывает, что в группу будуть выделятся названия смартфонов, причем группа буде представлять об'єкт List.
