Sealed-класи та інтерфейси

Останнє оновлення: 27.09.2025

sealed-класи (или запечатанные классы) и sealed-інтерфейси (или запечатанные інтерфейсы) в мові Java управляют тем, якие именно классы можуть расширять данный класс або інтерфейс. Эта функция вносит уровень контроля и предсказуемости в иерархии наследования, який ранїї був недоступен, кроме як через менїї гибкие механизмы, такие як final або ограничения доступа.

Прежде всего встает вопрос: зачем же нужны sealed-класи? До появления sealed-классов в Java иерархия классов могла быть або полностью открытой для расширения (любой класс мог наследоваться), або быть полностью закрытым для наследования за допомогою модификатора final. Sealed-классы предлагают золотую середину. Они дозволяють контролировать расширяемость - явно указать, якие классы можуть быть прямыми подклассами, предотвращая неожиданное и несанкционированное наследование. Крім того, они предоставляют эффективный способ моделирования ADT (Алгебраических типов данных) в Java, где базовый тип имїїт фиксированный набор вариантов.

Щоб оголосити класс або інтерфейс як sealed, використовуються два ключевых слова: sealed и permits:


sealed class имя_класса permits подкласс_1, подкласс_2, ююю подкласс_N { 

    // содержимое класса
}

Ключевое слово sealed застосовується к суперклассу або інтерфейсу для оголошення его запечатанным.

После имени класса указывается ключевое слово permits, за которым йде список классов, которым разрешено быть прямыми наследниками. Далїї будем також называть эти классы разрешенными классами.

Приклад:


sealed class Operation
    permits Sum, Subtract, Multiply
{ }

В даному випадку ми дозволяємо класам Sum, Subtract та Multiply бути спадкоємцями класу Operation. Однак для всіх інших класів встановлено запроет на спадкування. Наприклад, у разі ми зіштовхнемося з помилкою на етапі компіляції:


sealed class Operation
    permits Sum, Subtract, Multiply
{ }
class Divide extends Operation   // Ошибка - для класса Divide не разрешено наследование
{ }

Обмеження для дозволених підкласів

Классы, перечисленные после слова permits, також повинні следовать определенным правилам. Прежде всего повинні соблюдаться правила расположения классов:

  • Дозволені підкласи sealed-класу мають бути доступні. Підни не можуть бути приватними класами, вкладеними в інший клас або класами, які доступні лише на рівні іншого пакета.

  • Разрешенные публичные подклассы повинні находиться в том же пакеті, что и sealed-класс. Если застосовуються модули, то разрешенные подклассы повинні находиться в том же модуле, что и sealed-класс.

  • sealed-класс может быть оголошений без оператора permits. Тогда все его прямые подклассы повинні быть оголошенийы в одном файле.

Крім того, дозволені класи повинні явно оголошувати, як вони продовжують (або припиняють) ієрархію запечатаного класу. Для цього до дозволеного класу застосовується один із трьох модифікаторів:

  • final: запрещает любое дальнейшїї наследование, завершая ветвь иерархии:

    
sealed class Operation
    permits Sum, Subtract, Multiply { }

final class Sum extends Operation { }
final class Subtract extends Operation {}
final class Multiply extends Operation {}

  • sealed: разрешает ограниченное дальнейшїї наследование, но лише тем классам, которые идут после permits:

    
sealed class Operation permits UnaryOp, BinaryOp{}

final class UnaryOp extends Operation { }

// разрешаем дальнейшїї наследование, но лише для классов Sum, Subtract, Multiply
sealed class BinaryOp extends Operation
    permits Sum, Subtract, Multiply { }

final class Sum extends BinaryOp {}
final class Subtract extends BinaryOp {}
final class Multiply extends BinaryOp {}

  • non-sealed: открывает цей подкласс для расширения любыми классами, як обычный класс Java, продолжая ветвь иерархии

    
sealed class Operation permits UnaryOp, BinaryOp{}

final class UnaryOp extends Operation { }

// открываем для наследования произвольным классами
non-sealed class BinaryOp extends Operation {}

class Sum extends BinaryOp {}
class Subtract extends BinaryOp {}
class Multiply extends BinaryOp {}

Приклад sealed-класів

Розглянемо найпростіший приклад із sealed-класами:


class Program{
      
    public static void main(String[] args) {
          
        Operation add = new Sum(5, 4);
        Operation sub = new Subtract(5, 4);

        System.out.println(add.execute());      // 9
        System.out.println(sub.execute());      // 1
    }
}

sealed abstract class Operation
    permits Sum, Subtract
{
    int op1;
    int op2;

    Operation(int op1, int op2){

        this.op1 = op1;
        this.op2 = op2;
    }
    abstract int execute();
}

final class Sum extends Operation{

    Sum(int op1, int op2){ super(op1, op2); }

    @Override int execute() { return op1 + op2;}
}
final class Subtract extends Operation{

    Subtract(int op1, int op2){ super(op1, op2); }

    @Override int execute() { return op1 - op2;}
}

Здесь абстрактный класс Operation определен як запечатанный sealed-класс. Он представляє арифметическую операцию, которая хранит два операнда в виде переменных op1 и op2 и определяет абстрактный метод execute() для виконання цей операции. И цей класс можуть наследовать лише классы Sum и Subtract.

В классах Sum и Subtract соответствующим чином реализуем метод execute() для виконання соответствующей операции. Причем варто відзначити, что наследование от цих классов запрщено.

Розширений приклад

Розглянемо більш розширений приклад:


class Program{
      
    public static void main(String[] args) {
          
        Operation add = new Sum(5, 4);
        Operation sub = new Subtract(5, 4);
        Operation neg = new Negation(5);
        Operation if1 = new IfOperation(true, 3, 7);
        Operation if2 = new IfOperation(false, 3, 7);

        System.out.println(add.execute());      // 9
        System.out.println(sub.execute());      // 1
        System.out.println(neg.execute());      // -5
        System.out.println(if1.execute());      // 3
        System.out.println(if2.execute());      // 7
    }
}

sealed abstract class Operation permits BinaryOperation, IfOperation{

    abstract int execute();
}

// запрещаем дальнейшїї наследование
// операция отрицания
final class IfOperation extends Operation { 

    boolean condition;
    int op1;  
    int op2; 
    IfOperation(boolean cond, int op1, int op2){ 
        
        this.condition = cond;
        this.op1 = op1; 
        this.op2 = op2; 
    }

    @Override int execute() { return condition ? op1 : op2; }
}

// разрешаем дальнейшїї наследование для типов Sum, Subtract
sealed abstract class BinaryOperation extends Operation
    permits Sum, Subtract
{
    int op1;
    int op2;

    BinaryOperation(int op1, int op2){

        this.op1 = op1;
        this.op2 = op2;
    }
}

final class Sum extends BinaryOperation{

    Sum(int op1, int op2){ super(op1, op2); }

    @Override int execute() { return op1 + op2;}
}

non-sealed class Subtract extends BinaryOperation{

    Subtract(int op1, int op2){ super(op1, op2); }

    @Override int execute() { return op1 - op2;}
}

class Negation extends Subtract{

    Negation(int op){ super(0, op); }   // эквивалентно варажению 0 - op
}

Тут у нас є єдиний sealed-клас, який представляє операцію – Operation. Від нього можуть успадковуватись класи BinaryOperation (операція з двома операндами) та IfOperation (операція з трьома операндами)

Класс IfOperation определен як final и запрещает дальнейшїї наследование, он реализует своего рода тернарную операцию.

А вот класс BinaryOperation дозволяє унаследоваться двум классам - Sum и Subtract. Причем Subtract определен с модификатором non-sealed, что дозволяє наследоваться от него произвольным классам. Как наприклад, классу Negation, який реализует операцию унарного минуса.

Sealed-інтерфейси

Концепция sealed-типов також применима к інтерфейсам. Sealed-інтерфейс может быть реализован або расширен лише теми классами або інтерфейсами, которые указаны в его предложении >permits. Например:


public sealed interface Shape permits Rectangle, Circle {
    // Методи інтерфейса
}
